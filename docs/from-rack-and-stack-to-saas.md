# 从机架和堆栈到 SaaS

> 原文：<https://thenewstack.io/from-rack-and-stack-to-saas/>

欢迎来到由 [Scalyr](https://www.scalyr.com/) 首席执行官 [Christine Heckart](https://www.linkedin.com/in/christineheckart/) 主持的[新堆栈制造商](/podcasts/makers):攀登新高度系列访谈，涵盖了工程经理在扩展架构以支持业务需求时所面临的挑战。

当企业工程师谈论构建软件即服务(SaaS)时，机架和堆栈内存经常出现。当人们回忆起其他人所做的工作时，就有了尊重。

[攀登新高度 Ep # 6–从机架和堆栈到 SaaS](https://thenewstack.simplecast.com/episodes/scaling-new-heights-ep-6-from-rack-and-stack-to-saas)

大约在 2011 年至 2012 年，当初创公司涌入企业领域时，这种东西经常会丢失。社交技术很时髦。云很迷人。API，webhooks，RSS 向社交技术的演变——就像这种突然的兴奋，令人陶醉的服务热潮——松散耦合的技术改变了世界！

[force point](https://www.linkedin.com/in/nicolasfischbach/)的首席技术官 Nicolas Fischbach 提供了一个经常从科技公司的企业经理那里听到的观点:传统技术将会继续存在，但建设一个新的 SaaS 会令人兴奋。

在不考虑现有技术的情况下，如何开发一种基于新方法形成模型的方法在 Forcepoint 上无法完成。出现了什么？一种安全平台，能够根据人们使用的数据适应实时行为模式，同时保留其作为数据防丢失公司(DLP)和云安全公司的历史根基。

开发人员、工程师和运营人员率先发起了 DevOps 运动，以支持一种依赖于不同应用程序开发、部署和管理方法的新思维。DevOps 团队必须更快速地迭代构建应用程序。Forcepoint 有一个推论。管理层意识到新的 SaaS 集团与众不同。他们是产品和工程部门的一群人，被认为是“奇怪的团队”

那些“古怪”的团队通常是为重大技术项目设定路线的人。他们就像各地的 DevOps 团体。在 Forcepoint，他们需要奇怪的团队和多年来在构建 DLP 技术方面想了很多的人。Forcepoint 需要一种新的思维方式，这种思维方式需要借助公司员工的集体精神。

Forcepoint 有一个云安全产品，但他们希望通过其新的 SaaS 平台将其提升到一个新的水平。他们面临着赫克特所说的“死亡之谷”，即除了完成项目之外别无出路的地方。

最终，Forcepoint 团队找到了公司在应对现有企业产品时所面临的妥协，为“l”ever aging的一些 新的 东西、 或 利用 的一些的 旧的 帮助 到 过渡遗留 DLP 到 实际上 风险 自适应 DLP 和 帮助 客户端、 你

T 帽子 漂亮 酷。 我 的意思是， 但是 耶， 它 是的艰难旅程
