# Coders 的作者 Clive Thompson 谈编程是如何变化的

> 原文：<https://thenewstack.io/coders-author-clive-thompson-on-how-programming-is-changing/>

9 月 28 日，来自 83 个国家的近 5000 名在线与会者收听了 Stack Overflow 网站的[首届“流动状态”会议](https://stackoverflow.blog/2022/10/07/missed-our-flow-state-conference-catch-up-on-all-the-sessions/)(来自纽约的网络直播)。

其中一个亮点是 Stack Overflow 的首席营销官哈立德·艾尔·卡提布采访了[克莱夫·汤普森](https://www.clivethompson.net/)，这位热爱科技的记者为他 2020 年的书*Coders:The Making a New Tribe and The Remaking The World*采访了数千名程序员。

在它结束之前，汤普森对编程本身的未来做出了一些有趣的预测，推测人工智能辅助编码会如何影响代码的质量。他还讨论了远程(和混合)工作场所、行业对表示问题的反应，甚至一个庞大的“代码邻近”兼职开发人员阶层。

但最重要的是，汤普森对他在与世界各地的程序员交谈后所学到的东西做了一个有趣而生动的总结…

## 新兴工具集

展望未来，Thompson 看到了一个“代码相关人员的海洋——仅次于全职开发人员……”并使用“新兴工具集”来自动化其业务逻辑的世界。(Thompson 举了一个例子，应收账款中有人使用 Zapier 或 Google 电子表格或“某种低代码/无代码工具”。你瞧，你创造了某种基本上是应用程序的东西，迎合了你遇到的任何奇怪的问题。”)

这种现象一直存在——汤普森承认，它是上世纪 90 年代微软的 Visual Basic 所针对的主流商业用户的受众。但是“我认为我们正在看到这种大规模的东西爆炸…几乎是编码人员数量的 10 倍，做这种代码相关的东西。

“无论好坏，”汤普森补充道。虽然它允许个人定制他们的解决方案，而无需引入昂贵的全职开发人员，“但缺点是:其中一些将会非常不稳定，非常怪异…

“有人向我展示了这些长达 10，000 行的 Microsoft Visual Basic 脚本，这些脚本是一些疯狂的全局变量，它们相互碰撞在一起。然而，它已经运营他们的公司——你知道，他们的七人公司——10 年了。而且随时都会分崩离析。”

汤普森预测会有更多…

## 艾的影响

汤普森最终不可避免地被问到，人工智能和机器学习是否会改变编程职业。汤普森承认，在对 Copilot 进行试验后，他得出结论，它“在让你快速开发方面表现得令人震惊。这给人一种怪异的感觉，‘哇。这是人工智能接管开发任务的最后时刻吗？"

但在与开发人员交谈后，他得出了一个更微妙的结论。“我认为这不会让任何人完全失业，但它会改变发展的本质。”几个经验丰富的开发人员告诉他“我可以快 2 倍，因为它真正做的是花费我去堆栈溢出和查找如何做某事的时间，因为它*知道*…”

但更重要的是，Thompson 认为，像 Copilot 这样的工具也将“超级大国”更大的代码邻近兼职程序员军团。

“也许这并没有雇佣或雇佣不足那种你可能会在 Fiverr 上花 500 美元为你构建逻辑的程序员。但很可能它也将为那些*不是全职开发人员的人打开新的功能，从某种程度上摆脱一些繁重的工作，让他们也能发挥更大的作用。”*

## 全球集中化

人工智能更大的问题是它有可能*集中*新兴技术释放的所有力量。“现在，当我在全球范围内与人们交谈时，在全球南部和西方以外的国家，人们有很多担忧，许多人工智能开发都在几个热点地区。美国部分地区有，多伦多和加拿大有一点，英国有，德国有，日本有，中国有很多。”

就目前的发展而言，人工智能需要巨大的(也是昂贵的)能量和计算能力的爆发——汤普森认为，如果人工智能的发展方式没有一些范式转变，这种情况不会改变。这让全球各地担心的地区担心他们会落后，他们会被降级为消费这些其他国家制造的人工智能 API。这是一个非常无能为力的情况。

“也许七八年前，他们感到有点兴奋——tensor flow 是开源的，他们喜欢，‘哦，我们这里可以卷起并训练一个人工智能，就像有人在旧金山或波士顿或伦敦做的一样好。’现在他们感觉更像是，‘太好了。我可以订阅在北京或旧金山开发的真正高端的人工智能，但这对我们的代码存量来说并不太好……就像，我们将坐在一起做即时工程，而不是真正的工程。'

“这是一个有趣的挑战，我真的不知道有什么办法可以摆脱它。”

## 什么影响了代表性？

汤普森个人对编程的热情是显而易见的。在某一点上，他描述了年轻时的发现，“你可以用这些咒语将数字内容带入生活，”他说这有助于他理解激励程序员的激情。但这也让汤普森明白了为什么科技界一直在努力争取代表权。

“部分原因是，在这些大学预科水平中，没有足够的卷须说,“嘿，你想试试这个东西吗，这个告诉机器做你想让它做的事情的奇怪任务，并弄清楚你是否是这些获得电击刺激的孩子之一？因为不是每个人都这样。"

汤普森认为，他们分布在所有不同的人口统计数据中——估计他们占任何给定群体的 5%——但你必须给他们机会，让他们知道他们是感受到那种电击刺激的人之一因此，他赞成新的编程课程和针对年轻学生的课外项目，因为这让人们认识到他们的热情。

但是有趣的是不同的国家经历了不同的人口模式。“如果你去印度，那是一个拥有大量开发者的庞大国家，从人口统计学上看，他们比该国其他地方的人年轻得多，而且他们对移动非常感兴趣。如果你去中国，或者斯里兰卡，甚至亚洲的一些地方，你会看到更多的女性进入编码领域。这在很大程度上是出于一系列文化原因——你知道，他们的父母和周围的人正在以一种在美国可能不太常见的方式鼓励他们。

“所以实际上，在许多方面，我认为每个国家都可以通过观察其他国家擅长和不擅长做什么来学习一些东西。”

最终，他更担心的是学院和大学实际上难以跟上利息。“我有点担心，我们将面临另一种时刻，他们开始设置障碍，赶走人们。这不是我编造的。研究过这个问题的学者对此感到担忧。”

## 远程工作与混合工作

Thompson 还谈到了远程工作获得了什么，也失去了什么。程序员喜欢在“流动状态”下获得长时间不受干扰的时间——Thompson 称这是经理可以让他们的工程师保持快乐和高效的第一件事。但是这对大公司的新程序员有什么影响呢？当他们三个月不能跟在高级工程师后面“肩并肩冲浪”时，他们失去了什么？“有很多处于发展中期的人不希望回到办公室，也有很多处于高端和入门级的人，他们迫切地希望 T2 回到办公室。各家公司都在努力寻找如何打破这个怪圈。”

那么解决办法是什么呢？汤普森首先承认，“坦白地说，我不知道是否有人做得很好……我觉得和我交谈的每个人都觉得他们仍然只是在拼凑。”但是那些看起来心满意足的人已经确定了一个系统，在这个系统中，人们只能在一周的*部分*呆在办公室，“这样他们就可以进行面对面的同步，当你面对面时，同步会更快。剩下的时间，每个人都可以去他们想去的地方。”Thompson 甚至在疫情之前就听说过这种模式——有趣的是，这种模式在中小型公司中效果最好。“我认为真正的大公司很难应对——对他们来说，这感觉太分散了，你知道吗？”

接近尾声时，一名观众问远程工作是否会最终降低代码质量——仅仅因为最好的 QC 是其他人使用我们的代码的反应。"我们可能更有生产力，但是我们会更快地创造出可怕的东西来使用吗？"汤普森分享了一个相关的经验——两对眼睛实时地看着同一个问题，似乎是最有效的交流方式。这在未来可能会消失——谁也说不准。但汤普森也指出，人工智能辅助编程也存在同样的问题。

“如果这让我的工作速度提高了两倍，那是不是意味着我现在会产生更多蹩脚的代码，或者更多松散的代码——因为我可以做得更快？”
