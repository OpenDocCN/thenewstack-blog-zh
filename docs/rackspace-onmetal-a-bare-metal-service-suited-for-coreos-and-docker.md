# Rackspace OnMetal，一款适合 CoreOS 和 Docker 的裸机云服务

> 原文：<https://thenewstack.io/rackspace-onmetal-a-bare-metal-service-suited-for-coreos-and-docker/>

云服务市场打开了一扇窗。这是一扇不会永远打开的窗户，但是现在开源社区正在建立一个技术网络，它将在未来一段时间内影响市场。

最新的例子来自 Rackspace，它今天宣布 OnMetal 云服务器进入有限可用性，并作为一个开源项目。

OnMetal 云服务让人想起新堆栈的真正含义，因为新技术为更快的应用开发和自动化基础设施打开了窗口。它不是一种“仅限于云”的服务，而是一种可以在云和托管环境中运行的服务。

OnMetal 云服务器被设计成可编程的 API 驱动的服务。例如，它可以帮助正在扩大规模的公司，Rackspace 的企业战略总监保罗·奎尔纳在 Structure Conference 的一次采访中说。如果一家公司每六个月增加一倍的计算能力，而他们只需要 200 个金属实例，这比部署大得多的虚拟机要好得多。像这样发展可以简化管理。成长的时候一切都要更好的运转。OnMetal 云服务器有助于减少差异。这些差异包括对共享服务的竞争，这可能会影响延迟和原始性能。在虚拟化环境中，随着虚拟机数量的增加，必须优化工具和其他技术。例如，监控通常会占用大量工程时间，需要人才来提高竞争优势。但是，在没有虚拟机管理程序技术的情况下运行意味着磁盘和网络性能可以得到提升。

正是这种将云服务和主机服务相结合的能力，作为一种在复杂快速的基础设施上管理应用程序的方式，正在不断发展。OnMetal Server 运行在单租户环境中。顾客按分钟付费。单租户服务可以连接到数据中心的其他服务，这些服务在本质上可能更加分散。这可能意味着客户使用混合服务，如对象存储或虚拟化服务器，通过 Rackspace OnMetal 进行测试和开发。

OnMetal Cloud 正在使用 OpenStack 和 OpenCompute 技术。这些服务器都是运行在万兆网络上的固态硬盘。它们是为高性能和低延迟的数据库而构建的。它有很多 I/O，这使得它可以用于 Cassandra 等数据分析处理技术或 Postgres 或 MongoDB 等 NoSQL 数据库。

单租户技术可以在 CoreOS 上运行，core OS 编排 Docker 容器。容器因素很重要。如果使用虚拟机管理程序技术来实现这一点，将需要成千上万的虚拟机。相反，像 Pantheon 这样的公司可以使用 Linux 容器作为一个巨大的统一平台。它通过在专用基础设施上使用容器的多租户平台来提高效率。

CoreOS 是一个基于 Linux 的操作系统，它编排 Docker 容器。操作系统会自动更新，无需客户手动更新。一旦 OnMetal 上线，客户将能够在 Rackspace 的 OnMetal 云上提供实例并运行 CoreOS。这种组合使客户能够利用自动更新来旋转容器，以及在裸机上运行所带来的马力。

这种创新开始真正推动云服务市场。它是开源的，是为快速开发和性能而设计的。这是下一代新堆栈基础架构中的强大组合。

但这不是唯一的方法。来自亚马逊网络服务的大量证据表明，运行虚拟机的共享环境可以非常高效和强大，足以运行像网飞这样的高性能服务。Xen 虚拟机管理程序有开销，但经过多年的发展，它已经能够以前所未有的满意度运行数万个工作负载。
