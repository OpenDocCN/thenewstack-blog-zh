# 新的堆栈环境:内存计算与云计算相遇

> 原文：<https://thenewstack.io/the-new-stack-context-in-memory-computing-meets-cloud-native-computing/>

欢迎来到[新的堆栈环境](/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了 [Hazelcast](https://hazelcast.com/) 的高级解决方案架构师[迈克·延恩](https://www.linkedin.com/in/michaelyawn/)，他谈到了内存计算对微服务和云原生工作负载进行增压的潜力。

新的 Stack 编辑和营销总监 [Libby Clark](/author/libby/) 主持这一集，旁边是 TNS 高级编辑 [Richard MacManus](/author/richard/) 和 TNS 执行主编 [Joab Jackson](/author/joab/) 。

[第 131 集:内存计算遇上云端原生计算](https://thenewstack.simplecast.com/episodes/episode-131-in-memory-computing-meets-cloud-native-computing)

哈欠最近[给 TNS 发了一篇帖子](/future-proof-high-performance-microservice-applications-with-in-memory-technology/)，解释内存技术如何让微服务运行得更流畅。Hazelcast 提供内存数据网格 [Hazelcast IMDG](https://hazelcast.com/products/imdg/) ，以及流处理软件 [Hazelcast Jet](https://hazelcast.com/products/jet/) 。我们想了解更多关于内存如何与微服务一起使用的信息。虽然内存中提供了缓存，就像 Redis 等键值数据库一样，但它也提供了额外的计算能力，这有助于动态处理数据，哈欠解释道。

“内存数据网格的标准定义是，您可以将一系列不同系统的 RAM 汇集在一起，这样您就可以将多个节点聚集在一起。他说:“如果你有四个 8GB 的系统，那么你可以有效地拥有一个 32GB 的系统，你可以根据你的内存使用情况增加到几十或几百个节点。

“这不仅仅是共享内存的问题。您可以使用这些系统的附加处理器和不同接口卡的附加网络带宽。因此，它真的可以让你通过添加许多廉价的系统来扩展，而不是扩展到非常昂贵的超级计算机，”哈欠说。

他还谈到 Kubernetes 在该公司用户中的部署越来越多，这些用户往往是高端企业用户，如银行。

“我们支持 Kubernetes，因为我们的用户告诉我们这对他们很重要，”哈欠告诉我们。
