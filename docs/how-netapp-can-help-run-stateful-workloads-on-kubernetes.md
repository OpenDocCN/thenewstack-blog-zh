# NetApp 如何帮助在 Kubernetes 上运行有状态工作负载

> 原文：<https://thenewstack.io/how-netapp-can-help-run-stateful-workloads-on-kubernetes/>

[VMware](https://www.vmware.com/company.html) 赞助了这次播客。

在本周的 [The New Stack Context](https://thenewstack.io/podcasts/context) 播客中，我们与 NetApp 云基础架构业务部门的业务总监[戴尔·德根](https://www.linkedin.com/in/daledegen/)在本周于拉斯韦加斯举行的[年度用户大会](/netapp-turns-to-developers-with-low-hassle-cloud-native-storage-management/)、 [NetApp Insight 2019](https://insight.netapp.com/) 上讨论了 Kubernetes 和云原生存储。

自从去年[收购 StackPointCloud](https://thenewstack.io/netapp-goes-cloud-native-with-stackpointcloud-acquisition/) 以来，存储巨头 [NetApp](https://www.netapp.com/us/index.aspx) 已经成为 Kubernetes 管理领域的领导者。 [NetApp Kubernetes 服务](https://cloud.netapp.com/kubernetes-service)为运行任何商业甚至本土的 Kubernetes 发行版提供了一个控制平面。用户只需三次点击就可以部署一个完整的 Kubernetes 发行版。当与 NetApp on tap 存储操作系统和云卷结合使用时，用户可以设置多云 Kubernetes 部署。

我们与 Degen 讨论了 NetApp 如何支持 Kubernetes，以及建立覆盖整个企业的数据结构的重要性。我们还聊到了一款全新的 NetApp 对象存储设备[，它由高速闪存驱动器](https://blocksandfiles.com/2019/09/16/netapp-all-flash-storagegrid-array/)构建而成，据 Degen 称，这在存储行业尚属首次。

[NetApp 如何帮助在 Kubernetes 上运行有状态工作负载](https://thenewstack.simplecast.com/episodes/how-netapp-can-help-run-stateful-workloads-on-kubernetes)

然后在稍后的播客中，我们聊到了 Linux 基金会的[开源峰会+嵌入式 Linux 会议欧洲 2019](https://events19.linuxfoundation.org/events/open-source-summit-europe-2019/) ，TNS 创始人兼主编 [Alex Williams](https://twitter.com/alexwilliams) 出席了该会议。我们还讨论了来自 TNS 网站的一些头条新闻，包括 Pivotal 最近的[春季平台会议](/pivotal-wants-to-let-developers-forget-about-kubernetes/)，在生产中运行 Kubernetes 的一些提示，以及[无服务器和云计算的未来](/this-week-in-programming-whither-serverless/)。

TNS 的编辑和营销总监 Libby Clark 和 Williams 以及 TNS 的执行主编 Joab Jackson 一起主持了这个播客。

Linux 基金会、NetApp 和 Pivotal 是新体系的赞助商。
