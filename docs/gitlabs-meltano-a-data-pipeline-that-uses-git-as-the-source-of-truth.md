# GitLab 的 Meltano，一个使用 Git 作为真理来源的数据管道

> 原文：<https://thenewstack.io/gitlabs-meltano-a-data-pipeline-that-uses-git-as-the-source-of-truth/>

GitLab 赞助了这个播客。

数据工程落后于软件工程世界 10 年，并且在许多方面仍然深陷“电子表格的土地”，[专注于数据操作的创业公司](https://www.linkedin.com/in/daniellemorrill)[梅尔塔诺](https://meltano.com/)的总经理丹妮尔·莫里尔说。作为弥补缺乏可行的数据工程流程的一种方式，Meltano 可以满足组织的需求，而无需软件开发人员的内部专业知识，他们可以从头开始发明量身定制的解决方案。“很多人擅长电子表格，不会写代码或者认为他们不会写代码，但从概念上讲，他们已经是了，”Morrill 说。“所以，我认为(梅尔塔诺的)路线图在很大程度上就是要把他们带过来。”

在 9 月份在纽约布鲁克林举行的 [GitLab Commit](https://about.gitlab.com/events/commit/) 会议期间录制的这一集 New Stack Makers 播客中，Morrill 描述了 Meltano 如何将 wide net 作为数据建模、分析和管理的开源替代方案。

“类似于 GitLab 是开发者体验的管道工具，我们是数据分析师和工程师体验的管道工具，”Morrill 说。“我们将一流的开源工具结合在一起，这样您就可以从数据一直到仪表板，而不必挑选工具并单独设置它们。”

Morrill 说，GitOps 的“关键时刻”是 Git，而 Meltano 的关键时刻是“运行转换后得到的数据集”。“我们肯定认为人们应该使用版本控制和数据工程，但这实际上还没有发生，”Morrill 说。

然而，Morrill 说，Git 也应该用于数据管理。“我们希望你使用 Git，”Morrill 说。“这是我们认为事情应该发展的方向。”

Morrill 说，她作为 Mattermark 联合创始人的经历帮助她在 2013 年了解到建立数据管理解决方案有多困难后，为 Meltano 奠定了基础。“我们必须推出自己的产品，”莫里尔说。“当麦克摩尔在 2013 年起步时，数据工程的技术还远未发展起来。所以，我当时目睹了很多痛苦，我认为我们有机会简化这种痛苦。”

通常，一个组织可能需要先进的数据管理仪表板和工程能力，“但我们不能支付 100，000 美元来获得一个昂贵工具或许多次几个昂贵工具的许可证，”Morrill 说。“所以，我们真的在采取一种自下而上的方法来接近中小企业和小型科技公司，说‘看，你拥有所有这些属于你的数据，你不需要花一大笔钱来真正了解你拥有什么，’”Morrill 说。“所以这是我们看到最多的情况。”

Morrill 说，Meltano 1.0 将“很快”推出，“我们可以开始告诉人们可以在这个基础上建立东西，我们将支持它。”Morrill 说:“我们需要让它成为这样一种工具，即使你不是工程师，它仍然是一种友好的工具，可以让你通过这些工具进入工程领域。”
