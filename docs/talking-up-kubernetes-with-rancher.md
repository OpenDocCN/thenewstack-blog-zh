# 和牧场主谈论 Kubernetes

> 原文：<https://thenewstack.io/talking-up-kubernetes-with-rancher/>

[与牧场主谈论库伯内斯](https://thenewstack.simplecast.com/episodes/talking-up-kubernetes-with-rancher)

20 多年来，香农·威廉姆斯(Shannon Williams)在发布软件方面一直扮演着亲力亲为的角色。最近，在观察到集装箱化的力量和潜力后，威廉姆斯在 2014 年共同创立了[牧场主实验室](https://rancher.com/)，此前他成功创建了[Cloud.com](https://www.cloud.com/)(现由思杰系统公司所有)。

从 2014 年开始，事情发生了变化。当然，Kubernetes 已经成为标准的集装箱化平台，促使 Rancher 在最近发布的 Rancher 同名产品 Rancher 2.0 中使用“所有 Kubernetes”。

在本期[新堆栈制造商播客](https://thenewstack.io/podcasts/makers/)中，Williams 回顾了他对 Kubernetes 的看法，以及 Rancher 为何以及如何转向该平台。

“到 2017 年初，我们看到 [Kubernetes](https://kubernetes.io/) 是运行集装箱化的最成熟、最适合生产的选择，”威廉姆斯说。

当时，已经非常清楚的是，Kubernetes 在不同的可用编排平台中拥有迄今为止最丰富的功能集，并且“比我们自己的编排功能更加全面，”Williams 说。“Kubernetes 让整个行业致力于改善它，”威廉姆斯说。“我们当时决定用 Rancher 2.0 把一切都押在 Kubernetes 上。”

Rancher 2.0 的一个经常被引用的功能是它的用户界面(UI)，当使用基于角色的访问控制( [RBAC](https://en.wikipedia.org/wiki/Role-based_access_control) )的平台进行集中认证、访问控制、监控和健康检查时。active directory 集中管理 GKE、EKS 和阿拉斯加的 Kubernetes 集群。由于牧场主 Kubernetes 引擎(RKE)部署 Kubernetes 角色的方式，Kubernetes 也可以在 vSphere、generic EC2 或 Azure 上运行。

威廉姆斯还描述了为什么 Kubernetes 吸引了如此强烈的兴趣，这样做，可能回答了许多问题，人们不敢问。在寻求采用 Kubernetes 时，Williams 谈到了组织必须如何决定如何迁移工作负载，典型的部署成本是多少，当然，还有与该平台相关的真正好处是什么。他还描述了云上的典型部署以及本地和持久存储选项。

尽管 Kubernetes 在行业中获得了巨大的关注，但只有一小部分企业将 Kubernetes 集群集成到了他们的 IT 基础架构中。但是通常只需要组织中的一个团队开始测试或部署 Kubernetes，该平台就会在企业中迅速普及。威廉姆斯将这种现象描述为他所谓的库伯内特“传染”

“我有时会想到像麻疹或水痘这样的容器和 Kubernetes 它们具有高度传染性，”威廉姆斯说。“一旦一个团队开始使用它，你会很快发现另一个团队很快开始说‘嘿，这很酷，我们希望能够做到这一点。"
