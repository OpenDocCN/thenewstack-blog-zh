# 面向当今开发者传道者和社区管理者的事件营销

> 原文：<https://thenewstack.io/event-marketing-todays-developer-evangelists-community-managers/>

随着技术行业中社区经理的角色演变为现在所知的开发者福音，许多经理发现自己正在转向更加以开发者为中心的营销策略。与开发人员和工程师在他们的水平上联系，允许联系基于真正的兴趣，而不是试图推销产品或服务。

在新一集的播客 [The New Stack Makers](/tag/the-new-stack-makers/) 中，我们采访了开源分布式跟踪项目 [LightStep](http://lightstep.com/) 的营销和合作伙伴负责人 [Priyanka Sharma](https://twitter.com/pritianka) ，以及数据库容器管理平台提供商 [ClusterHQ](https://clusterhq.com/) 的营销副总裁 [Michael Ferranti](https://twitter.com/ferrantim) ，讨论当今的营销和社区专业人士如何在当今的开源和云原生生态圈中建立社区。这段对话发生在最近于本月在西雅图举行的 KubeCon 2016 会议上。

[面向当今开发者传道者和社区管理者的事件营销](https://thenewstack.simplecast.com/episodes/event-marketing-for-todays-developer-evangelists-and-community-managers)

从以开发者为中心的角度来看待营销和社区管理可能会令人生畏，尽管 Ferranti 建议记住开发者也是人。

“我要说的是，当你在开发一个产品时，与传统营销中的其他产品相比，最大的区别之一就是它不是一个数字游戏。而在开发者营销中，这是一种更加个性化的方法。您希望与每个开发人员建立联系。这是在开发人员领域特别成功的地方，”Sharma 表示同意。

Sharma 指出，参加活动的每个团队成员都应该对该产品有足够的了解，以便能够作为思想领袖就该产品进行 10 分钟的详细交谈，并强调在整个活动中放大自己声音的重要性。

“我们考虑为这次活动配备人员和进行计划的方式是，如果这里的其他思想领袖也对寻人感兴趣，我们可以放大这一信息。这是关于让人们知道他们在你的社区里。夏尔马说:“做好充分准备，通过 Twitter 扩大你的信息，并获得盟友，真的很有用。

Ferranti 指出，随着越来越多的技术会议变得对家庭友好，即使是像打印儿童尺寸的优质 t 恤这样简单的事情，也能让公司受到关注。

随着谈话的结束，Sharma 和 Ferranti 谈到了在 KubeCon 结束后 Lightstep 和 ClusterHQ 是如何填补空白的，以及他们从参加过去的活动中学到了什么。“如今真正困难的事情之一是填写您的工具需要使用的工具矩阵，以便提供完整的解决方案，”Ferranti 说，Sharma 补充说，“参加这样的会议真的让我大开眼界。根据你听到的问题，你知道什么文件更重要，然后你改进它。我认为这次会议对我们来说很重要，让我们知道虽然现在不是每个人都在做分布式跟踪，但这并不意味着他们不想听案例研究。现在我们知道这是一个重要的优先事项。”
