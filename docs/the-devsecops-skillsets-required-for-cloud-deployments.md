# 云部署所需的 DevSecOps 技能

> 原文：<https://thenewstack.io/the-devsecops-skillsets-required-for-cloud-deployments/>

一个组织如何确保它的 DevOps 团队有必要的技能来完成一个项目？

在最新一集的 New Stack Makers 播客[中，](https://www.linkedin.com/in/ashley-ward-jp/?originalSubdomain=uk)[帕洛阿尔托网络](https://www.paloaltonetworks.com/cloud-security?utm_content=inline-mention)首席技术官办公室技术总监 Ashley Ward 讨论了云部署所需的 DevSecOps 技能组合。TNS 创始人兼发行人 [Alex Williams](/author/alex/) 主持了这个播客。

[迎接面向云部署的 DevSecOps 技能挑战](https://thenewstack.simplecast.com/episodes/meet-the-devsecops-skillset-challenge-for-cloud-deployments)

在其[“2021 年勒索软件威胁报告](https://unit42.paloaltonetworks.com/ransomware-threat-report-highlights/)”中，Palo Alto Networks 的研究机构 Unit 42 指出，当组织迁移到云环境时，他们往往缺乏必要的技能。沃德说:“我看到人们正在做的是，他们伸出手来——他们通常与合作伙伴交谈，他们与人们交谈，以获得其他公司正在做什么以及人们如何应对技能差距的信息。”。

从哪里开始？沃德说:“无论是什么组织，你都要着眼于能够带来回报的事情——不一定是立竿见影的效果——而是持续的改进。“所以，如果是我，因为我的背景是集装箱和集装箱安全，我会说从集装箱图像开始，这是一个简单的东西，我们可以从货架上拿下来，我们可以立即开始展示好处，我们可以立即向图像的消费者展示事情正在改善。”

通常，当 Docker 文件用于创建容器图像时，组成图像的层可以被重用或构建在共享的现有层之上。然而，这项功能也带来了一些警告。“你可以借用其他人的层，你可以在他们和所有那些伟大的东西的基础上进行构建，所以这使得开发变得非常非常快。不幸的是，这确实意味着你在借用东西……而你不一定知道那些层里面是什么，”沃德说。

这就是为什么安全扫描图像如此重要。它用于揭示容器的配置，确定每个容器中到底在运行什么，其潜在的漏洞和依赖性——包括开源依赖性、相关的许可协议和关于容器的其他重要信息。“这听起来很多，事实上……因为这是我们正在做的一件事，我们可以把它分解成所有这些部分，我们可以把它变成最佳实践，”沃德说。
