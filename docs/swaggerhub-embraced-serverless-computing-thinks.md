# 观看 SmartBear 用 SwaggerHub 驯服 AWS Lambda

> 原文：<https://thenewstack.io/swaggerhub-embraced-serverless-computing-thinks/>

当软件开发人员继续考虑使用[无服务器计算](/category/serverless/)时，这项技术也在继续发展。随着尘埃落定，现在有更多的教育和机会让企业和初创公司在其堆栈中采用无服务器技术。为了帮助简化使用无服务器技术的学习曲线，如 [AWS Lambda](https://aws.amazon.com/lambda/) ，软件提供商 [SmartBear Software](https://www.smartbear.com/) 最近为其流行的 API 框架 [Swagger](https://swagger.io/) 引入了一个集成平台，被称为 [SwaggerHub](https://swaggerhub.com/) 。

在这一集的[下面嵌入的新堆栈制造商](https://thenewstack.io/podcasts/)中，我们简要介绍了 SwaggerHub 如何与 AWS Lambda 集成，记录自 [Gluecon 2016](http://gluecon.com/) ，Smart Bear Software 负责 Swagger [产品的副总裁 Tony Tam](https://www.linkedin.com/in/tonytam) 。我们也有一些关于 SwaggerHub 功能的深入视频演示，以及 SwaggerHub 如何与 AWS Lambda 一起工作。

[SwaggerHub 已经接受了无服务器计算，并认为你也应该这样做](https://thenewstack.simplecast.com/episodes/swaggerhub-has-embraced-serverless-computing-and-thinks-you-should-too)

“我们用 Lambda 做的是建立一个桥梁，这样当你在 SwaggerHub 中保存一个定义时，它可以推送到 Amazon API 网关，并将配置每一个路由指向一个现有的 Lamba 函数，或者它会为你创建一个。当它这样做时，它会对代码进行注释，并说，‘这是你可以预期的变量，’”Tam 说。

Tam 解释说，SwaggerHub 所做的是将 API 生命周期管理的概念重新整合到无服务器计算中，因此开发人员的任务可以在一个步骤中完成。对于那些与 [AWS Lambda](https://aws.amazon.com/documentation/lambda/) 一起工作来映射 API 定义的人来说，他们可能会发现这个任务很乏味。SwaggerHub 旨在消除其中的许多痛点，将 Swagger 定义转换为与 Amazon 兼容，连接和工具化 API，甚至创建新的 Lambda 函数(如果它们不存在的话)。

“工程师就是这样，在我看来，好的工程师最擅长的就是解决问题。优秀的工程师不喜欢坐在那里点击按钮来连接东西。它也在一定程度上消除了人为错误。当你整天在某个用户界面上点击时，不管它有多好，你都会把事情搞砸，”Tam 说。

Tam 解释说，无服务器计算为许多开发人员提供了一种新的工作方式。“对于人们来说，这里将有一些学习曲线，不是通过使用集成，而是通过思考‘当我不需要做所有这些我通常需要做的事情时，这意味着什么，以及我如何在这个叫做 Lamba 的环境中工作？非常孤立。它不是一个一直在运行的服务器。它确实促进了更好的程序设计，所以很多人需要时间来适应。"
