# PagerDuty CTO:有多少“运营”应该与“开发”结合在一起？

> 原文：<https://thenewstack.io/pagerduty-cto-much-re-imagined-ops-combine-dev/>

[page duty CTO:一个重新设想的‘运营’应该和‘开发’结合多少](https://thenewstack.simplecast.com/episodes/pagerduty-cto-how-much-should-a-re-imagined-ops-combine-with-dev)

对于一个组织来说，接受" [DevOps 文化](/category/devops/)"的信息或理想是一回事但是，当所有的拥抱和握手都在进行时，流程和工作流的变化会延伸到组织结构图吗？几周前，在旧金山举行的 2017 年 page duty 峰会开始前，我们向 page duty CTO 兼联合创始人 Alex Solomon 提出了这个问题。

“有多种方法可以做到这一点，但需要确保他们在同一间屋子里一起工作，”所罗门回应道，“他们的运营人员与开发人员在一起，[*和*]开发人员自己不仅可以编写代码，还可以测试代码，将其交付生产，并在生产中实际拥有它。”

所罗门进一步详细阐述了这种“所有权”的实际含义。目前，在一些组织中，它指的是在出现问题或失败时指定责任。毫无疑问，他认为所有权包括在失败情况下的反应。Solomon 将如何在组织中扩展所有权的概念，使得开发人员不仅仅是负责任的、负责任的、代码相关情况下运营经理的替身？
