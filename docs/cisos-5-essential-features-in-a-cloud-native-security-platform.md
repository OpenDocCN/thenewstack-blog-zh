# CISOs:云原生安全平台的 5 个基本特性

> 原文：<https://thenewstack.io/cisos-5-essential-features-in-a-cloud-native-security-platform/>

[Twistlock](https://www.paloaltonetworks.com/prisma/cloud) 赞助本帖。

您决定更新您的安全工具和策略，以应对现代云原生威胁。现在，您需要弄清楚在安全平台中应该寻找什么。

这可能是一个很难解决的难题。随着越来越多的供应商推出为容器和云定制的安全工具，可供选择的下一代安全工具解决方案也越来越多。但是有些比其他的更适合实现整体的云原生安全策略。

让我们来看看云原生安全平台应该为您的团队提供的主要功能。

## 什么是云原生？

首先，简要说明一下云原生的含义。由[云本地计算基金会](https://www.cncf.io/)制定的[云本地](https://github.com/cncf/toc/blob/master/DEFINITION.md)定义是官方版本。

但对于本文来说，当我们说云原生时，我们不仅仅指运行在云中的工作负载，我们还想到了首先为以云为中心的基础架构设计的技术。

云原生环境包括容器和无服务器功能等技术。这两种技术都不需要在云中运行(是的，如果你愿意，你[可以做无服务器的内部部署](https://www.channelfutures.com/channel-futures/prem-or-cloud-highlighting-top-serverless-options))。但它们往往是主要或完全基于云的基础设施的重要组成部分。

清楚了这一点之后，让我们来探讨一些保护云原生工作负载所必需的特定特性，无论它们采取何种形式。

## 宽度

也许当今许多以云为中心的安全工具之间最大的区别在于它们的广度。

一些工具被设计成只支持特定类型的基础设施(容器往往是最常见的)。然而，其他平台采取了更广泛、更全面的方法，不仅针对容器，而且针对云原生基础架构的所有组件(从无服务器功能到普通虚拟机)提供安全监控和威胁补救。

即使你今天只使用一种类型的基础设施技术，你也不知道明天你的需求是什么。这就是为什么提供广泛覆盖不同技术的云原生安全平台将提供最大的价值。

## 基于白名单的访问控制

有两种方法可以控制访问和保护资源。

一种是黑名单，默认情况下允许所有人进入，同时阻止某些人进入。

另一个是[白名单](https://www.networkworld.com/article/3179349/security/how-to-achieve-security-via-whitelisting-with-docker-containers.html)。在这种方法下，默认情况下您会阻止所有人，只允许您认为合法且值得信任的人访问。

黑名单方法是传统的方法。它来自一个网络配置和基础设施都是静态的、威胁是可预测的时代。

然而，对于当今的动态云原生环境，白名单提供了更高程度的安全性。与其假设您在所有威胁来袭之前就知道它们在哪里，并且知道应该将谁列入黑名单，不如尽可能地限制访问。

## 经得起未来考验的

如果在云原生时代有一件事情已经变得很清楚，那就是没有人知道哪些技术将在未来以有意义的方式出现。

2013 年，当 Docker 首次亮相时，容器是部署应用程序的最新和最伟大的方式。然后，随着 2014 年 AWS Lambda 的推出，无服务器开始成为主流。今天，有人[说单核](https://techbeacon.com/containers-20-why-unikernels-will-rock-cloud)是未来的潮流。与此同时，云供应商正在相互竞争，以创建更新更简单的方法来管理容器(这就是为什么我们看到了像 Fargate 这样的解决方案在 AWS 上的首次亮相)。

这意味着您不知道将来需要保护哪种类型的环境。因此，最好选择一个在设计时考虑到未来需求的安全平台，并且该平台有不断发展以适应新型技术的历史。仍然主要集中在容器上的工具不会让您为未来做好准备。

## 自动化

我们整天听到自动化有多重要。但事实仍然是，当涉及到安全性时，自动化仍然不是所有情况下的规则。由于解释和响应安全威胁的复杂性，一些团队仍然采取主要是手动的安全方法。

然而现实是，手动安全性在云原生时代不起作用。环境变化太快，配置变化太快，您的工程师无法手动解释安全威胁并及时做出反应。因此，您需要能够为您做出基于数据的明智威胁决策的工具，然后在威胁造成损害之前采取行动阻止它们。

## 服从

保持云原生环境的安全还不够。您还需要能够证明它们是安全的，并展示出哪里出了问题，以及在发生违规时您如何应对。这是让审计员满意并避免监管罚款的唯一方法。

因此，您需要一个集成了报告和[合规性](https://www.twistlock.com/managing-compliance-container-environments/)的云原生安全平台。否则，遵约必须是一个单独的过程，这不仅增加了复杂性，而且容易忽视与遵约相关的重要义务。

## 结论

选择正确的安全解决方案是任何成功的云原生部署的绝对要求。虽然许多供应商可能声称他们可以提供您需要的东西，但请只考虑那些能够提供未来验证、白名单、合规性和自动化的供应商。当然，找到正确的解决方案并不容易，但是一旦到位，您将能够控制保护您的基础架构和数据免受云原生的最令人生畏的潜在威胁。
