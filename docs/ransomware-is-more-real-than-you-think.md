# 勒索软件比你想象的更真实

> 原文：<https://thenewstack.io/ransomware-is-more-real-than-you-think/>

你有一只想要爱护和保护的泰迪熊。一个哥哥或姐姐拿走了泰迪熊，并威胁要拿它来勒索赎金，直到你付清为止。你是做什么的？

泰迪熊的类比当然过于简单，但它也反映了组织日益面临的勒索软件攻击的现实。攻击者封锁对关键数据的访问，以换取越来越离谱的赎金。根据 Palo Alto Networks 的 Unit 42 报告，2020 年的最高赎金为 3000 万美元，高于 2019 年的 1500 万美元。

在最新一集的 New Stack Makers 播客中，我们采访了帕洛阿尔托网络公司 [Prisma Cloud](https://www.paloaltonetworks.com/cloud-security?utm_content=inline-mention) 的产品营销经理[贾森·威廉姆斯](https://www.linkedin.com/in/iamjasonwilliams/)，讨论组织应该如何保护自己免受勒索软件攻击。TNS 的创始人兼发行人亚历克斯·威廉姆斯(Alex Williams)主持了这一集。

[勒索软件比你想象的更真实](https://thenewstack.simplecast.com/episodes/ransomware-is-more-real-than-you-think)

勒索软件的主要变化不是攻击本身的性质，而是攻击者变得更加厚颜无耻，要求——在许多情况下，收到——比过去更多的钱。贾森·威廉姆斯说，攻击的生命周期或攻击的风格并没有真正随着时间的推移而演变，发生变化的是策划攻击的人的大胆程度。

在当今日益高度分散和集装箱化的环境中，以及远程工作人员相互连接的情况下，攻击者有更多的机会进入网络而不被发现。他们可能会耐心等待几周或几个月，然后想出如何策划一次横向攻击，以获取关键数据，加密数据，然后要求赎金来解密数据。

> “您需要对网络进行分段，因为不是每个应用程序或每个工作负载都需要相互通信，只是因为它可以这样做。”

—贾森·威廉姆斯，帕洛阿尔托网络公司

“当你进入云时，有一个网络提供所有这些不同应用程序、所有虚拟机和容器以及无服务器功能等之间的连接，”贾森·威廉姆斯说。“这些事情需要一个网络来沟通和支持业务，勒索软件攻击者利用了(这些机会)。”

然而，在勒索软件攻击者能够访问“网络允许他们访问的”关键资产之前，该组织有可能通过分割关键数据来限制访问，贾森·威廉姆斯说。“您需要对网络进行分段，因为不是每个应用程序或每个工作负载都需要相互通信，只是因为它可以，”Williams 说。

美国政府在 6 月发布了备忘录[“我们敦促你做些什么来防范勒索软件的威胁”](https://s3.documentcloud.org/documents/20796934/memo-what-we-urge-you-to-do-to-protect-against-the-threat-of-ransomware.pdf)，以传达勒索软件威胁的严重性，同时呼吁私营和公共部门的组织采取更严格的措施。备忘录还概述了组织保护自己的具体方法。这些步骤包括改进备份、下载安全补丁、实施网络分段和其他保护措施。但是，尽管所有这些步骤都很重要，但补丁管理和隔离关键系统“远离不太关键的系统尤其重要”，Williams 说。

“这是我个人觉得有趣的几个最佳实践，当白宫真正开始强调它们时，因为在帕洛阿尔托网络公司，我们也认为这是你应该做的，作为网络安全打击勒索软件的最佳实践，”贾森·威廉姆斯说。
