# Grsecurity 厂商起诉开源先锋 Bruce Perens 在 GPLv2 上的分歧

> 原文：<https://thenewstack.io/open-source-pioneer-bruce-perens-sued-grsecurity/>

开源的指路明灯之一，[开源倡议](https://opensource.org/)的联合创始人[布鲁斯·佩伦斯](https://perens.com/)，因为对 GNU GPLv2 许可证的分歧，被开源安全公司起诉，该公司是为 Linux 内核开发 [Grsecurity](https://grsecurity.net/) 补丁管理软件的公司。

根据提交给美国加州北区地方法院旧金山分院的诉状，开源安全公司声称，Perens [在一篇博客文章中做出了“滥用和虚假”的声明](https://regmedia.co.uk/2017/08/03/grc_lawsuit.pdf)，导致“对 Grsecurity 的声誉、商誉和未来业务前景造成了重大损害”。

Perens 自己的律师 Heather Meeker 认为诽谤诉讼是“对自由软件社区在公共利益问题上自由交流思想的攻击。”开源安全没有回应置评请求。

Grsecurity 为最近发现的 Linux 内核漏洞提供了补丁。Perens 认为该公司禁止用户重新分发补丁，尽管该公司自己否认这种情况。尽管如此，Perens 在 6 月 28 日发布的[博客文章认为，由于这一禁令，Grsecurity 违反了开源 GPLv2 许可，因此，该公司的客户也将承担连带侵权责任。](https://perens.com/blog/2017/06/28/warning-grsecurity-potential-contributory-infringement-risk-for-customers/)

“目前，Grsecurity 是一种商业产品，仅面向付费客户。在他们的稳定补丁访问协议下，客户被警告如果他们重新发布 Grsecurity 补丁，这是他们在 GPL 下的权利，他们将被评估一个惩罚:他们将不再被允许成为客户，并且将不会被授权访问任何 Grsecurity 的进一步版本。GPL 第 2 版第 6 节明确禁止添加诸如这种禁止再分发的条款，”Perens [在他的帖子中争辩道。](https://perens.com/blog/2017/06/28/warning-grsecurity-potential-contributory-infringement-risk-for-customers/)

Grsecurity 坚称其软件和商业行为都没有违反 GPL2 许可。

但是，虽然表面上看，这起诉讼似乎与 GNU GPL v2 有关，但事实并非如此。这是一起诽谤案，佩伦的律师在电子邮件采访中说。

“实际上，这场诉讼与 GPL 无关。它是关于表达对许可证合规性的意见的权利——这是一个对自由软件开发者非常重要的法律话题。Perens 先生只是表达他的观点，并不打算也不需要攻击任何人。他的动机只是帮助社区和公众理解自由软件的好处以及如何最好地支持它，”米克说。"开源安全公司正通过其商业模式和这起诉讼制造自己的问题."

开源社区中的其他人也表达了类似的观点。

“我认为原告起诉某人评论他们可能不遵守 GPL 的立场显然是一种沉默策略。自由软件的长期倡导者 Deb Nicholson 说:“很难不把这看作是压制记者的一种令人担忧的趋势的一部分，这些记者竟敢报道不是每个人——特别是主题——都喜欢听的事情。

这并不是开源安全/Grsecurity 与 Linux 开源社区的第一次冲突。在 Linux 内核邮件列表 Linux creator 上，Linus Torvalds 写道:

*不要担心 grsecurity。*
*他们的方法一直是“我们不在乎我们是否打破了什么，我们只是声称这是因为我们特别安全”。*
