# 软件定义的网络和安全方法

> 原文：<https://thenewstack.io/software-defined-approach-networking-security/>

在本期[新堆栈分析师](/tag/the-new-stack-analysts/)中，我们将了解 Nuage Networks 如何通过使用软件定义网络(SDN)来解决容器安全问题。

为了准备我们最新的电子书: *[联网、安全和存储与 Docker 和 Containers](https://thenewstack.io/ebookseries/)* ，New Stack 创始人 Alex Williams 和 TNS 电子书编辑 Benjamin Ball 采访了[Nuage Networks](http://www.nuagenetworks.net/) 产品管理和营销负责人 [Hari Krishnan](https://www.linkedin.com/in/hkrish) 和 Nuage Networks 产品线管理总监 [Harmeet Sahni](https://www.linkedin.com/in/harmeetsahni) 。

随着容器的兴起，围绕如何最好地安全管理基于容器的基础设施的讨论显著增加。Sahni 解释说，目前还没有编排平台或容器的“标准”，Nuage Networks 对其客户如何使用其平台采取了不可知的方法。“我们支持客户做出的选择。我们并不是说 X 比 Y 好，”萨尼说。

基于容器的环境允许开发人员部署云原生应用程序，并具有快速扩展的能力。然而，如果没有合适的工具，支持基础设施很快就会成为一个棘手的问题。“这就是 SDN 的用武之地。人们将基础设施视为代码，而 SDN 的正确做法确实如此。你把你的网络当成代码。当您部署这些动态应用程序时，您的 SDN 会随之跟进。你可以很快改变事情，因为它已经是可编程的了。Sahni 解释说:“应用程序团队不再被网络团队使用传统方法所做的事情所束缚。

Nuage Networks 最初是作为在虚拟机上创建云网络的平台而取得成果的，现在已经发展到支持裸机设置，然后是容器。“一些早期的方法最终为容器创建了筒仓。我们说，‘你知道，当你开始部署应用程序时，这将成为一个问题。’”Sahni 提到。从运营的角度来看，Nuage Networks 希望在问题开始之前就解决问题，提供强大的日志记录、审计和策略管理作为其平台的一部分。Krishnan 解释说，容器天生比虚拟机共享更多的信息，这影响了策略管理的处理方式，“它们需要有所不同，”Krishnan 说。

容器安全性无疑将继续成为未来许多讨论的主题，但 Nuage Networks 希望采用 SDN 将有助于开发人员在使用容器时更好地创建和管理策略。“挑战仍然是，你如何确保开发商实施正确的政策？我们正在经历的那种交换是我们把完全控制权交给开发者，让他们完全掌控一切吗？相比之下，您的运营团队根据预定义的策略进行创建，然后应用程序开发人员在部署应用程序时必须从这些模板中进行选择。”
