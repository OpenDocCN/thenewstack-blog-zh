# 尼科尔·福斯格伦博士谈 DevOps:“你衡量什么，你就是什么”

> 原文：<https://thenewstack.io/dr-nicole-forsgren-on-devops-you-are-what-you-measure/>

CloudBees 赞助了这个播客。

[妮可·福斯格伦博士在 DevOps 上说:“你衡量什么，你就是什么”](https://thenewstack.simplecast.com/episodes/dr-nicole-forsgren-on-devops-you-are-what-you-measure)

[Nicole Forsgren 博士](http://nicolefv.com/)是 [DevOps 研究与评估](https://devops-research.com/) (DORA)的首席执行官兼首席科学家，也是 DevOps 年度报告的主要作者。在许多场合，她都坚持“你衡量什么，你就是什么”——一家公司的能力可能会受到它能够感知它如何做它所做的事情的程度的影响或限制。我们在这里讨论了如何衡量新的云原生技术的成功。

“我想我会同意这一点，当我们改变我们的技术堆栈和应用程序以实现云原生时，”首席执行官回应道，“这可能会改变我们的工作方式，无论是通过直接改变，因为我们已经改变了我们的工作，还是通过我们现在用来跟踪和评估工作的测量方法。

“不过，我想我会有一点不同意，”她继续说道，“因为迁移到原生云必然会让人们失业。举例来说，您可以迁移到原生云，但仍然保留本地云。在这种情况下，你没有必要把人从工作岗位上撤下来；他们现在正在扮演不同的工作角色。当我们这样做的时候，这里有很多机会和承诺。这并不一定意味着每个人都要失业，每个人都要被解雇；另一方面，我们将雇佣所有这些新人。”

正如 IT 运营专业人士所说，业务经理和高管往往缺乏远见，无法意识到他们工作线的基本流程——容量规划、质量保证、合规性、事件响应以及福斯格伦博士最喜欢的测量——不会像蓝色肥皂泡一样在旧世界的基础设施被云平台取代的幸运场景中消失。他们将云视为托管服务，具体来说，是由他人管理的服务。
