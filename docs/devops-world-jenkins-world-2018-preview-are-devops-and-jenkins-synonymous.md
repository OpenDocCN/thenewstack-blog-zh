# DevOps World | Jenkins World 2018 预告:DevOps 和 Jenkins 是同义词吗？

> 原文：<https://thenewstack.io/devops-world-jenkins-world-2018-preview-are-devops-and-jenkins-synonymous/>

[DevOps World+Jenkins World 2018 预告:devo PS 和 Jenkins 是同义词吗？](https://thenewstack.simplecast.com/episodes/devops-world-jenkins-world-2018-preview-are-devops-and-jenkins-synonymous)

詹金斯经常被称为 DevOps 的[引擎，近年来通过插件的帮助，成为持续交付管道](https://www.infoworld.com/article/3046038/application-development/why-jenkins-is-becoming-the-engine-of-devops.html)的[枢纽。它被](https://jenkins.io/doc/book/pipeline/)[解释为一个自动化工具](https://www.zdnet.com/article/the-dawn-of-automation-and-the-promise-of-a-formula-for-business-success/)，并且是一个组织的 DevOps 策略所基于的主要软件组件。

然而，在其核心，詹金斯本质上是一个自动化服务器。IT 专业人员将其广泛用于数据中心自动化和基础设施管理的报道严重不足。事实上，从许多数据中心经理的角度来看，Jenkins 是他们发起的一场运动，软件开发人员对它的接受是因为他们的坚持。

所以 Jenkins 应该是这两个 IT 部门的共同点。一年一度的詹金斯世界大会扩展到 DevOps 世界，这是否表明这些团体可能最终在同一个会议厅里达成一致？

“Jenkins 不再只是一个通用的自动化服务器或 CI 服务器，”CloudBees 的 DevOps 传播者 Brian Dawson 评论道，CloudBees 是商业 Jenkins 以及最近围绕 Jenkins 构建的名为 [CloudBees Core](https://www.cloudbees.com/products/cloudbees-core) 的云原生平台的生产商。“相反，Jenkins 已经成为支持组织和团队实施 DevOps 的重要组成部分。

"现在，我们认识到詹金斯并不是德沃普斯的一切，道森继续说道. "DevOps 还有很多其他方面没有单独作为一个工具编入 Jenkins。因此，通过将展览从 Jenkins World 扩展到 DevOps World，我们有空间引入演讲者和内容，不仅将 Jenkins 作为一个 DevOps 解决方案，还将 DevOps 转型的整个过程超越 Jenkins。"

在过去的四年时间里，詹金斯已经成为了一种 DevOps 生态系统的中心。围绕 Jenkins 构建的平台和流程适应或实现了 DevOps 技术的版本，这些版本在某种程度上适用于各种组织。这些工具一直以“[端到端 DevOps](https://www.kaiburr.com/blog/continuous-integration-simplified) ”的形式出现——这确实假设 DevOps 是线性的，有两端。

事实是，正如今年旧金山会议上的一些专题讲座所表明的那样，该流程中的其他利益相关者，如安全和营销，最终在不同的维度上延伸了 DevOps，使其不像一些附加平台制造商最初想象的那样线性和一维。

现在请听布莱恩·道森和他的 CloudBees 同事 r·泰勒·克罗伊和本·威廉姆斯在[的新堆栈制造商](https://thenewstack.io/podcasts/makers)播客上预告[devo PS World/Jenkins World 2018](https://www.cloudbees.com/devops-world)(今年 9 月 16 日至 18 日在三藩市举行)。
