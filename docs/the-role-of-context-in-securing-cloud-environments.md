# 上下文在保护云环境中的作用

> 原文：<https://thenewstack.io/the-role-of-context-in-securing-cloud-environments/>

由于云环境不断变化的特性，安全团队浪费大量时间手动匹配来自许多安全产品的高容量、低风险警报数据。[警报疲劳](https://orca.security/resources/blog/how-alert-fatigue-almost-flooded-our-basement/)以及这些警报中缺乏优先级和可操作信息可能会导致团队错过关键问题。

这个问题正在升级，特别是因为单点解决方案策略无法端到端地保护现代云环境。传统的单点安全工具旨在解决独立、孤立的架构问题。在云原生时代之前，它们一直工作得很好，云原生时代更加扩展、灵活，并且充满了无数的配置选项。

需要基于上下文的保护来检测云中的漏洞、错误配置和[恶意软件](https://orca.security/resources/blog/cloud-malware-challenges-best-practices/)，并确定其优先级。为了加强其云原生环境的云安全状况，组织必须投资于基于上下文的智能。

## **什么是与云安全相关的环境？**

那么“[上下文](https://orca.security/resources/blog/not-all-risks-are-equal-why-context-matters-in-cloud-security/)”在云安全方面意味着什么呢？根据标准定义，上下文是关于潜在威胁、漏洞和错误配置的信息[集合](https://thenewstack.io/how-to-evolve-cloud-security-posture-management-for-compliance/)，这些信息可能会导致云环境中的安全漏洞。从攻击者的角度来看，上下文为您提供了云环境背景中发生的事件和情况的完整画面。除了增强其云环境的安全态势之外，这还为安全团队提供了上下文安全情报和跨云资产的统一可见性，使他们能够更高效地改进安全运营。

上下文安全智能系统使用软件和硬件来自动收集和分析工作负载内部的数据以及云配置详细信息，从而提供关于表面风险及其根本原因的统一信息。这种环境和暴露关键攻击路径的能力使组织能够正确看待安全问题，从而在云环境中有效消除风险并最大限度地提高工作效率。

## **上下文如何帮助实现云安全**

云原生环境面临各种[安全风险](https://orca.security/resources/blog/the-top-emerging-cloud-based-security-threats/)，包括人为造成的错误配置、受损和未受保护的资产、加密不当的数据、云服务的危险默认设置、有风险的权限以及常见漏洞和暴露(CVE)。这些为攻击者提供了一个广阔的攻击面，从中可以设计出针对关键资产的各种攻击路径。这就是上下文发挥作用的地方。环境有助于组织描绘出潜在风险的完整图景，并确定最关键威胁的补救优先级。

### **确定云安全风险的优先级**

上下文有助于减少误报安全警报，从而降低团队中的警报疲劳。当安全工具配备了关于潜在威胁的上下文信息时，它们能够分析这些上下文数据，以帮助确定风险的优先级。为此，他们应该考虑三个关键因素:如果发生违规，威胁的严重性、攻击者对底层资产的可访问性以及对业务的潜在影响。一旦确定了风险的优先级，团队就可以专注于首先解决最重要的威胁，以便更快地进行补救。

### **从攻击者的角度了解云环境**

攻击者利用您的环境的最常见方式是首先分析整个攻击面，以发现最薄弱的环节，并形成通往您宝贵资产的直接路径。通过利用这一漏洞，他们利用资产之间的任何关系与您的环境一起横向移动。借助上下文安全智能，安全团队可以在一个可视化图形中可视化潜在的攻击路径(攻击者可能采取的横向移动选项，以转移到其他工作负载),从而在自己的游戏中击败攻击者，该图形包含所有相关云实体及其风险的数据，包括漏洞状态、错误配置风险、信任和授权、数据以及它们之间的关系。

此外，上下文安全情报使安全团队能够识别皇冠珠宝资产的位置，包括个人身份信息(PII)、机密暴露、知识产权、财务信息和其他[敏感数据](https://orca.security/resources/blog/cloud-data-security-best-practices/)，以及主动威胁的运行位置。这对于理解威胁环境和确定哪些威胁最危险至关重要。它还允许安全团队立即了解哪些攻击路径对业务最为关键，以便他们可以首先补救这些路径。

### **攻击路径分析和评分**

最常见的云安全态势管理(CSPM)工具错误之一是未能考虑可能危及公司资产的众多配置和组合。考虑下面的例子:您有一个不安全的接口，可以通过互联网访问第三方集成，您的 Amazon RDS 存储未加密的客户端数据。由于 API 的不安全互联网暴露，获得端点访问权的攻击者可以轻易地更改存储的数据。

您可以使用[攻击路径分析](https://orca.security/resources/blog/cloud-attack-path-analysis/)来寻找可能暴露公司最有价值资产的危险风险组合。这种环境对于帮助保护云至关重要。上下文通过对攻击者的众多路径以及它们之间的联系进行评分和优先排序来帮助安全团队。

## **结论:采用上下文感知云安全**

为了使云安全工作取得成效，必须在具备必要智能的背景下看待风险，以便根据严重程度确定威胁的优先级，从攻击者的角度可视化潜在的攻击，并识别、分析和评估针对有价值的云资产的多种可利用的攻击路径。基于上下文的安全智能还提供补充信息，以改进安全决策并减少警报疲劳。

组织必须投资于可靠的云安全平台，该平台使用统一的数据模型来收集和关联每项资产的上下文数据，包括云工作负载和配置中潜在风险的详细信息，以及关于外部和内部云连接的信息。

[Orca Security](https://orca.security/resource/literature/orca-security-solution-overview/) 正是通过扫描真实世界的云部署来寻找潜在漏洞，可视化攻击者的潜在路径并优先考虑风险补救来做到这一点(以及其他)。Orca 的 [CSPM](https://orca.security/platform/cloud-security-posture-management-cspm/) 增强了上下文感知安全数据，有助于在多云环境中保护云原生、Kubernetes 和微服务的安全。
