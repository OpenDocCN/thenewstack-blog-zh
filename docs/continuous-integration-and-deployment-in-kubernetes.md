# Kubernetes 中的持续集成和部署

> 原文：<https://thenewstack.io/continuous-integration-and-deployment-in-kubernetes/>

[持续整合并部署在库伯内](https://thenewstack.simplecast.com/episodes/continuous-integration-and-deployment-in-kubernetes)

云经济学家科里·奎因说“每天，我解决一个问题:我解决 AWS 账单。它往往会分解成几个部分:数据传输和找出区域套利。”在他的工作中，Quinn 看到了许多企业云环境，并对大型企业采用微服务仍然面临的问题有所了解。

“有工程背景的人往往有一种天生的偏见，我们关注的是工具和方法。例如，关于 vi 和 emacs 的战争已经持续了 30 或 40 年，仍然没有明显的赢家。有些人说 vi 就是答案，有些人就是错了。所以有这种自然的倾向去了解工作是如何完成的，几乎到了遮蔽工作是什么的程度。随着微服务革命的到来，你会看到这种元素周期性地出现。人们现在开始疯狂地崇拜他们在会议上看到的东西，在公共汽车广告上，谁知道在哪里。他们开始扭曲一种旨在解决政治和文化问题的技术，并试图以不恰当的方式使用它。奎恩说:“你可以制造一个神奇的扭矩扳手，但除非你的握法非常错误，否则它不会帮你更有效地把钉子钉进去。”。

“微服务的来源是你有 500 名工程师在一块巨石上工作，所有人都踩着彼此的脚趾。他们非常迅速地到达一个他们不能有效地发布代码的地方，基本上，你会以一句老话结束:一群鲸鱼被称为一个 pod，一群乌鸦被称为谋杀，一群开发人员被称为合并冲突。如果你把它分解成不同的小组，让小团队一起工作，它会变得更加有用，”奎因说。

他接着说，如果使用微服务的实际原因在数百个不可维护的个人服务的混乱中丢失，微服务很容易变得比它们解决的问题更麻烦。“他们最终有效地试图制定一个针对这种非常具体的设置的全球解决方案，他们没有关注他们的问题是什么，”奎因说。

Quinn 说，在 Kubernetes 环境中处理微服务需要重新思考你的整个方法。“专注于核心竞争力的东西。一般来说，考虑一个复杂的、基于容器化微服务的环境的维护、供给和部署。首先，市场上没有很多人有能力胜任这个领域的工作。同样，这也不是你的公司在大多数情况下要做的事情；就是卖小配件或者服务小配件。所以找一个知道鲨鱼藏在哪里以及如何帮你度过这个过渡期的人会增加巨大的价值，”奎因说。
