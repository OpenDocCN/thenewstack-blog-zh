# 高效云检测和响应的 5 个步骤

> 原文：<https://thenewstack.io/5-steps-to-effective-cloud-detection-and-response/>

[Gartner 预测](https://www.gartner.com/en/newsroom/press-releases/2022-04-19-gartner-forecasts-worldwide-public-cloud-end-user-spending-to-reach-nearly-500-billion-in-2022)2022 年全球公共云服务支出将增长 20.4%。这是由于云原生应用程序架构(如 containers、Kubernetes 和 serverless)的出现，使组织更容易在云中部署其应用程序，从而导致公共云基础架构的采用增加。

与此同时， [IDC 调查](https://securityboulevard.com/2021/06/data-breaches-rise-as-organizations-shift-to-the-cloud/)显示，2021 年，98%的公司遭遇了云数据泄露，这一比例相当高，高于 2020 年的 79%。不幸的是，组织不能仅仅将他们的内部安全解决方案用于云。

许多云安全工具侧重于识别和警告潜在风险，如控制平面配置错误、工作负载和应用程序漏洞以及不安全的机密管理。鉴于数据泄露事件的明显增加，公司需要在[现代云安全](https://orca.security/resources/blog/making-modern-cloud-security-a-reality/)中采用云检测和响应(CDR)。

## **什么是云检测和响应(CDR)？**

[云检测和响应](https://orca.security/resources/blog/how-orca-detects-cloud-attacks/)是一种新的云安全方法，专注于检测和响应云中的攻击。

云检测和响应工具在大型和小型组织中都有使用。大型组织使用安全运营中心(SOC)或事件响应(IR)团队来防止攻击。SOC 和 IR 团队使用 MITRE ATT 和 CK (MITRE 敌对战术、技术和常识)和 NIST CSF(网络安全框架)作为他们的团队框架。另一方面，小型组织使用自己的 IT 团队或外包服务，如托管检测和响应(MDR)。

## **CDR 如何工作？**

有两种类型的 CDR 解决方案:[基于代理的解决方案](https://orca.security/resources/blog/why-agent-based-security-fails/)和无代理的解决方案。

*   **基于代理的解决方案**使用安装在工作负载上的代理。它们结合了来自端点、流量分析、云流量和审计日志的信息，以及云服务提供商提供的数据。
*   **无代理解决方案**采用快照扫描方法，从工作负载的运行时块存储中外部收集数据，并通过 API 检索云配置元数据。

CDR 工具在以下阶段为客户带来价值:

1.  **检测**–它们对云服务中的攻击进行持续监控，并提供警报功能。
2.  **调查**——他们审查攻击步骤、技术、时间表和对可用数据的分析，以确定对威胁的必要回应。
3.  **响应**–他们的能力侧重于确保在云威胁造成损害之前将其遏制，例如自动修复或自动转发到票务系统。
4.  **弹性**——他们帮助调查，并根据可用数据提供潜在的补救措施。

## **CDR 如何帮助确定警报优先级**

云检测和响应解决方案应用警报优先级来帮助团队首先解决最关键的风险，并消除误报。CDR 工具提供了对云资产和工作负载数据的全面了解，这些数据是云安全的上下文数据。

CDR 工具可以根据警报对业务的影响、云资产对攻击者的可访问性以及利用上下文感知安全智能时[横向移动](https://orca.security/resources/blog/lateral-movement-attack/)的可能性来分析警报的严重性。结果，安全团队被引导到最关键的攻击路径，利用这些路径可能会对业务造成损害。

## **CDR 入门的 5 个步骤**

要有效获得云检测和响应的好处，请遵循以下步骤。

### **1。实现完整的资产覆盖**

有效的云检测和响应需要完整的资产覆盖。在这种情况下，您必须选择具有无代理功能的可靠 CDR 解决方案，不仅可以自动覆盖所有云资产，还可以检测和监控空闲、暂停和停止的工作负载、孤立的系统和无法支持代理的设备。

因为很难在每个资产上安装代理，所以基于代理的解决方案不适合资产覆盖，因为它们不能提供完整的覆盖。

### **2。深入了解云环境**

为了更好地了解整个云环境内部的情况，您必须了解以下各层中现有的[风险和威胁](https://orca.security/resources/blog/the-top-emerging-cloud-based-security-threats/):

*   **云基础设施级别**–您需要了解哪些资产在哪些网络上运行，以及谁可以访问这些资产。
*   **操作系统级别**—除了检查正确的操作系统配置，CDR 工具还应检查用户权限合规性，以及是否应用了所有必需的补丁。
*   **应用程序级别**—组织需要了解所有已安装的应用程序及其配置，并且应该扫描漏洞和不安全的补丁。
*   **身份级别**–您需要了解身份和访问管理系统权限和帐户，以便能够检测用户和角色行为的异常。
*   **API 级别**–您必须实现可见性，以便检测潜在的恶意行为，并理解攻击者可能如何利用环境中现有的 API 漏洞和风险。
*   **数据级别**–数据清单的可见性对于保护组织的皇冠上的宝石和包含敏感数据的服务器至关重要。

### **3。获取全面的云遥测数据**

有效的 CDR 解决方案应该能够收集数据。云服务提供商(CSP)提供自己内置的云威胁检测功能，CDR 解决方案可以访问其中的许多服务。大多数 CSP 使用遥测源的组合来识别攻击，包括利用分析和威胁情报补充来源的网络流量日志。当考虑 CDR 检测和响应选项时，请寻找一个单一的集中式平台，该平台可以接收、聚合、分析和呈现数据以及遥测数据。

### **4。实施情境智能**

有效的 CDR 安全平台应使用中央数据模型来收集和关联关于每项资产的[上下文信息](https://orca.security/resources/blog/not-all-risks-are-equal-why-context-matters-in-cloud-security/)，例如关于云工作负载和配置的详细信息，以及外部和内部云通信中的潜在风险。这种环境感知数据是确保安全团队通过根据严重性评分集中精力于最具利用性的攻击路径，快速识别和修复最关键问题的关键。

### **5。开发工作流程集成**

为了快速评估和解决问题，安全团队必须将 CDR 解决方案纳入其工作流。通常使用补救协调、警报服务、SIEMs、SOA 和票务系统，并将 CDR 解决方案纳入流程技术集成。这些集成将允许 SOC 和 IR 团队提高自动化和生产率，并缩短补救时间。它们还应该使安全团队能够组织、修改自动警报，并将其整合到正在进行的操作中。

## **结论:将 CDR 集成到您的云安全战略中**

网络攻击者以越来越多的速度攻击存储在云中的应用程序和信息。因此，对于组织来说，确保云检测和响应能力成为其云安全运营不可或缺的一部分非常重要。此外， [CDR 平台](https://orca.security/platform/cloud-detection-and-response-cdr/)必须提供关于活跃威胁的清晰且可操作的信息，并在不产生额外噪音的情况下实现快速调查和响应。
