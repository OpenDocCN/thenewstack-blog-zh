# 来自《新遗迹:分布式追踪的现代可视性》的主题演讲

> 原文：<https://thenewstack.io/the-keynotes-from-new-relic-modern-visibility-with-distributed-tracing/>

[来自《新遗迹:分布式追踪的现代可视性》的主题演讲](https://thenewstack.simplecast.com/episodes/the-keynotes-from-new-relic-modern-visibility-with-distributed-tracing)

在《新栈制造者》的这一集里，TNS 创始人兼主编 Alex Williams 在俄勒冈州波特兰的 New Relic 办公室现场预览了 [FutureStack 2018](https://newrelic.com/futurestack) 会议主题演讲，并从产品管理的 New Relic SVP[亚伦·强森](https://www.linkedin.com/in/aaronjohnson)那里了解了更多关于 New Relic 的分布式跟踪方法。

展开对话时，Williams 谈到了 New Relic 从 FutureStack 得到的启示:“我们可以从关注现代开始。我们将现代定义为:云、容器、微服务、无服务器和开发运维。Johnson 强调了容器已经成为的“避雷针”,他解释说，DevOps 的创建和选项结合在一起时会产生一个“地震事件”,导致许多以前不专注于软件的公司首先成为软件公司，用这些技术转变他们的业务。

谈到 New Relic 的创始故事，Johnson 指出，在开始时，APM 并没有广泛使用。New Relic 一开始的目标是“使 APM 民主化”，并让大众都可以使用它。当一个组织转向微服务并开始打破他们的垄断时，分布式跟踪就开始发挥作用了。“当你这样做时，你在 New Relic 上看到的曾经非常深入的事务跟踪变成了你需要跨许多许多系统可视化的东西，你需要能够看到这些系统是如何连接在一起的。很可能，许多性能问题来自网络内部或这些不同层之间。对我们来说，当你移动到容器和微服务时，分布式跟踪是一个非常重要的工具。这是我们将在 FutureStack 发布的公告之一。”

讨论的另一个热门话题是服务网格，Johnson 强调了围绕 New Relic 和 Istio 的新闻。“它有两个部分。我们将宣布或总结我们的用户运营的 Kubernetes。Kubernetes 已经成为部署容器的人的事实上的标准。对于我们来说，为 Kubernetes 提供一个真正深入和深思熟虑的监控解决方案是必不可少的，”Johnson 说，并指出 IBM 正在利用 New Relic 的 Kubernetes 集成来运行超过 5，000 个 pod，并补充说，“他们使用我们的可扩展平台在 New Relic 的基础上构建了一个 Istio 集成，该集成将基础设施中的数据与他们的容器相结合，并利用 Kubernetes 集成为 Istio 构建了一些非常棒的可视化功能。”他们将在未来堆栈的舞台上展示。
