# 即将发布的 Npm 更新将添加双因素身份认证

> 原文：<https://thenewstack.io/forthcoming-npm-update-add-two-factor-authentication/>

Npm，事实上的 Node.js 存储库，正在其即将发布的 [5.5.1 版本](http://blog.npmjs.org/post/166044150500/v550-2017-10-04)中集成对双因素认证的支持。这一变化带来了对安全令牌的额外粒度控制，现在可以设置为只读。该更新还包括人们期待已久的从命令行更改用户电子邮件和密码的能力，而不是要求访问国家预防机制网站。

双因素身份验证不仅需要用户名和密码，还需要其他形式的身份验证，例如，在 npm 的情况下，在移动电话上或其他应用程序中放置安全令牌。首席技术官 [CJ Silverio](https://www.linkedin.com/in/ceejbot/) 、 [npm Inc.](https://www.npmjs.com/) 表示，这种安全技术现在是当今以开发者为中心的服务提供商的筹码，并指出双因素认证已经在 GitHub、Gmail 和 AWS 中就位。

“从 2014 年我开始在 npm 工作的那一刻起，我就一直想这么做。像我们使用的所有其他工具一样，我们需要双因素身份认证。你在 [GitHub](https://www.github.com) 上获得一个账户，你设置了双重认证。你在 [AWS](https://aws.amazon.com/) 上获得一个账户，你打开双因素认证。这就是你要做的。你让人们更难窃取你的证书。”

Silverio 还表示，npm 是第一个包含双因素身份验证的包管理器。

Silverio 说，这些功能带来了“对 npm 身份验证令牌的更好控制。您可以生成用于测试 CI 服务的只读令牌。您可以限制授予身份验证令牌的权限范围。如果你在 Travis CI 上测试，你可以给 Travis 一个只读令牌。如果你把它泄露出去，这种事情发生的频率比应该发生的要高得多，那它就是可以阅读但不能发表的东西。”

这也意味着在 npm registry 中拥有自己的包的用户可以在对可能被成千上万的 JavaScript 开发人员使用的软件进行更改时验证他们的身份。

再加上 npm 引入的 Orgs 包组织功能，该功能允许企业(和其他人)在 npm 注册中心内描述他们自己的名称空间，以便为他们的开发人员创建一个受祝福的企业选择库，这将进一步提高企业用户的安全性。

Silverio 说，最终的计划是，“让一个组织启动需要对其中的所有人进行双重认证。这是我们几乎肯定会在下一个次要版本中推出的功能:发布特定包时需要双重身份验证。”

因此，虽然不是每个人都需要使用双因素身份认证，但对于 npm 中更大、更受欢迎的项目，双因素将成为规范。

Silverio 还暗示，即将发布的 npm 将进一步细化对授权令牌的控制，让团队能够更好地控制每个用户及其软件的使用。该团队还致力于改进 npm 的整体性能。
