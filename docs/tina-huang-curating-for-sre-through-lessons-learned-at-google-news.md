# 蒂娜·黄:通过谷歌新闻的经验教训为 SRE 策展

> 原文：<https://thenewstack.io/tina-huang-curating-for-sre-through-lessons-learned-at-google-news/>

20 世纪 90 年代末，许多孩子迷上了视频游戏编程。Tina Huang 喜欢开发她的 GeoCities 网站，但不喜欢制作游戏。黄喜欢将自己的网站自动化。

“说让我对编程感到兴奋的是自动化，这不是谎言，”[Transposit 的联合创始人黄在本周我们的科技创始人播客系列中说道。“现在，你可能会想:‘哪个中学生喜欢自动化？’"](https://www.transposit.com/)

[通过谷歌新闻](https://thenewstack.simplecast.com/episodes/curating-for-the-sre-through-lessons-learned-at-google-news) 的经验教训为 SRE 策展

黄喜欢用一点点代码将平凡的任务自动化的想法，所以她不必像杰森一家和机器人罗茜一样手工打字，这是人们想要的机器人。那里可以叠你的衣服，但不能剥夺人们喜欢做的事情的乐趣。

黄就像我们采访的许多创始人一样。她的工作可以是她想要的。但黄也必须处理好所有需要完成的事情。所有的工作都归结于换位网站在主页上说的话:让混乱归于平静。通过互联的工作流，为 TechOps 和 SREs 提供跨人员、流程和 API 的可见性、上下文和可操作性。

这些陈述反映了她自己使用自动化提供高质量信息的经验。

“当我在谷歌和推特这样的公司工作时，我总是逆流而上，你知道，当时谷歌新闻的口号是“机器人新闻”，黄说。他们心目中的理想是如何让机器人做所有的新闻报道。这很有趣，因为现在我想我们有了不同的看法。但在当时，人们普遍认为机器人播报的新闻会更真实、更民主。"

黄在谷歌从事一个项目，探索如何使用算法为人类编辑策划第一轮策划，然后将这种人情味添加到新闻中。这部作品反映了她对长篇新闻的热爱和对信息的人情味。

Transposit 提供了类似的下一级集成。有 RSS 粉丝吗？黄对 RSS 又爱又恨。她喜欢它能喂养的东西，但如果饲料没有经过过滤，那么它就会变得势不可挡。例如，当多个集成从 Slack 和其他来源开始分层时，就会被信息淹没。

“突然间，你被信息淹没，因为这些信息是为机器消费而设计的，而不是在人类范围内，”黄说。“你需要在此基础上的下一层监管。比如，你如何允许人们对这些信息进行注释？”

提供订阅选择会有所帮助。但是在什么水平上呢？这也是黄希望通过换位解决的问题之一。
