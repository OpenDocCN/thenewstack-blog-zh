# 新的堆栈环境:Kubernetes 是新的应用服务器吗？

> 原文：<https://thenewstack.io/the-new-stack-context-is-kubernetes-the-new-app-server/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了 Kubernetes 管理服务 [Spectro Cloud](https://www.spectrocloud.com/) 的产品副总裁 [Tina Nolte](https://www.linkedin.com/in/tinanolte/) ，关于为什么我们不应该认为容器/Kubernetes 只是另一种形式的虚拟化。

TNS 编辑和营销总监[利比·克拉克](https://thenewstack.io/author/libby/)主持了这一集，旁边是 TNS 执行主编[约阿布·杰克逊](https://thenewstack.io/author/joab/)。

[第 117 集:Kubernetes 是新的应用服务器吗？](https://thenewstack.simplecast.com/episodes/episode-117-is-kubernetes-the-new-app-server)

Nolte 最近为我们写了一篇受欢迎的帖子，讲述了为什么我们不应该将容器和 Kubernetes 视为另一种形式的虚拟化——它开辟了一种全新的方式来思考应用程序开发和部署。所以我们想更多地了解这个概念。

“Kubernetes 实际上是基础设施和应用程序之间中间地带。因此，应用程序本身能够采用不同的架构，因为运营 PaaS 层，如果你愿意的话，”她解释道。“这不仅仅是旧应用程序到新基础设施的提升和转移。”

过分关注 Kubernetes 的基础设施方面最终[会错过其真正的价值](https://thenewstack.io/why-apps-are-fundamentally-different-in-the-kubernetes-era/)，这是诺尔特在为一家广受好评的基于 OpenStack 的初创公司 Nebula 工作时获得的一种见解，该公司最终被关闭[。从我们的对话中:](https://www.networkworld.com/article/2905673/openstack-company-nebula-shutters.html)

大多数企业 OpenStack 供应商专注于某种公共云竞争路径，如果你愿意的话。然而，我认为由于对基础设施的关注根深蒂固，大多数供应商没有认识到公共云提供的平台服务的价值。我曾经听说过，每个人都认为他们在堆栈中的层是困难的问题所在。它们上面的每一层都很简单。如果你深深地陷入了基础设施的思维，你就不会欣赏这个生态系统在你之上发展的方式。"

我们还深入研究了 Spectro Cloud 提供的 Kubernetes 服务。我们一次又一次地听到管理 Kubernetes 的操作复杂性。因此，我们了解了[公司如何简化这一过程](https://thenewstack.io/spectro-cloud-saas-kubernetes-your-way/)。

“因此，我们的目标是在企业考虑他们的 Kubernetes 基础架构堆栈时，消除控制和易用性之间的权衡。她说:“不管这些堆栈部署在哪里，我们都为用户提供了从基础操作系统开始构建基础架构堆栈的能力，这种能力称为集群配置文件。然后，我们管理这些社区基础架构堆栈的部署和维护，这些堆栈从集群配置文件中部署，所有这些都来自我们的 SAS 管理系统。”
