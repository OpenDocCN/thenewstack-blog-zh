# 渲染:较少工程的云部署

> 原文：<https://thenewstack.io/render-cloud-deployment-with-less-engineering/>

在帮助建立支付初创公司 [Stripe](https://www.stripe.com/) 的过程中， [Anurag Goel](https://www.linkedin.com/in/anuragoel/) 意识到公司在管理亚马逊网络服务的日常事务上花费了太多的时间和金钱。即使对顶级工程师来说，这也是一个黑箱。

在他离开公司后，他自己的经历与其他前 Stripe 工程师的经历相符——即使 Docker 和 Kubernetes 越来越受欢迎，在线部署应用程序还是太难了。这让 Goel 创建了云托管平台[渲染](https://render.com/)。

“我们只是看到了对比 AWS 更简单，但给人们类似功能的东西的直接需求，特别是在健康检查和私人网络等方面，”Goel 说。

“如果你告诉开发人员建立一个虚拟专用网，他们的目光会变得呆滞，甚至系统管理员也不会真正期待建立 [VPC](https://aws.amazon.com/vpc/) 控制、安全组和防火墙等等。事实是，大多数应用程序……70%到 80%的应用程序不需要像 AWS 那样复杂。”

它的概念类似于 Rancher Labs 为 Kubernetes 开发的[Project Rio](https://thenewstack.io/rancher-rio-taking-care-of-the-first-half-of-the-developers-day/)——自动化重复性任务。

“Render 是在公共云之上实现隐形基础设施承诺的公司之一。它提供了一个自动化操作角色的平台，使开发人员能够专注于构建代码，而不是建立云基础设施的重复任务，”分析师 Nicole Henderson 在一份关于该公司的 451 研究报告中写道。

## 自动设置

Goel 说，Render 正在从 AWS、Heroku 和 Digital Ocean 等网站的用户那里赢得客户，起初通常是作为一个单一的静态网站，然后随着客户体验到它有多简单，越来越复杂的应用程序。

皮特·布蒂吉格总统竞选团队从 GKE 转移过来。该活动的技术负责人 Shreyes Seshasai 表示，他们在 Render 上部署了 20 多项服务。

“我们从一家主要的云提供商那里获得渲染，因为这是最好的解决方案，可以帮助我们由四名软件工程师组成的小团队满足活动的大规模扩展和灵活性需求，”他说。

戈尔说，在每次总统辩论结束时，当 Buttigieg 将人们引向他的网站时，Render 处理了“疯狂的流量”——每秒数万次请求。即使在新冠肺炎关闭期间，随着用户群的流量不断增加，它每个月仍能服务超过 10 亿个请求，尽管他并不确定为什么关闭会加速事情的发展。

Render 的构建是与提供者无关的。其服务器基础设施使用 Go、Docker 和 Kubernetes。它的仪表盘是用 React 和 GraphQL 搭建的；PostgreSQL 和 Redis 用于存储和缓存；让我们为 SSL 管理加密。

Render 的云平台运行在 AWS 和谷歌云平台(GCP)上。用户将其 GitHub 或 GitLab 存储库连接到该平台，该平台处理所有的部署和配置，包括设置和部署虚拟私有云、数据库、存储、web 服务器、安全组、防火墙和负载平衡。它还可以围绕性能自动进行扩展、监控和警报。

“从一个简单的网站到复杂的 Elasticsearch 集群或 Kafka 集群，我们以一种我们的用户觉得比其他任何事情都更容易的方式使所有这些成为可能，”Goel 说。

他指出，在应用程序中添加一个持久磁盘可以让 Render 变得更容易。

你只需要告诉 Render 你想要的磁盘大小和你的应用程序将要读取它的挂载点。Render 负责备份，确保备份始终正常运行。每次部署新版本的应用程序时，您都不必担心它在哪个区域运行。

另一件简单的事情是建立私人网络。它完全基于用户界面，您可以像创建 web 服务一样创建它。

“你给我们你的应用程序的某些关键部分，即如何构建和如何启动它。我们要么推导出所有其他东西，要么在某些情况下，我们给你某种高级选项，在你需要更大灵活性的地方填充你的应用。

“而且一旦你启动了私有服务，你就不必担心服务发现。你再也不用看 IP 了。这些都是内部 IP。我们只是给你一个服务和内部服务的网址。…如果它是一个私有的 Redis 实例，或者是你不想在网上公开的 Elasticsearch 实例，它只是它只是 Elasticsearch: 9200，而且它只在你自己的网络内可用，其他人看不到它。它不能从互联网访问，其他渲染用户显然没有办法访问它。

“如今，建立专用网络极其复杂。我们已经看到许多人对 Render 的易用性感到非常兴奋，一些人已经能够构建他们不会拥有的应用程序，只是因为它在 AWS 上太复杂了，”Goel 说。

设置 cron 作业的便利性是 Render 用户增长最快的领域之一。

## 减少工程时间

整个向云的迁移本质上是从运行自己的虚拟机到使用提供商的虚拟机，Goel 主张将这些抽象掉。

“计算的历史总是不断增加抽象层次。我们相信，我强烈认为，基础设施需要被引入下一个抽象概念，即运行您的服务。不要担心服务器。不用担心缩放。这就是为什么我们看到如此多关于无服务器的宣传。Render 是无服务器的，因为你根本不用考虑服务器，”他说。

Render 于 2019 年 4 月推出，宣布获得通用催化剂和南方公园共同基金 225 万美元的种子轮投资。10 月，它击败了其他 20 家早期创业公司，在 Disrupt SF 2019 上通过 TechCrunch 初创公司战场竞赛赢得了 10 万美元。

“应用程序开发人员显然希望专注于他们工作的功能和可用性，而不是服务器的设置、部署和扩展。General Catalyst 董事总经理史蒂夫·赫罗德说:“Render 正是实现了这一目标。

Render 的平台根据运行客户应用程序所需的资源进行定价。亨德森指出，虽然不比 AWS 或其他公司便宜，但“该公司表示，其客户通过任务的自动化获得了更多价值并节省了资金，否则这些任务需要 DevOps 工程师，这对湾区的初创公司来说尤其昂贵(而且具有竞争力)”。

Goel 说，虽然目前专注于个人开发者和创业公司的需求，但它希望像 Stripe 一样与客户一起成长，随着客户需求的增长增加功能。

Render 现在是 Gatsby、Create React App、Hugo、Vue 等开源框架的官方部署选项。

该公司刚刚在欧洲增加了一个托管区域，并计划在今年晚些时候在亚洲增加一个。他说，该公司计划增加更多托管服务——Redis、Kafka、elastic search——并改进现有的托管 Postgres 服务。它还刚刚推出了一个针对日志和监控等第三方服务的附加市场。

亚马逊网络服务和 Redis 是新堆栈的赞助商。
