# 为 OpenStack 配置裸机服务器

> 原文：<https://thenewstack.io/day-life-openstack-ironic-core-member-ruby-loo/>

开发人员的日常生活可能会从一个时刻到下一个时刻发生巨大的变化，特别是如果一个人正在从事开源项目的话。英特尔云软件高级开发人员 [Ruby Loo](https://www.linkedin.com/in/ruby-loo-7a01b3) 将她的时间花在裸机供应软件 [OpenStack 讽刺](https://wiki.openstack.org/wiki/Ironic)上，这是一款允许 OpenStack 供应裸机服务器的软件，是其核心成员之一。

虽然 Loo 受雇于英特尔，但她的大部分日常互动都是在上游开源社区进行的。当补丁进来时，Loo 与其他核心成员一起审查它们，确保 OpenStack Ironic 的功能优先级得到满足。在 OpenStack 峰会上录制的今天这一集的[中，Loo 与 TNS 创始人 Alex Williams 坐在一起，探讨她的背景、OpenStack 核心项目成员和积极的开源社区参与者的日常任务，以及 OpenStack 的下一步是什么。](https://thenewstack.io/podcasts/Makers)

[open stack 讽刺核心团队成员 Ruby Loo 的一天](https://thenewstack.simplecast.com/episodes/a-day-in-the-life-of-openstack-ironic-core-team-member-ruby-loo-o-vrlqq)

对于那些对在裸机上而不是在容器或虚拟机上运行软件感兴趣的开发人员来说，OpenStack Ironic 希望为他们提供这种灵活性，直到完全不用 OpenStack 也能运行。“我们正在开发一个独立的讽刺工具，这样你就可以在没有 OpenStack 的情况下部署讽刺工具。我个人更感兴趣的是在 OpenStack 雨伞内部讽刺使用[煤渣](https://wiki.openstack.org/wiki/Cinder)、[扫视](http://docs.openstack.org/developer/glance/)等等。当我们开始讽刺时，它是 OpenStack 的一部分，随着时间的推移，人们开始对讽刺感兴趣，”Loo 说。

Loo 最初是在雅虎的一个职位上接触到开源软件的，她发现自己与 OpenStack 一起工作，这很有讽刺意味。Loo 拥有计算机科学硕士学位，在直接参与开源项目和平台之前，他已经在该领域工作了几十年。

从为雅虎这样的传统公司工作到开源社区，最初对 Loo 来说有点震惊。“你做的任何事情都有利弊。除了在峰会上，大部分时间都没有面对面的交流，而在公司里，你会和几个人一起工作。在一家公司工作的好处是你认识这些人。这些项目不像它们在开源中那么大。你在开源中有所有这些不同的项目，完成工作和特性落地需要一点时间。这更多的是民主，”卢说。

总的来说，对 OpenStack Ironic 等开源项目进行修补和更新可能是一个挑战，因为来自社区的贡献在质量上各不相同。Loo 强调，当投稿人的主要语言不是英语时，这可能会特别棘手。这就是[核心团队成员](https://wiki.openstack.org/wiki/Ironic/CoreTeam)对保持项目整体质量至关重要的地方。能够在+2 或-2 的范围内对贡献的质量进行评级。

“并没有那么多等级制度。每个人都可以查看补丁，并说他们喜欢或不喜欢它。Loo 解释说:“核心是唯一有权使用 2 并可以批准某些事情的核心。

对于那些可能有兴趣成为 OpenStack Ironic 等大型开源项目核心的人，Loo 指出，实现这一目标的关键是通过积极和高质量的社区参与。“你可以通过贡献代码、审查代码以及展示你对项目的理解来成为核心成员。”
