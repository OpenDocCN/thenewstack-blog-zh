# Cloudflare 推出不受限制的工作人员、无服务器和无限制的 CPU 使用

> 原文：<https://thenewstack.io/cloudflare-launches-workers-unbound-serverless-with-unthrottled-cpu-usage/>

自从 [Cloudflare](https://www.cloudflare.com/) 首次推出其 [Cloudflare Workers](https://workers.cloudflare.com/) 以来，三年过去了，现在它描述为部署在 95 个国家 200 个城市的数据中心的无服务器代码。当时，人们关注的是这些基于 JavaScript 的“服务人员”如何在边缘运行并提供无与伦比的速度。现在，该公司正在发布 Workers Unbound，这是一个无服务器平台，它说它提供了“无与伦比的”灵活性、性能、安全性、易用性和价格，而且重点已经改变。

“边缘计算，这是锦上添花。这不是人们一开始想要的。他们真正想要的只是一个简单的平台，在那里他们可以编写代码，而不是思考，这就是工作，”Cloudflare 首席执行官[马修·普林斯](https://twitter.com/eastdakota)告诉新的堆栈。“三年前，我们犯了一个错误，我认为这些年来几乎每个边缘计算平台都犯了一个错误，那就是他们以速度领先。这又是锦上添花，它不是一个真正伟大的无服务器平台。”

相反，普林斯说，团队和开发人员真正想要的是成本效率、易用性，以及最重要的合规性。

“实际上，比易用性更重要的是合规性。不太性感，谈论起来也不太有趣，但如果你的总法律顾问或首席信息官说你必须做 X 或者你不能做 Y，那就真的束缚了开发者的手脚，”普林斯说。“所以合规是非常重要的一环。实际上，我认为这将是任何边缘计算平台未来的杀手锏。”

虽然 Cloudflare Workers 在 CPU 时间、内存使用和程序大小等方面受到一定的限制，但未绑定的 Workers 相对不受限制，具有不受限制的 CPU 使用、自动扩展以及“运行繁重的工作负载而不必担心过度受限的 CPU 限制”的能力这允许用户执行比工人更复杂的任务，同时花费更少。据该公司称，对于相同的工作负载，“工人自由联盟”比 AWS Lambda 便宜 75%，比微软 Azure 功能便宜 24%，比谷歌云功能便宜 52%。”

普林斯解释说，有几个因素使得非绑定工人比其竞争对手便宜得多。

“我们必须从零开始，我们真的只是在一个超高效的计算平台上构建它。事实证明，如果你能从别人的错误中吸取教训，你就有了优势，”普林斯说。“第二件事，也是最重要的一件事，是我们真正建立了一种更有效的运行代码的方式。当传统的无服务器平台正在启动虚拟机或某种具有巨大开销的容器时，我们采取了不同的方法。我们使用了一种叫做隔离的技术。隔离是一种不同形式的沙盒，不同于轻量级得多的虚拟机或容器。”

普林斯说，他们使用 Chrome 项目的技术，使用“V8 的大规模修改版本”，可以从任何 TCP/IP 应用程序输入或输出。

自由工作者的其他好处包括调试工具，即时代码更新，支持多种语言，如 JavaScript，C，C++，Python，Go，Rust，Scala，Kotlin 和“甚至 COBOL”，以及该公司所说的 0 纳秒冷启动时间。这个启动时间是通过在 TLS 握手期间预热工作人员来实现的，这有效地消除了否则将是 5 毫秒的冷启动。

至于合规性的“杀手级功能”，Prince 认为，Cloudflare 能够在其庞大的全球网络中定位所有这些工作负载，这使其能够遵守当地关于用户数据和处理的法规。

“由于 Cloudflare 的网络如今覆盖了全球 100 多个国家的 200 多个城市，我们已经处于那些政府制定规则要求本地数据必须保留在本地的地方。workers platform 允许您确保可以在一个地方编写代码，然后我们将帮助确保您符合日益复杂的监管环境。我们认为这将是边缘计算向前发展的杀手锏。这是我们最大的客户对我们的要求。普林斯说:“我们已经有了一些工具来确保我们符合这一点，但我的直觉是，在未来三年内，这将越来越成为任何计算平台的一个关键功能。
