# 与 Dremio 的 Kelly Stirman 一起探索数据湖

> 原文：<https://thenewstack.io/dipping-into-data-lakes-with-dremios-kelly-stirman/>

[与德雷米奥的凯利·施蒂曼](https://thenewstack.simplecast.com/episodes/dipping-into-data-lakes-with-dremios-kelly-stirman) 一起进入数据湖

数据湖是一个诱人的想法:将公司的所有信息——结构化和非结构化的——倒入一个系统，然后每个人都可以根据需要访问和分析它。这是一个伟大的想法，但在实践中，将数据从湖中取出并送到需要它的人手中可能比听起来要困难得多。

[凯利·施蒂曼](https://www.linkedin.com/in/kellystirman/),[dre mio](https://www.dremio.com/)的首席营销官兼副总裁，几十年来一直与数据打交道。上辈子，他在 [MongoDB](https://www.mongodb.com/) 从事策略方面的工作，在此之前， [MarkLogic](https://www.marklogic.com/) 。在他的最新职位中，他的任务是帮助公司找到解决数据湖问题的方法。他说，这些问题中最主要的是让那些需要的人可以使用这些数据。

“现在还早。公司已经想出了如何获取数据，现在他们正在通过下一组项目从他们的计划中获得价值，”Stirman 说。“公司谈论的更重要的事情是将他们的工作负载整合到他们的数据湖计划中。例如，我们如何将已经在 [Teradata](https://www.teradata.com/) 中做了 20 年的事情转移到我们新的数据湖中？我们如何做到这一点，以便我们的数据科学家可以专注于构建他们的模型，而花更少的时间来挖掘数据并为模型开发做好准备？我认为这就是你看到许多公司流露出的兴奋和战略。”

Stirman 说，企业内部对数据湖的需求源于企业经历数字化转型的需要:他们需要更快地生产更多的软件。正如云为开发人员带来了自助式工作负载一样，Stirman 表示，数据湖可以为数据带来类似的自助式访问范式。

“真正推动这些不同类型的变化的力量都与应用程序开发中的这一巨大变化有关，这一变化在过去的 20 多年中真正展现出来，也就是说，过去我们看的是将一个应用程序从鸡尾酒餐巾纸后面投入生产的总成本；主要的成本因素是基础设施，而将应用程序投入生产的人员可能不到 5%。现在这完全颠倒了，对任何公司来说，压倒性的和占主导地位的成本是相关人员。基础设施成本持续下降。当你看到这一点时，聪明的首席信息官会说，‘我该如何针对这一特定问题进行优化？”施蒂曼问道。
