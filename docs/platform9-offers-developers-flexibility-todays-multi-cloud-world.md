# Platform9 管理 Kubernetes 并构建一个新的无服务器平台

> 原文：<https://thenewstack.io/platform9-offers-developers-flexibility-todays-multi-cloud-world/>

对于一些开发人员来说，使用 [Kubernetes](/category/kubernetes/) 容器编排引擎可能是一个挑战。对于基础架构跨多个平台或在裸机上运行的用户来说尤其如此。为了帮助开发者解决这些痛点，混合云和容器编排公司 [Platform9](https://platform9.com/) 为 Kubernetes 推出了[托管服务](https://platform9.com/press/platform9-makes-easy-deploy-docker-containers-production-scale/)，这是一个[云本地计算基金会](https://www.cncf.io/)开源项目。该公司还推出了一个新的基于 Kubernetes 的开源无服务器框架，名为[裂变](https://platform9.com/fission/)，为开发者提供了 AWS Lambda 托管无服务器服务的替代方案。

在这一集的[中，新堆栈制造商](https://thenewstack.io/podcasts/makers)，平台 9 产品副总裁 [Madhura Maskasky](https://www.linkedin.com/in/madhuramaskasky) 与 TNS 副播客制作人 Kiran Oliver 讨论了裂变和平台 9 管理的 Kubernetes。

[platform 9 如何在当今的多云世界中为开发者提供灵活性](https://thenewstack.simplecast.com/episodes/how-platform9-offers-developers-flexibility-in-todays-multi-cloud-world)

对于 Platform9，裂变向用户提供了自由使用多个云的承诺

“[Lambda]的一个问题是将你的开发者和用户锁定在一个系统中。Lambda 本质上不是多云友好的。美国建筑裂变相信开发商从根本上喜欢选择。我们用 Kubernetes 解决了另一个挑战，因为它抽象出了许多底层细节。裂变提供了一个层，你可以给它你的代码，告诉它如何运行它，它会在幕后做你需要的事情，让你的代码运行，”说

Platform9 最初是作为 OpenStack 服务提供商建立的，当它开始研究 Kubernetes 时，发现自己面临着是否在 OpenStack 上开发 Kubernetes 服务的挑战。Maskasky 指出，这一决定最终可能会疏远社区的很大一部分人，要么是那些对使用 OpenStack 不感兴趣的人，要么是不想通过添加 Kubernetes 使其堆栈进一步复杂化的 OpenStack 开发者。

“我们问了一个问题，‘什么是支持容器编排的正确框架，它将成为事实上的标准？’我认为我们非常强烈地感觉到，Kubernetes 是一个将赢得所有人的框架，”Maskasky 说，并补充说，“我们意识到有一个重要的客户群对 Openstack 不感兴趣。这些客户不会欣赏我们通过在 OpenStack 上运行而使堆栈变得复杂。"

最终，帮助开发人员摆脱单一来源供应商的束缚，更好地管理他们的多云生态系统是 Platform9 使命的核心部分。“这是 IT 和运营团队在后云时代将继续面临的核心挑战。开发者从根本上喜欢选择。他们喜欢他们有不同的方法和框架。处理这一问题的最佳方式是拥抱多样性，然后投资于合适的平台，通过消除复杂性来帮助您简化 it。

[云本地计算基金会](https://www.cncf.io/)是新堆栈的赞助商。
