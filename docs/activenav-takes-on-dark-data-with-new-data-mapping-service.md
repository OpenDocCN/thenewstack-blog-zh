# ActiveNav 通过新的数据地图服务呈现“黑暗数据”

> 原文：<https://thenewstack.io/activenav-takes-on-dark-data-with-new-data-mapping-service/>

专注于数据隐私和治理的供应商 ActiveNav 发布了其新的[库存套件](https://activenav.com/solutions/inventory/)，提供其所谓的“[数据映射即服务](https://activenav.com/use-cases/data-mapping/)，或者更简单地说，DMaaS。虽然“数据映射”听起来似乎只是关于数据沿袭跟踪，但 ActiveNav 的套件远远不止于此，还解决了数据质量、数据发现、数据分类、数据目录和冗余数据检测等方面的问题。

## **适时转型**

鉴于围绕数据的监管格局、向在家工作的转变、由此产生的用户数据的激增以及最近备受瞩目的数据泄露事件，ActiveNav 似乎很好地抓住了市场时机和受众。

数据保护、数据隐私和获得对敏感数据的控制是许多企业组织关注的问题，采用基于云服务的简单方法可能会鼓励组织主动发现和管理其数据，无论是敏感数据还是其他数据。

## **瞎猜(数据)**

《新堆栈》采访了 ActiveNav 的首席收入官 Dean Gonsowski，他对推动 ActiveNav 套件发展和崛起的背景提供了一些见解。

Gonsowski 提到，该公司在帮助组织“查找、归类、分类和修复”大量数据方面的背景为其当前帮助组织实现零“暗数据”的使命提供了来源。该术语指的是可能被挖掘到应用程序或非结构化内容中的数据，这些数据通常无法被发现、清点，因此也无法被分析和保护。

Gonsowski 说,“照亮”这种黑暗的数据“是第一步，下一步是绘制地图，然后将其可视化。所以我们……创造了这项新服务……我们从根本上建立了一份清单，一旦你有了一份清单——一份好的、持久的、永远在线的清单，你就可以将它汇总并放大成一张地图。”

在其研究过程中， [ActiveNav](https://www.gartner.com/reviews/market/file-analysis-software/vendor/activenav/product/activenav) 证实，发现敏感数据是许多组织的当务之急，并且由于不断通过新的法规，这一问题变得越来越重要，但是这些组织却在努力寻找合适的工具和方法来帮助他们。

## **超越行列
超越行列**

市场上已经有大量的数据治理解决方案，其中许多侧重于结构化的表格数据，无论是关系数据库还是数据湖。但 ActiveNav 的库存套件也扫描用户生成的内容，包括文档文件、电子邮件，甚至微软 SharePoint 和团队。

这些是黑暗数据的主要来源，也是在查找和保护敏感数据方面的潜在漏洞。ActiveNav Inventory 可以使用预先构建的规则来评估客户的合规性风险状况，以突出显示该公司在其数据环境中所谓的“毒性热点”。ActiveNav 表示，它应用人工智能来确保数据地图保持最新。

## 时间的紧要关头？

几十年来，数据治理和数据管理一直是必不可少的，但直到 GDPR 和 CCPA/CPRA 法案等监管框架的实施，主流技术市场才开始关注数据治理。监管有时是技术最好的朋友。

将数据映射的原则和技术应用于隐藏在非结构化内容中的数据，应该有助于组织管理和治理这些数据的“防御”和“进攻”两方面。由于有如此多的内容存在于云中，云服务在其中映射数据是合乎逻辑的、智能的，也是一个很好的优势。
