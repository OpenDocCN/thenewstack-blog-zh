# Aerospike 创始人 Srini Srinivasan 从 DB2 到实时

Aerospike 创始人 [Srini Srinivasan](https://www.linkedin.com/in/drvsrini/) 加入 IBM 时刚刚在威斯康星大学完成博士学位，并在第一个商业关系数据库管理系统 DB2 的创造者 [Don Haderle](https://en.wikipedia.org/wiki/Donald_Haderle) 手下工作。

哈德勒在创办实时数据平台 Aerospike 时，成为了斯里尼瓦桑的主要影响者。直到今天，哈德勒还是气塞公司的顾问。

“他是我第一个回去寻求成功建议的人，”斯里尼瓦桑在新的 Stack Maker 系列的最新一集《科技创始人奥德赛》中说

年轻、雄心勃勃的工程师斯里尼瓦桑离开 IBM 加入了一家初创公司。斯里尼瓦桑对他认为缓慢的步伐感到不耐烦，他会见了哈德勒，哈德勒告诉他去挑战自己，尝试可能会不舒服的新事物。

如今，Srinivasan 寻求研究和产品开发之间的平衡，类似于他在 IBM 学到的方法——在非常困难和不可能之间的平衡。

技术创业公司的创始人发现自己一直面临复杂的技术问题。斯里尼瓦桑谈到了灵感来解决那些问题，但是灵感到底是什么意思呢？

灵感是一个需要解析的复杂话题。它可以被认为是微不足道或肤浅的讨论。Srinivasan 说，当灵感成为工作的一部分时，以及一个人如何诚实地面对工作，灵感就变得很重要。灵感就是诚实。

“因为一旦一个人诚实了，你就能得到与你一起工作的人的信任，”斯里尼瓦桑说。“所以诚实导致信任。一旦你有了信任，我认为就会有合作，因为现在人们不必担心保护自己。你可以犯错，然后你知道这是一群值得信任的人。他们会，你知道，保护你的。然后，有了这样的团队，你现在可以设定看似不可能的目标。但是通过诚实、信任和协作的结合，你可以带领团队从根本上解决那些困难的问题。在某些情况下，你必须足够诚实地认识到，你并不具备解决问题所需的所有技能，你应该愿意走出去，让新的人来帮助你。”

Srinivasan 在 Aerospike 的软件开发中使用诚实的原则。这在气塞公司的工作中是如何体现的？它带来了关于 Unix、Linux、系统技术以及构建在基础设施之上的一切的各种见解。这也是 Srinivasan 非常喜欢的工作——构建基础技术，这可能需要几年的时间，但随着时间的推移，它将成为重要的、可扩展的、性能卓越的工作。
