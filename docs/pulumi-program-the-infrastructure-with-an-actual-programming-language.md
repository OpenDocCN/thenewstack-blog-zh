# Pulumi:用实际的编程语言对基础设施进行编程

> 原文：<https://thenewstack.io/pulumi-program-the-infrastructure-with-an-actual-programming-language/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了[乔·达菲](https://www.linkedin.com/in/joejduffy/)、[普鲁米](https://www.pulumi.com/)的创始人兼首席执行官，以及[普鲁米的技术营销经理索菲亚·帕拉菲娜](https://github.com/spara)。在本次会议中，我们深入研究了最近发布的 Pulumi 2.0 版本，该版本允许团队重用代码、应用策略并对基础设施进行集成测试，就像他们对应用程序开发所做的一样，这一概念被称为“架构即代码”

TNS 编辑和营销总监[利比·克拉克](https://thenewstack.io/author/libby/)主持这一集，旁边是 TNS 创始人兼发行人[亚历克斯·威廉姆斯](https://thenewstack.io/author/alex/)和 TNS 执行主编[约阿布·杰克逊](https://thenewstack.io/author/joab/)。

[第 114 集:用实际的编程语言对基础设施进行编程](https://thenewstack.simplecast.com/episodes/episode-114-program-the-infrastructure-with-an-actual-programming-language)

去年九月，我们报道了 Pulumi 的 [1.0 版本，揭示了 Pulumi 提供了免 YAML 的基础设施管理。它提供了使用标准编程语言(如 JavaScript、TypeScript、Python)和 Go 来部署和管理 It 资源的能力。新版本](https://thenewstack.io/pulumi-reaches-1-0/)[扩展了这个平台](https://thenewstack.io/infrastructure-is-code-and-with-pulumi-2-0-so-is-architecture-and-policy/)，提供了一种围绕安全性、合规性和成本管理实施最佳实践的方法。

Duffy 在播客中指出，使用与应用程序相同的语言对基础设施进行编程有许多优势。

“你有类，你有函数，你有包管理器；他说:“我们在过去几十年的应用程序开发中学到的所有伟大的东西也可以应用到基础设施开发中。

他举了一些例子:“如果你在 AWS 中建立一个网络，你可能会想为每个可用区域建立一个子网，只是因为你想适当地分配流量。好的，那么[使用]大多数 YAML 工具，您所做的是复制并粘贴在一个可用性区域中设置单个子网的相同的 50 行 YAML，您复制并粘贴 N 次。在语言中，有一个 FOR 循环。你说，对于每个可用性区域‘这样做’，”他说。“如果由于某种原因，您的开发环境与生产环境不同—可能是节点数量不同，或者您需要模拟一些服务，因为它们在开发中运行太昂贵—那么，现在您已经有了 If 语句。”
