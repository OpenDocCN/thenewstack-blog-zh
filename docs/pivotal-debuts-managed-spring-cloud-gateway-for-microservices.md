# Pivotal 首次推出面向微服务的受管 Spring 云网关

> 原文：<https://thenewstack.io/pivotal-debuts-managed-spring-cloud-gateway-for-microservices/>

Pivotal 赞助了这篇文章。

上周在奥斯丁举行的 SpringOne 平台会议上， [Pivotal](https://pivotal.io/) 发布了其管理版本的 [Spring 云网关](https://docs.pivotal.io/spring-cloud-gateway)，该公司在[博客文章](https://content.pivotal.io/blog/against-the-backdrop-of-vmware-tanzu-here-s-how-pivotal-platform-s-new-release-helps-you-build-modern-apps#Spring%20Cloud%20Gateway)中将其描述为“一种开发者友好的方式来配置 API 请求并将其路由到应用服务。”随着测试版的发布，Spring Cloud Gateway 将获得 Pivotal 的持续维护和支持，以及 Pivotal 平台的[单点登录](https://pivotal.io/platform/services-marketplace/identity-and-security/sso)模块的添加，该模块是该公司针对应用程序和 API 的身份和访问管理工具。

[Pivotal 营销副总裁 Richard Seroter](https://www.linkedin.com/in/seroter/) 称 Spring Cloud Gateway 不仅仅能够配置和路由 API 请求，还能添加安全性、缓存、负载平衡等功能，他在与新堆栈的访谈中强调了这些功能。

“Spring Cloud gateway 是 Spring Boot 的一个开源项目，是 Spring 框架的众多项目之一，将 Spring Boot 应用程序转变为 API 网关。你可以不用购买商业产品，而是去做路由、安全、缓存和速率限制，”塞罗特说。“这是一种相当轻便的方式。你可以进行测试驱动的开发，拥有一个部署管道——一些你通常不会用打包软件做的真正有趣的事情。”

除了作为一个托管服务，Spring Cloud Gateway 还提供了一个新的用户界面，提供对路由的监控、度量和洞察。

Spring Cloud Gateway 进入测试版的举动在该公司的 Spring Framework 开发者大会上受到了关注，部分原因是它正好位于该公司的两个主要焦点之间——它简化了开发者体验，并有助于从整体服务转向微服务。Seroter 解释说，Spring Cloud Gateway 使开发人员部署网关成为一种简单快速的体验，并使其脱离了只能费力部署一次的昂贵产品类别。

“如果我一天要编写 15 次应用程序代码，但我必须等待数据库团队，那么我的速度有多快？只有和我的约束一样快。Seroter 说:“这越成为代码，就越成为你作为应用程序的一部分部署的工件，对开发人员来说，这让我重新掌握了控制权。“传统上，这些都是大型商业产品。你可以买一个网关。这是基于操作的，而不是说也许每个应用程序都有自己的网关，每周部署和更新一次——这些是你在旧世界甚至不会想到要做的事情。这实际上开始将一些微服务机制转化为代码，而不仅仅是我必须购买的产品。”

在简化开发者体验方面，Seroter 将 Spring Cloud Gateway 描述为消除了理解基础设施以构建酷东西的需要。

“我是一名正在恢复的开发人员，我仍然以写代码为乐。我会想到我不擅长的事情，以及我做得不好的事情。我想做一些很酷的东西。我不会放弃构建负载平衡器、DNS 设置和配置存储，而且由于很多原因，我也不会做得很好。Seroter 说:“Spring Cloud Gateway 正在帮助一些东西融入基础设施。“我如何帮助下一亿人成为软件开发人员？这不是因为他们了解最基本的东西，而是因为他们能够快速构建、迭代并交付。这是令人兴奋的事情。”

随着迁移到测试版，Pivotal Network 上客户早期访问组中的 Pivotal 客户已经可以下载并安装 Spring Cloud Gateway。其他客户需要联系他们的平衡账户团队请求访问。
