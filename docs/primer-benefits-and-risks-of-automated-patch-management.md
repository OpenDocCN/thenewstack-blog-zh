# 初级读本:自动化补丁管理的优势和风险

> 原文：<https://thenewstack.io/primer-benefits-and-risks-of-automated-patch-management/>

这个故事是最新的

[in a series](https://thenewstack.io/securing-the-software-supply-chain-with-a-software-bill-of-materials/)

我们将在文章中探讨安全在开源软件供应链中的作用。

自动化应用程序操作的最大吸引力之一是自动化工具总是能够比人工干预更快地做出响应。对于像扩展和故障转移这样的任务来说是这样，对于响应安全漏洞来说也是这样。

开源安全公司 [Snyk](https://snyk.io) 的首席产品官[安内尔·马祖尔](https://www.linkedin.com/in/aner-mazur-95ba1950/?originalSubdomain=il)说:“归根结底，许多组织在其基础设施和应用程序中面临的风险是‘如果在我有时间做出反应之前，有一个安全漏洞被利用了怎么办？’”。“他们在谈论打补丁。”

补丁并不总是解决安全问题的最佳长期解决方案，但它们可以填补漏洞，同时组织可以找到最佳解决方案，这可能是使用不同的操作系统、更改工具或更新工作流程。但是当发现共同的漏洞和暴露(CVE)时，组织需要尽快做出反应。这意味着修补，修补越自动化，完成的速度就越快，组织暴露的时间就越短。

乍一看，自动化补丁管理似乎是解决这个问题的简单方法。一旦发现 CVE 病毒，系统会自动进行更新或打补丁，并在几分钟内修复 CVE 病毒，而安全团队则彻夜不眠。

现实稍微有些混乱。安全风险必须与操作风险相平衡。就补丁程序管理而言，有时停机风险大于花费额外时间进行手动干预所带来的安全风险。DevOps 安全公司 Accurics 的联合创始人兼首席技术官 Om Moolchandani 说:“业界尚未找到解决这个问题的灵丹妙药。

那么，自动补丁管理是如何工作的呢？什么时候是好主意，什么时候不是？我和几个安全专家聊了聊。

## **它是如何工作的？**

自动化补丁管理背后的想法很简单。集装箱安全公司 [StackRox](https://www.stackrox.com) 的首席产品官[希拉里·本森](https://www.linkedin.com/in/hillary-benson/)解释说:“我有一些 CVE，它们会出现在我的软件中的某个地方，我不必进去手动检查它，并找出我需要升级到什么版本才能摆脱 CVE 之类的东西，我只需要自动化就可以做到这一点。”。简单吧？

但是自动化工具到底应该放在哪里呢？它到底应该如何工作？Benson 说:“我的总体感觉是，仅仅因为人们部署和运行软件的方式有如此多的变化，就有一种适合所有人的工具来统治他们，这种方法不太可能成功。”

Snyk 提供自动补丁管理，马祖尔说用户反馈清楚地表明 CI/CD 管道是自动补丁管理的最佳场所。在他们的流程中，如果有适用于 CVE 的补丁，Snyk 将自动触发一个拉请求和一个 git 代码更改。开发人员会收到补丁警告，并在构建开始前批准补丁。

Benson 不太确定自动补丁管理是否能局限于工作流程的一部分。“这就像是你所有系统的集成，”她说。“我不认为自动化补丁程序管理系统必须位于您环境的任何一个特定部分，并实际覆盖它需要的全部范围。”

## **在现实世界中…**

您可能会注意到 Snyk 的自动化补丁管理…它不是完全自动化的，因为它包括一个开发人员手动批准构建的步骤。“我们得到的反馈之一是，客户希望自动修补，但他们希望批准补丁，”马祖尔说。换句话说，真正自动化的补丁管理让人紧张。

人们一致认为自动化补丁管理并不总是一个好主意。Moolchandani 说:“特别是在任务关键度高于安全性的情况下，自动化补丁管理没有多大用处。

> 现实情况是，安全与风险管理有关，虽然自动化补丁程序管理可以改善组织的安全状况，但也会增加运营风险。

现实情况是，安全与风险管理有关，虽然自动化补丁程序管理可以改善组织的安全状况，但也会增加运营风险。如果应用程序非常关键，以至于一分钟的停机时间都会导致巨大的收入损失，那么使用自动化补丁管理就不是明智之举。

自动化补丁程序管理涉及的运营风险还取决于您使用的基础架构类型。Benson 说:“如果您手动部署和管理您的基础架构，自动补丁管理是不合理的。因此，部署和管理运行软件的整个流程越自动化，补丁程序管理的自动化程度就越高

如果您有一个脆弱的遗留应用程序，自动补丁管理不是一个好主意。Benson 说:“在云原生基础设施的旅程中走得越远，就越容易找到以实际可行、可重复且风险更低的方式自动化补丁管理的方法。”

## **你更安全吗？**

Moolchandani 说:“最大的误解之一是自动补丁管理可以让你安全。“安全永远是一个旅程。这不是目的地。自动补丁管理提高了您的水平，让您在安全游戏中表现得更好，但这不是安全的全部。”

然而，自动补丁管理对于保持强大的安全态势显然是必不可少的，即使该过程只有 75%的自动化程度，它仍然允许组织更快地响应漏洞。Moolchandani 说:“发布的漏洞数量如此之多，以至于如果你手动修补所有的漏洞，你根本无法跟上速度。”“经常修补和快速修补以领先攻击者一步非常重要。”

根据 Maner 的说法，行业可能更倾向于完全自动补丁的一种方式是通过应用机器学习来确定补丁是否“安全”例如，如果修补程序在 1000 个 CI 构建中有一个失败，那么该修补程序将足够安全，无需人工批准即可部署。用户将能够配置多少百分比的失败构建是足够安全的，然后完全自动化修补。
