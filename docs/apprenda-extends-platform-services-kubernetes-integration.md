# Apprenda 通过 Kubernetes 集成扩展平台服务

> 原文：<https://thenewstack.io/apprenda-extends-platform-services-kubernetes-integration/>

PaaS 提供商 Apprenda 正在通过增加对 Google 的 Kubernetes 的全面支持来扩展其云服务的范围，Kubernetes 可用于编排基于 Docker 的大型工作负载。

该公司还发誓要贡献一些程序员来帮助开源项目 [Kubernetes](http://kubernetes.io/) 的努力。

“在考察了 Docker Swarm 和 Cloud Foundry 之后，我们发现 Kubernetes 是云原生应用的最佳平台，”Apprenda 首席执行官兼联合创始人 Sinclair Schuller 说。

Apprenda 当前的集群管理堆栈和操作套件将通过添加 Kubernetes 进行扩展。这种方法将允许 Apprenda 服务的当前用户运行。Net 和企业 Java 应用程序也可以使用 Kubernetes。Kubernetes 的引入将在未来几个月推出，不会中断任何现有的服务。

或许更重要的是，新产品还将允许 Apprenda 的客户使用容器和[微服务架构](https://thenewstack.io/microservices-changed-matter/)构建新的云原生应用。

舒勒说，Apprenda 的客户“受益于现有应用程序和云原生应用程序的云平台”。“他们不会同时运行两三种策略。他们不会说，‘好吧，这就是我们对现有应用的看法，然后这就是我们对全新应用的看法。’"

RedMonk 的 RedMonk 行业分析师 Fintan Ryan 在一份声明中表示:“通过将对 Kubernetes 的全面支持纳入其平台，Apprenda 在保护现有企业投资的同时，为下一代容器应用程序进入生产提供了一条清晰的道路和一致的愿景。”。

该公司提供了许多选择 Kubernetes 作为其容器编排引擎的理由。一个是谷歌的专业技术，它开发了 Kubernetes 作为其内部“ [Borg](https://thenewstack.io/google-lifts-the-veil-on-borg-revealing-apache-auroras-heritage/) ”调度程序的开源版本。“如果你看看技术世界，你会问自己的一件事是，谁是云规模架构和计算机管理和协调方面最好的公司，”舒勒说。

New Stack 最近的一项调查发现，72%的 Kubernetes 是由谷歌自己完成的，另外 15%来自 Red Hat。Schuller 已经谈到了单个公司主导一个开源项目的危险，所以来自 Apprenda 的额外投入肯定会有助于给 Kubernetes 一个更加多样化的贡献者基础。

Apprenda 也对 Kubernetes 的开源模型感到满意。舒勒指出，虽然谷歌迄今为止已经完成了 Kubernetes 的大部分工作，但该公司没有从该软件中赚钱的计划。与 Docker 相比，这使得谷歌成为一个更仁慈的捐助者，Docker 希望通过使用企业版和订阅支持，直接从其开源 Docker 容器中产生收入。

Kubernetes 也不知道它可以处理哪种容器，不仅支持 Docker，还支持其他容器。谷歌云平台提供自己的自动化[容器管理服务](https://cloud.google.com/container-engine/)，该服务基于 Kubernetes。

在下周于伦敦举行的 [Kubecon Europe](https://kubecon.io/) 上，Schuller 将更多地谈论 Apprenda 与 Kubernetes 的工作。

Apprenda、Docker 和 Red Hat 是新堆栈的赞助商。
