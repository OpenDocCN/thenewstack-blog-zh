# 当 90%的组件都是开源时，自动实现许可合规性

> 原文：<https://thenewstack.io/automating-licensing-compliance-90-percent-components-open-source/>

FOSSA 是一家开源软件公司，帮助公司管理开源依赖，强调自动化许可合规，FOSSA 创始人兼首席执行官王聪在本月早些时候索诺马开源领袖峰会的《新堆栈制造商》节目中说道。

[https://www.youtube.com/embed/AUnHE9hnh80](https://www.youtube.com/embed/AUnHE9hnh80)

视频

王来到我们的 TNS Makers Livestream 工作室，在费尔蒙特酒店的白色大会议帐篷里，他拿着一个活页夹上的卡片，标题是“开源许可的状态”。我们浏览了这些卡片，回顾了 FOSSA 在这一年中建立公司广泛知识库的过程中发现了什么。

FOSSA 正在使用这些数据来帮助建立一个自动化的合规引擎，该引擎基于 3000 多家公司正在使用的开源软件。除此之外，该公司还:

*   从 Mapbox、Docker 和 Hashicorp 等公司收集了 2300 万个开源包的数据和见解，这些公司都拥有现代化的工作流程。
*   通过工作，FOSSA 发现大约 90%的组件是开源的，但是管理的复杂性是巨大的。
*   发现构建和管理现代构建系统是多么复杂。工具并不总是决定性的，这使得对管理系统的投资成为更高的优先级。FOSSA 已经开发了基于开发人员如何配置 FOSSA 的行为工作流构建的反馈循环，这些工作流是从 GitHub 下载的。随着 FOSSA 嵌入到工作流中，该工具可以通过简单的绿色或红色警报通知开发人员和开发团队。

王是我们采访的第一个带着活页夹上的卡片来讲述他的故事的人。它是有效的，并为任何组织都必须面对的问题提供了一些见解。例如:我使用开源组件构建的应用程序是否符合所有正在使用的许可证？
