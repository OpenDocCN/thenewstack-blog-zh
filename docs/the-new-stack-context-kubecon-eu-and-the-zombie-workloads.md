# 新的堆栈环境:KubeCon EU 和僵尸工作负载

> 原文：<https://thenewstack.io/the-new-stack-context-kubecon-eu-and-the-zombie-workloads/>

欢迎来到[新的堆栈环境](/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了 [Intuit](https://www.intuit.com/) 的产品开发副总裁 [Pratik Wadher](https://www.linkedin.com/in/pratikwadher/) ，讨论该公司作为 Kubernetes 最终用户的体验，以及它在 Argo Flux 项目中的参与情况——一个使用 GitOps 进行持续部署和自动化工作流的单一工具链。我们还分享了参加本周“虚拟”举行的[kube con+CloudNativeCon EU 2020](/kubecon-eu-cloud-native-developers-now-an-army-6-5-million-strong/)的经历。

《新书库》编辑和营销总监[利比·克拉克](/author/libby/)主持了这一集，旁边还有 TNS 创始人&出版商[亚历克斯·威廉姆斯](/author/alex/)，TNS 高级编辑[理查德·麦克马努斯](/author/richard/)，以及 TNS 执行主编[约阿布·杰克逊](/author/joab/)。

[第 130 集:KubeCon EU 和僵尸工作量](https://thenewstack.simplecast.com/episodes/episode-130-kubecon-eu-and-the-zombie-workloads)

Intuit 是向个人和小企业提供金融软件和服务领域的巨头，也许毫不奇怪，该公司早期投资 Kubernetes 是为了更好地管理其庞大且极其动态的工作负载。该公司两年多前开始使用 Kubernetes，现在运行一个完全可操作的生产环境，有 200 多个集群支持 2，500 种服务，包括公司旗舰产品 properties、TurboTax 和 QuickBooks。

Wadher 精通该公司庞大运营的许多方面，因此我们有很多要谈的，包括:

*   在生产规模上使用 Kubernetes 的挑战。
*   Intuit 对 Istio 服务网使用情况的调查。
*   Intuit 在[的价值加入了](https://www.cncf.io/announcement/2020/08/18/cloud-native-computing-foundation-announces-intuit-as-gold-member/)云本地计算基金会。
*   Intuit 在 Argo 中的角色以及 Argo 为 Intuit 做了什么。
*   Argo 如何使用 [Flux](/gitops-made-simple-with-flux/) 来启用 GitOps。
*   红帽在 OpenShift 上的新 [Argo 操作符](https://developers.redhat.com/blog/2020/08/17/openshift-joins-the-argo-cd-community-kubecon-europe-2020/)。
*   Intuit 新的[基于 Kubernetes 的数据处理平台 Spok](https://www.youtube.com/watch?v=NQL4wmMEOZk) ？
*   对数千款应用进行多区域追踪的挑战。
*   今年 Kubecon EU 的外卖。

在展览的后半部分，我们讨论了 KubeCon 的亮点，包括 Holly Cummins 对环境友好型计算的讨论，Rancher 的 K3s 对边缘工作负载的使用的[增长，以及云计算原生计算基金会对最终用户的日益重视。](/kubecon-eu-surge-of-kubernetes-edge-deployments-leads-to-cluster-sprawl/)

[云本地计算基金会](https://www.cncf.io/)和 [KubeCon + CloudNativeCon](https://www.cncf.io/kubecon-cloudnativecon-events/) 和[红帽](https://www.openshift.com/)是新堆栈的赞助商。
