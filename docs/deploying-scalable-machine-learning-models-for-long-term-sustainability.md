# 部署可扩展的机器学习模型以实现长期可持续性

> 原文：<https://thenewstack.io/deploying-scalable-machine-learning-models-for-long-term-sustainability/>

[机器学习](https://thenewstack.io/category/machine-learning/) (ML)模型的承诺正在扩散企业，但将它们部署到云或边缘计算环境被证明是跨过多工具和框架扩展的一个重大挑战。根据 [Algorithmia 的](https://algorithmia.com/)“[2020 年企业 ML 状态](https://info.algorithmia.com/2020)”，平均而言，40%的公司表示将 ML 模型部署到生产需要[一个多月的时间。"](/add-it-up-how-long-does-a-machine-learning-deployment-take/)

在 [AWS re:Invent](https://reinvent.awsevents.com/) 录制的这一集 New Stack Makers 播客中，[OctoML](https://www.linkedin.com/in/luis-ceze-50b2314/)的联合创始人兼首席执行官 Luis Ceze 讲述了如何在任何硬件、云或边缘设备上优化和自动化机器学习模型的部署。

《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了本期播客。

[为长期可持续性部署可扩展的机器学习模型](https://thenewstack.simplecast.com/episodes/deploying-scalable-machine-learning-models-for-long-term-sustainability)

从自动化机器学习模型到设计、管理和优化，快速将这些模型交付到生产中，同时节省成本并长期维持这些模型的能力正在挑战开发人员的生产力。“机器学习模型是非常计算密集型的，一旦你准备好部署一个模型，它需要你了解硬件，并真正调整以优化它，为部署做好准备，”Ceze 说。

基于[Apache Tensor Virtual Machine(TVM](https://tvm.apache.org/))，一个由 Ceze 和他的联合创始人创建的开源机器学习编译器框架，OctoML 建立在这个工具上，使开发人员能够自动优化机器学习，并部署它们在任何硬件上大规模运行。“Apache TVM 所做的是创建一组跨各种不同硬件的通用原语…然后它在内部使用机器学习来生成高效的机器学习代码。这一点之所以重要，是因为准备部署模型的工作涉及大量手工软件工程，”Ceze 说。

Ceze 说，通过使用机器学习，Apache TVM 帮助公司简化部署机器学习模型所需的洞察力。“一旦你获得了一个机器学习模型，你就有数十亿种方法可以实际生成代表你的模型的代码，并在目标硬件上运行它。”策泽问:“你没时间把它们都跑完，怎么挑最快的？而且有几十亿，不实用。”

Ceze 说，由于复杂的工具链和支离破碎的生态系统，将机器学习模型投入生产的实践似乎将势不可挡，因为“模型经常更新，因为数据变化或更好的想法可以使模型更加准确和有效，无论它需要做什么，因此需要再次训练模型”。“如果没有自动化，每次模型发生变化时，您都必须进行手工操作，以便为交付周期做好准备。”
