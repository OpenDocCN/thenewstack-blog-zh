# 工程师创业:副业还是全职？

> 原文：<https://thenewstack.io/entrepreneurship-for-engineers-should-open-source-project-be-a-full-time-job/>

工程师的企业家精神是一种

[monthly column](https://thenewstack.io/entrepreneurship-for-engineers-from-open-source-to-monetization-profit/)

由 New Stack 的长期撰稿人 Emily Omier 撰写，探讨了希望为其他开发人员构建工具的开发人员所关心的问题——并围绕他们的创新建立业务。我们欢迎您对未来专栏的反馈和想法。

每年的这个时候，人们通常会想知道自己的职业轨迹，并思考新一年的目标。如果您是一个开源项目的创建者和/或维护者，您可能还想知道现在是否是将您的兼职开源项目变成全职工作或者在业余时间继续工作的时候了。

听取其他企业家的意见可能有助于做出这种决定，但当被问及如何确定某件事情是否已经成熟，可以从旁门左道转向商业时，我采访过的企业家通常会回答说，“我希望我知道。”

这是因为一个开源项目是否可以赚钱是一个问题，而一个特定的人是否能够实现这种转变——以及对那个人来说，这是否是当时最好的职业发展——完全是另一个问题。

“这取决于你想做什么，”云运营解决方案公司 Glasnostic 的联合创始人兼首席执行官托拜厄斯·昆泽说。“你有一份轻松的工作，只是想兼职做些有意义的事情吗？还是想让它成为你生活中更大的一部分？”

关键是追求一个绝对商业上可行的想法并不一定是每个人的正确选择。这取决于风险承受能力、你的责任和支出，以及创业听起来是刺激还是噩梦。

从副业转向商业需要[巨大的思维转变](https://thenewstack.io/entrepreneurship-for-engineers-adopting-the-founder-mindset/)。尽管如此，还是有一些事情需要考虑。

## 全职创业的途径是什么？

人们往往倾向于认为只有一种“正确的”方式来赚钱开源项目，或者风险投资的创业公司是唯一的途径。这不是真的——即使最终走上这条路的创始人也不总是首先这么做。

出售与开源项目相关的咨询和/或服务，甚至只是从依赖某个项目的公司那里获得赞助，是许多开源维护者开始的一种方式[。](https://thenewstack.io/entrepreneurship-for-engineers-bootstrap-or-take-vc-money/)

虽然不是每个人都认为这是一个好主意。

容器化应用服务交付平台 Portainer 的首席执行官兼联合创始人尼尔·克雷斯韦尔表示，试图通过支持或咨询来赚钱的自由开源软件项目并不是一个好计划。“这从来没有工作，”他说。

这是因为通常采用开源的工程师有技能——或者至少认为他们有技能——来养活自己。直到一个项目被广泛使用和/或投入生产，才需要支付支持费用。

## 能不能调整一下思路？

克雷斯韦尔提倡让开源项目成为你的商业产品的一部分，而不是让商业产品成为开源项目的一部分，而不是通过提供咨询来赚钱。

这似乎是一个微妙的转变，但从维护者/创始人的角度来看，这是一个思维方式的重大变化，因为它迫使维护者/创始人首先关注商业产品，而不是开源项目。

昆泽回应了这一点，他说，他从根本上认为，在大多数情况下，开源项目已经成为公司发起和维持的一种营销活动，以达到某种战略目标(其中之一是销售开源项目的商业味道)。

## 这个开源项目可以扩展吗？

并不是所有的项目都容易商业化，在迈出这一步之前，重要的是要清楚你的项目前景如何。

它离用户的关键任务能力有多近？对用户来说是必备还是好看？它在生产环境中是最有用的，还是个人开发人员使用的一种工具，不能用于生产环境？所有这些因素都有助于决定项目商业化的难易程度。

“有如此多的开源项目在商业中被大量使用，如 OpenFaaS，它们很难获得任何形式的商业补偿，”克雷斯韦尔说。因此，仅仅看到大量下载或明星数量的上升不足以作为你做出决定的依据。

## 现在是创业的时候吗？

比方说，你非常自信地想把你的开源项目变成一项全职事业。现在是时候吗？“一旦你看到足够的证据表明你的产品在商业环境中被接受，并且你知道他们为什么使用你的产品，那么就是认真对待的时候了，”克雷斯韦尔说。

这种反馈是创办开源公司的一个好处，而不是仅仅建立一个基于想法的公司。昆泽说，社区可以成为信息和支持的重要来源，也可以减少进入业务的摩擦。任何减少阻力的东西，尤其是在早期，都是好的。

## 你应该继续你的日常工作吗？

有些人被推着去开一家公司，看看会发生什么，但想要继续工作以获得稳定的薪水并没有什么本质上的坏处。这也不意味着你不能在开源项目上投入大量的时间。

有许多维护人员的例子，他们在项目上的工作基本上是由他们工作的公司赞助的，这给了维护人员在项目上工作和推广项目的自由，而不必担心创业的挑战。有时这项工作由内部开源项目办公室支持。

因此，如果你的目标是花更多的时间在开源项目上，但创业听起来并不吸引人，有其他方法可以让开源成为你职业生涯中更完整的一部分。
