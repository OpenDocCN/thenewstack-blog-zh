# Sysdig 推出了一个共享 Kubernetes 安全性最佳实践的中心

> 原文：<https://thenewstack.io/sysdig-launches-a-hub-for-sharing-best-practices-in-kubernetes-security/>

Portworx 赞助了 New Stack 对 KubeCon+CloudNativeCon 北美 2019 的报道。

随着各公司争相采用 Kubernetes，许多公司直到最后一刻才开始忽视安全性。但是，将安全性引入 DevOps 工作流并尽早这样做还有其他后果——也就是说，安全应用程序的更多责任在于开发人员自己，而不是一个单独的安全团队。

根据云本地安全公司 [Sysdig](https://sysdig.com/) 的说法，这造成了一个危险的知识缺口，因为一个普通的开发人员在安全最佳实践方面没有专家那么熟练。解决这个问题的一个方法是使用工具——这是 Octarine 的护栏产品的目标之一。但是 Sysdig，开源安全工具 Falco 的幕后公司，采取了一种稍微不同的方法:开源获得安全配置背后的技术

为此，该公司推出了 [Cloud Native Security Hub](https://securityhub.dev) ，这是一个分享 Kubernetes 安全最佳实践的社区网站。该网站于上周在圣地亚哥举行的 KubeCon+CloudNativeCon 北美 2019 年展会上首次亮相。

Sysdig 的首席营销官珍妮特·松田隼说:“真正的问题在于连接性。这种方法旨在降低现在负责配置安全性的开发人员(或任何非安全专家)的门槛。这也是安全专家分享知识、了解其他公司如何管理配置以及就一套标准最佳实践进行协作的一种方式。

“对很多人来说，问题是‘我如何开始？’”她说我们试图提供一个很好的用户界面，这样就不会令人生畏。

DevOps 团队可以访问安全中心，直接下载规则和配置文件，既可以按原样使用它们，也可以针对特定用例进行定制。目前，安全中心只有 Falco 规则，但计划是在未来随着社区的扩大和更多专家的贡献，为其他云原生安全工具提供支持。

Sysdig 的产品营销总监豪尔赫·萨拉梅罗·桑斯说:“我们的目标之一是公开讨论，让每个人都同意什么是最佳实践。

“[米特里·ATT&CK](https://attack.mitre.org/)框架被广泛接受，但是你如何将它映射到 Kubernetes？”松田隼解释道。不同的人可以有不同的解释，为他们提供一个协作和教学的场所可以帮助云原生生态系统在安全最佳实践方面走向成熟。提供易于共享的配置文件也可以节省大量时间。

如果单个开发人员和 DevOps 团队要负责安全性，他们需要工具来成功地完成这项工作。Sysdig 希望云原生安全中枢最终能够节省时间，创造出更安全的云原生应用。

KubeCon+CloudNativeCon 是新堆栈的赞助商。
