# 罗宾汉的库伯内特之旅:一条比看起来更危险的道路

> 原文：<https://thenewstack.io/robinhoods-kubernetes-journey-a-path-more-treacherous-than-it-appears/>

欢迎来到[新堆栈制造商](/podcasts/makers):攀登新高度，一系列对工程经理的采访，他们谈论他们面临的问题和他们寻求的解决方案，由嘉宾主持[Scalyr](https://www.scalyr.com/)CEO[Christine heck art](https://www.linkedin.com/in/christineheckart/)主持。

工程领导者面临的挑战决定了如何做出技术决策来支持快速增长的业务需求。

赫克特的第一次采访是与亚当·沃尔夫(Adam Wolff)进行的，他是罗宾汉(Robinhood)的前工程副总裁，回忆起两年前 Kubernetes 看起来如此正确的时候——以及随后当沃尔夫授权采用该平台时出现的困难。Robinhood 是一项在线投资服务，提供免费的金融交易所交易应用。

[罗宾汉的库伯内特之旅:一条比看起来更危险的道路](https://thenewstack.simplecast.com/episodes/robinhoods-kubernetes-journey-a-path-more-treacherous-than-it-appears)

“所以我阅读了 Kubernetes 的所有营销材料，”沃尔夫说。“我玩了一个演示。我当时想，‘这看起来太棒了。这似乎正是我们所需要的。"

现在，两年过去了，罗宾汉团队仍在努力向库伯内特迁移，沃尔夫说，他在采访录制完成后离开了罗宾汉。

“我们已经取得了很大进展，但这比我们预期的要困难得多，”沃尔夫说。“这有很多原因。我认为我们的团队并不是唯一发现这条道路比看起来更危险的团队。”

他说，回想起来，更好的做法是专注于如何用不变的基础设施实现更快的发布，特别是对于一个只有几百名工程师的公司来说。

“对于像 Robinhood 这样的公司来说，部署像 Kubernetes 这样的技术，不仅仅是一个团队，你有一个基础设施平台团队正在寻求部署这一点，你有一个安全团队需要就我们如何做到这一点以及潜在的问题达成一致，”Wolff 说。“然后你的产品团队需要决定采用这项技术是否值得。这些团队中的每一个都将为这个项目带来他们自己的目标、愿望和幻想。”

采用 Kubernetes 并不是 Robinhood 面临的唯一挑战。上周，[彭博报告称](https://www.bloomberg.com/news/articles/2020-10-09/robinhood-users-had-accounts-looted-say-there-s-no-one-to-call)许多客户报告账户受损，但他们找不到人来解决问题。在给彭博的一份声明中，罗宾汉表示，网络罪犯已经锁定了有限数量的账户。他们说，问题在于，客户用于罗宾汉账户的个人电子邮件是在罗宾汉平台之外被访问的。Robinhood 发言人表示，他们正在与受影响的客户合作，以更好地保护他们的账户。

今天，学到的课程是关于基础的，是工程师学到的基础。当然，工程很酷，新东西令人兴奋。但是工程并不领导业务。引领工程的是业务。
