# 开发者福音、结对编程和开源企业支持

> 原文：<https://thenewstack.io/pivotal-pair-programming/>

当今天有如此多的[开源选项](https://thenewstack.io/for-open-source-software-foundations-are-the-new-cathedrals/)可用时，为什么任何组织会选择为软件付费呢？据[迈克尔·科特](https://twitter.com/cote)、 [Pivotal](https://pivotal.io/) 软件技术营销总监说，对于较大的企业来说，这一切都归结于支持。

在这一集的[中，新堆栈制造商](https://thenewstack.io/podcasts/)嵌入下面，新堆栈创始人[亚历克斯·威廉姆斯](https://twitter.com/alexwilliams)以及共同主持人[李·卡尔科特](https://twitter.com/lcalcote)和[凯尔·麦克唐纳](https://twitter.com/kylemacdonald)与[科特](https://twitter.com/cote)谈论了开发人员传道者的角色变化，为什么公司选择企业支持的开源软件，以及 Pivotal 如何定位其支持产品，以帮助其客户通过结对编程取得成功。

[Michael Coté，Pivotal 软件总监，云铸造峰会技术营销](https://thenewstack.simplecast.com/episodes/michael-cote-pivotal-software-director-technical-marketing-at-cloud-foundry-summit)

Coté首先解释了他在 Pivotal 中作为传道者的角色，其核心是与开发人员和客户联系，讨论如何充分利用 Pivotal。Coté指出，这已经通过口口相传取得了指数级的回报。

他解释说，开发者传道者的角色不仅仅是推销品牌或产品。福音传播者不仅知道他们组织的来龙去脉，而且他们还参与面向客户的互动，用最新的技术连接社区，并从目前使用他们服务的公司寻求反馈。

“在过去的九个月里，人们对云铸造有了更多的普遍认识。我们注意到客户说得更多了。他们更愿意谈论正在发生的事情，因此，有更多的常规公司在谈论 Pivotal 和 Cloud Foundry，”他说。

Coté讲述了他与 Express Scripts 信息技术高级总监 Brian Gregory 的一次讨论。两人试图确定是否有一个行业不需要定制软件。对于那些这样做的公司来说，尤其是在大型组织中，更新一个人的技术堆栈就像瘟疫一样被避免了。

“这些庞大的组织一直在运行软件，但在过去的十年中，他们没有更新他们的软件层。他们需要。它们运行在破旧的堆栈上，从基础设施层一直到应用程序和交付层。十年前，获得完全集成的体验是昂贵的。现在，我们有了 Cloud Foundry 和其他替代产品，因此他们可以将这些东西拼凑在一起，构建一个新的堆栈。”

支持和维护是试图建立平台或创建新堆栈的企业通常需要的。Coté指出，无论企业是需要一次性支持，还是希望引入 Pivotal 团队成员来与他们配对程序，付费支持选项都允许企业获得开源项目可能无法提供的支持级别。

“我们将编程、设计、产品管理和运营结合起来。把它想象成你带来关键人物，他们和你配对。他们会观察、指导，最终，你会知道如何独立完成一切。”
