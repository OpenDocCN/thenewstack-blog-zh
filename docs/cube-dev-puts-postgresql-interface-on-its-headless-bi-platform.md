# Cube Dev 将 PostgreSQL 接口放在其无头 BI 平台上

> 原文：<https://thenewstack.io/cube-dev-puts-postgresql-interface-on-its-headless-bi-platform/>

云数据栈的崛起可谓一日千里。在短短几年内，它已经从初创公司和早期采用者的领域变成了全世界企业的首选。

它成功的关键在于它的灵活性和易用性。像[亚马逊红移](https://aws.amazon.com/redshift/)和[谷歌大查询](https://cloud.google.com/bigquery)这样的数据仓库使得在云中存储大量数据变得容易，而各种商业智能(BI)工具使得访问和可视化这些数据变得容易。这催生了新一代数据驱动的应用和功能，从小型企业到财富 500 强企业，每个人都在使用这些应用和功能。

本月早些时候，开源无头 BI 平台 Cube 的制造商 [Cube Dev](https://cube.dev/) 宣布增加一个 PostgreSQL 接口，这将允许熟悉 Postgres 和与之兼容的 BI 工具(实际上所有 BI 工具都是如此)的开发人员查询一个单一的、可消费的数据源，以轻松执行分析。

## 云数据堆栈和商业智能

BI 应用程序旨在汇总和分析大型数据集。它们被全球数百万的企业所使用。预计到 2022 年，BI 软件的全球市场将超过 250 亿美元，成为软件行业中最大的细分市场之一。虽然 BI 应用程序可以在内部运行，但它们通常托管在云中，这使得它们非常适合云数据堆栈。

Cube 支持 BI 工具，包括商业工具，如 [Tableau](https://www.tableau.com/) 和微软 [Power BI](https://powerbi.microsoft.com/) ，开源工具，如 [Apache Superset](https://superset.apache.org/) ，甚至是使用嵌入式分析平台的定制应用程序，来查询相同的数据模型。这意味着不同类型的查询都将使用相同的度量和维度定义，因为它们利用了单个可查询的数据源。

Cube Dev 的首席执行官兼联合创始人 Artyom Keydunov 表示:“让各种各样的数据消费者都能获得 Cube 的力量，这一直是我们的抱负。“数据需要一致和高性能，不仅对开发人员如此，对分析师和一般业务用户也是如此。这需要将 Cube 引入这些用户已经在使用的工具中。”

## 一次建模，处处查询

企业花费大量时间和金钱来配置员工将直接连接的数据源。同样的员工需要组织和协调这些资源，以便可以一起查询。这导致了重复的工作和每个相关人员缓慢的洞察时间。

无头 BI 解决方案通过抽象化所有个性化配置来消除这些问题，因此任何应用程序都可以利用公司的企业指标，而无需了解底层数据模式以及如何转换它们。数据模型中的定义可以在组织内的不同工具团队之间重用，这些团队可以使用相同的定义有效地合作，从而实现更高效的协作。

组织可以配置一次多维数据集来访问数据源、定义数据模型、缓存查询并充当数据的单一连接点。这导致开发人员和业务用户可以使用相同的定义，而不管他们可能使用的编程语言、工具或应用程序。这意味着“季度收入”或“净销售额”等重要指标在高管的仪表盘、营销人员的活动报告和财务团队的收入模型中意义相同。

## 开发者免费访问

Cube 的无头 BI 平台对 [Cube Cloud](https://cube.dev/cloud) 的用户开放，该平台提供可处理高达 1GB 数据通过的免费账户。Cube 在 GitHub 上也有[的开源形式。无论哪种方式，开发人员都可以从他们的代码中访问熟悉的 Postgres 接口，并可以将分析功能引入他们的应用程序。](https://github.com/cubedevinc/cube-private/blob/master/README.md)
