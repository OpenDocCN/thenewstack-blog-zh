# 补偿开源维护者，Tidelift 如日中天

> 原文：<https://thenewstack.io/compensating-open-source-maintainers-tidelifts-on-a-roll/>

总部位于波士顿的初创公司 Tidelift ，专注于“让开源软件更好地为每个人服务”，已经迅速扩张，并为应用程序开发团队增加了新的功能。

通过与客户的订阅模式，它使用支付的费用来补偿没有赞助商支持的开源项目的维护者。

[Tidelift 宣布](https://blog.tidelift.com/managed-open-source-tidelift-expands-to-1000-open-source-projects-launches-new-capabilities-for-teams):

*   订阅数据库监控 300 万个开源项目。
*   它为 1，000 多个最受欢迎的社区主导的开源项目提供保证，维护人员可以从 JavaScript、Python、PHP、Ruby、Java 和。网络生态系统。
*   订阅现在包括开源依赖分析，与开源维护者直接合作识别和解决安全、许可和维护问题。

“我们正在与大量社区主导的开源项目的创造者和维护者合作，以引入托管开源的概念，组织可以通过向 Tidelift 的参与维护者支付费用来节省时间和降低风险，以确保他们的软件包符合统一和全面的商业标准，”Fischer 说。

虽然公司通常不再在内部维护服务器或数据库，但 Tidelift 联合创始人兼首席执行官 [Donald Fischer](https://www.linkedin.com/in/donaldfischer/) 说，“在大多数情况下，应用程序开发团队仍然承担着自己维护开源组件的所有痛苦和精力”。

“我们也在努力让他们摆脱这种局面。它让您的开发人员能够开发您的应用程序，而不是其中的部分，但他们仍然有信心这些部分是安全的，许可是锁定的，他们得到了积极的维护，他们将继续工作。”

New Stack 分析师 Lawrence Hecht [之前报道过，糟糕的文档和不良的安全记录是公司不使用开源技术的主要原因。近 75%的开发人员每月花费超过 10 个小时来处理开源漏洞。](https://thenewstack.io/ossmaintainer_security/)

虽然维护人员辞职还为时过早，但 Tidelift 的薪酬正在增长。

“我们对单个开源包的最大支出大约是 30，000 美元，并且您可以维护多个包。…这真的是 Tidelift 的终结游戏…我们认为世界上应该有这种新的工作类别——全职开源维护者，你不只是在兼职，或者这是你日常工作的一部分。这应该是人们的日常工作，”费舍尔说。

新的软件工具包括安全漏洞、许可问题和跨依赖关系的技术问题的概述，帮助开发人员衡量软件包更新如何影响他们的应用程序的概览指标，以及关于何时升级关键框架和库的建议。

该公司正在提供一项免费服务，即[开源依赖分析器](https://tidelift.com/subscription/dependency-analyzer-top-unmaintained)，它使开发团队能够了解他们正在构建的应用程序中的依赖状态。通过共享其中一个项目的包管理器文件，Tidelift 将对它们进行分析，并对今天要解决的问题提供三个建议。

[micro match、Enquirer 和其他 JavaScript 库的维护者 Jon Schlinkert](https://github.com/jonschlinkert) 说，他不想花时间在筹款、记账或强调妥协、交易和获得赞助商的承诺会如何影响项目的路线图上。

“如果获得资金需要大量的时间和维护，超出了项目本身的要求，我只是在用一个可持续发展问题交换另一个，”他说。

" [Tidelift](https://tidelift.com/) 是我见过的第一个解决方案，它让我可以从我作为一名开发人员和维护人员所创造的价值中获得报酬，而不用担心筹资的问题和压力。我非常喜欢他们的模型。它没有压力，而且可扩展。”

今年 1 月，该公司宣布在 B 轮融资中获得 2500 万美元，此前 7 个月曾获得 1500 万美元的 A 轮融资。

Tidelift 是新堆栈的赞助商。
