# SXSW 的 Capital One:用多样性对抗确认偏差

> 原文：<https://thenewstack.io/new-stack-makers-sxsw-diversity-fights-cognitive-bias/>

“坚定的信念既能帮助我们，也能伤害我们。我们有一个明亮的光，我们正在跟随，我们相信这是要走的路。我们寻找并解读能让我们更接近光明的信息。但有时这是以考虑其他可能更亮的光为代价的，”Capital One Labs 的设计研究员 Allison Abbott 在上周本月早些时候的 SXSW 会议上在 Capital One 小组讨论会上发言时说。

雅培和 Capital One Labs 的首席产品经理[艾玛·萨根](https://www.linkedin.com/in/emma-sagan-1a47263a)都谈到了确认偏差和人类思维中的其他错误如何影响团队成员、他们的研究和创作过程。

心理学术语，[确认偏差](https://www.sciencedaily.com/terms/confirmation_bias.htm)(或证实性偏差)是人类解读信息以证实自己先入之见的倾向。

## 消除对更强团队的偏见

在他们的小组讨论中，Abbot 和 Sagan 讨论了与人们如何思考、选择性注意、如何看待和检查自己的偏见和微表情等相关的系统处理。在对抗确认偏差时，他们指出，人们并不总是正确的——即使他们知道避免偏差的“屡试不爽”的技巧。

Capital One Labs 在试图消除偏见时首先考虑的领域是多样性。“我们相信不同的思想和不同的思想家群体。萨根说:“拥有这些不同的群体将有助于消除确认偏见。当处理偏见时，企业通常会开始创建大型团队，希望更大的团队会自动等同于更多的多样性。然而，通常情况并非如此，尤其是当一大群人有着相同的偏见时。大型团队也会影响整个开发工作流程，因为个人会纠结于向谁汇报或与谁一起解决问题。

[资本一号的艾利森·阿博特和艾玛·萨根:确认偏差:创新的盲眼](https://thenewstack.simplecast.com/episodes/capital-ones-allison-abbott-and-emma-sagan-confirmation-bias-innovations-blind-eye)

Capital One Labs 开发了各种金融应用程序，旨在帮助消费者更好地管理他们的财务，未雨绸缪，甚至开发了[大数据学院](https://www.capitalonelabs.com/#/projects)——一个基于利用个性化数据力量的自我指导程序。

大数据学院还帮助 Capital One own 的开发人员利用自己的内部工具解决系统和基于云的解决方案中的问题。凭借更好地定制、跟踪和分析行业和用户数据的能力，Capital One Labs 希望为其团队成员提供真正深入概念背后的数字的机会。

Sagan 和 Abbott 指出，Capital One Labs 的做法是让许多团队同时工作。在任何时候，他们可能有一个团队与研究人员一起解决某个问题的核心问题。拥有与产品上市无关的研究人员有助于 Capital One Labs 团队确保重点完全放在研究上。“让另一个没有投资项目的人加入进来真的很重要。这让团队后退一步，弄清楚除了积极的方面，还应该关注什么，”萨根说。

## 形成想法:走向完美的过程

当头脑风暴新想法时，Sagan 和 Abbott 建议团队碰碰运气。虽然这看起来矛盾，但测试被认为是“极端”的解决方案可以让开发人员更好地确定底线在哪里。特别是当团队成员处于“最佳状态”时，提出想法并敲定如何分配时间和资源有助于更好地简化工作流程。

当测试“坏”的想法时，人们还必须记住，用户的反馈对于重新审视一个概念是至关重要的。Sagan 指出，如果一个开发人员抛出一个他们知道是极端的想法，而用户喜欢它，这将表明该过程的某些部分需要重新工具化。“归根结底，我们都关心自己做了什么。如果我们不相信我们在做正确的事情，我们就不会成为创新者，”阿博特说。

萨根和阿博特指出，所有的偏见都不是基于恶意的意图。也不是主动忽略可能让一些人不舒服的事情。相反，偏见是人类自身条件的结果。因此，开发新的方法来对抗并更加意识到自己的偏见是至关重要的。Capital One Labs 希望让它的团队不仅意识到自己的偏见，还能让他们有意识地成为更成功的产品开发人员。
