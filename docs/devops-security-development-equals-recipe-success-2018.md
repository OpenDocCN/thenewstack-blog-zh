# DevOps，安全和发展是 2018 年成功的秘诀

> 原文：<https://thenewstack.io/devops-security-development-equals-recipe-success-2018/>

马尼什·古普塔

Manish 是 FireEye 的首席产品和战略官，帮助该公司从大约 7000 万美元增长到超过 7 亿美元，产品组合从两种产品增长到 20 多种产品。此前，他是思科 20 亿美元安全产品组合的产品管理副总裁。在 McAfee，他担任副总裁/总经理，在五年内将网络安全业务从大约 1 亿美元增长到超过 5 亿美元。2005 年，他在 iPolicy Networks 担任产品管理副总裁，帮助创建了下一代防火墙类别。他拥有凯洛格管理研究生院的 MBA 学位、马里兰大学的工程硕士学位和德里工程学院的工程学士学位。

在对 2018 年的预测中，Forrester Research 称这将是企业[发展](/category/devops/)的一年。然而，DevOps 通过加速软件开发生命周期(SDLC)而带来的巨大创新也破坏了传统的 AppSec 方法。随着越来越多的组织采用 DevOps 来加速应用程序交付，将需要自动化来限制安全和开发团队流程之间固有的安全风险。

DevSecOps 的出现大部分是理论上的，并且经常遇到减缓发布速度的质疑。首先，经常会有来自应用程序开发团队的推诿，因为安全审查会延迟部署，并且安全工具会为应用程序开发人员制造误报的“噪音”。其次，传统的安全工具通常是为了从外到内保护应用程序而构建的:它们依赖人类知识来理解应用程序是如何工作的。事实上，使用正确的工具，优秀的开发人员可以通过消除手动流程来加快 SDLC 的速度。

企业必须学会将开发、安全和开发作为其商业文化的基本组成部分。然而，这必须伴随着对应用程序安全性的观念的改变。现代 DevSecOps 要求完全自动化将“安全性作为代码”[融入 CI/CD 流程](https://blog.shiftleft.io/enhancing-security-for-your-ci-cd-pipeline-72120b1e35a8)，并充分了解开发和生产周期，而不是依赖于固有的孤立的单个和手动步骤。

安全、DevOps 和开发团队将需要一个通用的安全基础，该基础可以与每个团队的所有工具和流程无缝集成。创建这样一个基础将有助于深入了解敏感数据应该如何流经微服务、开源库和外部 API，以及生产中实际发生的事情。

此外，通过了解应用程序应该做什么和它实际在做什么，CISOs 可以推动良性的反馈循环。运行时保护可以由既全面又精确的开发洞察力来驱动。由生产分析提供信息的开发可以优先处理哪些代码缺陷。

为了实现这种持续改进循环，公司必须努力打破由于期望值不匹配而存在的传统工作孤岛。以速度衡量，应用程序开发人员需要准确、早期地接收安全信息，并有足够的上下文来纠正软件开发过程中的问题。安全专家同样可以通过将他们的知识扩展到开发周期的更深处，在 DNA 层次上理解代码而受益。只有真正理解代码，运行时保护才能既全面又精确。

要实现这一点，需要巨大的文化转变。而文化就像航空母舰一样，不会很快改变方向。这并不是说传统的 AppSec 方法在今天没有价值。问题是传统的方法太慢，太不精确，这就是为什么当组织采用现代 CI/CD 时，它们越来越不有效。祝你建立一种文化，让每个开发人员和安全工程师彻底检查他们遇到的每一个警报。

DevSecOps 必须冲浪，而不是对抗潮流。他们必须采用现代 CI/CD 的速度，并通过在 SDLC 中构建自动化安全性来简化每个人的工作。这意味着通过构建时代码分析来实现运行时保护的自动化，而不是使用依赖手动干预的方法。正如 DevOps 运动的基础是将基础设施转化为代码一样，2018 年将是 DevSecOps 腾飞的一年，组织必须学会将安全自动化为代码。
