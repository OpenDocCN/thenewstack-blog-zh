# 新的谷歌平台整合了云服务和物联网

> 原文：<https://thenewstack.io/google-launches-iot-core-beta/>

在谷歌 I/O 开始的前一天，消息已经开始传出。今天，该公司为[谷歌云物联网核心](https://cloud.google.com/iot-core)开放了一个私人测试版，这是一个管理、更新和处理设备及其产生的数据的平台。私有测试版预计将持续到今年晚些时候正式发布。

[谷歌云](https://www.linkedin.com/in/indrachak/)[产品经理 Indranil Chakraborty](https://cloud.google.com/) 表示，物联网核心平台允许开发者和管理者利用谷歌的分析和机器学习能力来处理设备网络可能产生的大量数据。他认为这是谷歌物联网核心与微软上周在西雅图的 BUILD 会议上吹捧和扩展的类似平台[微软 Azure 物联网边缘](https://blogs.microsoft.com/iot/2017/05/10/microsoft-azure-iot-edge-extending-cloud-intelligence-to-edge-devices/)之间的关键区别。

“如果你看看谷歌云平台提供了什么，我们有最先进的技术、市场领先的服务和数据分析。 [BigQuery](https://cloud.google.com/bigquery/) 可以用来运行数千亿行和列，在几秒钟内得到答案。有了谷歌云物联网核心，现在它允许开发人员和客户安全地将他们连接到[谷歌云平台](https://cloud.google.com/)【GCP】，并真正使用我们最先进的分析和机器学习服务从设备中获得实时可行的见解，”Chakraborty 说。

他说，谷歌已经与客户合作，并召集了 10 个合作伙伴，以简化将 GCP 整合到物联网战略的过程。为此，一家欧洲公用事业提供商 [EnergyWorks](http://www.energyworks.com/) 经过短短几个月的开发，就能够实现大规模实时能源需求可视化。查克拉博蒂说，一些原型在几天内就完成了。

Chakraborty 说，Android Things 操作系统的出现使过渡更加容易。“我们也有 Android 系统，这是我们为边缘设备开发的操作系统。Android Things 是使用 Android 构建的，所以它带有 Android 的一些好处:它是一个由 Google 管理的安全操作系统。如果你是一个 Android 应用程序开发者，你也可以将 Android 应用程序集成到 Android 设备中。

“有了 Android，我们去掉了不需要的模块，也减少了占用空间。它与物联网核心无缝协作。你可以在边缘运行一个经过训练的 TensorFlow 模型，然后使用云物联网核心将数据发送回云中。你训练你的模型，降低训练过的模型，提高整体优势表现，”Chakraborty 说。

至于现有的物联网市场参与者，谷歌已经与 [IBM](http://www.ibm.com) 和[微软](http://www.microsoft.com)等公司纠缠不清，但 [GE](https://www.ge.com/) 也在物联网领域，其商业模式与那些软件公司非常不同。Chakraborty 表示，他将通用电气视为潜在的合作伙伴，而不一定是竞争对手。

“如果你想想我们在谷歌做什么，我们真的是一个平台玩家。我们确实提供基础设施，我们在数据分析方面有多层基础设施，物联网核心使您的设备可以轻松连接到 GCP。当谈到像通用电气这样的公司时，我们将它们视为合作伙伴。通用电气带来了什么；他们有很多关于涡轮机的机构知识。我们确实将大公司视为合作伙伴，我们可以携手合作，他们带来他们的机构知识，我们带来我们的机器学习和可扩展的基础设施，为客户构建成功的应用程序，”Chakraborty 说。
