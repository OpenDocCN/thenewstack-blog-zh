# 首席技术官寻求志愿者帮助美国政府的新冠肺炎工作

> 原文：<https://thenewstack.io/ctos-seek-volunteers-to-help-the-u-s-governments-covid-19-efforts/>

来自脸书和凯鹏华盈(Kleiner Perkins)等组织的一群技术领导者发起了一项志愿者倡议，寻找 IT 专业人员来建立和运行这项技术，以帮助美国联邦、州和地方政府应对新冠肺炎疫情。

这项名为[美国数字响应](https://usdigitalresponse.org)的努力将把数据专家、工程师、设计师和医疗专家与需要帮助的政府官员匹配起来。到目前为止，他们已经招募了 1000 名志愿者。

这项倡议[是由一些前联邦首席技术官创立的](https://www.usdigitalresponse.org/about/)，他们一直致力于帮助政府进入数字时代:前美国副首席技术官[詹妮弗·帕尔卡](https://twitter.com/pahlkadot)，美国代码的创始人；[柯里·查莱克](https://www.codeforamerica.org/)，现就职于乔治城大学[贝克社会影响与创新中心](https://global.georgetown.edu/georgetown_units/beeck-center-for-social-impact-and-innovation)；瑞安·潘查德萨拉姆，他现在是凯鹏华盈的技术顾问。脸书和 Stripe 的前工程主管瑞琳恩·容也是联合创始人之一。

对志愿服务感兴趣的科技专业人士以及寻求帮助的政府官员被要求填写一份简短的表格。然后，USDR 团队将审查志愿者，并根据项目需要匹配他们的技能。

政府机构[可以要求](https://www.usdigitalresponse.org/faqs)整个团队，而不需要由政府机构本身雇佣(尽管机构本身可能有采购政策需要考虑)。在大多数情况下，不需要资金。志愿者将使用他们自己的设备，因为 USDR 预计大多数工作本质上都是远程的。

USDR 网站列出了两个正在进行的项目。

一个是 N95 项目，国家新冠肺炎医疗设备信息中心。该倡议与有能力生产个人防护设备(PPE)如口罩的制造商合作，并与政府合作确定需求和优先次序。

第二个项目，邻居快递，将帮助那些需要帮助的人提供食物和帮助。目前正在加利福尼亚州康科德市进行，在“车轮上的饭菜”的帮助下，该倡议希望将这一平台推广到其他地方。
