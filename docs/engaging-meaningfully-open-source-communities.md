# 有意义地参与开源社区

> 原文：<https://thenewstack.io/engaging-meaningfully-open-source-communities/>

随着开源软件的不断发展，大大小小的组织都开始在自己的平台上实现开源项目和服务。无论是利用开源来改进他们自己的产品，还是帮助最终用户实现他们自己的商业目标，开源贡献和社区已经证明了它们的持久性。

在本期的 [The New Stack Makers](https://thenewstack.io/podcasts/) 播客中，我们将探讨开源社区是如何长期维持的，项目如何将贡献与他们自己的发布路线图结合起来，以及组织如何授权和培养他们自己的开发人员、运营人员和工程人才。新堆栈技术编辑 [Benjamin Ball](https://twitter.com/benballjr) 与 [Hazelcast](https://hazelcast.com/) 高级解决方案架构师 [Viktor Gamov](https://www.linkedin.com/in/vikgamov) 和 [Aerospike](http://www.aerospike.com/) 产品副总裁 [Alvin Richards](https://www.linkedin.com/in/alvinrichards) 交谈，听取他们对这些主题的想法以及今年早些时候在 2016 年 OSCON 期间的更多想法。Hazelcast 提供了一个开源的数据网格，而 Aerospike 提供了一个开源的 NoSQL 键值存储。

[有意义地参与开源社区](https://thenewstack.simplecast.com/episodes/engaging-meaningfully-in-open-source-communities)

当构建一个[开源社区](https://thenewstack.io/for-open-source-software-foundations-are-the-new-cathedrals/)时，项目维护者应该在他们所处的过程中会见贡献者，同时理解不是每个贡献者在他们如何与开源项目交互方面都有相同的技能水平或兴趣。理查兹解释说:“我认为有一种能够与你的社区互动的两种方法。有更深、更黑暗的东西更难贡献，但你会得到更有回报的贡献，然后有第二个流更容易整合。这吸引了更多的人，通常是因为他们有特定的问题需要解决。”

当在他们的内存数据网格上工作时， [Hazelcast](https://hazelcast.com/) 的工程师们不仅考虑了它如何作为一个整体与社区合作，还考虑了来自大型企业的贡献将如何影响它的发布路线图。在一些开源项目中，贡献者经常会采取一种态度，期望他们的每一个拉取请求或合并都能毫无疑问地被项目接受。

对于一些组织，特别是那些从其他企业获得大量捐款的组织，这可能无法实施。“你可以找到各种方法来确保人们了解正在发生的事情，并且他们有能力感受到他们有发言权，他们的声音被倾听。理查兹说:“你最终会得到一个巨大的工具箱，里面有各种各样的焊接方法，适用于社区的不同部分、流程的不同部分和开发周期。

Gamov 专注于与其他开发人员处于同一水平，希望通过参加 OSCON 等活动和通过演讲分享知识来支持整个社区，并指出，“最重要的是:我认为聪明人不会让其他人失望，而是让他们进步。”
