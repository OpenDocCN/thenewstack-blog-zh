# 云安全 2021:不容错过的 4 大趋势

> 原文：<https://thenewstack.io/cloud-security-2021-4-key-trends-not-to-miss/>

[帕洛阿尔托网络公司的 Prisma](https://www.paloaltonetworks.com/prisma/cloud) 在 2020 年 2 月 11 日举行的[云本地安全直播，2020 年虚拟峰会](https://register.paloaltonetworks.com/prisma-cloud-native-security-virtual-summit)之后，赞助了这篇文章。

云服务已经成为一个价值数十亿美元的产业——这个新兴市场的增长没有减弱的迹象。随着 [Gartner 预测【2020 年公共云收入同比增长 17%,很明显，世界正朝着所有行业更广泛采用云的方向发展。](https://www.gartner.com/en/newsroom/press-releases/2019-11-13-gartner-forecasts-worldwide-public-cloud-revenue-to-grow-17-percent-in-2020)

展望 2021 年及以后，我们可能会看到越来越多的企业(甚至是政府)将工作负载迁移到云上。

当然，向云的转移也意味着完全重新设想和优化他们的堆栈，以实现云原生实施。这意味着，在未来的几年中，一些新的安全趋势可能会占据主导地位。

为了确保您为快速到来的云原生未来做好准备，IT 和安全领导者应该关注以下四个云安全趋势。

## 云安全整合将继续发展。

Matt 拥有近二十年的安全领导经验，目前是 Palo Alto Networks 公共云的首席安全官。他经常在 RSA 等行业活动上发表博客和演讲。他目前领导着云威胁团队，这是一个专注于公共云问题的安全研究人员精英团队。他还是罗格斯大学网络安全证书项目的顾问委员会成员，也是 IANS Research 的成员。](https://www.linkedin.com/in/mattchiodi/) 

组织了解使用数十种安全产品来管理云基础架构的内在风险。即使是一个小型组织也可能管理 15-20 个独立的安全工具。另一方面是一些最大的组织，他们经常发现自己对管理 120 多种独立的安全产品感到疲惫不堪。

安全团队渴望能够为他们的云环境提供清晰的风险解决方案。像 Azure、谷歌、阿里巴巴和 AWS 这样的云提供商都有自己的原生安全功能，但这些新生产品在能够保护的方面受到限制，并且往往没有很好地集成。除了自己的私有云或内部硬件之外，大多数组织还使用多个云服务提供商。这种混合运营模式是云服务提供商尚无法解决的安全问题。

展望未来一年，我们将开始看到云原生安全平台(CNSPs)的崛起。这些平台将使安全团队能够从单一控制台管理公共云、私有云和混合云中的所有安全。孤立的安全解决方案将集成到这些全面的平台中:治理和法规遵从性等功能；威胁检测和响应；集装箱安全；数据丢失防护和无服务器安全性都将存在于单个控制台中。请关注这些平台在未来两年内的发展和成熟。

### 1.面向安全的机器学习(ML)将不再是一个时髦词汇

机器学习长期以来一直是科技领域最热门的词汇之一。许多营销宣传将其定位为几乎可以应对任何挑战的近乎神奇的解决方案。然而，在实践中，ML“解决方案”还没有提供许多安全的实际应用。

在接下来的 18 个月里，这种情况将会改变，因为我们开始看到在一些非常具体的用例中部署 ML 的例子。它将开始出现在数据分类、恶意软件检测和自动推理等领域，即可以像攻击者一样从多个不同角度评估安全配置的 ML。身份和访问管理领域就是一个很好的例子。将利用自动推理来确定政策是否有效、是否过度许可等。

### 2.随着开发速度的提高，安全团队将被迫左移

大多数组织都明白左移的战术优势，但采用却滞后了。然而，随着云安全整合的进展和 CNSPs 变得更加普遍，安全团队和他们的 DevOps 同行将有一条更加可行的左移之路。这些平台将使组织能够将安全性无缝集成到开发流程中，并确保跨所有云的云无关保护。

例如，许多组织正在快速采用基础设施即代码(IaC ),因为他们试图在云中自动化更多的构建流程。当团队转移到 IaC 时，他们可以避免手动创建和配置基础设施，而有利于编写代码——IaC 模板是一致地实施安全标准的一种很好的方式。CNSPs 可以扫描 IaC 模板以发现开发流程中的问题，还可以提供自动补救，以快速解决云环境中的任何安全问题。

然而，今天的挑战是 DevOps 团队没有审查 IaC 模板的错误配置。最近的 Unit 42 研究表明，虽然 IaC 为安全团队提供了一种可预测的方法来执行安全标准，但这种强大的功能在很大程度上仍然是未知的。在同一份报告中，Unit 42 的研究人员发现近 200，000 个不安全的 IaC 模板正在使用中。

### 3.供应商整合将扩大可见性

多云策略的广泛采用带来了可见性问题。随着我们进入新的十年，安全团队和 IT 领导将继续寻找方法来获得他们曾经在内部享有的可见性。高度动态的云资源使得可见性变得困难。

然而，利用云提供商 API 的安全平台将为基于云的工作负载提供更丰富的环境，包括容器、无服务器和 PaaS。其中很大一部分将由较小或单一功能供应商的整合推动(例如，提供云工作负载保护或数据丢失预防的提供商)。到 2021 年，这些利基供应商可能会发现自己在收购或清算之间做出选择，因为市场开始需要集成平台而不是单点解决方案。

### 4.立即为云原生安全性的未来做好准备

随着我们进入云安全的新时代，毫无疑问，公共云平台将继续以极快的速度发展。这种前所未有的变化速度将迫使组织不断反思他们构建和管理应用程序的方式。云提供的可访问性和易用性打开了自由创新的大门，使业务线所有者能够快速行动并提供巨大的价值。

这种可访问性和易用性也产生了负面影响，使得许多传统的安全和治理模型几乎过时。因此，安全团队需要重新评估现有的流程和工具，以确保他们能够跟上风险和业务在云原生时代的需求。
