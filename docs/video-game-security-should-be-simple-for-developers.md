# 视频游戏的安全性对开发者来说应该很简单

> 原文：<https://thenewstack.io/video-game-security-should-be-simple-for-developers/>

视频游戏继续[流行](https://en.wikipedia.org/wiki/Video_game_industry)，游戏中潜在攻击媒介的数量也继续增加。在最新的新堆栈制造商播客中，联合主持人[阿历克斯·威廉姆斯](/author/alex/)(TNS 的发行人和创始人)和[巴拉特·巴特](https://www.linkedin.com/in/bharatbhat) ( [Okta](https://developer.okta.com/?utm_content=inline-mention) 开发者关系营销主管)讲述了为什么以及如何让视频游戏平台和连接更加安全，嘉宾 Okta 高级开发者倡导者[尼克·甘布。](https://www.linkedin.com/in/nickgamb)

[视频游戏安全对开发者来说应该很简单](https://thenewstack.simplecast.com/episodes/video-game-security-should-be-simple-for-developers)

游戏行业经常成为一些行业内最伟大的编程人才的展示平台。作为一个恰当的例子，[约翰·卡马克的](https://en.wikipedia.org/wiki/John_Carmack)支撑“毁灭”的 C++代码被认为是编程的历史伟人之一，不仅仅是对游戏，而是对整个软件。对于 Gamb 来说，在成长过程中，在研究这些游戏的代码之前玩“地震”和“毁灭战士”是他进入软件行业的切入点。

甘布说:“这些引擎的源代码一发布，我就开始把(游戏)拆开，对东西进行逆向工程，弄清楚它们是如何工作的，并在很小的时候就把我自己的东西加入其中。“这实际上是我学习编码的方式，而且我后来做的几乎所有软件开发都是建立在这个基础上的。”

尽管不断吸引优秀的软件工程人才，视频游戏公司生产的软件并不像它应该的那样安全。身份验证过程实现起来可能很复杂，尤其是在给定游戏中可能有数百万个连接需要同时管理的情况下。对于拥有多个游戏的大型游戏发行商来说，连接数量以及潜在的攻击媒介可能会呈指数级增长。与此同时，开发人员通常也将用户的性能作为他们的首要任务，就像许多领域的开发人员一样，所以安全性通常是事后才想到的。

Gamb 说，与 Gamb 互动的大多数游戏引擎都缺乏“一个内置的身份安全概念”。“他们有一些非常通用的建议，但归根结底，没有内置的东西，可用的东西也没有真正遵循安全最佳实践，这是一个很大的问题。Gamb 说:“开发人员通常会尝试从其他不起作用的开发环境中翻新概念，他们使用有限或不完整的开源解决方案，或者试图在很少或没有专注于游戏的学习资源的情况下自己构建它。“所以，这就是我所看到的一个大问题:人们被放任自流。”

考虑到认证是“一件复杂的事情”，Okta 的使命之一是“使其更容易”构建游戏平台和开发的安全性，尤其是认证。Bhat 说:“你承担的风险更小，我们让它变得更容易，因为你不必真正深入地学习所有这些协议。”
