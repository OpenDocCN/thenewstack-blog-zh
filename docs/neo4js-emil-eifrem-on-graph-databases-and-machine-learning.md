# Neo4j 的 Emil Eifrem 谈图形数据库和机器学习

> 原文：<https://thenewstack.io/neo4js-emil-eifrem-on-graph-databases-and-machine-learning/>

[Neo4j 的 Emil Eifrem 关于图形数据库，机器学习，还有更多](https://thenewstack.simplecast.com/episodes/neo4js-emil-eifrem-on-graph-databases-machine-learning-and-more)

当谈到机器学习和 GraphDB 数据库系统时， [Neo4j](https://neo4j.com) 的首席执行官兼创始人 [Emil Eifrem](https://www.linkedin.com/in/emileifrem/) 说，“我们正处于九局游戏的第一局”。  “你看机器学习算法，是用图形语言写的，或者可以用图形语言表达。”

在今年早些时候的 GraphTour 活动中，Eifrem 在旧金山的 Fort Mason 发表讲话，在这一集的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中谈到了图形数据库，它们与关系数据库有何不同，以及这项十年前的技术如何跟上新技术的步伐。

图形数据库是关于连接数据的，这样就可以通过它与其他数据的关系来访问它。Neo4j graph 数据库最初是在瑞典的一家小初创公司看到一个特定的痛点而产生的想法。  “我们一半的工程师把时间花在了与关系数据库的斗争上，”他解释道。突然间，数据库从加速器变成了绊脚石。  解决方案是远离关系数据库模型。

“我们意识到，如果我们只有非常简单的节点构建块，这些节点之间的关系，以及可以附加到节点和关系的键值对，那么您就可以对一切进行建模。”

了解到他们需要什么，他们搜索 Alta Vista(一个早期的搜索引擎)，但是没有找到。所以他们说“管它呢，我们就建吧。  能有多难？”

15 年后，事实证明从零开始建立一个数据库是很困难的，这就是为什么他们在这个领域遥遥领先的原因。
