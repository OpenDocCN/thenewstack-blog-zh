# 背景:eBPF 如何为高级集装箱监控奠定基础

> 原文：<https://thenewstack.io/context-how-ebpf-sets-the-stage-for-superior-container-monitoring/>

[eBPF 如何为高级集装箱监控做准备](https://thenewstack.simplecast.com/episodes/how-ebpf-sets-the-stage-for-superior-container-monitoring)

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论来自云原生社区的最新新闻和观点。本周，我们将讨论如何使用扩展的 Berkeley 数据包过滤器(eBPF)来改进容器监控。

参加这次讨论的有[丹妮拉·庞特斯](https://www.linkedin.com/in/daniellapontes/)，她是 [InfluxData](https://www.influxdata.com/) 的产品营销高级经理，还有[卢卡·德里](http://luca.ntop.org/),[ntop](https://www.ntop.org/)网络监控工具的创始人。Deri 和 InfluxData 合作开发了一个集装箱监测工具，以 eBPF、ntop 和 Influx DB 时间序列数据库系统为基础。

庞特斯和德里在 TNS 最近的一篇文章中写道:“容器时代的 IT 监控:利用 eBPF 的可观察性”他们写道，在容器和微服务的世界里，只从外部监控服务是行不通的。相反，IT 团队应该直接读取系统指标，以深入了解容器内部和容器之间的活动。eBPF 是获取关于容器上运行的工作负载的系统信息的一种方式。

在节目的后半部分，我们将讨论最新的新闻和播客。我们重点介绍了[最近的一个播客](/how-to-build-open-source-sustainability/)，由 VMware 的 Bryan Liles 和 Cloud Native Computing Foundation 的 Dan Kohn 录制，内容是关于支持开源社区。我们讨论了[最近发布的 Nats 2.0](/nats-2-0-like-containers-for-messaging/) ， [Box 通过 Kubernetes 和 GitOps](https://thenewstack.io/the-box-way-to-microservices-kubernetes-devops-gitops-and-the-desired-state/) 向微服务的转移，以及 [HashiCorp 的 Consul 被扩展为全服务 mesh](https://thenewstack.io/hashicorps-consul-adds-layer-7-to-become-a-full-fledged-service-mesh/) 。我们还讨论了 IBM 对 Red Hat 的收购。

TNS 的编辑和营销总监 Libby Clark 在 New Stack 的创始人兼出版商 Alex Williams 和 New Stack 的执行主编 Joab Jackson 的帮助下主持了这次播客。

云原生计算基金会、KubeCon + CloudNativeCon、InfluxDB、Linux 基金会、Red Hat 和 VMware 是新堆栈的赞助商。
