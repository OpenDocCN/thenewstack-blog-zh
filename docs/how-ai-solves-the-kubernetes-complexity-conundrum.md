Kubernetes 在企业 IT 从过去的独石向今天更加动态的微服务、容器和云计算基础设施转变的过程中，无疑赢得了关键角色的声誉。

但这种转变是一把双刃剑。

一方面，Kubernetes 是加速企业数字化转型的主要催化剂，为构建、部署和管理应用和云提供了新的机遇。在 2018 年[云原生计算基金会(CNCF)](https://www.cncf.io/) [对 5000 家企业](https://www.cncf.io/blog/2018/08/29/cncf-survey-use-of-cloud-native-technologies-in-production-has-grown-over-200-percent/)的调查中，40%的企业(5000 多名员工)表示他们在生产中运行 Kubernetes，58%的受访者在生产中使用它。在 CNCF 的研究中，Kubernetes 也大大超过了其他容器管理工具，被 83%的企业采用。

但另一方面，Kubernetes 也引入了新的复杂层次。这意味着开发运维团队和 IT 部门将更少的时间和资源用于推动新的主动创新，而将更多的时间和资源用于被动解决问题以及在比以往任何时候都更加动态和不可预测的环境中灭火。

Kubernetes 为企业提供了一个宝贵的新工具，将数字化转型和上市时间表推向新的阶段。但为了充分利用它，it 和 DevOps 需要推动向人工智能和自动化的文化转变；大多数其他现有的 IT 方法根本无法在这个新世界中充分工作或扩展。

## Kubernetes 带来了什么

Kubernetes 的本机集群和节点健康数据有助于更全面地描述复杂的云环境。这些数据提供了更精确的答案和对多云环境的洞察，为更快的创新创造了机会，并加快了新产品、新服务和新部署的上市时间。

但是，在我所见过的企业 Kubernetes 环境中，有数十亿的相互依赖性需要考虑。如此庞大的数据量让它比以往任何时候都有更多的工作要做，但对于人类来说，这也是一个无法在瞬间理解的信息量。这产生了连锁效应，损害了从性能到用户体验再到生产部署速度的方方面面。

但是，将 Kubernetes 中编排的云原生工作负载与 AI 以及围绕它的 AIOps 文化结合在一起，为企业 it 提供了一种释放 Kubernetes 所提供的全部潜力的方法。

### 将 IT 文化推向人工智能方向

我一直认为 DevOps as 是关于文化的。不仅仅是在改变 IT 文化以弥合开发和运营双方之间的差距方面，而是在进行文化变革以将人工智能恰当地集成到 IT 工作流中。

人工智能使它能够更快地解决问题。为什么这对管理 Kubernetes 部署很重要？因为人工智能可以理解云环境和应用程序中更详细的数据，这是任何人都做不到的。当你有一个像 Kubernetes 这样动态和复杂的环境时，能够像 AI 一样快速和自动地筛选它对于充分利用 Kubernetes 为企业所做的事情至关重要。

想想人工智能带来了什么:

*   实时检测出现的问题及其根源。
*   自动触发补救流程，快速彻底地解决这些问题。
*   自动执行相互依赖关系映射和全栈监控，以解放 IT 和开发人员，让他们更专注于在生产环境中构建和部署新功能。
*   使开发人员能够进一步将自动化工具构建到他们的管道中，加快新构建的部署，并加快增强这些构建的反馈周期。

确定性人工智能已经针对未来的现代混合云应用进行了优化，可以处理人类无法处理的数据，而支撑这一切的 AIOps 文化变革对于实现这一切至关重要。

IT 和 DevOps 在他们的生活中最不需要的就是更多的复杂性。没有人工智能，来自 Kubernetes 的环境复杂性决定了它和 DevOps 的工作方式，而不是相反。具有讽刺意味的是，艾把权力放回了人民手中。

### **用“确定性人工智能”释放 Kubernetes 的全部潜力**

AI 和 Kubernetes 相辅相成。人工智能触发对动态环境的快速、自动、实时的洞察——绘制出应用程序、微服务和容器之间的相互依赖关系。这有助于更快的创新、更明智的决策、成功的 Kubernetes 部署、优化的用户体验和更强的业务成果。同时，将 Kubernetes 集群数据、节点健康数据和利用率指标与确定性人工智能结合在一起，使人工智能更加智能，使其在优化 Kubernetes 工作负载和加速云创新方面处于更有利的地位。

Kubernetes 本身会给已经不堪重负的 IT 和 DevOps 团队带来更多的复杂性。但是，通过将 Kubernetes 与确定性人工智能和围绕它的 AIOps 文化集成在一起，这些团队可以绕过这种不必要的复杂性，并利用 Kubernetes 部署来推动新的数字转型和云创新机会。
