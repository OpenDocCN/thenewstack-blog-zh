# 利用开源许可的指南

> 原文：<https://thenewstack.io/a-guide-to-leveraging-open-source-licensing/>

从最广泛的意义上来说，开源可以指你能读懂的任何代码。但是开源软件通常指的是自由代码和公开工作的行为。对于《纽约时报》的高级工程经理马里恩·戴利来说，这是关于以许可或版权许可的方式发布你的源代码。这是你开放整个过程的地方，包括让公众对你的代码库和文档做出贡献。

“你可以发布你的源代码而不收取任何费用——但那时你并没有真正从中受益，”Daly 说。

在本期[新堆栈制造商](https://thenewstack.io/podcasts/makers)播客中，我们将与 Daly 探讨如何超越开源这个时髦词汇，走向真正的公共协作，收获随之而来的所有业务收益。

[马里恩·戴利——利用开源许可](https://thenewstack.simplecast.com/episodes/marion-daly-leveraging-open-source-licensing)

首先，尽管开源绝对是一个不错的公关举措，但为什么一家公司要冒着商业秘密公开的风险呢？如果它是开源的，难道人们不能以你不希望的方式使用它吗？

Daly 认为开源是一个极好的不对称策略，它(开始)使大型软件公司和可能走向全球的小型创业公司之间的竞争更加公平。

她还表示，开源软件使得专利和法律审查更加容易。拥有一个好的专利会保护你免于一事无成。在这次采访中，Daly 分解了两种开源软件专利，它们以不同的方式允许人们和组织阅读和使用您的开源代码:

*   Copyleft 许可证:有点像版权的反义词，该许可证要求对 copyleft 代码的任何使用或修改也必须发布 copyleft。因此，你不能关闭你在 Linux 上构建的东西的源代码，因为它有 GNU 通用公共许可证。
*   **许可自由软件许可证:**要求你对原始代码的任何改变必须是开源的，但是它允许你在它的基础上构建一些封闭的源代码，甚至从中获利。麻省理工学院许可是一个很受欢迎的例子。

Daly 认为许可通常在许多企业中更受欢迎，因为 copyleft 可能更困难，尤其是在与第三方集成时。但是，如果你真的担心有人窃取你的作品，她建议选择 copyleft，因为这样“没人能拿走你的作品并为此收费。”

但是你不需要全身心投入到某种开放或封闭的资源中。例如， [Chromium](https://www.chromium.org/) 是许可的，而谷歌 Chrome 浏览器(主要基于 Chrome)是专有的。另一方面， [Mozilla 公共许可证](https://en.wikipedia.org/wiki/Mozilla_Public_License)被认为更“弱左”，介于版权所有和许可之间，而 Mozilla 的 foxy 标志仍然是专有的。

说到这一点，Daly 建议您尽量灵活使用许可，因为您的需求可能会发生变化。这一集的《新栈制造者》继续深入探讨伟大和不那么伟大的开源之旅的例子，以及优化您自己组织的体验的方法。
