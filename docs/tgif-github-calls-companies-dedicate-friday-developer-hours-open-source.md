# TGIF: GitHub 呼吁开发者将他们的周五奉献给开源

> 原文：<https://thenewstack.io/tgif-github-calls-companies-dedicate-friday-developer-hours-open-source/>

本周，GitHub 启动了一项名为[开源星期五](https://opensourcefriday.com/businesses)的新计划，这是一个结构化的社区计划，旨在鼓励人们在星期五的工作时间定期为开源做出贡献。该方案为定期缴款提供了一个框架，并为雇主提供了资源。

这个项目[源于 GitHub 内部鼓励周五开源工作的努力](https://github.com/blog/2386-contribute-on-open-source-friday)，以及本月早些时候发布的 GitHub[开源调查](https://opensourcesurvey.org/2017/)的结果。一个关键的发现是，几乎所有的受访者都在他们的工作场所使用开源软件，而且绝大多数人(85%)都有兴趣回馈——但不清楚在他们目前的工作场所如何做。

调查结果量化了我们已经直观理解的内容:越来越多的企业用户依赖开源解决方案。(W00t！开源赢了！)幸运的是，许多公司意识到，通过允许甚至鼓励员工在工作时贡献开源工作来回报操作系统生态系统有助于实现良好的业务。为你已经使用的软件做贡献并不是无私的——这是对你公司所依赖的工具的投资。

但调查结果也表明，许多雇主缺乏清晰明确的政策和知识产权协议，不知道到底允许哪种贡献，以及在什么条件下。因此，作为第一步，参与公司——甚至个人——可能应该在前进之前考虑 GitHub 的[平衡员工知识产权协议](https://github.com/github/balanced-employee-ip-agreement)。

[![](img/705993441862ac87d797eae1e326cd10.png)](https://github.com/blog/2386-contribute-on-open-source-friday)

Open Source Friday 提供了一个结构化的、程序性的框架来参与开源，一次几个 TGIF 小时。首先是 GitHub 的[如何为开源做出贡献](https://opensource.guide/how-to-contribute/)指南，它向初学者和老手强调开源需要的不仅仅是代码贡献。设计师、作家、活动策划人——这需要整个社区的努力来孵化，然后维护一个繁荣和成功的开源项目。像[你的第一个拉式请求](https://yourfirstpr.github.io/)这样的资源为绝对 n00bs 铺平了道路，而[对一个开源项目的剖析](https://opensource.guide/how-to-contribute/)解释了它们是如何组合在一起的。

从那里，那些想加入开源星期五乐趣的人可以遵循 GitHub 的便利指南来选择最好的项目。专业建议包括重要但有时被忽视的尽职调查，比如确保项目在投入之前积极接受新的贡献。检查拉取请求是否真的得到了审查，维护人员是否对问题做出了有益的回应，以及围绕开源工作的社区是否友好和受欢迎(专业提示:大量的激烈争论是反对加入项目的一个巨大的危险信号，除非这种事情是你的乐趣所在)。

简而言之，开源星期五程序拥有一切。一步一步的牵手给那些一直想做贡献却不知道如何做的人。一个框架，让经验丰富的开源专家来管理甚至增加他们的贡献。该计划有助于跟踪和共享开源的周五特定贡献，甚至有材料/资源来帮助您说服您的雇主允许您的部门在周五下午在您的日常工作职责之外从事 cool OS 项目工作。有点像开源的欢乐时光。

一旦老板们听说像[谷歌](https://www.google.com/)、[微软](https://www.microsoft.com/)、[脸书](https://www.facebook.com/)、[康卡斯特](https://www.xfinity.com/)、[特拉维斯 CI](https://travis-ci.org/) 和 [thoughtbot](https://thoughtbot.com/) 这样的公司都鼓励员工开发者做开源工作，作为有偿雇佣的一部分，嘿，他们甚至可能会买啤酒。
