# 人工智能的可观察性如何降低库伯内特的复杂性

> 原文：<https://thenewstack.io/how-ai-observability-cuts-down-kubernetes-complexity/>

Kubernetes 时代已经使多种云环境上的横向扩展应用成为现实。但它也给 it 部门带来了巨大的复杂性。

我在本期 [The New Stack Makers](/podcasts/makers) 播客中的嘉宾是来自软件智能平台 [Dynatrace](https://www.dynatrace.com/) 的 [Andreas Grabner](https://www.linkedin.com/in/grabnerandi/) ，他[最近提到](/how-ai-solves-the-kubernetes-complexity-conundrum/)“在我所见过的企业 Kubernetes 环境中，有数十亿的相互依赖需要考虑。”是的，*十亿*。

[Dynatrace:Andreas grab ner——AI 可观测性如何降低 K8s 复杂度](https://thenewstack.simplecast.com/episodes/dynatrace-andreas-grabner-how-ai-observability-cuts-down-k8s-complexity)

自称为“DevOps 活动家”的 Grabner 认为，人工智能技术可以驯服这种原本势不可挡的 Kubernetes 复杂性。正如他在[一篇投稿文章](/how-ai-and-full-stack-observability-can-overcome-todays-kubernetes-challenges/)中所说，“人工智能驱动的可观测性为企业提供了大量新的能力，以更好地部署和管理他们的 Kubernetes 环境。”

在播客中，我们深入探讨了人工智能——以及一般的自动化——如何影响 Kubernetes 环境中的可观察性。为了开始这场表演，我请 Grabner 澄清他所说的“人工智能的可观察性”是什么意思。

“我们称之为确定性人工智能，”他回答说，“它真正的意思是，在核心上，它是关于从许多不同的数据仓库中捕捉大量数据，然后你需要弄清楚我如何才能把(那些)数据放在仪表板上，并从中获得意义。我们所说的‘人工智能可观察性’，或者让我们更好地称之为‘确定性人工智能可观察性’，是指我们如何将数据与上下文信息联系起来。”

Grabner 指出，Kubernetes 环境中的数据可能来自很多地方——主机、pod、容器、应用程序和其他服务。挑战在于确定所有这些数据如何协同工作。

那么，Grabner 对 AI 可观测性的定义与可观测性的标准定义(度量、日志和分布式跟踪这三大支柱)相比如何呢？

Grabner 首先指出 Dynatrace 也有三个支柱，然后解释说，人工智能的可观察性在此基础上增加了一个上下文层。

“缺少的和我们添加的是关于整个堆栈的上下文信息，这意味着哪个服务在哪个端口[和]哪个特定主机上运行。因此，我们不仅有我所说的水平依赖性，对于分布式跟踪，我们还有垂直依赖性。”
