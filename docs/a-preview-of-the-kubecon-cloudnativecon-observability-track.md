距离上次在圣地亚哥的 KubeCon 北美已经过去了将近一年，在那里我[在主题演讲台上现场演示了 OpenTelemetry](https://www.youtube.com/watch?v=W_8MHdtrgZE) 并向世界强调了我们的 alpha。该项目在去年取得了相当大的进展，我们很高兴很快就能全面投入使用。虽然我会想念 10，000 多名技术人员为我欢呼的热情，但我很高兴今年的云计算原生计算基金会/KubeCon 北美活动比以往任何时候都更容易为全球观众所接受。门票定为 100 美元，不需要旅行，这是一个在舒适的办公桌前学习 OpenTelemetry 和其他云原生技术的绝佳机会，无论你的雇主的旅行预算是多少。

在过去的一年里，我们从[分析师](https://redmonk.com/jgovernor/2020/02/05/reframing-and-retooling-for-observability/)、[供应商](https://thenewstack.io/why-observability-needs-to-stay-weird/)和最终用户那里看到了越来越多的清晰观点，他们都认为可观察性对于实现云原生之旅的生产力和扩展优势至关重要。当我们企业的软件系统和我们的员工协同工作时，可观察性，即理解我们运行中的系统的能力就产生了。稳健的遥测(如日志、轨迹和/或指标)是必要的，但不足以实现有意义的可观测性。只有当我们能够询问数据并增强我们对系统的了解时，我们才能看到遥测的好处，这使我们能够在发射、混乱和平静中安全地操作我们的系统。

随着社区团结在 OpenTelemetry (OTel)标准周围，供应商和最终用户合作将仪器代码直接构建到最流行的框架中，产生与流行的开源解决方案和专有提供商兼容的遥测技术。OpenTelemetry 将遥测生产与消费分离，这意味着您不会被绑定到特定的供应商或后端。数十家[知名公司和开源项目](https://github.com/open-telemetry/community/blob/master/ADOPTERS.md)在生产中使用 OpenTelemetry 来实现可观测性结果。鉴于最佳实践发展如此之快，CNCF/库比肯商学院提供了一个向同行学习的绝佳机会，既可以学习播下可观察性种子所需的技术选择，也可以学习让可观察性蓬勃发展的文化。

今年，我很荣幸地在第 0 天[启动了](https://sched.co/fNzw)开放遥测社区日，并共同主持了 CNCF/库比康的可观测性跟踪。以下是我所期待的:

现在[为 KubeCon 和 OpenTelemetry 社区峰会注册](https://events.linuxfoundation.org/kubecon-cloudnativecon-north-america/)还为时不晚。KubeCon 本身的门票为 100 美元，另一个共同主办的 OpenTelemetry 社区日活动的门票为 30 美元。当您参加活动时，不要忘记在蜂巢展位前停下来，挑选一个虚拟的 OpenTelemetry 初学者工具包。

*考虑为你的遥测数据提供开源后端？Honeycomb 的* [*开源可观察性指南*](https://www.honeycomb.io/the-open-source-observability-landscape/?&utm_source=tns&utm_medium=blog-links&utm_campaign=referral&utm_content=open-source-observability-landscape) *帮助您评估运行自己的开源后端与使用托管服务的 TCO。*
