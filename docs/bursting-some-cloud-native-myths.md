# 戳破一些云原生神话

> 原文：<https://thenewstack.io/bursting-some-cloud-native-myths/>

当我们将云原生计算与 DevOps 相结合时，一切都向左移动了。或者至少，这是经常被高举的黄金终极愿景。平台工程师围绕不同开发人员的需求优化基础设施配置，他们实现自动化，并以自助菜单的形式提供。只需点击几下鼠标，开发人员就可以直接进入开发阶段，从流程的一开始就正确地完成基础设施。

但是在这一愿景中隐藏着一些神话和误解。

## **开发者不关心…**

 [丽莎-玛丽·南菲

Lisa-Marie 最近被任命为 Ondat 顾问委员会成员。她是蟑螂实验室的开发者关系负责人，云计算原生计算基金会(CNCF)大使，也是开源的长期倡导者。](https://www.linkedin.com/in/lnamphy) 

在 DevOps 和云原生计算领域，我们越来越多地听到这个词。无论是存储、数据库、网络、计算还是基础设施的任何其他领域，我们都被告知开发人员并不真正关心它。我知道人们这么说的确切意思，但对我来说，这有点太轻率了。

开发人员最关心的是构建优秀的应用程序——开发推动业务发展并帮助我们保持竞争力的软件。像大多数专业人士一样，我们希望有所作为。我们中很少有人愿意把时间花在配置存储卷、构建故障转移架构或微调数据库性能上(尽管少数这样做的人现在越来越有价值)。

但是没有*有* *关心*某件事，离*不关心*还很远。开发人员非常清楚地意识到，一些基础设施技术比其他技术更能有效地解决问题。每个开发人员都希望获得最好的底层技术，让我们能够构建、扩展并确保我们的应用程序是最好的。所以开发者确实关心——非常关心。我们希望确信我们为应用程序选择的是最佳选择。

## **基础设施是商品**

随着 AWS 开始的趋势，人们相信云原生计算已经完全商品化了基础设施:计算、内存、网络、存储——它们都只是你从平台中获取的普通资源。

虽然已经有了商品化的明确元素，但我们正在看到云原生技术的激增，以支持个性化的用例。但不管是不是商品，这都不是母体；你不能仅仅通过他们脑中的 CAT5 端口直接将这些资源输入到你的开发团队中。

任何平台工程师都会告诉你，他们的工作是自动化，是的，但也包括工具。组织用来在公共、私有和混合云平台上构建的工具、平台和架构正在成为竞争优势的重要来源。正如我所说的，我们都意识到一些基础设施技术比其他技术更能解决问题。这在云中没有什么不同。

正如我上面提到的，云原生基础设施平台正在成为商品的对立面。它们越来越多地对应用程序的各个方面产生最重要的影响:从开发和运营成本，到性能和弹性，再到支持不同的架构可能性，使应用程序一开始就可行。

## **我们已经到达天堂**

今天热门话题列表上的最后一个主要神话是，云原生开发和持续交付的世界(现在非常包括有状态应用程序)本身就是某种数字化天堂。这不是旅程的终点；这只是一个非常令人兴奋的开始。

任何业务变更管理都是关于保持人员、流程和技术之间的持续平衡。云原生开发运维极大地改变了技术和流程。现在的重点需要放在人民身上，他们的需求，并将其反馈到循环中。

出现了非常真实的技能短缺，这意味着公司不仅需要从他们的团队中获得最大的价值和生产力，也许更重要的是，表明他们最重视他们的团队。在这两种情况下，这将意味着选择新的云原生技术，首先也是最重要的，真正让企业关注人！
