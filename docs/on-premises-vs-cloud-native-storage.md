# 本地存储与云本地存储

> 原文：<https://thenewstack.io/on-premises-vs-cloud-native-storage/>

[云原生计算](https://thenewstack.io/what-are-cloud-native-patterns-and-how-should-you-use-them/)正迅速成为企业的标准。[容器](https://thenewstack.io/category/containers/)，像 [Kubernetes](https://thenewstack.io/primer-how-kubernetes-came-to-be-what-it-is-and-why-you-should-care/) ，[云原生解决方案](https://thenewstack.io/category/cloud-native/)像 storage orchestrators，以及其他云原生解决方案正在使应用和基础设施比以往任何时候都更高效、更可用、更易于维护。

但是，许多组织仍然在使用内部存储解决方案，即使他们已经为其技术堆栈的其他元素采用了云原生技术。是什么在左右着内部存储？

如果您有本地存储解决方案，这意味着您的服务器和其他硬件设备是在您组织的基础架构和物理场所中托管和管理的。

本地存储解决方案的显著特点是它们有很大的惯性。他们倾向于:

*   附带多年服务合同。
*   与相关供应商的生态系统中的其他解决方案配合得最好，或者只能配合使用，
*   需要内部专家来维护解决方案并排除故障，

一旦您投资了本地存储解决方案，上述问题就会让它变得非常棘手。更具体地说，内部存储解决方案具有高度的供应商锁定性。

但是，本地存储真正面临的困难是如何与更灵活的云原生技术合作，这些技术在当今的数字领域占据主导地位。如果您想要使用本地存储解决方案来存储[有状态、容器化应用](https://storageos.com/containerized-apps-need-stateful-storage)的数据，您将需要创建定制来处理负载平衡、可用性、可伸缩性、更新、持久性、联网等。

这种让本地存储基础架构与云技术协同工作(尽管并不完美)的场景相当普遍。有道理；组织希望使用基于云的现代技术，同时从其原有的内部基础架构中获得尽可能多的价值。

因此，他们以零敲碎打的方式将技术堆栈的元素迁移到云中。然而，这与云原生方法形成对比，并没有获得它所提供的好处。

## 云原生存储有什么不同？

正如云计算基金会对的定义[,](https://github.com/cncf/toc/blob/main/DEFINITION.md)

*"Cloud nativ* *e 技术使组织能够在公共云、私有云和混合云等现代动态环境中构建和运行可扩展的应用程序。容器、服务网格、微服务、不可变基础设施和声明式 API 就是这种方法的例子。*

这些技术使得松散耦合的系统具有弹性、可管理性和可观察性。与强大的自动化相结合，它们允许工程师以最少的劳动频繁地、可预测地做出高影响力的改变。”

本质上，这是一种云优先的开发、托管和管理应用的方法。就存储而言，云原生方法假设堆栈的其他元素将基于云技术，因此可以充分利用云原生堆栈所提供的优势。

默认情况下，在云原生方法下，您的更多基础架构可以由软件定义。因此，您的基础架构元素(如存储解决方案)可以在任意数量的环境中公开工作。这可以防止供应商锁定，使您能够与服务或产品更适合您的应用的供应商合作，或者与在您希望的地区运营的供应商合作。

通常，这种服务和产品是在自助服务的基础上提供的；云原生解决方案，尤其是云原生存储，让您可以按需付费，而不是为您可能会或可能不会充分利用的整体技术支付高额费用。

## 首先，您为什么想要内部存储？

很多组织都没有。但是，正如我们之前讨论的那样，本地存储有很大的惯性。转向基于云的存储如此困难有两个主要原因。

### 1.沉没成本谬论

组织已经在他们的内部解决方案上投入了大量资金。他们聘请服务提供商来管理这些解决方案，聘请内部专家，购买必要的硬件，这样的例子不胜枚举。不使用所有这些可能会感到痛苦，即使这是一个不太有效的方法。

### 2.数据安全性和合规性

传统上，内部存储被认为更安全，更容易遵守数据法规。

有时，[数据](https://thenewstack.io/category/data/)必须保留一定的年限，将这些数据迁移到不同的解决方案会带来风险。或者，也许数据需要托管在某个司法管辖区。切换到云原生存储解决方案可能会导致数据跨越国界移动到受不同法律管辖的数据中心。

因此，内部存储解决方案并没有为云原生存储提供不同但同样有吸引力的优势。相反，内部解决方案正在挟持其所有者。对于许多组织来说，转换到云原生解决方案的感知成本和难度太高，难以自圆其说。

## 幸运的是，切换变得越来越容易

就价格、风险、工作量和所有其他与云原生相关的成本而言，从内部部署转换到[云原生存储](https://storageos.com/what-does-it-mean-for-storage-to-be-cloud-native)变得越来越容易。

主要的云提供商现在几乎在每个国家都有数据中心，这使得遵守数据法规变得更加容易。云原生生态系统中的提供商比以往任何时候都多，这降低了成本，并确保有足够的多样性来满足最特定的需求。这很快成为常态。随着[云原生存储解决方案](https://thenewstack.io/category/storage/)变得越来越普遍，将内部解决方案的成本计入产品或服务价格标签将变得越来越难。

## 这对企业意味着什么

如果您的组织有机会放弃本地存储解决方案，您应该抓住这个机会。供应商锁定和惰性是阻碍许多企业全面现代化的强大力量。陷入另一个五年采购周期将使赶上和实现现代化变得更加困难。

我们已经提到了这一点，但改变到云原生技术堆栈也很重要。采用云原生存储解决方案本身有很多好处，但是升级到整个云原生基础架构有更多好处。毕竟，任何技术组合都是由最不现代的组件组成的。
