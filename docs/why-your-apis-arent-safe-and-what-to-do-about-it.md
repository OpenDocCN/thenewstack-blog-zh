## 影子 API 的危险

对任何组织的关键应用程序和数据来说，最大的威胁之一是 it 不知道的东西:[“影子 API”，](https://thenewstack.io/are-your-development-practices-introducing-api-security-risks/)没有记录的端点，因此不安全。

例如，根据 Imperva 的数据，30%的 API 流量流向影子 API，自 2021 年以来增长了 89%。

“一个组织拥有超过 1000 个不同的 API 端点并不罕见，”Klimek 说。“最终，这些 API 本身往往很少被记录，尤其是在处理大量遗留 API 时。”

他说，对 API 的攻击激增有几个原因。首先，有许多 API，创造了一个更大的攻击面，而且许多 API 都被遗漏了。

Klimek 说，针对 API 的网络攻击增加的另一个原因是，它们类似于 SQL 接口，使查询数据变得容易，但公开暴露其逻辑，“以便你可以有效地管理和使用它。”

这使得 API 成为坏人的诱人目标，他补充说:“事实上，它是机器可读的，这不仅使开发者更容易解析，也使攻击者更容易解析。”

## 降低 API 风险

为了保护 API 并防止您的端点成为黑客的入口，Klimek 推荐了一些步骤。其中包括:识别和记录所有影子 API，尤其是不再需要的遗留端点。

“你不能保护你不知道存在的东西，”他告诉观众。这应该包括对每个 API 的请求和响应负载的理解。

保护 API 的另一种方法是汇集安全团队和开发人员的知识，安全团队了解威胁形势，但可能不了解其组织 API 的业务逻辑，而开发人员的情况正好相反。

“对于开发者来说，更大的挑战是教育和理解攻击者经常使用的攻击媒介，特别是针对你的 API 的攻击媒介，”Klimek 说。
