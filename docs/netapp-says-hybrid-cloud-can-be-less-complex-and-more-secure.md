# NetApp 的目标是降低混合云的复杂性，提高安全性

> 原文：<https://thenewstack.io/netapp-says-hybrid-cloud-can-be-less-complex-and-more-secure/>

为多云和混合云管理提供一个简化的流程，包括存储和安全功能，这是当今云世界的一个神圣目标。因此，DevOps 团队正在寻找至少能够简化分布式数据和应用部署的混合多云基础架构管理的方法。如果供应商能够提供这种按需 SaaS，那就更好了。

[NetApp](https://spot.io/?utm_content=inline-mention) 表示，通过向其混合云产品组合中添加新的安全、存储和其他功能，它可以提供所有这些功能以及更多功能。

随着该版本的发布，NetApp 旨在帮助开发团队努力采用混合多云基础架构， [Ronen Schwartz，](https://www.linkedin.com/in/ronenschwartz/)NetApp 云卷服务高级副总裁告诉新堆栈。总的来说，作为开发应用和推动数字化转型的新平台，混合多云可以带来巨大的业务优势。

然而，内部部署和多种云资源之间缺乏互操作性和一致性可能会“导致严重的问题，可能会减缓采用速度并限制灵活性，从而抵消优势，”Schwartz 说。“不同的环境需要不同的管理工具、服务和工作流，这增加了运营上的混乱。客户最终可能会将其数据锁定在跨多个云的不同孤岛中。这也扩大了各种安全威胁的攻击面，尤其是勒索软件攻击。”

## 简单且更好

Schwartz 表示，借助 NetApp 的最新增强功能，网络服务提供商正在提供一种简单、更好的混合多云存储和数据管理，帮助客户购买、部署、运营和优化统一的云体验即服务。“我们还将网络弹性和勒索软件保护功能直接嵌入到我们的混合多云解决方案中，”Schwartz 说。

特别是，产品和服务更新包括:

*   “作为服务”提供的简化混合多云体验
*   增强的反勒索软件集成，可在攻击发生时进行保护、检测和恢复
*   **NetApp ONTAP** :更好的数据安全性、勒索软件保护、性能和操作简单性
*   **NetApp 云管理器**:全面集成 ONTAP 系统管理器并增强订阅管理
*   **NetApp 重点**:混合云 STaaS。内部部署和云存储服务的单一订阅
*   **VMware** : NetApp 是唯一一家被选中的合作伙伴，可作为 VMware 云服务的外部补充数据存储支持您选择的云。

这些版本解决的主要问题之一是提高混合多云环境中的存储，因为 DevOps 团队会使用 NetApp 平台在混合多云环境中迁移虚拟机工作负载。

Schwartz 说，虽然以前可以将虚拟机工作负载迁移到公共云上的 VMWare 云中，但在以前的模式下，客户或者 必须使用 VMware 云中的嵌入式存储。

“这限制了灵活性和可扩展性，并伴随着更高的成本，或者需要更复杂的程序来直接从每个来宾虚拟机访问 NetApp 存储，Schwartz 说。现在，NetApp 将成为唯一一家能够将共享存储提供的可扩展外部补充数据存储库作为单个池提供给所有虚拟机，而不是基于每个虚拟机的供应商。这更类似于同类最佳的内部 VMware 环境和外部统一存储所使用的通用体系结构。”

## ONTAP 改进

为了更好地集成反勒索软件，NetApp 表示，其新的 [ONTAP](https://www.netapp.com/data-management/ontap-data-management-software/) 数据管理软件功能有助于提高数据安全性和勒索软件保护，同时在其为混合云产品组合发布的新功能的保护下保持简单易用。NetApp 传达的 ONTAP 改进包括:

*   通过要求对关键管理任务进行多重批准，提供更多保护。
*   借助机器学习和 “增强算法”快速检测新威胁，NetApp 称这些算法可以检测更复杂的恶意软件。
*   保护多 Pb 数据集的能力。
