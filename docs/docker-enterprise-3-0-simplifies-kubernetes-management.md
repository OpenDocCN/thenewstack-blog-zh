# Docker Enterprise 3.0 简化了 Kubernetes 管理

> 原文：<https://thenewstack.io/docker-enterprise-3-0-simplifies-kubernetes-management/>

Docker 发布了 Docker Enterprise 的第三个主要版本，这是该公司在专业环境中管理容器的平台。

Docker Enterprise 3.0 完全整合了该公司的桌面开发人员界面，提供了从开发人员到运营人员的自动化工作流程。它还提供了简化的 Kubernetes 服务，以及对多服务应用程序的新支持。该公司还与凯捷合作，提供 Docker Enterprise 托管服务。

“有些早期技术采用者喜欢深入了解配置，但随着 Kubernetes 的采用增加，你不能再只吸引那些前沿个人了。Docker 产品营销总监 Jenny Fong 断言道:“我们必须让这种产品为大众服务。

Fong 声称，随着 Docker Desktop 的加入，该软件是唯一一款可以无缝管理从开发人员桌面到 Kubernetes 管理的基于云或内部的生产环境的容器的商业软件。

一个有助于自动化应用交付的新工具包是一个名为 Docker Applications 的新工具包，它使用由微软和 Docker 开发的[云原生应用捆绑包](https://cnab.io/) (CNAB)标准，提供了一组构建应用所需的通用元素。它可以从 Docker Hub 识别和构建所需的容器，并创建 Docker Compose 和 Kubernetes YAML 文件、舵图和其他所需的配置设置。

有了 CNAB，开发人员可以通过使用参数化字段来定义应用程序，而不管它运行在什么平台上。

“通过简单的命令行，你可以改变默认端口或默认 orchestrator，即使你想在 Kubernetes 或副本数量之间切换，”Fong 说。

## 一次编码，随处部署

Docker 应用程序为在多种不同环境中部署应用程序做好了准备——结合其他现有的 Docker 包，如应用程序模板、应用程序设计器和版本包。

这里的另一个核心组件是新的 Docker Kubernetes Services (DKS)，它提供了一种跨不同环境安装、配置、管理和扩展基于容器的应用程序的方法。除了设置 YAML 和掌舵图，DKS 还提供安全和 RBAC(基于角色的访问控制)、PKI 认证和自动化生命周期管理。

Fong 断言:“Docker Kubernetes 服务是唯一一个能够为桌面和服务器环境提供 Kubernetes 并确保应用始终一致的平台。

作为 Kubernetes 的替代方案，用户还可以部署 Docker Swarm Services (DSS)来协调容器操作。

Docker 企业即服务旨在解决云原生工程师短缺的问题，构建于 Docker 企业之上，是一种全面管理的产品，将在全球范围内提供，可以在任何云或内部运行。

Fong 说，有了 Azure Container Service (AKS)或亚马逊网络服务(Amazon Web Services)的 Elastic Kubernetes Service (EKS)，“你得到的是托管集群”。相比之下，Docker 即服务是“管理一切，一直到把你自己的容器放在上面。”

通过这项工作，该公司协调努力，提供了一个端到端的平台堆栈。

“当我们开始考虑创新时，我们不仅关注一种应用的创新，还关注整个应用组合的创新，以帮助组织管理其传统应用和全新的绿色领域云原生和微服务应用，”Fong 说。

特别是，该公司着眼于在开发人员和运营团队之间转移容器的交接。

“当我们研究主流企业组织如何管理这种复杂性时，特别是如果他们的员工在 Kubernetes 方面的专业知识有限，并且如果他们没有能力雇用和找到人才，”Fong 说。

Docker Enterprise 3.0 现已推出测试版，完整的量产版将于今年晚些时候发布。
