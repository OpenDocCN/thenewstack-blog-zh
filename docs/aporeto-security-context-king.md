# Aporeto:当涉及到安全时，上下文就是王道

> 原文：<https://thenewstack.io/aporeto-security-context-king/>

在最新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/) 播客中，我们采访了 [Aporeto](https://www.paloaltonetworks.com) 联合创始人兼首席执行官 [Dimitri Stiliadis](https://twitter.com/dstiliadis) 以了解更多关于 Aporeto 如何在当今云优先的生态系统中实现安全性的信息。采访由 TNS 创始人亚历克斯·威廉姆斯和共同主持人[网络安全管理软件产品](http://www.solarwinds.com/)技术战略高级总监[李·卡尔科特](https://www.linkedin.com/in/leecalcote)主持，在 2016 年西雅图 Kubecon/CloudNativeCon 期间举行。

“人们正在努力解决网络上的许多安全问题。我认为在过去的 20 到 30 年里，我们一直在尝试猜测数据包在做什么。我们在云方面的机会是，我们现在了解了应用程序在做什么。Stiliadis 说:“通过将安全性转移到更靠近应用的地方，我们可以简化网络。

Aporeto 的应用程序分段工具 [Trireme](https://www.aporeto.com/trireme/) 允许组织在应用层加强安全性。例如，对于 Kubernetes ，它允许任何两个由 Kubernetes 管理的应用程序只能通过网络 API 相互通信。作为 Trireme 的 Kubernetes 1.4 安全 API 集成的一部分，Aporeto 能够通过部署一个简单的库为开发人员提供对 Kubernetes 安全策略的访问。“安全性还有另外一个方面，如果将关注点分开，情况往往会更好。理想情况下，您希望安全性与其试图保护的东西保持一定距离。”

“我强烈认为，安全必须在开源中完成，这样人们就可以检查你在做什么，”Stiliadis 说，并补充说，对于不太短暂的应用程序，“我们可以用相同的统一方式对待它们。在这两种情况下，我们有越来越多的背景。说到安全，上下文才是王道。”

[云计算基金会](https://www.cncf.io/)赞助了这个播客。 [Aporeto](https://www.paloaltonetworks.com) 是新堆栈的赞助商。
