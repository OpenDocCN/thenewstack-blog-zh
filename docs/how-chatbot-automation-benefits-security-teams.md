# 聊天机器人自动化如何让安全团队受益

> 原文：<https://thenewstack.io/how-chatbot-automation-benefits-security-teams/>

当你听到“聊天机器人”这个术语时，你首先想到的可能是零售网站上的机器人客户支持服务——聊天机器人的一个相对普通的用例，如果你是一名安全工程师，可能很难对此感到兴奋。

但事实是聊天机器人能做的远不止提供客户支持。他们还可以简化安全自动化，并帮助团队在识别、研究和应对威胁时更有效地合作。

## 什么是聊天机器人？

聊天机器人是一种自动化工具，可以传播信息，促进对话和/或采取行动，以响应命令。

一些聊天机器人被设计成模拟人类演员，这意味着它们应该能够与活跃的人类进行类似的对话。然而，并不是所有聊天机器人都以这种方式工作，(那些用于安全自动化的聊天机器人通常不会这样做)。从更一般的意义上来说，聊天机器人是任何类型的有助于简化对话的工具，不一定是设计来模拟人类对话智能的工具。

## **安全聊天机器人如何工作**

在安全操作环境中，可以部署聊天机器人来发起和管理人类参与者之间关于安全活动的对话。

例如，安全聊天机器人可以被配置为在空闲信道中宣布检测到安全风险。从那里，Slack 通道中的工程师可以向聊天机器人询问有关威胁的更多信息，例如哪些日志与之相关，或者威胁的严重程度如何。在某些情况下，他们还可以向聊天机器人发出命令，指导它采取自动行动，如[阻止违规的 IP 地址](https://torq.io/blog/automate-ip-address-management-with-cloudflare/)。

当聊天机器人以这种方式使用时，它们可以实现安全操作的 [ChatOps](https://www.techtarget.com/searchitoperations/definition/ChatOps#:~:text=ChatOps%20is%20the%20use%20of,communication%20channel%20for%20ongoing%20work.) 方法。ChatOps 是一种使用自动聊天(即人机对话)来简化工作流程和信息共享的实践。

## **聊天机器人自动化为安全运营带来的好处**

通过以这种方式利用聊天机器人，安全团队获得了一系列好处:

*   **集中沟通**:聊天机器人有助于提供一个集中的沟通界面，所有利益相关者都可以在研究安全事件时使用。由于与聊天机器人的交互对多个参与者可见，因此信息尽可能可见，并且每个人都可以跟踪安全事件响应的状态。
*   **简化响应**:工程师可以直接要求聊天机器人在共享频道中提供信息，而不必手动整理后端系统来查看安全事件的背景。这不仅节省了时间，还有助于确保每个人都看到相同的信息。
*   **随时可用**:与人类工程师不同，聊天机器人从不睡觉——甚至不会分心。这意味着他们可以即时宣布现有安全工作流的新威胁或更新，而不管一天中的什么时间或谁在待命。
*   **响应记录**:聊天机器人促成的聊天记录了对事件采取的行动。如果您需要在事后确定谁做了什么，您可以使用您的历史聊天数据来这样做。

在所有这些方面，聊天机器人和聊天工具有效地充当了安全自动化的大门。虽然聊天机器人本身不能实现完全的安全自动化(为此，您需要一个[安全自动化框架](https://torq.io/use-cases/)，但是它们提供了一个集中的、用户友好的界面，利益相关者可以使用它来请求信息或触发自动化的安全工作流。

## **安全聊天机器人入门**

聊天机器人在安全方面的优势在于它们相对容易设置。如果你有一个像 Torq 这样的安全自动化工具，你可以利用与聊天平台的内置集成，如 [Slack](https://docs.torq.io/v1.0/integrations/slack) 或 [Teams](https://docs.torq.io/v1.0/integrations/teams) 只需几个步骤就可以部署机器人。

一旦部署到您选择的聊天系统中，聊天机器人就提供了一个与您的安全自动化框架的其余部分进行交互的前端。你也可以配置自定义命令(比如 [Torq slash 命令](https://docs.torq.io/v1.0/integrations/slash-commands))，然后你的团队可以用它来触发你的聊天机器人的任何动作。

你不需要聊天机器人来利用安全自动化。但是聊天机器人可以帮助您最大限度地利用您已经部署的其他安全自动化工具。无论您的团队使用哪种通信工具，它们都可以集中安全操作并最大化可见性。
