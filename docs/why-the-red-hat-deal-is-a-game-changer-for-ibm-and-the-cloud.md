IBM[上周提出收购 Red Hat](/red-hat-acquisition-will-be-the-key-to-ibms-future-in-the-cloud/)让业内许多人大吃一惊，但这一举动非常合理。它将 IBM 定位为在云计算的第二幕中扮演领导角色。

第一幕是关于基础设施即服务(IaaS)，提供基本存储、网络和计算，我们知道这是如何实现的。亚马逊通过以更具成本效益的方式提供计算和存储，从 HPE、戴尔和 IBM 等硬件厂商那里分流了基础设施支出。

但现在正在进行的 Act II 超越了基本计算，提供了真正的应用基础设施，数据库、分析、消息队列和机器学习/人工智能都作为 API 驱动、容器化和由 Kubernetes 编排的服务提供。这是一个混合的世界，企业在自己的数据中心和公共云中构建这种云原生基础架构。Red Hat 交易将允许 IBM 通过混合云应用基础设施跨越两个世界。

IBM 可以使用 Red Hat 的软件来构建自己的云服务，将 Watson、区块链和 DB2 等功能与 Red Hat 的 OpenShift、Ansible 和其他基础设施软件相结合，向云现任者发起有效的挑战。对于许多企业来说，IBM 仍然是一个巨大且值得信赖的合作伙伴，Red Hat 将允许它在这些客户开始下一阶段的云之旅时将他们带回 IBM。

诚然，AWS 和其他云公司已经开始自己提供这些应用服务，但这不是他们的专长。IBM 在分析、人工智能和区块链等领域拥有更强大的技术。Red Hat 的软件将允许它在云中充分利用这些服务，通过提供将它们联系在一起的应用程序基础设施。IBM 也将拥有对开发者最友好的云操作系统，这并没有什么坏处。

IBM 一直坚持不懈地关注数据，数据是现代应用程序的命脉，也是商业的新货币。分析和机器学习的力量在于数据的质量和监管，这是实现洞察力和创新的核心。IBM 在 DB2 和其他数据技术方面的悠久历史只会加强它在云中的地位。

Portworx 也处于数据管理革命的中心，其领导地位是 Kubernetes 跨公共、私有和混合云的公共数据平面。越来越多赢得技术竞赛的企业将会把这种类似云的基础设施带到其公共和私有 IT 环境的各个方面，使他们能够构建实时数据和应用堆栈，从而为客户提供洞察力和价值。

这种无处不在的云模式将在未来几年推动技术的发展，这也是 IBM-Red Hat 交易如此有意义的原因。企业正在内部运行虚拟化和容器，因为他们想要云模式提供的速度和敏捷性。有了 Red Hat，IBM 将同时拥有应用程序和云基础设施，在这个混合世界中处于领先地位。这看起来是一个明智的 340 亿美元的赌注。
