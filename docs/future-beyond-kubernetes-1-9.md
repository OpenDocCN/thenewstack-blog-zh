# 超越 Kubernetes 1.9 的未来

> 原文：<https://thenewstack.io/future-beyond-kubernetes-1-9/>

[未来超越 Kubernetes 1.9](https://thenewstack.simplecast.com/episodes/the-future-beyond-kubernetes-1-9)

在过去的六个月里，Kubernetes 已经发布了两个新版本。公司如何保持同步？在这个开源社区中，开发人员的拥护者、维护者和主要开发人员在为客户服务中的角色是什么？

这些问题和其他问题在本月早些时候从 2017 KubeCon 和 CloudNativeCon 直播的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中得到解决。[Ihor Dvoretskyi](https://twitter.com/idvoretskyi),[云本地计算基金会](https://www.cncf.io/)的开发者倡导者；[莎拉·诺沃特尼](https://twitter.com/sarahnovotny),[谷歌云平台](https://cloud.google.com/kubernetes-engine)开源战略负责人；微软的高级工程师兼 KubeCon 的联合负责人 Michelle Noorali 都加入了 TNS 创始人 Alex Williams 的讨论，讨论如何让一个项目达到 CNCF 项目令人垂涎的地位，新当选的 CNCF 治理委员会正在做什么，以及未来会如何。

“我们想要长期、忠诚的贡献者，”诺沃特尼说。“我们正在努力建立一个指导项目。因此，我们正在考虑增加来 Kubernetes 的人数，但我们不一定要为那些只是进来尝试新事物的人进行优化。”

Novotny 说，8 月选举产生的 CNCF 治理委员会已经从隐性治理转向显性治理。例如，CNCF 在项目开始时不要求核心价值。但是，Dvoretskyi 说，“这个项目成功的主要原因是核心价值观。”

该委员会正致力于编纂一套核心价值观(已经是草案形式)，以及诸如重新基准、新贡献者指导和项目文档等概念的基线。这一切都以闪电般的速度发生。
