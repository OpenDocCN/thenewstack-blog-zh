# 数字平台成功和事件架构

> 原文：<https://thenewstack.io/digital-platform-success-and-event-architecture/>

如果使用[事件驱动架构(EDA)](https://thenewstack.io/how-an-event-driven-architecture-drives-business-agility/) 来构建数字平台，那么组织拥有成功数字平台的可能性会增加一倍。“ Gartner 2020 构建数字平台研究”基于半年前对 200 名 IT 领导者的调查，确定了数字平台的目标(如效率、业务增长)，受访者根据这些标准对成功进行了评级。总体而言，拥有高度成功的数字平台的 IT 领导者中有 67%利用了 EDA。在研究的其余四分之三中，只有 28%在其平台中使用 EDA。 

事件驱动架构本身能让平台更成功吗？可能不会，但这是一个更好的指标，表明现代软件设计方法正在被使用，而不是说云原生和敏捷/DevOps，这两者都已成为先决条件。虽然流数据通常利用 EDA，但它只是描述了一种狭窄的技术实现，而不是一种可以产生商业价值的方法。像微服务和无服务器这样的术语没有包含在调查问题中，但它们已经被营销炒作所玷污。目前，“事件驱动架构”似乎是现代软件工程实践正在被利用的一个很好的指示器。

数字技术平台是一个广泛的类别，描述了一组汇集在一起创建应用程序、应用程序和工作流的服务和功能。外部供应商通常不会将它们作为一个单元出售，但它们可能会与自称为 PaaS 的产品竞争，尤其是那些声称拥有利基用例的产品。一个这样的专业是[多体验开发体验平台(mxdp)](https://www.gartner.com/reviews/market/multiexperience-development-platforms)，它集中开发活动，以确保跨 web、移动、可穿戴、对话和沉浸式接触点的一致用户体验。虽然我们对新创造的技术市场名称持怀疑态度，但这引起了我们的注意，因为像 [OutSystems](https://www.outsystems.com/) 和 [Mendix](https://www.mendix.com/) 这样的低代码供应商正在将产品定位到这个领域。对开发人员的价值主张是，如果他们能够减少使用的工具数量，他们会变得更有效率。

平台的采用仍不普遍，在[“2020 年开发运维状况报告”](https://thenewstack.io/puppets-2020-state-of-devops-report-why-automation-and-self-service-are-key/)中，63%的受访组织使用用于构建和交付应用或服务的自助服务平台。如果有一个平台，可能有很多。在拥有平台的公司中，有一半至少拥有四个平台。该报告还发现，随着一家公司的 DevOps 成熟度逐渐上升，它将平台视为产品本身的可能性也在上升。当这种情况发生时，平台团队变得负责，成功变得更有可能。
