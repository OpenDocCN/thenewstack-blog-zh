# 新冠肺炎对微服务、多云和 Kubernetes 意味着什么

> 原文：<https://thenewstack.io/what-covid-19-means-for-microservices-multicloud-and-kubernetes/>

Kubernetes 已经成为管理容器的实际选项，而微服务则作为 Kubernetes 集群的底层分布式架构。多云基础设施的持续崛起也被视为继续采用微服务来为这种广泛分布的基础设施部署 Kubernetes 的渠道。在这个快速膨胀的多云世界中，创建应用程序和管理如此多样的基础架构的需求。

但是，突然之间，新型冠状病毒疫情在全球范围内以我们尚未完全意识到的方式颠覆了世界。

在这个[新堆栈制造商](/podcasts/makers)播客中，软件即服务(SaaS)提供商 [Volterra](https://www.volterra.io/) 的创始人兼首席执行官 [Ankur Singla](https://www.linkedin.com/in/ankur-singla-4863801/) 讨论了 Kubernetes、微服务、多云环境和开源对当今计算和 IT 的深远影响，以及它们在[新冠肺炎世界](https://www.cdc.gov/coronavirus/2019-ncov/index.html)中可能产生的影响。

[Volterra 首席执行官 Ankur Singla——新冠肺炎对微服务、多云和 Kubernetes 的意义](https://thenewstack.simplecast.com/episodes/volterras-ceo-ankur-singla-what-covid-19-means-for-microservices-multi-cloud-and-kubernetes)

辛格拉说，新冠肺炎把一切都搞乱了。“我们意识到的第一件事是，我们的许多大型企业客户的企业网络正在成为一个大瓶颈，”Singla 说。“为了减少负载，许多企业已经在考虑如何快速将应用从私有数据中心迁移到云，因为网络到云比网络到私有网络好得多。”

Singla 说，向云的迁移还包括“转向 SaaS 服务”。“SaaS 服务显然需要规模，越来越多的服务转向微服务，”Singla 说。

可以有把握地假设，在经历了最初的炒作周期，并被普遍接受和可靠之后，特定的技术或架构已经成为主流。与成本节约和明显提高的效率相关的技术也是用于确定特定技术是否获得主流地位的另一个标准。因此，可以认为微服务正在成为主流。Singla 说:“越来越多的企业正在迁移到 SaaS 服务，而 SaaS 最重要的是规模，而利用微服务进行规模扩展要容易得多。”。

大约五年前，当大型企业大量采用微服务时，Singla 开始看到微服务的潜力。“但我越来越清楚，这将是一种架构模式，而且随着无服务器的出现，这种模式也越来越明显——因此，模式转变开始发生了。我们认为这是一个很好的机会，可以成立一家新公司，帮助解决许多问题，成为多家云提供商的主流，并能够建立高度分布式的边缘位置…随着分布式应用程序和数据的融合，”Singla 说。“我们说，‘现在是开公司解决分布式应用数据问题的大好时机。’这就是沃尔泰拉的背景。"
