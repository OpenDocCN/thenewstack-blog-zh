# 裂变. io 如何为 Kubernetes 带来无服务器

> 原文：<https://thenewstack.io/how-fission-io-brings-serverless-to-kubernetes/>

[与裂变. io 创造者 Soam Vasani](https://thenewstack.simplecast.com/episodes/talking-serverless-with-fission-io-creator-soam-vasani) 无服务器交谈

无服务器已经变得不仅仅是[亚马逊λ](https://aws.amazon.com/lambda/)。今天，有几十种不同的无服务器平台，从云中的，到那些设计在数据中心内运行的，到那些在 [Kubernetes](https://kubernetes.io/) 中运行的。最后一批是过去一年中最感兴趣的地方，现在随着 [Knative](https://github.com/knative/) 作为一个旨在[平滑 Kubernetes](https://thenewstack.io/with-knative-google-brings-multicloud-serverless-to-the-enterprise/) 上无服务器体验的项目的宣布，这种兴趣更加浓厚。

上周在旧金山举行的 [Serverlessconf](http://serverlessconf.io/) 大会上，我们采访了 [Soam Vasani](https://www.linkedin.com/in/soamvasani/) ，他是 Kubernetes 无服务器框架[裂变. io](https://fission.io/) 的创建者。他在 2016 年底开始了这个项目，并表示它已经成为 Kubernetes 用户的一个受欢迎的功能即服务系统。这个项目是在[平台 9](https://platform9.com/) 上创建的，目的是帮助他们的客户在安装他们自己的 Kubernetes 集群的第一天就提高工作效率。“你可以用同样的方式构建函数，例如，在 AWS Lambda 上，但是你可以在任何 Kubernetes 集群上的任何地方运行它们，”Vasani 说。

Vasani 说，为客户提供一种使用功能的方式，让他们更快地理解与迁移到基于容器的环境相关的巨大思维转变。“你必须为集装箱生态系统学习的概念数量非常大，现在仍然如此，”Vasani 说。“它与时间一起成长。部署、服务、映像注册、版本管理、如何进行更改等概念。从您的笔记本电脑到容器集群的整个流程是什么？即使在今天，作为一个正在进入这个系统的人；这是一个每个用户都必须自己构建的领域，因此生态系统仍然是一个需要一些组装的解决方案。”

Vasani 和 Platform9 团队构建了裂变，作为打破这些复杂概念的一种方式，并提供了一种更简单的方式来开始与容器生活方式合作。开发人员可以使用裂变来快速构建保持整个系统运行所需的应用程序类型。“你有功能、环境和触发器。这些是我们需要的概念。如果你学会了这三个，它会给你所需要的，你就准备好建立裂变了，”瓦萨尼说。“我们有一体化的工具，让您从笔记本电脑上的源代码开始，在您的集群上进行构建、部署和监控。我们与监控系统、 [Istio](https://istio.io/) 和日志聚合系统进行了集成，因此您可以为您的功能获得一个自以为是的小堆栈。”
