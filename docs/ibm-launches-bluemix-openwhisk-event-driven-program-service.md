# IBM 推出了 Bluemix OpenWhisk，这是一种事件驱动的编程服务

> 原文：<https://thenewstack.io/ibm-launches-bluemix-openwhisk-event-driven-program-service/>

跳入新兴的无服务器计算范式，IBM 推出了一个新的事件驱动编程服务，名为 Bluemix OpenWhisk。

使用事件驱动模型，该服务自动执行用户代码以响应外部事件，例如到达的一点传感器数据。IBM 在该公司本周在拉斯维加斯举行的 [InterConnect 2016](http://www.ibm.com/cloud-computing/us/en/interconnect/) 大会上介绍了这项服务。

除了 OpenWhisk，IBM 还推出了 IBM 云托管的 [GitHub Enterprise](https://enterprise.github.com/home) 服务，允许组织管理自己的 GitHub 代码库，供企业云和混合云使用。

IBM 还为苹果 Swift 语言的用户提供基于云的运行时，为苹果设备构建后端功能。

OpenWhisk 的事件驱动方法通常被称为无服务器计算，可能会简化开发和部署新应用程序的过程，因为开发人员不必担心设置服务器和中间件来支持业务逻辑，这可以在 OpenWhisk 运行的代码中捕获。

IBM 还鼓励用户考虑使用这种服务来创建基于微服务的应用程序，在这种应用程序中，单独的任务是单独维护的。应用程序的各个部分可以由不同的团队并行维护。这种方法还允许应用程序操作并行运行。

开发人员可以用 Swift 或 JavaScript for Node.js 编写代码。或者，他们可以将自己的定制代码打包在 Docker 容器中，供那些需要导入遗留代码或担心被局限于单一服务提供商的人使用。

IBM 并不是第一个提供无状态的[，也称为无服务器](https://thenewstack.io/year-ahead-stateless-computing/)，即云计算服务的公司。自 2014 年以来，亚马逊网络服务已经提供了一项类似的服务，AWS Lambda。本月初，谷歌推出了一项类似的服务，名为[谷歌云功能](https://thenewstack.io/google-cloud-functions-arrives-challenge-aws-lambda/)。包括[无服务器](https://thenewstack.io/serverless-offers-framework-aws-lambda/)和 [Iron.io](https://thenewstack.io/iron-io-brings-aws-lambda-compatibilities/) 在内的许多初创公司都提供了用于建立内部无服务器平台的软件包。

然而，使用 Bluemix 可以让用户轻松接入其他 Bluemix 服务，为他们的应用增加分析和其他变革性服务。

IBM 将开源 OpenWhisk 本身的代码库。

## Swift 和 GitHub

IBM 衷心鼓励使用苹果的 Swift 语言，部分原因是该公司去年与苹果达成合作，以更好地装备 iPhone 用于企业工作。为此，该公司发布了一个 Swift 运行时预览版和一个 Swift 包目录，两者都是为了帮助开发者为苹果设备构建企业应用。

去年，IBM 发布了用于在服务器上运行 Swift 的运行时软件 Swift Sandbox。迄今为止，已有超过 100，000 名开发人员使用了该服务，执行了超过一百万次代码运行。

企业 Github 服务是通过 IBM Bluemix 平台提供的，可以通过 [Bluemix 专用和本地](https://thenewstack.io/ibm-plant-bluemix-cloud-behind-firewall/)管理交付模式获得。

通过 Bluemix 上的 Github repo，用户可以更容易地链接到其他基于 Bluemix API 的服务和数据源。IBM 将提供代码扫描服务来监视可能渗入代码库的安全漏洞。
