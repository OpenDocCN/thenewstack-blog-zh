# 为什么医疗保健研究需要多租户协调来拯救生命

> 原文：<https://thenewstack.io/healthcare-research-needs-multi-tenant-orchestration-save-lives/>

我们经常谈论使用混合架构在混合平台上运行混合工作负载的组织。我们鼓吹杂交，好像这是组织应该采取的自愿进化步骤。但是我们很少考虑混合组织(T1)，即一个平台的用户，这个平台本身是一个企业集团，或者是分散在地球上不同能力的各类用户的混合体。

[华盛顿大学健康指标与评估研究所](http://www.healthdata.org/)表面上看，是一个单独的研究机构，却声称拥有世界上最大的人类健康数据仓库。但是，这个储存库是依靠一种独特的组织结构，这种结构把杂交的整个问题抛到了脑后(假设这样的组织把耳朵放在同一个地方)。从它使用的应用程序来看，IHME 是一个混合组织。全世界几百个研究机构，都有收集医疗保健数据的责任，也共享这些数据。

然而平台本身实际上是*而不是*云。事实上，它是一个集中的数据中心，位于大学校园内。它使用了 [Univa](https://www.univa.com/products/navops.php) 的 [Navops](http://www.navops.io/) ，[开源 Kubernetes 容器编排引擎](https://thenewstack.io/univas-new-powerful-navops-command-working-kinks-kubernetes/)的多租户改编版，使所有这些设施能够像使用云一样使用集中式平台。它确保 UW 的 It 团队管理治理和安全，这是其用户居住的许多国家所要求的。

但是这个特别版的[新堆栈制造商](https://thenewstack.io/podcasts/makers)播客不仅仅是关于抽象的基础设施和多租户。这是一个用例，其结果导致关于国家和文化可以更好地对待自己人民的方式的见解——帮助他们避免死亡并提高他们的生活质量。在一个良好的医疗保健往往岌岌可危的世界里，信息共享可能成为成千上万人的生命线。

现在来听听 IHME nav ops 的故事，以及我们每天都在处理基础设施和自动化的真正原因。
