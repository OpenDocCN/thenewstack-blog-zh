# 新的堆栈环境:通往云本地计算的漫长道路

> 原文：<https://thenewstack.io/the-new-stack-context-the-long-road-to-cloud-native-computing/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。本周，我们与 [Catherine Paganini](https://www.linkedin.com/in/catherinepaganini/?locale=en_US) 聊天，她是 Kubernetes 软件提供商 [Kublr](https://kublr.com/) 的市场营销主管，谈论她最近持续发表的一系列文章，这些文章向商业领袖解释了 IT 概念。我们讨论了撰写技术文章的过程，以及学习 DevOps 驱动的云原生计算的艰难过程。然后在稍后的节目中，我们聊起了本周的头条新闻。

[Ep。#101 通往云原生计算的漫长道路](https://thenewstack.simplecast.com/episodes/ep-101-the-long-road-to-cloud-native-computing)

帕格尼尼在她给我们的最新帖子中写道:“揭开 DevOps 的神秘面纱——数字转型帷幕背后的一瞥”。

*数字化转型是由客户、合作伙伴和供应商的互动推动的，这些互动越来越多地进入数字化领域。客户被网飞、优步或 Monzo(一家云诞生的英国银行)等云创新者提供的快速可靠的交互宠坏了，现在他们要求更传统的企业提供同样水平的便利。如今，软件已经成为一个关键的战略业务区分因素，其快速、可靠和便捷的交付是数字化转型的核心。*

*简而言之，DevOps 是实现这种快速、频繁且不间断的服务交付的方法，而云原生技术使这种方法成为可能。*

我们讨论了沿着 DevOps 这条路走下去的挑战，以及为什么它对许多人来说是如此的挑战。

然后，在节目的后面，我们谈论了当天的一些头条新闻，包括一个新的 [Kubernetes 扫描工具](/octarine-open-sources-a-kubernetes-tool-to-check-for-configuration-security-errors/)，它不寻找软件漏洞，而是寻找不正确的配置，一个据称[更容易管理 Kafka 数据流的方法](/dataops-lenses-io-drives-data-with-sql-kafka/)，以及一些开发人员[过于热衷于干净代码实践的概念](/this-week-in-programming-clean-code-zealotry/)。

TNS 的编辑和营销总监 Libby Clark 与 TNS 的执行主编 Joab Jackson 一起主持了这一集。
