# 塑料 SCM 合并机器人:CI/CD 的版本控制

> 原文：<https://thenewstack.io/plastic-scm-mergebots-version-control-for-ci-cd/>

[塑料 SCM 合并机器人:CI/CD 版本控制](https://thenewstack.simplecast.com/episodes/plastic-scm-mergebots-version-control-for-ci-cd)

在本期 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，我们采访了 [Pablo Santos](https://www.linkedin.com/in/pablo-santos-94b49138/) ，他是 [Plastic SCM](https://www.plasticscm.com) 的首席工程师，这家公司是他在 2005 年创立的，就在科技大爆炸给我们带来 Git、GitHub、Dropbox 以及 iPhone 之前。

在许多方面，桑托斯走在了时代的前面。在像微软这样的公司每两年发布一次软件更新的时代，他设计的每周更新三到四次的软件系统是如此具有革命性，他们不得不对他们的公司客户隐藏，每年提供一次品牌更新。一旦行业开始转向连续交付，或者至少是“更频繁交付”的模式，该公司的客户想知道塑料是如何实现这种神奇的。因此，该公司将其软件捆绑到一个专有平台上。

Santos 承认，PlasticSCM 从 Git 的流行中受益匪浅。

## 合并机器人和版本控制

塑料为 CI/CD 提供了一些不寻常的东西:版本控制。这部分管道一直是持续集成的祸根。Plastic 通过将流程分解成小的分支来管理版本控制，然后仅在准备好投入生产时对分支进行版本控制。“让版本控制与所有部分合作，但驱动过程，”桑托斯说。

为了促进这一过程，开发团队为 Mac、Linux 和 Windows 操作系统构建了自己的 GUI。GUI 提供了对分支和合并的可见性，使过程变得直观。所有这些都是在拥有大量数据的环境中完成的。塑料从来没有把大小限制在它的回购上；桑托斯说，许多客户都是游戏开发商，对他们来说，4TB 回购不是什么大事。

去年，该公司推出了一款全新的产品:mergebots。这些合并机器人充当自动化的指挥者，在生产过程中管理分支。mergebot 能够跨 CI/CD 流程中使用的所有软件管理您的存储库的所有分支。它通过问题追踪器的更新来监控流程，并实现所有流程的自动化。

我们认为这是为你的管道团队雇佣一个合并机器人，桑托斯说。这个想法来自于问一个问题“我们怎样才能给我们的软件增加真正的价值？”通过安装 mergebots，开发人员可以腾出时间来提高工作效率，推动产品向前发展，而不是在吉拉提交分支或监控票证。
