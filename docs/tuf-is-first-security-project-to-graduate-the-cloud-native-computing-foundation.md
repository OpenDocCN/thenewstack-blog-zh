# TUF 是第一个完成云计算本地计算基础的安全项目

> 原文：<https://thenewstack.io/tuf-is-first-security-project-to-graduate-the-cloud-native-computing-foundation/>

本周，[更新框架(TUF)](https://theupdateframework.github.io/) 完成了云计算原生计算基金会(CNCF)的项目开发过程，加入了 Kubernetes、Prometheus、Envoy、CoreDNS、containerd、Fluentd、Jaeger 和 Vitess 的行列。虽然这是第九个达到 CNCF 分级状态的项目，但它是第一个以安全为重点的项目，也是第一个这样做的规范。

顾名思义，TUF 是一个处理软件更新的框架，用于保护软件更新系统并防止不良行为者将恶意或过时的易受影响的软件注入更新中。

“我们设计了 TUF，所以一个组织不需要在他们的运营安全方面完美，”TUF 的联合创始人和“共识建设者”[贾斯汀·卡波斯](https://engineering.nyu.edu/faculty/justin-cappos)在一份声明中说。“如果一家公司不小心公开了签名密钥，有黑客闯入了他们的软件库，或者一名心怀不满的员工叛变，他们所能造成的损失是有限的。深度防御是安全性的关键，软件更新基础架构的安全性是实践中最关键的问题之一。”

TUF 于 2009 年首次建造，并于 2017 年加入 CNCF，但卡波斯表示，尽管加入 CNCF 后技术成熟，但该项目在许多方面受益于这一过程。

“我们对治理之类的事情有了更多的了解。许多这些过程背后的机制得到了澄清。我们在很早的时候就已经做了很多，我们一直试图在这一领域非常积极主动，”卡波斯在接受新堆栈采访时说。

与此同时，TUF 为 CNCF 毕业进程带来了一些挑战，因为它无法获得标准项目所能获得的相同使用统计数据。

“我们不符合项目的正常模式。大多数 TUF 人并不一定和我们交流过或者下载过我们的代码。当你有一个实现，人们使用你的实现，他们给你错误报告，告诉你发生了什么等等。因为我们是一个规范，我们是从所有这些中移除的一个层次，”Cappos 解释道。“我们有一些人们写的博客帖子，谈到他们正在使用它，但他们在哪里使用它以及如何使用它，我们真的不知道。”

不考虑容易获得的统计数据，TUF 可以声称其用户中有许多大牌公司，如亚马逊、微软、谷歌、Cloudflare、Datadog、DigitalOcean、Docker、IBM、RedHat 和 VMware。TUF 也是 Docker 的[公证](https://docs.docker.com/notary/getting_started/)的规范，也是 CNCF 的项目，还有一个叫 [Uptane](https://uptane.github.io/) 的项目，是 TUF 的一个汽车应用，被广泛使用。Cappos 说，多年来，TUF 被视为“开放源代码、无阻碍、免费供所有人使用的通用更新程序”，从而获得了广泛的采用。

现在，已经满足了所有的[毕业标准](https://github.com/cncf/toc/blob/master/process/graduation_criteria.adoc)，TUF 可以将 CNCF 毕业添加到它的成就列表中，卡波斯指出，这还包括 CNCF 项目中的第一个银 [CII 最佳实践](https://bestpractices.coreinfrastructure.org/projects/1351)徽章。

“我真的很兴奋，TUF 是第一个达到这种成熟程度的安全项目。我认为这表明安全是随着技术的成熟而出现的。“这对 CNCF 来说是一个好兆头。我们真的很高兴成为这一过程的一部分，我们希望更多的安全项目将加入我们的行列。”

云计算原生计算基金会、Red Hat 和 VMware 是新体系的赞助商。
