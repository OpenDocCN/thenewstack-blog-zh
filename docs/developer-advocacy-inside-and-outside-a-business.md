# 企业内外的开发者宣传

> 原文：<https://thenewstack.io/developer-advocacy-inside-and-outside-a-business/>

在本期[新堆栈制造商](https://thenewstack.io/podcasts/makers)播客中，我们与 [GitHub](https://github.com/) 的开发者代言人 [Brian Douglas](https://twitter.com/bdougieYO) 坐下来讨论开发者代言人的角色。Douglas 将开发人员倡导者定义为向开发人员“宣讲”新产品和新功能的人，确保有一条清晰的途径来利用这些产品。

[Github 和开发者倡导的内外业务](https://thenewstack.simplecast.com/episodes/github-and-developer-advocacy-in-and-outside-a-business)

“我认为，作为工程师，很多时候我们可以建造超级复杂的东西，但永远不知道人们是否真的能使用它。因此，作为一名开发者倡导者，我的工作是确保我可以使用它，然后确保初学者、中级和高级人员也可以使用它，”道格拉斯说。

这可能是识别文档中的缺口或缺失的边缘案例。他说，这也是关于能够一次开始——30 到 60 分钟真正开始使用一种工具。

在 GitHub 的 1200 多名远程员工中，有两名开发人员和一名社区人员。道格拉斯的角色在理论上被定义为 50%的编码和 50%的开发倡导者。开始时，他在营销团队，但现在他直接向新的产品经理汇报。他的角色让他整天与项目经理、工程、营销和客户打交道。

GitHub 做极端的狗食。甚至财务和市场营销也可以处理拉式请求。对于所有形式的 GitHub 用户，Douglas 发现自己不断地询问用户知道什么，他们在使用什么，并确保他们知道新的功能。

道格拉斯的大部分工作是寻找品牌倡导者——包括内部和外部的。这可以是轮询用户故事，了解每个功能是如何使用的，或者确保用户有他们需要写的关于新功能的任何内容。

一个人可以做什么来帮助其他人在科技行业获得优势？成为品牌和开发商的倡导者，道格拉斯说。他认为回报社区的最好方式是写内容和分享你的经验。他说缺乏初学者的内容，人们可以通过谷歌来开始学习。

道格拉斯最后说，再也没有竞争优势这种东西了。我们都需要一起努力。
