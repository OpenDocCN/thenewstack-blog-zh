# 上下文:LaunchDarkly 的轨迹，下一个谷歌云

> 原文：<https://thenewstack.io/context-launchdarklys-trajectory-google-cloud-next/>

[launch ryly 的轨迹，谷歌云下一个](https://thenewstack.simplecast.com/episodes/launchdarklys-trajectory-google-cloud-next)

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，这是一个总结本周新闻的每周播客。本周，我们采访了奥克兰新堆栈记者 [TC Currie](https://twitter.com/tccurrie) ，他参加了本周在奥克兰举行的[launch darky](https://launchdarkly.com/)的[轨迹](https://www.trajectoryconf.com/)会议。然后在节目的稍后部分，我们自己的 [Alex Williams](https://twitter.com/alexwilliams) ，The New Stack 的创始人兼发行人，和 [Joab Jackson](https://twitter.com/Joab_Jackson) ，TNS 的执行主编，讨论了本周在旧金山来自 Google Cloud Next 的新闻。《新书库》的编辑主任利比·克拉克主持了这场秀。

LaunchDarkly 是一个功能管理平台，为 CI/CD 管道提供功能标记。早在 2 月份，Currie [在我们的 Makers 播客上采访了 launch crystally 的首席执行官兼联合创始人 Edith Harbaugh](/eliminate-the-stress-of-deployment-with-launchdarklys-feature-flags/) ，她是今年云-Now 的“[最佳云女性](https://cloud-now.org/7th-annual-top-women-in-cloud/)”奖的获奖者之一。

然后在节目的后半部分，我们谈论谷歌下一次会议。我们的云服务记者 Mary branscome 为这个节目带来了巨大的能量和报道。我们谈到了谷歌的 [Anthos 混合云平台](https://thenewstack.io/anthos-kubernetes-infrastructure-to-make-developers-more-productive/)，它于本周结束了测试，以及[它与七个领先开源项目的新合作关系](https://thenewstack.io/google-offers-7-popular-open-source-data-projects-as-managed-services/)，它将开始提供完全托管的服务。

## 提到的其他故事

*   [CloudBees，Atos 提供完整的 CI/CD 服务模式](https://thenewstack.io/cloudbees-atos-partner-to-offer-a-ci-cd-stack-for-google-cloud/):服务和解决方案提供商 Atos 与 Jenkins 提供商 CloudBees 联手，共同提供集成服务，作为持续集成和交付(CI/CD)的托管平台。Atos 的应用程序转换服务可帮助团队实现应用程序以及应用程序交付和管理方式的现代化；由 Jenkins 和 Jenkins X 支持的 CloudBees 的云原生 CI/CD 平台可以在云中或内部运行 Google Cloud technologies 谷歌为 Kubernetes 和 Istio 提供云技术。
*   [Gremlin-Spinnaker 集成用于自动化混沌工程](https://thenewstack.io/gremlin-spinnaker-integration-for-automating-chaos-engineering/):开源多云连续交付平台 Spinnaker 背后的人正在与混沌即服务供应商 Gremlin 合作。与 Gremlin 的集成意味着 Spinnaker 用户可以跨多个云提供商自动进行混沌实验，包括 AWS EC2、Kubernetes、谷歌计算引擎、谷歌 Kubernetes 引擎、谷歌应用引擎、微软 Azure、OpenStack 等。
*   [OpenStack“Stein”承诺 5 分钟 Kubernetes 部署](https://thenewstack.io/openstack-stein-focuses-on-kubernetes-bare-metal-nfv-and-more):在下个月丹佛举行的开放基础设施峰会之前，Open stack 发布了 Stein，这是其同名开源云基础设施软件的第 19 个版本。Stein 带来了数十项增强功能，包括裸机网络和管理的新功能，使用 GPU 运行高性能工作负载的能力，改进的容器和 Kubernetes 功能，以及支持 5G、边缘计算和网络功能虚拟化(NFV)用例的网络升级。
