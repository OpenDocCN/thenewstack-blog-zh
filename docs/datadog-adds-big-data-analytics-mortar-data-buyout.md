# Datadog 通过砂浆数据收购增加大数据分析

> 原文：<https://thenewstack.io/datadog-adds-big-data-analytics-mortar-data-buyout/>

总部位于 SaaS 的云监控初创公司 [Datadog](https://www.datadoghq.com/) ，两周前刚刚宣布在超额认购的 C 轮融资中筹集了 3100 万美元，已经花费了一部分资金收购总部位于纽约的大数据初创公司 Mortar Data。

Datadog 的首席产品官阿米特·阿加瓦尔(Amit Agarwal)开玩笑说，这笔钱在公司的口袋里烧了一个洞。

Agarwal 说，Datadog 已经将 Mortar Data 作为客户使用了一段时间，并看到其机器学习能力特别有助于 Datadog 提供更深入的分析。

Mortar Data 的平台基于允许开发人员和数据科学家通过 Pig 和 Python 编程语言的定制混合，轻松而廉价地利用 Hadoop。用户可以从强大的管道中集成不同的数据源，并使用定制的机器学习应用程序执行复杂的分析。今年 1 月，它宣布将 Apache Spark 集成到其平台中。

“使用 Mortar 数据技术，我们将能够加速我们领域的发展，这是从我们的客户那里获取大量数据，并根据长期模式向他们提供见解，”Agarwal 说。

Datadog 为客户监控成千上万的服务器，以获取应用程序或特定云实例的性能等指标。

“考虑到规模，有时人们很难看到数据中出现的问题模式。这就是迫击炮数据的机器学习部分的用武之地。阿加瓦尔说:“迫击炮数据可以确定长期趋势，例如，如果这一群机器表现非常不同，就可以提醒人们可能存在他们可能看不到的问题。”

Agarwal 说，Mortar Data 的 6 名员工将加入 Datadog 的 75 人团队，致力于建立更深层次的能力。尚不清楚这是否会带来新产品。

Mortar Data 宣布其服务将于 5 月 15 日关闭，并发布了使用工作流引擎 Luigi 在 Mortar 服务之外运行 Mortar 代码的指南。

Constellation Research 的首席分析师和创始人 R“Ray”Wang 表示，收购非常有意义，因为你可以利用 Mortar 的 piping 和 DataDog 的 analytic 的框架，并简化将两者结合在一起的过程。

鉴于数据量巨大，Constellation Research 预测，世界上 90%的数据将在明年创建，到 2020 年将有 800 亿个传感器。这将是一个巨大的数字。

“这个想法是不断提出见解，帮助 IT 运营部门识别他们可能没有看到的模式，并对他们的数据提出问题，这可能有助于他们继续改善运营，并改善他们的战略思维方式，”他通过电子邮件表示。

Datadog 专注于云计算，但也关注从传统基础设施向云迁移的公司。它汇集了来自服务器、数据库、应用程序、工具和服务的数据，在自定义仪表盘中呈现应用程序、工具和 web 服务的统一视图

它的客户包括 Spotify、斯坦福大学、Salesforce 和美国职业足球大联盟。

“这实际上是现代建筑服务于现代建筑，”正如 Index Ventures 的负责人 Shardul Shah 在宣布融资时所说。

Datadog 已经集成了近 100 种现代云应用中常用的技术，包括亚马逊 Web 服务(AWS)、微软 Azure、谷歌云平台、Red Hat OpenShift 和 OpenStack。

此次收购将帮助 Datadog 赶上已经在兜售分析能力的竞争对手，最著名的是 [New Relic](http://newrelic.com) 。这是一个拥挤的空间，有[边界](https://gigaom.com/2011/11/15/boundary-launches-with-a-new-network-monitoring-angle/)、[云](https://gigaom.com/2014/01/28/which-is-cheaper-for-your-work-amazon-or-google-cloud-ask-cloudyn/)、[日志](http://www.loggly.com/)和 [Splunk](http://www.splunk.com/) 。

谷歌正在将收购的 Stackdriver 应用于跨竞争平台的监控服务中。
