# 云原生部署给开发人员带来了新的复杂性

> 原文：<https://thenewstack.io/cloud-native-deployments-bring-new-complexities-to-the-developer/>

许多组织发现，迁移到云原生环境变得比过去更容易了。然而，一旦大规模部署开始，复杂性和随之而来的挑战仍然可以克服。

在这一集由 TNS 创始人兼发行人 Alex Williams 和 TNS 执行主编 Joab Jackson 主持的 New Stack Makers 播客中，应用部署标准是今天讨论的话题。特邀嘉宾有 [Bruno Andrade](https://www.linkedin.com/in/bruno-a-a369028/) ，创始人， [Shipa](https://www.shipa.io/) ，Kubernetes 的框架提供商；以及 [Bassam Tabbara](https://www.linkedin.com/in/bassamtabbara) ，创始人兼首席执行官 [Upbound](https://www.upbound.io/) ，为多集群管理提供通用控制平台。

[云原生部署给开发者带来新的复杂性](https://thenewstack.simplecast.com/episodes/cloud-native-deployments-bring-new-complexities-to-the-developer)

“我想说的是，原生云非常关注基础架构的准备、基础架构的设置和群集的准备。有了这些优秀的工具，事情变得越来越容易，越来越好，”安德拉德说。“但人们意识到的一件事是，现在您必须部署应用程序并维护这些应用程序，而应用程序不再是您部署到单个虚拟机中的那个整体。”

例如，今天的软件部署通常依赖于无状态环境中运行的大量微服务。安德拉德说，由于数百甚至数千个微服务需要可能几十个开发者的支持，“复杂性开始了”。“我们看到人们非常关注如何部署我的应用程序？不仅如此，我如何操作我的应用程序？”安德拉德说。

一旦开发了应用程序，尽管云原生基础架构和操作工具有所改进，但许多因素仍然会延迟其发布。Tabbara 说:“一旦[开发人员]得到了他们想要推向生产的东西，就要考虑为了生产而必须发生的所有事情，以及为了发生这些事情而需要的所有系统。”。“因此，根据他们使用的云提供商，或根据他们是否运行混合基础架构，或根据哪种策略、治理和法规遵从性[已到位]，在一些组织中，实际上可能有 120 人需要参与。”

开发人员报告说，他们甚至“感觉有时开发人员忘记了应用程序的开发人员部分”，必须更多地参与到操作中。“现在，[开发人员]必须深入了解基础设施和架构是如何工作的，同样，如果你正在部署第一打应用程序，这很好，因为你可能可以管理，”Andrade 说。“但如果你正在部署，比如为成百上千的用户提供成千上万的服务，开发者的体验就会受到影响，复杂性也会增加。这是我实际看到的一个大话题，关于云原生如何影响当今的开发人员工作效率。”
