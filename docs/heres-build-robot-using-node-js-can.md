# 为什么应该使用 Node.js 构建机器人:因为你可以

> 原文：<https://thenewstack.io/heres-build-robot-using-node-js-can/>

关于物联网的增长，最令人兴奋的部分是制作新事物带来的创造力和娱乐性。只有当我们拥抱混乱和无拘无束的快乐时，最好的创新才会出现。这是像 [NodeBots](http://nodebots.io/) 这样的项目背后的驱动力，这是一个覆盖全球的 [Node.js](/tag/node.js/) 驱动的机器人社区，它将人们聚集在一起，在 Raspberry Pi、Arduinos 和其他低成本微控制器上构建他们的创作。

在最新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/) 播客中，我们采访了 Salesforce 首席开发人员布道者[艾米丽·罗斯](https://twitter.com/nexxylove)和[图灵软件与设计学院](https://www.turing.io/)前端工程项目总监[史蒂夫·金内](https://twitter.com/stevekinney)来探索世界各地的开发人员是如何通过 [NodeBots](http://nodebots.io/) 社区来创造诸如遥控猫和咖啡烘焙机器人之类的东西的。TNS 创始人亚历克斯·威廉姆斯和业内资深人士[达里尔·塔夫特](https://www.linkedin.com/in/darryltaft)在 Node.js Interactive 2016 大会上接受了采访:

[这就是为什么你应该使用 Node 来建造一个机器人。因为你可以](https://thenewstack.simplecast.com/episodes/heres-why-you-should-build-a-robot-using-node-js-because-you-can)

金内指出，随着成本壁垒不仅从 JavaScript 社区，而且从整体上来说从开源社区不断被消除，人们有兴趣看看日常爱好者将开始创造什么。

“现在制造相当强大的微控制器和片上系统要容易得多，也便宜得多，这些微控制器和片上系统能够运行 V8 或 JavaScript 或类似的语言，而这些语言过去是禁止使用资源的。再加上 JavaScripts 非常容易访问，我认为这是推动我们处理计算方式快速转变的两个因素，”Rose 补充道。

作为最早参与 NodeBots 的人之一和 NodeBots 旧金山 GitHub 的核心贡献者之一，Rose 指出 NodeBots 从一开始就是一个社区的努力。Rose 说，一个人“用 Node.js 制作了一个带有面包机和热风枪的咖啡烘烤器”。“我认为它的核心是向人们展示进入门槛比熟悉这项技术的人所期望的还要低。”
