# 软件规划和部署应该预见到失败

> 原文：<https://thenewstack.io/software-planning-and-deployment-should-anticipate-failure/>

无论我们做了多少准备，部署并不总是按计划进行。在本期的[新堆栈制作者](/podcasts/makers)播客中，由新堆栈的创始人兼发行人 [Alex Williams](/author/alex/) 、 [Isabelle Miller](https://www.linkedin.com/in/isabelle-miller/) 、[launch darky](https://launchdarkly.com/?utm_content=inline-mention)的软件工程师主持，描述了 DevOps 团队如何构建流程来帮助消除发布周期中不必要的意外——以及为什么他们不需要有压力。

米勒说，自 2020 年初加入 LaunchDarkly 以来，她发现的一个主要问题是，当事情确实出错时，制定程序非常重要，“因为事情会出错，”她说。

[软件规划和部署的不同视角](https://thenewstack.simplecast.com/episodes/a-different-perspective-on-software-planning-and-deployment)

米勒说:“你需要能够尽快处理这个问题，并在事情失去控制之前将伤害降到最低。”“所以，在 LaunchDarkly 工作的一个好处就是我可以使用我们的产品。LaunchDarkly 的功能标志的一个奇妙之处在于，你可以直接关闭它。”

“我的团队曾经有过一次内部事故，当时一些东西停止工作，支持团队从一个客户那里报告给我们，”Miller 解释说。“能够关闭该功能是一种解脱，因此我们可以深呼吸，后退一步，尝试找出问题所在，而不是担心它会在调查过程中持续影响客户数据和体验。”

米勒的大部分知识和经验都来自于分析政府和技术交汇处的数据和相关模式。她的背景也使她能够批判性地思考私营部门和行业与政府在软件开发和一般技术方面的区别。

米勒回忆了她在德国科技公司(Tech for Germany)为德国政府所做的工作，她将其描述为“德国对数字政府机构的低投资回应”。

米勒说:“当时，这似乎是一个非常酷的机会，让我重拾对政策的热情，我会说，深入了解德国政府的社会化努力真的很酷——尽管出于各种原因，这也让我相信，从个人角度来看，我进入私营部门绝对是正确的。”“快速交付的能力——或者更确切地说，没有能力——可能是最主要的原因。我认为，很多人，尤其是初创企业和私营部门的人，在批评政府行动缓慢或未能承担风险时，无法理解的一点是，激励结构是如何截然不同地建立起来的，这样，如果你的初创企业失败了，你就可以为新企业筹集资金。”
