# HPE 一圈:一个多云平台，引擎盖下有 Docker 和 Kubernetes

> 原文：<https://thenewstack.io/hpe-onesphere-a-multi-cloud-platform-with-docker-and-kubernetes-under-the-hood/>

[HPE OneSphere:一个拥有 Kubernetes 的多云平台](https://thenewstack.simplecast.com/episodes/hpe-onesphere-a-multi-cloud-platform-with-kubernetes-under-the-hood)

[HPE OneSphere](http://hpe.com/OneSphere) 是一个为 Kubernetes 建立的平台，当时 Kubernetes 已经成为一种足够稳定的技术，使其能够在一个 SaaS 部署中为 it 运营和开发人员服务。

随着 [Docker](https://www.docker.com/) 和 [Kubernetes](http://kubernetes.io) 的出现， [HPE OneSphere](https://www.hpe.com/us/en/solutions/cloud/hybrid-it-management.html) 讲述了共享相同环境的 IT 和开发人员体验的新维度，使用容器技术进行打包和版本控制，以允许跨不同基础设施使用多个组件。

可移植性是使多云方法可行的本质。容器本身是可移植的，可以在 SaaS 平台上管理，比如 HPE OneSphere。Kubernetes 使得跨云平台和内部基础设施运行 HPE OneSphere 服务成为可能。

最近，我们采访了 HPE 总监、杰出的技术专家和首席设计建筑师布莱恩·雅克特，请他告诉我们更多关于 HPE one sphere 及其所秉持的理念。特别是，我们希望了解它如何支持这样的信念，即客户是数字意识的，因此，企业必须在他们为应用程序开发、部署和管理建立的架构中受到数字驱动。

听听 Jacquot 的描述，各行各业各种规模的公司都在成为技术驱动型企业。他们想:

*   以数字化的方式与客户互动，这意味着通过数字化客户体验创造竞争优势。
*   利用数字化转型创造新的收入来源，并转变产品、服务和运营模式，以更有效地参与竞争。
*   优化运营-数字化供应链，并通过移动性和生产力应用为员工提供支持。

这些具有数字意识的人使用的平台可能是云服务、内部部署或不同架构的某种组合。他们的应用程序可能是虚拟化的或者在容器上。HPE 在一个混合的背景下描述市场，他们非常清楚地坚持这种语言。不管这种语言的本质是什么，微服务将会继续存在，而 T2 的 Kubernetes 将会成为底层架构，容器将会是真理的核心单元。
