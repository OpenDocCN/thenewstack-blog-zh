# 持续性成为木桶，并专注于将 Hadoop 引入企业

> 原文：<https://thenewstack.io/continuuity-becomes-cask-and-keeps-focus-on-bringing-hadoop-to-the-enterprise/>

围绕 Hadoop 构建的数据应用平台 Continuuity 正在开源其平台，推出新流媒体工具的开发者版本，并更改其名称。[连续性](http://www.continuuity.com/)现在是木桶。Cask 将继续致力于将 Hadoop 引入企业，并增加让开发人员使用 Hadoop 进行实时活动的功能。

该公司看到 Hadoop 在企业中发挥着越来越重要的作用。大型企业通常使用传统数据和关系数据库来实现 CRM、供应链管理和其他传统企业应用。“我们相信，许多企业将通过类似类型的应用程序使用大数据，除非它们会有根本的不同，因为它们将更多地与数据有关，”公司首席执行官约翰·格雷说。

然而，问题是 Hadoop 很难使用，他说。像格雷的出生地脸书和他的联合创始人工作的雅虎这样的大公司，拥有围绕 Hadoop 这样的开源技术构建系统的资源、专业知识和兴趣。然而，企业、初创公司和小型企业没有人才或资源来推行类似的计划。“在个人英雄行为的支持下，我们在脸书做了很多伟大的事情，但这并不能在整个商业世界中推广，”他说。

木桶是激光聚焦于企业中的开发者。“我们坚信，开发者将成为围绕大数据创造颠覆性应用模式的个人，”格雷说。

该公司的主要产品是其数据应用平台，帮助用户以更智能的方式构建使用 Hadoop 的应用程序，而不仅仅是构建低成本的大型数据仓库。

Cask 还提供 Cooper，这是一个帮助用户快速轻松地配置 Hadoop 集群的工具。部分由于其分布式特性，Hadoop 可能难以部署。事实上，木桶把库珀建成了一个内部工具，但客户开始询问它。Cooper 在公共、私有和 OpenStack 云中工作，因此开发人员可以使用它在这些位置中的任何一个提供 Hadoop。

随着名称的改变，木桶还推出了一项与美国电话电报公司实验室合作的联合开发项目，名为 Tigon，用于 Hadoop 上的实时流处理。

格雷指出，整个行业都在努力让人们以更实时的方式使用 Hadoop。他说，这是有道理的。“传统上，您的数据放在其他地方，而您的处理放在不同的机器上，因此您必须将数据从一台机器移动到另一台机器。Hadoop 的固有优势是它是一个分布式系统。你不必移动数据来处理它，”他说。

然而，如果用户在数百台机器上存储大量数据，并希望进行流处理，那么数据和流处理引擎之间将会出现瓶颈，他说。Tigon 及其竞争对手 [Apache Storm](http://storm.incubator.apache.org/ "Storm") 背后的想法是构建一个 Hadoop 原生的流处理引擎。然后，用户可以在 Hadoop 集群中的多个节点上运行流处理，通常是在数据所在的同一台机器上。

Gray 说，Databricks 和 T2 的 Spark 和 Cloudera 的其他努力表明，业界开始将 Hadoop 视为一个可以构建各种功能的操作系统。

虽然木桶公司已经开源了它一直以来开发的其他技术，但它今天宣布也开源了它的核心 CDAP 引擎。它是在 Apache 许可下发布的。

“我们知道我们将是一个开源公司，”格雷说，从一开始就谈到了木桶的意图。然而，部分是为了更快地行动，该公司决定在向社区发布之前尝试自己开发一个可靠的产品。
