# 为什么运行良好的 DevOps 依赖于不变的基础设施

> 原文：<https://thenewstack.io/why-well-oiled-devops-rides-on-immutable-infrastructure/>

采用“不可变的基础设施”已经成为改进 DevOps 过程和文化的可行方法。通过在应用程序部署和管理中引入更多的标准化，不可变基础设施有助于在开发人员、操作人员、安全团队和其他利益相关者之间形成更好的协作环境。

[德勤咨询](https://www.linkedin.com/in/michael-liedike-3791b531/)的创新和平台团队经理迈克·李代克说:“不可变的基础设施使您能够在整个系统群中拥有一致的环境，这使您的部署更简单、更可预测。”。“它允许您更加一致地进行测试，并从开发到测试再到生产提升您的环境。”

换句话说，采用不可变的基础设施通常是高功能开发运维的标志。

[运行良好的 DevOps 依赖于不可变的基础设施](https://thenewstack.simplecast.com/episodes/well-oiled-devops-rides-on-immutable-infrastructure)

在本期[中，Liedike 在加利福尼亚州圣克拉拉的](https://thenewstack.io/podcasts/makers) [Palo Alto Networks](https://www.paloaltonetworks.com/) 工作室现场录制的 New Stack Makers 播客提供了关于采用不可变基础架构对您的组织意味着什么的进一步见解和分析。

描述不可变基础设施如何工作的一个好的起点是首先详细描述它是如何工作的——或者更具体地说，什么是“可变”基础设施，以及它与不可变基础设施相比有什么不同。Liedike 以 Apache 服务器为例，说明了管理员如何通过安装带有配置管理工具的最新版本的 Web 服务器软件来升级服务器。Liedike 说，问题是“在 1000 个实例中，你有很多出错和不一致的空间。”Liedike 说:“使用不可变的基础设施，而不是在适当的位置进行这些更改，您实际上可以建立一个新的服务器，所有的升级都已经到位，然后部署您的系统并淘汰旧的系统。”

不可变的基础设施也与基础设施作为代码的概念相联系，因为组织本质上使用代码来管理基础设施。Liedike 说:“基础设施即代码允许您提供所有的基础设施组件，如您的实例、网络和安全性，并将其推送到您的开发环境中。“然后，当您将这些升级到开发、测试和生产时，您知道您可以轻松地以一致的方式重复您的环境，一直到生产，并让您的应用程序开发人员知道，无论他们处于什么环境，他们都将处于一致的环境中，在部署应用程序时不必担心这一点。”

最终，不可变基础设施的很大一部分价值在于应用程序开发人员如何将代码一致地部署到不会改变的环境中。“你总是知道你的代码将被部署到哪里，如何被部署，系统将如何运行这些代码，”Liedike 说。“系统管理员没有进行配置或补丁更改，而应用程序开发人员并不知道，或者即使他们知道，也没有准备好让这种更改在他们的系统上发生，这就会导致问题。因此，当您构建新系统时，不可变基础架构所做的就是为应用程序的运行提供一致、可靠的基础架构。”
