甲骨文为何选择Kubernetes

原文：https://thenewstack.io/oracle-embraced-kubernetes/#0001-01-01


甲骨文为何拥抱库伯

不要问谁拥抱，问谁不拥抱。因为似乎每个人都在用Kubernetes。包括Oracle，无可争议的数据库之王，它正稳步(并且悄悄地)进入公共和私有云空间。

Oracle从事容器管理已经两年多了。该公司通过2016年收购stacken engine的获得了集装箱专业技术和解决方案。在Kubecon + CloudNativeCon 2017上录制的最新一期 The New Stack Makers 播客中，我们采访了StackEngine的联合创始人 Bob Quillin ，他现在担任副总裁领导 Oracle Container Group ，以更好地了解Oracle参与Kubernetes社区的情况。

Quillin说:“客户正在寻找开放社区驱动的云中立解决方案，以便在Kubernetes上运行他们的编排和容器管理解决方案。

他说，人们希望避免被云锁定。然而，与此同时，这些客户还希望与堆栈的其余部分进行一定程度的集成，以便他们能够在企业平台之间获得无缝体验。云本地计算基础”和像Kubernetes这样的开源解决方案让他们有机会在任何他们想去的地方运行它。这使他们能够灵活地在多种不同的云中运行，包括本地云，”Quillin说。

甲骨文去年加入CNCF成为白金会员。该公司也是CNCF几周前发起的整合项目的一部分。甲骨文有许多工程师在从事CNCF项目。

“CNCF希望致力于此，我们是解决方案的一部分，”奎林说。
