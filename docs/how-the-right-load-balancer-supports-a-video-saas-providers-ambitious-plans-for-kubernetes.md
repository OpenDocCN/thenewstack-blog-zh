# 合适的负载平衡器如何支持视频 SaaS 提供商对 Kubernetes 的宏伟计划

> 原文：<https://thenewstack.io/how-the-right-load-balancer-supports-a-video-saas-providers-ambitious-plans-for-kubernetes/>

Citrix 赞助了这个播客。

如果说 [8×8](https://8x8.vc/) 提供其视频和语音软件即服务(SaaS)的能力需要高带宽，这是一种保守的说法。虽然该公司一直在寻求提高其吞吐量的方法，但新冠肺炎疫情给该公司的带宽需求带来了巨大的压力，以维持和改善其用户的网络体验。例如，今年早些时候，该公司的[流量在不到一个月的时间里激增了 50 倍](/how-kubernetes-prepared-8x8-for-a-50x-spike-in-videoconferencing-usage/)。

最终，8×8 的 DevOps 在很大程度上依赖于 Kubernetes 基础设施和负载平衡器以及应用交付解决方案提供商 [Citrix](https://www.citrix.com/networking/microservices.html) 提供的其他支持，以帮助管理前所未有的流量。

在本视频中，[The New Stack Makers](/podcasts/makers)podcast， [Alex Williams](/author/alex/) ，The New Stack 的创始人兼发行人，与 Citrix 的产品营销、cloud native、DevOps、安全、分析和网络高级总监 [Pankaj Gupta](https://www.linkedin.com/in/pankajgupta100/) 以及 8×8 的工程、云研发总监 Lance Johnson[进行了交谈。他们讨论了 Kubernetes 和 Citrix 如何帮助 8×8 实现并保持敏捷性，同时为其协作产品组合提供更好的客户体验](https://www.linkedin.com/in/lancejohnsn)

[合适的负载均衡器如何支持视频 SaaS 提供商对 Kubernetes 的宏伟计划](https://thenewstack.simplecast.com/episodes/how-the-right-load-balancer-supports-a-video-saas-providers-ambitious-plans-for-kubernetes)

约翰逊说，以数据传输来衡量，8×8 在峰值时的吞吐量为每秒 50 千兆比特，因此代表了“最终用途的巨大飞跃”。然而，很大程度上得益于其负载平衡器功能和 Kubernetes 环境，8×8 不仅仅能够维持客户和开发人员的体验。

例如，Citrix 的 Netscaler 负载平衡器“完全符合要求”，Johnson 说。“它能够提供我连接流量所需的一切。”

Citrix 负载平衡器支持在帮助支持开发人员体验方面也发挥了关键作用。

“我们的主要要求是，当开发人员迁移到原生 Kubernetes 时，无论如何都需要相同的体验——他们甚至不知道他们被部署在哪里，只知道他们的清单定义了他们的工作流，并且可以工作，”Johnson 说。“所以我们想让他们留在那里，让他们有更好的体验。Citrix 是提供相同体验的最佳产品，这种体验与我们在云中或本地部署时获得的体验相同。”

古普塔解释说，事实上，Citrix 客户非常欣赏负载平衡器的一致性。“学习曲线较低，功能一致性也很好，”他说。“他们可以更快地从现有的整体应用程序过渡到基于微服务的应用程序，同时保持运营一致性。”

8×8 向 Kubernetes 和云的转变反映了随着远程员工继续严重依赖网络连接，其他具有大规模基础设施需求(尤其是视频带宽需求)的公司如何满足客户需求。将基础设施迁移到云上当然有助于按需自动扩展网络带宽和其他基础设施需求。“这是我们目前看到的最大趋势，”古普塔说。
