# 无论是开发还是运营，可观察性都至关重要

> 原文：<https://thenewstack.io/dev-ops-doesnt-matter-need-observability/>

每个软件工程师都应该能够回答这些问题:刚刚发生了什么？现在发生了什么？将要发生什么？调试平台[蜂巢](https://honeycomb.io)的首席执行官 Charity Majors 认为，这不仅仅是运营团队需要知道的事情——每个人都需要那种程度的可观察性。构建智能管理数据的服务是每个人的任务，在某种程度上，也是每个人的问题。

新堆栈技术编辑 [Benjamin Ball](https://www.linkedin.com/in/benballjr/) 在 2017 年的 ServerlessConf Austin 上与专业人士就最新版本的[新堆栈制造商](https://thenewstack.io/podcasts/makers)进行了交谈。Majors 谈到了消除 NoOps 的神话，并谈到了一个简单的事实，即监控、测试和调试不再是孤立的行为。正如她所描述的，下一代 DevOps 浪潮不仅减少了响应持续时间，还减少了整体事件的数量，并改进了团队维护和操作软件的方式。但它需要关注人们所扮演的角色，以及扮演这些角色的人们之间的互动。
