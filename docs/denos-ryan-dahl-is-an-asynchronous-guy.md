# Deno 的 Ryan Dahl 是一个不同步的家伙

> 原文：<https://thenewstack.io/denos-ryan-dahl-is-an-asynchronous-guy/>

[Ryan Dahl](https://tinyclouds.org/) 是 [Deno、](https://deno.land/)的联合创始人和创造者，Deno 是一个基于 V8 JavaScript 引擎和 Rust 编程语言的 JavaScript、TypeScript 和 WebAssembly 运行时。他也是 [Node.js.](https://nodejs.org/en/) 的创作者

[Deno 的 Ryan Dahl 是个异步的家伙](https://thenewstack.simplecast.com/episodes/denos-ryan-dahl-is-an-asynchronous-guy)
我们为新栈技术创始人奥德赛系列采访了 Dahl。

“是的，所以我们有一个 [JavaScript 运行时](https://dev.to/snickdx/what-is-the-javascript-runtime-4n09)，”达尔说。“本质上，它与 Node 非常相似。它执行一些 JavaScript，但更现代。”

Dahl 说，Deno 项目始于四年前。他讲述了编写代码如何帮助他重新思考如何开发 Node。Dahl 写了一个现代服务器端 JavaScript 运行时的演示。他认为它不会去任何地方，但果然，它去了。人们对此非常感兴趣。

Deno 有“很多很多”组件，这些组件是它的基础。它是用 Rust 和 C++编写的，带有不同类型的事件循环库。Deno 和 Node 一样有非阻塞 IO。

达尔将他的工作建立在异步技术的使用上。这种信念体系延续到他如何管理公司。达尔是一个不按部就班的人，他以这种方式经营自己的公司。

作为一名工程师，达尔知道他不喜欢被会议打断。工作应该尽可能异步，以避免中断。

达尔说，Deno 公司是在疫情时期创立的。每个人都很遥远。他们经常结对编程，并专注于简短、高效的对话。这是一个很好的社交和深入研究问题的方式。

达尔是如何从编程[到 CEO](https://thenewstack.io/a-technical-founders-story-jake-warner-on-cycle-io-2/) 的？

“我要说这是相对具有挑战性的，”达尔说。我非常喜欢编程。理想情况下，我会花大部分时间在编辑器中解决编程问题。这并不是首席执行官的工作。"

达尔说，随着首席执行官在更大范围内运作，沟通会更多。工程团队需要管理来确保他们有效地合作，为开发人员提供功能和解决问题。

总的来说，达尔过一天算一天。他没有基本的管理理论。他只是想在问题出现时解决问题。

“我的意思是，我的名声就像用非阻塞 IO 和其他东西把异步套接字带到了主流。所以，你知道，异步是根深蒂固的，我正在考虑。当谈到公司组织时，异步意味着我们有轮换的会议安排，以适应不同时区的人。我们做了很多会议录音。所以如果你因为任何原因不能按时到达，你没有在正确的时区，你，你知道，你在接你的孩子，无论什么。你可以回去看录音。所以我们基本上记录了每一次会议，我们尽量让会议简短。我认为这很重要，因为没有人想看几个小时的视频。我们经常聊天。聊天和电子邮件是异步交流的形式，你不需要和人一对一的交流。是的，我想…另一方面就是尽量少开会。就像有…一些情况下你真的需要让每个人都在房间里。我的意思是，当然有些时候你需要这样做。但我尽可能避免这种情况，因为我认为这真的会干扰很多人的工作。”
