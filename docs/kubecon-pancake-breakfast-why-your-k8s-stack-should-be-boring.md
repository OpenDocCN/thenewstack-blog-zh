# KubeCon 煎饼早餐:为什么你的 K8s“栈”应该是无聊的

> 原文：<https://thenewstack.io/kubecon-pancake-breakfast-why-your-k8s-stack-should-be-boring/>

Kubernetes 正在变得无聊，这是一件好事——Kubernetes 上面的东西才是有价值的。

在这个 [The New Stack Analysts](/podcasts/analysts) 播客中，TNS 创始人兼发行人 [Alex Williams](/author/alex/) 邀请 [KubeCon](https://www.cncf.io/kubecon-cloudnativecon-events/) 与会者参加我们的“虚拟煎饼早餐和播客”来讨论“你的堆栈上有什么？”

播客邀请了嘉宾演讲人[Janakiram Associates 首席分析师 Janakiram MSV](https://in.linkedin.com/in/janakiramm)； [Priyanka Sharma](https://www.linkedin.com/in/pritianka) ，总经理，[云原生计算基金会；](https://www.cncf.io/kubecon-cloudnativecon-events/) [Patrick McFadin](https://www.linkedin.com/in/patrick-mcfadin-53a8046) ，Apache Cassandra 首席传道者兼开发者关系副总裁[DataStax](https://www.datastax.com/)；以及[比尔·扎亚茨](https://www.linkedin.com/in/billjzajac)解决方案工程区域总监 [Dynatrace](https://www.dynatrace.com/) 。该小组通过虚拟糖浆和 Kubernetes 交谈。

Sharma 解释说，Kubernetes 可能还没有完全无聊，但它已经非常接近了——这是一件好事。

“Kubernetes 是坚实而成熟的——这相当于无聊。我要对社区和生态系统的成功表示‘敬意’，”她说。“我非常严肃地认为，这不是一件可以低估的事情，因为所有的辛勤工作都是为了使产品稳定可靠。”

尽管 Kubernetes 的核心技术接近乏味，但仍有许多工作要做。许多行业在采用 it 方面仍然落后，如电信行业。

Sharma 表示:“对于电信行业来说，实现基础设施需求现代化和降低运营成本是一个巨大的机遇。

最近发表在新堆栈电子书[“Kubernetes 生态系统的状态”](/ebooks/kubernetes/state-of-kubernetes-ecosystem-second-edition-2020/?utm_source=thenewstack&utm_medium=twitter&utm_campaign=Series3Book1.2-Download)中的研究结果显示，超过五个集群的部署总体上继续增长，但这种现象主要发生在较大的公司。与此同时，集群的增长“实际上是在应用层”，麦克法丁说。“Kubernetes 中围绕数据的社区正在涌现，这是目前缺失的一环。我这周刚刚做了一个关于 Kubernetes 是否准备好接受数据的演讲。剧透:不，它不是——它是为应用程序而构建的。”

预计将会有所改进，以便开发人员可以完成他们的工作，而不会被与基础架构相关的管理任务所困扰。为此，服务网格应该继续帮助“填补 Kubernetes 堆栈中的一个非常重要的空白，”MSV 说。

“在创新方面，我们将会看到服务网络的巨大进步，”他继续说道。“服务网格将是一项非常非常重要的技术，也是 Kubernetes 之上的开发者体验的基础。”

[KCCNC 2020 欧盟虚拟煎饼早餐:为什么你的 K8s 栈’要无聊](https://thenewstack.simplecast.com/episodes/kccnc-2020-eu-virtual-pancake-breakfast-why-your-k8s-stack-should-be-boring)

Kubernetes 的持续采用也将继续改变和塑造 DevOps 在未来的角色。SREs 可能会扮演更重要的角色，而所需的管理员可能会更少。

扎亚茨解释说，一个由 10 名软件可靠性工程师组成的团队通常已经可以支持一个 IT 环境。“当我们谈论规模时，我们不仅仅是从技术角度谈论它，我们还从文化角度谈论它，”他说。“我们还从底层自动化的角度讨论了这一点，为了管理在高度复杂的环境中运行的所有这些不同的东西，必须实现这一点。所以这是一个巨大的文化转变。”
