# API 的发展:当前的漏洞和未来的趋势

> 原文：<https://thenewstack.io/apis-evolution-future-and-vulnerabilities/>

这一集是 Okta 的[新堆栈制造商](/podcasts/makers)系列，涉及与大规模开发和安全相关的所有主题，特邀嘉宾 [Anant Jhingran](https://www.linkedin.com/in/anantjhingran) ，GraphQL API 提供商 [StepZen](https://www.stepzen.com/about) 的首席执行官。在成为 StepZen 的首席执行官之前，Jhingran 拥有丰富的经验，包括在 IBM、Apigee 和 Google 的长期工作，这无疑使他成为 API 及其在当今 DevOps 环境中的作用方面的领先专家。由 New Stack 的创始人和发行人 Alex Williams 和 API 安全公司 Okta 的开发者权益负责人 Randall Degges 共同主持，Jhingran 讲述了 API 是如何发展的，它们对开发者社区的潜力，以及它们的成功在一定程度上是如何暴露漏洞的。

[Okta 系列——API 的演进、未来与漏洞](https://thenewstack.simplecast.com/episodes/okta-series-apis-evolution-future-and-vulnerabilities)

在过去的几十年中，API 在开发人员和运营团队成员如何能够大规模构建和管理应用程序方面发挥了不可或缺的作用。Jhingran 说，API 的基本驱动力是“真正地分解你的业务，并确保你可以真正地进行创新，而不仅仅是由企业的四面墙来驱动”。

Jhingran 描述了一个典型的场景，在这个场景中，创业公司可能不再“受到我们的员工或合作伙伴的创新的限制”。

“为什么第三方不能在我们真正擅长的东西上创新呢？这意味着:我们能分解我们的业务吗？”金格兰说。“我们是否可以拥有其他人可以利用的组件，这样当他们将他们的创新带到我们的能力中时，就可以一起实现双赢？所以这个概念一直保持不变。”

然而，API 如何促进开发运维团队分解的概念已经发生了变化和发展。“[以前]，你与关系非常密切的合作伙伴一起工作，在那里你可以形成一种长期的关系，因此，你可以公开复杂的接口——但他们会投资于它，以便他们能够用它做一些事情。然后，当我们说‘不’的时候，这个世界就出现了，它本身很好，很重要，但要真正驱动这个巨大的引擎，你真的想开放它，让人们能够自助服务，注册并利用你的能力。“例如， [REST](https://en.wikipedia.org/wiki/Representational_state_transfer) 、REST APIs 和其他一切随之而来的东西就是在那里真正起飞的。今天，如果你去看看 [ProgrammableWeb](https://www.programmableweb.com/) 和其他公司，会有成千上万的 API 存在，它们都是基于 REST 的 API。”

今天，第三个变化正在发生，涉及 [GraphQL](https://graphql.org/) ，代表着一种“范式转变”，Jhingran 指出。“这种模式转变实际上是关于前端开发人员和推动体验的开发人员的重要性。推动体验的开发人员正在研究这些新的 JavaScript 技术和 React 技术等。“但他们正朝着这种完全抽象的方向发展，并说，‘看，我们只关心数据，而不关心数据是如何产生的以及其他一切。’GraphicQL 被证明是一种奇妙的新思路。"

就安全性而言，API 在许多方面都是其受欢迎程度的受害者，当应用程序代码中的漏洞被利用时，它们通常成为主要的攻击媒介。退一步讲，Jhingran 指出，如上所述，API 也是创新的基本驱动力。“所以，关键的问题是——我几乎总是看到——在做好业务和保护业务之间找到平衡，这不是人们马上就能做到的事情，”Jhingran 说。“你看到的许多事情的发生，都是因为转盘转动不正确。”
