# VMware 收购 Octarine 以提高 Kubernetes 运行时安全性

> 原文：<https://thenewstack.io/vmware-to-acquire-octarine-to-boost-kubernetes-runtime-security/>

[VMware](https://www.vmware.com) 计划收购 DevSecOps 安全提供商 [Octarine](https://www.octarinesec.com/) ，这两家公司[周三](https://www.vmware.com/company/news/releases/vmw-newsfeed.VMware-Advances-Intrinsic-Security-Strategy-at-Connect-2020-Announcing-Intent-to-Acquire-Octarine-to-Expand-Workload-Security-Solution-into-Kubernetes-and-Creation-of-Next-Gen-SOC-Alliance.43db1b59-0607-41d2-b065-a277bfc170bf.html)在 VMware 的虚拟合作伙伴会议 [Connect 2020](https://www.carbonblack.com/connect20/) 上宣布。

VMware 计划将 Octarine 的 Kubernetes 安全平台并入自己的 [VMware 炭黑云](https://www.carbonblack.com/products/vmware-carbon-black-cloud/)云原生端点保护平台。这些功能将使 VMware 客户能够建立基于内容的策略来保护敏感信息和监控 Kubernetes 工作负载，将安全检查集成到开发生命周期中，并提供本机异常检测和威胁监控。

该公司还将把 Octarine 功能集成到 VMware Tanzu 开发者工具组合中，特别是服务网格软件和开放策略代理。该公司计划潜在地减少客户必须应用于其终端、工作负载、网络、用户接入点和应用的额外传感器的总数。

“DevSecOps 现在正成为许多组织的口头禅，这是有充分理由的。这些范式的转变激发了我们的想象力和创新——照亮了我们的道路，定义了我们的路线图，”Octarine 首席执行官 Shemer Schwarz 在宣布即将进行的收购的博客文章中写道。“VMware 对云计算原生计算和内在安全性的承诺，已通过其产品发布和最近的收购得到证明，这使其成为 Octarine 的理想之家。”

Octarine 提供了许多 Kubernetes 特有的安全技术。例如，Octarine Guardrails 提供了一个准入控制器，它在集群中运行并执行策略。例如，这种软件可以确保一个没有认证的容器不会真正投入生产。

今年 1 月，该公司开源发布了 Kubernetes 通用配置评分系统( [KCCSS](https://www.octarinesec.com/solution-item/kccss/) )，这是一个用于评估与未正确配置 Kubernetes 相关的安全风险的框架。Octarine 的关联 [kube-scan](https://www.octarinesec.com/solution-item/kube-scan/) 使用它，这是一个扫描 Kubernetes 配置和设置的运行时工具，可以识别和排列运行部署中的潜在漏洞。

显然，云原生安全性对 VMware 非常重要。去年 8 月，[公司以 21 亿美元收购了](https://techcrunch.com/2019/08/22/vmware-acquires-carbon-black-for-2-1b-and-pivotal-for-2-7-billion/)云安全公司 Carbon Black。
