# 新的 Stack @ Scale 播客，Show 11:由内而外的敏捷

> 原文：<https://thenewstack.io/agility-at-scale/>

无论是后端运营还是前端开发，规模无处不在。

在这一集的[中，我们来看看用于可扩展开发和运营的新工具和流程。一个是新的开发软件栈，](https://thenewstack.io/podcasts/) [HyperDev](https://hyperdev.com/) ，它承诺减少建立网络应用程序所需的时间，我们从[雾溪软件](https://twitter.com/strd6)的 HyperDev 负责人丹尼尔·x·摩尔那里了解到这一点。我们还与监控软件提供商 Rocana 的首席技术官 Eric Sammer 讨论了运营可扩展性。

[展示 11:实现大规模敏捷性的最佳实践](https://thenewstack.simplecast.com/episodes/show-11-implementing-best-practices-for-agility-at-scale)

Fog Creek 将五月份发布的 HyperDev 吹捧为建立网络应用的最快方式。

“很容易把它想象成开发人员的工具，让开发人员的生活更轻松。实际上比这要宽泛得多。就像我们在开发 HyperDev 中发现的那样，随着障碍变得越来越低，组织中更多的人，你可能传统上不会想到，因为开发人员能够做出贡献，能够构建应用程序，能够解决他们自己的问题，”Moore 说。

Sammer 提供了自己与寻求快速扩张的财富 100 强和 500 强公司合作的经验。他解释说，虽然许多公司希望拥有脸书和谷歌的运营能力，但他们没有支持这种能力的基础设施。“对我来说，我们可以谈论的都是过程。但从技术的角度来看，这也是关于拥有允许这样做的基础设施，”他说。

Sammer 说:“我认为，无论是工程师还是开发人员，当他们开始考虑系统的各个部分，而不是系统的整体视图时，他们会陷入很多麻烦。”

Sammer 建议企业不要采用导致大规模运营时效率低下的流程的工作流，而是要专注于创建更强大的流程，这些流程将在以后支持他们的基础设施走向敏捷性。

“敏捷性可能意味着各种各样的事情。这可能意味着你是一个认证的 scrum 大师，你严格遵循敏捷所规定的流程，也可能意味着你想让它意味着什么，但在我看来，它意味着你做出了改变。你在世界上得到一些东西，”摩尔同意道。
