# 《勇敢的新云》中的神话、传说和宗教

> 原文：<https://thenewstack.io/the-new-stack-analysts-show-15-myth-lore-and-religion-in-the-new-open-cloud/>

谈到我们如何看待我们定义、构建和管理新堆栈基础架构的方式，有一个误区。太多时候，谈话陷入了太宽泛的讨论，无法达成任何有意义的理解。但是，当我们开始谈论神话及其与数据中心的关系时，我们就可以真正开始探索我们的角色，以及我们如何与人们正在开发的新堆栈联系起来。阻止人们推进他们已经开发的基础设施的神话是什么？

昨天，在旧金山举行的[开放数据中心联盟 2014 年预测](http://www.opendatacenteralliance.org/forecast2014/)会议上，我与瑞银集团基础设施首席技术官 Corey Voo 进行了对话，讨论了这个神话。克里斯·斯旺， [CohesiveFT](http://www.cohesiveft.com/) 的首席技术官和乔纳森·金，[century link Technology Solutions](http://www.centurylinktechnology.com/)的云战略和业务发展副总裁。

[#15:《勇敢的新云》中的神话、传说和宗教](https://thenewstack.simplecast.com/episodes/15-myth-lore-and-religion-in-the-brave-new-cloud)

## 显示注释

*   ODCA 成员正在努力开发下一代产品。Docker 和其他下一代技术正在被采用，但重点更多的是基于数量的采用。
*   人们第一次使用云。高级人员不接触使用云服务是什么感觉，这几乎是可怕的。
*   IT 团队的文化被定义为“我们在这里做事的方式”仍然有很大的惯性。银行通常一半是商业，一半是基础设施。如果基础架构变得更薄，那将是破坏性的。但那不会发生。
*   遗产:它的作用是什么？有很多关于移民的讨论。老实说，已经没有多少了。人们现在正在构建云，并连接回云。
*   真的没有真正的开放云。提供商类似于运营商。
*   看到这种不推动开源方法的持续努力是很奇怪的。
*   在政策方面，语义环境得到了更多的关注。
*   正如阿德里安·科克罗夫特所说，组织是由伤疤组织构成的。
*   相信机器人，不要害怕新的自动化功能可能带来的潜在问题。
*   太多时候，是传说而不是法律主宰了一天。
*   开发商的神话是什么？正如 Red Hat 喜欢说的那样，开发者是新的王者。
*   Docker Hub 展示了一种新的开发方式，这种方式更多的是组装而不是从头开始构建。
*   有这样一个神话，即每个问题都是独特的和新的。集装箱化？它从 Solaris 开始就存在了，但是 Docker 已经解决了构建、发布和部署问题。
*   遗留技术的神话是什么？其中出现了一些牧师。与其说是解决问题，不如说是为宗教服务。
*   语言对企业有什么影响？
*   科技语言有时让人想起奥威尔和新话的概念。是流行语控制了语言，也默认了对话。
*   使用模型非常重要，并且正在成为更好地理解云生态系统中不同 c 群体的真正方法。

英特尔是新堆栈的赞助商。
