上下文:“无服务器”是什么意思？

原文：https://thenewstack.io/context-serverless-mean/#0001-01-01

在撰写本文时，无服务器计算是一种新兴的理想，它已经过了萌芽阶段，但还没有完全成型。它有各种各样的拥护者，有些人比其他人更有道理。

但是“无服务器”这个术语到底是什么意思呢？在最新一期的 The New Stack: Context 播客中，我们采访了两位这样的倡导者，他们提出了迄今为止最明智的案例: Michael Hausenblas ，他是 Mesosphere 的开发者和开发者倡导者，也是最近一篇名为《理解无服务器计算》的新Stack文章的作者；还有 Chad Arimura ，Iron.io 的首席执行官和亚历克斯威廉姆斯的前播客嘉宾。

我们与Hausenblas和Arimura的对话是分开的，但他们提出了几乎相同的案例:具体来说，是为了消除当前正在开发的软件与其底层资源和基础设施之间的依赖性。如果你还记得那个时代，你打开你的个人电脑，看到一个“准备好”的提示，你需要参考的所有东西都在内存地图上——这就像是它的现代演变。

当我们开始整个云原生开发范例时，我们的想法是云可以对开发人员隐藏部署和实现的细节。然后，当我们开始整个容器编排业务时，正是带着这个概念，开发人员是做部署和实现的人。Arimura和Hausenblas建议，如果软件实现的细节可以浓缩为更基本的形式，开发人员可以想象出满足服务级别协议基本要求的草案。通过这种方式，自动化部署管道可以测试其容器，失败或通过，在后一种情况下，重新附加必要的细节并将其部署到生产中。

想象一下，一个运营商请求一个开发者开发一个应用程序，这个应用程序的性能在10分制中被评为7分。开发人员不需要知道为什么性能水平位于平均值和最大值的中间。因此，与其说它是“无服务器的”，不如说是“不太像服务器的”也许我们应该在“无服务器”后面加一个星号，然后加一个脚注:“可能包含一些服务器的痕迹。”

然而，更大的问题是，这是否真的是“ DevOps ”也就是说，无服务器意味着开发人员和运营专业人员合作，还是仅仅参与同一管道？
