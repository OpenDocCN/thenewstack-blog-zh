IBM将Express框架留给了Node.js基金会

原文：https://thenewstack.io/node-js-foundation-adopts-express-framework-incubation-project/#0001-01-01

Node.js基金会已经将 Express Node.js框架收归旗下。

Express将是基金会新的孵化项目。去年九月收购了 Express维护者StrongLoop的IBM正在贡献代码。

“表达将作为它自己的独立实体发挥作用；类似于Node.js基金会如何通过开放治理支持Node.js，技术指导委员会、导师和贡献者将有效地支持该框架，”Node.js基金会的社区经理Mikeal Rogers在一封电子邮件中写道。

Express是一个用于构建web应用程序和框架的工具包。它通常与Node一起使用。可以通过 NPM (节点包管理器)安装。它通常与MongoDB数据库和AngularJS前端框架一起使用。Express是最流行的节点框架。在过去的两年里，它已经被下载了5300万次，仅上个月就被下载了5次。

据罗杰斯说，Express的发布节奏和数量将由其贡献者决定。“传统上，这些用户和贡献者更喜欢稳定性，而不是新功能，”他写道。

允许基金会监督Express的部分原因是为了建立一个多样化的贡献者基础，考虑到该框架的受欢迎程度，这一点很重要。

该软件用于PayPal的开源软件kraken.js中，这是Node的企业扩展。它还为 Sails.js MVC(模型-视图-控制器)Web框架和Loopback(node . js API框架)提供了基础功能。

基金会的孵化过程将为Express提供一个开放的治理和贡献协议，这应该会吸引依赖Express的组织，这些组织可能会担心一个供应商主导Express的未来方向。

“作为孵化器的一部分，导师将与Express的贡献者合作，组成一个‘Express技术委员会’，其成员完全由Express的贡献者和相关依赖者组成，”罗杰斯说。“技术委员会将决定谁是主席，如果他们觉得有必要话。当Express从孵化中毕业时，他们也可以选举一名代表加入Node.js基金会技术指导委员会。”

Express最初由TJ Holowaychuk开发，他最近为Lambda无服务器计算服务创建了T2 Apex框架。像Node本身一样， Express framework 是最小的，它提供了一个干净的接口来包含额外的特性作为插件。

Node.js Foundation 是在一年前成立的，目的是作为一个独立的治理机构来管理软件，为JavaScript提供一个服务器端运行时。Node.js基金会孵化器计划下的项目从基金会的技术指导委员会和相关工作组获得援助和治理指导。

IBM是新堆栈的赞助商。
