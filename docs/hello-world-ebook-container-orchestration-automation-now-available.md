# Hello World:我们关于容器编排和自动化的电子书现在已经可以使用了

> 原文：<https://thenewstack.io/hello-world-ebook-container-orchestration-automation-now-available/>

集装箱生态系统已经成为一个非常拥挤的空间。生产中容器采用率和使用率的提高意味着管理和编排空间变得更加重要。当想到在企业中使用容器时，使用容器的任何障碍或复杂性都被放大了。

为了给这个领域带来一些急需的清晰度，新的堆栈已经汇编了一本电子书，“用 Docker & Containers 实现自动化和编排”，这是我们今天发布的。[在这里下载](https://thenewstack.io/ebookseries/)。

在这本电子书中，新的堆栈探索了供应商和从业者在多种环境中自动化、编排和管理他们的容器技术的方式。我们仔细比较了主要的编排工具，包括编排平台和容器即服务，以及编排容器如何成为推进容器更广泛使用模式的关键组件。

我们举例说明了进入编排容器的基本组件，并触及编排如何涉及各种不同的功能，包括集群管理、调度、服务发现、供应、配置管理，甚至超出基础设施和平台的角色。我们将讨论这一领域与虚拟机管理的对比，以及在软件交付、网络、存储和管理领域中有多少相同的概念。

电子书的另一个主要焦点是我们围绕最终用户和供应商的实践和认知所做的原创研究。我们创建了一个定制的自动化和编排调查，主要是为了衡量我们的观众对编排工具功能的期望；我们还要求受访者报告他们当前和预计的工具使用情况、容器采用水平、容器技术在生产环境中的使用情况等等。由此产生的数据描绘了当前编排工具和服务市场的一幅非常有趣的画面，我们期待在未来几个月中进一步构建和分析这些数据。

这是我们在码头和集装箱生态系统系列的第三本电子书，也是我们总共五本电子书中的中间一本。由于下一本电子书将更多地讨论网络、存储和安全性，因此在本书中确立协调和管理的目标非常重要。围绕容器管理的主要概念在本书系列的其余部分都很普遍，这本电子书为理解用户需要对他们的技术和实践做出的选择提供了必要的基础。

这本书在某种程度上对我们来说是一个巨大的挑战，因为编排领域非常热门——可能是目前容器生态系统中竞争最激烈的领域之一。同时，它交叉了许多不同的工具和服务类别，代表了开源软件的一个广阔领域。编排本身涉及如此多类型的行为和实践，难怪供应商如此渴望解决这一问题并吸引开发人员和运营人员的注意。

正如我以前说过的，我们一直在寻找需要更多关注和教育的新主题，我们欢迎任何关于我们下一步应该解决的领域的反馈，即使是在容器生态系统之外。非常感谢您对我们的电子书系列感兴趣。请随时联系我们的团队，提供反馈、想法和对未来的想法。
