# CISA 为零信任云制定安全规则

> 原文：<https://thenewstack.io/cisa-lays-out-security-rules-for-zero-trust-clouds/>

如果你曾经是环城路的强盗或者其他联邦政府的 it 工作者，你应该知道如何通过监管来完成你的工作。有时候，这真的很让人头疼，但其他时候，联邦调查局会提供非常有用的指导。看来，[网络安全和基础设施安全局(CISA)](https://www.cisa.gov/) 及其[云安全技术参考架构(TRA)](https://cisa.gov/publication/cloud-security-technical-reference-architecture) 和[零信任成熟度模型](https://cisa.gov/publication/zero-trust-maturity-model)草案就是如此。

这些文件列出了应对安全世界的指导方针，在这个世界里，传统的网络墙千疮百孔。防火墙在它们的位置上仍然很好，但是如果你今天依赖它们来保证安全，你是在自找麻烦。就问比如[殖民地流水线](https://www.zdnet.com/article/colonial-pipeline-ransomware-attack-everything-you-need-to-know)。

因此，我们必须围绕云安全和[零信任](https://thenewstack.io/how-zero-trust-service-meshes-and-role-based-access-control-can-prevent-a-cloud-based-security-mess/)实施数据保护措施。这些云文档通过解释共享服务、云迁移和云安全状态管理的注意事项，为机构安全迁移到云提供了布局指导。然而，他们的建议不仅仅适用于联邦承包商。

为此，CISA 与[美国数字服务(USDS)](https://usds.gov/) 和[联邦风险和授权管理计划(FedRAMP)](https://www.fedramp.gov/) 合作。除非你有至少三个机构在一起工作，否则这不是一个真正的政府项目——而且希望不要有冲突。

云文档涵盖了云迁移和数据保护的推荐方法。它还介绍了[云安全态势管理(CSPM)](https://thenewstack.io/why-the-age-of-the-cloud-native-security-platform-is-here-to-stay/) ，并列举了在云环境中进行监控、开发、集成、风险评估和事件响应的相关安全工具。

CISA 的零信任成熟度模型帮助机构制定他们的零信任战略和实施计划。其目标是"[防止对数据](https://csrc.nist.gov/publications/detail/sp/800-207/final)和服务的未授权访问，同时尽可能细化访问控制实施。"换句话说，零信任从以位置为中心的模型转变为更以数据为中心的细粒度安全方法。

虽然保护云很困难，但这是一个可以解决的问题。我们有工具。我们知道我们需要做什么。零信任是另一回事。走向零信任并不简单。“从根本上说，”CISA 说，“它需要一个组织围绕网络安全的哲学和文化的改变。通往零信任的道路是一段需要数年才能实现的旅程。”

你可以帮助规划通往这两者的道路。这两份文件现已公开征求公众意见。CISA 网络安全执行助理主任埃里克·戈尔茨坦(Eric Goldstein)解释说，“我们现在正在征求公众意见，以确保我们推荐的云技术现代化和零信任工作分别实现最佳的可见性、灵活性和安全性。”

公众意见征询期已经开始，计划于 2021 年 10 月 1 日星期五结束。您可以通过电子邮件向 tic@cisa.dhs.gov 的[提交您的意见和反馈。意见征询期结束后，CISA 将与利益相关方合作评估反馈意见，并编制这些指导文件的新版本。](mailto:tic@cisa.dhs.gov)
