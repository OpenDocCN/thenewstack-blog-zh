# Eclipse 启动了一个工作组来构建云原生工具

> 原文：<https://thenewstack.io/eclipse-launches-a-working-group-to-build-out-cloud-native-tools/>

以 Java 为基础， [Eclipse Foundation](https://www.eclipse.org/) 推出了 [Eclipse 云开发工具工作组](https://www.eclipse.org/org/workinggroups/eclipse_cloud_development_charter.php) (ECD WG)，这是一个专注于[云开发工具](https://thenewstack.io/category/cloud-native/)的厂商中立机构。Broadcom、IBM、Red Hat 和 SAP 都是该计划的最初支持者。

“今天的开发人员正在为混合云构建应用程序。他们专注于合作和开发的速度，以便从侧面超越他们的竞争对手。然而，混合云操作环境可能会很复杂，Kubernetes、容器、微服务和服务网格为应用程序设计和实施增加了许多新的考虑因素，”Red Hat 开发工具和宣传副总裁 Brad Micklea 在一份声明中解释了成立新团队的必要性。

Eclipse Foundation 调查了开发人员，发现超过 80%的人计划在未来 12-18 个月内创建云原生应用程序。

该小组的重点是通过建立标准和与工具提供商合作来促进云集成开发环境(ide)和基于容器的工作空间管理。它将管理许多现有的 Eclipse 技术，如 [Che](https://www.eclipse.org/che/) 云 IDE、[忒伊亚 IDE](https://theia-ide.org/) 平台、 [CodeWind](https://www.eclipse.org/codewind/) 面向容器的开发工具、[飞船](http://www.dirigible.io/)云开发平台和 [Sprotty](https://www.eclipse.org/community/eclipse_newsletter/2018/october/sprotty.php) 图表框架，等等。

Eclipse 基金会成立于 2004 年，最出名的是由 IBM 首先开发并开源的 [Eclipse IDE](https://www.eclipse.org/ide/) 。它现在被 400 多万开发者积极使用，每月下载超过 200 万次。总的来说，该基金会有超过 370 个合作项目，总计超过 1.95 亿行代码。

云计算工具和平台的世界无疑一直在快速扩张。[云本地计算基金会](https://www.cncf.io/),[Kubernetes](/category/kubernetes/)的所在地，管理着 [22 个处于孵化或毕业阶段的项目](https://www.cncf.io/projects/)，以及[另外 21 个处于早期“沙盒”阶段的项目](https://www.cncf.io/sandbox-projects/)。在 CNCF 看来，这只是可以被视为“云原生”项目的一小部分。“ [CNCF 云原生景观](https://landscape.cncf.io/)”汇集了这一领域的技术，最新统计有 1239 个条目。

云计算原生计算基金会和 Red Hat 是新堆栈的赞助商。
