# 机器的崛起:阿帕奇布鲁克林和自主计算

> 原文：<https://thenewstack.io/rise-machines-apache-brooklyn-autonomous-computing/>

在本文中，云软创始人兼首席执行官

[Duncan Johnston-Watt](http://www.cloudsoftcorp.com/company/people/duncan-johnston-watt/)

提供 Apache Brooklyn 的更新，它最近从 Apache 孵化器中毕业，现在是 Apache Software Foundation 的顶级项目。

Apache Software Foundation 最近宣布， [Apache Brooklyn](https://brooklyn.apache.org/) ，一个与基础设施无关的应用管理平台，[最近从 Apache 孵化器](https://blogs.apache.org/foundation/entry/the_apache_software_foundation_announces83)毕业，现在是一个顶级项目。

这一升级表明，应用程序的可移植性和这些应用程序在多个数据中心的运行状况越来越重要。对于我们如何在一个新的分布式环境中定义应用程序堆栈来说，分布的能力以及为应用程序执行自我修复机制是一个基本概念。

随着现代应用程序由许多组件组成，以及人们对微服务架构的兴趣越来越大，已部署应用程序的部署和持续发展变得越来越困难。Apache Brooklyn 的[蓝图](https://brooklyn.apache.org/learnmore/blueprint-tour.html)模型提供了一种在部署到公共云或私有基础设施之前对应用、其组件及其配置以及组件之间的关系进行建模的方法。

基于策略的管理—建立在自主计算理论的基础上—不断评估正在运行的应用程序并进行修改，以保持其健康并针对成本和响应能力等指标进行优化。

鉴于新兴应用平台(Cloud Foundry、Mesos 和 Kubernetes)的数量以及新的使用案例(尤其是物联网、大数据和分析)推动了超大规模(VLS)分布式系统的创建，这些系统自我管理或自我组织的能力至关重要。

在这种背景下，我们看到人们对自治系统和涌现行为的概念越来越感兴趣，涌现行为是一种当许多简单的实体在一个环境中运行，形成更复杂的集体行为时可能出现的现象。

创建可行的 VLS 分布式系统是一个巨大的挑战。公开做这件事是必要的。

Apache Brooklyn 最近发布了[版本 0.8.0](https://brooklyn.apache.org/download/index.html) ，并在继续发展，目标是在 2016 年上半年发布一个稳定、功能良好的 1.0 版本。它是我们云软应用管理平台(AMP)产品的基础，我们继续鼓励我们的工程师随时随地为 Apache Brooklyn 社区以及其他开源项目做出贡献。

十多年前，由 IBM 研究人员 Jeffrey Kephart 和 David Chess 首次提出的自主计算愿景现在已经成为现实。Apache Brooklyn 为用户提供了一个开放源代码的自主管理控制平面，在此基础上可以构建下一代 VLS 分布式系统。

云软和 IBM 是新堆栈的赞助商。
