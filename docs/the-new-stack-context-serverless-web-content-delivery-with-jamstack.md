# 新的堆栈环境:使用 JAMstack 的无服务器 Web 内容交付

> 原文：<https://thenewstack.io/the-new-stack-context-serverless-web-content-delivery-with-jamstack/>

前端 web 开发有了新的架构: [JAMStack](/the-sweetness-of-jamstack-javascript-apis-and-markup/) 重新思考了当前的服务器浏览器架构，将开发人员从摆弄 Apache、Linux 或后端支持的其他方面解放出来。

在本周的[新堆栈环境](/podcasts/context)播客中，我们采访了 [Guillermo Rauch](https://github.com/rauchg) ，他是 [Vercel](https://vercel.com/) 的创始人兼首席执行官，该公司提供基于 JAMstack 的服务，允许开发者[简单地将他们的代码推送到 git](/vercels-frontend-and-the-rise-of-the-hybrid-developer/) 以更新他们的网站或应用。这个平台的关键是一个由 Rauch 创建的开源用户界面框架，名为 [Next.js](https://nextjs.org/) ，基于脸书的 [React](https://reactjs.org/) ，但经过调整，不仅为开发人员，甚至为设计人员更容易地构建用户界面。

TNS 编辑和营销总监 [Libby Clark](/author/libby/) 主持这一集，旁边还有 TNS 高级编辑 [Richard MacManus](/author/richard/) 和 TNS 执行主编 [Joab Jackson](/author/joab/) 。

[第 127 集:使用 JAMstack 的无服务器网络内容交付](https://thenewstack.simplecast.com/episodes/episode-127-serverless-web-content-delivery-with-jamstack)

关于使用 Vercel 等管理型 JAMstack 的优势(相对于传统的[灯组](https://www.ibm.com/cloud/learn/lamp-stack-explained) ), Rauch 指出:

您可以部署到一个本质上无服务器的基础设施，对吗？我总是告诉人们，内容交付网络是无服务器的——因为它们从不需要管理。他们被完全授权了。这是一个全球分布式系统，没有单点故障。您不必担心 Linux 和 Apache，因为您可以部署到任何分布式全球网络，这些网络基本上可以提供标记、JavaScript、CSS 和静态文件。显然，为了支持 API、服务器渲染和更高级的功能，Vercel 网络为您提供了无服务器功能。所以我们试图完成整个 JAMstack 方程。

在我们的采访中，我们还讨论了:

*   写 Next.js 的动机。
*   JAMStack 与典型的服务器浏览器模型有何不同。
*   Vercel 的开发人员体验。
*   Vercel 运营内容交付网络的重要性。
*   云平台的开发者体验。
