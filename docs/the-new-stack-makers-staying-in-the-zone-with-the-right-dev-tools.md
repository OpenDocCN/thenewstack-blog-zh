# 新的堆栈制作者:使用正确的开发工具保持“在区域内”

> 原文：<https://thenewstack.io/the-new-stack-makers-staying-in-the-zone-with-the-right-dev-tools/>

今天的开发人员似乎使用了比以往更多的工具。构建一个基于 Node.js 的 JavaScript 应用程序可能需要十几个工具才能将代码投入生产。很容易被吸进兔子洞而无法集中注意力。在生产中调试应用程序也是一个挑战:您需要尽可能多的上下文，同时保持合理的信噪比。

功能管理平台提供商[launch darky](https://launchdarkly.com/?utm_content=inline-mention)的软件工程师丹·奥布莱恩，对如何在开发新功能或任何一段代码时不分心/保持心流有着个人兴趣。

[新的堆栈制作者:使用正确的开发工具保持“在区域内”](https://thenewstack.simplecast.com/episodes/the-new-stack-makers-staying-in-the-zone-with-the-right-dev-tools)

在最新一集的 New Stack Makers 播客中，我们向 O'Brien 询问了他在当今开发人员工作流程中看到的复杂性，以及他在编写代码时必须保持“在最佳状态”的一些技巧。我们还将讨论 launch crystally 拥有的有助于加快应用程序开发的工具。TNS 创始人兼发行人亚历克斯·威廉姆斯(Alex Williams)和 TNS 执行主编(Joab Jackson)主持了这个播客。

“我经常被闪亮的新事物分心，”奥布莱恩承认道。为了帮助专注于代码，他禁用了桌面通知，并且在 VS 代码中使用 Zen 模式，这种模式[隐藏了](https://code.visualstudio.com/updates/v1_8#:~:text=Zen%20Mode%20lets%20you%20focus,by%20the%20shortcut%20Ctrl%2BK%20Z.)活动栏、状态栏和其他编辑器附件。

另一个绝对必须的是 [GitLens](https://gitlens.amod.io/) ，Git 的 VS 代码扩展。

在 GitLens 中，“我可以看到代码最近是否被人动过，最后一个编辑是谁。你还可以获取提交信息，”他说。“对我来说，有这样的背景真是太好了。我能保持我需要的上下文越久，我就越不会在不同的工具之间来回切换，越不会分心。”

特性标志也应该在开发人员的工具箱中，能够打开和关闭特性，以观察它们对整个应用程序的影响。

LaunchDarkly 的功能标志工具的可伸缩性最初吸引了 O'Brien 为该公司工作。

“当我在 IBM 工作时，我们希望编写自己的特性标记解决方案。你很快就会知道如何简单地开始，但它会变得多快(复杂)。任何事情都有边缘情况，”奥布莱恩说。“LaunchDarkly 背后的技术一直很棒——围绕它的所有技术债务管理对我来说都是巨大的。任何人都可以将功能标志添加到系统中，但要知道该功能标志的真实状态，这是一件大事。”
