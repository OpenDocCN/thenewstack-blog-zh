# AMD 在收购后制定了软件战略

> 原文：<https://thenewstack.io/amd-builds-out-a-software-strategy-after-acquisitions/>

芯片制造商 AMD 已经获得了与竞争对手英特尔(Intel)和英伟达(Nvidia)竞争所需的所有硬件，现在正将其重点转向创建一个全面的软件组合。

“软件非常非常重要，”AMD 首席执行官苏姿丰在上周的收益电话会议上说。“我们有自己的软件堆栈。你会看到我们统一这一点，这将是我们前进道路上的一个重要部分，”她补充道

AMD 过去的软件工作主要围绕其 GPU 和驱动程序的游戏。该公司在 2 月份发布了 ROCm 5.0，这是一个并行计算框架，旨在利用其 CPU、GPU 和加速器的联合计算能力，主要用于高性能计算。但 ROCm 并没有被广泛采用，OpenCL、Nvidia 的 CUDA 和英特尔的 OneAPI 等替代品更受欢迎。

软件的推动将填补 AMD 成为数据中心产品一站式商店道路上的一块缺失。该公司以 500 亿美元收购了制造浮点门阵列的芯片制造商 Xilinx，从而增加了一个广泛的软件堆栈。AMD 上个月宣布将支付 19 亿美元收购 Pensado，该公司为云和边缘环境制造可编程芯片。

AMD 现在正致力于统一所有的软件资产，并将在下个月的公司年度分析师会议上详细说明其软件战略。

“人工智能对我们来说是一个巨大的机会，我们正在从整体上考虑如何解决这个问题，”苏说。

Xilinx 的 FPGAs 是灵活的芯片，可以针对基于 PyTorch、Caffe 和 TensorFlow 框架构建的机器学习应用进行调整。这些芯片是专门为推理而设计的，可以连接到运行在 AMD GPU(包括最新的 MI250)上的训练模型。

赛灵思有一个用于嵌入式应用和边缘设备(如汽车)的人工智能引擎，赛灵思前首席执行官、现任 AMD 自适应和嵌入式计算集团总裁的 Victor Peng 在财报电话会议上表示。

“他们正在做大量的图像识别，各种推理应用，同样的架构可以扩展并纳入 CPU 产品组合，”彭说。

AMD[正在追随 Nvidia](https://thenewstack.io/omniverse-kit-nvidia-metaverse-sdk/)的方向，Nvidia 正在调整其业务，以围绕软件产生更大的收入，该公司认为这是一个 1 万亿美元的机会。Nvidia 于 2006 年推出的 CUDA 框架最初是一个并行编程框架，用于运行高性能计算应用的 GPU。

在 Nvidia 的 GPU 在人工智能领域获得成功后，CUDA 演变成了一个 ML 开发工具。英伟达现在正试图通过销售针对医疗保健、机器人和汽车等特定领域的预定义 CUDA 软件栈，来锁定其软件和 GPU 的客户。

英特尔正在采取一种更具包容性的方法，通过其 [OneAPI 开发工具](https://thenewstack.io/intel-pushes-cross-architecture-support-in-oneapi-toolkits/)瞄准开源社区，该工具可以在一系列 CPU、GPU 和 AI 加速器上运行。但 OneAPI 在 Nvidia 最新的 Hopper 上可能没有那么有效，它具有只能通过 CUDA 解锁的功能。

AMD 正走在与英特尔相似的道路上，为开源社区开发软件堆栈。

“我们确实相信开源。我们认为协作也是生态系统的重要组成部分。所有这些都是我们正在努力为我们的客户提供更完整的解决方案，”苏说。
