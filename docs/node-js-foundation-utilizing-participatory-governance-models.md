# Node.js 基金会如何利用参与式治理来构建其社区

> 原文：<https://thenewstack.io/node-js-foundation-utilizing-participatory-governance-models/>

在最新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，TNS 创始人 Alex Williams 采访了 [Node.js 基金会](https://nodejs.org/en/foundation/)社区经理 [Mikeal Rogers](https://www.linkedin.com/in/mikealrogers) ，探讨该基金会如何通过[参与式治理](http://www.oxfordhandbooks.com/view/10.1093/oxfordhb/9780199560530.001.0001/oxfordhb-9780199560530-e-32)来帮助建立 [Node.js](/tag/node.js/) 社区。采访在 Node.js 互动 2016 发布会上进行。

[当 Node.js 社区的规模翻倍时，管理它需要什么？](https://thenewstack.simplecast.com/episodes/what-is-needed-to-manage-the-node-js-community-as-it-doubles-in-size)

Node.js 基金会对其治理模型所采取的方法受到了已经在诸如 [LevelDB](https://github.com/google/leveldb) 等项目中使用的自由贡献协议的启发。“模块生态系统中的一些社区尝试了这种自由贡献者协议的模式，基本上很早就分发了提交权，[通过]使项目更具参与性，让更多的人参与审查过程，并将其作为保持速度而不是减慢速度的一种方式，”罗杰斯说。

随着对话的结束，罗杰斯谈到了基金会发展基金会、开源和 Node.js 社区的整体目标。“开源可持续发展是我整个职业生涯都在思考和写作的事情。老实说，当你有这种增长时，你需要把人们带到漏斗上。我们已经做了大量工作，尽可能让这条道路变得简单。”
