# 下一代可持续数据中心设计

> 原文：<https://thenewstack.io/next-generation-sustainable-data-center-design/>

CloudBees 赞助了这个播客。

作为 IT 行业，我们犯的最大错误之一就是称之为云。因为它不在那里。它在数据中心。我们远离了数据中心，也远离了数据中心对环境的影响。

IT 行业的碳足迹数据差异很大，争议很大，这取决于你是否只考虑数据中心或网络和硬件的整个循环生命周期。但是它们似乎都下降了，大约世界能源使用的百分之十在其中。如果我们预计数据使用量每四年翻一番，IT 行业的碳足迹就会翻一番，而其他行业的碳足迹却在减少。

基本上，如果我们是驱动未来的人，我们是最有责任考虑这些后果的人。

在本期 New Stack Makers 播客中，我们将与 [Susanna Kass](https://www.linkedin.com/in/susanna-kass-6753768/) 谈论所有这些以及更多内容，Susanna Kass 在设计、建设和运营数据中心 30 年后，现在是[联合国可持续发展目标项目](https://www.un.org/sustainabledevelopment/sustainable-development-goals/)的数据中心顾问。

[苏珊娜·卡斯——下一代可持续数据中心设计](https://thenewstack.simplecast.com/episodes/susanna-kass-next-generation-sustainable-data-center-design)

对于数据中心来说，这一切都归结于阻止 Kass 所说的“脏云”，即大多数数据中心都在消耗能源、土地和水。数据中心需要承诺实现消耗的每千瓦时的碳中和，负责淘汰煤炭，并通过使用和生产站点消耗的每千瓦时的可再生能源过渡到碳中和基础设施。

卡斯将此描述为使用“基于自然的技术解决方案来保护我们的地球、资源和环境。”

卡斯说，在我们这个永远在线的世界里，数据中心普遍存在过度供应的浪费行为。此时，为了实现 100%的正常运行时间，他们会提供双倍(2N 完全冗余)或 N+1，通常每运行四个单元就有一个额外的备份单元。这意味着许多数据中心正在运行两倍于其所需的资源。卡斯说，资源浪费需要从一开始就考虑到，初创公司需要增加，帮助他们保证正常运行时间，而不会过度浪费。

![](img/e7a10dfc7f2108731866486512964950.png)

这都是可持续数据中心设计的一部分。这从您的数据中心建设和运营计划开始，一直到您的资源生命周期，最后通过再利用、回收和再制造消除一次性使用，从而消除垃圾填埋的影响。

“我们的目标是随着我们向数字时代的发展，促进更好的数字福利，”卡斯说。

当然，我们都知道，众所周知，当我们失去信号时，我们的世界就结束了，所以这种下一代可持续能源设计必须包括 24/7，365 的正常运行时间、可扩展性和增长，但它必须不止于此。在这段对话中，Kass 谈到了土地、电力、水和网络基础设施的有效利用，这些都需要从一开始就进行规划，“这样我们就可以收集任何废物，以最大限度地减少对环境的影响。”

碳抵消肯定是一个受欢迎的作品，但只是把问题推远了一点——贴上创可贴而不是止血。

Kass 将真正可持续的数据中心定义为检查以下每一点:

*   仅生产该场所消耗的可再生能源
*   零碳足迹
*   零排放
*   零浪费

这包括数据中心的巨大废物制造者:供暖。现在主要是这样做的，他们冷却计算机散发的热量，这样他们就能保持凉爽和平稳运行。卡斯说，通过区域供热技术，这种热量可以很容易地用于加热周围城镇的供水。

每个数据中心策略的首要考虑的另一个问题是在不使用饮用水方面更加自律。

最终目标是卡斯所说的三重底线——经济、环境和社会效益。在这次谈话中，她特别谈到了即使是个人开发者也必须要求清洁能源。
