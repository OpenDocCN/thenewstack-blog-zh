# 如何让用户界面、管道和 Jenkins X 更容易契合

> 原文：<https://thenewstack.io/how-to-make-user-interfaces-pipelines-and-jenkins-x-an-easier-fit/>

这个想法是让所有 DevOps 的利益相关者能够更好地访问持续集成(CI)/持续交付(CD)管道。您的组织也很可能依赖 Jenkins 作为生产管道的主干，并依赖 Git 来共享和协作。

然而，长久以来的问题是，众所周知，Jenkins 对于许多组织来说很难实现和使用。转移到云原生平台和 Kubernetes 平台的复杂性进一步加剧了这些困难，但这个 Jenkins 和云原生部署的新时代也为 CloudBees 的开源产品 [Jenkins X](https://jenkins.io/projects/jenkins-x/) 的诞生奠定了基础。CloudBees 创建 Jenkins X 主要是为了帮助简化和自动化到 Kubernetes 和云原生环境的 CD 管道。这个想法也是为了让 [Jenkins X](/jenkins-x-and-the-cd-foundation-a-framework-to-help-developers-in-a-chaotic-cloud-native-world/) 更容易开发和部署云原生应用到 Kubernetes，而不必学习 orchestrator 的复杂性。詹金斯 X 也是[连续交付基金会(CDF)](https://cd.foundation/) 的四个项目之一，其中还包括[詹金斯](https://jenkins.io/)、[大三角帆](https://www.spinnaker.io/)和[泰克顿](https://github.com/tektoncd)。

CloudBees 表示，它还在继续尝试和改进 Jenkins X，简化开发人员和所有 DevOps 利益相关者使用该工具的方式。为此，该公司开发了有史以来第一个图形界面。

在上周于圣地亚哥举行的 KubeCon+CloudNativeCon 期间录制的 New Stack Makers 播客中，CloudBees 的副总裁 Moritz Plassnig 和首席架构师 Peter Muir 与 New Stack 的创始人兼主编 Alex Williams 谈论了 CloudBees 如何继续努力使 Jenkins X 更容易访问和使用。

自从 18 个月前 Jenkins X 诞生以来，CloudBees 选择了创建一个图形用户界面来帮助 Jenkins X 和 CD 变得“更加主流”，Plassnig 说。Plassnig 说:“很明显，在开源项目中，最终有许多其他人与产品进行交互，他们不是开发人员。“当涉及到 CD 时，特别大的公司中有更多的利益相关者参与进来，”DevOps 继续“一天多次”部署代码

Plassnig 说，对开发者来说，这个想法有助于将“运营和开发者更紧密地联系在一起”。“作为一名开发人员，你要考虑你正在构建什么，它将如何扩展以及如何保护它，等等。但对许多开发人员来说，这不是他们真正需要考虑的事情，因为他们总是将它交给某人，然后让它为他们工作。”

然而，Jenkins X 背后的一个主要想法是“利用 Kubernetes 提供的一些伟大的东西，让开发人员更容易构思，”Plassnig 说。Plassnig 说，除了寻求改善扩展和部署管理能力以及安全管理，CloudBees 还试图让所有利益相关者更容易访问 Jenkins X 和 Kubernetes。

对于开发者来说，图形用户界面“有助于他们理解他们实际上在构建什么，以及它实际上将如何运行，”Plassnig 说。“我想我们都知道这更容易做到”，而且图形用户界面比“在屏幕上打印出来的文本”效果更好，Plassnig 说。
