# 新的堆栈环境:云环境中的 SQL 数据库

> 原文：<https://thenewstack.io/the-new-stack-context-sql-databases-in-a-cloud-native-world/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了开源数据库软件和服务公司[的首席执行官](https://www.percona.com/)[彼得·扎依采夫](https://www.percona.com/about-percona/team/peter-zaitsev)。本周，Percona 举行了自己的虚拟 24 小时虚拟会议， [Percona Live Online](https://www.percona.com/live/conferences) ，会上讨论了开源、数据库和云原生计算。因此，我们向扎依采夫询问了传统 SQL 数据库如何在云原生世界中运行，以及 Percona 新发布的 MongoDB 性能和优化包。

TNS 编辑和营销总监[利比·克拉克](https://thenewstack.io/author/libby/)主持这一集，旁边还有 TNS 高级编辑[理查德·麦克马努斯](https://thenewstack.io/author/richard/)和 TNS 执行主编[约阿布·杰克逊](https://thenewstack.io/author/joab/)。

[第 118 集:云原生世界中的 SQL 数据库](https://thenewstack.simplecast.com/episodes/episode-118-sql-databases-in-a-cloud-native-world)

有了 Kubernetes，轻松扩展云计算的舞台已经搭好。但是如何处理有状态的应用程序，尤其是那些涉及到在数据库中存储用户数据的应用程序呢？对于许多 NoSQL 数据存储，如 Redis，发挥了宝贵的作用。但不是每个人都可以使用 NoSQL 的方法。需要使用关系型 SQL 数据库的公司使用什么？传统开源数据库如何在云原生世界中运行？

例如，扎依采夫认为 Kubernetes 对于数据库即服务(DBaaS)的出现至关重要，尽管并不是每个人都从这种模式中受益。

“如果你看看由许多供应商构建的数据库即服务平台，我知道其中许多都是使用 Kubernetes 构建的。他说:“这是漂亮外表下的 Kubernetes，它允许人们对数据库进行点击式开发。”尽管他警告说这些服务虽然容易使用，但仍然是专有的。“我们没有完全开源的解决方案，这使我们能够避免基于云的数据库服务解决方案所需的供应商锁定，但提供了可与数据库即服务相媲美的简单性。”

我们还讨论了新的 Percona 自己的用于 MongoDB 的[托管数据库服务，以及用于用户运行日常检查、获取警报和修复常见数据库安全问题的](https://www.percona.com/services/managed-services/percona-managed-database-services)[新安全威胁工具](https://www.percona.com/software/database-tools/percona-monitoring-and-management)

然后，在节目的后期，我们聊了聊[新堆栈](/)上的其他热点新闻帖子和播客。Red Hat 的 Diane Mueller 和 Google 的 Paris Pittman 进行了一次[播客采访](https://thenewstack.io/the-internet-is-awesome-the-kubernetes-community-proves-it/),深入探讨了由于全球疫情，开源项目的贡献者被禁足后，开源项目所经历的更大参与和挑战。

我们还讨论了亚马逊网络服务对开发人员友好的 Kubernetes YAML 文件的替代方案，上周的 RedisConf 2020 如何证明 Redis 不仅仅是一个缓存提供者，以及正如 Honycomb.io 的一位专家所说的那样,“AIOps”的概念是完全错误的。

Red Hat 和 Redis 是新堆栈的赞助商。
