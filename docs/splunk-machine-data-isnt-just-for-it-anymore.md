# Splunk:机器数据不再仅仅是 IT 的了

> 原文：<https://thenewstack.io/splunk-machine-data-isnt-just-for-it-anymore/>

就像新一代云 BI 和数据可视化供应商试图让组织中的更多人能够访问数据一样，Splunk 也在努力让更多类型的人对其机器数据感兴趣和可用。该公司今天宣布了一些新功能，包括与移动应用程序相关的数据收集，以及旨在使其产品更易于使用的更新。

Splunk 解决方案营销总监比尔·埃米特(Bill Emmett)说，Splunk 最初主要是为系统管理员提供的工具，他们在解决问题时使用它来访问日志文件。“但人们意识到，机器数据是多样的。它不仅允许 It 组织的人员，而且越来越多地允许业务线内部的人员使用 Splunk，不仅可以了解 IT 运营和应用程序问题，还可以了解业务中正在发生的事情，”他说。

该公司正在更新 Splunk Enterprise，以便企业中的更多人可以更容易地使用它。该公司表示，这一想法是为了让“临时和不太懂技术的用户”从机器数据中获得商业见解。例如，新的即时透视功能旨在让用户直接从搜索透视，因此他们可以轻松创建仪表板，而无需了解 Splunk 搜索处理语言。该更新还包括“预建面板”，旨在使用户更容易创建仪表板。

Splunk 还宣布全面推出其所谓的 Splunk MINT Express，这是一款收集移动应用信息的产品。该产品基于 Splunk 去年与 BugSense 一起收购的技术。

BugSense 技术为开发人员提供了关于其移动应用性能的数据，如崩溃信息，崩溃是局部的还是广泛的，应用上的每日活跃用户数，以及他们使用的操作系统版本。借助 MINT Express，Splunk 增加了更多功能，包括更深入地了解网络性能及其对应用响应时间的影响。MINT Express 还向开发人员展示了 Web 服务器或其他组件的瓶颈，这些瓶颈可能会降低响应速度。

开发者可以购买 MINT Express 作为独立服务。要使用它，开发人员必须在他们的移动应用程序中包含一行代码，以便 MINT Express 可以收集相关数据。

该公司还推出了 Splunk MINT Enterprise，它将移动数据与 Splunk 的企业数据相结合，处于测试阶段。企业产品允许用户根据移动运营数据构建警报、仪表盘和报告，为用户提供比 Express 产品更详细的分析。

除了 It 部门之外，这种企业产品在整个企业中可能更有意思。例如，营销人员可以使用它来查看移动应用程序用户的地理位置。一名业务人员可能希望根据人们是连接到 Wi-Fi 还是蜂窝网络来查看他们使用应用程序的方式的差异。这些用户还可以查看客户如何使用移动应用程序的数据，并与可能在网站上进行类似活动的客户进行比较。

Splunk 并不是唯一一个试图找出让更多企业员工利用其收集的数据的方法的公司。最近，ExtraHop】开始让用户向 MongoDB 传输数据，在那里他们可以更容易地将机器数据与金融数据结合起来，以挖掘新的见解。

这是一个更大趋势的一部分，即企业试图做更多的事情来理解他们从各种在线服务和应用程序中收集的大量数据。对于已经在 It 部门收集机器数据的企业来说，尝试使相同的数据可供其他企业用户访问和使用是有意义的。
