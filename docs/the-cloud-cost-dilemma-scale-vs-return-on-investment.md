# 云成本困境:规模与投资回报

> 原文：<https://thenewstack.io/the-cloud-cost-dilemma-scale-vs-return-on-investment/>

如果你在查看云发票时享受每月一次的鼓励，认为内部成本会高得多，那么风险投资公司安德森·霍洛维茨(Andreessen Horowitz)最近的报告可能是一个真正的扫兴。

A16z 深入研究了云的数量与内部总拥有成本，并提出了一个发人深省的结论:“如果你大规模运营，云的成本至少会使你的基础设施账单翻一番。”

## **了解云悖论**

 [马克西姆·梅拉梅多夫

马克西姆是 Zesty 的联合创始人兼首席执行官。马克西姆在科技行业拥有超过 13 年的经验，包括之前在国土安全领域的职业生涯，他擅长解决复杂的问题和打破以前建立的规范。](https://www.linkedin.com/in/maximmelamedov/) 

让我们对报告试图告诉我们的东西有一些看法，因为作者很清楚他们并不主张遣返。事实上，他们认识到立即可用的大规模基础设施的“难以置信的价值主张”。

这个悖论的核心是关于你的企业的成熟度。作为一家初创公司，云是完美的软着陆。首先，零资本支出和所有运营支出意味着您不需要那么大的初始投资，因为您将重点放在“即服务”订阅模式的运营支出上，而不是通过构建自己的基础架构的资本支出上。接下来，一个经验丰富的云提供商代表您处理基础设施，限制了您对内部开发人员和工程师的需求，您可以访问大量基于云的应用程序，只需一跳、一跳和一个 API，即可构建您的核心产品或服务。

随着你快速成长的初创公司变得更大、更稳定，你的初始投资需要转化为持续的利润，事情变得有点复杂。

云规模越大，管理云所需的工程师就越多。这些工程师在一个越来越以云为中心、技能缺口严重的市场中非常受重视，并且拥有要求高薪的利基技能。随着云的使用在整个公司范围内扩展，从存储和计算到备份、SaaS 产品或安全，许多公司觉得他们是在向云提供商签一张空白支票。

这是因为，随着您的业务规模不断扩大，以及您接纳新客户和构建新功能，您需要越来越多的计算资源来管理规模。与此同时，增长速度意味着很难在不影响业务连续性的情况下找到一条替代路径。另外，最终你的供应商关注的是他们自己的利润和增长；例如，AWS 每年有大约 30% 的[舒适的云利润率。](https://www.cnbc.com/2020/10/29/amazon-cloud-revenue-jumps-29percent-in-line-with-expectations.html#:~:text=AWS%20generated%20%243.54%20billion%20in,of%20its%20revenue%20from%20AWS.)

## **纵观全局**

那么，云永远是最划算的选择吗？虽然云肯定可以节省您的初始启动成本，但随着您的业务规模扩大，您可能会发现您的成本也开始上升。

然而，再问自己一个问题。不管它是否是最便宜的选择，云对您的企业来说是正确的选择吗？

如果您迁移到了云，或者首先开始了您的旅程，降低成本可能是您的清单上的众多项目之一。事实证明，你可能无法拥有一切。问问你自己:你还从你的云环境中获得了什么好处？

对于一些人来说，这将是使用 DevOps 管道快速交付新功能和更新，比您在内部实现的速度快 10 倍。对于其他人来说，这可能是满足高峰流量、季节性需求或突然的病毒式营销活动的按需扩展带来的安心。也许是灵活性、易于集成、对安全责任分担模式的信心，或者是云是唯一一个无疫情区的任何其他原因，[预计今年将增长 23%。](https://www.gartner.com/en/newsroom/press-releases/2021-04-21-gartner-forecasts-worldwide-public-cloud-end-user-spending-to-grow-23-percent-in-2021)

## 但是……钱很重要！有没有一种方法可以拥有一切？

如果这些好处对你很重要，那么偶尔你可能会发现成本优化需要让出司机的座位，为所有这些乘客让路。如果这是你的情况，如果其他因素也同样重要，那么这就是一个合理的策略。但是，这并不能让您免费减少云支出。您的投资回报非常重要，您不能被视为在没有数据支持的情况下随意向您的云环境投资。

如果您不考虑遣返，但希望保持持续云战略的最高成本，以下是一些帮助您开始的提示:

**让工程师远离重复而平凡的任务:**您是否在招聘和雇佣工程师上花费了太多时间，结果却在平凡的云管理任务上浪费了他们的技能？将它们释放出来，帮助人类茁壮成长的地方(提示:是战略、决策和客户支持)，并寻找将人工智能和自动化添加到最适合机器人的任务中的方法。(C3PO，是你发光的时候了。)

**制定 FinOps 治理战略:**在这里，您可以规划现有的使用情况和云支出，并使用商业智能工具来发现数字背后隐藏的趋势和模式。然后，您可以为云优化创建一个数据驱动的预算，包括系统设计变更、重新架构、第三方效率解决方案，甚至是有意义的遣返。

**使用你能得到的成本优化工具:**安德森·霍洛维茨并不完全悲观。他们确实评论说，明智地分配承诺使用折扣，如预留实例，可以大大降低您的云支出，正如依赖第三方优化工具一样，据 a16z 称，这些工具已被证明可以将云支出降低 10%至 40%。

**让云支出一个可衡量的 KPI:** 最重要的是，不要把“无论云的成本是多少”视为必要的支出。相反，强调云成本是整个组织都有责任的一个指标。为减少支出、创建目标和里程碑以保持专注、要求所有部门关注跟踪和管理云成本的智能思维建立激励机制。
