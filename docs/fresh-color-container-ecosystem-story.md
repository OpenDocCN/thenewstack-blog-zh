# 欢迎来到容器的新 IT 时代

> 原文：<https://thenewstack.io/fresh-color-container-ecosystem-story/>

像我一样，你可能喜欢阅读博客和书籍，但有时它们会有点单调。感谢上帝，我们有现场活动和视频可以依靠，以不同的方式消费信息。

举个例子，新任 Stack 主编 Alex Williams 上周在拉斯维加斯举行的 [SAP TechEd](https://events.sap.com/teched-2015-usa/en/home) 会议上的讲话，基于我们过去两年对 Docker 和[容器生态系统](/category/containers/)的不间断报道。作为一个已经通读了 TNS 的[新电子书](https://thenewstack.io/ebookseries/)的人，我可以证明[视频](https://events.sap.com/teched/en/session/22804)很好地总结了我们在那段时间学到的东西。

除了视频，如果您有兴趣借用幻灯片进行演示，请随时[通过 SlideShare](https://www.slideshare.net/TheNewStack/sap-teched2015-lasvegastnspresentationupdated) 阅读内容。

Williams 认为，容器吸引所有人注意力的主要原因是容器和 Docker 预示着整个 IT 领域将发生更大的变化(因此该网站得名)。

Docker 起源于 Linux 容器系统的扩展，植根于开源文化。它被反复引用作为开源社区如何激增并变得更加商业化的例子。[开放容器倡议](https://www.opencontainers.org/)和[云本地计算基金会](https://cncf.io/) (CNCF)都是企业如何在基础层面合作指定微服务应该如何运行的例子。

易于使用的编程语言的涌入加剧了这一趋势。苹果的 Swift 就是一个例子，说明了简化编程是如何带来一大批新程序员的。其他人则被谷歌的 Go 所吸引，[成为了 Docker](https://thenewstack.io/go-programming-language-helps-docker-container-ecosystem/) 的基础。

推动这一容器革命的另一个趋势是对自动化基础设施的需求。互联网规模的公司(如 Yelp，网飞)开发了自己的解决方案来自动化其基础设施，以处理从测试到日常使用的容器管理需求。

然而，手工制作的方法无法扩展，因此新一代供应商已经崛起，创造产品和服务来解决将容器集成到更大环境中的痛点。谷歌正在领导 CNCF 的努力，帮助公司开发自己的自动化基础设施，以便他们能够像互联网规模的公司一样行事。

Williams 演示的后半部分涉及实际的供应商和产品。它基于我们今年早些时候对 48 家公司的调查。TNS 发现，Docker 和 container 生态系统公司可能会提供或计划提供工具来帮助应用程序的整个生命周期，从应用程序开发到映像创建、部署、编排和管理。由于强调端到端解决方案，许多公司将 IBM、AWS 和 Heroku 视为潜在的竞争对手也就不足为奇了。

考虑到 AWS 经常被认为是最好的基础设施之一，SAP 明智地在同一家酒店——宽敞的威尼斯酒店——举办会议。这个基础设施对于构建 AWS 的注册认证服务至关重要。图像注册表至关重要，也是 Docker 的竞争优势之一。标准化基础映像的努力正在成为几家供应商产品的一部分。

Williams 的报告最后讨论了不同公司之间的伙伴关系。Docker 是大多数公司计划的核心。微软和谷歌也在积极促进他们的合作关系，但同时他们在提供 IaaS 方面与 AWS 竞争。CoreOS 正在提供构造，Red Hat 正在提供 OpenShift，这两家公司都可以作为他们自己的合作伙伴提供商业产品的基础。

*Docker 和 Red Hat 是新堆栈的赞助商。*
