# 衡量云本机安全性的状态

> 原文：<https://thenewstack.io/measuring-the-state-of-cloud-native-security/>

来自帕洛阿尔托网络公司的 Prisma 赞助了这篇文章。

 [宙斯·凯拉瓦拉

宙斯是 ZK 研究公司的创始人和首席分析师。](https://www.linkedin.com/in/zkerravala/) 

云安全可能是当今安全专业人员面临的最困难的事情。大多数组织没有为云安全做好充分的准备，因为他们试图应用一种为早已过去的时代设计的安全模型。传统的安全性基于这样一种模型，在该模型中，组织周围有一个硬壳，壳内的一切都是好的，需要受到保护；而外面的一切都应该被认为是不好的。当大多数计算平台都在外壳内时，这是可行的。

## **云的现状:挑战**

然后，云出现了，企业开始将资源移出这个外壳。然后出现了多云，应用程序和数据分散在众多的云提供商之间。这有许多挑战，因为大多数传统的企业安全工具没有扩展到云。此外，虽然云提供商提供他们自己的安全解决方案，但这些解决方案不会跨云扩展。例如，我不能使用 AWS 防火墙来保护 Azure 中的工作负载。对于多云，安全专业人员需要考虑一种可以跨越企业数据中心和所有主要云提供商的解决方案。

但是等等，还有更大的困难。现在，多云已成为常态，安全专家需要开始展望未来。这就是我称之为分布式云的东西，它将把安全性的复杂性提高到新的高度。没错，我们还没有真正搞清楚多云，我认为多云时代正在结束——分布式云已经开始出现。

分布式环境可以被认为是一组高度分布式的云服务，它们跨越私有云和公共云，也跨越无数的边缘位置。这包括物联网边缘、校园边缘、家庭边缘、5G 边缘等等。在这些云、边缘和数据中心中，客户可以在裸机、虚拟机或容器中运行工作负载。容器、虚拟机、云和边缘的这种组合为组织增加了一层难度，推动了对不同云安全模式的需求。

## **最新统计数据和趋势**

为了更好地了解世界各地的公司使用什么技术来管理云安全，Palo Alto Networks 进行了第一次“[*2020 年云原生安全状态”*报告](https://www.paloaltonetworks.com/state-of-cloud-native-security)。

该报告基于对云架构、信息安全、DevOps 和应用程序开发领域的 3，000 名专业人员的调查。一半的受访者来自美国，其余的来自德国、英国、新加坡和澳大利亚。

大约四分之三的受访者在三个行业工作:金融服务和保险、消费品和工业品，以及能源和资源。其他人要么在技术、媒体和电信领域工作，要么在生命科学和医疗保健领域工作。

该报告发现，大多数组织(94%)使用不止一个云平台，60%的调查受访者表示他们使用两到五个。那些正在利用云计算的企业已经接近转型的中间点。

没有单一的云计算服务占主导地位，因为工作负载几乎均匀地分布在不同的服务中。虚拟机(VM)平均占所有工作负载的 30%，然后是容器的 24%，容器即服务(CaaS)的 21%，平台即服务(PaaS)的 22%。大多数受访者(86%)预计他们对所有四种服务的使用在不久的将来会增加或保持不变。与此同时，推动云应用(IaaS、CaaS 和 PaaS)的架构的多样性也有望增加。

调查结果表明，在未来 24 个月内，云将成为组织的主导计算模式。虽然目前有 46%的组织在云中运行其工作负载，但该报告预计这一比例将在两年内增长至 64%。80%的受访者认为，这是因为云基础设施在不断发展。

## **组织最关心的问题**

毫不奇怪，当组织将工作负载迁移到云时，他们最大的担忧就是安全性。39%的受访者认为维护全面的安全性是最大的挑战。当放大组织在云服务方面面临的威胁时，他们列出了:数据安全和恶意软件、应用程序漏洞、薄弱和不可靠的身份认证、内部威胁、凭据泄露、不安全的 API、过度授权访问和错误配置。

出于多种原因，组织很难提供全面的云安全。有些人缺乏对安全漏洞的了解。其他公司没有对员工进行足够的安全工具和安全实践培训。

云安全团队结构因组织而异。该报告发现，77%的公司的云安全团队超过 20 人。进一步细分显示，47%的企业拥有集中的云安全团队和在交付团队中工作的安全专家，31%的企业拥有集中的云安全结构，22%的企业采用跨职能的云安全结构。

许多组织不明白他们要对云中的应用程序和配置负责，因此他们必须自己保护这些应用程序和配置，而不是假设提供商会为他们保护它们。绝大多数情况下，73%的受访者不清楚他们的云安全责任和云服务提供商(CSP)的责任。

部署更多安全工具并不能转化为对云基础设施的更好保护。相反，获得更多的工具会导致效率低下，并使员工培训更加困难。在报告调查的人中，71%使用第三方供应商工具，65%使用 CSP 提供的安全工具，62%使用开源工具。

在投资 5000 万至 1 亿美元的云计算公司中，30%的公司使用 11 家或更多的云安全供应商。对于在云领域投资超过 1 亿美元的公司，供应商的数量会显著下降。在这个高消费群体中，53%的人使用少于五种安全工具。拥有较大云预算的公司能够通过消除重叠的工具和供应商来整合和合理化其投资。

拥有最大年度云预算(1 亿美元或更高)的组织也在云安全上投入更多。在高消费人群中，34%的人将 16%或更多的云预算用于安全性。

Palo Alto Networks 开发了一个“云安全准备度”指标来确定受调查组织的准备程度，从低到中到高。只有 18%的组织为保护其云做好了充分准备，而 29%的组织处于准备最差的类别。这个数字的可怕之处在于，大多数公司都倾向于高估自己的能力。在声称准备好的 18%的人中，我猜可能有一半实际上准备好了，而另一半认为他们准备好了，但实际上没有。

拥有最高级别云安全准备的组织正在做出更明智的决策。在准备最充分的公司中，45%将安全性嵌入到开发运维中，41%将安全性集成到软件开发生命周期的至少四个阶段中。相比之下，21%准备最差的公司在开发运维中嵌入了安全性，12%在开发生命周期中涉及安全性。

该报告还发现了安全准备与组织使用的云安全工具数量之间的相关性。准备充分的组织认为，拥有许多工具会使确定风险优先级和预防威胁变得更加困难，而那些准备不足的组织不认为拥有多种工具有问题。

一半(50%)准备充分的组织表示，他们正在积极减少使用的安全工具的数量。这些组织的共识是，单一、全面的安全解决方案将改善他们的整体云状况。这在过去可能是一个白日梦，现在仍然很难实现，但一些领先的安全供应商已经采用了一种平台方法，允许第三方轻松地插入他们的框架。这使安全提供商能够提供单一供应商解决方案，但确保任何缺口都可以由与底层平台协同工作的第三方来填补。
