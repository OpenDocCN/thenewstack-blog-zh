# OpenFaaS 开源社区资助模式的创建者

> 原文：<https://thenewstack.io/openfaas-creator-on-open-sources-community-funding-model/>

[OpenFaaS 创始人谈开源社区资助模式的成功](https://thenewstack.simplecast.com/episodes/openfaas-creator-on-the-success-of-open-sources-community-funding-model)

在最新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，我们与 [OpenFaaS](https://www.openfaas.com/) 的创建者 [Alex Ellis](https://www.linkedin.com/in/alexellisuk) 坐在一起，讨论他的开源无服务器平台的最新更新，这个新概念“[无服务器 2.0](/context-the-stateful-services-of-serverless-2-0/) 意味着什么，以及支持开源项目的困难。

Ellis 几年前创建了 OpenFaaS，希望将 Amazon Web Services 的 function-as-a-service 提供的功能扩展到任何容器化的计算环境，这一功能后来被其他无服务器产品效仿。这个想法是，你不仅可以把你的应用程序打包，还可以把所有的支持库和依赖项打包到一个容器中，这样它们就可以无服务器运行了。您不局限于一个云提供商或一个语言运行时。“我想找到一种方式，将我对 Docker 的热爱和对编码的热爱结合起来，”Ellis 说。

有了无服务器 2.0，“你可以运行任何代码，无论是二进制还是 HTTP 服务器，以任何你喜欢的方式——你的笔记本电脑，内部，在 OpenShift 上，在云中，”埃利斯说。Kubernetes 提供了一个共同的基础。

虽然 Ellis 自己创建了 OpenFaaS，但他已经围绕该软件建立了一个健康的贡献者社区，以及一个非常健壮的最终用户社区，包括许多大公司。他承认“让飞轮转动起来”需要大量的工作，尽管通过志愿者捐赠机制只找到了有限的支持，如 GitHub 赞助商或 Linux 基金会的社区桥。

“它们都基于同一个模型:如果我们创造一种让你的支出透明化的方式，人们就会给你钱。我认为事实并非如此，”埃利斯说，称这些努力在大多数情况下是“失败的模式”。

(自从接受我们的采访后，埃利斯启动了一个[内部人士计划](https://github.com/users/alexellis/sponsorship)，每当赞助商和支持者在 GitHub 或 Patreon 上赞助他时，都会给他们带来附加值。他说，这已经比捐赠模式成功得多。这从 5 美元开始，上升到主页赞助水平。内部人员将定期收到“内部人员更新”，涵盖 Alex 的所有 OSS 工作、博客、视频、行业更新，以及对 OpenFaaS 新闻和功能的早期访问。)
