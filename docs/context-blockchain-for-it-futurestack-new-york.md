# 背景:面向 IT 的区块链，FutureStack 纽约

> 原文：<https://thenewstack.io/context-blockchain-for-it-futurestack-new-york/>

在本周的[新堆栈环境](https://thenewstack.io/podcasts/context)中，我们采访了[WSO2](https://wso2.com/)CTO 办公室的软件架构师兼研究副总裁[斯里纳斯·佩雷拉](https://www.linkedin.com/in/srinathperera/?originalSubdomain=lk)，了解他对区块链实际用途的研究。

[区块链为之，未来斯塔克纽约](https://thenewstack.simplecast.com/episodes/blockchain-for-it-futurestack-new-york)

WSO2 是新堆栈的赞助商，它为 API 管理、集成、身份和访问管理、分析和物联网提供了一个集成平台。而且它的研究也是云原生技术的前沿。也是[芭蕾云原生编程语言](/ballerina-an-api-first-programming-language/)背后的公司。

佩雷拉领导这些研究工作，最近为我们写了关于[无服务器技术](/how-to-get-over-the-serverless-integration-hump/)和[区块链](/what-you-need-to-know-about-blockchain-before-making-the-leap/)的文章。对于区块链，他的团队不确定哪些用例是可行的，并且也能交付清晰的价值。因此，他们使用该公司的[新兴技术分析画布](https://github.com/wso2/ETAC) (ETAC)对区块链技术进行了系统分析。佩雷拉写道:

*区块链提供了建立信任的机制，通过自动化建立信任所需的验证来降低与集中式系统相关的风险并实现敏捷性。然而，与当前的分散式或半分散式区块链解决方案相比，集中式解决方案速度更快、吞吐量更大、实施成本更低。也就是说，随着政府和市场需求解决区块链面临的技术挑战，相关成本和实施障碍将会减少。*

然后在节目的稍后部分，我们将报道本周网站上的热门播客和故事[，包括为什么](/) [Kubernetes 应该无聊的争论](https://thenewstack.io/cloud-foundry-summit-europe-why-kubernetes-should-be-boring/)，一种叫做 [Dark](/dark-deep-dive-from-editor-to-infrastructure-in-less-than-50-milliseconds/) 的新的“[无部署](/dark-a-new-programming-language-for-deployless-deployments/)编程语言，[CentOS 用户的一些受欢迎的更新消息](/red-hats-new-linux-distro-brings-centos-closer-to-rhel/)，以及[在纽约举行的 New Relic 的 FutureStack 会议的总结](/new-relic-expands-observability-platform-with-new-logging-tracing-and-services-metrics/)。

TNS 的编辑和营销总监 Libby Clark 在 TNS 创始人兼出版商 Alex Williams 和 New Stack 的执行主编 Joab Jackson 的帮助下主持了这个播客。

New Relic、Red Hat 和 WSO2 是新堆栈的赞助商。
