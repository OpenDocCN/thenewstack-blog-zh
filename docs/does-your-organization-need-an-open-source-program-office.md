# 你的组织需要一个开源项目办公室吗？

> 原文：<https://thenewstack.io/does-your-organization-need-an-open-source-program-office/>

每个现代企业都使用一些开源软件，或者至少使用具有开源组件的软件。在一个企业环境中，一个组织可能使用的不同开源项目的数量很容易达到数十万个，也很容易有同样多的工程师使用这些开源项目。

虽然现实情况是企业使用开源软件，但开源社区有一种完全不同的文化——一种专注于协作的文化，这种文化与大多数标准商业环境格格不入。

“作为一家企业，这是一场文化变革，”Jeff McAffer 解释道，他多年来负责微软的开源项目办公室，现在是 GitHub 的产品总监，专注于在企业中推广开源。“许多公司不习惯协作。他们不习惯与公司以外的团队合作。”

到底什么是开源项目办公室(OSPOs)？他们做什么，谁需要他们，为什么？为了了解更多信息，我们采访了一些领导开源项目办公室的人。

## 谁，什么，为什么

让我们从谁需要开源办公室开始:使用开源软件的大公司——换句话说，几乎所有的大公司。需要指出的是，OSPOs 不仅仅是谷歌或微软等“科技”公司的领域。[托多集团](https://todogroup.org/)，一群运行开源程序的公司，建立了一个分享最佳实践的社区，包括许多高科技公司，但也包括丰田、Capital One、Target 和其他不属于科技垂直领域的公司。

“我们正处于开源应用的第四次浪潮中，传统的财富 1000 强公司开始改变他们消费开源的方式，”IBM[开源办公室负责人](https://www.ibm.com/)[杰夫·博瑞克](https://www.linkedin.com/in/jeffrey-borek-a23a0/)解释道。“十年前，IBM 需要一个开源项目办公室，但全球可能只有几十家公司有类似的东西。”

现在，公司不仅希望被动地使用软件，还希望积极地参与开源社区——也许不是典型企业可能依赖的所有成千上万个项目，但肯定是那些对业务特别重要的项目。

“开源办公室可以推动创新，但也提供了护栏，以便当你改变消费开源的方式时，你仍然可以管理收益与风险，”Borek 说。“它还为您提供了一种有效的方式，向您公司的其他人传达开源的重要性以及适当使用它的重要性，但不会制造太多障碍，让开发人员感到沮丧。”

开源办公室的核心角色之一是审计许可合规性，以确保公司遵循开源项目的许可要求。开源办公室的这部分工作通常涉及到与法律团队的合作。但这也是一个非常具有战略意义的角色，涉及到组织应该如何为开源社区做出贡献，什么应该或不应该发布回社区，以及开源如何为业务的不同部分带来价值。

[Spotify](https://www.spotify.com) 开源负责人 [Remy DeCausemaker](https://www.linkedin.com/in/decause/) 解释道:“工程、商业和法律利益相关者都有自己的目标和角色，经常在速度、质量和风险之间做出权衡。“OSPO 致力于平衡和连接这些单独的目标，形成一个减少摩擦的整体战略。”

## 领导一个开源办公室

大多数读者可能认为开源办公室是工程组织的一部分，由工程师领导。有时候，甚至经常是这样……但不总是这样。

首先，本文采访的每个人都同意开源办公室在组织结构中的正确位置是在工程中，但是给出了在法律或营销团队中有开源办公室的公司的例子。

但是在 OSPOs 工作的人是工程师吗？不一定。“我是一名工程师，我已经做了 30 年的开源开发者，”[VMware](https://www.linkedin.com/in/dirkhohndel/)的首席开源官 Dirk Hohndel 分享道。“但是，如果你观察这个行业的一小部分，观察在大公司管理开源项目办公室的人的类型，你会看到有许多不同背景的人，营销副总裁或项目经理，来自完全不同领域的人。”

Hohndel 说，成功运营一个开源办公室的关键与让一个人擅长写代码的技能非常不同。“你必须能够与非工程师的人交谈，你需要能够与背景非常不同的人辩论，”他说。“你需要能够与营销专业人士会面，进行建设性的对话。你必须与企业主交谈，了解他们的目标，还要与律师交谈。”

开源办公室本质上是跨职能的。拥有工程背景可能会有所帮助，但对于帮助组织从战略上考虑如何使用开源来说，这既不是必要的，也不是充分的。

## 不要低估文化的变化

管理一个开源项目办公室不仅仅是“推广开源”它是关于教导组织中的每个人战略性地思考开源如何适应更大的商业目标。“这从根本上说是一个工程挑战，”麦卡弗说。“但是工程师们，我也是其中之一，倾向于认为，哦，我们可以为此编写一些工具，对吗？”他说，这是看待开源办公室的错误方式——这是一个非常战略性的功能，涉及到根本的文化变革，而不仅仅是另一个工具。

OSPOs 也不尽相同。尽管 OSPO 的工作方式和开源社区的工作方式都有共性，但是每个 OSPO 都必须考虑行业垂直、技术基础设施和特定于其组织的商业目标。

“开源不仅仅是一个‘社区’，而是一个‘社区的社区’，”Spotify 的 DeCausemaker 说。“认识到这些开源社区相互依赖的复杂性，对于跨垂直行业和拓扑结构提出建议至关重要。”

## 双行道

开源办公室还帮助公司决定如何参与开源社区并为项目做出贡献。“当人们说‘回报社会’时，我通常会很生气，”麦卡弗说。“感觉像是慈善捐款什么的。”

当一个企业为开源项目做出贡献时，这并不是正在发生的事情。如果企业要使用开源项目，他们就要依赖这些代码和项目的持续健康。他们也有兴趣确保 bug 被修复，项目按照他们想要的方式发展。公司为开源项目做贡献是因为这样做符合他们的商业利益，而不是因为他们想做一些温暖而模糊的事情。

Hohndel 说:“越来越多的开源项目办公室也有帮助开源社区成长、成功、吸引更多人的功能。“你希望他们成长、成功、包容、适应项目面临的挑战。”

VMware 是新体系的赞助商。
