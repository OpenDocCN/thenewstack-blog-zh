你不会用骡子来拉重型拖拉机。同样，您不应该期望使用为遗留系统构建的技术来支持当今动态的 Kubernetes 环境。

Kubernetes 已经迅速成为各种云原生应用和基础设施的首选企业平台。它受欢迎有很多原因。云原生技术，尤其是 Kubernetes，是推动当今数字经济的高级应用和服务的开发和交付的核心。

此外，由于关系型和非关系型数据库(如 NoSQL)经常使用 Kubernetes 来支持这些应用程序、环境和服务，许多开发人员也依赖于它的存储能力。Kubernetes 消除了确保应用程序和服务的高可用性和可伸缩性的许多痛苦，但不幸的是，这些好处并没有扩展到数据。在 Kubernetes 环境中，数据管理必须是一个关键的优先事项，但传统的数据管理技术(涵盖备份/恢复、灾难恢复和应用程序迁移等操作)在云原生系统固有的敏捷性、可扩展性和性能方面远远落后。

由于应用程序的快速增长和大规模生产部署的增加，许多企业不仅要关注应用程序开发生命周期，还要关注“第二天”操作以及生产中应用程序和服务的挑战。这些包括数据管理、安全性和可观察性。虽然 Kubernetes 的数据复制和可移植性可以增强系统的可靠性，但它不能保护开发者和运营商免受基础设施故障、数据损坏或数据丢失的影响。事实上，如果有一个意外导致数据库被删除的编码错误，该错误将会随着周围的一切如实复制，从而导致进一步的数据丢失。如果没有独立且适当的数据管理系统，企业的高价值数据以及依赖于这些数据的应用程序和服务可能会一直暴露在外，从而产生广泛的潜在业务风险。

[Cloud Native Computing Foundation](https://www.cncf.io/)最近的一项研究显示，超过 40%的调查受访者正在使用 Kubernetes 进行存储，其余 55%的受访者正计划这样做。这些危险不仅仅是理论上的——如今在 Kubernetes 上运行的有风险的工作负载和数据集比大多数 IT 人员意识到的要多得多。随着这一趋势的继续，引入风险的机会成倍增加。

当然，将备份和恢复责任委托给为遗留基础设施构建的遗留工具的诱惑是可以理解的；这在过去是有效的，可以立即使用，并且不需要额外的投资。然而，经验表明，当数据丢失时，与容易出错的传统数据重建方法相关的冗长恢复时间会给本已紧张的局面增加巨大的成本和风险。

为了在云原生的、Kubernetes 协调的环境中保护业务关键型数据，需要两件事情:Kubernetes 原生数据管理策略(专门用于可扩展的、敏捷的和动态的环境中)和在独立位置维护的完整数据集的全面但冗余的独立副本。Kubernetes 使用自己的放置策略在所有服务器上分发应用程序组件，以提高容错能力和性能。此外，不同的应用程序通常位于同一服务器上。相比之下，传统的数据管理系统将会失败，因为它们永远无法独立地捕获某个特定应用程序的状态，而不把不相关的应用程序拉进来。

> Kubernetes 的核心是以应用为中心，而不是以基础设施为中心。

相应地，Kubernetes 编排也在不断地重新平衡，因此它们的 IP 地址也在不断变化。备份解决方案需要理解并遵循这种模式。但是，大多数传统解决方案都假定应用程序和数据的定义和地址是固定和稳定的，如果不引入不相关的应用程序，就永远无法独立捕获特定应用程序的状态。另一方面，Kubernetes-native 备份将能够发现和捕获动态应用程序及其所有相关内容，无论它们在哪里。

Kubernetes 的核心是以应用为中心，而不是以基础设施为中心。正因为如此，它有助于支持数字创新速度的高速应用程序开发。因此，与 Kubernetes 并行采用的 DevOps 理念将对基础设施和部署的控制权交给了开发人员。然而，这种巨大的能力也伴随着风险，即使是简单的配置错误，如果不加检查，也可能删除关键数据并导致业务中断。

这就是为什么确保第 2 天操作的连续性如此重要。为了防范这些风险，适当的备份是必要的。能够自动发现新的和变化的应用程序的系统是一个重要的附加值，而且这样做时不需要开发人员改变他们的工作流程或工具。在 Kubernetes 环境中，这可以通过支持安全协议的本地 API 来实现，比如身份验证和授权，以及 CI/CD 和工作流集成。

所有这些都反映出 Kubernetes 是一个完全不同的计算平台。它比以前的系统更复杂，对于许多在传统数据中心运营中成长起来的 It 人员来说是不熟悉的，并且它的管理职责以不熟悉的方式分配。除了云环境中数据丢失的传统原因，Kubernetes 增加了意外数据丢失的风险。

然而，与此同时，有一个快速增长的技术生态系统，使 Kubernetes 环境能够像他们那样运行，提供敏捷性、性能和可扩展性，以帮助用户从他们的部署中获得全部价值。作为一名用户，你必须关注专门的、Kubernetes-native 的解决方案，这些解决方案随着云原生技术的采用而快速发展，否则就有暴露宝贵投资的风险。
