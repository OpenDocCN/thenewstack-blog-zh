# Qumulo 为协作在线视频编辑扩展分布式存储

> 原文：<https://thenewstack.io/qumulo-scales-distributed-storage-for-collaborative-online-video-editing/>

当 2015 年 [Qumulo](https://qumulo.com/) first [推出](https://thenewstack.io/qumulo-its-not-about-the-storage-its-about-the-data/)时，据说该公司正致力于解决如何扩展数据的问题，而不是通过其 Qumulo Core 来扩展存储，qu mulo Core 是一个可扩展的数据分析平台，可以为企业提供数据和存储资源的视图。

两年后，Qumulo 将其分布式文件存储系统与亚马逊网络服务配对，今天又将谷歌云平台加入其中。利用所有可用和可扩展的计算，该公司还推出了两款新产品:Qumulo CloudStudio 和 QumuloDR Cluster。

Qumulo 的分布式文件存储系统与谷歌云平台(GCP)和亚马逊网络服务的弹性计算资源的结合，在 [CloudStudio](https://qumulo.com/solution/cloudstudio/) 中得到了展示，根据公司声明，“安全地将传统的本地工作空间，包括桌面、应用程序和数据，移动到公共云”。

Qumulo 的全球产品总监 Molly Presley 提供了一个视觉效果工作室的轶事，证明了 CloudStudio 的潜力。她说，以前，工作室需要花几个星期的时间来渲染新的视觉效果，他们开始落后了。她说，有了 Qumulo，他们可以“冲到亚马逊，在一千个节点上运行”，而不是错过最后期限和落后。

普雷斯利还提到，与 GCP 的整合提供了额外的战略[计算地点](https://cloud.google.com/vpc/docs/edge-locations)，因为该公司已经在多伦多、洛杉矶和其他媒体和娱乐组织(如创意机构、后期制作工作室、广播公司和视觉效果工作室)可能设立的地点之外增加了地点。然而，她强调，Qumulo 也为其他行业提供了类似的好处，如石油和天然气行业。

“我们的石油和天然气客户也在做同样的事情。整个想法是移动整个工作负载，它有专门的应用程序、计算能力和元数据要求，并在云中建立这种足迹，”普雷斯利说。“我们所做的一切都是为了将传感器的数据从边缘转移到云中。数据位于我们的环境中，计算在 AWS、GCP 或本地进行。以前你能做这种工作的唯一方法是，如果你是一个超级计算型的人。”

Qumulo 还推出了 QumuloDR Cluster，它使用公共云增加了一层额外的数据保护。根据该公司的一份声明，该功能将 Qumulo 与 AWS 和 GCP 相结合，为用户提供“将他们的本地集群数据复制到云中运行的集群以获得额外保护”的能力，并在发生灾难性故障时“故障转移到云集群，在那里他们将可以访问他们所有的本地数据”。此外，用户能够保持群集连续运行，或者“在复制之前以编程方式打开它，以降低成本。”
