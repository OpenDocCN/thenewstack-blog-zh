# 如何让你的项目在 CNCF 变得更容易

> 原文：<https://thenewstack.io/getting-project-cncf-just-got-easier/>

管理和理解这些新的云原生架构是[云原生计算基金会](https://www.cncf.io/) (CNCF)旨在帮助全球开发者变得更容易的事情。在今天的[新堆栈制造商](https://thenewstack.io/podcasts/makers/)播客中，我们与 CNCF 执行董事[丹·科恩](https://www.linkedin.com/in/dankohn)和 CNCF 首席运营官[克里斯·阿尼斯奇克](https://www.linkedin.com/in/caniszczyk)谈论 CNCF 和云原生计算的整体方向。面试在 KubeCon/CloudNativeCon 进行，面试是上个月在西雅图进行的。

“我们试图避免的故事是，‘哦，这是如此新鲜和可怕的事情，我将什么也不做，坐几年，’我认为像这样的会议的真正信息之一就是围绕动力。科恩说:“做出这些部署决定的人不会独自去一个阴森恐怖的地方。

除了在 CNCF 任职(在 Kohn 签约之前，他是那里的临时执行董事)之外，Aniszczyk 还是[开放容器倡议](https://www.opencontainers.org/) (OCI)的执行董事，该倡议专注于容器映像格式和相关运行时的标准化。“我们需要就如何存放容器、如何布局以及如何运行容器达成一致，并尽可能保持标准，以便每个人都可以重复使用，我们不会为此争吵。相反，我们可以在上面的层上创新，”Aniszczyk 说。随着 OCI 即将发布图像格式的 1.0 版本，Aniszczyk 敦促社区继续参与改进该项目。“如果你在集装箱领域有股份，请参与进来。我们将不胜感激，”他说。

在随后的谈话中，Kohn 讨论了一种新的方式，让开发人员将他们的项目提交给 CNCF 进行考虑。对于那些考虑向 CNCF 提交项目的人来说，将他们的工作与现有的 CNCF 牧羊人库伯内特和普罗米修斯进行比较的想法可能会令人生畏。科恩说:“这产生了一个真正的问题，因为如果唯一能进入 CNCF 的项目本身已经取得了巨大成功，那么 CNCF 就没有真正增加多少价值。”。

正是这个因素导致科恩和 CNCF 技术监督委员会决定将开发人员提交的早期项目划分为一个新的类别，称为“初始阶段”“它允许不太成熟的项目在更早的阶段加入 CNCF，但不像孵化过程那样，如果他们加入了，他们基本上就加入了，初始阶段需要每 12 个月进行一次新的 TOC 投票。"

[云本地计算基金会](https://www.cncf.io/)赞助了这个故事。
