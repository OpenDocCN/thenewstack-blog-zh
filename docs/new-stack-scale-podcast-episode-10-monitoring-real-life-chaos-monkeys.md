# 新的 Stack @ Scale 播客节目 10:平台对混沌猴子

> 原文：<https://thenewstack.io/new-stack-scale-podcast-episode-10-monitoring-real-life-chaos-monkeys/>

对于平台的维护者来说，开发人员有时看起来像活着的、会呼吸的[混沌猴子](http://techblog.netflix.com/2012/07/chaos-monkey-released-into-wild.html)，永远以热情的、有时完全新奇的方式在基础设施上戳新的洞。

“热情的人试图以创造性和独特的方式使用平台，”[慈善专业](https://twitter.com/mipsytipsy)承认，他是 [Parse 移动平台](https://thenewstack.io/shuttering-parse-future-mobile-backend-service/)的基础设施技术负责人，这项技术最终被脸书收购。"银河系中最大的混乱来源是人类，对吗？"

在本期 [The New Stack @ Scale](/tag/the-new-stack-at-scale/) 播客中，梅杰斯接受了 New Stack 创始人亚历克斯威廉姆斯和 New Relic 主编弗雷德里克保罗[的采访。新遗迹的开发者托里·维瓦特也加入了他们的行列。他们讨论了广泛应用监控的优势和局限性，以及它可以为平台提供商的工程师提供的巨大支持优势。](https://twitter.com/TheFreditor)

[显示 10:手机应用监控](https://thenewstack.simplecast.com/episodes/show-10-mobile-application-monitoring)

与任何优秀的平台提供商一样，Parse 会安排值班工程师来调试客户问题，这些问题可能包括从深层基础设施故障到客户的 WiFi 无法工作等各种问题。随着 Parse 成为大约 100，000 个应用程序的所在地，解决问题所需的时间失去了控制。

“他们都不一样，”梅杰斯谈到收到的问题罚单时说。公司的工程师们“从零开始做每一个产品，试图弄清楚(客户的)体验是什么。当工程师们试图追踪非常困难和耗时的事情时，他们真的会精疲力尽。”

然而，当 Parse 被脸书收购时，工程师们可以利用这个社交网络巨头自己的[水肺监测工具](https://www.facebook.com/notes/facebook-engineering/under-the-hood-data-diving-with-scuba/10150599692628920/)，这就产生了很大的不同。

“这对我们来说简直改变了生活，”梅杰斯说。“如果我们无法找出根本原因，我们就开始加入键值对，您可以立即搜索它们，并在任何维度上进行聚合。”

梅杰斯说，诊断一个严重问题的平均时间从“几个小时到不可能”到大约 3 到 10 分钟。

然而，即使在最好的情况下，监控也有其局限性。

“每个人都想要银弹。他们希望新遗迹不仅能监控，还能预测和修复。只是现在还不存在。如果这很容易，它已经是你手机上的一个应用程序了，我就可以在别的地方工作了，”威瓦特说。“这需要时间的投入。在设备运行时查看，而不是在紧急情况下查看。”

[新遗迹](https://newrelic.com/)是新栈的赞助商。
