# VMware 的 Susan Wu:你不一定要成为一名程序员才能为开源做出贡献

> 原文：<https://thenewstack.io/vmwares-susan-wu-dont-coder-contribute-open-source/>

下面的采访是“开源领导者”系列的一部分，在这个系列中，我们描述了开源 IT 社区中的项目领导者，以了解他们如何开发他们的软件，以及运行开源项目带来的挑战和好处。

关于 [Susan Wu](https://twitter.com/susanwu88) 最令人难以置信的是，虽然她没有发布一行商业代码，但她已经管理了几十个第三方工程项目，允许开发者和公司通过开源的力量来沟通他们的工作。她帮助将 Oracle Enterprise Manager 与 BMC 和 HPE 等第三方系统管理软件之间的双向连接器推向世界，并将第三方网络和存储插件引入 Citrix Cloud Platform。

从 2015 年到去年 6 月，吴()在网络软件创业公司 Midokura 担任技术营销总监。“我从事内容和产品营销。我为首席技术官代写博客，在会议上发言，开发信息，向媒体和分析师做简报，发布推文/社交媒体，创建在线广告，页面更新。我还推动了需求挖掘，并负责第一阶段的销售渠道，”吴说。

如今，吴是 VMware 的高级产品营销经理。她与来自 IBM、Intel、Hitachi、Mirantis、SUSE、Walmart 和许多其他公司的作者共同撰写了三本 OpenStack 企业手册。

在业余时间，她还是 CloudNOW 的董事会成员，这是一个为从事云计算工作的女性设立的非营利性领导论坛。下个月，该组织将在加州山景城举办第六届年度“[云上顶尖女性](https://thenewstack.io/cloudnow-names-linkedins-neha-jain-one-top-women-cloud-work-using-ai-reduce-bias-hiring/)”活动。

吴是 OpenStack 的坚定支持者，他认为当权威人士向该平台扔石头并预言它的灭亡时，诸如 OpenStack 企业工作组的章程等努力表明 OpenStack 正在企业中获得牵引力。

她还提到了她的前公司 Midokura，她说该公司自 2011 年以来一直致力于 OpenStack 项目，该项目极大地帮助了 Midokura 开发自己的 MidoNet 网络虚拟化平台。“参与 OpenStack Neutron(最初是 Quantum)项目让我们可以无限制地接触开发人员和操作人员。我们通过参与中子核心团队来帮助塑造项目方向。对运营商的访问帮助我们增强了 MidoNet 插件，特别是针对大规模(超过 1000 个节点)OpenStack 部署，”她说。

在一个更大的公司，吴的角色可以很容易地分成三个部门:产品营销，分析师关系/公共关系和需求挖掘。"[基于客户的营销](https://azalead.com/blog/blog/what-is-account-based-marketing/) [ABM]帮助我专注于对销售至关重要的客户。自动化和专注是与企业客户打交道的关键驱动因素，”吴补充道。

吴说，吸引她去开源的是有机会与来自世界各地的人才合作，创造一个文化身份和优势的拼凑。“因为你的工作是公开的，同行监督促使你把工作做到最好。她说:“开源让我成为一名更好的营销人员。

除了 OpenStack 和 MidoNet，她参与的一些主要开源项目包括 Ubuntu、Apache CloudStack 和 Docker。

她的第一个项目是 Ubuntu。吴实际上离开了甲骨文公司，她在那里担任产品经理，加入了 Canonical 公司，在 Ubuntu cloud 公司担任产品营销经理。她说这对她来说是一个巨大的变化。“Oracle 的产品管理是一个面向流程的封闭系统。Ubuntu 是开源和透明的缩影。在 Ubuntu 社区，价值是基于技术贡献，而不是资历，”她说。

*(注:本文最初的采访是在她还在 Midokura 做开源产品营销人员的时候进行的)。*

[OpenStack 基金会](https://www.openstack.org/)和 [VMware](https://www.vmware.com/cloud-solutions/app-modernization/cloud-native-apps.html) 是新堆栈的赞助商。
