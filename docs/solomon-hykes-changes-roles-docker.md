# 所罗门·海克斯在码头改变角色

> 原文：<https://thenewstack.io/solomon-hykes-changes-roles-docker/>

[码头公司](https://www.docker.com/)的创始人所罗门·海克斯已经辞去了公司首席技术官的职务。据 Docker 的代表称，Hykes 已经担任了董事会副主席和首席架构师的新职务。此前，该公司的另一位主要联合创始人、前首席执行官本·戈卢布[于 3 月](https://venturebeat.com/2017/05/02/docker-replaces-ceo-ben-golub-with-former-concur-cofounder-and-ceo-steve-singh/)离开公司。

Hykes，或者网上所说的 Shykes，一直是硅谷宠儿 Docker 的避雷针。2007 年，Hykes 和 Sebastian Pahl 一起创立了这家公司。他们被著名的硅谷孵化器 [YCombinator](http://www.ycombinator.com/) 拒绝了两次，最终被心血来潮的保罗·格拉厄姆[接受。当时，这家公司名为 DotCloud，在 YCombinator 融资后不久，Pahl 于 2011 年离开。今天，帕尔在中间层担任工程总监。](http://www.paulgraham.com/)

Hykes 继续推动 DotCloud，这是一个面向云开发者的 PaaS 产品，直到 2013 年该公司对 Linux 容器的采用获得了牵引力。此后不久，该公司迅速发展，获得了近 2.5 亿美元的风险投资。

在此期间，Docker 的竞争环境发生了巨大的变化，公司越来越专注于解决企业市场的需求。就在最近，该公司宣布支持 [Kubernetes](https://kubernetes.io/) 开源容器编排引擎，似乎是为了自己内部开发的容器编排游戏，即 [Swarm Mode](https://docs.docker.com/engine/swarm/) 。

虽然 Docker 仍然声称 Swarm 在安全方面有一席之地，但 Docker 企业版现在将包括 Kubernetes 和 Swarm。该公司是管理 Kubernetes 的[云本地计算基金会](https://www.cncf.io/)的创始成员之一，Hykes 是该组织的[技术监督委员会](https://www.cncf.io/people/technical-oversight-committee/)成员。

去年，Hykes [在开源容器](https://thenewstack.io/container-format-dispute-twitter-shows-disparities-docker-community/)社区引发了争议，在推特上发布了一些煽动性的东西，比如指责 [OCI 容器](https://www.opencontainers.org/)规范是“假标准”，并怀疑任何其他声称提供完全 Docker 支持的公司的声明。他当时断言，Docker 行动太快，无法得到 90%的供应商社区的支持。

该公司周一还宣布，柯达首席执行官杰夫·克拉克被任命为公司董事会成员。Hykes 角色变化的消息是通过电子邮件发布的，并得到了 Docker 的证实。
