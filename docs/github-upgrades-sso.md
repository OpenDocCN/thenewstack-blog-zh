# GitHub 现在提供单点登录

> 原文：<https://thenewstack.io/github-upgrades-sso/>

支持 5200 万个项目的开源代码共享平台 GitHub 已经升级了其代码托管服务，提供单点登录(SSO)认证。该公司使用了 [SAML](https://www.oasis-open.org/standards#samlv2.0) (安全声明标记语言)，这是一种用于在身份和服务提供商之间交换认证和授权数据的行业标准数据格式，可以支持各种联合身份服务。

GitHub 用户可以立即开始使用新功能，这也包含在今天发布的[GitHub Enterprise](https://enterprise.github.com/home)2.9 版本中。

GitHub.com 用户现在可以从联合身份服务登录，如 [Ping Identity](https://www.pingidentity.com/en.html) 、 [Okta](https://www.okta.com/) 、 [OneLogin](https://www.onelogin.com/) 、 [Azure Active Directory](https://azure.microsoft.com/en-us/services/active-directory/) ，以及开源的 [Shibboleth](https://shibboleth.net/) 项目。

GitHub 产品高级总监 Connor Sears 在一次电话采访中说:[向 SSO 的转变代表了市场的一个更大的变化。公司正在从孤立的软件和服务转向软件即服务(SaaS)，如聊天、电子邮件和代码托管。](https://www.linkedin.com/in/connors/)

[SSO](http://searchsecurity.techtarget.com/definition/single-sign-on) 允许跨这些服务自动共享登录凭证，简化了用户和系统管理员的流程。对于企业而言，它允许自动配置和取消配置员工，并允许员工记住一个用户名/密码组合。此外，当员工离开公司时，SSO 提供了一个可以撤销所有前员工访问权限的单点，从而提高了安全性。

与 SSO 一起，GitHub 正在推动系统升级，以将正常运行时间保证提高到 99.95%。它还为工作时间内提出的支持请求提供不到 8 小时的响应时间。从周一到周五，GitHub 将提供 24 小时支持和 8 小时响应时间。

对于那些希望或需要在防火墙后运行内部、数据隔离的 GitHub 的客户，GitHub Enterprise 版使他们能够比以前更容易地跨更多平台连接内部实例。例如，现在支持谷歌云。“GitHub 可以去他们在的任何地方，”康纳斯说。

他指出，有趣的是，对内部解决方案的需求与公司规模无关。几家大公司在 GitHub.com 经营他们所有的开发项目。但其他公司，特别是金融或医疗保健行业的公司，要求数据隔离，主要是出于法律合规性的原因。其中一些公司运行 GitHub 的混合版本，一些项目在 GitHub.com 上运行，一些在企业版的防火墙后运行。

康纳斯说，随着时间的推移，观察内部公司向开源的转变将是有趣的，他预测公司将在 GitHub.com 完全转向 SaaS 模式。
