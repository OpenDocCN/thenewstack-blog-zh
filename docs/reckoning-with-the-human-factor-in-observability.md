# 在可观察性中考虑人的因素

> 原文：<https://thenewstack.io/reckoning-with-the-human-factor-in-observability/>

可观察性被广泛误解，但在一个安全漏洞增加、更多业务在网上进行的时代，它从未如此重要。组织应该如何考虑他们在多云环境中的资源？他们应该采取什么策略在黑客之前发现安全漏洞？此外，DevOps 团队可以采用哪些文化变革来加强他们的可观察性？

在这一集的 New Stack Makers 播客中，Check Point 的技术营销工程师兼云原生和网络安全宣传员 [Maya Levine](https://www.linkedin.com/in/maya-levine-2ba05aa7/) 与共同主持人[New Stack 的创始人兼发行人 Alex Williams](/author/alex/) 和[TNS 的特写编辑 Heather Joslyn](https://www.linkedin.com/in/heather-a-joslyn/) 一起讨论了可观察性现在意味着什么。

考虑人的因素的可观察性

莱文告诉制作者们，可观察性不仅仅是挫败重大攻击，或者用她自己的话说，“抓住网络罪犯留下的痕迹。”它是关于计算在创建和延续安全漏洞中的人为因素:例如，组织广泛地在内部授予权限，如果坏人利用打开的门获得更多的访问权限，这种便利很容易变成安全噩梦。

多云和混合云架构的流行增加了可观测性策略的复杂性。莱文告诉我们，团队再也不能假设他们“在云中是安全的”

她说，要适应这种新的现实，就需要对可观察性和安全性的思考发生重大转变。“挑战在于，你需要为它的每一步提供安全保障，”她告诉创客们。“这就是为什么您需要将实现安全性作为开发过程的一部分。

Levine 说，虽然安全性需要从一开始就嵌入到应用程序开发中，但 DevOps 团队也需要认识到人工智能是如何被黑客用作工具的。

莱文解释说，如果黑客正在使用人工智能来利用你的系统的漏洞，这种情况需要使用类似的技术来应对。“对自动攻击的手动响应总是太慢，尤其是在云中，”Levine 说。“因此，你可以应用的自动部署的保护越多，甚至无需人工干预，你就会变得越好。”

另一方面，她指出，[人工智能可以提高自身的可观察性](/how-ai-observability-cuts-down-kubernetes-complexity/)，有助于更快地捕捉异常:“我们有能力识别什么是正常行为，什么是不正常行为。”
