开源是免费的。那么如何围绕它建立一个公司呢？

肯定有一些人认为开源和金钱不应该混为一谈，真正的开源应该永远是免费的和由社区驱动的。这是一种我觉得很幼稚的态度——毕竟，维护和贡献开源的软件工程师需要吃饭。

不仅如此，开源还用于商业环境。如果开源项目被公司用来赚钱，为什么不应该以某种方式向创造它们的人支付报酬？

**这是针对软件工程师创业的各个方面的每月专栏的第一部分。在接下来的几个月里，我计划写一些关于开源项目货币化的各个方面，创建和扩大一家软件基础设施公司，以及如何从一名工程师转变为一名企业家，无论这意味着创建一家风险投资公司还是建立一个人的商店。**

有很多方法可以解决开源软件和金钱的问题。在本专栏中，我将从一群创始人的角度来阐述开源商业策略，他们最初的目标是创建一家软件公司。这与一些人形成对比，他们开始一个开源项目，多年来培育和发展这个项目，然后决定将其货币化。这两种方法通常会导致相同的结果，但是第一步是不同的，初始策略也是不同的。

## 获得报酬的挑战

对于许多有基础设施工程背景的人来说，出于哲学原因、实践原因或者两者兼而有之，创办闭源公司的想法是不切实际的。

“开源几乎已经成为企业基础设施工具的默认采用模式，”数据 API 公司 [Hasura](https://hasura.io) 的联合创始人 [Rajoshi Ghosh](https://www.linkedin.com/in/rajoshighosh/?originalSubdomain=in) 说。“我们很早就决定 Hasura 平台的]数据访问部分将是开源的。”

[Armon Dadgar](https://www.linkedin.com/in/armon-dadgar/) 、[联合创始人](https://thenewstack.io/hashicorps-mitchell-hashimoto-on-when-to-step-down-to-the-job-you-love/)和 [HashiCorp](https://www.hashicorp.com/?utm_content=inline-mention) 的首席技术官指出了围绕开源项目建立业务所固有的挑战，尤其是在早期。

“当我们创建 HashiCorp 时，我们想解决一系列具体的问题，但我认为我们对 HashiCorp 的业务是什么没有任何概念，”Dadgar 说。“那是一家 C 公司，我们正在创建一家公司，但我们不清楚如何将它商业化，从这个意义上说，我们实际上是从一系列开源产品开始的。”

但是，如果你决定免费赠送一定数量的知识产权，你最终如何获得报酬呢？

有三种主要的[开源商业模式](https://thenewstack.io/open-source-licenses-who-holds-the-power/)，它们适用于任何规模的公司，从单个工程师赚钱的副业项目到[红帽](https://www.openshift.com/try?utm_content=inline-mention)。它们是:

*   支持和服务(即咨询)。这通常被称为红帽模式，以最著名的从业者命名。
*   企业发行版，公司出售具有一些附加功能的专有软件，客户自己运行软件。
*   软件即服务(SaaS)选项。

那么这些选项是如何工作的呢？个人或公司如何决定哪一个最适合特定情况？让我们开始吧。

### 红帽模型

支持和服务模型是最古老、最成熟的开源商业模型。

“如果我们回到 2015 年，开源成功故事的例子并不多，”Dadgar 说，谈到如何将 HashiCorp 的开源项目转化为可行、盈利的业务的过程。“那是红帽子。就这样。没有人真正在任何有意义的程度上复制红帽。”

这种模式是一种咨询模式——除了开源项目之外没有任何软件产品，但公司提供专业服务。这通常是开源项目盈利最少的方式，但它也需要较少的产品开发初始投资——因为除了免费的开源项目，没有其他产品。

### 企业版

在这种模式中，有一个软件版本在商业许可下出售，但通常做的事情与开源项目相同。这有时被粗略地描述为具有漂亮用户界面的开源项目，但是企业版通常不仅仅有更好的用户界面。企业版通常还附带支持合同。

在大多数情况下，企业版将基于开源项目——但情况并非总是如此。无论如何，产品开发过程与开发一个闭源软件产品没有什么不同。

### SaaS

最近，开源项目的 SaaS 版本越来越受欢迎，比如 [MongoDB 的](https://www.mongodb.com/cloud/atlas/?utm_content=inline-mention) Atlas。这是你所能得到的离开源体验最远的体验——SaaS 提供商会处理所有的事情，用户所要做的只是输入一张信用卡。

这意味着 SaaS 公司要承担与运营相关的一切。这也意味着 SaaS 公司不得不担心所有 SaaS 公司都担心的事情，比如他们的云成本与实际使用和定价的关系。

## 做出选择

开源项目的货币化选择并不是无限的，但它仍然是一个重要的选择——尤其是当你的工作资源有限的时候。

Dadgar 说，潜在的开源企业家在决定他们的战略时需要问的第一个问题是“谁是我们的客户？”在哈希公司，“这是我们第一次也是最大的一次灵魂探索。”

他说，如果你的客户是中小型企业，你希望他们能够刷信用卡，每月支付几百美元并使用该产品。如果你的客户是福布斯全球 2000 强，他们不会刷信用卡，他们会让你经历一个采购过程。

对于那些想同时进入两个市场的人来说，Dadgar 说这很好——当你是一家 1500 人的公司时，就像 HashiCorp 现在这样。一开始，对于 1、5 或 15 个人来说，专注于一个市场和一个盈利策略是很重要的。

“开源企业很难，”Ghosh 说。意向性是关键。明确什么是开源，什么是付费选择，以及遵循什么商业策略将使这个过程变得容易得多。
