# 对于企业来说，Kubernetes 是值得投资的

> 原文：<https://thenewstack.io/for-enterprises-kubernetes-is-worth-the-investment/>

Kubernetes (K8s)的推出为企业铺平了道路，为他们提供了创业公司以前享受的工具和能力，以大规模发展他们的业务。Kubernetes 已经被大多数诞生在云端的新公司广泛使用。但是企业对 K8s 的采用并不普遍。

首先，K8s 很复杂，涉及到一个陡峭的学习曲线。一旦掌握了额外的技术，包括缓存管理和负载平衡，实现 K8s 的真正好处就会显现出来；它还需要使用一套新的监控和健康检查工具。K8s 的引入要求对交付渠道进行重大变更，并因此改变了既定的程序和监管控制。任何成本节约都是不确定的——过度分配资源是非常容易的。

K8s 有潜力为企业做它为互联网创业公司所做的事情:缩短上市时间，改善服务水平协议(SLA)和提高底线。今天，每个企业都是软件企业。企业首席信息官的任务是交付高质量的应用程序和客户体验，与所有常见的巨头相媲美。对创新速度和敏捷性的需求正在推动公司构建、运行和保护其现代应用的方式，这反过来又将软件架构转变为微服务。反过来，这些微服务依赖于容器化的应用程序和流程编排来加快改进和新功能的部署，这对于维护高度可用、安全的客户体验至关重要。Kubernetes 可以通过在许多关键领域引入自动化来做到这一点，例如应用服务的部署、应用网络的配置和跨基础设施的服务分发，等等。

Kubernetes 不仅仅是一个短暂的幻想，有迹象表明它将长期存在。根据最近对 it 专业人员的一项调查，IT 部门发现超过一半的人在生产环境中运行 Kubernetes，三分之一的人运行 26 个以上的集群，五分之一的人运行 50 个以上的集群。虽然这些信息没有考虑公共云服务的持续使用，但它无疑表明 Kubernetes 正在本地数据中心环境中获得牵引力。对于 IT 专业人士来说，考虑新工作时，薪资似乎不是主要驱动因素——通常情况下，职业发展和学习的机会是更强大的动力。K8s 等技术的引入不仅提高了 IT 员工的工作效率，还提供了从事有趣项目的空间，从而提高了工作满意度并留住了员工。随着企业继续投资 k8s，对于掌握 K8s 的员工来说，竞争只会加剧——因此，培养自己的才能是一种越来越有吸引力的方法。

## 我们应该准备什么？

就像任何其他主要的 IT 投资一样，实施 K8s 也会有一定的风险。但是，如果你什么都不做，也要准备好衡量失去市场承受能力的代价。做好准备，明白如果你接受这项新技术，你可以做一些事情来减少风险。任何 K8s 实施都需要在与其他技术集成方面做出巨大的技术努力和投资。K8s 依靠其他项目来管理注册、安全、遥测、网络和自动化等服务，其中许多项目是开源的。您需要能够认识到所有这些，并相应地将其纳入您的实施计划。尝试使用一种可用的经过测试(和集成)的“企业就绪”产品。
