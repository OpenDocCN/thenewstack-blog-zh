借助Cloud Foundry，Cloud.gov为政府工作提供了符合联邦政府标准的云

原文：https://thenewstack.io/community-compliance-strengthening-federal-government-open-source/#0001-01-01

确保美国政府机构拥有合规的基于云的基础设施是总务管理局 18F数字服务的任务，该服务创建了cloud.gov，这是一个专门为联邦机构提供的基于云铸造的托管云服务。

在这一集的中，新的Stack Makers 嵌入下面，新Stack创始人 Alex Williams 和联合主持人 Lee Calcote 与18F软件工程师兼cloud.gov总监Diego lapid uz、18F创新专家兼产品负责人Bret Mogilefsky以及戴尔首席技术官办公室高级首席工程师Barton George坐在一起。采访发生在2016年云铸造峰会期间。请继续阅读，详细了解18F和戴尔如何利用开源和Cloud Foundry不仅为当今的联邦机构开发安全技术，还跨云开发灵活的基础架构。


社区遵从:用开源加强联邦政府

对于18F来说，通过Cloud Foundry，面向政府的开发者平台【cloud.gov T21】的创建已经成为可能。Lapiduz解释说，通过防止供应商锁定，最初无法将其现有堆栈迁移到公共云的政府机构可以选择在私有云上使用Cloud Foundry。

莫吉列夫斯基指出，像cloud.gov这样的项目是开源的，随着越来越多的政府机构从传统技术中转移出来，这为未来的持续创新创造了条件。“我们开始用Cloud Foundry解决我们自己在部署、运营和管理方面遇到的问题。我们决定，如果我们要为自己使用它，我们也可以把它提供给其他机构。我们是完全公开的，所有的东西都在GitHub上。让cloud通过联邦合规流程意味着其他机构可以跟随我们的脚步，轻松构建联邦合规云。”

在任何基础设施中，支持多个堆栈都是一项挑战，但当政府机构参与进来时，这可能会很快变得更加详细。“Cloud Foundry确实帮助我们跟踪构建包以及基础设施和平台的基础层。Cloud Foundry允许我们一路向下修补所有东西。”拉皮迪兹说。Mogilefsky强调，Cloud Foundry的 Concourse 也有助于快速构建新版本的Cloud Foundry平台，主要依靠社区上游获得新的Cloud Foundry版本和构建包。

对于戴尔来说，整合其开源团队是一项挑战，尽管这仍然是该公司致力于解决的问题。George提出了一个许多组织在开发应用程序时应该考虑的问题:“我们已经看到了许多不同的努力。我们关注的是如何将这些不同的开源团队整合到一个更统一的过程中？我们如何让整体大于部分之和？”

Cloud Foundry 是新堆栈的赞助商。
