# 为什么 Azure App Service 拥抱容器，现在运行在 Linux 上

> 原文：<https://thenewstack.io/azure-app-service-embraces-containers-now-runs-linux/>

[为什么 Azure App Service 拥抱容器，现在运行在 Linux 上](https://thenewstack.simplecast.com/episodes/why-azure-app-service-embraces-containers-and-now-runs-on-linux)

在使分布式系统成为可能的社区中，一个关于复杂性的长期叙事多年来一直是一个故事情节。比如[微软](https://azure.microsoft.com/en-us/?v=17.14)最近宣布 [Azure App Service](https://azure.microsoft.com/en-us/services/app-service/) ，一个创建基于 Azure 的应用的平台，现在[运行在 Linux](https://azure.microsoft.com/en-us/blog/general-availability-of-app-service-on-linux-and-web-app-for-containers/) 上。底层基础设施依赖 Docker 来运行容器，这些容器要么是内置的特定堆栈版本，要么是开发人员自己定制的容器。通过加入开源社区并使用技术为开发人员抽象复杂性，微软可以为开发人员提供更容易构建 web 应用程序的切实方法。

微软项目经理 Ahmed Elnably 在开源峰会上接受采访时表示:“它允许(开发者)快速部署、运行和扩展网络应用，而无需考虑基础设施。

大多数云原生技术首先由开发人员和工程师采用，他们希望解决在不同类型的基础设施上运行多种服务所带来的底层问题。他们创建自己的服务，并经常以开源技术的形式提供服务。随着时间的推移，Docker 等技术已经成为平台的核心。早在 2013 年，Docker 创建者 [Solomon Hykes](https://github.com/shykes) 曾[将](https://techcrunch.com/2013/07/28/the-matrix-of-hell-and-two-open-source-projects-for-the-emerging-agnostic-cloud/)在不同基础设施上运行服务描述为一个地狱矩阵。今天，我们看到了这一运动的实现，服务建立在开源技术上，使用容器来提供开发者可以用来构建应用的服务。
