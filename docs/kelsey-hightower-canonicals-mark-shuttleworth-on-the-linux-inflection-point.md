# Kelsey Hightower，Canonical 的马克·舒托沃尔斯谈 Linux 拐点

> 原文：<https://thenewstack.io/kelsey-hightower-canonicals-mark-shuttleworth-on-the-linux-inflection-point/>

Canonical 广受欢迎的 Ubuntu Linux 发行版(这位作者的最爱)继续在 Kubernetes 的广泛采用中扮演着重要角色。谷歌云平台首席开发者倡导者[凯尔西·海塔尔](https://twitter.com/kelseyhightower?lang=en)和 [Canonical](https://canonical.com/) 首席执行官[马克·舒托沃尔斯](https://en.wikipedia.org/wiki/Mark_Shuttleworth)在最新一集[新堆栈制造商](/podcasts/makers)播客中总结道，这种默默的支持是理所应当的。[TNS 的创始人兼发行人亚历克斯·威廉姆斯](/author/alex/)主持了这一集。

退一步来说，Ubuntu 和一般意义上的 Linux 已经变得更容易使用，超越了许多人曾经认为的服务器操作系统和 Windows 的神秘替代品。

[凯尔西·海塔尔，马克·舒托沃尔斯:Kubernetes 依赖 Linux](https://thenewstack.simplecast.com/episodes/kelsey-hightower-mark-shuttleworth-kubernetes-relies-on-linux)

Hightower 说:“这是一个转折点，Linux 已经从命令行服务器端的东西变成了可以在桌面上运行的有意义的用户界面的东西，感觉我们正在缩小与所有其他流行的开放操作系统的差距。”。

如今，操作系统在后台运行，开发人员甚至操作人员能够更专注于运行在 Linux 之上的内容。Hightower 说，例如，那些在云本地空间和 Kubernetes 环境中工作的人甚至可能倾向于报告他们“可以忽略底层细节”。

Hightower 说:“每当出现技术转型时，我们都会听到这样的话:人们认为我们站在肩膀上到达我们现在的位置……可以默默无闻。”

舒特尔沃斯说:“如果你能造出一些让人们感到舒服的东西，让他们不再担心，你实际上释放了所有的智力能量，然后专注于可以说更有趣的问题。”。“他们可能会浪费在细节上的时间，现在你节省了这些时间，最终，这些时间变成了无人驾驶汽车和无人机，以及各种令人兴奋的机器人技术，这些发明有效地推动了未来的发展。”

最终，Kubernetes 的采用和管理也有望减少开发人员和运营团队的顾虑。例如，Canonical 的 [MicroK8s](https://microk8s.io) 是为了使 Kubernetes 更易于管理而创建的，其方法之一是自动化 Kubernetes 数据存储的节点选择过程。

舒特尔沃斯说，有了 MicroK8s，就“没有你必须连接的选项、旋钮或转盘”。“MicroK8s 是一个例子，我们曾经说过，‘看，如果你发现自己远离基础设施，你不在谷歌云和 [VMware](https://tanzu.vmware.com?utm_content=inline-mention) 上，你在某个尴尬的地方，你只是想要最简单的 Kubernetes，那么我们可以用 MicroK8s 为你做一些事情:一个命令安装，一个命令[创建]集群，你就大功告成了。"

两人还讨论了最近[发布的一份来自 Canonical 的报告](https://juju.is/cloud-native-kubernetes-usage-report-2021)，该报告发现 45.6%的受访者报告在生产中使用 Kubernetes，只有 15.7%的受访者报告只使用 Kubernetes，21.4%的受访者管理超过 500 台机器。该调查基于来自 1200 名受访者和分析师的 50 个问题，他们来自代表谷歌、[亚马逊网络服务](https://aws.amazon.com/?utm_content=inline-mention)、[云本地计算基金会](https://cncf.io/?utm_content=inline-mention)和其他组织的行业思想领袖。
