# 为什么第三方安全采用必须变得更好

> 原文：<https://thenewstack.io/why-third-party-security-adoption-must-get-better/>

来自帕洛阿尔托网络公司的 Prisma Cloud 赞助了这次播客。

几年前， [Target](https://krebsonsecurity.com/2014/02/target-hackers-broke-in-via-hvac-company/comment-page-3/) 和 [Equifax](https://thenewstack.io/equifax-data-breach-shows-perils-not-patching-timely-manner/) 黑客攻击等重大攻击成为头条新闻。但是，这些臭名昭著的攻击并没有给许多组织(如果不是大多数的话)敲响警钟。他们缺乏适当保护其数据所需的安全工具、流程和文化，[雨资本](https://www.linkedin.com/in/chenxiwang88/)的执行普通合伙人[王晨曦](https://raincapital.com/)博士说。

“每个人都读过这些头条新闻，但将它们转化为(组织)今天所做的工作，我认为仍有差距，”王说。“作为安全行业的专业人士，包括我自己在内，我们需要更多地接触邻近的社区，尤其是现在的开发人员。我的意思是，软件正在吞噬世界，而开发人员是驱动软件的人，所以我们需要与开发人员合作，让这一切成为现实。”

在本期由 New Stack 创始人兼发行人 Alex Williams 主持的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，王谈到了这些以及其他第三方安全趋势。该播客是在预计于 6 月 24 日举行的虚拟峰会上录制的。

[王晨曦博士为什么第三方安全采用必须变得更好](https://thenewstack.simplecast.com/episodes/chenxi-wang-ph-d-why-third-party-security-adoption-has-to-get-better)

像 Target 和 Equifax 的漏洞一样，政府支持的攻击也成为头条新闻。例如，根据美国司法部的消息，一个朝鲜黑客团队涉嫌策划了针对勒索软件的 [WannaCry 漏洞](https://www.us-cert.gov/sites/default/files/FactSheets/NCCIC%20ICS_FactSheet_WannaCry_Ransomware_S508C.pdf)。

不管攻击的来源、安全策略和所采用的工具如何，都必须考虑到绝大多数攻击都是出于经济动机。“总的来说，黑客会追踪金钱的踪迹，”王说。“他们希望攻击拥有最有价值数据的地方和系统。”

随着越来越多的组织向云迁移，存储在云上的数据通常被视为潜在的金矿，这些人会寻找漏洞来访问敏感且通常会泄露的数据，以进行代理或利用。与此同时，DevOps 和 [DevSecOps](/devsecops-not-just-about-security/) 团队经常缺乏他们需要的工具集来锁定对许多组织来说是一个新的世界。监控云日志的困难是他们在尝试管理云服务安全性时面临的挑战的一个例子。

“我无法告诉你有多少安全专业人士告诉我，他们淹没在这些云日志中，他们需要一些东西来帮助他们聚合、理解、分析和可视化它们，”王说。“我认为这是云服务提供商目前没有提供的一个缺口。”

DevOps 和 DevSecOps 团队需要的工具还必须能够包含 DevOps 对开源工具的日益依赖和采用——这可能会带来挑战。

“开源工具易于操作，许多开发人员都喜欢它们，但对于企业安全来说，要完成企业安全任务，它们往往不能提供正确的方法来分析数据或轻松地与其他工具集成，所以你必须建立这些工具，”王说。“因此，我认为在[信息安全](https://en.wikipedia.org/wiki/Information_security)中有一个地方，有人可以介入并说，‘好吧，我将管理你的具有安全隐患的开源工具，并在这些工具之上给你企业级的控制。’"

归根结底，安全还必须解决保护受害者的问题。王说，在这方面，安全政策必须有“利他的一面”。“我们正在保护消费者的财务数据，而在电话线的另一端，消费者是有银行账户的 80 岁老人，”王说我喜欢能够提供保护的那一面。个人消费者和用户。"
