## DevOps 是在 Kubernetes 上交付应用的关键

Kubernetes 上的应用交付从在基于容器的云原生架构中将应用构建为一组微服务开始。但是开发运维实践才是真正改变一个组织的应用架构的东西；它们是使应用程序在 Kubernetes 上运行的所有模式和实践的基础。开发运维转型只有在组织的价值观与其开发应用架构的方式保持一致的情况下才能实现。

在这种新优化的组织云原生转型方式中，Kubernetes 是推动者，而不是完整的解决方案。您的组织必须实现最适合您自己的业务需求和结构的工具和实践，以便实现这个开源平台的全部承诺。Kubernetes 项目文档本身就是这么说的。

> Kubernetes“不部署源代码，不构建您的应用程序。持续集成、交付和部署(CI/CD)工作流由组织文化和偏好以及技术要求决定。”

这是怎么做到的？这不是一套规则。它是流入组织并影响应用程序架构开发方式的一组实践。这就是 DevOps，它的潮流现在已经深入到拥有现代应用程序架构的组织内部，通过持续的交付得以体现。
