# 哈希公司如何扩大领事服务网络的覆盖范围

> 原文：<https://thenewstack.io/how-hashicorp-widened-the-reach-of-the-consul-service-mesh/>

HashiCorp 通过扩大其不同高度分布式服务和环境的范围来扩展其 Consul 网络控制平面，同时简化和扩展其合规性和策略管理功能。

该公司表示，通过在今天发布的 Consul 1.8 中添加网关选项和合规功能，HashiCorp 已经使控制平面能够在单个界面中管理许多不同的环境。这些可能包括在容器、裸机上的 Kubernetes 或虚拟机(VM)、传统数据中心或地理位置广泛分散的多云环境中运行的服务和应用程序。

“我们对客户很有用，因为我们通过单个管理平台提供了一个跨[不同环境]的层。[哈希公司](https://www.hashicorp.com/)产品营销副总裁 [Amith Nair](https://www.linkedin.com/in/amith-nair-11b793) 告诉新堆栈:“客户面临的挑战是，他们有许多服务位于服务网络之外，如传统应用程序，需要将它们纳入同一框架。“那么，在您的服务网格中，服务如何与您的应用程序对话，服务网格中的应用程序如何与外界对话？咨询 1.8 解决了这个问题。”

Consul 1.8 的审计日志和单点登录(SSO)功能(是企业版的一部分)扩展了控制平面的治理和合规性功能。Nair 说:“这些提供了服务和应用程序层发生的事情的更多可见性，如服务的健康状况、上次被触及的时间、使用的策略等。”

HashiCorp 的 Consul 1.7 改进包括帮助 DevOps 团队提高在共享集群中创建隔离管理环境的能力。Consul 1.8 有助于更好地管理许多组织越来越需要管理的大量服务。这包括在多个环境中控制和协调不同服务的挑战，同时还要设置防火墙策略和映射所有各种相关端口。

“在你的网络堆栈中有大量的依赖性，而你在新的世界中进行网络连接的方式与传统的静态 IP 非常不同，”Nair 说。“现在，网络必须更加动态，因为服务的数量越来越多，开发人员如何构建所有这些应用程序和服务也带来了额外的挑战。”

有了这些新服务，组织通常会有一些在传统服务网格环境中工作的应用程序，而其他的则不能，例如在传统数据中心环境中管理的应用程序。

奈尔说，Consul 1.8 有助于整合所有新旧服务，以便“你的网络内部和外部的所有内容都可以相互交流”。

在一次演示中，HashiCorp 列出了 Consul 1.8 的新功能:

*   网状网网关上的广域网联盟。
*   终止网关。
*   入口网关。
*   JWT 认证方法。
*   第 7 层流量管理增强。
*   ACL UX 增强。
*   审计日志记录(企业版)。
*   单点登录(使用企业版)。

在这种情况下，组织通常依赖服务网格来“更好地发展容器集群并加速基于 Kubernetes 的平台的采用。”[汞合金洞察](https://amalgaminsights.com/)的分析师汤姆·佩特罗塞利告诉新堆栈。“这就是 HashiCorp 与 Consul 合作的重要性，”Petrocelli 说。

人们常说，在 Kubernetes 环境中管理应用程序和数据需要服务网格。例如，除了简单集群的管理，控制应用流量(第 7 层流量)“南北和东西两个方向”也很重要。“这是大多数组织采用服务网格的时候，”Petrocelli 说。

“随着集群的增长，Kubernetes 中简单的网络选项是不够的，”Petrocelli 说。当您开始拥有多个大型群集时，管理群集之间的流量以及多个群集与外界之间的流量会变得更加困难

Petrocelli 说，组织通常也在扩大他们必须管理的新平台和应用程序的数量。与此同时，Petrocelli 说，微服务部署在集群中，通常分布在“整个组织中，跨地理位置，跨公共云和私有云的混合环境”。“一旦这种情况开始大规模发生，管理微服务集群、通过 API 使用其功能的应用程序以及 n-tier 等传统系统之间的流量就不可能手工完成，”Petrocelli 说。“在这种情况下，您需要一个能够管理所有不同流量的控制平面和服务网格。这就是《执政官》的新功能。”

Petrocelli 说，如上所述，随着集群规模和数量的增长，服务网格控制平面需要能够管理所有流量，同时集群目标的多样性也在扩大。“领事正在寻求解决这些规模问题，”帕特塞利说。

至于治理和合规，主要目标之一是制定政策并记录已经实施的内容。我认为大多数服务网格控制平面在某种程度上都是这样做的。Consul 只是在企业范围内这样做。"
