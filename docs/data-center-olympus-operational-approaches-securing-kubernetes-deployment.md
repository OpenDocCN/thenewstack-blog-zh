# 数据中心是奥林巴斯:确保 Kubernetes 安全的操作方法

> 原文：<https://thenewstack.io/data-center-olympus-operational-approaches-securing-kubernetes-deployment/>

[数据中心是奥林巴斯:保障 Kubernetes 部署的操作方法](https://thenewstack.simplecast.com/episodes/the-data-center-is-olympus-operational-approaches-to-securing-a-kubernetes-deployment)

如果数据中心在奥林匹斯山，Kubernetes 的环境是什么？这是我在本期 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中向 [Alcide](https://www.alcide.io/) 联合创始人兼首席技术官 [Gadi Naor](https://www.linkedin.com/in/gadinaor/) 提出的第一个问题，该播客包含在我们将于下周发布的最新电子书“[Kubernetes Deployment&Security Patterns](https://thenewstack.io/ebooks/kubernetes/kubernetes-deployment-and-security-patterns/)”中。Naor 将神话中的奥林匹斯山(众神之家)与现代数据中心进行了类比。

这种比较带来了一个关于整个组织如何看待基础设施的概念。Olympus Naor 描述的数据中心现在拥有 Kubernetes 的能力。有了这种力量的能量和力量，今天的组织如何评价和保护他们的核心资产就成了头等大事。

第一个价值必须由组织如何对待 Kubernetes 中的安全性来定义。Naor 说，安全性并不是最好的附加功能。它必须是部署本地的。第二个考虑因素是零信任，这在安全第一的方法中得到了最好的体现。必须考虑集群以及微服务如何在互连的 Kubernetes 架构中进行事务处理。开发人员发现微服务很有吸引力，但是操作人员需要可观察性来了解组件的行为。安全层为运营团队提供了一种监控集群异常情况的方法。现有的网络安全不能满足容器带来的新需求。它可能会提供一个自顶向下的政策，但需要的不仅仅是这些。现在需要的是 Naor 所说的“政策融合”。

[cyclone slider id = " kubernetes-series-book-2-赞助商"]

云原生技术相当复杂。对虚拟机基础架构仍有实际需求。这是毫无疑问的。实际上，当今 Olympus 的复杂性需要复杂的策略融合，允许以一种内聚的方式统一多个策略，以便安全可以大规模运行，并且组织可以真正利用 Kubernetes 的可扩展性。
