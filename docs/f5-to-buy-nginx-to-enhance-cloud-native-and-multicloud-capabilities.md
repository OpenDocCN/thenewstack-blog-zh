# F5 将收购 NGINX 以增强云原生和多云功能

> 原文：<https://thenewstack.io/f5-to-buy-nginx-to-enhance-cloud-native-and-multicloud-capabilities/>

应用控制器交付软件提供商 [F5](https://www.f5.com/) 正计划收购 [NGINX Inc.](https://www.nginx.com) ，后者管理着流行的开源 web 服务器/负载平衡器和同名反向代理。在此次收购中，F5 将以 6.7 亿美元的价格收购所有私人持有的 NGINX 股票。

F5 声明称，F5 计划使用 NGINX 软件和人才“在所有环境中实现多云应用服务”。

据该公司称，F5 计划用 F5 自己的安全技术以及一系列“云原生创新”来增强 NGINX 软件，以增强负载平衡。

该公司还发誓要支持 NGINX 开源项目。现任 NGINX 首席执行官格斯·罗伯逊和创始人伊戈尔·塞索耶夫和马克西姆·科诺瓦洛夫将继续运营 NGINX。品牌名称也将保留。

“我们有互补的优势，”罗伯森在博客文章中解释道。“我们的 web 和应用服务器、微服务和 API 管理解决方案补充了 F5 的应用管理、应用安全和基础设施安全解决方案。即使在应用交付控制器(ADC)有一些重叠的情况下，NGINX 也创建了一个轻量级的纯软件版本，补充了 F5 云、虚拟和物理设备选项。”

展望未来，该交易将使 F5 在服务网格解决方案中占据强势地位，这将有助于控制基于 Kubernetes 的云原生系统的网络、安全和监控方面。反过来，NGINX 将更多地接触 F5 的全球销售团队和合作伙伴生态系统。

根据 2018 年下半年发布的 CNCF 调查，Kubernetes 最广泛使用的入口提供商是 NGINX，占受访者的 64%。TNS 赞助商 HAProxy 以 29%的份额位居第二，F5 Networks 以 15%的份额紧随其后。

有了 F5 的支持，NGINX 公司不必担心其开源商业模式会在短期内取得成功。据 Netcraft 报道，NGINX(发音为“engine-x”)于 2004 年作为 Apache web 服务器和其他服务器的低延迟、高吞吐量替代物作为开源软件发布，现在已被超过 3.74 亿个网站使用，其中包括世界上百万个访问量最大的网站中的 15%。

这笔交易的另一个方面是 DevOps 的角度。由于 DevOps 的角色越来越多地涉及管理应用交付，这使得 F5 在来自开发人员背景的人员中占据了重要地位，因为开发人员更有可能积极看待具有开源传统的产品。
