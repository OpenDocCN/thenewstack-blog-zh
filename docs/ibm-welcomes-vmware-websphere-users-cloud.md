# IBM Cloud 轻松复制 VMware 工作负载和企业 Java

> 原文：<https://thenewstack.io/ibm-welcomes-vmware-websphere-users-cloud/>

通过与 VMware 的合作，IBM 让组织能够非常轻松地将其 VMware 工作负载迁移到 IBM 云。蓝色巨人还通过为其 Websphere enterprise Java 应用服务器提供云连接器，为其企业 Java 用户铺下了众所周知的欢迎地毯。

对于 IBM 在新兴的云市场获取更多份额的努力来说，这次合作是个好消息。VMware 表示，几乎所有的财富 500 强都以某种形式使用 VMware。IDC 估计，到 2017 年，80%的组织将运行混合云架构，其中计算资源在内部服务和云服务之间进行分配。

IBM 在本周于拉斯维加斯举行的 [InterConnect 2016](http://www.ibm.com/cloud-computing/us/en/interconnect/) 上宣布了这一合作关系。

这两家公司设计了一种架构，允许用户在 IBM cloud 上配置 VMware 环境。

IBM Cloud 复制了 VMware 的软件定义的数据中心架构，允许用户轻松地将其内部基于 VMware 的工作负载(包括其所有安全和网络设置)迁移到全球 45 个 IBM 数据中心，这些数据中心共同构成了 IBM Cloud。

在 IBM 方面，该公司的[云构建器](http://www-935.ibm.com/services/us/en/it-services/cloud-services/cloud-builder/)工作负载自动化功能可以基于 [VMware 的软件定义的数据中心](https://www.vmware.com/software-defined-datacenter/)模式自动配置预配置或定制的工作负载。

在 VMware 方面， [VMware 的 vRealize Automation](https://www.vmware.com/products/vrealize-automation) 和 [vCenter](https://www.vmware.com/products/vcenter-server) 管理工具将能够在 IBM 云中部署和管理环境，就像它们为客户自己的内部计算和网络资源所做的那样。

两家公司还将合作，共同开发和销售利用其组合技术的新软件包，包括那些有助于无缝工作负载迁移、灾难恢复、容量扩展和数据中心整合的技术。

IBM 和 VMware 还将联手为混合云服务提供支持。定价将基于灵活的按月消费模式。

## 企业 Java

对于那些希望将其现有的企业 Java 工作负载迁移到云中的企业，IBM 已经发布了 IBM WebSphere Cloud Connect。这个连接器为运行[IBM WebSphere](http://www.ibm.com/software/websphere)Java Enterprise Edition server 的 100，000 个企业用户提供了一种将服务器端 Java 应用程序连接到云的简单方法。

IBM 估计全球大约有 1300 万 Java 程序员。

除了与 VMware 的合作关系，IBM 还宣布，link shortening service Bitly 也正在将其 250 亿个链接以及相关的指标和元数据迁移到 IBM 云平台。平均而言，用户每月通过 Bitly APIs 创建 3 亿个链接。Bitly 希望 IBM 平台能让用户更快地加载链接。

IBM 和 VMware 是新体系的赞助商。
