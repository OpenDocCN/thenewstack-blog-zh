# 煎饼早餐播客:OpenStack 和 Kubernetes 现在是最好的朋友了，真的

> 原文：<https://thenewstack.io/pancake-breakfast-podcast-openstack-kubernetes-besties-now-really/>

周三上午在奥斯汀举行的 OpenStack 峰会上，300 多人一边吃着煎饼，一边听一个小组讨论工作负载协调的未来。

由英特尔数据中心集团副总裁 [Jonathan Donaldson](https://www.openstack.org/summit/austin-2016/summit-schedule/speakers/5875) 、OpenStack 分销商 [Platform9](https://platform9.com/) 、CoreOS 基础设施负责人[Brian ' red beard ' Harrington](https://www.openstack.org/summit/austin-2016/summit-schedule/speakers/2267)和 New Stack 执行主编 [Joab Jackson](https://twitter.com/Joab_Jackson) 组成的小组讨论了基础设施互操作性的重要性。TNS 创始人亚历克斯·威廉姆斯主持了这个小组。

![Jonathon](img/3ba1245922df0d24877374184b6545b3.png)

英特尔公司数据中心事业部副总裁兼软件定义基础设施总经理 Jonathan Donaldson

Donaldson 谈到 Kubernetes 是部署 OpenStack 等应用程序的一种很好的方式，并且容器的可移植性带来了巨大的运营好处，但他也扮演了外交官的角色，声称[英特尔](https://www-ssl.intel.com/content/www/us/en/cloud-computing/intel-cloud-based-solutions.html)拥有庞大的客户群，该公司会听取他们的请求。他们从客户那里看到了一些共同的主题，比如“我如何更高效地运行我的基础设施？”或者‘我如何提供这种服务？’他说:“英特尔在每种特定情况下都会选择最适合工作的工具，有时这种情况就是 Kubernetes。

哈灵顿插话说，他已经私下谈论过很多次 Kubernetes，但还没有听到任何人公开谈论它。

“作为一名技术专家和一名运营商，Kubernetes 是我的答案。这是我个人的看法。这是因为 Kubernetes 为您提供了一个统一的 API 来调度这些工作单元，而不管您在哪里运行集群。它为您的部署提供了高度的统一性。”

值得注意的是，哈灵顿的雇主 CoreOS 提供了一个商业支持版本的 Kubernetes，名为[constructive](https://thenewstack.io/coreos-debuts-tectonic-a-commercial-kubernetes-distro/)。

其他小组成员似乎不愿意做出如此大胆的声明，红胡子说他很乐意在技术层面上讨论他的立场，如果有人能向他证明他基于技术是错误的，他很乐意修改他的观点。但是，现在，Kubernetes 是答案。

杰克逊说，最终，市场会做出决定。“Kubernetes 与 OpenStack 在互操作性方面合作良好，这是一个好消息。但我认为，最终市场将确定两三种常见配置，我们不知道会是什么。”

在小组讨论结束后，来自思科的观众成员 [Ken Wronkiewicz](https://github.com/wirehead) 回应了 Harrington 对 Kubernetes 的喜爱，他说，Kubernetes 提供了最小的启动集，非常具有可扩展性，并允许根据贵公司的需求添加更多功能。

Wronkiewicz 说:“我有很长的创造自愈事物的历史，”Kubernetes 使这变得容易。Kubernetes 中的高级抽象可以创建保护级别。如果一台服务器停止运行，功能将会切换。Wronkiewicz 解释说，自我修复系统需要前期的努力，但最终，“会有一个改进的速度。”

市场在决定。

思科、CoreOS 和英特尔是新堆栈的赞助商。
