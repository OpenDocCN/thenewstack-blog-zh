# 用容器和 AIOps 实现 DevOps 梦想

> 原文：<https://thenewstack.io/realizing-devops-dreams-with-containers-and-aiops/>

[#175:用集装箱和 AIOps 实现 DevOps 梦想](https://thenewstack.simplecast.com/episodes/175-realizing-devops-dreams-with-containers-and-aiops)

在这一集的[新堆栈分析师](https://thenewstack.io/podcasts/analysts)播客中，TNS 创始人亚历克斯·威廉姆斯与[Janakiram Associates 的首席分析师兼新堆栈的定期撰稿人](https://thenewstack.io/author/janakiram/)以及 [Harness.io](https://harness.io/) 的营销副总裁[史蒂夫·伯顿](https://www.linkedin.com/in/burtonsays/)一起，不仅讨论了容器和 Kubernetes 对实现我们的 DevOps 梦想的影响，还讨论了机器学习如何随着 AIOps 的发展而将它带到下一个层次。

“在过去的五年里，DevOps 实际上已经成熟了。所以，我们从虚拟机开始，DevOps 完全是关于配置和配置管理，然后，最终 CI/CD 进入，Jenkins 成为构建管理和发布管理的前沿和中心，但当容器成为主流时，整个游戏被带到了一个新的水平，”Janakiram 说。“我们进化了。基本上，当前阶段主要是由像 Kubernetes 这样的容器编排管理器驱动的，这使得搭建一个登台环境或测试环境变得非常容易。然后，我们将 Docker 映像作为部署单位。这从根本上改变了游戏规则。”

虽然容器可能会使一切更容易自动化，部署更快，从而有助于缩短开发周期，但 Burton 希望机器学习能够真正提高效率。

“如果您正在进行连续交付，您希望您的管道在几分钟内运行。那么，你是怎么做到的呢？机器学习是一种可以精简部署管道的方式。你可以实现端到端的自动化，”伯顿说。“从我的角度来看，这是关于压缩交付管道和使用机器学习来自动化更多开发人员、开发人员和 sre 通常自己做的手动任务。”

这就是 AIOps 发挥作用的地方。AIOps 是机器学习在传统 DevOps 管道中的应用，其中更多的过程可以使用机器学习技术自动化。例如，除了自动化应用部署之外，AIOps 还可以在发布进入生产之前对其进行分析，并帮助确定构建是否会成功，或者特定的微服务是否会增加延迟或按预期工作。

伯顿认为机器学习不仅仅是提高速度的关键，而是提供必要的洞察力，否则可能无法及时交付。

“DevOps 不是关于你一周能发布多少版本。我认为这里有一点误解，”伯顿说。“日志文件不会告诉您性能提高对收入的影响，因为开发人员获取这些信息需要几个小时。他们需要在几分钟内知道的是“我的应用程序性能如何？”机器学习可以实现即时反馈循环。我们谈论的是数十亿的指标，数十亿的事件。即使是这个星球上最聪明的 DevOps 工程师也没有足够的时间将它们整合在一起。机器学习可以很快给你这些见解。"
