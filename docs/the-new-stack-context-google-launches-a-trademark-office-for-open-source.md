# 新的堆栈环境:谷歌为开源启动了一个商标局

> 原文：<https://thenewstack.io/the-new-stack-context-google-launches-a-trademark-office-for-open-source/>

欢迎来到[新的堆栈环境](/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了[的克里斯·迪博纳](https://sites.google.com/a/dibona.com/www/)，他是[谷歌](https://www.google.com)的开源主管，关于[谷歌推出](/google-launches-organization-to-protect-trademarks-for-istio-angular-and-other-open-source-projects/)[开放使用共享](https://openusage.org/?utm_source=thenewstack&utm_medium=website&utm_campaign=platform)的，这是一家帮助开源项目更好地管理其商标的独立公司。

在一篇博客文章中，DiBona 指出商标处于法治和开源哲学的结合点。因此，在这一集里，我们想更多地了解他们是如何互动的，以及谷歌是如何试图以开源的方式改善商标管理的。我们还想澄清这样一个谣言，即这个组织是为了管理谷歌的 Istio 开源服务网格[而不是云本地计算基金会](https://www.protocol.com/google-cloud-kurian-istio-foundation)(DiBona 的回答是:不)。

TNS 编辑和营销总监 [Libby Clark](/author/libby/) 主持这一集，旁边还有 TNS 高级编辑 [Richard MacManus](/author/richard/) 和 TNS 执行主编 [Joab Jackson](/author/joab/) 。

[第 125 集:克里斯·迪博纳——谷歌推出开源商标局](https://thenewstack.simplecast.com/episodes/episode-125-chris-dibona-google-launches-a-trademark-office-for-open-source)

谷歌承诺，开放使用共享资源将“将开源的理念和定义扩展到项目商标。”该公司希望，创建这些商标的中立、独立所有权将让贡献者/消费者“安心”，他们的项目名称正以安全和透明的方式使用。最初，谷歌将支持 Istio、Angular 和 Gerrit 基于网络的代码协作工具。

在我们的采访中，迪博纳澄清了开放使用共享的范围:

我想我应该说一下我们不会提供什么。开放使用共享的要点是解决围绕在开源项目中使用商标的问题，并确保它们被适当地共享，并与开源定义保持一致。我们不从事基金会业务。我们不是做营销的。我们不是做会议生意的。我们不是这些项目的管理或指导委员会。我们的目标不是成为 Linux 基金会、Apache 软件基金会或软件自由保护协会。
