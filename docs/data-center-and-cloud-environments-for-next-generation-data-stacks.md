# 面向下一代数据堆栈的数据中心和云环境

> 原文：<https://thenewstack.io/data-center-and-cloud-environments-for-next-generation-data-stacks/>

[AppDynamics](https://www.appdynamics.com/) 赞助本帖。

随着新工具和平台的出现以满足当今的部署需求，数据中心和云基础设施也需要发展。

在本期[的新堆栈制造商](/podcasts/makers)播客中，思科 AppDynamics 产品营销高级经理 Shrey Parekh 和 AppDynamics 产品营销总监 [Whitney Satin](https://www.linkedin.com/in/whitney-satin/) 讨论了数据中心和云环境如何改变以适应下一代应用和数据堆栈。《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了这一集。

[面向下一代数据堆栈的数据中心和云环境](https://thenewstack.simplecast.com/episodes/data-center-and-cloud-environments-for-next-generation-data-stacks)

Satin 解释说，组织正在看到“云边缘计算等技术对数据中心的巨大破坏”，以及它们对基础设施的影响，如电力和冷却，以及其他外形因素的影响。“实际上，考虑到这种程度的转变，所有关于基础架构变得更容易访问、更现代、更不单一，以支持一些新的应用技术，这是非常了不起的。”

随着 DevOps 团队在多云环境中扩展新技术(如边缘部署),复杂性也随之增加。Kubernetes 环境公司也将继续在满足这些新兴的大规模需求方面发挥重要作用。

“在思科，Kubernetes 真正让我们兴奋的是能够部署我们所有的产品，并且能够在我们所有的产品投入实际部署之前轻松、敏捷地测试它们，然后能够根据需要迭代和修改我们的配置，”Parekh 说。“Kubernetes 和一般的容器使我们能够无缝地做到这一点，然后[我们可以]在思科内部的开发团队之间分享发现。”

当使用 AppDynamics 和思科提供的产品[app dynamics Kubernetes Cluster Agent](https://www.appdynamics.com/blog/product/introducing-appdynamics-cluster-agent-effectively-scale-monitor-kubernetes-confidence/)为客户部署产品时，它满足了“我们的客户查看其容器所需的安全性和运营需求……下至容器、节点、集群和名称空间”, Parekh 描述道。

Satin 解释说，为客户提供下一代云和数据中心部署以及技术基础设施需求的“主要可见性”也将仍然至关重要。

“无论应用程序位于何处，我们都努力提供可见性。因此，Kubernetes 集群代理使我们能够满足一些处于领先地位的客户，并真正确保我们为他们提供对代码的深入了解，以及了解应用程序和基础架构如何相互协作以提供客户体验，”Satin 说。

对于边缘应用和环境，客户“正在以一种高效且最符合其需求的方式使用云原生”，Parekh 说。“因此，对于那些希望采用‘边缘’技术的‘前沿’客户来说，‘边缘’是一种自然的发展。”"
