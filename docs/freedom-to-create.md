对你来说，什么更重要:你用来创建云应用的工具，还是围绕创建它们的自由？如果你像我们交谈的开发人员一样，工具来来去去，但是自由是一种持久的品质，它激励你做任何事情。

让计算机做你想让它做的事情的创造性过程——以及看到最终结果实际工作——都是至关重要的。当你可以花更多的时间编码，而不是坐在会议上，维护，或其他世俗的任务，工作更有趣。很过瘾。开会，没那么多。

如果你已经在云中工作了一段时间，你会比过去完成更多。一个你以前可能花了几天的任务现在变成了一个快速的 API 调用。开发周期正在加快。这么多都是自动化的，你过去花在常规任务上的时间被释放出来，让你有时间花在有趣的事情上。

随着你做得越来越多，你的角色也在演变。这不仅仅是因为你是第一个看到新技术或使用新技术的人；您的建议和代码现在推动了总收入。你不再是一个成本中心；你在重新定义你的事业。

## 不是轮子上的齿轮

有些公司明白这一点。其他人没有。认可你的专业知识和你所提供的价值的经理们可能会完全接受数字化转型。然而，我们经常听到开发人员说他们觉得自己不受重视。他们觉得他们的经理把他们当成齿轮，或者装配线上的工人。他们要求你做更多，交付更多，做得更快。

如果你不被重视，坚持住。好消息是，因为你在第一线，你能够研究和使用新技术。这种修补导致学习。学习会带来好的想法和出色的工作。

企业 IT 瞬息万变，加上工作期限的压力，有时很难跟上变化。正如我们在[“识别开发人员缺口”](https://www.cloudfoundry.org/developer-gap-2016/)报告中发现的那样，公司越来越多地选择培训，而不是外包或雇佣，以满足对更多开发人员不断增长的需求。这就是为什么我们最近推出了[云铸造认证开发人员认证和培训计划](https://www.cloudfoundry.org/training/)。培训和认证是切实可行的解决方案，对您和您的组织都有好处，让您蓬勃发展。

我的希望是，随着公司雇佣更多成千上万的开发人员，他们不会认为你是一个轮子上的齿轮，而是承认你在引领潮流，让你展示更多的才华。

## 自由与约束

这种创造的自由意味着让你专注于解决有趣的问题，同时避免妨碍你的过程。一些最好的工具实际上对你所做的事情施加了约束——但有点讽刺的是，这些约束在简化不太有趣的必要任务时，带来了更多的创造力(甚至是自由)。平台是一个约束的完美例子，它加速了整个过程，但最终给了你创造的自由。工具只是没有自由的苦工。
