我们都听说过数字原生代:那些出生于 20 世纪 90 年代或更晚的人不记得在互联网、流媒体音乐和电影、网上购物以及通常在他们的指尖上拥有世界上所有信息之前的时间。

我会尽量避免和自己约会，但我不是真正的数字土著。我记得报纸投递和电视新闻，当 AOL 第一次告诉我有邮件时，我就在那里。我的第一个网页是在 [JavaScript](https://www.javascript.com/) 和 [CSS](https://www.w3.org/Style/CSS/Overview.en.html) 出现之前建立的——比 Space Jam 网站有更多的滚动字幕和闪烁的 gif(取决于你什么时候长大，你可能知道我在说什么)。

然而，我直到 2018 年才真正学会编码，当我在代码学校尝试容器时，我的第一个应用程序是使用[无服务器](https://martinfowler.com/articles/serverless.html)架构启动的。

我很幸运，因为我的第一个真正的开发人员角色是在一家建立并采用无服务器架构的公司。无服务器是我现在知道的唯一构建后端的方法。

换句话说:我是一个没有服务器的本地人。但这意味着什么呢？

## 天生高效

无服务器的本地用户永远不必学习构建自己的硬件，为其提供 IP 地址，并且因为办公室停电而导致整个服务中断。无服务器本地用户在选择云资源时不需要学习预测容量，也不会因为过载而导致网站崩溃。

无服务器的本地人把他们本该花在学习虚拟机或容器上的时间用来学习编写更好的代码。新开发人员可能是从无服务器原生中看到最大好处的人:他们可以构建他们的前端，并使用云服务的组合来创建他们的后端，从存储到数据库到用户认证。

哦，没有服务器的本地用户不会为不必要的计算资源付费。

## 函数构成了完整的堆栈

无服务器本机编写无状态函数，通过 API 与应用程序或服务的其他组件进行通信。

传统的开发基于编写客户端或服务器端代码。无服务器原生开发人员仍然可以编写客户端代码，如网站的前端或移动应用程序界面，但该架构是事件驱动的，而不是让后端代码无限期地驻留在服务器上。

这意味着今天，一个无服务器的本地前端开发人员只需要能够编写 Lambda 函数，就可以成为一个全栈开发人员。

## 企业无服务器

作为无服务器的本机也可以扩展到企业。一个存在了几年或几十年的公司会有大量的技术债务。在考虑添加无服务器架构之前，它可能会考虑如何将传统的整体架构分解为微服务。

然而，一个无服务器的本地公司从一开始就以无服务器的方式构建它的应用程序。It 可能需要考虑使用哪些云提供商，或者他们的资源将如何通过 API 进行通信，但是组织可以节省其他人将他们的后端迁移到云上所花费的工程时间。

## DevOps，天生的

无服务器概念自 2012 年左右就已存在。自 2014 年末 AWS Lambda 推出以来，真正的无服务器计算已经可以实现。今天任何学习编码的人都可能是#无服务器本地人——这意味着从一开始就考虑云架构。
