# 随着德国地区的推出，AWS 努力消除隐私担忧

> 原文：<https://thenewstack.io/with-launch-of-germany-region-aws-strives-to-quell-privacy-concerns/>

亚马逊网络服务今天早上宣布，它已经推出了一个新的区域，这次是在德国，该公司努力向欧洲企业保证，它的服务是完全安全的，即使是美国政府的监视。

安迪·雅西在今天早上的一次网络直播中说，新的数据中心位于法兰克福，使用碳中和能源。该服务包括多个可用性区域。

虽然 AWS 已经在欧洲建立了一个数据中心，但在爱尔兰，扩展到一个新的地区有点冒险，因为欧洲人在美国国家安全局的间谍活动曝光后产生了不信任。间谍活动的细节曝光后，欧洲出现了强烈的反弹。一些分析师对美国云提供商的损失做出了宏伟的预测，因为欧洲企业将决定将业务转移到其他地方。

在通话中，Jassy 详细介绍了 AWS 的安全政策。“安全性是 AWS 的首要任务。如果我们认为我们需要做些什么来加强它，我们就会放弃一切，”他说。他说，策略从物理访问控制开始，包括允许用户围绕谁可以在哪些场景下访问哪些 AWS 功能来设置策略的控制。

他提到，AWS 工具有助于防止工作人员在 IT 部门不知情的情况下启动实例，因为管理员可以跟踪谁执行了哪些操作。但是，该工具仅在员工使用公司帐户登录时有用。用户可以轻松地用自己或公司的信用卡访问 AWS 资源。

但欧洲企业感兴趣的真正问题是 AWS 如何保护他们的数据免受美国政府机构的攻击。Jassy 说，企业可以使用的最好工具是加密他们的数据，并自己持有加密密钥。“如果你这样做，这不是问题，”他说。

时间会证明这是否足以让欧洲公司满意。企业可能会认为 AWS 有意或无意地保留了一些保存这些密钥的方法，以便美国政府的窥探可以访问他们的数据。

Jassy 进一步解释说，如果任何政府要求 AWS 提供客户数据，该公司不会回应，除非是具有法律约束力的请求。在收到这种请求的“奇怪情况”下，AWS 会确保请求不会超出范围或不准确，他说该公司“积极地”挑战超出范围的请求。如果请求成立，AWS 会在“法律允许的情况下”告诉客户，这样他们就有机会自己质疑请求。

然而，关于美国正在监视世界各地的个人和公司的披露引起轩然大波的一个原因是，这仅仅是一个披露。公司不知道这种做法。在某些情况下，这是因为在某些情况下，政府要求服务提供商不要告诉客户他们的数据移交。

尽管如此，Jassy 淡化关注。“讨论起来很有趣，但实际上它并没有影响到客户。你可以问我们的顾客。这不是我们平台上的人必须处理的事情，”他说。

Jassy 列举了一些潜在敏感行业的大型欧洲公司，如保险、铁路票务、银行和电信，它们已经在使用 AWS。
