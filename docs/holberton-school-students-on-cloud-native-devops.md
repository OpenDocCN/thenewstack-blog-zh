# 云本地 DevOps 上的霍尔伯顿学校学生

> 原文：<https://thenewstack.io/holberton-school-students-on-cloud-native-devops/>

[TNS 背景:云上霍尔顿学校学生本土 DevOps](https://thenewstack.simplecast.com/episodes/tns-context-holberton-school-students-on-cloud-native-devops)

欢迎来到 New Stack Context，这是可伸缩 IT 开发和部署领域最热门新闻的每周总结。本周我们将与 [Essence Boayue](https://twitter.com/girlsaregeeks2) ，一位正在[霍尔伯顿学校](https://www.holbertonschool.com)接受软件工程培训的全栈工程师，以及[马克斯·约翰逊](https://twitter.com/Pypestream/status/1064677962019041281)，一位前霍尔伯顿学生，现在是对话式人工智能提供商 [Pypestream](https://www.pypestream.com/) 的 DevOps 工程师。

Boayue 和 Johnson 在一组投稿文章中研究了“云原生 DevOps”对于新堆栈的意义。Boayue 的帖子[讨论了云原生 DevOps 工程师的角色和职责](/roles-and-responsibilities-of-cloud-native-devops-engineers)。下周五请回来阅读 Johnson 的帖子，他将于下周五发布，内容是关于如何以全新的方式管理为在云中运行和扩展而构建的应用程序。

我们与 Boayue 和 Johnson 讨论了他们自己的开发运维之旅，什么是云原生开发运维，以及开发运维在构建和维护云原生应用中的一些角色和职责。我们还讨论了霍尔伯顿培训下一代全栈工程师的方法。

在节目的后半部分，我们将讨论 Pivotal DevOps 工程师 Paul Czarkowski 的 [2019 预测](https://tech.paulcz.net/blog/future-of-kubernetes-is-virtual-machines/)，即虚拟机——而不是容器——是 Kubernetes 的未来，这一想法已经在 Twitter 上获得了前 Docker 核心维护者 Jessie Frazelle 等的支持。我们将回顾本周的其他头条新闻。

TNS 编辑部主任利比·克拉克和 TNS 总编辑 T21 一起主持这个节目。

## 故事

*   [云原生 DevOps 工程师的角色和职责](/roles-and-responsibilities-of-cloud-native-devops-engineers):最优秀的 DevOps 工程师将有能力使用或学习各种开源技术，并熟悉大量用于脚本的编程语言。点击此处了解更多信息。
*   [读心术，自我复制:回顾年度最佳人工智能故事](https://thenewstack.io/mind-reading-self-replicating-a-look-back-at-the-years-best-ai-stories/):对于人工智能领域来说，这是不平凡的一年。我们的人工智能科学记者 Kimberly Mok 列举了 2018 年最显着的几项进展，包括解码脑电波以试图读取人的想法的算法，可以预测未来行动的机器，使用“视觉预见”，以及模仿社交技能的数学发展，或者机器之间的合作，就像人们在一个小组项目中一样。
*   [科技伦理新年决心:不要开发你会后悔的软件](https://thenewstack.io/tech-ethics-new-years-resolution-dont-build-software-you-will-regret/):那些想把新年作为一个机会，把他们的个人伦理与他们谋生的工作联系起来的开发人员可能想读读 Jennifer Riggins 关于伦理是软件开发的帖子。道德软件的理念是不会造成负面的社会影响，也不会让这个世界变得更糟糕。
*   [本周编程:Kubernetes 在虚拟机中的未来](https://thenewstack.io/this-week-in-programming-kubernetes-future-in-virtual-machines/) : Paul Czarkowski 是 Docker 早期的容器爱好者，同时也是 Pivotal 的员工，他在本周写了一篇文章，这篇文章被广泛传播，他大胆地宣称 Kubernetes 的未来是虚拟机——人们对此并没有哄笑，而是表示赞同，甚至是强烈赞同。
