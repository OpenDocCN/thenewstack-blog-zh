# Datadog 与前端 Web Tester Madumbo 收购实现全栈

> 原文：<https://thenewstack.io/datadog-goes-full-stack-with-front-end-web-tester-madumbo-acquisition/>

基础设施和应用监控服务 [Datadog](https://www.datadoghq.com/) 收购了人工智能网站测试工具 Madumbo，Madumbo 首席执行官 [Gabriel-James Safar](https://fr.linkedin.com/in/gabriel-james-safar-44789260) 解释说，这是扩展 Datadog 全栈监控能力的第一步。

“我们可以关联到应用程序性能管理，如果后端请求花费太长时间，Datadog 可以给出跟踪信息，说明它为什么不工作，显示它是来自 UI 更改、后端问题还是停机，”Safar 说。“Madumbo 的发现可以显示在 Datadog 的仪表板上，从而完善了 Datadogs 的产品套件。现在有了前端可视性，它允许 Datadog 变得更加全栈。”

Datadog 对 Madumbo 的收购将是它的第三次收购，最后一次[将在 2017 年](/datadog-brings-data-visibility-log-management-acquisition/)到来，当时该公司收购了另一家法国公司 Logimatic.io。当时，我们写道，这次收购完善了“其产品组合，该公司宣传其在单一平台内提供基础设施指标、应用性能监控(APM)和日志管理的能力”，现在这种能力将移动到包括 Madumbo 的前端。

作为一个独立的产品，Madumbo 为开发人员和项目经理提供了执行一些关键网站测试功能的能力。首先，该产品的“漫游器”自动识别页面上的元素，并与所有元素进行交互，使用算法触发网站的所有可能状态，而不会陷入无限循环，以确保所有基于 JavaScript 的前端功能都按预期运行。

其次，Madumbo 的用户能够记录他们通过网站功能的路径，并自动再现该路径，即使元素被移动或改变。这些产品会显示错误代码、网络错误和潜在问题。

“Madumbo 在其测试产品中发现的错误对于排除 web 应用程序的故障非常重要，但这些错误通常源于后端或基础设施中的底层问题，”Safar 补充道。“将我们的技术引入 Datadog 的平台，最终用户不仅可以通过测试识别这些错误，还可以通过 Datadog 的基础设施监控、日志管理和 APM 产品找到这些错误的根本原因。”

根据 Datadog 关于此次收购的声明，Madumbo 团队将加入 Datadog 巴黎研发办公室，并正在开发将于今年晚些时候发布的新产品。

“我们很高兴有 Madumbo 团队加入 Datadog，”Datadog 首席执行官 Olivier Pomel 在声明中说。“我们看到核心技术增强了我们的平台，并为我们的客户扩展了许多新的数字体验监控功能。”
