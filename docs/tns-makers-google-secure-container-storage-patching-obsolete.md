# TNS 制造商:安全容器存储和为什么补丁应该过时的谷歌

> 原文：<https://thenewstack.io/tns-makers-google-secure-container-storage-patching-obsolete/>

在本期 [TNS Makers](https://thenewstack.io/podcasts/makers/) 播客中，我们采访了谷歌的产品经理 [Kit Merker](https://twitter.com/KitMerker) 以了解更多关于谷歌的[容器注册表](https://cloud.google.com/container-registry/)的信息，该注册表于去年 8 月正式发布。采访发生在 11 月在巴塞罗那举行的欧盟会议上。Merker 还流利地讲述了云存储，为什么今天的修补流程已经过时，以及持续集成和交付的障碍，请听这里:

[Google 的 Kit Merker 在 DockerconEU](https://thenewstack.simplecast.com/episodes/googles-kit-merker-talks-container-registries-at-dockerconeu) 谈容器注册

## 突出

谷歌注册服务上的 Merker*:*

“它真的是为从事私人注册业务的人准备的。我们努力成为最快、最具可扩展性的产品。我们最近进行了一系列性能改进，从上周开始，下载速度提高了 40%。因此，我们已经很快的服务甚至更快…我们加密所有静态图像，所以我们有一个非常安全的解决方案。它还非常具有成本效益，因为我们对存储收费，而不是对私人注册表的每个图像收费。”

*对虚拟机打补丁*:

“我认为现在容器的一个主要难点是安全向量。实际上，我认为打补丁是一种我们需要摆脱的过时的东西；我们需要摆脱它。对我来说，打补丁是一种职业，是颠覆性的，这么说吧。我与客户的许多 IT 团队交谈过，他们似乎都有一个孤独的人维护虚拟机的黄金映像，必须使它们保持最新，必须确保每个人都使用正确的版本，这是一个巨大的时间消耗。”

*关于正确使用 devo PS*:

“上周末，我在一个小型创业活动上发表了演讲。我不得不问…'你如何知道你的软件开发团队是否健康？'我说，我寻找的一件事是，如果你的团队在随叫随到的情况下一次有超过一个人，你就是不健康的。你需要像这样，想一想你将如何减少你的随叫随到，并得到控制，然后有大量的实践可以做到这一点。进入这种持续交付和 DevOps 的心态不仅仅是…工具驱动的，它还能消除你的恐惧，并使用工具来实现它，这样你就不会逞英雄了…”
