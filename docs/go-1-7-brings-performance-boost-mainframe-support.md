# Go 1.7 带来了性能提升和大型机支持

> 原文：<https://thenewstack.io/go-1-7-brings-performance-boost-mainframe-support/>

自 Go 1.6 发布六个月后的周一，谷歌发布了 Go 编程语言的第八个版本，1.7 版。

Golang 的这个版本带来了一些重大的变化，比如更新了编译器，增加了“上下文”包，以及分层测试。谷歌也将这种语言移植到了 [IBM LinuxOne](http://www-03.ibm.com/systems/linuxone/) 大型机上。

IBM z 系统上的 Linux 端口是 1.7 中的一个实验性版本，伴随着早期的 Plan 9 端口。

虽然 Go 的编译器众所周知很慢，还需要进一步的改进，但是这个版本的 Go 开始对该语言的编译器进行重大改进。

根据 Google 的基准测试，新的编译器将性能提高了 35%。据报道，与 1.6 版本相比，二进制文件的大小减少了 20-30%，编译速度显著提高。

编译器的后端已经过重新设计，以静态单赋值形式表示源代码，每个变量只赋值一次。目前只有 64 位 x86 平台支持静态单赋值。然而，编译器前端现在可以更有效地处理导入声明，并且似乎可以跨所有平台进行更新。

在被证明对涉及网络和微服务的应用至关重要后，**golang.org/x/net/**上下文现在已经作为包“上下文”成为 Go 标准库的一部分

核心库中的其他包现在可以使用“上下文”来允许超时、取消和传递请求范围的数据。Package context 向 Go 标准库的转移是为了鼓励它的进一步使用。

子测试和子基准现在都支持分层测试和表驱动基准。这些功能在“测试”包中启用。

在 Go 1 的兼容性承诺下，几乎所有的 Go 程序仍然可以像以前一样编译和运行。go 1.7[可在](https://golang.org/dl/)下载，并欢迎新的贡献者。

上面提到的功能和改进只是这个版本的 Go 所做的许多改变中的一部分。要查看完整列表，请点击这里[。](https://golang.org/doc/go1.7)

IBM 是新堆栈的赞助商。
