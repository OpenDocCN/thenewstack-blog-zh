# 团队无服务器

> 原文：<https://thenewstack.io/serverless-for-teams/>

Stackery 赞助了这个播客。

[无服务器组队](https://thenewstack.simplecast.com/episodes/serverless-for-teams)

现收现付的定价模式和速度的提高是无服务器技术对应用交付的两个众所周知的影响。同样重要但经常被忽视的是，无服务器如何影响开发人员的工作流程和团队动态。开发人员不再完全关注业务逻辑和代码，而是承担更多配置云资源的责任。

“[无服务器]重新分配了运营工作的责任，在大计划中，它实际上执行了 DevOps 模式，”[Stackery](https://www.stackery.io/)的首席执行官[内特·塔加特](https://www.linkedin.com/in/nathantaggart)说。“无服务器从根本上说是 DevOps。开发人员必须在与他们的开发工作相同的周期内迭代操作工作。”

塔加特说，无服务器的采用不是从首席技术官开始的。相反，组织通常采用无服务器，因为他们的工程师已经开始自己使用 AWS Lambda，以努力更快地发布功能。治理的缺失会给组织带来弹性和伸缩问题，因为单个开发人员会将他们自己的工作流带入团队动态中。运营团队很快失去了对整个应用程序的可见性和控制，回滚和故障排除成为一个问题。

“实际上，两个工程师进行无服务器开发比一个工程师更难。而且[问题]越来越严重，”塔加特说。

为了保持效率和速度，对一个组织来说，为无服务器架构设置标准实践变得很重要。这确保了单个工作流可以在整个组织中扩展，并在应用交付中提供可见性和一致性。例如，团队可以采用剧本来了解如何回滚变更或者知道谁提交了一个提交。

在本播客中，了解无服务器架构如何影响开发人员工作流和团队动态，无服务器如何实现 DevOps ideal，以及团队可以开始使用 Stackery 的无服务器工具包标准化无服务器应用程序开发实践的一些方法。
