# 英特尔的 Das Kamhout: Kubernetes 和 Mesos 可以在数据中心协同工作

> 原文：<https://thenewstack.io/intel-engineer-das-kamhout-kubernetes-lock-developers/>

英特尔解决方案工程总监 Das Kamhout 表示:“我们看到，Kubernetes 作为一个面向容器的应用编排层，正在获得巨大的吸引力。“现在任何人都在构建应用程序——也许他们正在使用 Docker 作为他们的容器运行时，但当你看到他们是如何编排的时，发现 Kubernetes API 在适当的位置是很常见的。”

Kamhout 在上个月的英特尔云日活动中与新堆栈的 Alex Williams 进行了广泛的采访，并在[新堆栈制造商](/tag/the-new-stack-makers/)播客中捕捉到。

Kamhout 的观察有些令人惊讶，因为英特尔继续寻求与 Mesosphere 的合作安排，后者为[提供 Apache Mesos](https://thenewstack.io/mesophere-marathon-1-0-newly-released-supports-stateful-containers/) 的商业版，并生产基于 Mesos 的 DCOS(数据中心操作系统)。有些人可能会将 Mesos 视为 Kubernetes 的竞争对手，但 Kamhout 并不认为这是必然的。

Kamhout 告诉 Williams，Mesos 和建立在 Mesos 上的平台显然是为了组织大型部署。这证实了我们几周前在圣克拉拉的开放网络峰会上听到的对话。但这位工程师说，客户亲自告诉他，Kubernetes 正在为他们的开发者取得突破。

威廉姆斯问卡姆豪特，英特尔现在是否会投资 Kubernetes。

“我们在投资方式上相当宽泛，”他回应道。“我们仍然投资于 Mesos，[*它是一个非常强大的资源分配器。”他点名表扬了 Mesos 的创建者和 Mesosphere 的联合创始人 Ben Hindman，称赞他为大规模部署“创造了一个非常惊人的东西”。*

“但是，我们也看到了梅索斯顶上的库贝内特斯，”他继续说道。“Kubernetes 更多是直接为应用程序开发人员构建的，而 Mesos 则是为超大规模集群资源管理构建的。”他确实预见到了 Kubernetes 在大型数据中心资源管理和协调方面进入 Mesos 领域的可能性，但目前这两者之间存在明显的区别。

Kamhout 还与 Williams 和记者 Scott Fulton 详细讨论了开源软件组件领域碎片化的危险，以及英特尔正在努力帮助 Kubernetes 和其他 orchestrators 在其最新的至强服务器处理器中使用新的资源管理器。这是英特尔的最新技术，用于减少运行在同一处理器上的虚拟机和容器之间的干扰——特别是当它们是同一主设备的克隆时。

英特尔和 Mesosphere 是新堆栈的赞助商。
