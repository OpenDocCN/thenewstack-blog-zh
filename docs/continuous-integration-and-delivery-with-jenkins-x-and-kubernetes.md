# 与 Jenkins X 和 Kubernetes 的持续集成和交付

> 原文：<https://thenewstack.io/continuous-integration-and-delivery-with-jenkins-x-and-kubernetes/>

本播客由 [CloudBees](https://www.cloudbees.com/) 赞助。

[CI/CD 与詹金斯 X 和库伯内特](https://thenewstack.simplecast.com/episodes/ci-cd-with-jenkins-x-and-kubernetes)

当技术行业支持 Kubernetes 时，应用程序生命周期管理可能会变得容易得多。如今，只有持续集成和持续交付(CI/CD)的 CI 部分是自动化的。CD 由脚本、管道、元数据和配置手工组装而成。但是 Kubernetes 实现了 CD 自动化，像 Jenkins X 这样的工具使得在 Kubernetes 上部署变得简单。

“CI/CD 应该越来越像一个只为您提供 CI 和 CD 的设备。CloudBees 的首席架构师詹姆斯·斯特拉坎(James Strachan)说。“我们试图与 Jenkins X 合作的地方不是隐藏 Kubernetes，也不是隐藏 CI、CD 和管道，而是实现自动化。”

在本播客中，[CloudBees 联合创始人兼工程经理 Michael Neale](https://www.linkedin.com/in/michaelneale/) 和 Strachan 讨论了 Kubernetes 如何改变 CI/CD、发展 GitOps 等工作流，以及 cloud bees 如何努力改善开发人员在 Kubernetes 上部署应用程序的体验。
