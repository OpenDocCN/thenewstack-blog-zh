# 理查德·斯托尔曼评估了 34 年来自由软件的斗争

> 原文：<https://thenewstack.io/richard-stallman-assesses-34-year-fight-free-software/>

我记得在 90 年代中期看过理查德·斯托尔曼在网景公司的演讲。作为自由软件基金会的创始人(同时也是 GNU 通用公共许可证的作者)，Stallman 激励观众为一个软件自由的世界而战，甚至以唱他的非官方颂歌结束。

因此，很高兴看到这位 64 岁的倡导者仍在为正义而战，[本月接受了布莱恩·伦杜克(Bryan Lunduke)的罕见视频采访](https://www.youtube.com/watch?v=S0y0oXU8YNk)(他是南加州 Linux 博览会上一位熟悉的开源话题主持人，现在[在 YouTube 上有一个“每日计算/书呆子秀”](http://lunduke.com/))。伦杜克还提倡开源软件，并完全用免费软件录制了采访——因为“理查德对 YouTube 有非常强烈的看法”,所以确保一份拷贝也出现在 archive.org 上——无数字版权管理，并有多种格式。

https://www.youtube.com/watch?v = s0y 0 0x u8 ynk

当伦杜克开始他对时事的业余采访时，烟火几乎立即开始了——特别是蒂姆·伯纳斯·李爵士(以及万维网联盟的首席执行官杰夫·贾菲)支持在即将到来的 HTML 标准中包括数字版权管理(DRM)保护的方式。“你对此有什么想法吗？”伦杜克害羞地问。

斯托曼立即作出回应，描述“一片受到强大帝国威胁的土地。”他的音频中断了，回来时他说“减轻随后的占领和征服的残酷…”

*“那些目光短浅的人会说，‘好吧，我们打不过他们，让我们加入他们吧，’所以他们会想办法签约成为征服者的仆人，他们告诉自己，他们会减轻征服者的压迫……征服者利用他们来完成他们无论如何都要完成的工作，如果那些人拒绝帮忙，事情会变得或多或少一样，除非会有更多的反对意见和更好的机会最终把他们赶出去……*

伦杜克很快发现自己成了斯托曼欢快地呼吁许可革命的陪衬者。一个没有 DRM 的世界真的可能吗？

“这基本上就是说，‘我们应该不战而屈人之兵吗？’"斯托曼回应道。"因为事实就是如此。如果人们告诉自己我们永远不会赢，那么他们就不会打，所以他们不会赢，这是肯定的。然而如果他们打起来，也许最终我们会赢。只需要足够的人去战斗……”

那么我们这些软件用户应该如何反击呢？“拒绝使用 DRM 的产品，”斯托曼笑着回答。“永远不要使用任何旨在限制你的产品，除非你手头立即有打破手铐的东西。”

很快，这位快乐的反向投资者引入了另一个话题。“当然，你无论如何都不应该使用网飞，因为它可以识别人们的身份并跟踪他们观看的内容，而且它强加了一个不公平的合同，你肯定永远不会同意。”

斯托曼对用户隐私有很多话要说，坚持认为“没有人使用脸书，脸书使用他们。脸书没有用户。它已经使用了。如果你有一个脸书账户，脸书正在利用你获取关于你和你认识的其他人的信息，这种危险已经到了他们可能能够控制选举或向他人出售数据来实现这一目的的地步。现在已经不仅仅是对被利用者的威胁了。所以你真的最好不要成为脸书的一个习惯……”

就伦杜克而言，他深思熟虑地利用长达一小时的采访，引出了一些关于这位活着的传奇人物的人性化细节。例如，作为一个坚持自由软件的人，理查德·斯托尔曼自己是如何获得他自己的数字娱乐的？“嗯，关键的一点是，没有一部电影、一场演出或一首歌曲值得我为之放弃自由，”斯托曼回答道。“所以我不要。

“我获得出版物副本的唯一途径是有一种合乎道德的方式，一种不会虐待我，不会对使用它的人不公正的方式。”第一个要避免的不公正是 DRM——但是，当然，理查德·斯托尔曼没有任何播放器可以播放受 DRM 保护的软件。不公正#2 呢？*最终用户许可协议……*斯托曼拒绝同意这些协议。

“永远？”去问伦杜克。

斯托曼坚持说:“T5 永远不会是 T6。”。“这意味着我无法通过互联网获得音乐，因为他们强加了 EULAs。我可以为音乐买一张 CD，我也这么做了，但是如果我不得不同意 EULA 的话，我就不能得到和文件一样的音乐。当您购买 CD 时，您不同意任何许可……”

"同样，如果他们有 EULAs 或 DRM，我也不会得到电子书."

不公正#3 呢？追踪。(通常见于在线出版物……)“我讨厌向不公正屈服。我读什么书，听什么音乐——这其实不是秘密，我担心如果有人知道会危及我，这不是重点……我想对抗监视！我不想屈服。如果我得到一份能追踪到我的出版物，我就放弃了。我会在我可以抵抗的地方投降。”

“所以我从实体店购买音乐光盘……问题出在美国。现在很难再找到这样的商店了。”

但在整个采访过程中，他也提到了他熟悉的主题，提醒伦杜克“专有软件是不公平的，因为它由一个实体控制，即所有者——通常是开发者，但不总是——它赋予所有者控制每个用户的权力。这是专有软件的根本不公平之处。这就是为什么我拒绝使用它。这就是为什么它不应该存在，它*必须*不存在。”

这就不可避免地引发了后续问题:“你觉得这场战斗已经进行到什么程度了？”

斯托曼兜售[尊重你的自由证书](http://www.fsf.org/resources/hw)，这是自由软件基金会对外围设备和计算机的高度认可。“所以你可以看到我们在向各个方向前进。然而，你看到所有这些移动设备都充满了需要系统级非自由软件的外围设备，所以除了大量艰苦的逆向工程，没有办法释放它们并让它们工作。进展缓慢。”

但是后来，Stallman 说他在 ThinkPad x60 上运行 Trisquel 的 GNU/Linux 发行版，“这是一种可以运行免费 BIOS 的电脑，在 BIOS 或 Linux 中没有二进制程序，在 GNU 系统的任何级别都没有专有软件。这基本上是我们 34 年前的目标。”

斯托曼似乎从未失去他的信仰——或者他的热情。“我们不断进步，但新的计算领域不断涌现，这些领域完全是奴役或征服的区域。你可以用自由软件做的事情越来越多——有些领域，例如图形设计和动画，现在你可以用自由软件做得相当好……但与此同时，大多数使用互联网的人正在使用基本上利用他们来获取个人信息的有害工具……”

但在新的转折中，斯托曼对联网设备也有一些严厉的措辞，他称之为“刺的互联网”

“就我个人而言，除了我的电脑之外，我不会容忍家里有任何东西在与互联网对话。”斯托曼在这些设备中看到了无数的问题，从监控、DRM 和后门，到审查和与远程服务器的连接。"如果现在任何专有程序都没有恶意功能，那基本上是运气."

不过话说回来，斯托曼把所有问题都追溯到了它们的原罪。“最终，专有软件总是不公平的，因为你不能自己改变它……有了自由软件，你可以删除任何恶意功能[或者]其他几个用户可以聚在一起做这件事并发布他们的修改版本，你只需要使用它。有了自由软件，用户群体可以保护自己免受恶意功能的攻击…

“这就是为什么[仅仅是专有软件的事实是不公平的](https://www.gnu.org/philosophy/free-software-even-more-important.en.html) …唯一已知的对恶意软件的防御是自由软件…对于专有软件，用户毫无防备。他们在老板的控制下无能为力。所以你应该引导专有软件的方式已经脱离了你的生活。”

伦杜克:最好是一双大铁头靴。

斯托曼:*【微笑】*耶！

但偶尔也能瞥见这个传奇背后的人。斯托曼开玩笑说，“我是不由自主地自我提升到管理层的，”伦杜克表示同情，说他开始想念编码了。“你有没有那种冲动，回到那里，只是坐在一个黑暗的房间里编码一个周末？”

“问题是，”斯托曼回应道，“现在有如此多的人致力于自由程序，如果我开始一个项目，当我完成它的时候，其他人可能已经写了一个了。”

所以他关注一些不太常见的东西。“我们需要更多的人谈论自由软件，坚定地拥护它，并且说‘我们必须继续战斗，因为投降导致失败。’"

伦杜克说，“如果你不投降，至少还有机会。”

斯托曼:“你刚才是在引用 T4 和惊爆银河系的话吗？

伦杜克:我是…

伦杜克觉得他必须转达一些观众对斯托曼所有灵感的热情感谢——斯托曼已经做好了回应。“感谢我和成千上万为 GNU 工作的人的最好方式就是帮助我们进步。所以看看[GNU.org/help](http://www.GNU.org/help)，你会看到几十种你可以做的不同类型的工作或者你可以做出的贡献——而且不全是编程……所以卷起袖子开始工作吧……！”

“有很多不同的方式来为这项事业做贡献。”

当伦杜克问了一个终极问题时，采访达到了高潮。“如果现在，就在今天，世界各国联合起来，通过一项一致通过的法律，宣布所有专利软件为非法，并使自由软件成为事实上的自由软件，那么在你余下的日子里，你会做些什么？”

斯托曼回避说，解决办法不是法律。专有软件就像一种危险的毒品，“我们必须教人们戒掉它。”但是如果他所有的愿望都实现了，理查德·斯托尔曼会怎么做呢？

“嗯，如果世界上还有其他的不公正，我相信我能够为结束这些不公正做出一些贡献。”
