# 微软描绘了它的云未来

> 原文：<https://thenewstack.io/microsoft-maps-cloud-future/>

掌握微软庞大的云平台可能会令人望而生畏。涵盖了从内部硬件到管理工具到开发平台到 Azure(以及其他)的一切，这是一套复杂的技术，具有非常不同的开发时间表。一些元素，如 Windows Server 及其系统中心管理工具变化缓慢，其他元素，如 Azure 每隔几周就会推出新功能。

持续发布和云技术的结合使我们很难掌握已经发布的内容，以及可能很快发布的内容。因此，对于微软来说，提供一个路线图，让 it 部门和开发人员了解已经推出的产品和当前正在开发的产品是有意义的。该公司的云和企业业务是其最大的业务之一，提供 Windows Server、Azure、Intune 等管理工具，以及 StorSimple 混合云存储设备等硬件。

随着云，更具体地说，Azure 成为微软当前战略的核心，公司计划指南是一个有用的工具。它可以帮助 It 经理掌握一套快速变化的工具，为他们自己的发展计划指明方向。

[得出的云平台路线图](http://www.microsoft.com/en-us/server-cloud/roadmap/public-preview.aspx?TabIndex=0)是微软在云上使用的工具和技术的有用指南。该路线图通过网络交付，采用网格结构，其中包含云基础设施、企业移动性、数据管理和分析、开发以及物联网等部分。然后可以过滤每个主题，以跟踪 Redmond 云和企业业务关键领域的最新发布、公开预览、开发中的产品和已取消的项目。

深入到该网站揭示了大量有用的功能，链接到进一步的细节，并在可能的情况下，到测试程序应用程序。你在这里找不到 Office 365 等服务的细节——它们有自己的路线图。它也不太可能列出微软正在做的一切，尤其是如果它正在被开发来与亚马逊网络服务和谷歌的公共云竞争。

即便如此，这里也有很多细节，例如，云基础设施部分显示了微软将其软件定义的数据中心雄心从内部基础设施扩展到 Azure。它目前正在开发一系列网络虚拟设备，旨在为虚拟基础架构添加急需的网络和安全功能。你还会发现隐藏在细节中的惊喜，微软详细介绍了在 Azure 的站点恢复业务连续性工具中扩展其对 VMware 虚拟机的支持的计划，让 VMware 虚拟机在 Azure 服务器上运行，这是雷蒙德重大变化的另一个迹象。

也许路线图中最有用的部分是公开预览列表。列出可能在未来几个月内发布的应用程序和服务，IT 部门将能够在发布之前使用它来确定要使用的原型和测试。

有了这样的路线图，你可以提前计划，而不是等待斯科特·格思里在博客上发布的消息。
