# VMware 的开源方法

> 原文：<https://thenewstack.io/vmwares-approach-to-open-source/>

[VMware](https://www.vmware.com/company.html) 赞助了这次播客。

组织对开源有许多共同的理解——但是这些理解也有很大的不同。尤其是在描述一个组织在开源开发中的角色时，以及利用开源工具和组件持续增长的最佳方式时。如果你愿意，可以称之为文艺复兴。

开源对一个组织意味着什么——以及它应该意味着什么——是 New Stack Makers 播客最新一集的主要话题，该播客是在圣地亚哥的[开源峰会](https://events.linuxfoundation.org/events/open-source-summit-north-america-2019/)期间录制的，预览了 New Stack 的第二次年度调查结果“[企业中的开源项目](https://www.surveymonkey.com/r/OSS_Prog_19_T)[VMware 副总裁兼首席开源官 Dirk Hohndel](https://www.linkedin.com/in/dirkhohndel) 讨论了他对这些结果的看法以及这些结果对 [VMware](https://www.vmware.com/opensource.html) 的意义，VMware 与[Linux Foundation 的](https://www.linuxfoundation.org/) [TODO Group](https://todogroup.org/) 共同发起了此次调查。

[您可能不知道开源对您的组织意味着什么](https://thenewstack.simplecast.com/episodes/what-you-may-not-know-about-what-open-source-means-for-your-organization)

调查结果也有助于量化 Hohndel 在开源社区工作期间监督的许多操作。鉴于企业越来越多地将自己描述为软件公司，一个关键的考虑是确定软件在您的组织中的特定角色。Hohndel 说，更具体地说，大多数企业将软件视为“他们工作的关键”

“在大多数情况下，是您的增强应用程序让您能够实现业务价值。在今天的所有这些应用程序中，我想说它们都有开源组件，它们依赖于开源基础设施，并且通常运行在开源之上，”Hohndel 说。“现在的问题是，在这些种类繁多的软件组件中，哪些部分对您的业务至关重要，哪些部分可以让您脱颖而出，您可以在哪里提供业务价值，哪些部分只是您的基础架构，或者只是“部署表面”，因为没有更好的术语。”

Hohndel 说，组织的差异化通常“更接近他们的内部应用，而不是基础设施”。“通常情况下，你参与任何特定的开源项目越多，你就越会说‘这是我的战略优势所在’，你在这上面花费的金钱或时间就越多。”

自三年前成立以来，VMware 开源计划办公室的一项任务就是帮助企业确定什么对他们具有战略意义，以及他们希望在软件方面投资的领域和方式。Hohndel 说:“我只是想在哪里使用一些别人可能是专家的东西……而且你经常会看到这突然改变了你对开源软件的看法，这取决于你谈论的是整个软件栈的哪一层。”
