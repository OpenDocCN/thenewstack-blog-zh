# Convox 如何利用开源社区驯服 AWS

> 原文：<https://thenewstack.io/convox-open-source-communities/>

将应用程序从白板上拿下来并投入使用应该很简单。即使有 Docker 和 Amazon Web Services 这样的平台可用，开发人员可能仍然会发现自己在努力让他们的应用程序运行起来。各种开源项目已经满足了满足这种需求的挑战，不仅为开发人员提供了容器服务，还提供了管理、维护和部署他们的应用程序的方法。

在这一集的[新 Stack Makers](https://thenewstack.io/podcasts/) 嵌入式下方，新 Stack 创始人 Alex Williams 在 [OSCON 2016](https://thenewstack.io/new-stack-analysts-diversifying-open-source-oscon/) 期间与 [Noah Zoschke](https://twitter.com/nzoschke) 、 [Convox](https://convox.com/) 创始人兼首席技术官坐在一起。Convox 提供了一个开源工具，用于在云基础设施上部署、管理和监控应用程序。请阅读或聆听，了解更多关于 Convox 如何利用开源来帮助开发其同名软件，它旨在帮助开发人员最大限度地利用 Amazon 或 Docker 设置的方式，以及开源的好处。

[用 Convox 构建更强大的开源社区](https://thenewstack.simplecast.com/episodes/building-stronger-open-source-communities-with-convox)

在康沃克斯之前，佐施克在赫罗库工作。在那里，他负责管理 AWS 基础设施。这次经历的结果是，他了解到尽管 AWS 非常强大，但它也“非常难以使用”因此，Convox 的诞生源于这样一个想法:将 Zoschke 在 Heroku 期间学到的关于 AWS 的所有知识打包，同时使 AWS 易于使用。

Convox 利用简单而强大的语言混合来帮助实现这些目标，特别是 Ruby 和 Go。Zoschke 认为，简化容器服务不仅有助于开发人员在其基础设施中支持 Docker 和 Amazon，而且有助于他们更有效地工作，而不是阅读 800 页的手册来设置特定的服务。

“我们不需要亚马逊所有的花里胡哨。他们需要一个负载均衡器、某种扩展，以及非常好的工作流来指导如何部署、运行迁移或扩展。我们项目的很大一部分是从桌面上去掉一些选项，并找到最小和最简单的方法从亚马逊获得这些选项。我们有一个 Go API 和一个基于 Go 的 CLI。这是一种自动化基于云 API 的事物的伟大语言。另一项技术是解开亚马逊本身的秘密。

Convox 的大部分都是使用开源构建的，鼓励社区做出贡献。虽然一些开源项目花时间让贡献者知道他们的建议是否在日程上，但这不是 Zoschke 和 Convox 团队的运作方式。

“Convox 拥有开源 API、开源客户端和开源 CLI。有一个很小的闭源网络界面。也就是说，大部分工作都是在开源平台上进行的。我非常坚信，任何人都应该能够提出一个想法，并尝试立即给他们反馈，看它是否在路线图中，”佐施克解释道。

当社区成员、贡献者和维护者转向新事物时，许多开源项目半途而废，Zoschke 的目标是长期支持 Convox。随着开源社区及其项目的持续增长，长期维护、社区增长和长期项目更新应该在“新项目”的影响消失后的时期加以考虑。

“设置一个集群来玩很容易，设置它并说‘嘿，我们将永远维护它’要困难得多。”
