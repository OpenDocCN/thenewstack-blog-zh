您想了解像富达这样的大型组织是如何大规模实施 DevSecOps 的吗？Capital One 如何使用 Jenkins 管道简化其软件交付？为什么一家全球软件和服务公司半开玩笑地说 DevOps 是“彻底的失败”？(提示:主要是因为组织经常低估采用 it 的挑战。 **)**

这些仅仅是在 [DevOps World 2020](https://www.cloudbees.com/devops-world) 的 125 场会议议程中涉及的几个主题，这是业界最大、影响最深远的致力于卓越软件交付的活动。

DevOps World 已经发展了多年，吸引了成千上万的与会者，并成为所有软件信息共享和教育的焦点。2020 年大会将于 9 月 22 日至 24 日以虚拟方式举行，让与会者能够在自己舒适的家中参加任何高质量的会议。

当会议首次广播时，与会者有机会与其他与会者以及(在许多情况下)发言者交谈。是的，如果有必要的话，你可以稍后观看，但是如果你这样做了，你就失去了与专家互动的机会。(换句话说，我们鼓励您观看会议的原始广播。)

会议主题从配置管理到包容性和多样性，从功能交付自动化到作为数字化转型战略的开发运维。演示将包括来自各种行业的用户故事，包括农业、矿业、旅游和酒店业。行业专业人士可以参加旨在娱乐的通用会议、重点技术会议以及介于两者之间的任何会议。

你想了解什么？这里有一个议程来刺激你的胃口。如果你不能决定参加哪些会议——让我们面对现实吧，很难缩小名单——这里有五个你不想错过的会议。

**在大型金融服务机构——富达投资公司 Nagaraju Kakarla 成功实施 DevSecOps**

富达投资是世界上最大的金融服务公司之一，管理着近 2.5 万亿美元的资产，员工超过 5 万人。当 it 需要扩展一项计划时，团队会面临可能会破坏整个地区流程的挑战。

印度富达企业计算高级总监兼分会负责人 Nagaraju Kakarla 将讲述他的团队在富达数字化转型项目中遵循的一些 DevSecOps 和站点可靠性工程(SRE)最佳实践。Kakarla 将描述 DevSecOps 在其他大型组织(包括 Mindtree、Intuit 和 Cisco)工作或担任顾问时遇到的挫折和挑战。

**管理 CI/CD 管道:最小化开发人员的工作，最大化企业合规性——Linda oyo Lu 和 Tahmina Khan，Capital One**

近年来，第一资本公司在 T5 前后被誉为 T4 更具前瞻性的金融服务公司之一。该公司已经在云上孤注一掷，采用敏捷方法和开源技术，并在全国电视广告活动中推广其技术。

Capital One 对詹金斯管道的实施改变了该公司。在这个面向 DevOps 从业者的演示中，Capital One 的 [Linda Oyolu](https://www.linkedin.com/in/lindaoyolu/) 和 [Tahmina Khan](https://www.linkedin.com/in/khantahmina/) 将描述如何在大型组织中实施托管 CI/CD 管道框架，从而在提高企业合规性的同时让软件工程团队的生活更加轻松。

**混沌工程混沌工程师——杰森·易，小妖精**

我们都听说过很多关于公司“吃自己的狗粮”的故事——本质上是拿走他们开发的技术或服务，并消费它们以确保他们自己的业务更有效地工作。(“喝自己的香槟”是一个更令人愉快的比喻，但出于某种原因，它并没有流行起来。)

Gremlin 有一个有趣的狗粮故事。该公司创建了一个混沌工程平台，客户用它来对自己的应用程序进行严格的测试。为了完善和改进自己的平台，Gremlin 使用其技术在它能想到的最混乱的环境中测试和重新测试其产品。

但是测试不仅仅是运行一个工具。成功取决于公司运行流程的能力 Gremlin 发现它在自己擅长的领域有所欠缺。宣传总监 Jason Yee 将会讲述 Gremlin 如何试验自己的混沌工程实践:公司尝试了什么，什么有效，什么无效。与会者将学习保持他们自己的混沌工程实践轻量级和快速的技术，因此他们可以在他们组织的特性工作和可靠性之间取得正确的平衡。

**Adobe GitOps—卡洛斯·桑切斯，Adobe**

召集一些开发人员在一个房间里——或者进行视频通话——你可能会听到有人热情地谈论他们组织中的 GitOps。 [GitOps](https://thenewstack.io/what-is-gitops-and-why-it-might-be-the-next-big-thing-for-devops/) 毕竟给了开发者控制权。通过坚持 Git 是真理的唯一来源的原则，GitOps 使开发人员能够执行通常属于 it 运营范围的任务。

在本次会议中，Adobe 高级云软件工程师 [Carlos Sanchez](https://www.linkedin.com/in/carlossg/) 将重点介绍他的公司如何开发 GitOps 流程；如何管理应用、Kubernetes 集群和 CDN。他还将讨论如何使用 Jenkins 和 Git repos 来存储代码和工件，跟踪版本历史和维护单一的真实来源。

**devo PS 彻底失败的五个原因——Savinder Puri，Zensar**

DevOps World 举办了超过 125 场会议，清楚地说明了 DevOps 为什么为当今的组织创造价值，以及它如何在未来改善他人的命运。它还包括一个环节，讽刺地指出，整个学科是对时间的巨大浪费。

[Zen sar Technologies 的 DevOps 传播者 Savinder Puri](https://www.linkedin.com/search/results/all/?keywords=Savinder%20Puri%20zensar&origin=GLOBAL_SEARCH_HEADER) 经常撰写和谈论组织在实施 DevOps 时遇到的[挑战——他们如何经常低估这些挑战，感到沮丧并放弃。在本次 DevOps 世界会议中，他将阐述 DevOps 计划失败的原因。](https://devops.cioreviewindia.com/cxoinsight/5-things-santa-claus-taught-me-about-devops-nid-4137-cid-151.html)

本次会议还将为与会者提供一个机会来尝试会议提供的反馈组件。DevOps 在你的世界里失败了吗？你和你的组织能做些什么？
