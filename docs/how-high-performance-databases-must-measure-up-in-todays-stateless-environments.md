# 在分布式世界中测量高性能数据库

> 原文：<https://thenewstack.io/how-high-performance-databases-must-measure-up-in-todays-stateless-environments/>

CloudBees 赞助了这个播客。

对于具有高容量和复杂数据管理需求的组织来说，寻找最适合的数据库一直是一个挑战。随着企业越来越多地在无状态云原生平台上混合搭配内部和多云环境，对数据库的性能要求变得更高。

“如今应用程序开发人员的目标是将除了数据库之外的所有东西都视为无状态的，”[首席技术官、](https://www.linkedin.com/in/eliothorowitz) [MongoDB](https://www.mongodb.com/) 的联合创始人埃利奥特·霍洛维茨说。对于数据库提供商来说，这意味着“数据库变得越来越复杂，数据平台也必须变得越来越复杂，才能处理这种复杂性。”

在这个 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，Horowitz 在拉斯维加斯的 [IFX2019](https://ifx.packet.com/) 现场录制了当今数据库开发挑战的复杂性，并描述了 MongoDB 的发展，以跟上应用程序变得越来越以云为中心和无状态的变化步伐。

[高性能 w/Eliot Horowitz MongoDB](https://thenewstack.simplecast.com/episodes/high-performance-w-eliot-horowitz-mongodb)

随着开发人员寻求能够满足应用程序日益苛刻的需求的数据库，数据库创建者必须想出如何满足性能需求，同时管理由此带来的平台复杂性的增加。

“在我们的行业中，我们考虑的一件事是，尽管任何数据库都可能是好的，但人们总是希望用数据做更多的事情。霍洛维茨说:“最近发生的有趣的事情是，存储数据的方式如此廉价，真的没有必要删除任何东西。“如果人们想存储数据，从中学习，让它变得互动，让他们的用户跟踪他们的数据，这只会产生更多的问题。”

在具体的数据库开发挑战中，有一些选项允许组织利用成本较低的存储或诸如[亚马逊网络服务(AWS)](https://aws.amazon.com/) [S3](https://aws.amazon.com/s3/) 等替代方案来共享数据。

“如何将更多的功能融入进来，并从一个数据库和一组组件转变为一个平台？”霍洛维茨说。数据库客户可能会说，“好吧，这是我的数据，你要知道使用哪些数据，并知道如何对其进行组织、分层和管理，这是我需要为我的业务用例做的事情，我不想去想它。让它发挥作用吧。"

例如，对于 AWS 客户来说，MongoDB 最近增加了一些功能，使得在该平台上管理 MongoDB Atlas 变得更加容易。此举是 MongoDB 持续努力的一部分，旨在覆盖更广泛的网络，以满足不同云替代方案组织的需求。“使用 MongoDB 的绝大多数人可能在公共云中运行它——这是最近大多数新开发发生的地方。MongoDB Atlas 运行在所有三大公共云上:亚马逊、谷歌和微软，”Horowitz 说。“它让您只需通过 API 或 UI 使用 MongoDB，而无需担心任何事情，这非常简单，因此您可以使用信用卡，从零成本的免费层开始，一直到非常大的集群。”
