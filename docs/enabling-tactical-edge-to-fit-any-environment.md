# 使战术优势适应任何环境

> 原文：<https://thenewstack.io/enabling-tactical-edge-to-fit-any-environment/>

对于许多 DevOps 团队来说，让管道几乎没有破损地运行就是胜利。然而，对于一些组织来说，这可能意味着成功或失败的区别。

对于软件开发公司 [Jasper Solutions](https://www.jaspersolutions.com/) ，这再正确不过了。Jasper Solutions 为世界各地的客户提供一系列商业技术服务，包括联邦、州和地方政府机构。它为组织提供的技术解决方案之一是战术优势。战术优势是指在受到有限通信、连接或存储可用性限制的特定环境中运作的能力。许多 it 服务和系统被设计为在强大的 IT 环境中运行，但是当部署在 IT 环境之外时，例如在存在间歇性连接或没有连接的现场操作中，这些服务和系统无法扩展。

Jasper Solutions 以首席执行官兼所有者[安迪·帕特尔](https://www.linkedin.com/in/anshuman-andy-patel-35a48a/)创造的“盒子里的 DevOps”的形式提供不断更新的进化数据这使操作处于战术边缘，并提供了修改任何应用程序以适应任何环境的能力，并在网络建立时间歇地更新。像空军、海军陆战队、水兵和其他军事服务部门这样的公共部门客户非常依赖 Jasper 解决方案来满足其关键的战术优势需求。这种决定性的依赖要求技术不仅在某些时候有效，甚至在大部分时间有效，而且每次都有效。

五年来，Jasper Solutions 的开发人员将各种管道工具(如 Atlassian、BitBucket、吉拉、Jenkins 和 SonarQube)“科学地”结合在一起，以便为客户及其独特的开发需求提供“盒子中的开发工具”管道解决方案。然而，Jasper Solutions 开发人员不断遇到单个工具更新中断的挑战——当堆栈中的一个工具更新时，它会中断与其他工具的通信。

“我们更担心的是管道何时会破裂，以及如何实际修复管道或回滚管道，以便我们可以继续进行开发，”Patel 说。他说，然后他们会介绍每个不同产品的附加特性和功能，以便对其进行全面配置和测试。"很多时候，开发和测试环境会有不同供应商的各种测试包."

为了创建和维护这种“盒子中的 DevSecOps”方法，Jasper solutions 需要一个易于部署、提供代码管理并具有安全意识的平台，最好是在一个工具中。单一平台将允许团队专注于开发，而不是由来自多个工具的更新导致的持续的管道故障。这种能力不仅可以帮助团队成员，还可以提高客户满意度——以更快的速度交付更好的产品。

在测试了 260 多种 DevOps 工具后，Jasper Solutions 找到了一个满足所有需求的平台。GitLab 是一个 DevOps 平台，在一个应用程序中涵盖了 DevOps 生命周期的所有阶段，最初因其透明性而脱颖而出。据 Patel 称，不仅工具的可用性是透明的，而且公司的路线图也是透明的。

“从价值主张的角度来看，特别是在不断发展和变化的 DevSecOps 领域，讨论 GitLab 目前没有提供的特定问题或差距并将其实际放在路线图上的能力——没有其他公司在完全透明的情况下做到这一点，”Patel 说。"同时，解决 CI、CD 和 SCM 的整个堆栈问题."

GitLab 提供了 Jasper Solutions 的客户受益的所有功能、特性和优势。尽管最初只用于内部开发，但在团队看到 GitLab 支持部署是多么容易之后，GitLab 成为了 Jasper Solution 外部服务的一部分。GitLab 现在是他们用于 CI、CD 和 SCM 的单一堆栈软件。

Patel 说:“当我们进行一次升级时，整个管道都会升级，增加所有新的特性和功能。“99%的时间我们没有打破管道，我们仍然可以满足客户的软件开发需求，专注于应用程序的生命周期，而不是担心我们的管道。"

自从三年前采用 GitLab 来满足其 DevOps 需求以来，Jasper Solutions 已经将周期时间减少了 30%。部署频率加快了 25%, 90-95%的项目发布符合预算并按时完成，每个产品平均节省了近 350 个工时。

通过 GitLab，“我们从交付、执行以及管理角度提供了更好的价值主张。”
