# AWS Lambda 如何成为今天的样子

> 原文：<https://thenewstack.io/how-aws-lambda-became-what-it-is-today/>

首先是“无服务器”这个词，几年前，亚马逊网络服务公司(Amazon Web Services)开始称其产品为“Lambda ”,当时组织刚刚开始学习这个平台如何为他们服务。当然，虽然不是对每个人都适用，但是 Lambda 已经被证明比传统的虚拟机(VM)环境有很多优势。Lambda，以及来自 [Azure](https://azure.microsoft.com/en-us/solutions/serverless/) 、 [Google Cloud](https://cloud.google.com/serverless/) 和其他替代产品的无服务器平台也发生了重大变化。

[AWS Lambda 怎么变成了今天这个样子](https://thenewstack.simplecast.com/episodes/how-aws-lambda-became-what-it-is-today)

在最新的[中，New Stack Makers](https://thenewstack.io/podcasts/makers) 播客在拉斯维加斯 IFX2019 现场录制，无服务器和新兴云服务总经理 [Andrew Tunall](https://www.linkedin.com/in/andrewtunall) 和 New Relic 首席工程师 [Erica Windisch](https://www.linkedin.com/in/ewindisch) 讨论了 Lamba 和无服务器如何变化和发展，以及一些组织如何受益于这项技术。

当无服务器的概念被引入时，Windisch 说很多时间和精力都花在了向人们解释“什么是无服务器”上。

如今，组织通过在 Lambda 和无服务器上的部署引入了不同程度的复杂性。Windisch 说，例子包括分布式应用。“越来越多的人遇到这些更难的事情:这些具有更复杂架构的用例，”Windisch 说。“这不是因为无服务器很难，而是因为人们处于他们可以构建的成熟水平，所以他们在无服务器而不是在容器中构建复杂的应用程序。”

Windisch 说，Lambda Destinations 允许 DevOps 说“‘我的代码运行，如果它有一个异常，如果它没有异常，就做另一件事。“我们正在 Lambda 的基础上构建新的模式，但同时也降低了我之前描述的复杂性，同时做了以前用容器不可能做的事情，现在将不可能变得复杂变得简单。”

Tunall 表示，早期的一个典型用例可能包括使用 Amazon CloudWatch 事件及其 Chrome 功能来触发 Lambda 函数，该函数可以进行一些分析和编排工作。“我们仍然看到客户这样做，但越来越多的客户在 Lambda 上构建他们的整个平台，”图纳尔说。

New Relic 的客户 [Morningstar](https://www.morningstar.com/) 是一家金融服务公司，它在 Lambda 上重建了自己的网站，并在其实现中使用了 New Relic。“公司开始[向无服务器进行如此重大的转变]的原因是无服务器本身的许多发展，”

事实上，无服务器已经发展到这样的程度，例如，晨星可以使用该平台与关系数据库进行连接池。“他们没有采用旧的解决方案并试图将其倾倒在 it 中，但他们试图解决这个问题——也许通过不同的方式——同时使你不必基于某些权衡做出技术选择，这可能是负面的，”图纳尔说。“相反，您正在根据适合您工作负载的技术做出技术选择，”
