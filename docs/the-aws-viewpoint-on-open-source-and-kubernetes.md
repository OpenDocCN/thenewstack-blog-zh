# AWS 对开源和 Kubernetes 的观点

> 原文：<https://thenewstack.io/the-aws-viewpoint-on-open-source-and-kubernetes/>

[KubeCon+CloudNativeCon](https://www.cncf.io/kubecon-cloudnativecon-events/) 赞助了这个播客。

Kubernetes 当然在发展，但组织在云原生环境中无缝部署和运行应用程序还需要一段时间。[亚马逊网络服务](https://amzn.to/36oOGE9) (AWS)既是 Kubernetes 的早期支持者，也是云原生服务和支持的领先提供商，因此在过去几年中一直隐含地参与其变化。

在这段视频中，[The New Stack Makers](/podcasts/makers/)podcast、 [Bob Wise](https://www.linkedin.com/in/bobwise) ，AWS 的 Kubernetes 总经理和 [Peder Ulander](https://www.linkedin.com/in/ulander) ，AWS 负责企业、开发者和开源项目的产品营销主管，描述了 AWS 在 Kubernetes 中的角色以及 cloud native 如何融入该公司的开源战略。他们还讨论了 Kubernetes 如何在市场中发展，包括客户需求如何变化，以及为什么开源技术对于填补空白以实现云原生的全部潜力至关重要。

《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了这一集。

[AWS 对开源和 Kubernetes 的看法](https://thenewstack.simplecast.com/episodes/the-aws-viewpoint-on-open-source-and-kubernetes)

怀斯指出，在 AWS 推出亚马逊弹性 Kubernetes 服务(亚马逊 EKS)之前，客户已经开始在 Kubernetes 中构建系统。他说，一旦 EKS 上线运行，客户就“非常迅速地投入生产，部分原因可能是他们有在 AWS 上独立运行 Kubernetes 的经验”。

怀斯说:“但我要说的是，在过去的 18 个月里，客户的担忧发生了明显的转变，他们不再担心如何才能让一件东西首次投入生产。“我们已经看到了集群数量和规模的大幅增长，尤其是来自我们的一些大客户。”

随着客户开始扩大其集群，客户的兴趣和关注点也发生了变化。怀斯说，例如，安全性是一些客户在投入生产之前就关心的主要问题，而 AWS EKS 的其他客户“在生产过程中就知道了”，然后他们才开始意识到“这个系统关系重大”，并确保他们明白什么是风险。

“管理安全性是我们工作的一大部分，因为我们的主要工作是帮助客户以安全的方式运行 Kubernetes。当然，随着 Kubernetes 变得越来越大，它在那里变得更有吸引力，是一个更大的目标，”他说。“因此，我认为安全问题是一个很好的迹象，表明当客户开始在一项技术上投入巨资时，他们开始真正关注它的安全方面。”

客户“希望开源技术不仅能填补他们现有组件的空白，还能有效地帮助他们更快地创新。因此，当我们第一次与客户接触时，或者每当我们与客户接触时，总会围绕“我最喜欢的开源能力技术平台工具在 AWS 上可用吗？””乌兰德说。我想说的是，就在过去的七个月里，考虑到疫情，随着客户的增长，推动更快的数字化转型实际上增加了我们从开源角度和亚马逊开源获得的问题和兴趣的速度。"
