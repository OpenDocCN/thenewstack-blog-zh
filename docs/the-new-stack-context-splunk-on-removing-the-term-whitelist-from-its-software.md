# 新的堆栈上下文:Splunk 从其软件中删除了术语“白名单”

> 原文：<https://thenewstack.io/the-new-stack-context-splunk-on-removing-the-term-whitelist-from-its-software/>

欢迎来到[新的堆栈环境](/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了[埃里克·萨默](https://twitter.com/esammer)、 [Splunk](https://www.splunk.com/) 杰出的工程师，谈论了这家 IT 系统监控公司正在努力重新命名其术语，以删除在技术领域延续系统性种族主义和无意识偏见的语言。Splunk 召集了一个由整个组织的人员组成的工作组来制定其他建议、指南和程序，以识别和替换有偏见的语言，并防止将来发生其他情况。我们还与 Sammer 聊了聊自他联合创立的公司事件驱动服务监控提供商 [Rocana](/rocana-aims-eliminate-blame-game-microservices-troubleshooting/) 于 2017 年被 Splunk[收购以来发生的事情。](https://www.splunk.com/en_us/blog/it/rocana-joins-splunk.html)

TNS 编辑和营销总监[利比·克拉克](/author/libby/)主持这一集，旁边还有 TNS 高级编辑[理查德·麦克马努斯](/author/richard/)和 TNS 执行主编[约阿布·杰克逊](/author/joab/)。

[第 122 集:Splunk 关于从其 IT 系统中移除排他性语言](https://thenewstack.simplecast.com/episodes/episode-122-splunk-on-removing-exclusionary-language-from-its-it-systems)

六月十五日快乐！这一天标志着 1865 年的这一天，美国的黑人奴隶得知他们获得了自由。今年，由于黑人的命也是命的激烈抗议，我们看到 IT 公司将这一天视为正式的工作假期。多亏了 BLM，我们开始看到这个行业发生了一些其他有意义的变化。

本周，许多 IT 公司重新定义了一些长期以来带有潜在攻击性含义的传统 IT 术语。例如，GitHub 放弃了“主”和“从”，取而代之的是“主/默认/主要”和“次要”。同样在本周，Splunk 听取了合作伙伴的建议，取消了“白名单/黑名单”这一术语，转而使用更加中性的措辞。因此，我们采访了 Sammer，以了解 Splunk 的更多努力。

“我来自一个世界，当有人说，‘嘿，这东西伤害了我’，然后你就停止这样做，”萨默在采访中说。“作为一个社会，语言在进化，我们在学习，我们在成长。我认为质疑、挑战和改变一些根深蒂固的东西是有意义的。”
