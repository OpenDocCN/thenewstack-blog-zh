# 米科斯:“野心”是将 AWS 兼容性引入惠普的 Helion

> 原文：<https://thenewstack.io/mickos-the-ambition-is-to-bring-aws-compatibility-to-hps-helion/>

虽然惠普在关于[收购公司](https://thenewstack.io/marten-mickos-heads-to-hp-with-eucalyptus-acquisition/ "The New Stack")的新闻稿中没有提到 Eucalyptus 技术的未来，但 Eucalyptus 的首席执行官 Marten Mickos 听起来似乎希望将 Eucalyptus 整合到惠普的 Helion 产品中。

“我们的目标是找到一种方法，将 Eucalyptus 路线图融入 Helion OpenStack 路线图，”他在接受 New Stack 采访时说。然而，他提醒说，细节还没有制定出来。“我相信这是可以做到的，但我并不是说已经达成一致的路线图。”Helion 是惠普用于其私有和公共 OpenStack 产品的品牌。

那么合并桉树和太阳神意味着什么？这意味着兼容亚马逊网络服务以及其他潜在的领先云服务。

结果并不是 OpenStack，就像在开源软件栈中一样，将获得 Eucalyptus 所提供的那种 AWS 兼容性。“我们不能说 Eucalyptus 将成为 OpenStack 产品，因为 OpenStack 对 OpenStack 有严格的定义，但我要说的是，我们对私有云感兴趣的客户可以找到我们，说给我 Helion，它就会成为 Helion OpenStack。如果客户说我需要亚马逊的兼容性，我们也会提供，”他说。

他说，Eucalyptus 已经考虑如何适应 OpenStack 大约一年了。“我们知道这是可行的，”他说。

“也就是说，如果你看看惠普的战略，它是基于 OpenStack 的。什么都没变。他说:“有了 Eucalyptus，我们可以与需要亚马逊兼容性的客户开展更多业务。

这样的愿景不会在一夜之间实现。“到达那里需要时间。我们不会操之过急，因为那会威胁稳定，”他说。

Mickos 还指出，Eucalyptus 一直在考虑支持与其他公共云服务兼容的方法。“与公共云的兼容性非常重要。我们有 AWS 兼容性，但让我们明确一点。还会有其他主要的公共云，”他说。他说，在收购之前，Eucalyptus 的工程师们一直在沿着这些思路思考，希望能够兼容谷歌和微软这样的云。

尽管再次指出路线图尚未与惠普敲定，但 Mickos 建议将继续支持现有的 Eucalyptus 客户。他说:“Eucalyptus 如今拥有运行大规模云的出色客户，他们将继续这样做，他们将获得新功能，他们将成为 Eucalyptus 的成功用户。”

Mickos 说，现在还不清楚 Eucalyptus 与亚马逊的安排会发生什么。Eucalyptus 自己完成了所有的开发工作，但会向 AWS 寻求反馈。AWS 会提供一些需要考虑的事情的建议。Mickos 说，他不想猜测这种关系会发生什么，因为 Eucalyptus 将被惠普拥有。

惠普在 OpenStack 和云计算上下了很大的赌注，但该公司是否会赢得可观的市场份额并不确定。它正在与其他传统企业供应商竞争，如 IBM，他们也在使用 OpenStack，以及像 Red Hat 这样的大型开源公司。此外，它还将与微软和谷歌等非 OpenStack 用户争夺公共云中的业务。

不过，米科斯认为惠普处于有利位置。“我们在 Eucalyptus 了解到，客户说我们想从一个地方购买所有东西，”他说。他说，由于惠普提供软件、硬件和服务，它可以满足这种需求。
