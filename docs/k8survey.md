# TNS Research:告诉我们你在 Kubernetes 的经历

> 原文：<https://thenewstack.io/k8survey/>

新的堆栈提供了对 Kubernetes 开源容器编排引擎的全面覆盖，我们希望进一步投资于该社区。2016 年，我们的调查报告了“[集装箱编排的现状](https://thenewstack.io/tns-research-present-state-container-orchestration/)、*”*，但去年发生了很大变化。虽然 Kubernetes 目前的思想份额超过了许多竞争对手，但集装箱战争远未结束。Kubernetes 的成功依赖于早期采用者的满意度和它在许多不同用例中的成功记录。

这就是你进入画面的地方！我们正在调查已经评估过 Kubernetes 的人。我们希望更好地了解那些早期采用者如何以及为什么选择 Kubernetes，以及他们目前如何使用它。研究结果将为即将出版的关于 Kubernetes 生态系统状况的电子书系列奠定基础。

> [立即参加 Kubernetes 用户调查！](https://www.surveymonkey.com/r/TNS-K8s?c=article)

我们有兴趣了解更多的一些话题:

*   随着市场的成熟，为什么 Kubernetes 会(或者不会)被其他解决方案选中？
*   复杂性是否将用户推向了供应商分销或咨询服务？

无论您是从应用还是运营的角度来看，我们都希望得到您的答案。您的反馈将帮助我们回答以下方面的问题:

### 应用程序部署

*   Kubernetes 部署了哪些类型的应用程序？
*   Jenkins 和 Kubernetes 如何融入 CD 管道？

### 管理基础设施

*   连接了哪些类型的存储卷(例如，块、文件、对象)(例如，CephFS、Flocker、Portworx)？
*   使用哪种 SDN 产品(例如 Contiv、OpenContrail、Weave Net)？
*   如何监控 Kubernetes 集群(例如 cAdvisor、CoScale、Heapster、Prometheus)？

如果您想了解更多关于调查问题本身的信息，我们还提供了问题的 [PDF](https://cdn.thenewstack.io/media/2017/05/061380a2-tns-spring2017-kubernete_survey.pdf) 。但是，请花时间填写[调查](https://www.surveymonkey.com/r/TNS-K8s?c=article)部分。通过您的反馈，我们希望影响其他人如何评估 Kubernetes 以及他们如何选择在自己的环境中使用它。我们希望这些结果显示出不断增长的用户群和持续的承诺，尤其是我们希望在即将推出的 Kubernetes 电子书系列中反映这些结果。

我们需要您对评估和使用 Kubernetes 的反馈。[在这里接受我们的调查](https://www.surveymonkey.com/r/TNS-K8s?c=article)！
