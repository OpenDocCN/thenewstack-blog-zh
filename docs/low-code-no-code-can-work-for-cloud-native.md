# 低代码、无代码可用于云原生

> 原文：<https://thenewstack.io/low-code-no-code-can-work-for-cloud-native/>

“低代码，无代码”的定义仍然是一个争论的话题。对于一些人来说，这是缺乏软件开发培训和技能的所谓“公民开发人员”的能力——能够依靠一个平台来部署与专业软件工程师同等水平的代码。其他人将低代码、无代码描述为一种依赖于促进软件开发的平台的方式——同时自动化构建中的许多任务——既简化了没有经验的开发人员的过程，又节省了有经验的开发人员的时间和资源。在这两种情况下，在这个日益拥挤的空间中，低代码、无代码使得编码和软件开发过程更加简单，结果更加自动化。

在低代码、无代码平台提供商 [gopaddle](https://gopaddle.io/) 的例子中，这个想法是“为现代应用程序释放无代码平台的力量。”如何将低代码、无代码应用于在云原生环境中运行的以 Go 为中心的应用程序是本次新堆栈制造商播客的主题，gopaddle 创始人兼首席执行官 [Vinothini Raju](https://in.linkedin.com/in/vinothini-raju-9817ab5) 担任嘉宾。《新书库》创始人兼发行人亚历克斯·威廉姆斯和 TNS 新闻编辑[达里尔·塔夫脱](https://twitter.com/darrylktaft)主持了对话。

[低代码，无代码可为云原生](https://thenewstack.simplecast.com/episodes/low-code-no-code-can-work-for-cloud-native)

例如，Raju 说，低代码、无代码平台的一个关键特性是支持首次开发人员的学习过程，他们可能不完全了解构建一个成熟的 CRM 软件工具所需的最佳实践。第一次开发人员，或者“希望让他们的应用程序更快启动并运行的重复开发人员”是目标用户。Raju 说:“即使是第一次开发的人，如果没有完整的知识，也可以在更短的时间内让他们的软件运行起来。这与我们在云原生工作负载方面应用的理念相同，但这里有一个更广泛的问题陈述，因为我们正在考虑各种各样的开发实践和应用。此外，底层基础设施将会有所不同。"

如上所述，许多提供的低代码、无代码平台已经出现，使它成为一个越来越拥挤的空间。“有一些低代码、无代码的平台正在出现，所以，对我们来说，关键的区别在于我们可以装载更全面的工作负载，”Raju 说。“今天的现有平台有一些机制可以让您的简单工作负载启动并运行，但您是否可以将所有这些服务连接在一起，构建更复杂或更大的应用程序[还有待观察]…我认为我们的平台提供了一种机制来构建更全面的应用程序模板，类似于掌舵图。”

Raju 说，Gopaddle 提供了一种“脚手架机制”，作为“将你的应用程序放到 Docker 或 Kubernetes 上”的基础。虽然仍处于 alpha 阶段，但 gopaddle 的智能脚手架能够从应用程序源代码中建议 Docker 或 Kubernetes 配置文件。“我们实际上可以从你的源代码开始，动态构建你的 Docker 文件和你的 Kubernetes 工件，”Raju 说。
