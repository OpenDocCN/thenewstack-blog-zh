# Pokemon Go Creator 如何为开发者构建 Kubernetes

> 原文：<https://thenewstack.io/how-pokemon-go-creator-builds-on-kubernetes-for-developers/>

Kubernetes 在维护 Niantic 广受欢迎的增强现实开发 Pokemon Go 方面发挥了关键作用。Kubernetes 及其为 DevOps 团队提供的效率继续在 Niantic 发挥作用，因为该公司在游戏的架构上为第三方开发者构建。

在新堆栈制造商播客的最新一集中， [Niantic](https://nianticlabs.com/fr/) 的高级产品经理 [Ria Bhatia](https://www.linkedin.com/in/ria-bhatia-a8135b98) 讨论了为什么 Pokemon Go 平台仍然相关，以及为什么 [Kubernetes](/category/kubernetes/) 将仍然是该平台不可或缺的一部分，因为该公司希望引入更多的“开发者客户”

[Pokemon Go Creator 如何为开发者构建 Kubernetes](https://thenewstack.simplecast.com/episodes/how-pokemon-go-creator-builds-on-kubernetes-for-developers)

巴蒂亚解释说，Niantic 的使命是“激励人们走出去，一起探索世界”。该公司实现这一目标的第一种方式是发布游戏，如 Pokemon Go。该公司还在扩展该平台，以提供更多服务，如增强现实和地理定位服务。巴蒂亚说，这个想法是向“全世界”提供服务平台，这样他们就可以在我们的基础上构建应用程序。

巴蒂亚说，Niantic 已经依赖 Kubernetes 超过五年，成为“Kubernetes 最老的用户”之一。“当我们发布 Pokemon Go 时，一切都在 Kubernetes 上，”巴蒂亚说。“我们必须快速扩大规模，因为我们获得了意想不到的玩家使用量——我认为这是一个非常疯狂的时期。”

如今，每个游戏用户都可以访问单独的集群。巴蒂亚说:“我认为我们将保持这种与世隔绝的感觉，因为基础设施就在下面。”。

改善开发者体验仍然是 Niantic 正在进行的工作。Bhatia 解释说，这包括，在其他事情中，使用平台发布游戏更容易，这意味着使开发者创建应用程序更容易。

巴蒂亚说:“我们希望确保，如果你在第一天登陆 Niantic 的平台，你就能够根据你的概念建立原型——今天，这不是特别正确，Niantic 很棒，但也非常复杂。”

Niantic 今天面临的挑战是如何将用于创建 Pokemon Go 游戏的堆栈和平台转化为第三方开发客户更容易访问的东西。

“我们建立了一个非常硬的技术堆栈——我们实际上从 Pokemon Go 中分离了许多组件，并创建了一个平台，这很棒，因为我们喜欢这些核心组件。但我们没有考虑可用性或可靠性，所以，这就是下一步，”巴蒂亚说。“我们正在组建我们的核心平台团队，我们正在整个公司组建我们的站点可靠性工程和服务器工程团队，我们还有机器学习(ML)工程职位空缺。”
