# WebAssembly 的下一步是什么？

> 原文：<https://thenewstack.io/whats-next-in-webassembly/>

德克萨斯州奥斯汀。—对它的朋友来说，[web assembly](https://thenewstack.io/what-is-webassembly/)—Wasm——一种基于堆栈的虚拟机的二进制指令格式，允许开发人员用他们最喜欢的编程语言构建并在任何地方运行他们的代码，它的未来是什么？

对于 Fermyon Technologies 的首席执行官兼创始人马特·布彻(Matt Butcher)来说，Wasm 的未来在于在浏览器之外运行它，并在从代理服务器到视频游戏的所有东西中运行它。"

他补充说，“真正令人兴奋的是能够在云中运行它，以及像虚拟机和[容器](/category/containers/)这样的云服务。”

在新的 Stack Makers 播客的《上路》一集中，Butcher 接受了 TNS 专栏编辑 Heather Joslyn 的采访。

Butcher 说，随着 Ruby、Python 和 C#等关键编程语言增加对 WebAssembly 新功能的支持，Wasm 正在获得临界质量。

“我们现在谈论的是实现在 WebAssembly 中已经存在了很长时间的潜力。但随着人们变得兴奋，开源项目开始采用它，我们现在看到的就像是浪潮的开始。”

但是在广泛采用之前，Butcher 说，在准备下一波 Wasm 的环境方面还有工作要做:云计算。

与字节码联盟的其他成员一起，例如 Cosmonic、Fastly、Intel 和 Red Hat，Fermyon 今年正在努力改善开发者的体验和环境。他补充说，下一步是“开始构建第一波应用程序，真正突出它可以为我们带来什么。”

Butcher 指出，Wasm 的兴起代表了云原生技术的新时代。“我们喜欢容器。我们中的许多人已经参与 Kubernetes 生态系统很多年了。我最初建造[掌舵](https://helm.sh/)；在某种程度上，那还是我的孩子。

“但我们也很兴奋，因为现在我们找到了一些在容器生态系统中没有解决的问题的解决方案。这就是为什么我们把它称为下一波浪潮。”

## Wasm 和“无摩擦”开发体验

6 月，在 Linux 基金会的北美开源峰会上，Fermyon 介绍了它的“无摩擦”WebAssembly 平台。该平台建立在包括 [HashiCorp 的](https://www.hashicorp.com/?utm_content=inline-mention) Nomad 和 Consul 在内的技术之上，能够编写微服务和网络应用。Fermyon 的开源工具 [Spin](https://github.com/fermyon/spin) ，帮助开发者将应用从他们的本地开发环境推送到他们的 Fermyon 平台。

在我们的 Makers 讨论中，Butcher 强调了 Wasm 未来的一个方面，即它如何在保持轻量级的云资源消耗的同时实现可伸缩性。

“除了在一个安全的平台上创造这种伟大的开发者体验，我们还将帮助人们节省云成本，因为云成本已经有点失控了，”他说。

“如果我们能够真正注意我们使用的资源，并帮助开发人员理解编写灵活、资源使用量小的代码意味着什么。真正的目标是让他们在编写代码时，恰好具备这些特征。”

对于那些对 WebAssembly 感兴趣的人来说，Fermyon 创建了一个名为[挑剔的胡须](https://www.finickywhiskers.com/index.html)的在线游戏，旨在展示如何用 Wasm 重新想象[微服务](/category/microservices/)。
