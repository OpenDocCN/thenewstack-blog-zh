### 有人能解释这个“码头工人分叉”的废话吗？

*   [Coté的报道](https://cote.io/2016/09/01/deciding-where-the-docker-ecosystem-will-make-money/)。
*   [Docker Inc .不想成为商品化的积木](https://thenewstack.io/docker-fork-talk-split-now-table/)。
    [来自一位红帽人士](https://www.linkedin.com/pulse/forking-docker-daniel-riek):“今年夏天早些时候，当 Docker，Inc .利用其控制地位将它自己的 Kubernetes 风格的容器编排克隆体 Swarm 推入核心 Docker 项目时，冲突开始升级，使基本容器运行时与其生态系统的一个显著部分发生冲突。Docker Inc .接着在 Red Hat Summit 上指责 Red Hat 对 Docker 进行分流。之后，Docker，Inc .的 Solomon Hykes 强烈反对在 OCI 标准化容器运行时的努力——这是他的公司共同发起的倡议。"
*   回复:我们讨论 Docker 生态系统挑战的那一集:“然而，支持来自 Red Hat 客户用例的有效需求的 Red Hat 补丁经常被关闭，原因似乎很简单，它们不符合 Docker，Inc .的业务战略。"
*   关于在免费/开放/商品化和成本/专有/竞争之间划一条线的争论:“虽然我个人认为编排层是容器范式的关键，但这里正确的方法是将编排与核心容器运行时标准化分开。这避免了容器运行时不同层之间的冲突:我们可以就通用容器包格式、传输和执行模型达成一致，而不会限制在 Kubernetes、Mesos、Swarm 等之间的选择。”
*   不要带手枪去打火箭筒。企业热爱 RHEL——你有没有尝试过向企业推销 Ubuntu？这就像销售 NT 时的情形一样。
