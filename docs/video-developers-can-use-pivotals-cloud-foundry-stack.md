# 视频:开发者如何使用 Pivotal 的 Cloud Foundry 堆栈

> 原文：<https://thenewstack.io/video-developers-can-use-pivotals-cloud-foundry-stack/>

这是关于使用 Pivotal Cloud Foundry 平台的四部分视频教程系列的第二部分。

昨天，我们[提供了关于如何使用 Pivotal 的](https://thenewstack.io/video-get-started-pivotals-cloud-foundry-stack/) [Cloud Foundry](http://pivotal.io/platform) 构建云服务的高级概述，这是一款用于运行平台服务的开源软件。

在这个新版本中，Pivotal 首席技术专家 [Casey West](https://twitter.com/caseywest) 讨论了开发人员如何在日常工作中使用 Cloud Foundry。他向我们介绍了在 Cloud Foundry 中部署 Docker 的过程，对于那些[Perl](https://thenewstack.io/perl-back-ready-roll-big-data/)粉丝来说，West 以一个小小的惊喜结束了这个视频。

正如在上一期文章中所讨论的，迁移到原生云的一个好处是，平台服务可以处理运行应用程序所需的大量基本配置，例如处理网络通信。

“当 Cloud Foundry 启动应用程序实例时，它会提供一个动态绑定端口。因此，只要您在该端口上侦听传入的请求并在同一时间发送您的输出，您就能够为您的应用程序协商网络通信，”West 解释道。

这里使用的所有技术都是开源的，所以家庭观众可以在他们自己的机器上复制这里的演示。
