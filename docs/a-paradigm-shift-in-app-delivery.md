# 应用交付的范式转变

> 原文：<https://thenewstack.io/a-paradigm-shift-in-app-delivery/>

对于提供分布式数字体验的组织来说，提高应用交付和更新的速度并保持其在互联网基础设施上的可用性仍然是典型的挑战。

在这一新的 Stack Maker 播客中，Citrix 产品营销高级总监 Pankaj Gupta 讨论了为什么要对应用交付进行彻底的变革。

[Alex Williams](/author/alex/) ，New Stack 的创始人和发行人，主持了这个播客。

[应用交付模式的转变](https://thenewstack.simplecast.com/episodes/a-paradigm-shift-in-app-delivery)

随着向云的转变加速，组织也在寻找通过云交付应用时降低风险的方法。古普塔说:“我认为，最近疫情已经让每一家数字企业都变成了一家提供体验的公司。”。“如果几年前你谈到向云和 SaaS 的过渡，每个人都在朝这个方向努力。但现在的问题是，我能走多快，在降低风险的同时，我能有多自信地实现向云的超级过渡，这[创造]了许多新的机遇和挑战。”

Gupta 说，组织转向基于云的部署的另一个主要原因是受益于“自动扩展”。“但自动扩展的挑战是，你必须对 CPU 和内存进行大量猜测……如果你的意图或要求发生变化，你必须从头开始，多次重复这个循环，”古普塔说。

这也是为什么组织越来越多地重新思考他们的应用交付方法的原因之一。“现在是时候审视你的应用交付基础设施，并采取一种全新的激进方法来建立一个新的应用交付和安全基础设施，”Gupta 说。

为此，Citrix 推出了一种新的应用交付和安全服务，Gupta 称这种服务代表了“为用户交付应用方式的彻底转变”

“每个人都在寻找一种非常简单的方式来部署和消费应用交付基础设施，”Gupta 说。为此，Citrix 推出了[应用交付和安全服务(CADSS)](https://www.citrix.com/products/citrix-app-delivery-and-security/) 。CADSS 建立在“意图驱动的应用交付和安全服务”的“原则”之上，它代表了“一种范式的转变，不仅仅是今天，它还是未来十年的一个平台。"

Gupta 解释说，最终结果是可以消除缓慢的手动易错配置。“因此，我们消除了应用交付几乎每一步的复杂性。我们相信，借助这种出色的自动化，我们可以将运营效率提高 60%。”
