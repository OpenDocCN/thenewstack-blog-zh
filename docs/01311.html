ADP如何采用容器思维

原文：https://thenewstack.io/adp-adopted-container-mindset/#0001-01-01

对于任何企业来说，进入容器思维模式都是最重要的。但最终，这意味着与API进行交互，这是当今企业开发人员工作流的一部分，但在考虑扩展和消除手工劳动时仍然是一种调整。

詹姆斯·福特是T2 ADP T3的首席架构师，这是一家市值110亿美元的公司，也是人力资本管理领域的支柱。福特和他的团队在Docker上进行了大量投资，从整个企业使用的大约1000个集装箱增加到现在的3000多个。这只是该公司向外扩展的开始，但它已经做了足够多的工作，让人们对容器感到放心，特别是在安全性方面。

本播客中与福特的讨论反映了企业在转向容器架构方法时必须考虑的因素。ADP的价值在于Docker可以跨多种环境使用，包括虚拟机、大型机、裸机、Amazon Web Services和Azure。

福特的观点反映了在采用Docker时，拥有一个容器心态是多么重要。但福特也信奉另一个真理:总是扰乱自己，尽量不要控制欲太强。实验至关重要。
