# 背景:逆转康威卓越服务管理定律

> 原文：<https://thenewstack.io/context-reversing-conways-law-for-superior-service-management/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论来自云原生社区的最新新闻和观点。本周，我们采访了云铸造基金会的首席技术官 [Chip Childers](https://www.linkedin.com/in/chipchilders/) ，探讨了有助于定义现代企业的技术和架构决策。 [Cloud Foundry Foundation](https://www.cloudfoundry.org/) 是一个独立的组织，负责监督 Cloud Foundry 平台的开发和采用，并且是新堆栈的赞助商。

超过一半的财富 500 强企业使用 Cloud Foundry，因此 Cloud Foundry 基金会依赖于使用、贡献和推进其开源项目的企业。Childers 与基金会所有成员公司的开发人员和首席信息官密切合作，因此他对现代企业需要什么才能在当今快速变化的技术环境中取得成功有着深刻的理解。在最近的一篇关于新堆栈的文章中，他写道“现代企业…批判性地思考他们应该自己建造什么，他们应该从其他地方采购什么。”

[【逆向康威法则】卓越服务管理](https://thenewstack.simplecast.com/episodes/reversing-conways-law-for-superior-service-management)

我们与 Childers 讨论了开源如何改变企业的软件环境，以及是购买软件还是内部构建软件更好。我们还谈到了微服务如何帮助组织根据其提供的服务重新定义其劳动力——实际上逆转了[康威定律](https://www.thoughtworks.com/insights/blog/demystifying-conways-law)的观点，康威定律描述了官僚机构如何设计反映其内部组织结构的系统，这往往对他们不利。

然后，在稍后的播客中，我们讨论来自[新堆栈](/)的其他新闻和播客项目，包括来自本周 cloud bees devo PS World | Jenkins World conference 的[新闻](https://thenewstack.io/jenkins-x-and-the-cd-foundation-a-framework-to-help-developers-in-a-chaotic-cloud-native-world/)，Splice Machine 的[将 SQL 应用于机器学习数据集](/splice-machine-brings-sql-to-machine-learning/)，对开源软件当前状态的[持续担忧](/this-week-in-programming-whither-open-source/)，以及[如何在桌面上运行 Kubernetes](/how-to-install-docker-desktop-with-kubernetes-on-macos/)。

《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了这一集，并得到了《新书库》执行主编约布·杰克逊的协助。

CloudBees 和 Cloud Foundry Foundation 是新堆栈的赞助商。
