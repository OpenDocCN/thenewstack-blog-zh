# 战争故事:“提升和转移”对云本地安全性不起作用

> 原文：<https://thenewstack.io/war-stories-lift-and-shift-does-not-work-for-cloud-native-security/>

帕洛阿尔托网络公司的 Prisma Cloud 赞助了这个播客。

传统的内部安全实践和流程不能直接转移到云本地安全和管理。

在这一集的[新堆栈制造商](/podcasts/makers)播客中，新堆栈的创始人兼发行人 [Alex Williams](/author/alex/) 主持了与 IT 安全公司 [Palo Alto Networks](https://www.paloaltonetworks.com/cloud-security?utm_content=inline-mention) 的客户的圆桌会议，他们分享了关于云原生安全和其他相关主题的经验和见解。嘉宾有 [Brian Cababe](https://www.linkedin.com/in/brian-cababe-8504847) ，IT 咨询公司 [Cognizant](https://www.cognizant.com) 的网络安全、架构和治理总监；[商业仓储地产提供商物联网安全总监泰勒·沃伦](https://www.linkedin.com/in/tylerwarren1/)、安全渗透测试公司 [Cobalt.io](https://cobalt.io/) 信息安全经理[普洛斯](https://www.prologis.com/)和[亚历克斯·琼斯](https://www.linkedin.com/in/alex-jones-0585109)。

向云的转变当然不容易。这需要高水平的专业知识和大量的时间投入。“我见过一些公司因为某种原因在云中进行‘提升和转移’时遇到很多问题，”沃伦说。“如果你走这条路，请确保你的房子井然有序，因为你将在云中复制你在本地的任何问题，一旦你在云中，你将使它变得更加困难和复杂。”

[帕洛阿尔托网络虚拟事件:客户分享他们的战争故事](https://thenewstack.simplecast.com/episodes/palo-alto-networks-virtual-event-customers-share-their-war-stories)

Cababe 说，一些组织可能也看不到他们可能想象的在不久的将来会实现的成本节约和安全优势，因为他们开始转向原生云，但从长远来看，提高安全性和利用其他机会的前景是巨大的。

Cababe 说:“当你开始利用云中的许多服务时，你就真正开始利用节省成本的机会，在安全领域你可以做很多事情。“过去，你在云中的能见度很低。但是现在，我觉得有很多工具可以提供更多的可见性，以至于当我在现场工作时，有时我会对我在那种环境下所能做的事情感到失望。”

但是，尽管目前有数百种云原生安全和其他工具可用，组织也应该尽可能降低复杂性。Warren 说:“如果你有太多的工具，太多的现场闪光灯，安全团队可能会遇到问题。“你可以在云中遇到这个问题。”

因此，限制组织的开发运维所依赖的安全工具的数量是一个合理的策略。“我尽量减少工具、屏幕和提醒的数量，”沃伦说。“许多安全提供商和供应商都有这些可用的工具，所以他们确实接触了许多不同的云。
