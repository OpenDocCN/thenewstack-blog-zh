# 新堆栈上下文第 95 集:AWS Re:Invent，Packet 的 IFX 2019

> 原文：<https://thenewstack.io/the-new-stack-context-episode-95-aws-reinvent-packets-ifx-2019/>

CloudBees 赞助了这个播客。

[上下文第 95 集:AWS Re:Invent，](https://thenewstack.simplecast.com/episodes/context-episode-95-aws-re-invent-ifx)

在新 Stack Context 第 95 集，新 Stack 工作人员讨论了本周在拉斯维加斯举行的两场会议:亚马逊网络服务的 [Re:Invent 2019](https://live.awsevents.com/reinvent/) 和 [Packet](https://www.packet.com/) 的 [IFX 2019](https://ifx.packet.com/) 。TNS 出版商[亚历克斯·威廉姆斯](https://thenewstack.io/author/alex/)主持了这一集，我们这一集的嘉宾是 TNS 欧洲记者 [B .卡梅伦·增益](https://thenewstack.io/author/bruce-gain/)，他参加了两次会议。

今年，AWS 吸引了近 60，000 名开发人员和 IT 专业人员参加其年度用户大会，该公司推出了许多新服务，包括一项托管服务，该服务利用 Fargate 的无服务器模型来管理和供应弹性 Kubernetes 服务(EKS)上的集群。我们讨论组织[现在如何在 EKS 上为集群管理](https://thenewstack.io/why-aws-decided-to-let-developers-manage-kubernetes-with-serverless/)指定计算和其他标准，因为 Fargate 为 Kubernetes pods 管理复杂的配置和基础架构管理。我们讨论这一新产品对开发人员和 IT 运营人员的价值。

这是 Packet 2019 年 IFX 会议的第二年，Packet 每年都与 Re:Invent 一起举办。大会的任务是[庆祝基础设施的工艺和文化](https://thenewstack.io/why-infrastructure-has-become-a-key-differentiator/)，通过两天沉浸式编程，其灵感来自硬件、网络和软件的交叉。我们讨论 Packet 与 AWS 等大型云服务有何不同。

在 [The New Stack Context](https://thenewstack.io/podcasts/context) 播客上，New Stack 的编辑和营销团队回顾了本周云原生技术和大规模应用程序开发方面的最热门新闻，并展望了我们预计在未来几周将获得更多关注的话题。
