# 开发速度的可观察性的重要性

> 原文：<https://thenewstack.io/the-importance-of-observability-for-developer-velocity/>

提高开发者的可观察性是本期 New Stack 播客的重点。特邀嘉宾有 [Zain Asgar](https://www.linkedin.com/in/zasgar) ，他是[新遗迹](http://newrelic.com/?utm_content=inline-mention)开源 Pixie Kubernetes 监控工具的总经理； [Roopak Venkatakrishnan](https://www.linkedin.com/in/roopakv) ，电子商务零售商工具 [Bolt](https://bolt.com/) 的工程经理；[Ihor Dvoretskyi](https://www.linkedin.com/in/idvoretskyi/en),[云原生计算基金会的开发倡导者](https://cncf.io/?utm_content=inline-mention)和[高级解决方案工程师 Christine Wang](https://www.linkedin.com/in/christineywang),[Grafana Labs](https://grafana.com/)。主持人是《新书库》的创始人兼发行人亚历克斯·威廉姆斯。

[如何提高 Kubernetes 对开发者速度的可观察性](https://thenewstack.simplecast.com/episodes/how-to-improve-kubernetes-observability-for-developer-velocity)

Asgar 说，对于开发者来说，可观察性是“开发过程和体验的核心部分”。“我认为，在任何一种生产环境中，甚至在此之前，都应该遵守代码。”

阿斯加尔在 Pixie 为 Pixie 项目工作的主要任务之一，以及自收购 Pixie Labs 以来在 New Relic 的主要任务之一，是实现“即时可见性，以便东西安装后立即点亮，”阿斯加尔说。

Venkatakrishnan 说，目标是让开发变得“超级容易”。

Venkatakrishnan 说:“你希望他们能够更快地调试，并了解他们的应用程序发生了什么。“我们正在考虑的一个大问题是，我们如何确保如果有 bug，他们如何能够真正快速地找到问题。

一旦代码被编写并上传到 Git，开发团队成员应该已经可以立即访问可观察性工具提供的日志、度量和数据。王指出，她“钦佩”Grafana 的客户在生产周期的一开始就依靠它来实现可观察性。

她说，客户正在“随着项目的开发，作为一个一流的组件，而不是事后的想法”实施可观察性。“因此，最复杂的开发人员在产品和代码设计期间都在考虑可观察性，并且明白没有一种放之四海而皆准的方法——他们希望通过将它设计到代码中来密切了解他们的指标，”王解释说。
