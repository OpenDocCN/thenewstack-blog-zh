# HPE 通过收购 Scytale 收购云原生服务认证

> 原文：<https://thenewstack.io/hpe-buys-into-cloud-native-service-authentication-with-scytale-acquisition/>

两家公司周一宣布，为了在新兴的云计算原生计算市场占据重要份额，惠普企业(Hewlett Packard Enterprise)收购了服务认证技术提供商(T2)Scytale(T3)。

Scytale 的工程师是用于身份管理的 [SPIFFE](https://spiffe.io/) (面向所有人的安全生产身份框架)框架以及相关运行时环境[SPIRE](https://spiffe.io/spire/)(sp iffe 运行时环境)的创始贡献者，他们已经向[云本地计算基金会](https://www.cncf.io/)捐赠了两个开源项目。Scytale [成立于 2017 年](/scytale-launches-spiffe-based-service-identity-management/)，提供该软件的企业级版本。

“我们仍将积极致力于 SPIFFE/SPIRE，它们仍将是 CNCF(即 Scytale 联合创始人 Andrew Jessup 在推特上写道。

SPIFFE 和 SPIRE 已经准备好成为云本地计算的基石，云本地计算趋向于分布式[微服务架构](https://thenewstack.io/category/microservices/)，其中许多组件不断地相互通信。因此，服务需要一种机制来验证消息的真实性——这是 SPIFFE 定义的角色，SPIRE 可以强制执行。大多数现有的身份验证系统更多地面向对终端用户进行身份验证，而不是服务，并且它们对于云原生系统的动态、高度可扩展的性质准备不足。

[Scytale 公司首席执行官苏尼尔·詹姆斯为您解读 SPIFFE](https://thenewstack.simplecast.com/episodes/sunil-james-ceo-of-scytale-explains-spiffe-for-you)

SPIFFE 框架是根据零信任系统的原则构建的，在零信任系统中，安全性是基于端点和适当的身份验证，而不是基于某种中央防火墙。“随着越来越多的组织采用混合云、多云、容器和边缘计算，[零信任]的价值变得更加明显。Scytale 联合创始人 [Sunil James](https://www.linkedin.com/in/suniljames1/) 在[一篇关于此次收购的博客文章](https://scytale.io/blog/scytales-next-chapter/)中写道:“投资于【零信任】的组织能够更好地扩展其天文数字般增长的 IT 基础设施，而不会像竞争对手那样受到安全性、合规性和复杂性的困扰。

该规范和软件随后被云原生空间中的许多其他公司使用，[包括 Portshift](/portshift-attempts-more-granular-security-for-what-istio-service-meshes-lack/) 。

根据[公司的新闻公告](https://www.hpe.com/us/en/newsroom/blog-post/2020/02/hpe-acquires-scytale-to-advance-open-secure-edge-to-cloud-strategy.html)，HPE 将继续致力于这两个项目，并将它们用于自己计划的“动态、开放、安全的边缘到云平台”。

HPE 最近推出了自己的基于 Kubernetes 的 HPE 容器平台(T11)，该公司称这是“业界首个面向云原生和分布式非云原生应用的企业级容器平台”对于这个包，它借鉴了另外两家收购的开源数据分析软件提供商 [MapR](https://www.datanami.com/2019/08/05/hpe-acquires-mapr/) 和 [BlueData](https://www.hpe.com/us/en/newsroom/press-release/2018/11/hpe-to-acquire-bluedata.html) 。

交易条款没有披露。
