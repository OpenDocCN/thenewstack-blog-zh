容器是当今新旧应用程序开发和部署的标准。然而，对于容器开发人员来说，内置的安全性并不总是标准。

当涉及到容器映像安全性时，组织需要了解许多变量。了解图像中的内容、图像是如何创建的以及是谁创建的非常重要。

扫描容器中的潜在风险很重要，但更重要的是，开发人员要明白当扫描结果显示出数百个漏洞时，他们可以做什么以及应该做什么。对于开发者来说，使容器映像安全可行且高效的挑战需要人员、过程和技术。

扫描容器寻找潜在漏洞的想法并不新鲜，但是光靠它自己也是不够的。仅仅向开发人员提供一长串检测到的漏洞不足以修复容器内部的问题。我们需要的是一个集成的流程，它可以在部署容器之前帮助降低风险。

将安全性作为集成容器开发过程的一部分有助于加速整体开发，减少在构建容器后解决问题所需的时间。当容器安全性被构建到开发过程中时，开发人员能够更快地发布并减轻生产中的监控负担。

## **创建安全容器映像的三个关键步骤**

在构建容器映像后对其进行扫描不应该是降低潜在风险的第一步安全措施。扫描可以而且应该是集成到开发管道中的更大过程的一部分。开发人员在创建安全容器映像时应该考虑三个关键步骤。

概括地说，这些步骤是:

1.  保护您的代码及其依赖项。利用 Snyk 等工具，进行软件组合分析(SCA)、容器安全和静态应用安全测试(SAST)。通过注册一个[免费 Snyk 账号](https://snyk.co/signup)，可以自动发现并修复 Docker 镜像和开源库中的漏洞。
2.  **从一个可靠的来源建立一个最小的基础映像。**容器图像可以来自任何地方，所以一定要从可信的来源获取，比如 Docker Hub。选择操作系统占用空间最小的映像可以减少攻击面。更进一步说，Docker 和 Snyk 合作的原因之一是让您能够从建议到行动。Docker 桌面中集成的漏洞扫描功能实际上可以为您处理一些基础映像选择的工作。
3.  管理基础映像和代码之间的所有层。通过从最少的基础开始，然后添加工具来定制您的图像，使得以后很容易移除这些工具，只需将它们从 docker 文件中取出。例如，在映像进入生产阶段之前，可以删除构建阶段使用的工具或库。通过 exclude-base 选项，使用 Docker 扫描注释可以轻松扫描中间层。

在创建安全容器映像的三个高级步骤中，每个步骤都存在潜在的漏洞。在依赖关系中，甚至在最小的基础映像中，跨层和容器配置中，都可能(并且经常)发现漏洞。扫描每个阶段的风险，而不仅仅是最后阶段的风险，对于实现更安全的结果至关重要。

这三个高级步骤中的每一个都有许多细微差别和细节，这是一篇短文无法解决的。为了帮助开发团队，新的 Docker 和 Snyk eGuide 提供了可操作的信息，以帮助开发人员确定要修复什么，如何在容器映像和 Dockerfile 的上下文中修复它，以及在哪里集中精力。

提高容器安全性不是一个单一的步骤，甚至也不需要单一的工具。这是一个持续的、多步骤的过程，Docker 和 Snyk 在新的免费资源[中详细介绍了实施的最佳实践。](https://snyk.co/DockerSnykContainerSecurityGuide3)
