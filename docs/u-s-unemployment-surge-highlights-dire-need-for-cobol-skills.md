# 美国失业激增凸显对 COBOL 技能的迫切需求

> 原文：<https://thenewstack.io/u-s-unemployment-surge-highlights-dire-need-for-cobol-skills/>

面对大量的失业保险索赔，全国各地的州计算机系统正面临技术困难。不幸的是，这些系统通常运行在旧的大型机上，并且依赖于一种已经有 60 年历史的编程语言 COBOL。面对数量的急剧增加，这些系统已经失效，需要技术支持。

这种需求变得如此迫切，以至于新泽西州州长菲尔·墨菲和医疗工作者一起，现在正在向 T2 招募具有 COBOL 技能的志愿者。新泽西州有 40 年历史的大型机福利系统被 1，600%的使用量增长所围困，因为在过去的一个月中有超过 371，000 人提出了索赔。

州长专注于解决当前的紧急情况，但愤怒地问为什么这些关键系统依赖 COBOL。没有理由等待验尸。原因是大多数州不断推迟必要的多年项目，这些项目仍然不能完全消除对大多数大学不再教授的编程语言的依赖。

有些东西是古老的，并不意味着它应该被安乐死。事实上，根据正在进行的由 [Micro Focus](https://www.microfocus.com/en-us/home) 和 [Vanson Bourne](https://www.vansonbourne.com/) 对参与 COBOL 系统的 500 多人的[调查](https://www.brighttalk.com/webcast/16907/376805)，最常见的 COBOL 应用方法是现代化，而不是退休。63%的参与者正在更新这些应用程序，36%的参与者正在更换它们。这些应用程序正在更新新功能，重新搭建平台并迁移到云环境，甚至迁移到新的大型机系统。不幸的是，正如在[汤姆的硬件](https://www.tomshardware.com/news/new-jersey-cobol-coders-mainframes-coronavirus)中所解释的，许多州的失业系统并不是已经被重组的遗留系统。

几乎所有(92%)的上述调查都认为他们组织的 COBOL 应用程序是战略性的。许多非关键系统已经被基于现代语言的应用程序所取代，这些语言实际上是教给年轻开发人员的。留下来的东西往往没有动过，因为它们的更换往往需要很多钱和多年的过程。据《纽约时报》报道，去年，纽约州授予印度[塔塔咨询服务公司](https://www.tcs.com)一份价值 5600 万美元的五年期合同，以更新其基于 COBOL 的失业系统。虽然 TCS 可能会雇用美国人来管理这份合同，但它不太可能使用美国人，因为根据最近的 [Dice 调查](https://insights.dice.com/2020-technology-jobs-salaries)，2019 年美国 COBOL 开发人员的平均年收入为 106，194 美元，比前一年高出 10%。

在过去的 10-20 年里，美国的公共卫生系统没有被忽视。为许多州失业系统提供动力的计算机系统也是如此。在这两种情况下，公务员一再提出更新，但从来没有优先考虑。为什么我们需要等待暴雨来修补我们漏水的屋顶？
