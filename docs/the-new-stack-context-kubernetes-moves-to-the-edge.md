# 新的堆栈环境:Kubernetes 走向边缘

> 原文：<https://thenewstack.io/the-new-stack-context-kubernetes-moves-to-the-edge/>

上个月末，Rancher Labs [将其广受欢迎的 K3s Kubernetes 发行版捐赠给了云计算原生计算基金会。这个精简版的 Kubernetes 已经在云本地用户中](/ranchers-k3s-joins-cncf-sandbox-as-first-kubernetes-distribution/)[悄然走红](/k3s-expanded-ranchers-leadership-and-will-be-crucial-for-suses-kubernetes-success/)——许多人正在部署到边缘环境。

因此，在本周的[新堆栈背景](/podcasts/context)播客中，我们邀请了 [Rancher](https://rancher.com/) 联合创始人 [Darren Shepherd](https://twitter.com/ibuildthecloud) 来讨论 Rancher 在云原生生态系统中看到了什么。Rancher 正在被 SUSE 收购，因为交易仍在进行中，Darren 不能发表评论，但他确实谈到了 K3s 和 Kubernetes。

《新书库》*编辑和营销总监[利比·克拉克](/author/libby/)主持这一集，旁边还有 TNS 高级编辑[理查德·麦克马努斯](/author/richard/)和 TNS 执行主编[约阿布·杰克逊](/author/joab/)。*

[第 132 集:牧场主达伦·谢泼德——谁还需要 Kubernetes 运营商？](https://thenewstack.simplecast.com/episodes/episode-132-darren-shepard-of-rancher-who-needs-kubernetes-operators-anyway)

随着 Kubernetes 和 K3s 在边缘环境中的兴起，如快餐店和工业装配线，Shepherd 指出，edge 过去运行“相当简单”的应用程序，如销售点设备。但现在人们正试图在边缘上部署像 Kafka 这样相当复杂的软件。

“他们试图利用开源世界的所有生态系统和所有工具，以便能够部署更复杂的软件。但是部署这种软件的需求变得越来越复杂，所以他们需要一种更好的技术来做到这一点，”Rancher 解释说。
