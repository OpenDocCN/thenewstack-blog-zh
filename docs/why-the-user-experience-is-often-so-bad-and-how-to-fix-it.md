# 为什么用户体验经常这么差，如何修复

> 原文：<https://thenewstack.io/why-the-user-experience-is-often-so-bad-and-how-to-fix-it/>

当应用程序不能正常工作时，最终用户通常是受害最深的。在当今快速部署和“快速失败，经常失败”的世界中，在开发人员认为可行的东西和最终用户实际经历的东西之间，经常会丢失很多东西。

日常生活中的例子数不胜数。它们可能包括仅仅用优步的应用程序选择你希望你的优步司机去哪里接你的困难。或者以花旗银行为例，正如笔者亲身经历的那样，当试图完成基本交易时，错误和故障经常发生。对许多人来说，谷歌地图的界面在几个方面有所欠缺。除了仍有争议的接口之外，众所周知，web 应用程序的小错误也很常见，从典型的 SSL 问题到可能导致应用程序停止的微服务不兼容。

事情不一定是这样的。以人为中心的设计应该是更多的焦点，因为在开发周期中，应用程序的组件库在设计时考虑了更好的一致性和定制。

在本期 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，New Stack 创始人兼发行人 Alex Williams 在蒙特利尔[Node+JS Interactive](https://events19.linuxfoundation.org/events/nodejs-interactive-2019/)期间与 [Optimizely](https://www.optimizely.com/) 的高级 UX 设计师 [Ayesha Mazumdar](https://www.linkedin.com/in/ayeshakmaz) 谈论了需要做些什么，以及为什么以人为中心的设计不仅变得有好看的功能，而且变得有技术要求。

[Ayesha Mazumdar——为什么可访问性这么差，如何解决](https://thenewstack.simplecast.com/episodes/ayesha-mazumdar-why-accessibility-is-so-bad-and-how-to-fix-it)

Mazumdar 说，通常，当开发人员和工程师考虑以人为中心的设计时，有一种“非常明显的倾向，就是为像我们这样的人类设计”。Mazumdar 说:“我认为这是我们所有人潜意识中的受害者，但我认为[以人为中心的设计]正在成为这种技术要求。”“我们意识到，某些用户无法以特定方式与我们构建的应用程序进行交互，因为我们从一开始就没有考虑过这种使用情形。”

Mazumdar 说，主要的问题是一个设计对开发者来说可能是可访问的，但对最终用户来说不一定。例如，开发人员还必须考虑那些有视觉障碍的人，甚至那些网速慢的人。

“可访问性必须继续下去，并出现在开发人员的工作流程中，以便我们可以确保最终产品在各个方面都可以被任何人使用，”Mazumdar 说，“所以我认为，使其成为技术需求包括使其成为开发过程的一部分，”例如，包括使其成为 GitHub 和 Q&A 过程的一部分。“在开发过程中插入可访问性检查可以确保我们在开发周期的任何时候都不会丢失它，”Mazumdar 说。

当然，达到这些目标并不容易，但 Mazumdar 形容她作为 UX 高级工程师的角色“非常适合”她。她将自己的角色描述为涉及组件库设计系统工作、大量原型制作和产品本身的前端工作。“这是设计和工程之间的一种桥梁作用。因此，我的许多工作涉及确保设计方拥有工程所需的一切，并确保工程师在实现特定 UI 时拥有设计或我所需的一切。至于组件库本身，"
