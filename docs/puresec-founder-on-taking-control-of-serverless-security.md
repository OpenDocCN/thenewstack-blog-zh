# PureSec 创始人控制无服务器安全

> 原文：<https://thenewstack.io/puresec-founder-on-taking-control-of-serverless-security/>

[PureSec 创始人谈控制无服务器安全](https://thenewstack.simplecast.com/episodes/puresec-founder-on-taking-control-of-serverless-security)

无服务器的主要优势之一是能够将服务器管理和平台管理相关的任务转移给第三方，从而可以更加专注于开发和部署。但是这种自由是有代价的——例如，依赖无服务器也意味着给第三方更多的控制权。您受无服务器提供商支配的许多事情包括意外停机、价格波动，以及最终无法从针对您的特定需求定制的功能中受益，因为您可能是成千上万客户中的一员。底层安全参数的控制也是一个问题。

然而，使用正确的工具，有可能获得无服务器数据和应用程序的运行时安全性的可见性和控制。无服务器安全提供商 [PureSec](https://www.puresec.io/) 的首席技术官兼联合创始人 Ory Segal 在本月早些时候由 New Stack 的创始人兼主编 Alex Williams 主持的播客中，讨论了如何在今年早些时候的 [ServerlessConf 2018](http://serverlessconf.io/) 上实现这一目标。

Segal 说，无服务器的固有问题是用户不拥有运行时环境，因为“你是客人”。“能够坐在那里，在一个非常低的水平上监控一切……是具有挑战性的，”西格尔说。

但是正如前面提到的，Segal 说，通过正确的监控系统，如 PureSec 提供的监控系统，在运行时获得对无服务器安全的可见性是可行的。“只要我们为您提供库，我们基本上就是将自己编织到环境中。当你在你的代码中调用我们的库时，它会进行某种程度的包装——不是包装在代码中，而是将函数实例封装在我们的安全环境中，以便在非常非常低的级别监控事物，”Segal 说。

一个可靠的无服务器监控系统还必须包括适用于典型内部部署或云部署的功能。每个组织也有其特定的数据保护需求。“没有一种方法可以匹配或适合每种环境和每个客户，”Segal 说。"每个组织都有自己的需求和不同的环境."

但不管怎样，静态分析、扫描、库、认证库和扫描代码仍然很重要，而且“即使我们在谈论无服务器应用程序安全性，我也不会跳过这些东西，”Segal 说。“在部署之前，您需要静态扫描代码以找到长寿者。事后分派东西是没有意义的。”

当谈到无服务器的运行时和监控时，“事情已经发生了变化，”Segal 说。“我想我听到有人提到，在无服务器中，云就像操作系统，然后功能就是代码，所以这是完全不同的，”Segal 说。“它基本上就像一台完全完整的计算机，你需要这样对待它。因此，仅仅监控功能或配置还不足以看到全局并理解这是一个大系统。”
