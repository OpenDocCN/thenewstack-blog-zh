# 泰克顿为 MLOps 带来了实时机器学习

> 原文：<https://thenewstack.io/tecton-brings-real-time-machine-learning-to-mlops/>

用于机器学习(ML)的企业[功能商店](https://www.tecton.ai/blog/what-is-a-feature-store/)Tecton，本周向其功能商店推出了低延迟流媒体管道，使其用户能够构建具有超新鲜功能的实时 ML 应用程序，时间不超过 100 毫秒。

在 ML 中，特征是机器学习模型可以训练并从中推断信号的数据，而特征库是数据和模型之间的接口。要素存储处理各种任务，从转换和清理数据到提供要素服务，再到提供日志记录和监控管道健康状况的功能。

根据采用的复杂性，公司通常会沿着一个连续体采用 ML 管道，从传统分析(如每日仪表板)开始，然后转向分析 ML，这为预测和营销活动背后的批量生成 ML 预测提供了动力。复杂性的下一步是操作 ML，它可以在生产应用程序中支持实时预测，如产品推荐，并且通常是功能存储进入组合的地方。此时，特性存储可以帮助提供生产中使用所需的低延迟和服务级别协议(SLA)。最后一步是实时 ML，它不再仅仅依赖于批量数据，而是需要访问这些由实时和流式数据源构建的“超新鲜”功能。

泰克顿本周正在用其最新的功能解决这一最后的复杂性，它说，这种功能可以在不到一秒钟的时间内自动将流和实时数据(包括时间聚合)转换为 ML 特征。这里的创新不在于这样做的能力，而在于这样做的能力没有一个专门的工程师团队和潜在的几周甚至几个月的准备时间。

“很难找到能够实施和解决这类问题的人才。泰克顿联合创始人兼首席技术官[凯文·斯汤普夫](https://www.linkedin.com/in/kevinstumpf/)解释道:“即使你有这方面的天赋，也需要很长时间来解决这些问题，因为每当数据科学家需要一项新的机器学习功能时，工程师现在就必须花费数周或数月的时间来实际实现它，并使其为生产做好准备。”。“通过我们的系统，我们现在基本上可以为更多的企业提供这些功能。他们不需要雇用昂贵的工程师，并且交付时间大大缩短，因为该平台自动化了所有这些工作，否则工程师将不得不以容易出错的方式在数周或数月内手工实施这些工作。”

Stumpf 和泰克顿背后的其他创始团队非常熟悉手工完成这一切的过程，因为他们之前在优步一起工作过，创建了[米开朗基罗](https://eng.uber.com/michelangelo-machine-learning-platform/)，这是一个机器学习平台，用于在优步的规模上构建、部署和操作机器学习解决方案。他们现在的目标是把同样的能力带给那些不想做同样投资的公司。

通过提供这些“超新鲜”的功能，泰克顿的用户将有能力做一些用 ML 做不到的事情，比如欺诈检测。Stumpf 解释说，欺诈检测需要尽可能最新的功能，如果没有这些功能，它很容易错过重要信号。

“机器学习中最重要的特征之一就是这些时间窗聚合。例如，在过去的 5 分钟或 30 分钟内，某张信用卡完成了多少笔交易？”斯坦普夫说。“如果您有一个欺诈检测应用程序，您需要在那一刻知道，‘截至目前，这张信用卡在过去五分钟内使用了多少笔交易？’不是一分钟前或五分钟前。支持和服务旧的功能要容易得多，因为您可以在后台轻松计算，但正如您可以想象的那样，这还不够好。你真的需要知道在你做预测的时候，到目前为止已经发生了多少笔交易，因为如果你不知道这些信息，你基本上就放弃了一个真正重要的信号，而这个信号是准确预测这笔交易是否是欺诈交易所必需的。"
