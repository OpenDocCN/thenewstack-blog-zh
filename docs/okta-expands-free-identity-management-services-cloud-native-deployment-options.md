# Okta 扩展免费身份管理服务、云原生部署选项

> 原文：<https://thenewstack.io/okta-expands-free-identity-management-services-cloud-native-deployment-options/>

身份管理巨头 [Okta](https://developer.okta.com/?utm_content=inline-mention) 针对企业中日益增长的可扩展性重要性，为开发人员扩展了其免费的认证、授权和用户管理工具层。

[Okta Starter 开发者版](https://developer.okta.com/)现在可以容纳多达 15000 名月活跃用户，高于上一代的 1000 名用户限制。这意味着您的应用程序可以支持更多的用户，因为它正在进行测试和早期部署。该公司还简化了开发人员的界面，增加了对许多新的网络和云原生技术的支持。

Okta 长期以来一直认为身份管理是连接微服务、API 服务和分布式应用的秘密武器。Okta 负责开发者关系的副总裁[约翰·普里查德](https://www.linkedin.com/in/jpritchard/)说:“身份是让这种无缝集成成为可能的东西——所以开发者不会建立点对点的联系。

身份管理实施起来很麻烦，充满了大量容易出错的低级细节——如果搞砸了，真的很危险。该公司认为组织确实应该外包这部分应用程序开发。

“每次我走进一个大组织，发现一个大规模的本土身份团队，我都会问‘你为什么要经历这种痛苦？’“这是作为开发人员最难解决的问题之一，也可能是编写的技术中回报最少的部分，”Pritchard 说每个人都在试图攻击你，攻击面如此之大。"

如今，新的云原生技术的完美风暴和对更多开发人员的迫切需求正在使组织认真考虑最终将身份认证问题转移给可信提供商的想法。尤其是在经济上有意义的情况下。

“我的身份团队需要 26 个人吗？或者，如果我有一个软件作为服务身份提供者，我能创建一个 Scrum 团队来管理它吗？我更加相信专门研究该技术的提供商会做得很好，特别是如果我想与之合作的一批其他集成技术合作伙伴已经与他们进行了集成。”

Okta Starter Developer Edition 提供了一种测试这种外包方法的简单方法。

它为面向客户的应用程序和 API 以及基础架构支持运营提供了大量内置身份管理的自助服务功能。它包括 Okta 的旗舰自适应多因素认证和 API 访问管理服务。

此外，该公司修改了用户控制台，以提高易用性，加快部署速度，并为整个应用程序开发生命周期提供更大的支持。为所有 hip 新开发语言提供开箱即用的支持，包括 Go、Java、JavaScript、Python、VueJS、React Native 和基于 Spring Boot 的应用。

该服务还包括增强的文档、示例应用程序和新的集成，以融入用户的持续集成和持续交付(CI/CD)系统。

除了增强其免费层，该公司还发布了许多不同的集成，以将身份管理引入云原生计算世界。该公司与平台服务提供商 Heroku 合作，简化了将身份管理添加到基于该服务的 CI/CD 平台的流程。与[孔](https://konghq.com/kong/)的合作将身份服务引入公司的旗舰 API 网关。该公司还更新了对 [HashiCorp](https://www.hashicorp.com/?utm_content=inline-mention) 的 [Terraform](https://www.terraform.io/) 的支持，带来了跨环境复制 Okta 配置的能力——非常适合组织以基于代码的部署方式构建基础设施。

该公司将在本周举行的年度用户大会上披露所有这些新的和更新的产品的更多细节。
