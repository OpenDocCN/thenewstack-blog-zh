# 采用 Kubernetes 如何影响开发运维团队的 CI/CD 需求

> 原文：<https://thenewstack.io/codeships-laura-frank-orchestration-developer-culture/>

[Codeship ' s Laura Frank on Orchestration 和开发者文化](https://thenewstack.simplecast.com/episodes/codeships-laura-frank-on-orchestration-and-the-developer-culture)

在容器编排的年轻历史中，将[微服务](/category/microservices/)理念与企业开发和部署管道相融合的最成功实践者之一，是 CI/CD 平台提供商 [Codeship](https://codeship.com/features/pro) 、 [Laura Frank](https://de.linkedin.com/in/laurafrank) 驻柏林的工程总监。但是 Frank 接触的企业越多，她就越能理解当今组织中的开发人员所面临的文化问题，以及像 Kubernetes 开源容器编排引擎这样的平台在多大程度上可能无法催化自身的革命性变化。

“我认为这肯定是一场斗争。在我举办的研讨会上，我看到其他工程师已经习惯了这种‘把它扔过墙’的心态，”弗兰克在最新一期的[新堆栈制造商](https://thenewstack.io/podcasts/makers)播客中接受采访时告诉我们。这次采访是结合我们即将推出的*Kubernetes 生态系统的状态*电子书进行的，其中 [Codeship](https://codeship.com/features/pro) 是赞助商之一。

“他们编写代码，把它扔到墙上，他们不需要运行[*或*]支持它，”弗兰克说。“但是随着工程组织的发展，他们开始使用更多的‘devo PS-y’实践——比如持续集成[*和*]更多的自动化——代码被推倒的那堵墙已经不存在了。工程师必须在操作方面变得更加精明。但这绝对是一个渐进的变化。”
