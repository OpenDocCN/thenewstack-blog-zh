# Forrester: 96%的企业都有云安全问题

> 原文：<https://thenewstack.io/forrester-96-of-businesses-have-had-cloud-security-probs/>

我们都知道我们的云经常受到攻击，但是 Forrester Consulting 在对 154 名云安全决策者的调查中发现，他们分别来自[亚马逊网络服务(AWS)](https://aws.amazon.com/) 和[公共云安全公司 Sonrai Security](http://www.sonraisecurity.com/) 。嗷！

在该研究中，[身份控制是企业云安全计划的核心](https://sonraisecurity.com/resource/identity-control-study/)，其中 98%的攻击涉及与身份相关的安全挑战。根据 Sonrai 的说法，这些问题不在于人们的身份，而在于所有用于运行云应用程序的系统和服务身份。因此，“[云决策者正在努力应对过于复杂的访问控制策略](https://sonraisecurity.com/resource/identity-control-study/)、云平台身份的分散视图以及特权过多的云管理员用户。”

## 非人类身份

这些非人类身份包括机器人、无服务器功能、代码基础设施和计算资源。每一项新技术都带来了新的独特身份，同时也带来了一系列风险。正如埃里克·凯德罗斯基所观察到的，

由于数字化转型，[非个人身份远远多于个人身份](https://sonraisecurity.com/blog/rise-non-human-identities/)，这意味着您的风险状况正在增加，通常是以您不知道的方式和领域。"

该调查的受访者表示同意。超过一半(56%)的人说，不属于个人的身份在云中不受控制。因此，82%的受访者表示，他们预计到 2023 年将投资开发新的身份访问管理(IAM)工具来解决这一问题。74%的人甚至说云迁移需要不同的 IAM 方法。

那么你能做些什么呢？Kedrosky 同意受访者的观点，即机器学习、自动化和 DevOps 集成是解决安全问题的关键。具体到 IAM，他认为要了解公共云的真实风险情况，组织需要的不仅仅是身份本身的背景。“这意味着通过[持续云安全状态管理(CSPM)](https://sonraisecurity.com/cloud-security-platform/platform-cspm/) 和工作负载安全性，将身份与业务数据和整体平台风险联系起来。为了真正有效，它需要使用智能[工作流和自动化](https://sonraisecurity.com/cloud-security-platform/workflow-automation/)进行部署，因此它以云的速度和规模移动。

## 其他痛点

虽然身份是真正的云安全痛点，但它不是唯一的一个。其他包括:

*   超过一半的受访者是针对其云的内部事件的受害者。
*   49%的人表示，他们遭受过涉及商业伙伴或第三方供应商的攻击。
*   另有 49%的人报告由于云配置不当而丢失数据
*   而 49%的受访者表示不得不应对外部攻击。

在这些具体的安全问题背后是更普遍的问题。其中包括与云整合不好的遗留工具，45%；过于复杂的访问控制策略，40%；监管合规问题，40%；而过度特权用户，40%。

总而言之说实话，云安全是一个烂摊子。真的，难怪几乎所有人都有过云安全问题。

这一切的背后是一个简单的事实，云管理员和开发人员几乎没有云安全技能。在他们获得管理云级安全问题的自动化工具所需的专业知识之前，我们将始终面临重大的云安全问题。
