# 演示:牧场主 2.2 反映了库贝内特斯不断变化的动态

> 原文：<https://thenewstack.io/demo-rancher-2-2-reflects-the-ever-changing-dynamics-of-kubernetes/>

2017 年变得很明显，Rancher 背后的人需要将他们的软件作为关键的容器编排平台。当时，[联合创始人兼营销和销售副总裁 Shannon Williams](https://www.linkedin.com/in/smw355/),[Rancher Labs](https://rancher.com/)告诉我们，Kubernetes“比我们当时自己的编排功能更全……我们当时决定把一切都押在 Kubernetes 的 Rancher 2.0 上。”

现在，在 2019 年，随着 [Rancher 2.2 的发布，Rancher 更加拥抱 Kubernetes。](https://rancher.com/blog/2019/rancher-2-2-hits-ga-milestone/)“我们只是在说 2015 年发生了怎样的变化——但也许真的一点都没有。威廉姆斯说:“每天你都会发现新的东西，然而，从很多方面来说，它们只是我们以前已经做过的东西的抽象，但它们正在被整合，更容易管理。”。“与 Kubernetes、containers 和 Docker 一起工作是一段有趣的时光。但在过去的四年里，这很有趣——简直是一场爆炸。”

Williams 说，去年，Rancher 每天为数万个集群进行大约 8000 次活跃部署。现在，这个数字已经激增到 25000 人，比去年增长了 200%以上。“所以这是非常非常疯狂的一年——有点像库伯内特的增长，”威廉姆斯说。

在这个演示中，Shannon Williams 和 Rancher 的技术营销总监 Adrian Goins 展示并讨论了带有附加集群的 Rancher 2.2 的运行环境。威廉姆斯说，集群被旋转起来，各种各样的工作负载被创建，以显示平台的运营商和 Kubernetes 的用户将如何与它交互。

虽然 Williams 说可以在 Rancher 的网站上找到所有信息，但他在演示开始时展示了当任何人访问 Rancher 时，访问控制是如何执行的。管理员可以看到他们管理的所有集群，而用户只能看到他们有权访问的集群。

Rancher 以其用户界面(UI)而闻名。威廉姆斯说，对于管理员来说，这意味着使用用户界面来访问“全球范围内发生的所有事情”。“我们正在管理广泛部署的应用程序、用户访问策略、安全设置和生命周期，这是人们谈论的两个在集群级别处理的东西，”Williams 说。威廉姆斯说，用户的大部分时间实际上都花在了他们可以访问的集群上。在基础设施方面，Goins 描述了 Rancher 和 Kubernetes 如何“让我做过去需要交换机和路由器的事情——这是物理基础设施的另一层。”

戈因斯说:“现在，只要把东西送上来，然后到处移动就没什么了——便携性太棒了。”。“我们能够做事情的速度是惊人的，无论是在原型制作，测试，带来设备或带来容器图像，测试新的东西，还是将东西投入生产，在提供商之间上下移动 Kubernetes 提供的灵活性打开了我们以前无法进入的大门，”Goins 说。

威廉姆斯说，牧场主希望采取的最终方向是“在 Kubernetes 是什么方面保持灵活”。“Kubernetes 将变得非常标准化，”Williams 说，并指出 Rancher 的客户在各种基础设施上运行 Kubernetes，除了云之外，还包括虚拟机和裸机服务器。

“我们希望在如何部署这些集群方面非常灵活，然后围绕它们增加很多价值，以便人们可以用它做很多事情，”威廉姆斯说。

Alex Williams，The New Stack 的创始人兼主编，主持了这次演示。
