# 网络性能:没有行动的可见性是没有用的

> 原文：<https://thenewstack.io/cloud-optimized-networking-happening-teridion/>

随着基于云的应用程序的使用持续增长，公司发现他们必须使用一系列网络和应用程序监控即服务平台和个人工具来监控其堆栈的所有级别。然而，将混合应用于大规模运行的网络应用程序会很快导致混乱。

这就是泰瑞迪翁大放异彩的地方。在面向互联网的运营中，如[数字海洋](https://www.digitalocean.com/)，Teridion 利用云虚拟路由器做出基于性能的路由决策，帮助确保其客户的应用程序可以无延迟地访问。

历史上，如此大的外部网络流量是通过内容交付网络(CDN)来管理的。[cdn 有其局限性](https://thenewstack.io/fight-latency-killer-apps/)，因此 Teridion 希望通过其云优化虚拟主干技术，使用软件定义网络(SDN)、网络功能虚拟化(NFV)以及其他先进的网络技术，进一步简化会话交付。

在本期[新堆栈分析师](https://thenewstack.io/podcasts/)中，新堆栈创始人亚历克斯·威廉姆斯采访了 Teridion 首席营销官[戴夫·金斯伯格](https://www.linkedin.com/in/david-ginsburg-4774904)和 451 研究分析师吉姆·戴维斯，以了解更多关于 Teridion 的方式。

[#91:网络性能上的 Teridion 没有行动的可见性是相当无用的](https://thenewstack.simplecast.com/episodes/91-teridion-on-network-performance-visibility-without-action-is-pretty-useless)

当谈到当今存在的一些网络监控解决方案的流行词汇时，金斯伯格并不讳言。

“没有行动的可见性是没有用的。我们采取了一种不同的方法，将事情循环到云层面，因为那里是您的服务所在的位置，也是您的应用程序和数据库交付的位置。他们不仅会告诉你全球互联网出了什么问题，或者出了什么问题，在我们的案例中，我们通过虚拟路由器来告诉你，这将允许你根据这一情报采取行动。”

软件定义的网络还带来了第二个好处:额外的安全控制。

“我们正在寻找使用 cdn 和覆盖网络的不同方法来处理作为性能问题一部分的延迟。有时，这些技术中的一些也能在安全方面帮助我们。随着 DDOS 和 web 应用防火墙保护等服务的加入，我们将看到该市场正在以许多不同的方式发展。在企业层面，你会看到云正在影响网络资源的分配，”金斯伯格指出。

泰瑞迪翁和[数字海洋](https://www.digitalocean.com/)是新堆栈的赞助商。
