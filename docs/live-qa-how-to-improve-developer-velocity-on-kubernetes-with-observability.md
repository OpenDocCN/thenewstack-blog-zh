# 现场问答:如何用可观察性提高 Kubernetes 上的开发速度

> 原文：<https://thenewstack.io/live-qa-how-to-improve-developer-velocity-on-kubernetes-with-observability/>

Kubernetes 上的开发者体验正在慢慢改善，但仍远非理想。尤其是监控和调试应用程序非常耗时，开发人员通常依赖定制的仪表盘和代码工具。开发人员可能很难获得正确的指标来帮助他们解决变更问题并提高性能。

去年 12 月被 [New Relic](http://newrelic.com/?utm_content=inline-mention) 收购的初创公司 Pixie Labs ，开发软件工具，让 Kubernetes 上的开发者更容易观察到，也更有用。开源 Pixie 使用 [eBPF](/ebpf-put-the-kubernetes-data-plane-in-the-kernel/) 从堆栈的所有层(应用层、基础设施层和内核层)收集数据，并将其与 Kubernetes 元数据配对，以帮助查明集群中的任何性能问题。开发人员体验的核心是使用社区构建的开源脚本或通过现有的可观察性工具来访问和分析数据的能力。对开发人员的承诺是能够通过一个 CLI 命令访问可操作的应用程序性能数据。

在 Kubernetes 上提高开发速度的最好方法是什么？可观察性如何适应？太平洋时间 4 月 29 日星期四上午 11 点加入新的堆栈，我们将超越面糊，进入与我们的赞助商 New Relic 的实时 Pancake & Podcast 问答，了解开发人员如何充分利用 Kubernetes 上的 observability 平台。
