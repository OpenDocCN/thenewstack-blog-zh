# HPE 正转向集装箱和 Stackato 的微服务

> 原文：<https://thenewstack.io/hpe-empowers-enterprise-developer-part-2-2/>

谈到云原生开发，HPE 身体力行，最近承担了将一组整体信息服务应用程序迁移到基于微服务的方法的任务。

“在许多情况下，我们实际上使用了[staccato](http://www8.hp.com/us/en/cloud/helion-devplatform-overview.html)、 [Cloud Foundry](https://www.cloudfoundry.org/) 和 [Docker](http://docker.com/) 作为支撑这些转变的架构原则。HPE 产品和服务副总裁 Omri Gazitt 在 New Stack 创始人 Alex Williams 和技术编辑 Benjamin Ball 关于最新一集 [The New Stack Analysts](/tag/the-new-stack-analysts/) 播客的两部分采访的第二部分中解释道:[我们非常喜欢 Stackato 的一点是将这些生态系统结合在一起的想法。](https://twitter.com/omrig)

由于使用了 Stackato，HPE 能够为使用 Docker 容器的开发人员提供更强大的解决方案。“通过教授我们的平台 Stackato 如何基本上获取和运行这些容器，并能够将所有 PaaS 功能(如集成日志记录和监控、自动扩展、应用程序版本控制等)引入 Docker 堆栈，效果非常好，”Gazitt 指出。

点击这里收听播客:

[#84: HPE 正转向使用集装箱和堆垛机的微服务](https://thenewstack.simplecast.com/episodes/84-hpe-is-moving-to-microservices-with-containers-and-stackato)

虚拟化和 API 管理也是 HPE 正在内部解决的技术。这两者都使该公司能够将其数十个 API 打包成可供开发者使用的第三方解决方案。

对于那些对改造 CI/CD(持续集成/持续部署)工作流程感兴趣的人，HPE 现在提供其 StormRunner 软件作为 SaaS。

“它在 AWS 运行的每个地区运行，因此您基本上可以在该地区或跨地区进行模拟负载测试。它还与我们基于微服务的应用开发平台相结合，使其成为我们 CI/CD 管道的一部分。

CI 和 CD 是 HPE 认为的云原生开发的基石，企业需要更好的工具来简化他们的工作流程。

Gazitt 解释说，最终，过渡到云原生工作并不完全是关于目的地，而是关于 IT 团队将沿着这条道路走下去的旅程。“教他们(开发人员)如何构建云原生应用，并向他们展示如何将这些应用与他们的一些传统资产联系起来——我们将这种转变视为一次真正的旅程。”
