基于 OpenStack 和 [Kubernetes](/category/kubernetes/) 的应用将越来越多地使用容器开发，但这些平台将定义开发如何与基础设施本身集成。

Kubernetes 正迅速成为 OpenStack 的主流编排平台。两年前，在巴黎 OpenStack 大会上，从容器的角度来看，Docker 是人们谈论的焦点。但现在不同了。话题已经转向了编排，Kubernetes 是这场讨论的中心。

Mirantis [正在使用 Kubernetes 和 Docker 容器构建一个更加模块化的 OpenStack 应用版本](https://thenewstack.io/mirantis-containerizes-openstack-to-ease-operational-challenges/)。Kubernetes 将在 OpenStack 上运行。

Mirantis 的 Alex Freedland 在我们的煎饼早餐会上说，集装箱将改变基础设施，Kubernetes 是最具社区动力的。基础设施将被“码头化”,每一部分都将通过编排交付。包括 OpenStack 在内的一切都将通过 Kubernetes 进行编排。

界线会变得模糊。两种方法都将被采用。谷歌的 Craig McLuckie 在早餐会上说，在 OpenStack 上运行 Kubernetes 是一个很好的起点。

OpenStack 拥有 Kubernetes 没有的功能。例如，OpenStack 支持对象存储——这是 Kubernetes 所缺乏的。正如我们在 [Kubernetes 1.3](https://thenewstack.io/kubernetes-1-3-supports-stateful-applications-federated-clusters/) 中看到的，Kubernetes 正变得更加面向企业。

仍然有很大的空间让编排平台更适合生产。容器生态系统中没有一个平台占据主导地位。Apache Mesos 可以被视为 T2 Kubernetes T3 的竞争对手，每天都在 Twitter 和 Airbnb 等公司的生产环境中使用。

[码头工人](https://www.mirantis.com/software/docker/kubernetes/)是新堆栈的赞助商。
