# OpsRamp 演示:用于混合基础设施管理的 AIOps 平台

> 原文：<https://thenewstack.io/opsramp-demo-an-aiops-platform-for-hybrid-infrastructure-management/>

在本演示中，我们直接看到了如何创建 [OpsRamp](https://www.opsramp.com/) 作为 AIOps 平台，以监控和分析该公司所描述的复杂且通常完全不同的混合基础架构。这些基础架构通常可能跨越多个数据云环境或容器或整体平台上的数据中心，OpsRamp 将其压缩到单个控制台界面中。

带领我们进行这次演示之旅的是 OpsRamp 的高级营销总监[乔丹·谢尔](https://www.linkedin.com/in/jordansher)和 OpsRamp 解决方案顾问[科特·索林](https://www.linkedin.com/in/curtisthorin/)。

该演示旨在模拟典型组织中生成的事件量，每周总计 43，000 个警报。“对于一个 IT 团队来说，这种[数据量]难以管理。当你(通过 OpsRamp)提出这个问题时，它大大减少了他们需要处理的原始警报数量，从而大大减少了他们需要上报的事件数量，”谢尔说。“它通过创建这些关联警报的推断简化了他们的生活，因此企业不再需要手动操作。”

在演示中，OpsRamp 的仪表板显示了人工智能如何处理来自混合基础设施网络的近 200 万个原始事件。索林说，这个想法是通过“理解我们过滤的噪音”来确定问题区域。分析显示，在近 200 万个事件中——这主要归功于人工智能分析——只有 70 万个警报被生成。索林说，总共产生了 10，000 个需要解决的问题，但“我们已经优化了其中的 92%”。

Thorin 展示了在可能出现内存泄漏或 CPU 使用率激增的情况下，OpsRamp 如何“几乎实时”显示性能。Thorin 说:“然后我可以深入研究，开始了解我们是如何从 ops 的运营基础设施中模拟这项服务的。在一个实例中，Thorin 展示了如何在数据库中检测到问题，更确切地说，是在特定的 SQL 集群上检测到问题。

当然，任何工具或平台的目标都是最终使组织的最终用户受益。在非常基础的层面上。例如，在在线零售领域，OpsRamp 通过在软件故障发生时自动处理和交流软件故障，充当了最终用户可能遇到的麻烦的直接渠道。例如，在特定的在线零售交易中，在线客户可能会遇到持续几秒钟的延迟，Thorin 展示了如何将信息快速传达给正确的 DevOps 团队以采取行动。谢尔说，这一功能展示了该平台是如何从混合企业对这种以服务为中心的应用平台的需求中诞生的。

“我们公司是由那些在各种不同组织中为大客户提供服务的人组成的，”谢尔说。
