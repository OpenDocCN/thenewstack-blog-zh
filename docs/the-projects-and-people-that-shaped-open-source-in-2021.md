# 2021 年塑造开源的项目和人物

> 原文：<https://thenewstack.io/the-projects-and-people-that-shaped-open-source-in-2021/>

开源项目生产一些最有价值和最复杂的软件，也可以降低公司的信息技术成本。但是，尽管今年公司投资的增加加速了开源软件的发展，但它在社区和商业伙伴关系中也面临着挑战。

根据 [Red Hat 的 2021 年企业开源状况报告](https://www.redhat.com/en/enterprise-open-source-report/2021)，90%的 IT 领导者正在使用企业开源，79%的人预计他们在未来两年内将增加对新兴技术的企业开源软件的使用。随着大多数企业使用某种形式的开源，从消费到生产和拥抱开源，仍然存在不同的成熟度。

从新工具和不断增加的贡献者活动，到重新定义开源含义的社区和领导标准，这些都是今年形成的开源人和项目的主要故事。

让我们开始吧。

## **2021 年顶级开源故事:**

在 Linux 30 周年之际，Tag1 Consulting 的创始合伙人兼首席执行官杰里米·安德鲁斯(Jeremy Andrews)在一次由两部分组成的电子邮件采访中，与 Linux 和 Git 创始人莱纳斯·托瓦尔兹(Linus Torvalds)分享了他对与他密切相关的话题的想法。看看他的观点，他问企业用户从开源项目中获利，却没有为那些努力谋生的实际维护者做出贡献或提供支持。

当自由软件基金会(FSF)再次选举理查德·斯托尔曼进入董事会时，这成了一个有争议的决定。被一些软件行业的人尊为自由软件的教父，斯托曼因发起了 [GNU 项目](https://www.gnu.org/software/software.en.html)而闻名，这是一个完全由自由软件组成的 UNIX 兼容操作系统。它的诞生被认为是现代自由软件运动和[开源软件许可](https://thenewstack.io/a-guide-to-leveraging-open-source-licensing/)的 copyleft 品牌诞生的时刻。这篇文章记录了机构群体对他再次当选的回应，以及开源世界为改善治理、包容性和开放性而必须采取的步骤。

如果开源项目被公司用来赚钱，为什么不应该以某种方式向创造它们的人支付报酬？在新的“工程师的企业家精神”专栏中，New Stack 的 Emily Omier 从一群创建软件公司的创始人的角度讲述了开源商业战略。这篇文章探讨了 HashiCorp、Hasura、Red Hat 和 MongoDB 等公司商业模式背后的决策。

GitHub 今年早些时候推出的 Copilot 不仅能自动完成代码行，还能根据你输入的代码和自然语言提供完整的代码块。这一新功能引发了关于版权问题的质疑，指向病毒式 [GPL 许可证](https://www.gnu.org/licenses/gpl-3.0.en.html)的想法，它要求所有衍生作品都带有相同的许可证。阅读这个故事，看看整个社区的反应。

亚马逊网络服务和 Elastic 在过去的几年里卷入了冲突，因为这两家公司在商业开源和专有软件的性质上争论不休。亚马逊的声明遭到了 Elastic 的回应，使得 OpenSearch 社区陷入混乱。这个故事探讨了供应商之间的关系，以及他们在平衡开源技术的可持续和可行的生态系统方面的最新举措。
