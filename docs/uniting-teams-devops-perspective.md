# 从 DevOps 的角度团结团队

> 原文：<https://thenewstack.io/uniting-teams-devops-perspective/>

在本期[新堆栈分析师](https://thenewstack.io/podcasts/)中，我们将探讨如何将现有的网络、安全和存储模式应用到新的集装箱化环境中。

思科云基础设施服务首席技术官 [Ken Owens](https://twitter.com/kenowens12) 讨论了如何在容器化工作流中采用以开发运维为中心的思维模式，从而进一步改善容器网络、安全性和整体存储。进行这次采访是为了准备我们新发布的第四本[围绕 Docker 容器网络、安全和存储的电子书。](https://thenewstack.io/networking-security-storage-docker-containers/)在 2016 年 DockerCon 展会上，New Stack 创始人 Alex Williams 和联合主持人 [Lee Calcote](https://twitter.com/lcalcote) 与 Owens 交谈。

[#108:以 DevOps 视角团结团队](https://thenewstack.simplecast.com/episodes/108-uniting-teams-with-a-devops-perspective)

采访也可以在 [YouTube](https://www.youtube.com/watch?v=l_hAAsdVz40) 上听到。

虽然开发人员已经将容器作为他们日常工作流程的一部分，但是当今企业 IT 团队的一些成员可能会发现他们正在努力将它们付诸实践。然而，Owens 解释说，将容器引入一个人的网络和安全工作流，再加上 DevOps 专注的心态，可以支持一个团队。“从某种意义上说，网络和安全团队越来越像软件开发人员。这是他们正在走向的 DevOps 模式。如果你今天是一个网络或安全人员，你不了解 DevOps。你可能想花些时间了解一下“什么是 DevOps？””欧文斯说道。

Owens 解释说，DevOps 的核心是自动化。通过自动化工作流程的各个部分，网络和安全团队成员可以开始像软件开发人员一样思考。“下一个思路将是，‘我如何浏览这个容器模型’，以及我如何将我对网络和 SDN 的理解以类似的方式应用于容器？”欧文斯说。一些人将容器编排和调度工具视为这些问题的答案，而 Owens 强调，今天的许多现有平台都不是为今天高度可扩展的基础设施的需求而构建的。

Owens 指出，最终，网络和安全团队将以最适合他们的方式工作。无论是通过 onboarding 容器、以更加以 DevOps 为中心的方式思考，还是实现工作流自动化。“开发人员将按照他们开发的方式进行开发。他们不会改变。他们可能喜欢围棋，并开发围棋。如果我对他们说，“这里有 [Mantl](https://thenewstack.io/internet-of-happy-things-pancake-breakfast/) ，我们想让你用 Python 编码，”他们会说‘去你的，我不会因为你用了 Python 就改变我的编码方式。’"
