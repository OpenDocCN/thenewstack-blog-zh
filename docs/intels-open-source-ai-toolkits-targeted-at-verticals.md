# 英特尔面向垂直行业的开源人工智能工具包

> 原文：<https://thenewstack.io/intels-open-source-ai-toolkits-targeted-at-verticals/>

英特尔迈出了一小步，重启了长达数十年的成功从软件中赚钱的计划。

这家芯片制造商的新机器学习工具包直接针对特定的垂直行业，作为快速部署人工智能应用的一站式商店。

这些工具包基于开源工具，面向制造业、公用事业、医疗保健和其他行业。

## OneAPI

该版本包括可执行模型和硬件级指令，可加速人工智能训练和推理。这些工具封装在[英特尔的 OneAPI](https://www.intel.com/content/www/us/en/developer/tools/oneapi/overview.html#gs.5ju2e0) 并行编程框架中，该框架在后台工作，以在包括 CPU、GPU 和 AI 加速器的服务器环境中提取最大马力。

这家芯片制造商一直在采取措施授权 OneAPI，该 API 被宣传为开源工具包。虽然英特尔宣传新的工具包是开源的，但它们已经专门针对使用英特尔芯片的服务器环境进行了调整。

这些工具包更像是垂直行业的赠品，最终可能会采用英特尔进行部署，而不是在内部完成。英特尔主导着服务器 CPU 市场，也提供人工智能芯片，包括[高迪](https://www.intel.com/content/www/us/en/newsroom/news/vision-2022-habana-gaudi2-greco.html#gs.5jufmo)加速器、图形处理器和现场可编程门阵列。

## SaaS 战略

英特尔正试图复制英伟达在 [CUDA](https://en.wikipedia.org/wiki/CUDA) 上取得的巨大成功，该产品最初是一套编程工具，但演变成了针对特定垂直行业的随时可部署的人工智能程序。英伟达的 CUDA 工具针对的是使用其 GPU 的系统，而英特尔的 OneAPI 适用于所有主要品牌的芯片。

软件已经成为掩盖人工智能计算复杂性的焦点，人工智能计算需要多个芯片和不同的硬件来执行。随着英特尔扩展其软件即服务战略，它也在收购 SaaS 公司。

英特尔上个月[收购了 Codeplay 软件](https://thenewstack.io/intel-grabs-codeplay-as-it-repositions-for-a-diverse-chip-future/)，该软件提供工具、运行时和执行模型，因此标准 C++代码可以适用于跨 CPU、GPU 和其他处理器的并发执行。Codeplay 非常适合 OneAPI 并行编程框架。

英特尔还在其持续收购 SaaS 公司的热潮中收购了 grate 和 Linutronix。

## 工具包功能

与埃森哲合作开发的人工智能工具包包括针对英特尔硬件和软件的优化。公共设施资产健康包括 1000 万个数据点和英特尔优化库，用于监控现场资产，帮助高效分配能源。根据英特尔的一份声明，该系统使用英特尔的人工智能分析工具包进行训练，“优化的训练和推理速度比标准的埃森哲工具包快 20%和 55%”。

其他工具包包括智能文档索引，通过使用人工智能对文档进行分类和归类，为结构化和非结构化数据添加更多上下文。芯片制造商表示，“与埃森哲的智能文档索引工具包相比，这些工具将数据预处理、训练和推理时间提高了 46%、96%和 60%。

英特尔的 [OpenVINO](https://www.intel.com/content/www/us/en/developer/tools/openvino-toolkit/overview.html) 运行时也是新推理工具包中的一个关键组件。

随着 CPU 发货面临延迟，英特尔的软件开发工作正在加速。英特尔承认已经推迟了其[蓝宝石芯片](https://en.wikipedia.org/wiki/Sapphire_Rapids)的发货。这一推迟正值英特尔计划在美国和欧洲建立新工厂的大规模半导体扩张计划。
