# 新堆栈分析 2016 总结:Docker、OpenStack 和开源

> 原文：<https://thenewstack.io/new-stack-analysts-2016-year-end-wrap-discussing-docker-openstack-open-source/>

本周的[新堆栈分析师](https://thenewstack.io/podcasts/Analysts)是我们的年终总结，TNS 员工回顾了 2016 年的可扩展技术。研究分析师 [Lawrence Hecht](https://www.linkedin.com/in/lawrence-hecht-2179301) ，资深记者 [Scott Fulton](https://www.linkedin.com/in/scottmfultoniii) ，技术编辑 [Benjamin Ball](https://www.linkedin.com/in/benballjr) ，执行编辑 [Joab Jackson](https://www.linkedin.com/in/joab-jackson-b6b108) 和创始人 [Alex Williams](https://www.linkedin.com/in/alexwilliams2) 讨论了过去一年中最大的三个话题:Docker fork，OpenStack(和 Kubernetes)的未来，以及开源对企业的影响。TNS 团队还提供了一些关于 2017 年可能会发生什么的预测。

### 1.码头叉到底怎么了？

去年 8 月，谣言围绕着一个想要分叉 Docker 代码库的小组，担心将 Docker Swarm 编排引擎纳入核心 Docker 运行时。

对许多人来说，这是码头工人为集装箱化的未来划了一条线。“在那之前，Docker，Inc .有一个政策，它叫‘包括电池，但可更换’，这个想法是，‘我们给你我们的建议，什么最适合我们的容器堆栈，但请随意使用你想用的东西，’嫁接把它放在一个更难更换它的地方。富尔顿说:“它变成了‘包括电池，但还过得去’。

### 2.OpenStack(和 Kubernetes)的未来

惠普出售其 OpenStack 资产和思科关闭其基于 OpenStack 的 Intercloud 的最新消息只会增加对 OpenStack 私有云软件在市场上失去势头的担忧。除了这些担忧之外，还有开源容器编排引擎 Kubernetes 的出现，它可以执行 OpenStack 在企业中的部分职责，尽管不是全部。

Williams 指出，驱使人们远离 OpenStack 的一个因素是它的整体复杂性。“我认为时间刚刚过去，现在我们处于一个新的时代，”威廉姆斯说。杰克逊后来指出，OpenStack 更适合服务提供商，尽管 Hecht 补充说，今天的开发人员将希望学习如何使用 Kubernetes 和微服务，而不是 OpenStack。富尔顿试图假定 OpenStack 是无辜的，他说，“我认为 OpenStack 今年的一些不幸不一定是 OpenStack 的错。”

### 3.开源在企业中日益增长的影响力

讨论的最后一个话题是开源软件的未来。Ball 向专家组提出了一个问题，即随着软件基金会影响力的增长，开源项目的管理在过去几年中发生了怎样的变化。富尔顿很快提出了对 Linux 基金会等组织的积极好处，指出它“在如何集体组织和达成某种类型的共同议程方面，提供了比往年更积极的经验。Apache 基金会更多的是为软件和单个项目提供支持结构，而 Linux 基金会更多的是交流。让每个人排队讨论需要做什么。”
