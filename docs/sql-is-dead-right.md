# SQL 死了吧？

> 原文：<https://thenewstack.io/sql-is-dead-right/>

它可能已经有 46 年的历史了，但是 SQL 仍然完成了它的工作。

2020 年，SQL 开始思考自己近半个世纪的统治地位。这是一个坚实、相对容易使用(当然，非常熟悉)、可靠的解决方案，只是完成工作。

正如大量了解并热爱 SQL 的分析师和数据库开发人员将会证明的那样，SQL 已经成为该领域的一个开拓者。毕竟，它是关系数据库的语言，提供了驱动每一个大企业的解决方案，并且已经这样做了几十年。

## 经受住时间和弹性的考验

SQL 工具生态系统包括从 Excel 和 Tableau 到 Spark SQL 的所有工具，被超过 60%的组织使用。令人印象深刻的壮举，尤其是考虑到它的年龄。

但并不总是这样。几年前，SQL 被遗弃了。

当时的原因很多。公司认为它无法扩展。它无法满足数据、存储和分析的需求。这导致了当时叛逆的 NoSQL 数据库的兴起，它是作为非关系数据库对关系数据库缺点的解毒剂而开发的。如今，NoSQL 拥有近 40%的市场份额。

NoSQL 数据库现在支持来自 Amazon、SAP、Oracle 和 Microsoft 的 SQL 和解决方案，并获得了巨大的吸引力。事实上，SQL 是 Oracle 和 SQL Server 如此具有粘性的原因之一，也是它们成功的秘诀。没有专有的查询语言能像 SQL 一样清晰明了地表达应该包含、关联和排除哪些数据。也没有其他语言像它一样受到各种不同工具、操作系统、连接器和编程语言堆栈的广泛支持。近 50 年的企业专用实践证明了这一点。

事实上，SQL 被证明是有弹性的，即使 JSON 和 sharding 获得如此迅速的流行，以至于新的数据库供应商很快放弃了 SQL，主要是因为它们不成熟和缺少像 joins 这样的东西。但是很快就变得很明显，SQL 不能被轻易抛弃，因为客户开始遇到奇怪的筒仓，并且需要从事大量繁重的工作以确保与其他工具的集成。许多组织失去了迁移前拥有的功能。与此同时，SQL 也在不断发展。完全支持 SQL 的新 SQL 数据库出现了，JSON 数据类型也加入了，扩展 SQL 的新方法也开始出现。

## SQL 仍然有很大的潜力

SQL 的潜力随着需求的变化而变化，越来越多的组织意识到 SQL 可以提供完成工作所需的可靠性。在谷歌 2017 年发布的一篇论文中，这家科技巨头解释了为什么要将其数据管理系统 Spanner 做成 SQL 系统。原因很简单——它需要一种健壮的查询语言来帮助绕过 OLTP 应用程序开发人员面临的许多困难。

如今，SQL 凭借其强大的功能保持着领先地位，但重要的是，SQL 要继续通过 [SQL++](https://www.couchbase.com/sql-plus-plus-for-sql-users) 等努力来适应现代世界的需求。如果没有别的原因，就让那些精通 SQL、Tableau 和 Excel 的分析师去做比学习多种基于 JavaScript 的查询语言更好的事情吧。他们应该能够将他们的技能用于现代数据库，并且随着 SQL 的不断发展，他们很可能会这样做。
