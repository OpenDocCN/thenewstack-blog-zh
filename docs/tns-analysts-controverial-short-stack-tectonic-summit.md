# 2015 年建筑峰会上的一堆争议

> 原文：<https://thenewstack.io/tns-analysts-controverial-short-stack-tectonic-summit/>

第二天的【2015 年构造峰会在纽约市短栈拉开帷幕(三个品种！)以及由《新堆栈》的执行主编[约阿布·杰克森](https://thenewstack.io/author/joab/)促成的小组讨论，并得到了《新堆栈》的技术编辑&制片人[本杰明·鲍尔](https://twitter.com/benballjr)和撰稿人[劳伦斯·赫特](https://thenewstack.io/author/lawrence-hecht/)的支持。小组:

*   [Gabriel Monroy](https://www.linkedin.com/in/gabrielmonroy) ，发动机厂首席技术官，Deis 的创始人
*   [Matthew Brender](https://www.linkedin.com/in/mjbrender) ，英特尔公司软件定义的基础设施团队的开发倡导者
*   CoreOS 公司的创始人兼首席执行官 Alex Polvi 。

在开始时，Polvi 承诺，作为对那些早早参加的人的奖励，该小组将使事情“尽可能有争议”，这种可能性因众多主题而增加——容器安全性、企业采用曲线、 [DevOps 作为工作描述](https://thenewstack.io/tns-research-devops-factions-software-development-world/)、如何与 AWS 竞争、与 Kubernetes 的预期锁定以及它将使哪些开源项目变得多余——在这个轻松的会议期间，这些主题在热烈的气氛中出现。

这一期的新 Stack Analysts 播客记录了所有争议。
