# 开源领导者:Andrew Wafaa 将 ARM 引入 OpenSUSE

> 原文：<https://thenewstack.io/open-source-leaders-andrew-wafaa-bridges-arm-opensuse/>

下面的采访是“开源领导者”系列的一部分，在这个系列中，我们描述了开源 IT 社区中的项目领导者，以了解他们如何开发他们的软件，以及运行开源项目带来的挑战和好处。

[Andrew Wafaa](https://github.com/awafaa) 是开源世界的知名代表。他在许多改变游戏规则的项目和产品中发挥了重要作用。他目前为[的 ARM](https://www.arm.com/) 工作，但是鉴于他在 [openSUSE](https://www.opensuse.org/) 上所做的大量工作，有一个流传的笑话说“Andrew 从 ARM 那里拿工资来开发 openSUSE”

Wafaa 还因在重大开源活动中展示其令人发指的时尚感而闻名。由于自动缩略图聚焦在错误的区域，openSUSE 团队一度不得不删除其中一张活动图像！

与许多开源专业人士不同，Wafaa 并不是从开源开始的。他有一个非常卑微的开始。他 17 岁离开学校，加入皇家空军团，只有 O-level/GCSE 资格证书。1996 年，他以行李搬运工的身份开始在航空业工作，并逐步晋升为初创航空公司的顾问。当他在航空业工作时，他也开始涉足 Linux，大约是在 2003 年。

但是 2001 年 9 月 11 日的美国恐怖袭击使 Wafaa 重新考虑了他在航空业工作的决定，因此他考虑改变行业。

Wafaa 意识到 Linux 在企业中的巨大潜力，但他没有 Linux 的企业经验，所以他参加了为期四周的速成班，希望获得[红帽认证工程师](https://www.redhat.com/en/services/certification/rhce)的认证，但却错过了更高级的[红帽认证技术员](https://www.redhat.com/en/about/press-releases/press-newrhct)。然后，他在一家政府机构担任低级别的支持人员，然后在 2004 年加入 Sun Microsystems，为 L2 L1 提供支持。

在 Sun，站点上只有两个人负责 Linux 支持，而其他人都负责 Solaris，所以 Wafaa 和他的同事决定由 Wafaa 负责 SUSE，另一个负责 Red Hat。这就是 Wafaa 参与 SUSE 和 openSUSE 社区的方式。当年，SUSE 归 Novell 所有。

Wafaa 是许多新项目的重要贡献者，包括为 SUSE 打包 Hula 服务器。他还深入参与了 openSUSE 中的 Gnome 团队。

当英特尔的移动 Linux 项目出现时，Wafaa 为它创建了一个 openSUSE 端口。后来，当莫布林[在 2010 年演变成 MeeGo](https://www.engadget.com/2010/02/15/meego-nokia-and-intel-merge-maemo-and-moblin/) 时，他移植 openSUSE 创建了一个名为 Smeegol (SUSE MeeGo Linux)的项目。随着 [Linux 基金会](https://www.linuxfoundation.org/)举起商标旗，这个原本可能领先的开源移动项目提前夭折，并被搁置。久而久之，MeeGo 还是死了。

虽然 Moblin 和 MeeGo 项目走进了死胡同，但它确实让 Wafaa 接触到了更多的 SUSE 技术。“这不是一个人的事情，而是建立在 SUSE 内部许多人作为其商业产品的一部分所做的工作的基础上。Wafaa 说:“我在玩早期进入 SUSE Studio 的游戏，发现它是构建发行版的一个很好的工具。

认识到他对 openSUSE 的参与，Wafaa 于 2011 年当选为 openSUSE 董事会成员。他当选了两届，然后被任命为董事会的司库。他已连续六年担任董事会成员。

## 将 openSUSE 引入 ARM

在加入 ARM 之前，Wafaa 曾供职于富士通。在 2011 年的一次 openSUSE 会议上，Wafaa 和一些 openSUSE 开发人员随意讨论了 beers Canonical 对 ARM 架构的 Ubuntu 支持。为什么 openSUSE 不能支持？不久之后，他接受了挑战，帮助创建 openSUSE 的 ARM 端口。

“我在 ARM 上使用 openSUSE 的第一个大项目是让 openSUSE 在三星 Chromebook 上运行，无需任何脚本或工作。这是一个真正的港口。当他为 ARM 开发 openSUSE 端口时，他最终被 ARM 聘用，成为 ARM 战略联盟团队的一员，该团队关注 ARM 需要参与的关键领域。由于开源是 Wafaa 的主要关注点，他参与了各种社区和开源项目。

Wafaa 在 ARM 上所做的工作导致了去年第一个用于 Raspberry Pi 的 64 位操作系统的发布。openSUSE 和 [SUSE Linux Enterprise](https://www.suse.com/products/server/) 现在在 Raspberry Pi 3 上都得到完全支持。由于他的支持和 openSUSE 社区的实施工作，寻求 ARM 驱动的数据中心的企业客户现在可以使用 Raspberry Pis 构建超廉价的开发人员环境，以构建企业级 SLES 驱动系统。SUSE 向 Raspberry Pi 用户提供一年免费订阅 SLES 的服务。

A few months ago, Wafaa changed roles at ARM and now he is looking after ARM’s upstream engineering contributions to open source projects. ARM created a new position for Wafaa called Software Director for Enterprise, Communities.

“The intention is to make sure that ARM is a first-class open-source citizen. ARM doesn’t necessarily have the best reputation of working in the open-source community. We acknowledge that, and we want to change and improve at that position,” said Wafaa.
