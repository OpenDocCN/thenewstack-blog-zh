# 加起来:理解 GitHub Octoverse 报告

GitHub 的年度报告包括对非美国贡献、支持依赖性的开发者和私人回购数量的见解。

Nov 7th, 2019 2:00pm by [Lawrence E Hecht](https://thenewstack.io/author/lawrence-hecht/ "Posts by Lawrence E Hecht")![Featued image for: Add It Up: Making Sense of the GitHub Octoverse Report](img/2212acb7a6c0e032841cfd413314a1df.png)

GitHub 刚刚发布了 Octoverse 的年度[状态报告，其中充满了值得分析的数据点。以下是一些快速拍摄和对未来报道的建议:](https://octoverse.github.com/)

1.  **非美国贡献者被夸大了。**报道[称](https://octoverse.github.com/#regions)近 80%的 GitHub 贡献者并非来自美国，但实际上，数据显示约 [20%](https://twitter.com/LawrenceHecht/status/1189167733700812801?s=20) 的开源贡献者来自美国。区别在于，许多 GitHub 用户没有标明他们的地理位置。例如，今年早些时候，当新的 Stack 分析了人工智能基金会领域的 7625 名项目贡献者时，只有 59%的人自愿提供了他们的位置，这包括地球上的 9 个人。当我们查看这些贡献者的前 100 个位置时，40%位于美国。
2.  **巨大的依赖性依赖于少数开发人员。**平均有 350 万个项目依赖于排名前 50 的 npm (Javascript 相关)包中的任何一个。该图包括了 forks 和私有存储库中的依赖项，其中包括不活动的项目和那些主要不是用 JavaScript 编写的项目。只有 35 个人是这些包的组件的直接贡献者。部分原因是由于不同编程语言的独特性质，Java (Maven)、Python (pip)、。NET (NuGet)和 RubyGems (Ruby)每个直接贡献者的依赖项目要少得多。
3.  **私人回购超过公共回购。**去年创建了超过 4400 万个存储库，但其中绝大多数是私有的。包括之前创建的项目，去年有 1200 万个公共库和分支被推送到 GitHub。随着 GitHub、GitLab 和 Atlassian [争夺市场份额](https://thenewstack.io/i-dont-git-it-tracking-the-source-collaboration-market/)，重要的是要认识到他们的大部分收入是基于管理私有库。

我们故意没有包括关于顶级项目和语言的数据，因为报告的这一部分值得与之前版本的 Octoverse 报告进行更详细的比较。在下周的 [GitHub Universe](https://githubuniverse.com/) 会议之前，我们鼓励该公司让这些旧报告变得容易获得，这样有事业心的记者就不必依赖[互联网档案馆的回溯机](https://web.archive.org/web/20190410172307/https://octoverse.github.com/projects)。
