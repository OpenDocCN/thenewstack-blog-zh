# 金融行业如何成为云原生的晴雨表

> 原文：<https://thenewstack.io/how-the-financial-sector-is-a-barometer-for-cloud-native/>

[帕洛阿尔托网络公司的 Prisma Cloud】赞助了这次播客。](https://www.paloaltonetworks.com/prisma/cloud)

在本期的[节目中，由 New Stack 创始人兼发行人 Alex Williams 主持的 New Stack Makers](/podcasts/makers) 播客、信息安全工程 IT 安全运营副总裁 feli PE Medina[以及 BankUnited N.A .的云架构和运营经理](https://www.linkedin.com/in/michael-lehmbeck-1656465)Michael lehm beck 讲述了他们在金融领域的云原生之旅。该播客是为 6 月 24 日举行的[云本地安全虚拟状态](https://start.paloaltonetworks.com/state-of-cloud-native-security)峰会录制的。

[BankUnited N.A.](https://www.bankunited.com/) 属于中型银行类别。总部位于佛罗里达州迈阿密湖，总资产约 329 亿美元，服务于消费者和商业部门。bank United N . A(bank United 的子公司)不是美国最大的银行之一，也不是最小的银行之一，但可以被视为云原生采用的晴雨表，以实现向云原生的转变。

[金融部门如何成为云原生的晴雨表](https://thenewstack.simplecast.com/episodes/how-the-financial-sector-is-a-barometer-for-cloud-native)

大约三年前，BankUnited N.A .开始转向云计算。Lehmbeck 说，最初的想法是“试水”，以获得“一些切实的回报”。该团队开始测试其灾难恢复能力。“我们证明了我们能够在灾难恢复类型的场景中，在我们的主数据中心和[亚马逊网络服务(AWS)](https://aws.amazon.com/) 之间进行故障转移的能力，”Lehmbeck 说。“因此，这基本上使我们能够获得初始足迹，并证明我们的任务关键型系统实际上可以在该云资产中运行。”

该银行在转向原生云时，还采用了非常以安全为中心的流程。

“我们对我们的云之旅采取了非常谨慎的态度，”Medina 说。“我们决定继续做的第一件事是展现典型的网络场景或传统网络场景。”

该银行首先依靠 Palo Alto Networks 的防火墙来满足其数据流量检查需求，并使用 [Amazon Web Services](https://aws.amazon.com/) 及其“云房地产”进行灾难恢复。“去年，我们从云中运行了整个灾难恢复练习，”Medina 说。

灾难恢复练习非常成功，也是该银行从传统环境向云原生基础架构转变的一个里程碑。“我们有一定程度的信心，我们知道这些系统可以运行，我们可以维持这种环境，”Lehmbeck 说。“所以，让我们继续，现在继续，翻转脚本，而不是将云作为我们的灾难恢复，让我们继续，现在将它用于我们的全职生产。”

Lehmbeck 说，最终的结果是“完全退出数据中心业务”。

在 BankUnited N.A .向云迁移的过程中，该银行的成功转型在很大程度上归功于 AWS 和 Palo Alto Networks 的工具。Lehmbeck 说:“我们所看到的是，在我们的 CSP 中利用 AWS 的一些原生功能，以及利用 Palo Alto Networks 等第三方合作伙伴来真正尝试和改善我们的整体安全状况方面取得了进步。”。“与我们当前的内部状态相比，我们有一个很高的基准，但我们真正做到的一点是，我们希望在这一领域做出改进。”

最后，DevOps 团队试图展示该银行向云原生环境的迁移如何改善其整体安全状况。“老实说，这只能通过非常紧密的合作伙伴关系来实现，不仅是与我们的 CSP，还与 Palo Alto Networks 等关键的第三方合作伙伴，以及在团队内部进行大量投资，真正帮助推动教育和熟悉平台，”Lehmbeck 说。

[亚马逊网络服务公司(AWS)](https://aws.amazon.com/) 是新堆栈的赞助商。
