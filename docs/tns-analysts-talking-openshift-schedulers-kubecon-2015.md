# 在 KubeCon 2015 上谈论 OpenShift 和调度程序

> 原文：<https://thenewstack.io/tns-analysts-talking-openshift-schedulers-kubecon-2015/>

在旧金山庄严的皇宫酒店召开了 KubeCon 会议，在会议结束的几个小时里，New Stack 创始人 Alex Williams 与会议组织者 [Patrick Reilley](https://twitter.com/preillyme) 以及 Red Hat 的 [Martin Klaus](https://www.linkedin.com/in/mklaus123) 和 [Grant Shipley](https://github.com/gshipley) 一起录制了这一期的 New Stack Analysts。该小组讨论了调度程序的多样性，企业面临的标准化选择，以及 Red Hat 最近在刚刚发布了 3.1 版本的 [OpenShift](https://www.openshift.com/) 上取得的成就。

Kismatic 的创始人兼首席执行官 Reilley 为 Docker 和 Kubernetes 提供制作支持，他在谈到首届 KubeCon 时说:“对我们来说，最重要的是确保它以社区为中心，而不是由赞助商驱动。我们希望人们谈论他们用 Kubernetes 解决的现实世界中的问题，并真诚地谈论项目的状态，以及他们希望它在未来几个月和几年内解决的问题。”

对于库伯内特项目的年轻时代，“它有一种前所未有的成熟感，”他说。虽然与 Kubernetes 最接近的竞争对手 Apache Mesos 在运营方面领先，但 Reilley 说，“Mesos 今天能做的和 Kubernetes 今天能做的之间的差距并没有你想象的那么大。”

负责 OpenShift 营销的 Klaus 表示，这种成熟吸引了企业客户，但他承认早期采用者和修补者给社区带来了一定程度的热情。“我们需要灵活地将上游社区的创新更快地转化为生产质量的产品，以便更多的客户和用户能够利用它。”

“让我们确保培育社区，”他说，“并做出贡献，培育那些冒出来的想法，看看我们可以用它解决哪些商业问题。”

“Kubernetes 项目目前的独特之处在于围绕它的纯粹的兴奋，”Shipley 说，他是一名从第一天起就加入 OpenShift 团队的 Red Hat 开发人员。“这是第一次活动，有超过 500 人参加，这非常了不起。”

“我认为我们都很幸运能成为这个项目的一部分，”他继续说道。“人们开始意识到在容器上运行 Kubernetes 和调度器编排层的好处…当你玩完容器，并且真的想开始在生产中使用它们时。”

事实上，会议的大部分对话都是关于调度程序的，这很可能是 KubeCons 未来的重点。

“你看看参加会议的 Univa 的人，”Reilley 说。“他们继承了 Sun grid 技术，现在他们能够使用其中一些技术，在 Kubernetes 上的通用调度程序之上构建更好的调度程序，并提供价值。你看看马尔特·施华蔻，他是谷歌欧米茄白皮书的主要作者；他正在开发 Poseidon 调度程序，该程序使用的是他的基于苍穹流的调度程序。

“他们没有受到影响，也没有寻找新的原型，”他指出。“他们已经在那里了。他们拥有所需的构件。”
