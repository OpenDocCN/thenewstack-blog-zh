# 这些数字说明了当今服务网格的使用情况

> 原文：<https://thenewstack.io/what-the-numbers-say-about-how-service-meshes-are-used-today/>

服务网格技术已经成为在微服务环境中管理可观察性、安全性和流量管理的可靠方法，通常使用 Kubernetes 进行容器编排。服务网格的具体用例和[需求也各不相同。](/when-you-do-and-dont-need-a-service-mesh/)

新堆栈最近完成了一项关于服务网格用例的[调查](/survey-results-service-mesh-useful-for-security-observability-and-traffic-control/)。虽然三分之一的受访者表示他们的组织已经使用服务网格来控制微服务和 Kubernetes 环境之间的流量，但受访者的采用率和使用情况差异很大。例如，16%的受访者表示他们的组织在生产环境中广泛使用服务网格，17%的受访者表示服务网格在生产环境中的使用有限。

在最新一集的[新堆栈分析师](/podcasts/analysts)播客中，[服务网格提供商](https://www.linkedin.com/in/leecalcote/) [Layer5](https://layer5.io/) 的分析师和创始人 Lee Calcote ，以及[红帽](https://www.linkedin.com/in/brianredbeard/) [OpenShift](https://www.openshift.com/) 服务网格的主要产品经理 Brian "Redbeard" Harrington ，讨论了调查数据真正含义的许多细微差别。

[第 195 集——李·卡尔科特和布莱恩·“红胡子”·哈灵顿](https://thenewstack.simplecast.com/episodes/episode-195-what-a-mesh-w-lee-calcote-and-brian-redbeard-harrington)

Calcote 指出，在众多不同的服务网格功能中，流量管理被视为一个关键特性，但它对高级用户最有用。谈到流量管理功能的使用，Calcote 说:“人们在进入这一领域时往往会更先进一些，因为他们在这一点上实际上影响了流量，然后以不同的方式路由请求，而不是像纯粹观察或获得他们环境中的‘只读’视图那样。”

哈灵顿同意了。哈林顿说:“我很高兴李指出了交通控制的具体区别，因为在我与之交谈的用户中，这是一语双关的‘门户药物’”。Harrington 说，拥有传统裸机环境和“非常昂贵的硬件设备”的组织在“迁移到动态环境”和“降低”一些传统硬件的优先级时面临挑战，流量管理功能服务网格可以在这种转变中提供帮助。

调查结果和现场经验还表明，组织仍在考虑服务网格的最佳用例。当被问及一个组织是否应该采用或他们应该如何开始依赖服务网格时，通常是“不管他们是从该领域中更简单的……还是更复杂的[可能性]开始，”Calcote 说。“建议通常是相同的，那就是你应该开始并一次采用一点价值，这种价值多少取决于你在 mesh 之外寻找什么，”Calcote 说。“但是，在您对已部署的产品感到满意和从已部署的产品中获得价值之间，[组织应该]从那里迈出下一步，希望在某个时候利用网格的所有功能。”

服务网格的采用也应该导致最终提供业务结果。“当我开始与用户交谈时，我最讨厌的一种情况是，我总是有一个非常具体但看似简单的问题:当你去部署一个网格时，它已经改变了你的组织、你的过程和你开发代码的方式，它做了什么？这个结果如何让你的业务变得更好？”哈灵顿说。“几乎毫无疑问，我得到的回应只是一份营销首页的要点列表。你知道，这就是我为什么问这个具体问题的原因，因为虽然功能很重要，但功能只是软件的一小部分。”
