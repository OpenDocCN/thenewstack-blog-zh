# 裸体自拍将暴露在灰色阴影的数据世界中

> 原文：<https://thenewstack.io/naked-selfies-will-get-exposed-in-a-world-of-data-shaded-in-gray/>

裸体自拍如今风靡一时，但这篇文章不是关于这个的。不管你怎么定义，这都是关于特权数据的。这是你需要控制的东西，不是任何人(甚至任何人)都可以被允许看到的。它可能是财务数据、医疗信息、法律文件、知识产权，或者——是的——你的调皮照片。

当我们点击表示我们已经阅读并理解了云应用的服务条款(您确实阅读了服务条款协议，不是吗？)在我们使用的每个云服务的上下文中，我们对我们的特权数据做出了有约束力的决定。云服务提供商认为数据泄露是不可避免的。即使他们对自己的安全性有信心，他们也知道用户在习惯方面是出了名的马虎。

> 错误时有发生，没有一个云服务在发生违规时不承担任何责任。

在企业 IT 领域，我们希望适用于消费者的规则不应该适用于我们。除了 iCloud 中提供的安全性之外，Workday 中还应该有其他安全性吗？这是个有趣的问题。最后一次有人和他们的 SaaS 供应商讨论合同中的赔偿条款是什么时候？

本周泄露的名人裸照对其所有者来说是高度保密的数据。私下拍摄，并期望隐私，出了点问题，但因为被泄露的数据是女明星的照片，对这一事件的关注被放错了地方。

让我们不要裸体，而是把数据看作是存储在一家公司的 Salesforce.com 环境中的信息。如果变得个人化，可能会更容易:在您的企业 Salesforce 实例中存在多少特权数据？哪些用户可以访问这些数据？您能从经验上证明拥有这些用户凭证的实体就是用户本身吗？如果数据落入他人之手，您的组织会遭受什么样的损失？

也许将云服务比作停车场会有所帮助。有些车库只是给你提供一个停车的地方；这个社区可能很简陋，灯光很差，服务员受到了 Ferris Bueller 休息日的启发，但价格合适，你不介意步行。其他车库更贵，但它们光线充足，配有保安，装有监控摄像头，位置便利，并保持最佳状态。

虽然后者可能会提供更大的安全感，但两个车库都不会对留在车内的贵重物品负责。这里的问题很简单:你应该得到安全，但安全无法得到保证。

你不能因为意外被钓鱼而受到责备，但当它发生在你身上时，你也不能责怪服务提供商。在云中，就像在生活中一样，有很多灰色。坏事发生在好人身上，但那些教育自己认识到风险并积极投资以减轻这些风险的人不太可能成为猎物。
