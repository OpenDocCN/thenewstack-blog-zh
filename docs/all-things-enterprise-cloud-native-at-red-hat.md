# Red Hat 自带的所有企业云

> 原文：<https://thenewstack.io/all-things-enterprise-cloud-native-at-red-hat/>

[万物企云原生于红帽](https://thenewstack.simplecast.com/episodes/all-things-enterprise-cloud-native-at-red-hat)

五月初在哥本哈根的 [KubeCon + CloudNativeCon，我们和](https://events.linuxfoundation.org/events/kubecon-cloudnativecon-europe-2018/) [Red Hat](https://www.openshift.com/) 的 [Brandon Philips，](https://www.linkedin.com/in/brandonphilips/)一起讨论 CoreOS 和容器的整体状态。飞利浦之前曾与 CoreOS 合作，CoreOS 于今年 1 月被红帽收购，此次合作将大量 Linux 思想领导力集中在了 Kubernetes 项目上。在哥本哈根，他和 Red Hat 社区发展总监 Diane Mueller 坐在一起谈论合并 CoreOS 和 Red Hat 团队的感受。

CoreOS 团队和 Red Hat 一直与 CNCF 紧密合作，为企业用户扩展 Kubernetes 的功能。飞利浦说，这项努力的一个重要部分是 Kubernetes 运营商项目。

“我们在公共云中看到的基本上是两个软件部分:托管部分已经有很长一段时间了，然后是这种软件交付部分，现在单个服务器之外的开源软件暂时还没有办法在集群和多服务器环境中交付其价值，”飞利浦说。“我认为，通过[运营商](https://coreos.com/operators/)和我们在 CoreOS 尝试做的事情，我们在 Red Hat 继续执行的愿景是:我们如何让人们能够跨越他们的数据中心、跨越他们的云，将软件——无论是开源还是非开源——带到所有这些不同的环境中，而不是每次他们必须移动到托管这些服务器的位置时重复工作。”

“这里的想法是开箱即用，Kubernetes 为您提供这些抽象的 pod、部署和服务。这些都是基本的计算工具，本质上允许你运行一个软件，然后给它网络连接。这些是基本的基本构件。我们发现人们正在做的事情，以及为什么公共云对开发人员来说如此重要的原因是，他们不仅可以说我需要一个虚拟机，还可以说我需要一个数据库，或者我需要一个负载平衡器。操作员正在做的是向计算、网络和存储这一基本语言添加新名词。现在，我们可以说是计算、网络和存储，也可以说是负载平衡器、数据库和火花队列。正是这些抽象让开发者变得如此高效，”飞利浦说。

穆勒说，运营商现在是红帽 Kubernetes 计划的一大部分。“我认为这是一种进步，因为首先，我们试图说服每个人把东西放在容器里。我认为这是社区教育的下一步，我们将在接下来的几个月里做。什么是操作符，如何创建操作符，它们有哪些不同的用例。因此，从我的角度来看，我认为这真的是社区必须加强并开始工作和创造这些新的运营商，”穆勒说。
