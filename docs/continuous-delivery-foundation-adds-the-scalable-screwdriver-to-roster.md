# 持续交付基金会将可伸缩螺丝刀加入名册

> 原文：<https://thenewstack.io/continuous-delivery-foundation-adds-the-scalable-screwdriver-to-roster/>

自从[持续交付基金会(CDF)](https://cd.foundation/) 成立以来已经[将近一年](/continuous-delivery-foundation-launches-with-jenkins-jenkins-x-spinnaker-tekton/)了，其目的是为持续集成和持续交付(CI/CD)领域的开源项目提供一个供应商中立的家园，以及定义一些行业标准，上周[正式接受了自成立以来的第一个新成员](https://cd.foundation/blog/2020/02/19/screwdriver-joins-cd-foundation-as-its-first-incubation-project-treating-continuous-delivery-as-a-first-class-citizen-in-the-build-pipeline/)，并增加了[螺丝刀](https://screwdriver.cd/)。

在 Linux 基金会的庇护下，该基金会托管了至少六个其他基金会和超过 175 个开源项目，CDF 最初以 [Jenkins](https://jenkins.io/) 、 [Jenkins X](https://jenkins-x.io/) 、 [Spinnaker](https://www.spinnaker.io/) 和 [Tekton](https://github.com/tektoncd) 项目为创始成员，但在开始成长之前，它有一些成长的烦恼需要克服。然而，螺丝刀的加入向基金会的其他项目发出了一个信号，表明基金会准备向前推进，CDF 项目经理 Dan Lopez 说。

“有几个[项目]已经开始做提案过程，已经在技术监督委员会(TOC)或特定的特殊利益集团(SIG)面前演示了他们的工具，”洛佩兹告诉 New Stack。“螺丝刀带头冲锋陷阵，我认为这也为其他人提供了一个舒适区，他们可以站出来说，嘿，我也想把我的工具带到桌子上来。”

螺丝刀加入了孵化级别，是 CDF 仅有的两个级别之一，在毕业级别之前。与其他同类基金会一样，CDF 提供资源来帮助其孵化项目扩大规模和增长，包括项目管理、正确程序指南、开源治理、活动管理、营销服务、技术基础设施以及围绕法律和知识产权问题的援助。

Jithin Emmanuel ，现在是威瑞森媒体的营销经理，曾是雅虎团队的一员。他制作了螺丝刀，他[写下了创作的](https://opensource.com/article/19/3/screwdriver-cicd)，解释说没有其他 CI/CD 工具可以处理当时公司所需的规模。

“2012 年，在经历了我们使用的 CI/CD 解决方案的性能和扩展限制后，我在雅虎的团队开始寻找另一个平台。我们没有找到满足我们快速增长的开发者需求的解决方案，所以我们开发了螺丝刀，”他写道。“在看到增值螺丝刀后，我们决定在 2016 年开源它，与全球 DevOps 社区分享我们的解决方案。”

洛佩兹类似地描述了该工具的创造，评论说“螺丝刀源于对雅虎基础设施的巨大努力，随着时间的推移，他们建立插件架构的方式，以及在雅虎内部无数不同场景和生态系统中配置和部署的方式，真正开始发挥效率。”

螺丝刀的插件架构允许用户交换架构的各个部分，甚至是动态的，例如将 golang 构建发送到 Kubernetes 执行器，而将 iOS 构建发送到 Jenkins 执行场。

螺丝刀最初于 2019 年 7 月提出了加入中国发展基金的提议，尽管许多成员认为该项目令人兴奋且成熟，但一些人认为基金会尚未做好准备。当时，TOC 成员 Andy Glover 对他投票反对螺丝刀加入 CDF 提出了以下解释。

当时，CDF TOC 还没有“为现有的成员项目制定一个正式的互操作性计划/策略”。我认为，在接受新项目之前，我们应该澄清所有这些项目如何为了最终用户的利益而结合在一起，”Glover 在给 CDF TOC listserv 的一封电子邮件中写道。“我认为螺丝刀很有趣，我们都可以从这个社区中学习，但我希望推迟添加它(或其他项目)，直到我们有了一个清晰的创始项目的故事。

Lopez 解释说，开始接受新项目的[过程的一部分是以开发大量 SIG 的形式出现的，以更好地划分和征服由 CDF 处理的连续交付的不同方面。2019 年末，该基金会集思广益](https://github.com/cdfoundation/toc/blob/master/PROJECT_LIFECYCLE.md#project-proposal-requirements)[九大战略目标](https://cd.foundation/blog/2019/10/17/cdfs-governing-board-unveils-9-strategic-goals/)，此后又为[安全](https://github.com/cdfoundation/sig-security)、 [MLOps](https://github.com/cdfoundation/sig-mlops) 和[互操作性](https://github.com/cdfoundation/sig-interoperability)创建了三个 SIG。Lopez 说，随着流程和 SIG 被更好地定义来处理即将到来的项目，他们很高兴能够向前推进。

“我们相信螺丝刀是一个很好的开始，我们很高兴能一起工作。通过加入 CD 基金会，螺丝刀将能够更快地扩展，在开发和部署方面迈出更大的步伐，”洛佩兹在一份声明中说。“有了这么多支持的集成，螺丝刀提供了开发团队所需的开放性和灵活性。”

Linux 基金会是新堆栈的赞助商。
