# 尽可能向左移动，以保护云原生应用

> 原文：<https://thenewstack.io/shift-as-far-left-as-you-can-to-protect-cloud-native-applications/>

来自帕洛阿尔托网络公司的 Prisma Cloud 赞助了这次播客。

许多(如果不是大多数)IT 专业人员都非常清楚在生产管道的最开始嵌入安全流程的重要性。

在本期的[新堆栈制造商](/podcasts/makers)节目中，来自 [Palo Alto Networks](https://www.paloaltonetworks.com/prisma/cloud) 的思想领袖讨论了为什么软件生产流程中的安全性“左移”对当今的开发运维至关重要，该节目是为[6 月 24 日举行的云原生安全状态](https://start.paloaltonetworks.com/state-of-cloud-native-security)虚拟峰会录制的。讨论的主题包括向左移动的趋势如何在 DevOps 中扎根，它与连续交付(CD)的集成，安全不仅在软件开发过程中而且对企业的作用，以及最终向左移动如何帮助确保软件安全。

作为 Palo Alto Networks 的嘉宾，Prisma Cloud 的产品经理 [Aqsa Taylor](https://www.linkedin.com/in/aqsa-taylor-b7b81961/) 、Palo Alto Networks 的云首席技术官 Ashley Ward[、Prisma Cloud 的产品营销主管](https://www.linkedin.com/in/keithmokris)Keith mo Kris[以及 Prisma Cloud 的云首席技术官](https://www.linkedin.com/in/vinayvenkat)Vinay Venkataraghavan。

[尽可能向左移位保护云原生应用](https://thenewstack.simplecast.com/episodes/shift-as-far-left-as-you-can-to-protect-cloud-native-applications)

DevOps 团队经常表达的一个主要问题和关注是，他们如何在开发过程的最开始集成安全过程，而不妨碍以更快的节奏部署代码的需求。然而，从一开始，左移就可以帮助 DevOps 避免在“要求开发人员重新构建他们的应用程序以满足安全性和合规性要求”之前等待安全团队完成安全检查，Taylor 说。

泰勒说，这是“通过从安全角度引入对他们正在构建的东西的可见性”来实现的。“在早期的周期中，我们帮助或帮助开发人员更快地交付应用程序，同时在应用程序进入运行时之前确保良好的质量。因此，通过在早期阶段提高安全性，您实际上减少了应用交付的时间。”

泰勒说，通过在周期的开始就进行安全测试，“你让他们对整个画面和安全设备有了更多的了解”。“通过在开发人员进行测试时将他们结合起来，你也使他们更有能力。”

当在生产周期的最开始使用正确的工具实现测试时，这也意味着 Q&A 过程是自动化的。“如果你正在通过测试的金字塔，并使事情自动化和易于使用，那么我认为这对 QA、安全性和整个 DevOps 团队来说都是令人兴奋的，”Ward 说。

在整个 CD 中，自动化对于成功的左移方法也是必不可少的。泰勒说，对 GitOps 使用自动化“总是会减少工作量和人工交互”。“想象一下规模更大的公司，您没有时间逐个管理集群，您希望能够在几分钟内看到结果。”

最终，今天的开发运维团队必须拥有合适的安全工具，以便在生产周期的最开始以及整个部署过程中锁定代码，同时不影响应用及其更新的部署速度。Venkataraghavan 说，这就是 Prisma 云套件发挥作用的地方。“我显然有偏见，但我非常兴奋，因为我们在他们所在的地方会见了开发人员，并为他们提供了进行这些修复的能力，就在他们对这些代码记忆犹新的时候，这是无与伦比的。”
