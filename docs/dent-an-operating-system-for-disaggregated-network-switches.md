# DENT:用于分散网络交换机的操作系统

> 原文：<https://thenewstack.io/dent-an-operating-system-for-disaggregated-network-switches/>

Linux 基金会启动了一个名为 [DENT](https://dent.dev/) 的项目，致力于为分散式网络交换机创建一个网络操作系统。

它汇集了包括亚马逊、Cumulus Networks、Delta Electronics、Marvell、Mellanox 和 Wistron NeWeb 在内的成员，为企业边缘属性(如远程校园位置和零售店)构建开源网络操作系统。

这是 Linux 基金会旗下的[最新开放网络项目](https://thenewstack.io/linux-foundation-takes-networking-fd-io/)，该项目于今年年初推出了 [LF Edge](https://www.lfedge.org/) 框架，将三个现有项目和两个新项目整合在一起，努力促进边缘计算和物联网开发社区之间的互操作性和协作。它在 9 月份又增加了两个项目: [Baetyl](https://www.lfedge.org/projects/baetyl/) (发音:beetle)，最初由百度捐赠，为开发者提供了一个可以构建边缘计算系统的平台；和由 Dianomic 贡献的 Fledge，Dianomic 是一个专注于工业边缘环境中的关键操作的开源框架。

凹痕将与左前边缘分开操作。

据 Linux 基金会网络总经理 Arpit Joshipura 说，DENT 的目标是创建一个开源技术社区，专注于各种行业和供应商的网络操作系统、控制平面和管理平面用例。

传统供应商将网络硬件和软件作为一个整体出售，无法针对每个市场和每个使用情形(如电信、云或企业数据中心)进行分离和定制。他们使用专有芯片进行数据包处理，并使用封闭操作系统在网络交换机上运行工作负载和应用程序。

据该组织称，虽然开放网络正在数据中心和电信基础设施中使用，但企业边缘用例仍有差距。

DENT 将使用 Linux 内核、Switchdev 和其他基于 Linux 的项目，使开发人员能够像对待任何其他硬件一样对待网络芯片。这个想法是为了简化目前存在于这些交换机和其他边缘用例的开放软件中的抽象、API、驱动程序和开销。

5G、物联网和人工智能等新技术正在将工作负载和服务拉近应用和用户。据该小组称，拥有一个简单的基于 Linux/SwitchDev 的分解开关将实现一个应用生态系统，并有助于标准化整个生态系统的集成。

Mellanox Technologies 的以太网交换机副总裁 Amit Katz 在一份声明中说:“DENT 促进网络分解，通过消除供应商锁定使客户受益，并允许硬件供应商在公平的竞争环境中竞争，最好的交换机 ASICs[硅]和系统可以通过提供尽可能高的 ROI 而获胜。”

在 New Stack 的前一篇帖子中，SnapRoute 的联合创始人亚当·卡塞拉(Adam Casella)提出了[需要在网络中采用云原生方法(T3)的理由，以便实现分解。](https://thenewstack.io/whats-preventing-disaggregation-in-networking/)

凭借其[云原生网络操作系统(CN-NOS)，](https://thenewstack.io/cn-nos-a-cloud-native-network-operating-system/) SnapRoute，最近[被 InfoBlox 收购](https://siliconangle.com/2019/11/07/networking-heavyweight-infoblox-acquires-microsoft-backed-startup-snaproute/)，一直专注于将 DevOps 敏捷性和开发人员经验引入网络软件的编写。
