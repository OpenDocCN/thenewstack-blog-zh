# puppet compliant 自动执行安全策略

> 原文：<https://thenewstack.io/puppet-comply-automates-security-policy/>

Puppet 推出了[Puppet compliant](https://puppet.com/products/puppet-comply/)，这是一款新产品，可评估、补救和大规模执行合规政策，同时还提供服务，帮助发现合规问题并构建内容来解决这些问题。此次发布与该公司过去一年的其他发布步调一致，后者更专注于进一步的自动化和补救，而[傀儡](https://puppet.com/)首席技术官[艾比·卡恩斯](https://www.linkedin.com/in/abbykearns/)将其视为这一趋势的延续。

“我经常听到的说法是,(合规)成本很高，被认为是进步或创新的障碍，而且是一个很大的负担。最终，如果你不合规，你的动力、你的进步、甚至你作为一家公司的品牌都会受到打击，”卡恩斯说。“我们如何开始真正帮助我们的客户实现合规性，让他们享受与我们帮助他们管理基础架构和管理漂移时同等的舒适和支持？对于我们来说，合规性是继续与客户进行自动化之旅的自然进程。”

Kearns 还指出，Puppet 的许多客户正在转型，将工作负载转移到云，并应对日益复杂的环境。因此，第一步是让 Puppet 专注于基础工作，包括发布时的互联网安全(CIS)基准测试中心。然而，这只是一个起点，木偶产品经理 Alex Hin 说。

“当我们考虑我们试图解决的问题时，首先是需要在节点、服务器或工作站上安装的控制装置本身。当我们看到像 FedRAMP 和 NIST 这样的东西时，它们都尽可能地映射回控制，映射到存储或配置这些服务器的特定方式，因此我们从安全配置方面的合规性的广泛视角开始。随着我们产品的增长，我们将寻求帮助我们的客户实现其他合规计划，如 FedRAMP、Sox、HIPAA 和所有其他计划，”Hin 说。

Puppet Enterprise 允许它的用户定义一个期望的状态配置，它会自动将其放置到位并加强。如果有人在服务器上做了与期望状态不同步的更改，Puppet 将恢复它，强制执行期望的状态。Puppet compliant 采用了这一基本原则，并将其扩展到合规性，Hin 指出，这使 Puppet 客户“能够根据基准进行评估，因此它不再只是强制一个特定的状态。”与此同时，Puppet compliant 超越了其他法规遵从性解决方案，不再只是提供一份报告，然后需要由运营团队手动制定。

“传统上，当我们谈到自动化合规性评估时，CIS 和 DISA 之类的事情是由通常属于安全团队的扫描技术完成的。这一领域的许多供应商传统上只关注评估和可见性，很少有人会将补救作为其软件或服务的一部分，更不用说强制执行了，”Hin 说。“没有木偶，真的没有办法让人们大规模地执行这些类型的动作。我们只做一次，然后将它大规模应用到所有这些节点，这样我们就真正减少了推出更改所需的工作量和时间，然后确保这些更改不会被恢复或意外更改为其他内容。”

虽然 Puppet compliant 有助于补救和执行，有助于在尝试确保大规模合规性时消除人为错误因素，但满足法规要求仍然很棘手，因此 Puppet 发布的最后一部分是配套的专业服务。Hin 说，他们希望帮助客户成功，“而不仅仅是给他们一个产品，然后让他们去解决它。”

展望未来，Hin 预计 compliant 将增加上述基准，如 FedRamp、DISA、NIST、Sox 和 HIPAA 等。

Hin 说:“我们不仅希望继续增强我们作为产品的能力，还希望扩大我们基准的范围。”。“我们从 CIS 开始，但我们希望扩展到其他领域和其他基准供应商或拥有基准的监管机构。”
