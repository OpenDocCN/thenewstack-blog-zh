# 好事达借助云铸造部署进入开发运维快车道

> 原文：<https://thenewstack.io/allstates-devops-honeymoon-pivotal-going-strong-one-year-later/>

好事达(Allstate)云工程总监马特·库里(Matt Curry)表示:“当你实施 Cloud Foundry 这样的平台时，你很快就会发现:突然之间，你的交付管道的一部分变得非常非常快。”。

在本期[New Stack Analysts 播客](/tag/the-new-stack-analysts/)中，Curry 与 TNS 的 Alex Williams 和 [Benjamin Ball](https://thenewstack.io/author/benjamin/) 谈论了这家知名保险公司使用 Pivotal Cloud Foundry 实现持续集成、持续部署和测试驱动开发的历程。回到节目的是[布丽奇特·克罗姆胡特](https://twitter.com/bridgetkromhout)，Pivotal 云铸造公司的首席技术专家。

[#78:好事达借助云铸造部署进入 DevOps 快车道](https://thenewstack.simplecast.com/episodes/78-allstate-speeds-into-the-devops-fast-lane-with-cloud-foundry-deployment)

这个播客也可以在 YouTube 上看到。

鲍尔询问了导致好事达选择致力于 Cloud Foundry 的因素，这一决定要早于库里在好事达的时间，但他对此表示热烈欢迎。

“(这个决定)是出于加快应用程序部署速度的需要，他们意识到这涉及多个方面。有动态临时基础设施的观点，然后是支持连续交付，支持测试驱动的开发，并在整个应用程序生命周期中推动自动化，”Curry 说。

“我认为，Cloud Foundry 与 Pivotal Labs 咨询协议的结合最终真正推动他们转向 Cloud Foundry，而不是一个不同的解决方案，”他说。

> “我们没有尝试使用我们的虚拟化团队。我们没有尝试使用现有的 Linux 管理团队来支持它。我们从团队中挑选出个人，创建了一个新的平台即服务团队”——Matt Curry

并不是说 Cloud Foundry 的部署完全是不受约束的速度…

“也有现有的团队和筒仓和过程造成摩擦，”库里说。“无论你做什么，都不可能摆脱那些。所以你必须开始考虑如何优化这些东西。我们已经开始走这条路，”他补充道，“我们已经在身份空间取得了很大的进展；我们在一些围绕身份管理的 IBM 工具上进行了大量投资。”

他说，他们还一直在努力实现这一点，以便当开发人员部署新的 Cloud Foundry 应用程序时，该应用程序“可以与我们拥有平台之前开发的应用程序集成，用户可以在两者之间无缝切换，而不必重新登录”。“从用户体验的角度来看，这是我们真正想要推动的事情。”

“我们有很多大多数公司都会有的传统障碍，”库里说。“防火墙请求流程并不是真正为了促进敏捷和加速交付而构建的。数据库供应、批准和治理流程也不是真正为此而构建的。因此，我们开始挖掘这些东西，并找出如何优化这些东西。”

“我们知道，随着我们开始扩大这一努力，甚至超越我们已经取得的成就，能够使这些事情更有效、更精简、更自动化，将是我们成功的关键，”他说。

“当你开源这些工具的时候，”克罗姆胡特插话道，“你将会给所有不同的 DevOps days，Velocity 和 OSCON 发送一些精彩的会议提议。那太好了！”

“你需要真正的文化和共享，这样你就可以使用这些自动化和测量工具，以达到最佳效果，”她说。

库里称赞好事达的领导层，特别是基础设施服务高级副总裁安迪·齐特尼(Andy Zitney)不仅有远见卓识地做出改变，还知道如何去做。齐特尼和库里都在讲述好事达云的故事。

“当我们建立 Cloud Foundry 时，我们用一个专门的团队来建立它，”库里说。“我们没有尝试使用我们的虚拟化团队。我们没有尝试使用现有的 Linux 管理团队来支持它。我们从团队中挑选出个人，创建了一个新的平台即服务团队，他们是负责维护该平台的团队。”

他解释说:“这样做让我们能够挑战现有的平台运营规范。”。“围绕如何监控它，我们做出了不同的决定。我们围绕如何交付做出了不同的决定。我们认为这是一个真正推动“一切自动化”的机会“如果你与 Pivotal 的一些人交谈，我们很早就公开表示，我们并不真正关心 GUI，”他回忆道。我们希望能够通过 API 和管道来驱动一切，并在基本上按一下按钮的情况下交付平台，或者甚至可能不是这样。"

“其结果是，它位于现有基础设施之外，”对 Allstate 来说，现有基础设施由 WebSphere 组成。Net 和 Tibco。“我们有开发团队，他们在云铸造服务代理和集成点等方面帮助我们，”库里说。“我们只是找出摩擦点，然后在这些摩擦点出现时解决它们。”

“我们一直在边走边学习和反应，”他说。“我们已经以一种他们可以快速响应的方式动员了人们。”

IBM 和 Pivotal 是新堆栈的赞助商。
