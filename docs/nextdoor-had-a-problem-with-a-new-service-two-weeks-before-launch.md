# Nextdoor:发布前两周的测试挑战

> 原文：<https://thenewstack.io/nextdoor-had-a-problem-with-a-new-service-two-weeks-before-launch/>

欢迎来到[新的堆栈制造商](/podcasts/makers):攀登新的高度，由 [Scalyr](https://www.scalyr.com/) 首席执行官 [Christine Heckart](https://www.linkedin.com/in/christineheckart/) 主持的一系列采访，采访工程师们关于他们面临的问题和他们寻求的解决方案。

[【攀上新高度】第三集:下一门:发布前两周的测试挑战](https://thenewstack.simplecast.com/episodes/scaling-new-heights-episode-3-nextdoor-test-challenges-two-weeks-before-launch)

[麦乐](https://www.linkedin.com/in/mai-huong-le/)是社交媒体服务 [Nextdoor](https://nextdoor.com/) 的工程主管。她和她的团队知道他们必须为疫情遭受苦难的小企业做点什么。他们需要找到将他们与当地客户联系起来的方法。社交网络已经成为小企业的中心，Nextdoor 专门经营本地社区。

经过她的团队的巨大努力，发射应该是一帆风顺的。但是，在离这个大日子只有两周的时候，Le 无法让这项新服务投入使用。

一天早上，当乐(音)用自己创建的一家假遛狗公司在自己的 iPhone 上测试这项新服务时，恐慌的时刻来了。一个主要功能没有工作。她给她的工程经理发了一条松散的信息，询问她是否应该担心。他们开始调试，查看她如何建立她的假业务，检查她正在使用的应用程序版本。

“最后(首席工程师)说，‘让我和团队商量一下，然后给你答复。’”乐说。“那天很晚的时候，他回应说，‘正如你今天早上所经历的，我们还没有准备好，我们需要更多的时间来测试和稳定。’因此，在发布前不到两周的时间里，我不得不与我的执行利益相关者沟通，我们需要更多的时间，我们的上市计划不得不稍微改变，因为我们的产品还没有准备好。产品体验不是可预测的质量，也不在它需要的地方。"

在推出新服务的过程中，拥有足够的时间一直是最大的挑战。

乐解释说:“我们努力快速地跑了几个月，你知道，构思创造，建立许多测试堵塞，试图找出我们应该做什么，不应该做什么。”。“我们有很多计划来增强我们的产品。与此同时，我们有计划减少技术债务，但在这段时间里，所有的事情都突然发生了。”

Nextdoor 的团队不知道在疫情期间人们是否会有稳定的手机服务或 WiFi。他们审查了关键流程和数据，以了解如何进行优化。他们通过查看网络包、捆绑包和包的大小来研究如何减轻负荷。此外，他们还添加了更多的仪器分析，以实现团队的实时视图，从而在客户开始注意到之前解决问题。尽管如此，仍然缺乏资源来完成所有的工作。

“我不得不向我的同龄人乞讨和借钱来借用资源，”乐说。“他们非常支持我，尽管我肯定他们不是很开心。因为，通过帮助我，他们的路线图被推迟了。”

团队如何在大型项目中工作是一回事，但正如 Le 所了解到的，清晰是重要的。否则，这通常意味着每个人都认为他们仍然必须做他们负责的所有其他工作，尽管老板可能根本没有想到。
