# 人工智能和全栈可观测性如何克服今天的 Kubernetes 挑战

> 原文：<https://thenewstack.io/how-ai-and-full-stack-observability-can-overcome-todays-kubernetes-challenges/>

[Dynatrace](https://www.dynatrace.com/) 赞助了这篇文章。

Kubernetes 为组织旨在提供的最终用户体验带来了实实在在的改善，这反过来又转化为更强的业务成果。但是，要从您的 Kubernetes 管理中获得最大价值，需要有能力应对我们在当今 IT 环境中看到的不断升级的复杂性。人工智能驱动的全栈可观察性是任何现代企业 IT 团队的必备条件，以便了解 Kubernetes 如何利用资源，部署的容器如何随着工作负载的变化而变化，以及如何相应地优化配置和性能。

### 利用人工智能推动整个 Kubernetes 的全栈可观测性

关于人工智能在企业 IT 中扮演的关键角色，已经写了很多。Kubernetes 也不例外，特别是随着容器的采用继续飙升。根据[最近对首席信息官](https://www.dynatrace.com/global-cio-report-part-2/)的调查，68%的组织已经在使用容器，86%的组织预计在未来 12 个月内部署它们。

随着 Kubernetes 和容器变得越来越普遍，IT 团队需要用确保这些动态环境真正为他们工作的能力来武装自己。这意味着能够从他们的技术堆栈中获得精确的性能下降洞察。这还意味着利用这些见解，在这些问题影响到最终用户之前，快速识别和修复这些问题背后的根本原因。这些问题解决得越快，用户体验和任何相关的业务成果就越好。

人工智能可以提供这些非常缺乏的能力。具体来说，一个人工智能引擎可以接收 Kubernetes 事件和指标，如工作负载变化，状态变化和其他关键事件。拥有人工智能是 IT 更好地理解 Kubernetes 堆栈中存在的依赖关系和关系的关键，包括集群、容器、服务网格和其中运行的工作负载之间的依赖关系和关系。如果实施正确，高保真全栈可观测性可以帮助组织消除复杂性问题，并使用 Kubernetes 推动改善数字转换和用户体验。

### 可观察性始于在 Kubernetes 环境中建立监控

但是，当我们谈论 Kubernetes 的全栈可观察性时，我们必须谈论监控——具体来说，就是作为 Kubernetes 环境的内置功能提供监控。

考虑这些环境中存在的各种技术。在他们的 Kubernetes 集群中部署容器的不同开发团队可能会将他们自己的工具带入组合中。这个集群越大，你就会看到越多的应用程序运行在各种不同的技术上；想想 JAVA，。net，Node，Go 等等。当您拥有这种多样性的技术时，您也可以让团队引入他们自己喜欢的监控工具。

单靠人工智能无法解决全栈可观测性问题，因为人工智能算法依赖于它们摄取的数据类型。因此，使用他们自己的工具进行监控的 IT 团队将会把不同质量的不一致数据输入到 AI 中。

数据不一致让 AI 效果降低；这就是“垃圾进，垃圾出”的难题。这使得将单个监控工具作为自助服务功能内置到 Kubernetes 集群中变得更加重要，该工具既支持开发人员部署的各种技术，又为人工智能提供基线水平的高质量数据，这些数据包含开发人员需要的细节水平以及 it 管理员需要的系统和事件监控水平。

### 管理 Kubernetes 环境的清单

人工智能驱动的可观察性为企业提供了一系列新功能，以更好地部署和管理他们的 Kubernetes 环境，并在生产中正确扩展他们的 Kubernetes 应用程序和微服务。其中包括:

*   从 Kubernetes 控制面板以及所有部署的容器访问日志。
*   分析容器资源使用情况，并深入研究当前工作的容器运行时。
*   发现、检测和映射在 Kubernetes 环境中运行的容器技术。
*   使应用程序所有者能够更快地识别和纠正性能下降和可扩展性瓶颈。
*   优化资源管理和分配。

这是你的 Kubernetes 清单。如果您不能实现或支持这些功能，那么您就不能从 Kubernetes 提供的功能中获得全部价值。不仅如此，您还可能面临环境中日益增加的复杂性和混乱，这将产生连锁效应，影响从用户体验到您纠正损害用户体验的问题的能力的方方面面。

不过，这个清单上的所有项目都可以通过将确定性人工智能模型作为技术堆栈的一部分来添加到 Kubernetes 工具箱中。一个能够吸收 Kubernetes 指标和事件的人工智能引擎可以推动更高水平的可观察性，以及基于这些见解采取的行动，这比传统指标、仪表板或人工 IT 支持现在所能实现的要多。现代企业云和容器化环境已经超出了这些传统解决方案的能力范围。

### 人工智能让你有你的蛋糕，也吃它与 Kubernetes

现代 IT 组织经常发现自己进退两难。一方面，Kubernetes 为这些组织提供了一种在当今企业运营的更加动态的基于云的环境中检测和协调工作负载的方法。另一方面，采用 Kubernetes 和这种更灵活、更动态的 IT 工作负载管理方式也意味着突然将更高层次的复杂性融入到您的生活中。这种程度的 IT 复杂性使得了解您的技术堆栈中实际发生的事情变得更加困难，从应用程序和微服务之间的相互依赖，到诊断和解决最终用户体验性能下降背后的根本原因。

AI 可以让 IT 团队在这两个极端之间穿针引线，将全栈可观察性构建到他们的 Kubernetes 平台、容器和工作负载中，并帮助组织从大规模环境中获得更智能、精确的答案。对于当今的企业 IT 而言，数字化转型意味着多云和混合云架构；这意味着变得更加敏捷；这意味着用更动态的微服务和容器化应用取代旧的整体结构。但这并不意味着更复杂。将 Kubernetes 与 AI 相结合，利用 Kubernetes 事件和指标实现全面的全栈可观察性，提供了两全其美的优势。
