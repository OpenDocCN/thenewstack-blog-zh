# CircleCI Insights 帮助 DevOps 团队量化高绩效

> 原文：<https://thenewstack.io/circleci-insights-helps-devops-teams-quantify-high-performance/>

所有的工程团队都想成为高绩效团队，但是他们如何真正知道呢？专注于开发运维的持续集成和持续交付(CI/CD)服务提供商 CircleCI 发布了一项新功能，帮助团队量化他们的绩效到底有多高。

CircleCI 高级产品经理[库纳尔·贾恩](https://www.linkedin.com/in/kunalsprofile/)说:“我们的想法是，我们可以为你提供关于你在 CircleCI 上做什么的可见性和更多信息。“假设你正在 CircleCI 上运行一个工作流或管道。堆栈中有很多不同信息的信息。Insights API 向用户提供 CircleCI 上运行的所有不同工具的状态信息，并记录使用信息。”

在日益分布式系统的世界里，Jain 说你不仅可以监控你的时间和状态，还可以观察你从内部和外部端点得到什么。一旦 CircleCI 转向这种基于信用的系统，其客户就继续渴望更多的信息，以更好地做出决策和优化他们的支出。

“给你能见度，给你更多的洞察力。您是想购买更多计算设备，让您的设备运行得更快，还是想对它们进行优化？”他说，并补充说“从我们平台的角度来看，这只是将所有数据返回给用户。”

Circle CI Insights API 为客户提供了每项工作所需的所有信息。

Jain 举了一个例子:“我必须运行一个测试套件，在 Docker 中型机器上需要 15 分钟，如果我将其切换到 Docker 小型机器上，则需要 16 分钟，我可以通过减少 10%来优化我的信用支出。”

通过 Insights API 的四个端点发回的数据应该有助于用户优化决策。

这些指标基于 [Google 的 Jez Humble 的四个关键 DevOps 成功指标](https://thenewstack.io/googles-formula-for-elite-devops-performance/)，针对您在 CircleCI 平台上运行的每个工作流和每个作业:

*   接通率
*   故障率
*   吞吐量
*   MTTR(平均恢复时间)

CircleCI 信贷支出覆盖了这一切，以了解其 CI/CD 渠道的效率和费用。Jain 称之为“鸟瞰图”,可以做出更好的决策。

该公司的客户之一，电子商务提供商 [Bolt](https://www.bolt.com/) 已经发现了这项新功能的价值。“我们使用 CircleCI 的 Insights 端点构建了一个仪表板，以全面了解我们的集成测试。我们目前配置它来显示我们的主要工作流和 repos 的健康状况，这有助于查明潜在的问题。我期待着随着时间的推移，用这些端点构建更多的用例，”Bolt 软件工程师 [Roopak Venkatakrishnan](https://www.linkedin.com/in/roopakv/) 在一份声明中说。

Jain 说，他们正在考虑更复杂的方法来在检测到异常时做出改变，包括未来的自动修复。

自从 [CircleCI 本周刚刚宣布](https://www.businesswire.com/news/home/20200407005283/en/CircleCI-Raises-100M-Series-Funding)另一笔 1 亿美元的 E 轮融资以来，路线图看起来将会得到很好的维护。
