随着各行各业的企业对云的采用激增，现代云场景本身变得越来越分散。 [Pivotal Cloud Foundry](https://pivotal.io/platform) ， [Red Hat OpenShift](https://www.openshift.com) ， [AWS](https://aws.amazon.com) ，[微软 Azure](https://azure.microsoft.com/en-us/?v=18.07a) ，[谷歌云平台](https://cloud.google.com)只是云的冰山一角。它们都有不同的元素、工具和数字性能监控方法，但是，它们之间的潜在因素是它们需要完美地工作，因为这是用户对它们的期望。

但是，当全球超过四分之三的首席信息官认为 IT 变得如此复杂，以至于威胁到他们有效管理数字性能的能力时，云环境的碎片化不仅仅表明了这种复杂性；这实际上加剧了它。

因此，对于企业来说，找到新的方法来整合他们监控云的方式至关重要。如果我们不能简化云并解决其复杂性问题——及其所有的连锁反应——对每个人来说，事情只会变得更糟。

### 新技术的采用加快了 IT 复杂性的步伐

一般的企业 IT 组合很快变得过于拥挤，需要运行和监控的东西越来越多，而且这一速度丝毫没有减缓。事实上恰恰相反。

超过 85 %(高达 97 %)的企业已经或计划在未来一年内部署一系列新技术。这包括采用移动应用、软件定义的网络、多云设置、容器和微服务。

所有这些新技术的采用最终也加剧了 IT 复杂性问题。仅在过去五年中，运行一项交易所需的平均系统数量就增加了 55%以上，从 22 个系统或组件增加到今天的 35 个。接触点的激增意味着他们在任何给定时间都有比以前更多的事情要监控。

### 在数字性能问题上花费了太多时间

由于这种日益增长的复杂性，企业维护云对其系统影响的端到端可见性的能力也变得前所未有的困难。而且，试图实现这种可见性会增加时间和成本。

每个主要的云提供商都有自己的监控系统。但是，这意味着当 IT 团队部署多个云提供商时，他们也要花费无数的时间来检测正在添加到其产品组合中的每个新云的监控流程。

72%的首席信息官表示，被迫为不同的提供商设置监控让他们的 IT 团队感到沮丧，在时间越来越稀缺的情况下，这又是一次不必要的时间浪费。

## 整合 DPM 以简化复杂性

那么，80%的首席信息官表示甚至很难弄清楚数字性能对其业务的影响，这有什么好奇怪的呢？

如果云使 IT 比以往任何时候都更加繁忙，并不断消耗资源，超过 70%的首席信息官表示，IT 因此无法满足对他们提出的不切实际的要求，那么这就损害了云应该为企业带来的价值。

这是不能接受的。在云优先的世界里，云不能被视为问题。

因此，答案必须是采用集成和整合这些不同数字性能监控服务的基础设施——无论它们来自 AWS、谷歌云平台、Azure、Pivotal 还是任何其他服务——以简化性能监控。

如果 IT 和首席信息官没有所需的可见性来了解他们的云之间的关系，以及这些云对整体性能和客户体验的影响，那么他们就没有真正获得这些云最初必须提供的价值。

### 企业不能只是云优先或云本地，他们必须能够感知云

成功的现代 IT 团队*必须*知道云如何影响他们公司的绩效和客户体验，以及如何改进和利用这种影响来实现更好的业务成果。

为此，如今企业采用能够整合其云性能监控并提供端到端可见性的解决方案至关重要，以确保他们知道云确实非常适合他们。
