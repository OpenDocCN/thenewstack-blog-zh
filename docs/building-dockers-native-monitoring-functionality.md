# Docker:容器监控不同于监控传统的应用程序

> 原文：<https://thenewstack.io/building-dockers-native-monitoring-functionality/>

在新一集的[中，新的堆栈制造商](https://thenewstack.io/podcasts/Makers)，Docker 生态系统开发总监 [John Willis](https://www.linkedin.com/in/johnwillisatlanta) 讨论了容器如何改变应用程序和系统监控的传统做法。TNS 电子书编辑 [Benjamin Ball](https://twitter.com/benballjr) 进行了采访，为我们的第五本电子书做准备:[用 Docker 和 Containers 进行监控和管理](https://thenewstack.io/ebookseries/)。

[基于 Docker 的本机监控功能](https://thenewstack.simplecast.com/episodes/building-on-dockers-native-monitoring-functionality)

就监控而言，重要的是要记住，容器具有多个周界，将它与主机分隔成它们自己的单独隔离。“许多新时代的供应商都很好地处理了这一问题，但您必须认识到主机类型(我称之为外部边界)和容器本身(内部边界)之间的区别，然后认识到，例如，“容器实例的 CPU 利用率是多少，主机的总利用率是多少？”所以你在那里会遇到一些有趣的事情，”威利斯说。

[Willis 指出，将传统的监控实践应用于容器化的微服务](https://thenewstack.io/qa-james-turnbull-art-monitoring-age-microservices/)可能会有压力，但并非没有解决方案。" [Dave McCrory](https://blog.mccrory.me/) 有一个我很喜欢的数据重力概念，我想说，新世界的枪支、宝石和钢铁实际上是容器和微服务中的数据重力。因此，在这个世界上，有许多计算工作负载，您只需将计算转移到数据中。”

许多人心中的一个问题是，Docker 将在监控生态系统中发挥多大作用。威利斯解释说，Docker(在撰写本文时)没有计划与更大的监控公司如 [Datadog](https://www.datadoghq.com/lpg/?utm_source=Content&utm_medium=Blog&utm_campaign=Blog-TheNewStackNotebooks2016) 或 New Relic 正面交锋。相反，它正专注于支持项目，如[生态系统技术合作伙伴](https://www.docker.com/partners/partner-program) (ETP)项目，专注于生产产品和服务，以通过一套新的解决方案来扩展 Docker 的监控功能。

人们看着 Docker 说，“好吧，Docker，他们如何与他们的合作伙伴一起工作？“很多人没有看到的是[Solomon【Hykes】](https://twitter.com/solomonstre)和[Ben【Golub】](https://twitter.com/golubbe)给了我和我的团队与所有这些合作伙伴一起工作的自由，并把所有时间花在真正不产生收入的东西上，但这有助于行业。它有助于 Docker 生态系统。”
