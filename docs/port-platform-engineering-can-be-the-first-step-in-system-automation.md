# 港口:平台工程需要一个整体的方法

> 原文：<https://thenewstack.io/port-platform-engineering-can-be-the-first-step-in-system-automation/>

到目前为止，几乎每个人都同意[平台工程](https://thenewstack.io/whats-platform-engineering-and-how-does-it-support-devops/)是[可能是一个好主意](https://thenewstack.io/cyberark-decreases-cognitive-load-with-platform-engineering/)，其中一个组织建立一个[内部开发平台](https://thenewstack.io/internal-developer-portal-what-it-is-and-why-you-need-one/)来授权编码员和加速应用程序发布。因此，在最新一期的[新堆栈播客](https://thenewstack.io/podcasts/)中，我们采访了该领域的先驱之一， [Port](https://www.getport.io/?utm_content=inline-mention) 的首席执行官 [Zohar Einy](https://www.linkedin.com/in/zohar-einy-b6219612a/?originalSubdomain=il) ，以了解平台工程将如何在您的组织中发挥作用。TNS 总编辑约阿布·杰克逊主持了这次谈话。

Port 为开发者提供了据称是世界上第一个低代码平台。

[端口:平台工程可以是系统自动化的第一步](https://thenewstack.simplecast.com/episodes/port-platform-engineering-can-be-the-first-step-in-system-automation)

通过 Port，组织可以构建一个经过批准的工具的软件目录，导入自己的数据模型，并设置工作流。开发人员可以通过自助服务目录使用他们需要的所有资源，而不需要了解如何设置复杂的应用程序，如 Kubernetes。开发人员和平台团队自己维护平台。

Einy 在我们的会议中指出，应用程序所有者并不是自助服务目录的唯一潜在用户。开发运维团队和系统管理团队也可以使用该平台。DevOps 团队可以设置自动化“以确保[开发人员]以正确的心态使用平台，在合规性、安全性和性能方面符合他们的组织标准。”

对于那些希望尽可能自动化部署的人来说，甚至机器本身也可以从自助服务平台中受益。

Einy 提供了一个例子:CI/CD 过程可以自己创建一个构建过程。如果它需要检查某个工具的成熟度，它可以通过 API 调用来完成。如果它没有得到充分的认证，开发人员会得到通知，但是如果所有的工具都足够成熟，那么自动化过程可以完成构建，而无需开发人员的进一步干预。

另一个可以自动化的可能过程是当许可的最后期限过去时终止许可。想想过期数字证书的预警系统。“因此，这是降低成本和安全最佳实践的一大驱动力，”Einy 说。

## 选择太多，代码不够

但是开发者的选择呢？当开发人员被禁止使用他们最喜欢的工具时，他们不会感到沮丧吗？

但是这种使用任何可用工具的自由是导致我们在全栈开发中过度复杂的现状的原因，Einy 回应道。这就是为什么“全栈开发人员”的角色看起来是不可能的，因为在栈的每一层都有所有可能的排列。

就像艺术家在有限的调色板中寻找灵感一样，开发人员应该能够在一个精心策划的平台中找到他们需要的一切。

“过去，当我们谈到‘你建造它，你拥有它’时，我们认为开发人员需要了解任何东西的一切，他们有充分的所有权来选择他们想要的任何东西。他们厌倦了，对吧，因为他们需要知道的太多了，”Einy 说。“因此，我认为我们正在进入一个过渡时期，开发者可以通过点击按钮获得他们需要的东西，因为他们自己有太多的工作要做。”
