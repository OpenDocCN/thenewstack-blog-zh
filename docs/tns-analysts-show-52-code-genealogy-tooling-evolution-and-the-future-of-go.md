# 代码谱系、工具进化和围棋的未来

> 原文：<https://thenewstack.io/tns-analysts-show-52-code-genealogy-tooling-evolution-and-the-future-of-go/>

Go 编程语言会走哪些路？在这期新的 Stack Analysts 播客的开始，这个问题被提给了 CoreOS 的首席技术官兼联合创始人[布兰登·菲利普](https://github.com/philips)和[迪福拉克](https://deferpanic.com/)创始人[伊恩·埃伯格](https://github.com/eyberg)，昨天在 [GopherCon](http://www.gophercon.com/) 录制。

更多剧集，请查看新堆栈的播客部分[。](https://thenewstack.io/podcasts/)

[#52:代码谱系学、工装进化与围棋的未来](https://thenewstack.simplecast.com/episodes/52-code-genealogy-tooling-evolution-and-the-future-of-go)

这个播客也可以在 YouTube 上看到。

Go 是现代系统语言。句号。它有作为系统语言的传统，历史将继续作为其未来的基础。

“我们谈论的是不编写内核的系统——不编写直接与硬件对话的底层、低级位——而是跨数据中心或跨互联网交互的系统，”Brandon 说。

“酷的是，”布兰登说，“这是一项技术，它使这些领域的许多有趣的研究和开发成为可能。因此，你会看到时间序列数据库正在构建，你会看到这些一致的数据库正在构建，调度程序，以及与网络结构对话和配置网络结构的东西。”

Ian 说，Go 有一致的语法，很容易学习，他可以在一个环境和另一个环境中阅读它，并看到相同的风格。一旦学会了，你就可以在任何地方用简单易懂的语言阅读它，Ian 说，而 Erlang 是一种有一些相似之处的语言，但是当人们看到它时，很难理解他们在看什么。

Golang 核心团队之外的人可以导入外部包，Brandon 认为这是一个好的、有趣的、深思熟虑的想法。随着 Go 语言和工具的成熟，语言本身可以保持精简。

据伊恩，包装一般已成为一个有争议的问题。

Alex 建议说，仿制药也是如此。

“我认为一小部分人对[仿制药]有问题，”Ian 说。“缺乏这种能力的人有一个问题——他们太习惯于不同于其他语言的编程风格，这种风格允许他们自由地混合和匹配他们的代码，即使这样做可能不是最好的主意。”他还说，JavaScript、Ruby 和其他语言也有这个问题。因为这是一个相当普遍的问题，所以在 Go 社区中没有必要对此深感忧虑。

Brandon 说，对于 CoreOS 来说，围棋中最难的是依赖性管理。“每次 CoreOS 创建一个新的开源项目时，它都会复制。我们必须将这些东西捆绑在一起，重新出售。”

尽管一直需要更好的工具，Brandon 看到了一个令人鼓舞的趋势。“在过去的一年里，我们经历了许多不同的工具。我们编写了一个销售工具，其他销售工具已经出现，每个人都围绕着一组模式，Go 核心团队正在将这些模式应用到实际的主线中。”

然而，亚历克斯认为，这些工具必须包罗万象。布兰登对此表示赞同，并补充道，“这不仅仅是获取源代码和生成机器码。”

Ian 确信，从长远来看，也许在几年内，Go 将会取代 Java。但是，一些人的妥协仍然存在，包括不同的方言，“不管核心团队多么不喜欢这个想法，”伊恩说。“我已经在野外看到过例子了。”

最后，核心团队发布了 Go to the community，处理善后可能是最困难的部分，因为与一个从隐式信任转向显式信任的社区合作变得至关重要。

CoreOS 是新堆栈的赞助商。
