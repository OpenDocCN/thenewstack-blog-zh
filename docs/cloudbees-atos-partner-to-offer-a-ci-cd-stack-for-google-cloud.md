# Atos 合作伙伴 CloudBees 将为谷歌云提供 CI/CD 堆栈

> 原文：<https://thenewstack.io/cloudbees-atos-partner-to-offer-a-ci-cd-stack-for-google-cloud/>

对于那些选择谷歌云以及新的内部部署的组织，服务和解决方案提供商 Atos 和 Jenkins 提供商 T2 Cloud bees 表示，他们已经开始联合提供集成服务，作为持续集成和交付(CI/CD)的托管平台。

如果 Atos 和 CloudBees 确实能够提供一个可行的服务模型，DevOps 团队将在一个包中包含许多组件，用于在 Google 云和内部进行大规模生产部署。詹金斯工具，Kubernetes 管理界面，谷歌云集成和其他各种各样的工具选择和管理任务将被合并，例如 DevOps。

CloudBees 和 Atos 的最终目标是帮助 DevOps 团队解决跨数据中心和多云的整合、集成和自动化挑战。这两家公司在本周于三藩市举行的 [Google Next 大会](https://cloud.withgoogle.com/next/sf)上宣布了合作关系。

[Enterprise Management Associates(EMA](https://www.enterprisemanagement.com/))的分析师 [Torsten Volk](https://www.linkedin.com/in/torstenvolk) 说:“技术孤岛仍然是当今最大的 DevOps 疾病，DevOps 团队需要寻找一切可以帮助他们将讨厌的遗留组件引入企业 DevOps 管道的东西。

更具体地说，Atos 和 CloudBees 共同表示，他们为谷歌云提供以下服务:

*   Atos 的应用程序转换服务可帮助团队实现应用程序以及应用程序交付和管理方式的现代化；
*   CloudBees 的云原生 CI/CD 平台，由 Jenkins 和 Jenkins X 提供支持，用于云中或内部的 Google 云技术；
*   谷歌为 Kubernetes 和 Istio 提供云技术。

CloudBees 的联合创始人兼首席战略官 Francois Dechery 在一封电子邮件回复中表示，开发人员将通过可定制的 CloudBees CI/CD 平台 Kubernetes-native 以及由 Jenkins 和 Jenkins X 提供的支持，“获得持续和最新的 CI/CD 服务”，同时“结合了便捷和安心的“即服务”体验。”

“开发商和他们的经理可以百分之百地专注于增加业务价值，并依靠 Atos 来处理 CI/CD 平台的运营管理，”Dechery 说。此外，Atos 还提供应用程序现代化服务，帮助开发人员改造现有应用程序或创建新的应用程序，以更好地利用基于 Kubernetes 和 Istio 的现代谷歌云原生平台。”

Atos 和 CloudBees 还寻求帮助中小型企业寻找经济实惠且易于实施的工具，以简化 CI/CD 流程以及云本地和内部部署。

“中型企业无法获得跨不同技术平台集成和自动化开发运维及应用生命周期所需的资源。沃尔克说:“这些企业在数字化过渡期正努力提高安全性、合规性、性能和可用性。“关键是要明白，这一时期将会持续很多年，大量传统应用将会暂时保留下来。将这些应用从 DevOps 生命周期中剔除会带来运营风险，这种风险会随着时间的推移而不断增加，并且会阻止开发人员编写与传统组件相关联的现代云原生应用。”

Volk 说，这些服务可能对 DevOps 团队有用，正在进行测试，并提供更多有关该产品的细节。“如果 Atos 和 CloudBees 能够通过 GCP 实现一个集成良好的 DevOps 平台，将数据中心和公共云联系在一起，这将是一个有趣的价值主张，”Volk 说。

得益于这些新平台的混合特性，Istio 和 GKE 可能会紧密整合。“为了更深入地进行思考，这个平台也将很好地将谷歌的大数据、机器学习和人工智能能力引入企业 DevOps 管道，”Volk 说。

CloudBees 是新堆栈的赞助商。
