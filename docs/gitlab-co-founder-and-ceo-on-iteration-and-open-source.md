# GitLab 迭代和开源联合创始人兼首席执行官

> 原文：<https://thenewstack.io/gitlab-co-founder-and-ceo-on-iteration-and-open-source/>

GitLab 赞助了这个播客。

迭代软件开发模型可以帮助组织提高敏捷性和生产流水线的效率。 [GitLab](https://about.gitlab.com/) 是一个成功利用迭代并应用其经验教训的企业的例子——为自己的开源项目以及开源社区做出贡献和提供支持。

在最新一集的 [The New Stack Makers](/podcasts/makers) 播客中，New Stack 的创始人兼发行人 [Alex Williams](/author/alex/) 采访了 GitLab 的联合创始人兼首席执行官 [Sid Sijbrandij](https://www.linkedin.com/in/sijbrandij) ，谈论迭代、开源项目——包括 Meltano 和 Kubernetes——以及 SpaceX 在硬件行业的迭代开发流程可以给软件行业带来什么启示。

[不久前成功完成](https://www.spacex.com/)[美国宇航局自 2011 年以来的首次载人太空任务](https://www.space.com/spacex-crew-dragon-demo-2-home-port-arrival.html)的 SpaceX ，是迭代工作的典型案例研究。Sijbrandij 解释说，除其他外，SpaceX 已经向硬件和软件部门展示了如何“以复数形式而不是单数形式构建应用程序”。

SpaceX 的迭代也适用于其团队如何思考他们正在做的事情，以及它将如何影响他们的合作伙伴。Sijbrandij 说:“在 GitLab，我们希望确保开发人员、安全人员和运营人员聚集在一起，为一个平台做出贡献，并且使用 GitLab 的 10 万个组织感觉每个月都有能力做出贡献。”“我们有来自更广泛社区的 300 多项贡献，最终，这是为了让人们有能力做出贡献。”

也有一些迭代不一定有效的情况。Sijbrandij 说，一个项目可能不值得追求的测试案例是“我们做了一些东西，但没有得到任何用户”。

“我认为，作为一个组织，倾听是非常重要的，如果某样东西没有用途，我们就不应该再投资，我们应该把投资放在其他地方，”他说。“我认为没有人使用某个东西的问题可能会发生在任何事情上，我认为如果你迭代，那么至少有少量工作没有被使用，这就好得多了。”

Sijbrandij 解释说，有时一个项目可能“在交付之前就被放弃了”。

“因此，如果一个(项目)被取消或太难，而且工作量很小，这是一个小得多的问题，”他说。“这是我们从客户那里看到的一个更好的问题，因为他们迭代得越多，取消对他们能够完成的工作总量的影响就越小。”

Sijbrandij 说，GitLab 将继续投资于其开源数据管道管理项目 Meltano，因为“你不会放弃，但有时你必须改变你的方式”。“应该有一个开源 ETL 框架来加载数据的前提仍然是正确的，所以我们将继续投资，直到我们确信我们的前提首先是错误的。”

在大范围内，Kubernetes 可以为开源社区提供关于一个极其成功的项目的动态的见解，尤其是在 Kubernetes 日益成熟的今天。

“Kubernetes 变得越来越无聊，这很好，因为这意味着它是成功的。我不与 Linux 内核的细节进行交互，但我每天都在使用它，”Sijbrandij 说。“我们看到人们不再与 Kubernetes 直接互动。”
