# 牧场主如何帮助 Kubernetes 投入生产

> 原文：<https://thenewstack.io/rancher-labs-seeing-kubernetes-put-work-production/>

在最新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/Makers) 播客中， [Rancher Labs](http://rancher.com/) 首席软件工程师 [Alena Prokharchyk](https://twitter.com/lemonjet) 和 Kubernetes 用户 [Brian Scott](https://twitter.com/brainscott) ，目前是迪士尼的系统工程经理，在 Kubecon 2016 上与 TNS 创始人 Alex Williams 坐下来讨论 Kubernetes 在生产环境中的使用，以及 Rancher 如何帮助这种部署。

RancherOS 是一个 Linux 发行版，它本身支持和管理 Kubernetes 容器编排引擎，以及 Mesos 和 Swarm 编排器。Rancher 还提供[牛](https://github.com/rancher/cattle)，这是它自己开发的基础设施编排引擎。

Kubernetes 和 Rancher“为我们提取了许多关于供应节点和负载平衡的问题。Kubernetes 为我们提供了很多围绕其他工具编写的秘方。有了 Rancher，您可以使用任何您想要的编排平台，并让每个人都满意。这是一整块玻璃。你可以感受到任何类型的编排，可能适合某种类型的应用程序风格，”斯科特说。

Prokharchyk 强调了那些在多平台 Kubernetes 部署中与 Rancher 合作的人可以预期的一些附加功能，包括铺设用户覆盖网络、部署 Kubernetes 系统堆栈、替换生病或死亡的节点、设置内部 DNS 用于服务发现以及监控集群健康的能力。

“我们能够使用 Rancher 的一些挑战是消除 Kubernetes 的所有困难设置。现在，它更像是一个按钮设置，”斯科特说，并补充说，尽管如此，Kubernetes 仍然有一些特殊的增长领域需要解决。其中包括存储、角色管理和访问控制。

[![2016-11-30_grafanacon_nyc_pancakepodcast_eventspagepromo](img/dd03df97ae73062155f47235f5cb3e47.png)](http://grafanacon.org/)

随着 Rancher Labs 的不断发展，展望未来需要充分利用 Kubernetes。“我们希望探索用于 Kubernetes 集群部署的所有工具。我们想学习他们的经验，我们想跟随 Kubernetes 社区。看看他们增加了什么功能，Kubernetes 系统集群和管理应该改变什么，主要集中在 Kubernetes 架构上，以使用户体验更容易，”Prokharchyk 说。

当对话接近尾声时，Scott 提供了一个关键的收获，对于任何希望在生产中充分利用 Kubernetes 的开发人员来说都是至关重要的。
