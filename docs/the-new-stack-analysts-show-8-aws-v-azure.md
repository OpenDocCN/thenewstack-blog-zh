# AWS 与 Azure

> 原文：<https://thenewstack.io/the-new-stack-analysts-show-8-aws-v-azure/>

与 Rackspace 已经离开 IaaS 提供商市场的消息形成对比的是，微软和亚马逊最近的财报显示，这两个主要参与者正在进一步进入这个市场。但是有多舒服呢？根据他们透露的收入情况，很难说，但我们可以看看他们的历史。虽然众多竞争者争夺市场份额，但谷歌、Azure 和 AWS 似乎正在进行一场漫长、缓慢的三重对决。还是僵持不下。无论它可能如何展开，它都会引发引人入胜的讨论。

亚历克斯·威廉姆斯(Alex Williams)和共同主持人迈克尔·科特(Michael Coté)与南希·戈林(Nancy Gohring)谈论了微软 Azure 的成功，它对 AWS 可能意味着什么，以及它对苹果和其他云计算竞争对手可能意味着什么。

## 显示注释

微软和亚马逊都没有很好地公布他们的数字，所以对于具体行业的表现，大部分都是猜测。两家公司都在报告云市场的增长。

微软看到了云服务和服务器销售的增长，坚持其“混合”故事线，即客户既是增长的内部数据中心，也是扩展的云工具。

微软目前可能占据上风，因为亚马逊对桌面即服务等更高级别的服务相对较新。

亚马逊报告的云服务 90%的增长是原始使用量，不一定是收入；降价肯定会影响收入，华尔街似乎给亚马逊开了绿灯，让它在定价上玩长线游戏，这对亚马逊有利。

人们在这些云服务上部署了什么—定制应用程序、通用存储、旧的 VMware？亚马逊希望成为通用供应商，而微软则希望延续其传统产品。未知因素是谷歌，可能是纯粹的开发者云。

云的长期、非存储构成？取决于套装软件能坚持多久，SaaS 可能会在 10 年内完全接管。在 Azure、AWS 和谷歌之间，这将归结为开发者的投入和策略。

这可能是这些巨头之间一场漫长而缓慢的战斗，以免我们忽视苹果最近的举动。IBM/苹果的交易可能涉及令人满意的商业应用程序，但更大的公司将在内部定制开发。

桌面即服务听起来不错，但最终用户并不满意。存在几个大型安装，但是它们适合管理(约束)低层用户。高质量的虚拟桌面与实际桌面一样昂贵。桌面即服务的价值在于能够捆绑外围设备资源。

桌面即服务似乎不愿消亡，但倡导者在哪里？这是 AWS 在两面下注吗？亚马逊在基础设施上投入了大量资金，这意味着部署和消费者进入的低成本壁垒；但是营销和支持是不够的，那么谁会买他们提供的东西呢？

VMware 和 Citrix 都在鼓励工作空间即服务市场，但他们都是长期利益相关者。

AWS 试图在后端和中间件上深入。市场无法支持所有的大小供应商。AWS 和 Azure 正在自己构建这些服务，而不是收购较小的公司。

小公司保持私有的趋势，这可能会给他们更多的持久力来完成与大公司的长期合作。与以往一样，云安全仍然是许多人关心的问题。

显然，微软的客户忠诚度在这一阶段得到了回报，而 AWS 的客户受到了价格冲击，较小的公司正试图提供企业交易。

尽管获得了金星奖，但 Michael 仍然经常情绪低落，并坚持认为公共云是纯洁的。
