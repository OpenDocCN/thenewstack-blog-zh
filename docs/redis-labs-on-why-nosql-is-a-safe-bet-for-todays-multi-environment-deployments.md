# Redis Labs 解释为什么 NoSQL 是当今多环境部署的安全之选

> 原文：<https://thenewstack.io/redis-labs-on-why-nosql-is-a-safe-bet-for-todays-multi-environment-deployments/>

我们可以相当肯定地说:组织正在由传统数据中心和通常不同的云服务组成的混合环境中部署应用程序，而允许他们这样做的开源业务模型正在发生变化。Redis Labs 首席产品官 Alvin Richards 说，在这种情况下，对于数据库管理和使用来说，选择 NoSQL 仍然是当今部署的安全选择，尤其是对于多云环境。

在亚马逊网络服务(AWS) [re:Invent 2019](https://live.awsevents.com/reinvent/) 期间在拉斯韦加斯现场录制的播客中，Richards 讨论了数据库和开源是如何发展的，以及 Redis Labs 是如何调整其 NoSQL 产品和商业模式的。在线旅行社 HolidayMe 的首席技术官 [Rajat Panwar](https://www.linkedin.com/in/rajat-panwar-4a9b3510/?originalSubdomain=in) 也加入了他的行列，他也是 Redis Labs 的客户。

[Redis Labs 关于为什么 NoSQL 是一个安全的赌注](https://thenewstack.simplecast.com/episodes/redis-labs-on-why-nosql-is-a-safe-bet)

“人们正在采用技术和超高速率，但他们也在甲骨文、[IBM] Db2 和大型大型机方面进行了巨额投资，这些投资不会消失，”理查兹说。“所以，我认为关键是，无论你在云服务方面有什么，你都必须有一个混合环境，”由内部部署和通常的多云基础架构组成，需要能够适应不同部署模式的技术。

因此，横向扩展关系数据库可能会有问题。理查兹说，这是“因为你最终不得不加入你所有的服务”。“因此，有了 NoSQL 的概念，就可以说‘好吧，我们将使用不同的数据模型来表示不同的内容，这样我们就可以分发数据了。’"

他说，对于 Panwar 来说，Nosequel 数据库“给了你一种灵活性，如果你想拥有某个功能，那么很好，你可以添加几列，然后就可以运行了。”

潘瓦尔说，他的团队对图书馆的重新研究特别感兴趣。根据 Panwar 的说法，HolidayMe 能够在短短几天内建立并运行该模块。

与此同时，Redis 实验室也调整了其许可模式，因为另一家大型云提供商 AWS 已经开始提供开源代码服务，直接或间接与开源软件供应商竞争。Richards 说，云提供商的 SaaS 和其他服务模式“导致了许可模式的发展和变化”，但他没有提到 AWS

“公司不得不重新思考开源的真正含义和云服务的概念。理查兹说:“随着市场的变化，这个领域的所有公司都在不断发展自己的思维。“我认为，如果我们在五年后进行这样的对话并回顾过去，我们会发现世界在未来五年也会发生变化。我认为目前还不稳定，而且我认为许多像我们这样的公司以及其他数据库和 NoSQL 公司的地位也因为时代的不断变化而发生了变化。”

TNS 出版人亚历克斯·威廉姆斯主持了这次播客，TNS 欧洲记者 T2·b·卡梅隆·盖恩(T3)提供了协助。
