# Kubernetes 和云原生数据管理

> 原文：<https://thenewstack.io/why-cloud-native-data-management-day-is-about-stateful-data/>

数据管理不再像最初那样被认为是一个短暂的概念，它已经成为一个巨大的问题和挑战，尤其是在 Kubernetes 环境中管理无状态数据。

数据管理的状况是本期 New Stack Makers 播客的讨论主题，由 New Stack 的创始人兼发行人 Alex Williams 主持。嘉宾是高级全球技术专家[迈克尔·凯德](https://www.linkedin.com/in/michaelcade1/?originalSubdomain=uk)和[奈杰尔·波尔顿](https://fr.linkedin.com/in/nigelpoulton)，奈杰尔·波尔顿是[nigelpoulton.com](https://nigelpoulton.com/)的所有者，后者提供 Kubernetes 和 Docker 培训及其他服务。Cade 和 Poulton 还参与了云原生数据管理日的组织工作，这是上个月与 KubeCon+CloudNativeCon EU 联合举办的一项活动。

[为什么云原生数据管理日是关于有状态数据的](https://thenewstack.simplecast.com/episodes/why-cloud-native-data-management-day-is-a-about-stateful-data)

除其他外，同处一地的活动揭示了随着实践和解决方案的出现，云原生数据管理作为一个概念正在走向终结。“云原生管理日反映了目前技术的发展状况，我们现在可以做的不仅仅是谈论东西。Poulton 说:“实际上，我们有时可以在容器和 Kubernetes 的环境中使用这种数据管理来进行数据处理。“但同时，对它的需求也很大，因为我认为它的用途已经超出了——听起来很俗气，对吧？—但是你知道，那就像任何人都可以做的无状态应用程序一样容易实现。除此之外，我们还在不断进步，我们希望将更严肃的应用程序放到容器中，并用 Kubernetes 来管理它们。”

Poulton 说，虽然 Kubernetes 上的容器数据管理“对每个人来说都是新的，但我认为真正有趣的是，许多公司现在都加入了数据管理领域的派对”。
