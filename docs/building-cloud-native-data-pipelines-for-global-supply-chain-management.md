随着在线购物继续以每年 15%到 20%的速度增长，全球零售商越来越多地面临将产品运送到他们需要去的地方的复杂性挑战。如果公司希望创造回头客，那么从生产到最后一英里送货到顾客家门口，有许多供应链挑战必须克服。

许多企业求助于领先的供应链管理公司 [Elementum](https://www.elementum.com/) ，来协调他们的全球供应链运营，并帮助他们满足客户的即时满足期望。

Elementum 的产品图描绘了全球 25 万亿美元的产品经济，以揭示世界各地的货物流动，并促进跨生态系统事件管理，以确保产品在正确的地点、时间、数量和成本可用。Elementum 成功的一个关键是:在基于云的平台上部署微服务模型。

Elementum 的工程开发主管丹尼尔·阿维拉(Daniel Avila)肩负着为该公司的大规模全球数据收集和共享业务构建技术基础的艰巨任务。两年前，他上任后的第一项任务就是摆脱公司单一的基于应用的平台，将未来的发展转向更灵活、更易于扩展的微服务模式。

“我们从包括物联网设备和第三方在内的各种来源收集并整合数据，很明显，要获得我们需要的企业解决方案，我们最好的选择是使用[中间层 DC/操作系统](https://mesosphere.com/product/)，”Avila 说。

在 Elementum 不断发展的基于云且不可知的堆栈中，传统应用被迁移到 DC/操作系统。这一举动促使 Avila 半开玩笑地说，他的目标是通过创建一个数据平台来“一环统治一切”，该平台将与大型客户已经到位并希望继续使用的任何云基础设施良好合作。

“我们是百分之百的云，”阿维拉说。“我们甚至没有真正的数据中心。”

通过 Kafka、Elastic Search、Redis 和 DSE 等组件，Elementum 能够调整流数据并实现近实时更新，这些数据可以在优雅的仪表板上以最小的开销与客户共享。

“对我们来说，最具挑战性的事情是创建一种标准的方式来表示数据，”Avila 说。"为每个客户做定制开发项目是不可扩展的."

Avila 希望继续将工作负载、数据管道和数据摄取操作转移到 DC/OS 上，并参与机器学习。首要目标是确保 Elementum 客户能够查看、分析、管理和优化他们的整个供应链和交付物流，以便他们的产品能够顺利制造、装箱、装运并交付到家门口，而不会遇到任何障碍。

实现这一目标意味着保持对云技术和云不可知论的高度关注，同时规划[一条通往多云实施](https://d2iq.com/blog/dcos-1_11-hybrid)的道路，通过冗余实现可扩展性、灵活性和风险缓解，或许最重要的是，这种适应性对于在这样一个快节奏和不断变化的行业中取得成功至关重要。
