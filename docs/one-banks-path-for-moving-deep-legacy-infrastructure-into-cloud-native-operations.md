# 一家银行将传统基础设施迁移到云原生运营的途径

> 原文：<https://thenewstack.io/one-banks-path-for-moving-deep-legacy-infrastructure-into-cloud-native-operations/>

KubeCon+CloudNativeCon 赞助了这个播客，作为对 Kubernetes 终端用户的一系列采访的一部分。听听之前的故事[从一台服务器到 Kubernetes，一个创业公司的故事](/from-one-server-to-kubernetes-a-startups-story/)和[Box 的 Kubernetes 之旅的跌宕起伏](/the-ups-and-downs-of-boxs-kubernetes-journey/)。

一些传统基础架构无疑比其他基础架构更难以转变为云原生操作。例如，金融服务面临许多法规、合规性和数据管理方面的挑战，如果公司拥有特别大的国际业务，这些挑战会进一步加剧。

在本期的[New Stack Analysts](/podcasts/analysts)播客中，作为 New Stack 最近关于终端用户 Kubernetes 报道的一部分，总部位于东京的投资服务公司 [MUFG](https://www.mufg-investorservices.com/) (三菱 UFJ 金融集团)副总裁、高级创新工程师 Michael Lieberman 讨论了他的公司在金融服务领域的微服务和 Kubernetes 环境中向外扩展架构的历程。 [Alex Williams](/author/alex/) ，The New Stack 的创始人兼发行人，与[Cloud Native Computing Foundation(CNCF)](https://uk.linkedin.com/in/cheryljhung)的生态系统副总裁 Cheryl Hung 和[Spotify](https://www.linkedin.com/in/dzolotusky/)的高级职员工程师 Dave zolotsky 共同主持了播客。

[一家银行将深层遗留基础架构迁移至云原生运营的路径](https://thenewstack.simplecast.com/episodes/one-banks-path-for-moving-deep-legacy-infrastructure-into-cloud-native-operations)

从一开始，MUFG 向云原生基础设施的过渡就涉及到团队成员运营方式的文化转变。Lieberman 说，采用云原生工具和技术来帮助满足国际银行业的法律和监管要求，需要对其要求更加开放。

利伯曼解释说，在过去，银行通常将他们的工具和技术选择“非常保密”。这可能会在几个方面造成问题。

利伯曼说，有些“情况下，我们可以使用的工具和技术可能无法使用，因为开发它们的人不知道我们的需求或不知道我们的需要”。“因此，我们正在努力做的事情是，我们试图以更开放的态度讨论这些需求，并与其他供应商和整个开源社区讨论这些需求。”

Lieberman 说，作为一家金融服务公司的典型情况，MUFG 的堆栈由很大一部分传统基础设施组成，当然包括堆栈中有大量 Java 编写的代码的“旧大型机”。

MUFG 向云原生的转型包括采用 Red Hat 的 OpenShift 来帮助平滑流程。与此同时，MUFG 正在探索许多不同的选项和选择。“因此，我们仍在探索香草 Kubernetes 之类的东西…我认为我们仍处于相当早期的旅程…我们仍在评估许多其他云原生工具和技术。”

然而，尽管 MUFG 在很大程度上处于向原生云转变的早期阶段，但 Kubernetes 已经“对我们起到了巨大的推动作用，”他说。

“Kubernetes 允许我们做的一件大事就是使用 API 开发框架。[Kubernetes]使我们能够…更快地将反馈回路传递给开发人员——你可以想象，大型机并不是真正为快速变化而设计的，”Lieberman 说。

“也就是说，转向 Kubernetes，我们现在已经能够设置它，开发人员可以对我们正在运行的数百个微服务中的任何一个进行更改，我们可以开始在我们的开发环境中针对这些微服务运行一整套测试。”

云计算原生计算基金会和 Red Hat 是新堆栈的赞助商。
