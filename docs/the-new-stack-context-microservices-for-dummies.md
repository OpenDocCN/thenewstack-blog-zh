# 新的堆栈环境:虚拟微服务

> 原文：<https://thenewstack.io/the-new-stack-context-microservices-for-dummies/>

CloudBees 赞助了这个播客。

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。本周，我们与[凯尔·戴维斯](https://www.linkedin.com/in/kyle-davis-9336085a/?originalSubdomain=ca)、 [Redis 实验室](https://redis.com/)开发者权益负责人和[Redis 实验室开发者权益经理洛里斯 Cro](https://kristoff.it/) 就他们的新电子书《 [Redis 虚拟人微服务](https://redis.com/docs/redis-microservices-for-dummies/)》进行了交谈我们讨论微服务领域对数据库系统的新要求，以及数据流的出现。Libby Clark，The New Stack 的编辑和营销总监，主持了这一集，其中还包括 TNS 创始人兼出版商 Alex Williams 和 TNS 执行主编 Joab Jackson 的意见。

[第 98 集假人的微服](https://thenewstack.simplecast.com/episodes/episode-98-microservices-for-dummies)

在本书的[摘录中，Davis 和 Cro 写道](/how-redis-fits-with-a-microservices-architecture/) [Redis NoSQL 数据库](https://redis.io/)可以以三种主要方式用于微服务:作为存储的可扩展数据库，用于使用 Redis 流的微服务之间的通信，以及作为缓存以减少单个服务响应其自身请求的延迟。“理想情况下，服务数据将与其他数据层完全隔离，允许非耦合扩展和跨服务争用慢速资源。他们写道:“由于服务是专门为满足单一角色(就业务流程而言)而设计的，它们存储的状态本质上是非关系的，非常适合 NoSQL 数据模型。

然后，在节目的稍后部分，我们讨论本周的热门播客和新闻，包括关于 AWS Lambda 和 New Relic monitoring 如何[最大限度地减少对基于容器的架构的需求](https://thenewstack.io/how-aws-lambda-became-what-it-is-today/)的讨论，本·西格曼[对基于微服务的“深度系统”的出现的想法](/qa-ben-sigelman-on-the-emergence-of-deep-systems-in-microservices/)，[在网络管理平面上缺乏创新](https://thenewstack.io/network-automation-and-the-lack-of-innovation-in-the-management-plane/)，以及云计算原生计算基金会快速的新分布式文件系统 [ChubaoFS](https://thenewstack.io/chubaofs-the-cloud-native-computing-foundations-speedy-new-distributed-file-system/) 。
