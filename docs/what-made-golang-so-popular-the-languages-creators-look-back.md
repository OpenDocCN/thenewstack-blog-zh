# 是什么让 GoLang 如此受欢迎？这种语言的创造者回顾过去

> 原文：<https://thenewstack.io/what-made-golang-so-popular-the-languages-creators-look-back/>

自从 2009 年开源的那天起， [Go 编程语言](https://go.dev/)就越来越受欢迎。现在，最初创建[的五名谷歌软件工程师正在回顾是什么*推动了*的增长](https://cacm.acm.org/magazines/2022/5/260357-the-go-programming-language-and-environment/fulltext#R16)。

在 ACM 的*通讯中，Go 的五位原创者强调，即使是他们最早的作品也“从谷歌许多同事的建议和帮助中受益匪浅”论文的第二句强调，它现在是一个公共项目，“有成千上万的个人和几十家公司的贡献。”*

在强大社区的支持下，Go 已经获得了不可否认的广泛普及。在最近的 Tiobe 指数中，Go 被排在第 14 位。

他们的论文指出, [Docker](https://thenewstack.io/category/containers/) 和 [Kubernetes](https://thenewstack.io/category/kubernetes/) 都是用 Go 编写的，并补充说该语言也是“每个主要云提供商的关键基础设施的基础，并且是[云本地计算基金会](https://cncf.io/?utm_content=inline-mention)托管的大多数项目的实现。”

但是一个更有趣的问题是*为什么*围棋变得如此流行…

这篇论文认为是 Go 的“以发展为中心的哲学”帮助了它的社区蓬勃发展，然后赞扬了这个社区——以及它所建立的技术——最终使 Go 成为“现代云计算环境的重要组成部分”

简而言之，当回顾过去 13 年时，Go 的创造者们相信这种语言的成功是通过训练其对软件项目得到工程化的“整体环境”的关注。" Go 的方法是将语言特征视为不比环境特征更重要."

这篇论文的作者是[拉斯·考克斯](https://twitter.com/_rsc?lang=en)、[罗伯特·格里斯默](https://github.com/griesemer)、[罗布·派克](https://twitter.com/rob_pike?lang=en)、[伊恩·兰斯·泰勒](https://www.linkedin.com/in/ianlancetaylor/)和[肯·汤普森](https://www.computer.org/profiles/kenneth-thompson)。

## 早期

Go 来自谷歌，这并没有什么坏处，仅仅是这家搜索引擎公司在 1998 年首次推出的 10 年多一点。Go 的压缩二进制文件也更容易部署——因为(与 Java 二进制文件不同)它们不需要单独的运行时环境来执行。

其他语言特性也有助于使其更具吸引力，因为 Go 是包含“垃圾收集”特性的语言之一，可以自动释放变量不再使用的内存。为此，该论文指出，Go 利用了新的多核处理器，在专用核上运行垃圾收集，以降低对延迟的影响。

这种并发性是作为语言的核心部分提供的，而不是作为一个单独的可选库。事实上，这在很大程度上解释了为什么 Go 是以这种方式构建的。论文中题为“起源”的一节描述了 Go 如何从谷歌的经历中成长起来，拥有约 4000 名活跃开发者共享的庞大多语言代码库。

他们的日常经验表明，需要一种更好的方法来处理大规模负载，利用新多核芯片的能力。回过头来看，这种语言的创造者写道，Go“是我们对为应对这些挑战而设计的语言可能是什么样子的问题的答案。”Go 是专门为并发性和并行性提供顶级支持而设计的——这意味着 Go 不仅可以有效地处理多个任务，还可以同时执行多个任务。

这篇论文认为，在 Go 出现之前，工程师们一直受困于使用笨拙的语法和“大型、固定大小的线程堆栈”，而支持并发的线程不受欢迎，因为它们难以创建、难以使用和难以管理。一个脚注甚至引用了 TCl 脚本语言的创始人约翰·奥特 1995 年的一篇论文，题目是“为什么线程是一个坏主意(对于大多数目的)。”

“解决这种紧张是创造围棋的主要动机之一，”他们写道，后来称之为语言的“主要不寻常的属性……”

## 超越语言设计

该论文还承认，Go 今天的流行部分是因为更广泛的技术行业现在经常使用云服务提供商——以及他们支持的大规模生产环境(Go 旨在解决这些问题)。还有其他一些明显的优势。

后来，这篇论文指出 Go“移除了在 C 和 C++程序中导致许多问题的未定义行为。”(例如，如果代码尝试一些有风险的事情，如取消引用空指针或使用超出数组或片边界的索引，Go 会抛出运行时异常并停止运行程序。)

但是围棋的创造者将围棋的流行归因于其他东西。他们强调“更重要的是为打包、依赖、构建测试、部署和软件开发世界的其他日常任务建立基础的早期工作，这些方面通常不是语言设计中最重要的。”这吸引了那些用有用的包“播种”他们的生态系统的开发者。虽然最初的版本只支持 Linux 和 MacOS X，但这个热情的社区很快就为 Go 的编译器和库创建了 Windows 版本，并将其移植到其他操作系统。

然后，本文认为对开发者的关注贯穿了语言的发展。例如，早期它提到 Go 的高质量加密库(包括对安全通信协议 SSL 和 TLS 的支持)。Go 的标准库还包括一个内置的 HTTPS 客户端和服务器，用于与其他系统在线交互。

但是更重要的是 Go *处理*图书馆的方式。Go 的编译器被设计成只小心地导入必要的库以包含在其二进制文件中，这避免了在其他语言中看到的行为，即导入整个库，只是为了确保包含一个必要的功能。

让开发人员记住，Go 允许从其他领域轻松导入外部库(以及自动检查兼容版本的方法)。“作为适合分布式计算的语言，在 Go 中没有必须发布 Go 包的中央服务器，”论文指出。(尽管现在有一个可用 Go 包的公共*镜像*，以及一个加密签名的日志，其中包含它们的预期内容。)

Go 还以在标准发行版中支持优化技术(如程序剖析)以及测试功能(如模糊化)而自豪。这篇论文指出，Go 甚至有一个代码布局的约定。(而 Go 的`gofmt`工具将源代码解析成这种标准化的布局。)这个和其他内置工具有助于更容易地构建一切，从 IDE 插件和调试器，到框架和构建自动机。Go 的创造者认为，他们的语言是专门设计来鼓励工具和自动化的创造，“因此，Go 世界有一个丰富的、不断扩展的、互操作的工具包。”

因此，他们的论文认为，语言只是其吸引力的一部分。完整的故事必须包括整个 Go 环境:库、工具、约定和软件工程的整体方法。

## 保持一致

他们论文的另一部分也吹捧这种语言的一致性。Go 的创建者承认，在最初的几年里，“我们在每周的发布中对它进行了修补和调整。当更新到新的 Go 版本时，用户经常不得不改变他们的程序。”

但自 2012 年(随着 Go 版本 1 的正式发布)，“我们公开承诺只对语言和标准库进行向后兼容的更改，以便程序在用较新的 Go 版本编译时可以继续运行而不发生变化。”

结果呢？论文解释说，从那以后，这种语言“几乎被冻结了”——但是工具却有了显著的增长。具体来说，它是“更好的编译器，更强大的构建和测试工具，以及改进的依赖性管理，更不用说支持 Go 的大量开源工具了。”该论文认为，这有助于鼓励教育材料的创作，并吸引用户和“第三方软件包的蓬勃发展的生态系统。”

在某一点上，这篇论文甚至认为 Go 精心平衡的特性集避免了该语言的开发者过度扩展。但是本文最后看了这个规则的例外，当时 Go 实际上增加了一个重要的新特性。仅仅两个月前，go 增加了[参数多态性](https://github.com/golang/proposal/blob/4a54a00950b56dd0096482d0edae46969d7432a6/design/go2draft-contracts.md)，这是“为了很好地适应 Go 的其余部分而定制的……”

“在坚持一致性、完整性和社区原则的同时做出如此大的语言改变，将是对这种方法的严峻考验。”

Go 的创造者认识到，维持一门编程语言需要一个社区。也许这也需要一个社区来建立。文章最后对他们在谷歌的同事在 Go 早期的建议和支持表示了感谢，这预示着大量的社区支持即将到来。“自从公开发布以来，Go 已经得到了发展和改进，这要归功于 Google 的 Go 团队以及一大批开源贡献者。围棋现在是成千上万人的作品，在此不胜枚举。

"我们感谢每一个帮助我们走到今天的人。"
