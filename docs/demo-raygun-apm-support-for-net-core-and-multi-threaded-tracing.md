# 演示:Raygun APM 支持。NET 核心和多线程跟踪

> 原文：<https://thenewstack.io/demo-raygun-apm-support-for-net-core-and-multi-threaded-tracing/>

Raygun 赞助了这次示威游行。

[https://www.youtube.com/embed/0TY_oA-EbiI?feature=oembed](https://www.youtube.com/embed/0TY_oA-EbiI?feature=oembed)

有时候解决非常复杂的问题需要非常清晰的答案。对于在 Kubernetes、微服务或众多平台上运行的应用程序的开发人员来说，复杂性无疑是一个问题。不用说，监控这些充其量是一项挑战。

对于这一点，Raygun 有一个相对简单的解决方案。一直以来，Raygun 都是围绕着自己的开发者的考验和磨难来构建工具的，本月，ray gun 推出了其[应用性能监控(APM)](https://raygun.com/platform/apm) 支持。NET 核心，以及对多线程跟踪的支持。

这遵循了 [Raygun](https://raygun.com/platform/apm) 传统上寻求通过提供支持来解决监控难题的方式，正如 Raygun 的首席技术官 [Jeremy Norman](https://www.linkedin.com/in/jerms/?originalSubdomain=au) 恰当地描述的那样，提供“看到整个画面的能力”。

在 New Stack 创始人兼主编 Alex Williams 主持的演示中，Norman 在屏幕上展示了 Raygun 的 APM 如何支持 for.NET 核心工作的示例。这还包括演示如何使用 APM 捕获跟踪还涉及多线程跟踪支持。

事实上，诺曼描述了 Raygun 最新增加的 APM 功能如何“联系到我们现有的客户”

“这不仅仅是一个. NET 核心特性——它将使目前使用 Raygun 的 APM 和。Net 完整框架，”诺曼说。“这是因为人们多年来一直在考虑多线程开发。所以现在，我们将选择这些场景，这样你会有更好的体验。”

这意味着 Raygun 试图为多线程跟踪场景提供更通用的支持。“所以，我们现有的所有可能使用的客户。NET 完整框架支持也将从中受益，”Norman 说。

APM 支持概念的一部分。NET Core 是为了帮助开发人员更好地监控异步执行的线程。“这个框架管理着许多线程，”Norman 说。

“你需要充分地捕捉并分析它，看看你的请求是什么样的，所以你需要能够支持多线程和异步工作的捕捉，”诺曼说。

Norman 在演示开始时展示了一个多线程跟踪，其中使用了多个产品，以便捕获产生错误的跟踪。“所以，在这段代码中，你可以看到，现在我们不只是捕获了一个单独的线程，我们有多个线程，我们是如何做到这一点的，这些线程是从代码执行中执行的，”诺曼说。“我能做的就是点击我们，我们可以看到我们的 GitHub 与代码的集成，这些代码实际上是从我们的源代码库中执行的。您可以在这里看到，我有一个相当简单的用例，使用一个任务来运行一段将异步执行的代码。”

Norman 还展示了演示期间一个常见问题的解决方案，该问题涉及许多正在开发微服务的应用程序如何“主要请求可能会调用其他微服务”

“也许你可能会进行太多的微服务调用，这些调用可以通过批量方法或基准方法进行优化，或者可能会将其中一些调用拆分到可以并行完成的其他任务中，”Norman 说。“因此，我们有了这样一个概念，您对 API 进行了大量调用，这可能会在您的应用程序中引入性能瓶颈或降低速度，并为您提供了改进的机会。

如上所述，一般来说，Raygun 总是围绕自己的开发人员所面临的问题来开发工具。Norman 说，作为一名开发人员，他需要工具来“为我提供我需要的答案”

“过去，当我知道我的代码中有问题，但我不知道问题在哪里，我真的需要探索，试图找到这些问题，我们以不同的方式解决它，”诺曼说。“我们真的希望为人们提供行动——这是对我们有价值的，因为我们可以插入这个工具，它立即开始识别我们可以修复的东西。因此，通过使用我们自己的产品，通过在我们开始开发 APM 并在我们自己的代码库上使用它，我们已经取得了巨大的成功。”
