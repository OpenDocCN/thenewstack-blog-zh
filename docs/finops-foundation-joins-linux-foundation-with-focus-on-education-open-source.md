# FinOps 基金会加入 Linux 基金会，关注教育和开源

> 原文：<https://thenewstack.io/finops-foundation-joins-linux-foundation-with-focus-on-education-open-source/>

根据一份声明， [FinOps 基金会](https://www.finops.org/)(金融运营的简称)已经加入了 [Linux 基金会](https://www.linuxfoundation.org/)，以便“通过最佳实践、教育和标准来推进 FinOps 的纪律”。虽然 Linux 基金会经常发现自己是更专注于技术的努力的家园，但 FinOps 基金会执行董事 [J.R. Storment](https://www.linkedin.com/in/jrstorment) 将这一举动描述为将让 FinOps 基金会移交所有后台办公室操作，以便它可以更好地专注于某些问题，如开源。

“对我们来说，作为 Linux 基金会集成的一部分，一个大的类别是非常关注开源。该基金会的三个目标是围绕创建社区，促进从业者，然后建立开源标准。去年我们花时间在前两个项目上，”斯托蒙特说。“到目前为止，我们做得不好的是构建开源方面。我们正在开源原则，如何运行组织的框架，如何建立团队，诸如此类的东西，这些都是更具文化性的最佳实践，我们现在开始开放这些东西，以引入集成脚本之类的东西。”

从表面上看，FinOps foundation 似乎不太技术性，但正如 Storment 解释的那样，重点不仅仅是团队和文化，而是需要工具、集成和软件来明确云支出。Storment 进一步解释说，FinOps 部分源于向云原生和开发运维的转移，在这种情况下，快速采购基础设施的能力成为创新和快节奏开发实践的必要条件。

> 这是 FinOps 基金会寻求解决的基本问题:更好地装备所有相关人员，共同理解云支出与开发者和工程实践之间的联系。

“过去，要购买资源和服务器，你必须去找采购团队，而且需要很长的准备时间。有时需要几个月。在云的世界里，你基本上有这种模式，你可以即时访问资源，”斯托蒙特说。“你可以获得的资源数量没有限制，你需要它们的时间和你收到它们的时间之间没有延迟。这极大地促进了各种公司加快实验速度，更加敏捷，尝试新事物，采用新技术，但它从根本上改变了技术团队与采购和财务业务团队的合作方式，因为现在财务大权掌握在工程师手中。”

这是 FinOps 基金会寻求解决的基本问题:更好地装备所有相关人员，共同理解云支出与开发者和工程实践之间的联系。

斯托蒙特说:“所有这些团体现在正走到一起，以找出一种新的运营模式，你可以称之为 FinOps，你可以称之为云财务管理，但这些公司必须利用一系列新的人员、流程和技术来做到这一点。”“所以，去年我们成立了这个基金会，作为一个非盈利的贸易协会，目标是帮助这个群体建立一个社区。许多人都在处理云计算方面的支出，他们很难相互沟通，因为很多话题都集中在工程师身上。”

作为促进 FinOps 从业者的努力的一部分，该基金会推出了一个免费的介绍性 edX 课程，[FinOps 简介](https://www.edx.org/course/introduction-to-finops-course-v1linuxfoundationxlfs175x2t2020)，该课程将涵盖“fin ops 的基础知识，以及它如何通过围绕云使用建立问责文化来积极影响一个组织，从而帮助公司在云中做出良好、及时、有数据支持的决策。”该课程现已开放注册，将于 7 月 21 日开课。除此之外，FinOps 基金会还通过 Linux 基金会提供 [FinOps 认证从业者考试(FOCP)](https://training.linuxfoundation.org/certification/certified-finops) ，预计今年晚些时候会有更多的培训和认证项目。它们包括 FinOps 认证平台(FCP)、FinOps 认证服务提供商(FCSP)和 FinOps 培训提供商(FTP)。
