# Pulumi 追求多语言以扩展基础设施即代码

> 原文：<https://thenewstack.io/pulumi-pursues-polyglotism-to-extend-infrastructure-as-code/>

VALENCIA——devo PS 的目标是打破软件开发和运营之间的孤岛。副作用是模糊了开发者和运营者之间的界限。不管是好是坏。因为软件开发人员的角色不断扩大，导致认知超载和倦怠。这就是为什么开发人员工具市场爆炸式增长，以自动化和帮助开发人员在他们需要的时间和地点用他们已经知道的任何语言进行构建。

在这一集的[中，新堆栈制造商播客](https://thenewstack.io/tag/the-new-stack-makers)，在上个月[kube con+CloudNativeCon Europe 2022](https://events.linuxfoundation.org/kubecon-cloudnativecon-europe/)的大厅中录制，[基础设施即代码(IaC)工具提供商](https://twitter.com/mattstratton) [Pulumi](https://www.pulumi.com/) 的员工开发倡导者 Matty Stratton 讲述了 IaC 如何影响开发和运营团队。

[普鲁米追求多元化以扩大 DevOps 的影响](https://thenewstack.simplecast.com/episodes/pulumi-pursues-polyglotism-to-expand-impact-of-devops)

五月份， [Pulumi 发布了更新](/pulumi-takes-infrastructure-as-code-universal-with-crosscode/)，使该平台更接近于成为一个真正的[多语言方式来实施最佳云实践](/pulumi-uses-real-programming-languages-to-enforce-cloud-best-practices/)，包括支持:

*   完整的 Java 生态系统
*   YAML
*   所有普鲁米语的亚马逊网络服务的人行横道
*   以所有 Pulumi 语言部署 AWS 云开发工具包(CDK)

这些是重要的更新，因为它们极大地扩展了可用于在任何云上创建、部署和管理基础架构的语言。

“很多时候，在基础设施即代码中，我们使用配置文件来使用特定于领域的语言。我们称之为基础设施即代码，实际上并没有编写任何代码。所以我喜欢把 Pulumi 想象成基础设施即软件，”Stratton 说。

对于 Stratton 来说，这意味着使用通用编程语言编写基础设施管理代码，如 TypeScript、Python、Go 等。NET 语言，或者现在的 Java。

“最棒的是，你不仅可能已经知道这种编程语言，因为这是你用来构建应用程序的语言，而且你能够使用编程语言可用的所有东西，如条件、循环、包、测试工具、IDE[集成开发环境]和整个生态系统，”Stratton 说。

“这使得它更加强大，并为我们提供了许多可以使用的伟大抽象概念。”
