# Kubernetes 会推动云本地电信吗？

> 原文：<https://thenewstack.io/will-kubernetes-drive-cloud-native-telcos/>

也许 Kubernetes 最全球化的应用是在电信行业。毕竟它有大约 50 亿用户。它本质上是一个有硬件支持、监管良好的行业。

New Stack 创始人兼发行人 Alex Williams 在上个月的 KubeCon + CloudNativeCon 上与网络功能虚拟化开放平台( [OPNFV](https://www.opnfv.org/) )的社区和生态系统副总裁 [Heather Kirksey](https://twitter.com/heatherrkirksey) 以及 Vulk Coop 设计和开发合作社的合作伙伴兼创始人 [Taylor Carpenter](https://twitter.com/ixx) 一起谈论电信的云原生未来。Kirksey 和 Carpenter 所在的以电信为重点的不同协作 Linux 基金会和云本地计算基金会工作组见证了(有时是推动了)电信在过去五年中从单一硬件设备向现在称为云的方向发展。

[Kubernetes 会驱动云原生电信公司吗？](https://thenewstack.simplecast.com/episodes/will-kubernetes-drive-cloud-native-telcos)

对于电信公司来说，云原生意味着软件可以解决重型设备传统上解决的复杂问题。归结起来就是回答两个问题:

*   电信运营商试图解决的问题是什么？
*   创新的云原生技术如何帮助解决这些问题？

当然，电信的规模、法规和永不停机的要求意味着这些问题的答案有时可能需要是独特的，但这并不意味着从零开始。

## 电信采用的集成关键

在这次现场采访中，Kirksey 表示，云原生计算使电信公司能够与其他行业和垂直行业的人们合作，他们已经在 Kubernetes 上构建了一段时间。

柯克塞说:“我们实际上能够与其他合作伙伴一起创新，并利用其他人编写的软件，能够从其他地方吸收创新，而不是来自我们自己的 R&D。”。

此外，Kubernetes 的快速和广泛采用使其成为电信迁移到原生云的全球云基础设施主干的最热门竞争者。

Carpenter 认为，即使是电信硬件也不再是孤立的，而是完全有望尽早与其他资源集成。最终，即使是在一个建立在规范、原则和要求基础上的领域，你也将不得不开始与设备对话，并测试这些工作组在野外试验的一切。

在一个肯定不习惯如此快速发展的行业，这种合作开启了非凡加速的潜力。这个领域有更多限制性要求，如合规性、验证、现有标准和持续可用性等层面。

## 服务网格作为电信解决方案出现

电信需要更复杂和成熟的网络功能。目前，企业电信使用案例和网络功能相当基本，但 5G 网络非常复杂。

卡彭特解释说，“基础网络试图解决绝大多数需求，并使通信变得真正容易。”

[网络服务网格](https://networkservicemesh.io/)正在成为一种潜在的解决方案，充当网络顶部的代理，供不同的提供商在其上工作，并在其上构建应用程序。NSM 可能是通过特定信道将控制扩展到不同设备以确保质量的方式。

就像服务网格如何在应用程序的各个部分之间充当一个自以为是的筛子一样，网络服务网格可以满足数据规则和电信协议。

卡彭特说，任何选择都将归结为服务质量和成本之间的平衡。

正如柯克塞所说，“这是有趣的，大规模的难题。”
