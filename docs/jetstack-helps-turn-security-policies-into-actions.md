# Jetstack 有助于将安全策略转化为行动

> 原文：<https://thenewstack.io/jetstack-helps-turn-security-policies-into-actions/>

西班牙巴伦西亚——除非你一直躲在石头下面，否则你知道[保护软件供应链](https://thenewstack.io/lessons-learned-from-2021-software-supply-chain-attacks/)已经成为安全部门的头号工作。从[网络安全管理软件产品](https://thenewstack.io/solarwinds-the-worlds-biggest-security-failure-and-open-sources-better-answer/)、 [Codecov](https://thenewstack.io/not-your-usual-supply-chain-hack-the-codecov-bash-uploader-blunder/) 和 [Kaseya](https://www.zdnet.com/article/updated-kaseya-ransomware-attack-faq-what-we-know-now/) ，仅举几个例子，过去几年对软件供应链安全来说是糟糕的几年。因此，有许多文件和框架列出了保护供应链的最佳实践。

只有一个小问题:你如何将所有这些政策建议转化为具体行动的计划。云原生安全公司 [Jetstack](https://www.jetstack.io/) 有答案: [Jetstack 软件供应链工具包](https://jetstack.io/software-supply-chain/#overview)。

## 一系列控制

正如 Jetstack 的高级解决方案架构师和工具包开发人员 Steve Judd 所说，“解决这些挑战需要通过一系列控制措施，这些措施远远超出了软件材料清单(SBOMs)，这只是 54 项建议中的一项。”Jetstack 通过提供这项服务为云原生社区做了一件好事。

正如 Jetstack 的首席技术官和创始人 Matt Barker 在 KubeCon Europe 告诉我的那样。它还会对这些问题进行优先级排序，确定哪些问题应该优先解决，而解决这些问题的工作量最小。”

“有了这个，”巴克继续说，“你就可以开始保护你的供应链。你可以爬上梯子。要真正保护您的软件供应链需要付出巨大的努力，但有了这个，您今天就可以有所作为。”

他是对的。你可以。越早开始越好。如果你怀疑这一点，就去问太阳风。
