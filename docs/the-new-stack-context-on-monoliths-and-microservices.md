# 新的堆栈环境:单片和微服务

> 原文：<https://thenewstack.io/the-new-stack-context-on-monoliths-and-microservices/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。今天我们采访了新堆栈新闻记者[迈克·梅兰森](/author/mike-melanson/)，关于他最近对微服务价值争论的报道。我们还讨论了本周的热门播客和新闻故事，涵盖了开发者关系、移动平台开发和优化以及分布式架构等主题。

[第 104 集:单片与微服务](https://thenewstack.simplecast.com/episodes/episode-104-monoliths-and-microservices)

我们已经报道了一段时间关于微服务相对优势的争论。微服务在开发方面有很多优势。不同的团队可以使用一种语言的不同部分，甚至是他们喜欢的语言和工具。更新可以悄悄地插入，不需要刷新整个应用程序。但是，尽管围绕[微服务](/category/microservices/)的大肆宣传相当可观，行业观察家也开始认识到其局限性。许多人主张回到相反的方法:构建单一的大型“单体”应用程序。

最近，Melanson [写了一篇关于](/this-week-in-programming-forget-microservices-monoliths-are-the-way-forward/) Google cloud 的原生传播者 Kelsey Hightower 认为巨石代表未来的文章。

Hightower [在他自己的博客文章](https://changelog.com/posts/monoliths-are-the-future)中写道:“单片是未来，因为人们试图用微服务解决的问题并不真正符合现实。一个缺乏编码纪律的公司最终会导致代码库的灾难，它可能会将微服务视为一个答案，但实际上，这只是用类实例化来换取函数调用。Hightower 写道:“他们发起一些事情，把它扔进网络，然后希望它能回来。”

在本周的播客中，我们将讨论 TNS 驻伦敦记者 Jennifer Riggins[对 Mary Thengvall](/mary-thengvall-on-measuring-the-value-of-developer-relations/) 的采访，探讨开发者关系的价值以及如何衡量这种价值。我们还讨论了 Tricentis 工具如何用于[防止导致美国民主党爱荷华州党团会议灾难的应用故障](/how-id-load-test-the-iowa-caucus-app/)，易贝的[性能改进调整](/ebay-tweaks-mobile-and-web-platforms-for-faster-performance/)，微软的新 [Dapr 分布式计算架构](/tutorial-first-look-at-dapr-for-microservices-running-in-kubernetes/)，Humio 关于迁移到云的[建议](/ebay-tweaks-mobile-and-web-platforms-for-faster-performance/)，以及 Aspen Mesh 最近发现的 Istio 服务 Mesh bug。

TNS 编辑和营销总监 Libby Clark 与创始人兼出版商 Alex Williams 一起主持了这一集。
