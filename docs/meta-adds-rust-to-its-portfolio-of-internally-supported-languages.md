# Meta 为其内部支持的语言组合增加了 Rust

> 原文：<https://thenewstack.io/meta-adds-rust-to-its-portfolio-of-internally-supported-languages/>

周三，社交媒体巨头 [Meta](https://investor.fb.com/home/default.aspx) 宣布，它现在正式使用 [Rust](https://www.rust-lang.org/) 作为服务器端编程语言。它是该公司开发人员“正式”使用的四种语言之一，加入了 C++、Python 和脸书自己内部重写的 PHP、 [Hack](https://hacklang.org/) 。

“Meta 内部使用 Rust 的项目数量一直在加速增长，”Meta 的软件工程经理兼 [Rust Foundation](https://foundation.rust-lang.org/) 董事会成员 Eric Garcia 在他最近的[博客中解释了新增的](https://engineering.fb.com/2022/07/27/developer-tools/programming-languages-endorsed-for-server-side-use-at-meta/)。

Meta 加入了亚马逊网络服务、微软、Dropbox 和其他许多公司的行列，成为 Rust 的热情采纳者。脸书于 2021 年加入 Rust 基金会，并于 2016 年开始使用 Rust。

这并不令人惊讶，因为 Rust 在开发人员社区中的支持率非常高，这一点在上个月的 2022 年 Stack Overflow 开发人员调查中被评为“最受喜爱的语言”中得到了证明。

Garcia 的博客文章将“性能敏感的后端服务和 CLI 工具”列为 Rust 推荐的项目类型。Rust 和 C++在语法上很相似，对于性能敏感的后端服务，仍然鼓励使用 C++。但加西亚明确表示，“我们的产品和服务中存在快速增长的 Rust 足迹，我们致力于长期 Rust，并欢迎早期采用者。”

Meta 自己的基于 PHP 的开源语言 Hack，在涉及到业务逻辑和相对无状态的应用时，仍然是推荐的语言。2013 年和 2014 年，脸书使用 Hack 重构了其网站的大部分核心内容。Hack 继续拥有最高水平的支持和自动化。

Python 仍然是数据科学、机器学习应用和 Instagram 的首选语言。Python 不会很快在 Meta 被淘汰，因为这个生态系统仍然是 Meta 将要投资的。

## 极端值

尽管 Rust、C++、Hack 和 Python 是唯一官方支持的用于服务器端的编程语言，但在需要时也会有异常值。Java、Erlang、Haskell 和 Go 是特定用例的首选，但目前没有得到广泛支持。没有迹象表明它们将来是否会被广泛采用，也没有迹象表明它们具体用于什么。

Meta 高度重视对新编程语言的支持，因为一旦它被平台和工具采用，它就会永远存在。进行仔细的评估，以确定所考虑的语言实际上是否是各种用例的最佳工具之一。

社会媒体公司建议不要使用社区支持的语言，也就是“长尾”语言，除非团队本身对这种语言有丰富的经验。评估过程不可用。

对高质量开发人员体验的期望伴随着在 Meta 给某种东西贴上“受支持”语言的标签。高质量开发人员体验的特征是在 Garcia 的帖子中列出的“代码编辑、调试、构建、部署、核心库和互操作性”方面的良好体验。这需要很高的成本，但这是像 Meta 这样大规模运作所必需的。

就使用多少种语言而言，堆栈的碎片对安全性、隐私、对核心库的支持以及操作风险都有很大影响。Garcia 拐弯抹角地指出，公司使用的语言越少，就越容易降低整体风险和最小化挑战。
