# 为开发人员的工作流增加安全性

在现代数字世界中，唯一不变的是变化。随着快速发展的数字环境催生新技术和日益紧密相连的商业生态系统，安全威胁也在增加。这意味着今天的开发人员被要求执行任务和承担责任，这些任务和责任在他们最初开始工作时可能并不存在。

软件开发人员面临着跟上不断加快的数字化转型步伐的压力，这使得开发和运营团队更加紧密地合作，在许多组织中形成了敏捷的开发运维职能。然而，这种优先考虑速度的做法意味着安全性往往会退居二线。一次又一次，我们看到了破坏品牌与其利益相关者建立的信任的安全漏洞的例子，因此尽早将安全性融入软件开发周期是必要的。

这意味着普通开发人员的期望比以往任何时候都高——当他们被要求以更快的速度构建和推送代码时，他们也被要求安全地编码并对其代码的安全性负责。这是在威胁形势从未如此严峻的时候出现的，应用程序不断受到恶意行为者的监视，他们试图利用漏洞。

那么，开发者该怎么做呢？作为一名现代开发人员，要想获得成功，需要做一些调整来适应这些新的职责。

## 开发者的过去、现在和未来

现代开发人员的工作描述与几年前的开发人员大不相同。在过去，这个角色仅仅专注于代码和应用程序的创建。测试代码的漏洞是安全团队的工作。

今天，开发人员对他们代码的安全性负有更直接的责任。事实上，[最近的数据](https://blogs.microsoft.com/blog/2021/01/14/software-development-in-2021-and-beyond/)显示，20%的招聘专业人士正在寻找与疫情之前不同的技能。此外，对开发人员的需求也在增长，与过去两年相比，工程职位[的招聘量同比增长了 25%。](https://blogs.microsoft.com/blog/2021/01/14/software-development-in-2021-and-beyond/)

## 不断变化的安全格局

为什么在一个更注重创造软件而不是保护软件的领域，对安全知识有如此大的需求？首先，最近的[软件安全报告](https://www.veracode.com/press-release/veracode-state-software-security-half-application-security-flaws-remain-open-six)发现 76%的应用程序包含至少一个安全漏洞。此外，这些缺陷中有一半在发现后六个月仍未修复，这表明了定期应用程序安全测试的重要性。这些统计数据突出表明，安全团队正在努力自行识别和修复漏洞。安全团队开始依靠开发人员来帮助管理当前的需求。

## 开发安全的开发者

为了应对不断变化的形势，开发人员的培训和教育必须进行调整，以便让下一代维护者为未来做好准备。目前，美国只有 3%的本科学位提供网络安全，但对这一技能的需求正迫使更多学校将应用安全纳入其课程。这为 AppSec 组织创造了与学术机构合作并帮助培训和塑造下一代开发人员的机会。

一些领先的技术企业已经在投资网络安全教育。谷歌最近表示，它将在未来五年投资超过 100 亿美元来加强网络安全，并承诺通过职业证书计划在技术领域培训 10 万名美国人。此外，[微软](https://www.cnbc.com/2021/08/25/google-microsoft-plan-to-spend-billions-on-cybersecurity-after-meeting-with-biden.html)承诺在未来五年内投资 200 亿美元来提供更先进的安全工具，并将投资 1.5 亿美元来帮助政府机构升级安全系统和扩大网络安全培训合作伙伴关系。

其他组织通过与学术机构合作来微调年轻程序员的技能，变得越来越有创造力。像[黑客游戏、](https://www.veracode.com/events/hacker-games)这样的黑客马拉松是一项为期两周的比赛，让学生尽可能高效地找到并修复安全漏洞，为年轻的软件开发人员增加了网络安全技能。

同样重要的是向开发人员展示威胁参与者是如何工作的。帮助开发人员进入“坏人”思维模式将使他们能够更好地理解威胁参与者如何从折衷的角度处理应用程序，从而提供对其应用程序开发过程的哪些领域可能是潜在目标以及如何最好地防范这些威胁的宝贵见解。

开发人员可以向提供应用程序安全最佳实践的组织学习，如开放 Web 应用程序安全项目(OWASP ),该项目最近庆祝了其 20 周年纪念，甚至提供故意不安全的应用程序，如为开发人员提供学习和实践应用程序安全的[果汁店](https://nam04.safelinks.protection.outlook.com/?url=https://owasp.org/www-project-juice-shop/&data=04%257C01%257Ckgwilliam@Veracode.com%257Cffefbb0e951545594e4208d9adc9c4ee%257C3b627b68f21c4ed79fe3698efdedbe21%257C0%257C0%257C637731904374543700%257CUnknown%257CTWFpbGZsb3d8eyJWIjoiMC4wLjAwMDAiLCJQIjoiV2luMzIiLCJBTiI6Ik1haWwiLCJXVCI6Mn0=%257C3000&sdata=ID1kBKC6O3i8969vDu71dfaarzLkHiAy4Bv/P7Gd7i0=&reserved=0)。

此外，对于目前在该领域的开发人员来说，实现安全培训和安全编码最佳实践是至关重要的。一些最佳实践包括:

*   尽早将安全性集成到软件开发生命周期中。将 AppSec 工具嵌入到开发人员工具中，并使安全性成为日常事务的一部分。
*   验证输入:确保所有数据在输入系统时都是语法有效的。
*   通过将身份、认证和访问控制构建到开发的早期阶段来实现它们。
*   通过加密来保护数据。
*   记录安全问题，而不仅仅是排除故障并继续前进。
*   利用已建立的框架和受尊敬的第三方库，以及安全的代码和经过验证的安全控制。

国家标准和技术研究所(NIST)已经在制定行业标准，包括提供软件材料清单，以确保更安全的软件供应链和开发环境。随着这些新标准成为现实，网络安全教育和培训对于将现在和未来的开发人员转变为安全倡导者和实践者将变得比以往任何时候都更加重要。"
