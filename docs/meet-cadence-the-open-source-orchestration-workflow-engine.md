# 遇见 Cadence:开源编排工作流引擎

> 原文：<https://thenewstack.io/meet-cadence-the-open-source-orchestration-workflow-engine/>

当构建和操作涉及多个服务调用并需要持续协调的长时间运行的流程时，开发人员通常会面临复杂性。为了解决这一挑战，优步在 2016 年建立并推出了 Cadence，这是一种用于工作流编排的开源解决方案，使开发人员能够直接将复杂、长期运行的业务逻辑表达为简单的代码。自首次亮相以来，它不断吸引运行大规模、基于微服务的架构的开发人员。最近，Instaclustr 宣布支持 Cadence 的托管版本。

在本期 New Stack Makers 播客中，[insta clustr 首席产品官本·斯莱特、](https://www.linkedin.com/in/ben-slater-2720562/?originalSubdomain=au)和[艾姆拉·谢克尔、](https://www.linkedin.com/in/emrahseker/)优步软件工程师讨论了 Cadence，以及开发人员如何使用它来解决各种业务问题，使他们能够专注于编写业务逻辑代码，而不必担心分布式系统的复杂性。

[Alex Williams](/author/alex/) ，The New Stack 的创始人兼发行人主持了这次播客，共同主持人 [Joab Jackson](/author/joab/) ，The New Stack 的主编。

[遇见节奏:开源编排工作流引擎](https://thenewstack.simplecast.com/episodes/meet-cadence-the-open-source-orchestration-workflow-engine)

谢克尔说，Cadence 最初由优步在 2016 年开发，旨在应对技能挑战和不同产品团队日益增长的要求。“Cadence 的主要附加值在于，它允许产品开发人员陈述他们想在产品中做什么，描述他们的意图，并忘记经销商系统的问题。这使得他们的系统在幕后是分布式的、可靠的和可扩展的，”谢克尔说。

“Cadence 是对许多复杂性的伟大抽象，”Slater 说。“如果您从头开始编码，那么扩展到多台主机会有很大的延迟，最终您不得不自己编码。“Cadence 提供了一个很好的框架来插入你的业务逻辑代码，然后所有的东西就可以工作了，”Slater 补充道。

谢克尔说，传统的工作流解决方案需要规定数量的步骤来执行给定的操作，而 Cadence 可以以非常高的性能运行多步操作。它的独特价值在于“在分布式环境中，您必须考虑网络故障、节流、速率限制以及特定订单在其生命周期中所处位置的可观察需求。而 Cadence 基本上提供了所有这些现成的东西，”谢克尔说。

“它反映了当今的现代世界，因为我们希望开发人员能够了解业务，并直接与业务合作。“Cadence 允许他们用他们熟悉的语言表达这种理解，”Slater 说。

Slater 说，最终，当“你有大量的工作流需求，或者需要达到最高级别的规模时，Cadence 就变得有价值了。
