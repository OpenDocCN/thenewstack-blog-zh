# 云托管和资本一问，你的亚马逊云里面有什么？

> 原文：<https://thenewstack.io/cloud-custodian-capital-one-ask-whats-amazon-cloud/>

通常，当你想到 Capital One 的产品时，你会想到可爱的詹妮弗·加纳，或者塞缪尔·l . "[够了，我受够了这些](https://www.youtube.com/watch?v=amYzBQMT4VI) " Jackson，问你钱包里有什么。也许听说过[通过免年费的信用计划获得无限里程](https://www.capitalone.com/credit-cards/ventureone/)。也许你最不希望从同一家公司看到的是一个策略定义引擎，用于在 Amazon AWS 上运行的容器化环境中执行脚本化规则。

Capital One 高级杰出工程师 Kapil Thangavelu 在最近的 O'Reilly 新堆栈制造商开源会议上与 Alex Williams 的对话中解释说:“我们开发了云托管，为我们提供了一种对 AWS 公共云帐户中发生的所有策略的统一视图和执行方式。

Thangavelu 告诉我们，开发人员倾向于做一些愚蠢的事情——例如，将他们的整个数据库公开。随着 Capital One 构建其集装箱化战略，其工程师发现自己正在设计脚本，以确保这些愚蠢的事情不会产生灾难性的结果。但这些都是个案情况，涉及 YAML 和 Python，很快他们就需要一种方法来将更通用的脚本应用于更广泛的标准集，以强制执行策略，并在开发人员漫游到未探索的领域时应用护栏。

他说，云托管是从这一基本需求中产生的，尤其是当这些成千上万的一次性脚本为它们自己产生程序生命周期问题时。“你不想进入一个有数百个随机脚本的地方，这些脚本代表不同的控制，由不同的人部署——可能是代码审查，可能是测试……你不想进入一个你的控制已经有效地锁定了你的创新能力的地方。”

Thangavelu 进一步详细介绍了云托管如何工作，以及如何在自己的基于 Amazon 的环境中将其作为规则执行管理器来使用。他讨论了如何使用亚马逊的无服务器模型将一些规则实现为 Lambda 函数，并可以从其 [CloudWatch 监控服务](https://aws.amazon.com/cloudwatch/)查询实时数据。
