在最新一期的 New Stack Makers 播客中，[CNCF 首席技术官克里斯·阿尼斯奇克](https://www.linkedin.com/in/caniszczyk)与[New Stack 的发行人兼创始人亚历克斯·威廉姆斯](/author/alex/)谈论了云本土的热门话题。他们的对话让我们看到了正在发生的事情。

## 越来越多地采用 Rust，Wasm

对于不同的项目，编程语言的使用和采用已经发生了转变，Rust 越来越多地被用于创建 Kubernetes 的云本地工作语言 Golang(T21)所采用。

虚拟机语言 [WebAssembly](https://thenewstack.io/what-is-webassembly/) (Wasm)也取得了进展。最初创建它是为了帮助提高 web 应用程序的代码效率，后来它的使用和应用扩展到了云本地计算的范围。

“我发现生锈在项目中变得越来越普遍。Aniszczyk 说:“我认为，其中一部分也与近来人们对 WebAssembly 的兴趣有关，在许多核心的 WebAssembly 社区中，有许多问题。“所以，我认为，这很自然地把它带到了 CNCF。”

Aniszczyk 说，在作为网络浏览器的沙盒技术引入后，用户开始意识到 Wasm“可以用于新的环境”。

Aniszczyk 说:“这是一种全新的运行时，因此[Wasm]可以用于运行其他类型的工作负载。“人们总是试图让事情变得更快、更好、更便宜，而且会有新的方法。我认为 WebAssembly 只是其中一个例子。”

## GitOps 继续加速发展

对于 [CI/CD](https://thenewstack.io/category/ci-cd/) ，向以版本控制为中心的方法的转变——通常被吹捧为[GitOps](https://thenewstack.io/what-is-gitops-and-why-it-might-be-the-next-big-thing-for-devops/)——也应该继续加速。Aniszczyk 说:“许多东西正在向开发人员进行正常编程的地方转移，这些工件被版本化，通常存在于 git 这样的源代码控制系统中，并且都在典型的开发人员工作流程中进行管理。”

他补充说，虽然对许多人来说这并不奇怪，但开源应该继续在云原生应用程序和整个 IT 行业中发挥主导作用。

“任何人想要开发一个能被很多人使用的东西，我认为他们必须采取开源的、面向社区的方法。我只是觉得没有更好的方法了。”
