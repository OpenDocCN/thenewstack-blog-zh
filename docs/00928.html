DockerCon 2019:关于端到端容器生命周期的一切

原文：https://thenewstack.io/dockercon-2019-all-about-the-end-to-end-container-lifecycle/#0001-01-01

虽然Kubernetes可能在过去几年的云原生计算革命中抢了Docker的风头，但Docker正在接管让基于容器的开发真正为企业所用的工作。

在本周于三藩市举行的 DockerCon 2019 的开幕主题演讲中，Docker首席执行官史蒂夫辛格强调了 Docker Enterprise ，该公司最新更新的容器管理包，如何帮助整个开发和部署生命周期。

Docker Enterprise 3.0包括多项自动化开发和部署容器化应用的进步，包括开发人员直接从桌面将应用推入多个云部署的能力，直接在桌面上进行Kubernetes和Docker Swarm的低维护设置，甚至是Docker Enterprise的完全托管版本。该公司的立场是，大多数IT商店不具备在内部运行类似Kubernetes的专业知识，因此Docker提供了一些软件包和工具来尽可能多地自动化繁重的工作。

作为主题演讲的开始，Singh请来了一些企业用户的代表——Docker总共有大约750家企业客户——来谈谈他们在应用程序容器化方面的经验。对于嘉年华的技术副总裁托德·赫德(Todd Heard)来说，集装箱化给豪华邮轮带来的主要好处是能够通过该公司的Medallion奖励计划，将邮轮旅行的不同方面(如登记和船上服务)连接成一个统一的体验。“每个人都想谈论技术，但在旅游业，一切都是关于体验，”赫德说。

“我们都在云上，”保险公司Nationwide的云运营主管Eric Zielinski表示同意。该公司正在进行一项雄心勃勃的努力，希望在未来几年内将其大部分应用程序运行在云中，包括许多旧的应用程序。

遗留应用程序是Docker重点支持的另一个领域。在他自己的演讲中，Docker产品高级副总裁Scott Johnston 将企业应用分为三个阵营:绿地应用(新的云原生应用)、不再开发的遗留应用和“棕地”应用(仍在用新功能构建的遗留应用)。借助Docker应用程序和云本地应用程序捆绑包(CNAB)标准等工具，Johnston承诺所有三种类型的应用程序都可以容器化。

该公司还展示了其最近与ARM 在的合作成果。借助Docker Enterprise 3.0中的预览功能，开发人员可以使用(基于x86的)Docker桌面实例构建一个完整的基于ARM的应用程序。约翰斯顿指出，事实上，Docker Hub中的所有官方图像都可以在ARM上运行，这为使用亚马逊网络服务和分组网络现在提供的低成本ARM服务器芯片铺平了道路。
