在应用程序开发和交付领域，唯一不变的是变化。今天的软件测试专业人员已经见证了业务期望、开发方法、系统架构等方面的巨大变化。而且，对数字化转型的新关注意味着变革的速度和范围只会增加。

重新发明测试对于实现在数字未来蓬勃发展所需的速度和敏捷性至关重要。为什么？速度是新的货币——但是传统的软件测试是快速部署的头号敌人。

相同的基本故事总是出现:组织转变开发和运营，但是测试跟不上步伐，他们停滞不前。为传统的长达几个月的发布时间表构建的测试工具和过程根本不适合现代的交付节奏，现代的交付节奏要求对每个新的构建进行即时的质量反馈。

这是一个悲伤但简单的事实:如果你有一个处于高度加速的开发和操作过程之间的缓慢的测试过程，你就没有办法达到期望的交付速度。

许多公司已经认识到这一点，并通过现代化测试来激励他们的数字化转型计划。通过在他们的组织中重新检查和重新发明软件测试，他们不仅消除了“测试瓶颈”——他们还推动了积极的用户体验，这种体验现在可以成就或摧毁一个企业。

## 测试转型需要什么？

想要通过[连续测试](https://www.tricentis.com/products/what-is-continuous-testing/)(devo PS 和数字化转型所需的极端测试自动化)将测试转变为数字化转型的全球 2000 强组织需要克服以下挑战:

*   高度复杂的应用程序架构(例如，跨浏览器的流程加上 API 加上 mobile 加上 SAP 加上其他打包应用程序加上自定义 Java/。NET 应用程序加大型机)。
*   不同的应用程序类型、规划周期、开发方法和工具并行存在。
*   不同团队和项目中根深蒂固的质量流程。
*   稀缺的测试自动化资源。
*   “臃肿”的测试套件延迟了过程，同时提供了有限的商业价值。

没有灵丹妙药。转换需要时间，并且必须涉及与应用交付管道相关的所有人员、流程和技术。尽管如此，你可以放心，你的努力会有回报的。这些观点提供了许多例子，说明致力于测试转型的组织现在是如何发展业务和增强客户/用户体验的。

持续测试曾经只是一个愿望。现在，数字化转型的驱动力使得持续测试成为企业的当务之急。像这样的组织正在铺平前进的道路，证明持续测试能够——并且必须——在企业组织中成为现实。

## 重新想象质量保证的未来

随着组织转变他们的测试方法以推进他们的数字化转型计划，测试规程也在以前所未有的速度发展。云、人工智能、机器学习、RPA 和物联网不可否认地影响着测试的未来——但遗留系统、监管合规性和安全性也是如此。我们如何调和新旧，区分炒作和现实，并优先关注什么？

100 多页的凯捷报告《[重塑质量保证](https://www.tricentis.com/resources/reimagine-the-future-of-quality-assurance/)的未来》解决了这些问题。作为报告撰稿人和赞助商，Tricentis 很高兴能与 IT 领导分享这份报告。该报告包括:

1.  采访了 14 位 IT 领导者，他们成功推动了麦当劳、AGL、AXA、Equifax 和 Westpac 等组织的质量转型。
2.  深入探讨各种趋势，如测试数据的民主化、测试自动化和 RPA 的融合，以及 DevOps 和数字化转型如何影响质量指标和 KPI。
3.  对 CPRDT(消费品、零售、分销和运输)、能源和公用事业、金融、高等教育和政府的质量趋势进行行业分析。
