云铸造基金会的下一步是什么？与新任执行董事艾比·卡恩斯的谈话

原文：https://thenewstack.io/whats-next-for-cloud-foundry-foundation-abby-kearns/#0001-01-01

在担任 Cloud Foundry Foundation 执行董事的新职务中， Abby Kearns 计划在2017年与不同市场的开发者社区进行更紧密的合作。

在今天的 The New Stack Makers 播客中，Kearns讨论了Cloud Foundry如何将其 service broker API 引入其工具包的前沿，以帮助开发人员在一个完全关于跨平台兼容性的生态系统中快速简单地连接他们的服务，并强调了Cloud Foundry Foundation的一些未来目标。


执行董事Abby Kearns 云铸造基金会2017年的下一步是什么

正是Cloud Foundry service broker API迅速获得了开发者的青睐。“我认为服务业将会非常庞大。Kearns说:“当您构建应用程序时，如果能够连接和利用服务，最终您将无法创建一个没有可用服务的应用程序。

虽然Cloud Foundry service broker API隶属于Cloud Foundry Foundation并拥有自己的项目管理委员会，但该基金会欢迎社区对软件做出贡献。“你不一定要成为Cloud Foundry Foundation的成员才能参与。我们把它放在一边，在某种程度上，考虑到这种可及性。我们希望更多的人参与进来，”卡恩斯指出。虽然一些项目可能会迭代并期望他们的项目快速变化，但Kearns解释说，Cloud Foundry service broker API与此相反，由于它同时遵循标准和API规范，因此会逐渐变化。

Kearns解释说，Cloud Foundry service broker API提供了真正的跨平台灵活性，允许开发人员除了连接他们的服务之外，还可以监控他们的服务。“我特别想到那些编写和开发软件的小型软件公司。他们想让在某个平台上编写软件的人也能使用它。针对单个API编写并让该服务可以跨多种平台访问不是更容易吗？对他们来说，这让他们的生活更轻松。他们不必为每个平台定制软件。”

最终，组织转型是卡恩斯对2017年生态系统最期待的事情。快速创新和迭代曾经只是初创公司的领域，现在企业带着自己的创意和挑战出现在舞台上。“我对未来几年感到非常兴奋，因为越来越多的这些组织改变了他们编写软件的方式，以及他们编写的软件。他们如何参与？他们如何利用已有的数据，并以新的有趣的方式向用户群展示这些数据？我认为我们只是看到了冰山一角。”

Cloud Foundry基金会是新堆栈的赞助商。
