# Palo Alto Networks 将带外 Web 安全性引入云

> 原文：<https://thenewstack.io/palo-alto-networks-brings-out-of-band-web-security-to-cloud/>

你看到那个标题的第一个问题，可能也是我的第一个问题。什么是“带外”安全性？通常，我认为它是零日灾难的紧急安全补丁。例如，当微软采取措施发布非补丁日补丁时。但是，当云安全提供商[帕洛阿尔托网络](https://www.paloaltonetworks.com/cloud-security?utm_content=inline-mention) (PANW)宣布[将带外 web 应用和 API 安全](https://www.paloaltonetworks.com/blog/2022/06/prisma-cloud-introduces-oob-waas)(带外 WAAS)添加到 [Prisma Cloud](https://www.paloaltonetworks.com/cloud-security?utm_content=inline-mention) 时，他们正在谈论从其托管基础设施外部监控 Web 应用和应用编程接口(API)的安全问题。最终结果是什么？安全的服务交付，没有缓慢的性能。

这是一件大事。web 服务的一个肮脏的小秘密是，如果要在性能、可伸缩性和安全性之间进行选择，安全性要么最后被选中，要么完全被忽略。正如 [Melinda Marks](https://www.linkedin.com/in/melindamarks22/) ，[企业战略集团(ESG)](https://www.esg-global.com/) 高级应用和云安全分析师所观察到的，“随着组织越来越多地在云中构建和部署其应用，在不影响性能的情况下保护其关键业务应用已成为一项挑战。”

Web 服务外部攻击令人讨厌且常见。在最近的一项 Forrester 研究中，39%的外部攻击是 web 应用程序漏洞。这使得它成为最常见的外部攻击。

### 带外 WAAS

[PANW Prisma Cloud 高级副总裁 Ankur Shah](https://www.linkedin.com/in/ankur-shah-cloudsec/) 声称，有了带外 WAAS，“公司不再需要在应用安全性和性能之间做出选择。我们正在为客户提供灵活的安全选项，以满足他们不断发展的应用需求。”

怎么会？通过用带外 WAAS 替换内嵌 Web 应用防火墙(WAF)和点 API 安全工具。它还提供 bot 防御和拒绝服务(DoS)保护。它通过提供外部服务来快速监控您的任务关键型 web 服务，而不会降低您的性能。

Prisma Cloud 还增强了基础设施、工作负载、身份和应用程序的可见性。目标是消除公共云和多云环境中的安全盲点。

## 关键特征

它通过以下功能实现这一点:

*   云基础架构授权管理(CIEM)的多云图形视图:发现特权过多的帐户并了解跨多云环境的访问风险。Prisma Cloud 现在提供了跨 AWS、Microsoft Azure 和 Google Cloud 的净有效权限的图形视图。
*   多云无代理云工作负载保护:除 AWS 之外，扩展对 Azure 和 Google 云的云工作负载和应用程序风险的可见性，以补充现有的基于代理的保护。
*   基于 DNS 的威胁检测:云环境中的表面恶意活动和异常行为。Prisma 云威胁检测现在利用机器学习(ML)和高级威胁情报来识别隐藏在 DNS 流量中的不良行为者。
*   米特 ATT 和 CK 警报优先级排序:使安全团队能够根据业界最广泛采用的框架来确定风险和事件的优先级。

带外 WAAS 现可用于 [Prisma 云计算版](https://www.paloaltonetworks.com/resources/datasheets/prisma-cloud-compute-edition-aag)。它将在下个月推出企业版。
