# 阿迪达斯如何管理规模

> 原文：<https://thenewstack.io/how-adidas-manages-for-scale/>

运动服装巨头阿迪达斯是如何规模化经营的？三种方式:技术上，文化上，战略上。

在这一集的 New Stack Makers 播客中，New Stack 的创始人兼发行人 Alex Williams 与阿迪达斯平台工程高级总监[iniaki Alzorriz](https://es.linkedin.com/in/ialzorriz)和阿迪达斯解决方案架构总监 [Rastko Vukasinovic](https://www.linkedin.com/in/rastko-vuka%C5%A1inovi%C4%87-b4170235/?originalSubdomain=nl) 谈论了该公司如何在 Kubernetes 上扩展其运营。

“所以，毫无疑问，我们是在旅行。我们肯定有很多事情要做，”Alzorriz 说。“我们已经找到了到达那里的方法，但至少我们已经踏上了我认为重要的旅程。”

[阿迪达斯如何管理规模](https://thenewstack.simplecast.com/episodes/how-adidas-manages-for-scale)

Vukasinovic 表示，在通过“几项微服务”启动这一数字之旅后，阿迪达斯的 DevOps 团队开始为其阿迪达斯和锐步品牌“向不同市场推出这些数字产品”。Vukasinovic 说:“因此，我们将在那里推出一种数字产品，然后在不同的地方推广。”如今，该公司的跨国 DevOps 团队每天编译代码超过 10，000 次。

在创建了必要的基础设施来支持其 Kubernetes 环境、CI/CD 工具、监控工具和流媒体平台后，阿迪达斯专注于软件工程师的工作需求。工程师始终是阿迪达斯计划的“核心”，以保持该公司对最终消费者的“同样的消费者痴迷”，同时“试图理解并(对此)做出反应”，Alzorriz 说。

Alzorriz 解释说:“我们需要确保我们能够发展并响应最终消费者的需求，”通过保持 DevOps 的可扩展性、易于使用的新功能、弹性和其他基础组件。

事实上，阿迪达斯的数字化之旅与其 DevOps 战略的发展密切相关。Vukasinovic 解释了公司开始数字化之旅的决定是如何与它需要成为一个“产品主导的组织”联系在一起的。这意味着技术和业务利益相关者都要对公司的数字计划负责。

Vukasinovic 说，DevOps 的主要经验之一是不同的 DevOps 利益相关者如何“一起配合”，以支持软件和产品开发。Vukasinovic 说，这种推动是必要的，以便“成长为一个数字组织，而不仅仅是一个技术组织或商业组织”。
