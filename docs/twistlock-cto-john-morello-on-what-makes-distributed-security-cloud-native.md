# Twistlock 首席技术官 John Morello 谈什么使分布式安全成为“云原生的”

> 原文：<https://thenewstack.io/twistlock-cto-john-morello-on-what-makes-distributed-security-cloud-native/>

[为什么云原生架构天生更安全](https://thenewstack.simplecast.com/episodes/why-cloud-native-architectures-are-inherently-more-secure)

云原生应用部署最受推崇的优点之一是，它旨在将软件开发人员从担心其基础设施状态中解放出来。“开发商和消费者都依赖我们，”去年 10 月，产品营销总监 Michael Ducy 在新的堆栈中写道，“隐藏操作的复杂性并保持选择的自由。”

因此，人们可能会有这样的印象，称自己为“云原生安全性”的东西是代表云原生开发空间的维护者提供的服务，如今在越来越多的企业中，这些维护者不是运营人员。最近几个月，Twistlock 越来越不仅仅局限于保护集装箱，尽管这显然是该公司去年这个时候的重点。正如 Twistlock 首席技术官 John Morello 最近在《新堆栈制造商》中解释的那样，twist lock 对云原生安全性的看法是，从安全性的角度来看，CNCF 堆栈(我们都知道“CN”代表什么)的组件所占据的开发空间值得特别关注。

Morello 承认，无服务器功能确实减轻了开发者的负担，包括安全性。但是，“您可能不会运行从三十多个架构移植过来的应用程序，这些应用程序已经在您的组织中存在了十几年……成为一个功能或一组功能。

“功能能够为人们简化事情的原因之一，”CTO 继续说道，“是你放弃了很多能力。您将牺牲一些与主机底层交互的能力，并真正拥有一些在容器或虚拟机中运行时可能拥有的能力。这样做的代价是你不必担心底层的基础设施。但这也意味着一些应用程序或部分应用程序可能不太适合在无服务器功能中运行。”

Morello 认为，由于本质上总会有一类应用程序使用最新型号以外的型号运行，例如，从旧系统移植的应用程序，组织将继续选择更全面的安全服务，平等对待应用程序环境的所有方面。因此，从 [Twistlock 的](https://www.paloaltonetworks.com/prisma/cloud)的角度来看，拥有一个学习所有应用程序行为的安全系统，并将该系统与云原生应用程序一起部署在云平台上是有意义的。
