# 云铸造峰会欧洲:为什么 Kubernetes 应该是无聊的

> 原文：<https://thenewstack.io/cloud-foundry-summit-europe-why-kubernetes-should-be-boring/>

[Cloud Foundry](https://www.cloudfoundry.org/) 赞助了这个播客。

开发人员在转换到云原生平台时肯定会面临新的挑战。这个过程可能包括，例如，学习如何向 Kubernetes 集群添加代码，或者掌握 [etcd](https://etcd.io/) 和 [kubectl](https://kubernetes.io/docs/reference/kubectl/overview/) 的机制。云原生平台和 Kubernetes 提供的能力和伸缩灵活性等，往往比开发人员在采用这些技术时投入的时间和资源更有价值。

然而。

开发人员通常更关心的是他们需要实现的业务目标。他们可能不太关心底层基础设施是什么，而是关心如何使用它来创建代码，以提高组织的底线，或者对于公共机构来说，更好地满足公民的需求。换句话说，正如下面这一集所讨论的，我们需要停止吹捧 Kubernetes 有多棒，让它变得无趣。相反，我们应该专注于如何——如果可能的话——云原生和 Kubernetes 可以推进业务目标。

在本月于荷兰海牙举行的[2019 Cloud Foundry Summit Europe](https://www.cloudfoundry.org/event/summit/)上，分析师记录了这一集的新堆栈，其中讨论并辩论了开发人员的业务需求和 Cloud Foundry 社区的角色。

[云代工峰会欧洲:Kubernetes 为什么要无聊](https://thenewstack.simplecast.com/episodes/cloud-foundry-summit-europe-why-kubernetes-should-be-boring)

由 New Stack 创始人兼主编 Alex Williams 主持，Cloud Foundry Foundation 营销副总裁 [Devin Davis](https://www.linkedin.com/in/devinpdavis/) 共同主持，小组成员包括:

卡恩斯说，她同意 Kubernetes 应该是无聊的，考虑到“我们在这里不是因为我们关心 Kubernetes，我们在这里不是因为我们关心平台即服务。”这是因为“总的来说，我们正在解决更大的问题，这就是我们如何允许用户的客户编写代码，将代码部署到产品中并大规模运行？"

“所以对我来说，这总是关于我们如何考虑技术趋势的走向，用户行为趋势的走向，”卡恩斯说。“然后，我们如何确保我们作为一个社区、一个生态系统和一项技术也能继续发展。”

关于如何让 Kubernetes 变得无聊的想法，Coté说，“似乎每个人都想在较低的层次上谈论这个问题，在最高层次上，我们专注于如何添加功能和新的东西，但对于无聊的强迫功能，中间层没有任何事情发生得足够快。”

对于云原生和 Kubernetes 部署，开发人员可能永远不需要参与基础架构的管理和担忧。“如果我们要管理这个基础设施，有些政策对我们很重要，”Van Hove 说。“我们不希望我们的普通开发人员进入那个领域。”
