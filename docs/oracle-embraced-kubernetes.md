# 甲骨文为何选择 Kubernetes

> 原文：<https://thenewstack.io/oracle-embraced-kubernetes/>

[甲骨文为何拥抱库伯](https://thenewstack.simplecast.com/episodes/why-oracle-embraced-kubernetes)

不要问谁拥抱，问谁不拥抱。因为似乎每个人都在用 Kubernetes。包括 Oracle，无可争议的数据库之王，它正稳步(并且悄悄地)进入公共和私有云空间。

Oracle 从事容器管理已经两年多了。该公司通过 2016 年收购[stacken engine](https://thenewstack.io/stackengine-focuses-on-docker-tools-for-the-real-world/)的[获得了集装箱专业技术和解决方案。在 Kubecon + CloudNativeCon 2017 上录制的最新一期](https://thenewstack.io/oracle-settles-docker-ecosystem-stackengine-purchase/) [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，我们采访了 StackEngine 的联合创始人 [Bob Quillin](https://www.linkedin.com/in/bob-quillin-46802511/) ，他现在担任副总裁领导 [Oracle Container Group](https://cloud.oracle.com/en_US/containers) ，以更好地了解 Oracle 参与 Kubernetes 社区的情况。

Quillin 说:“客户正在寻找开放社区驱动的云中立解决方案，以便在 Kubernetes 上运行他们的编排和容器管理解决方案。

他说，人们希望避免被云锁定。然而，与此同时，这些客户还希望与堆栈的其余部分进行一定程度的集成，以便他们能够在企业平台之间获得无缝体验。[云本地计算基础](https://www.cncf.io/kubecon-cloudnativecon-events/)”和像 Kubernetes 这样的开源解决方案让他们有机会在任何他们想去的地方运行它。这使他们能够灵活地在多种不同的云中运行，包括本地云，”Quillin 说。

甲骨文去年加入 CNCF 成为白金会员。该公司也是 CNCF 几周前发起的整合项目的一部分。甲骨文有许多工程师在从事 CNCF 项目。

“CNCF 希望致力于此，我们是解决方案的一部分，”奎林说。
