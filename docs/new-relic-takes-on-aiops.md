# 新遗迹挑战 AIOps

> 原文：<https://thenewstack.io/new-relic-takes-on-aiops/>

[新文物亮相](https://thenewstack.simplecast.com/episodes/new-relic-takes-on-aiops)

在本周的 New Stack Context 播客中，我们将讨论 New Relic 在最近收购 significal 后在 AIops [领域的发展。](/new-relic-signifai-and-the-shifting-view-of-monitoring-technologies/)[New Relic 产品管理高级副总裁亚伦·强森](https://www.linkedin.com/in/aaronjohnson/)和 SignifAI 联合创始人兼前首席技术官、现任 AIOps 总经理兼 New Relic 产品工程副总裁的 Guy Fighel 分享了他们在收购背后的愿景，以及机器学习如何用于监控和诊断复杂的 IT 系统。

然后在节目的后面，我们将讨论如何摆脱技术债务。答案是停止使用开源库和框架吗？最近退休的眼镜供应商 Warby Parker 的首席架构师罗伯特·莱夫科维茨提倡这种方法，我们将在上周举行的奥莱利软件架构会议上深入研究他的演讲

TNS 编辑总监 Libby Clark 主持了这场秀，TNS 执行主编 Joab Jackson 和 TNS 联合创始人兼主编 Alex Williams 提供了帮助。

## 链接

*   [New Relic、signival 和监控技术的转变观点](https://thenewstack.io/new-relic-signifai-and-the-shifting-view-of-monitoring-technologies/) : New Relic 最近收购 signival 表明系统监控领域正在发生变化。技术工具提供商正在将机器智能和其他工具结合成单一的集成服务，可以更好地向 DevOps 技术专家和现场可靠性工程师(SRE)通报其系统的状态。
*   [AIOps:更大的数据上下文可以产生更丰富的结果](https://www.thenewstack.io/aiops-greater-data-context-can-yield-richer-results) : IT 从业者经常与上下文斗争。他们跳到数据操作或聚合，建立数据摄取模型，而没有首先考虑业务。在这篇供稿中，OpsRamp 的产品副总裁兼工程主管 Bhanu Singh 提供了许多技巧来为 AIops 获取正确的数据，包括学习调整目标和结果，使数据保持一致，并从小测试用例开始。
*   [WhiteSource:容器的开源安全管理](https://www.thenewstack.io/whitesource-open-source-security-management-for-containers):很有可能，你正在使用一个开源库。最近的一项调查显示，92%的应用程序至少使用一个开源库。另一项调查显示，这个数字甚至更高，达到 96%。WhiteSource 是一款持续的开源安全和许可证合规性管理工具，创建于七年前，旨在解决这一问题。本周，它将持续监控功能引入到容器化的应用程序和包含这些应用程序的映像中。它通过“Kubernetes 代理在后台作为生产集群中的 pod 静默运行，自动扫描新 pod 中部署到生产的任何映像”，同时监控 Docker Hub、Amazon ECR、Azure Container Registry 和 JFrog Artifactory 中存储的容器映像。
*   [Datadog 收购前端 Web 测试公司 Madumbo](https://thenewstack.io/datadog-goes-full-stack-with-front-end-web-tester-madumbo-acquisition/):基础设施和应用监控服务公司 Datadog 收购了人工智能网站测试工具 Madumbo，Madumbo 首席执行官 Gabriel-James Safar 解释称，这是扩展 Datadog 全栈监控能力的第一步。Datadog 对 Madumbo 的收购将是第三次，上一次是在 2017 年，当时该公司收购了另一家法国公司 Logimatic.io。
