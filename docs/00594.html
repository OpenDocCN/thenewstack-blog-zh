库伯内特社区20年健康计划

早在5月，世界上最大的 Kubernetes 用户和贡献者聚集在丹麦参加 KubeCon + CloudNativeCon 2018欧洲。在这次思想会议上，我们渴望看到的一个主题是Kubernetes的未来，以及整个云本地计算基金会 (CNCF)。为了了解Kubernetes项目的长期准备工作，我们采访了谷歌开源软件治理项目经理Jaice Singer DuMars、谷歌集团产品经理Aparna Sinha和CNCF执行董事Dan Kohn。

将Kubernetes建设成一个在未来20年能够继续成长和发展的开源项目的关键之一是围绕这个项目的用户和贡献者社区。Sinha说，“我们需要更好地解决贡献者路径，这样人们更容易参与。随着项目变得越来越大，人们不知道如何做出贡献，他们如何进入，他们去找谁？因此, Paris [Pittman] …一直在为贡献者设计一个更容易的入口，以及一个她非常热衷的指导项目。

Kohn说，CNCF允许子项目解决尽可能多的问题，解决方案来自社区底层，而不是自上而下。他说，“我们CNCF有一种哲学。运行一个开源项目没有“CNCF方式”。我们称之为“自带治理流程”。现在，事实上，到目前为止，没有一个来到CNCF的项目是完全成熟的，所以他们每个人都需要经历一个过程，对他们中的几个人来说，这实际上是有点漫长和痛苦的，比如Kubernetes和 Prometheus 。我们一路提供建议，提供帮助，但坦率地说，他们中的大多数人都能够提出他们满意的东西，并继续前进。”

辛哈说，CNCF通过为Kubernetes团队提供预算帮助了我们。因此，该团队现在考虑将bug奖金和扩展测试作为这些资金的良好用途。CNCF正在做的另一件事是认证发布。Kohn说，有这么多的供应商和云自己实现Kubernetes，一致的发行版对于为生态系统提供可预测性和可靠性是必不可少的。DuMars说，这种类型的工作是关注用户的需求，同时也解决供应商的需求。

“我认为，治理的核心是将社区的需求和产品用户的需求融合在一起。最佳形式的治理是一种提供透明度和跨决策制定的审计能力的方式。杜马斯说:“谷歌非常重视透明度，并试图建立超级健康、包容和多样化的社区，而治理是在这方面实现公平竞争的一种方式。
