# 为什么没有开源云是不可能的

> 原文：<https://thenewstack.io/why-cloud-is-impossible-without-open-source/>

亚马逊网络服务(AWS) 赞助了这篇文章。

 [马特·阿萨伊

Matt 是 AWS 的负责人，参与了开源及其支持的所有领域(云、机器学习、数据基础设施、移动等。)近二十年来，为各种开源公司工作，并定期为 InfoWorld 和 TechRepublic 撰写文章。你可以在推特上关注他(@mjasay)。](https://www.linkedin.com/in/mjasay/) 

很明显，云对开源产生了很大的影响，通过使开源运行起来更容易、更便宜，同时最小化各种风险，极大地增加了它的吸引力。

然而，开源对云计算的影响可能更加显著。[认为](https://mobile.twitter.com/pwramsey/status/1248373100317720576) PostgreSQL 开发者保罗·拉姆齐说，“没有开源，我们所理解的云就不会存在”。Selenium 和 Tapster 创始人 Jason Huggins 补充道[纯粹基于经济学，没有开源云是“不可能的”。](https://mobile.twitter.com/hugs/status/1248372795500724224)

真的吗？

是的，真的。“如果谷歌不得不向甲骨文支付每个数据库或每个 CPU 的许可费，世界上的谷歌将永远不会存在，”哈金斯说。这并不是一家大公司试图提高利润率，而是根据单位定价模式运行数百万台服务器是完全不可能的。“如果每个虚拟机都运行 PROPRIETARY _ UNIX，那么您就无法通过增加额外的虚拟机来扩展网络商店，因此部署每个**虚拟机需要花费您＄1000 美元的许可证费用。开源专家 Liam Proven [说](https://mobile.twitter.com/lproven/status/1248374985791430657)只有当增量许可成本为 0 美元时**才有效。****

这些数字不仅看起来糟糕。正如哈金斯所说，它们看起来“不可能”。

这也不仅仅是成本增加的问题。理论上，你可以协商降低甲骨文费用。(去吧！我们会等的！)但是，正如 GitHub 首席运营官埃里卡布雷西亚指出的那样，这并不能解决问题。“即使你通过改变收费结构来解决定价问题，仅仅跟踪许可和授权也会给已经具有挑战性的技术问题增加很多复杂性，”布雷西亚说。

除了软件的成本，还有代码的质量。据前 MySQL 首席执行官(现任 HackerOne 首席执行官)Marten Mickos 称，[开源提供了](https://twitter.com/martenmickos/status/1248385573959548931)“大量的几乎没有错误的可横向扩展的软件，云提供商可以免费获得(操作系统、虚拟化、数据库、中间件、开发语言和框架等)。)."利用(并回馈)这些丰富的代码，云提供商能够构建一系列服务，包括许多转变为服务的开源项目。

但还不止这些。

AWS 副总裁马特·威尔森[说](https://mobile.twitter.com/_msw_/status/1248379446211403782)，云是所有关于基础构建模块。毫不奇怪，这也是开源的核心，借鉴了早期 Unix 的影响。虽然肯定有整体的开源项目，但开源的精神实质是将模块化组件拼凑在一起以创建更大的组合。

## **(不是)多此一举**

虽然这在今天看起来有些明显，但并不是每个公司都是这样开始的。

例如，微软 Azure 开始是试图在云中重建 Windows。当微软的 Windows Azure 团队主管 Manuvir Das 在 2008 年(视频中大约 3 分 38 秒开始)介绍该公司的云方法时，他对 AWS(和类似 Unix 的开放系统)将 Memcached 等模块化组件“修补在一起”的方法不屑一顾，而是说云缺少的是“一个可以直接使用的统一操作系统”。他接着说，“它被称为 Windows Azure，因为它基于 Windows，我们认为它是云的操作系统。”

那是当时，这是现在；而“然后”对微软来说并不是很管用。虽然 Azure 继续运行在经过重大修改的 Windows 版本上，但它不再试图将 Azure 作为操作系统公开。取而代之的是，它跟随 AWS 和其他公司发展了一个由可组合部分组成的开放生态系统，开发者可以组合这些部分来运行他们的应用程序。不出所料，Azure 今天的大部分工作负载都运行在 Linux 上。

这是实现云计算的正确方法，这在很大程度上是拥抱开源精神的问题。分析师 Tony Baer [说](https://mobile.twitter.com/TonyBaer/status/1248420710432694275)通过这种方式，云提供商避免了“不得不(总是)重新发明轮子。”AWS 合作伙伴负责人马可·贝克[同意](https://mobile.twitter.com/MarkABaker/status/1248547982804312065)，他说:“开源使软件商品化。云是开源软件与硬件、用户界面和界面的打包，以实现服务交付的商品化。”

## **接下来…从开源开始**

这就把我们带到了一个重要的，存在主义的问题上。虽然云使得开源对于企业来说更容易获得，更有生产力，但是没有云开源并不是“不可能的”,就像没有开源就不可能有云一样。是的，好多了。更易使用、更易扩展、更易访问。是的。是的。是的。

但并非不可能。

AWS 交付实践经理 Adam Ulvi 认为，开源先于云。他在一次短信对话中告诉我，“我在这个行业已经呆了很长时间，足以看到我们从物理机转向虚拟机，从数据中心转向云。我不知道接下来会发生什么，但我可以肯定的是，不管是什么，它都是从开源开始的。”

从机器学习到大数据等众多当前技术趋势中，我们肯定看到了这一点——所有这些趋势都始于开源。此外，正如 Spark Labs 首席执行官[劳伦·库尼所说](https://twitter.com/lcooney/status/1248620822408785921)，几乎所有的现代软件开发原则和云工具都是开源的或者深受其影响。

换句话说，云可能是一个平台，但它是开源的，它带来了许多更有趣的事情。

Brescia 表示，开源协作也在继续帮助我们从云获得最大收益。她说，一个例子是网飞——它开放了混沌猴的源代码，并帮助其他人建立了更具弹性的系统。反过来，我们已经看到云提供商，包括 AWS，更积极地贡献代码——包括像[鞭炮](https://aws.amazon.com/blogs/aws/firecracker-lightweight-virtualization-for-serverless-computing/)这样的基本构件。这很好，有助于维持我们在整个行业看到的令人难以置信的云发展势头。

然而，我们需要更多。

最好的开源软件来自那些日复一日使用该软件的人，布雷西亚举的网飞的例子尤其有力。相比之下，威尔逊说，“软件供应商主导着企业软件领域。我认为这就是为什么 T2 如此糟糕的原因。因为开发软件的人不是**用** it 来解决问题。”相比之下，随着新一波企业对开源做出贡献，他[继续](https://twitter.com/_msw_/status/1207337670336114690)“这将会更好，因为将软件投入应用实践的正反馈循环。”

它来了，正是时候。世界上最好的开源软件越来越多地运行在云中，通常作为云服务，云提供商和该软件的其他用户积极地做出贡献。开源和云一起提供了一个非常积极的反馈循环，有望创造一个日益开放的云服务的黄金时代。
