# Codenvy:你准备好持续发展了吗？

> 原文：<https://thenewstack.io/codenvy-changing-face-continuous-integration/>

总的来说，如果没有一个可靠的 CI/CD 工作流策略，许多涉足软件开发的组织会发现自己很快就会陷入困境。onboarding 自动化工具不仅有助于简化开发人员的工作流程，还可以更容易地看到问题发生在哪里，如何最好地解决它们，并允许更简单地分析自动化流程的哪个部分未能正常运行。有了全面的 CI/CD 设置，组织能够轻松地执行回滚、安排自动化构建测试，或者深入到病态服务或容器背后的“为什么”。

在这一集的[新堆栈分析师](/tag/the-new-stack-analysts/)中，新堆栈创始人亚历克斯·威廉姆斯与 CI/CD 工具提供商 [Codenvy](https://codenvy.com/) 的首席执行官[泰勒·朱厄尔](https://twitter.com/TylerJewell)以及联合主持人、TNS 编辑本杰明·鲍尔聊天。三人讨论了 Codenvy 在持续部署领域的价值主张、集装箱化对 CI/CD 的影响以及生产自动化工作流程的未来。

[# 90:Codenvy——你准备好持续发展了吗？](https://thenewstack.simplecast.com/episodes/90-codenvy-are-you-ready-for-continuous-development)

“仍然有许多组织试图弄清楚如何做好 CI &自动化测试。组织仍在学习如何成为一种多编程语言。朱厄尔说:“我们仍然可以看到很多用一种语言写成的单行本。

许多商店面临的一个障碍是理解容器的价值。当将容器与单实例服务或应用程序进行对比时，最初可能会发现它们有所欠缺。然而，这就像把苹果比作橘子一样。在大规模运行应用程序时尤其如此。“考虑[Codenvy]的[Che](https://eclipse.org/che/)是一回事，它有一个单用户容器和一个工作区的单用户服务。当数百名并发开发人员自己构建需要隔离网络和长期存储的多容器应用程序时，情况就完全不同了。由于整个架构是建立在容器上的，了解我们如何在架构层面上改进这一点是我们的动力，”朱厄尔指出。

Codenvy 在快速增长的 CI 领域脱颖而出，为其用户提供了各种功能，不仅允许他们提供安全的工作空间，还允许他们利用强大的工具集(如 [RSync](https://ss64.com/bash/rsync.html) )来加强安全性和存储。RSync 允许 Codenvy 通过自定义实现 [GoRouter](https://github.com/cloudfoundry/gorouter) 来强制隔离，同时运行 RSync 来长期存储文件。朱厄尔指出，基于 Linux 的协议可以在一个系统上复制文件，在另一个系统上进行编辑，并在它们之间进行同步。虽然这些变化令人兴奋，但 Codenvy 希望在将容器作为当今 CI 和 CD 工作流不可或缺的一部分时，为开发人员和最终用户提供一种新的思维框架:

“容器作为应用程序架构的通用格式，意味着每个人都可以访问应用程序，包括用户。然后反馈会流向需要它的观众，”朱厄尔说。

Codenvy 是新堆栈的赞助商。
