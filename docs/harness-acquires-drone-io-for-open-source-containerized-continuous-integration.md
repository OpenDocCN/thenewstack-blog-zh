# Harness 收购 Drone.io 进行开源容器化持续集成

> 原文：<https://thenewstack.io/harness-acquires-drone-io-for-open-source-containerized-continuous-integration/>

两家公司周三宣布，连续交付(CD)软件提供商 [Harness](http://harness.io/) 收购了开源连续集成(CI)软件公司 [Drone.io](https://drone.io/) ，完善了其产品组合，提供了一个完整的软件交付平台。条款没有披露。

从 Drone.io 的角度来看，这次收购“仅仅意味着 Drone 将变得更好、更快。Harness 有一个巨大的愿景，一个伟大的公司文化，”Drone.io 的首席执行官兼创始人 [Brad Rydzewski](https://github.com/bradrydzewski) 说，“我们不会改变一切。我们仍然以开发者为中心。但现在我们可以利用更多的资源，我们可以为这个项目带来更多的雄心。”

除了扩大 Harness 在不断发展的 CI/CD 领域的存在，这次收购将是 Harness 首次涉足开源软件。Harness 首席执行官兼联合创始人 Jyoti Bansal 在接受 New Stack 采访时表示，就像 New Relic 上周宣布转向开源一样，Harness 正在努力将开源纳入其商业战略。

Harness 最初是一家严格专注于提供持续交付服务的公司。Bansal 指出，许多组织试图将他们的 CI 工具扩展到管理在 CI 链上构建和测试的成品软件工件的部署，但这并不合适。6 月，该公司[扩展了其平台](https://thenewstack.io/harness-io-translates-cloud-costs-to-developers/)，以提供基础设施使用的财务信息。

尽管如此，Bansal 认为集成 CI/CD 空间正在演变成一个更大的问题，即软件交付，拥有一个 CI 组件将是支持整个生命周期所必需的，他说。IDC 预计，到 2023 年，DevOps 软件市场将达到 150 亿美元。

Bansal 指出，Drone.io 吸引 Harness 的原因有很多。一个欣欣向荣的开源社区已经围绕着这个软件成长起来——根据最近的统计，有超过 50，000 名活跃用户。CapitalOne、思科、易贝和 VMware 都使用过 Drone.io。据 Bansal 估计，该软件易于部署和管理。它也符合[尖端的云原生 GitOps 驱动架构](https://thenewstack.io/drone-io-a-containerized-ci-cd-system-for-cloud-native-development/)。在新闻公告中，Harness 指出:

不需要脚本，持续集成管道可以作为 Git 中的代码来声明和管理，这意味着它们具有标准的语法，需要更少的工作，并且易于工程师创建、使用、维护和故障排除。Drone.io 插件都是容器化的，因此也是标准化的，这使得它们易于下载、使用、升级和创建。

Harness 估计，Drone.io 可以为工程师节省 5-10 倍的时间和持续集成任务的成本，让他们有更多时间专注于编写代码。

Harness 承诺支持 Drone.io 的开源社区，目前正在审查自己的软件，寻找可能的开源版本。开源无人机将更名为 Harness CI 社区版。目前的无人机企业版将作为HarnessCI Essentials Edition 出售，今年晚些时候， Harness 将推出 Harness CI 企业版。
