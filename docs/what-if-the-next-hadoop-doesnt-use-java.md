# 下一个 Hadoop 不使用 Java 怎么办？

> 原文：<https://thenewstack.io/what-if-the-next-hadoop-doesnt-use-java/>

随着人们对 Java 的兴趣逐渐减弱，我们不得不问自己是什么让 Java 存活下来的？这当然不是新的语言或语法特性。不，Sun/Oracle 的艰难管理并不是 Java 得以生存的原因。这是一个巨大的、充满活力的生态系统和社区。

Java 社区已经承担起推动语言和技术进步的责任。虽然 Java 缺乏一种动态语言，但是社区已经创建了好几次了。这些 Java 社区项目中的一些非常受欢迎，有移植到其他语言的地方。

下一个问题是为什么是 Java？为什么这些社区项目使用 Java 而不是其他语言？Java 通常等同于它的企业和跨平台能力。这一点，再加上它是“开源的”，导致其他社区项目本身也是真正的开源。

### 一个新的追求者

这就引出了微软最近宣布已经开源了. NET，看了这个公告，我还以为是个玩笑。我的第二个想法是这对 Java 意味着什么。现在 Java 有了一个真正的竞争对手。在 Java 停滞不前的时候。NET 已经添加了许多特性，这些特性让我们 Java 程序员渴望得到这种语法糖。

这一声明可能意味着下一个 Hadoop，或本应使用 Java 但使用。NET 将是一种真正的可能性。强大而创新的 Java 社区可能会变成强大而创新的社区。网络社区。这种差异可能会让 Java 逐渐减弱的兴趣变得更加强烈。

### **菜谱**

所有这些都表明，这需要一些时间来完成。NET 是一个真正可行的竞争对手。到现在，微软的。NET 解决方案仅适用于 Microsoft Windows。这就忽略了 Mac OSX 和 Linux。Linux 是这里的关键，因为绝大多数后端企业软件都运行在 Linux 上。做这个需要多长时间？NET VM 和 Java 的一样扎实？

说大多数 Java 开发人员和微软之间有爱恨交加的关系是一种保守的说法。它近乎宗教狂热。一个新开放的源代码。NET 与 Java 社区达成停战协议了吗？

### **风景**

Java 第一次有了一个开源的企业竞争对手。这将使下一件大事的创造者在选择语言时三思而行。Oracle 应该开始以一种有意义的方式推动 Java 向前发展。我把钱押在甲骨文上，让社区来决定；现在有一个选择。
