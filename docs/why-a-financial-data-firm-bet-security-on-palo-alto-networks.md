# 为什么一家金融数据公司将安全押在 Palo Alto Networks 上

> 原文：<https://thenewstack.io/why-a-financial-data-firm-bet-security-on-palo-alto-networks/>

来自帕洛阿尔托网络公司的 Prisma Cloud 赞助了这次播客。

数据和对数据的监管访问都是我们生活中不可或缺的一部分。在自由访问大量无处不在的数据的同时，确保保护到位的责任也随之而来。对于组织而言，一系列接入点都需要数据保护，包括应用程序、主机、容器和无服务器架构。

在本期由 [Alex Williams](/author/alex/) 主持的 [The New Stack Makers](/podcasts/makers) 播客中，我们采访了[路孚特](https://www.refinitiv.com/en)的公共云迁移风险产品安全首席架构师 [Darian Jenik](https://uk.linkedin.com/in/darian-jenik-6a32025) 。

[为什么一家金融数据公司将安全押在帕洛阿尔托网络公司身上](https://thenewstack.simplecast.com/episodes/why-a-financial-data-firm-bet-security-on-palo-alto-networks)

路孚特为全球 40，000 家机构提供金融相关信息、数据和分析。Jenik 讨论了路孚特如何使用 Prisma Cloud 作为其定制云安全报告应用程序的基础。在涵盖的主题中，他分享了他最初的安全挑战，这些挑战促使路孚特考虑像 Prisma Cloud 这样的第三方解决方案，以及推动创新的新定制应用程序需求的因素。

这段播客是为 6 月 24 日举行的[云原生安全虚拟峰会](https://start.paloaltonetworks.com/state-of-cloud-native-security)录制的。

作为世界上最大的交易平台和大多数顶级金融机构的“巨型数据轮”，该公司从[汤姆森路透](https://www.thomsonreuters.com/en.html)剥离后的第一个雄心是从传统的数据中心运营迁移到云环境。

Jenik 说，要找到合适的安全平台来满足路孚特的要求，就需要找到一家能够“找到并识别那些可能存在问题的东西”的提供商。该公司在查看了“所有主要品牌”后选择了 Prisma Cloud，而 Prisma Cloud“为我们检查了所有正确的选项”

“它让人们看到正在发生的事情，看到发生了什么变化，”詹尼克说。“我们的运营团队对做出这一决定至关重要。”

在这些挑战中，在内部构建必要的安全脚本所投入的时间没有得到回报。他解释说，Prisma Cloud 提供了一个自动化的替代方案，包括“数百种不同的规则，然后你可以根据环境定制和裁剪它们”。

在采用 Prisma Cloud 安全技术后，路孚特的下一阶段包括改进其白名单程序。“有数百份报告正在发布，这是我们正在完善的，也是我们正在学习的。我们在报告中看到一些东西，然后团队表示我们必须这样做，”Jenik 说。“我们正在了解白名单有多么重要……因为你不想让报告、警报和不真实的报告超载。”由于经常出现过量的假阳性，“我们必须能够关闭它，因为团队会产生警报疲劳，”Jenik 说。

关于路孚特采用 Prisma Cloud，Jenik 从 cloud native technology 中获得的一些信息是，以前“我们不确信我们看到了一切，或者我们甚至没有看到正确的地方。”

“这给了我们信心——我们看到了警报和我们没有想到会发生或我们应该担心的事情。所以，它教会了我们，你真的不知道你不知道什么，直到你得到这样的东西。”
