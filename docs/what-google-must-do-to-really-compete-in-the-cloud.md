# 谷歌要在云计算领域真正竞争必须做些什么

没有人质疑收入 400 亿美元的公共云巨头[亚马逊网络服务](https://aws.amazon.com/) (AWS)是否赢得了公共云战争的第一波。现在的问题是:其他服务，比如像[谷歌云](https://cloud.google.com/)如何在第二波中竞争，占据市场份额，并有可能克服 AWS 看似不可逾越的领先优势？

与直觉相反，谷歌通往企业的最佳路线与基础设施关系不大，而与软件关系更大。随着运营支出超过资本支出，基础设施一直是公共云的一个关键卖点。

在看谷歌应该怎么做之前，我们先了解一下他们为什么要这么做。既然谷歌在公有云的主要竞争对手是 AWS，那么值得一问的是 AWS 最大的优势是什么，如何用柔道般的招式来对抗他们？作为迄今为止最大的云提供商，AWS 受益于网络效应——对于客户来说，他们在 AWS 上运行的应用越多，在 AWS 上运行所有应用就越好。更多的应用程序意味着更大的规模经济和更低的用户成本，从而带来更多的智能，从而带来更好的服务，等等。因此，AWS 非常担心，如果市场份额开始侵蚀，那么网络利益也会侵蚀，从而形成恶性循环，导致 AWS 基础设施的商品化。

当这种恐惧导致在 AWS 贸易展上对“微软”、“谷歌”或“多云”等词过分热心的粘贴时，你很清楚应该把攻击的重点放在哪里。如果 AWS 想要禁止多云这个词，更不用说多云现实了，谷歌需要让多云茁壮成长。

那么，它是怎么做到的呢？在多个云中运行企业应用程序的能力是一个复杂的命题，但不是不可能的。它要求:

*   “消除”操作环境之间差异的软件；因为否则，多云仍然是一个白日梦。
*   一致管理大型资源池(计算、网络、存储、数据服务等)的软件，无论底层基础架构如何；因为否则您的 IT 管理员将无法应对额外的运营负担。
*   统一的管理界面，提供对不同资源的控制，就像它们不是不同的一样；因为不同的管理系统增加了复杂性。
*   能够轻松满足企业在灾难恢复、备份、数据安全、迁移、容量管理、合规性等方面的需求；因为否则，不会在所有内部和云环境中采用。

谷歌在这些方面做得怎么样？

我给他们打了个 B，考虑到提供大规模的、可行的服务来替代 AWS 的优势，这已经很不错了。

*   “消除”操作环境差异的软件= Linux 容器
*   管理大型资源池的软件= Kubernetes
*   统一的管理界面= Google Anthos
*   轻松满足企业需求:灾难恢复、备份、数据安全、迁移、容量管理、合规性等=不完整

Linux 容器为多云策略提供了基础。虽然它们不是必需的，但它们在消除云和本地数据中心之间的微妙环境差异方面做得很好，并为采用 AWS 提供了基本的构建块。

因为一个典型的应用程序拥有的轻量级容器比重型 VM 对等物多得多，所以需要一个系统来管理这么多的容器。极具先见之明的 Google 在 2015 年发布了 Kubernetes，解决大规模容器管理的问题。

Google Anthos 将 Kubernetes 向前推进了一步，为跨多个环境运行的许多 Kubernetes 集群提供了一个统一的管理层。Anthos 是这个难题的关键部分，但今天它只提供了企业自信地在 Anthos 上运行应用程序所需的一半体验。

Anthos 少了什么？真正集成的存储和数据管理。有多少企业 app 没有数据？有多少人不关心备份或灾难恢复？有多少不关心数据安全？财富 500 强是谷歌最富有的增长目标，没有一个成员不关心这些数据管理需求。然而，解决这些需求的存储和数据管理还不是 Google Anthos 软件大脑的核心。关心备份、灾难恢复、数据安全、迁移、容量管理的客户被要求在 Anthos 的之外构建一个解决方案来满足这些要求。这就像邀请你所有的朋友和家人在后院烧烤，提供汉堡包，但没有面包。

因为客户必须为最重要的企业应用需求构建自己的软件解决方案，所以谷歌将他们对抗 AWS 的成功留给了机会。如果在 AWS 上获得市场份额意味着让更多的应用程序运行多云，那么让客户自己为备份、灾难恢复、数据安全、容量管理和迁移带来解决方案是一种风险。

要想在云计算领域真正竞争，谷歌需要将跨所有环境的存储和数据管理置于其 Anthos 战略的核心。AWS 将很难掩盖这一点。
