InfluxData的时间序列查询语言，Exoscale的容器栈2019

本周，在我们的每周新闻总结播客New Stack Context中，我们采访了 InfluxData 的创始人兼首席技术官保罗·迪克斯，谈论为什么现在需要时间序列数据库。

新的Stack赞助商InfluxData在2013年推出了InfluxDB，一个开源时间序列平台。从那时起，该软件已经有了数百万次的下载，建立了一个不断扩大的企业客户名单，并培养了一个不断增长的社区。influx data CMO·马克·赫林本周在新的堆栈上写道大多数数据最好在时间维度上理解，所以时间序列空间才刚刚开始。Paul与我们讨论了为什么会出现这种情况，还分享了一些关于公司新的时间序列数据查询语言的信息，这种语言叫做Flux。

在节目的后半部分，我们将回顾本周的头条新闻，并讨论Exoscale的容器堆栈，这是一个关于与Kubernetes进行DevOps的会议，将于1月30日至31日在苏黎世举行。在此次活动中，来自制药、银行、媒体和研究行业的最终用户以及云原生专家将分享他们的经验和最佳实践。Aqua的Liz Rice、Weaveworks的Alexis Richardson和CERN的Ricardo Rocha将在会上发言。将涵盖堆栈的所有层，包括网络层、存储层、打包层和专业服务。

在最近一次关于新堆栈的直播中，Exoscale首席技术官Pierre-Yves Ritschard分享了他对欧洲市场集装箱化状况的看法。他说，虽然在容器化工作负载方面，欧洲组织可能有点落后于美国同行，但他们正在迅速赶上。

TNS编辑总监 Libby Clark 主持了这次活动，并得到了TNS创始人 Alex Williams 和TNS执行主编 Joab Jackson 的协助。
