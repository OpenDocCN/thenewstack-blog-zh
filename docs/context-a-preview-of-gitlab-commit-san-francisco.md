# 背景:旧金山 GitLab Commit 预览

> 原文：<https://thenewstack.io/context-a-preview-of-gitlab-commit-san-francisco/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们与 [GitLab](https://gitlab.com/) 的产品营销经理[约翰·耶利米](https://gitlab.com/johnjeremiah)就 1 月 14 日在三藩市举行的 [GitLab Commit](https://about.gitlab.com/events/commit/) 用户大会进行了交谈。

《新书库》的编辑和营销总监利比·克拉克(Libby Clark)在 TNS 创始人兼出版商亚历克斯·威廉姆斯(Alex Williams)和 TNS 执行主编约阿布·杰克逊(Joab Jackson)的帮助下主持了这一集。

GitLab 是新堆栈的赞助商，它以基于 git 的版本控制而闻名，但现在它正在寻求将持续集成和持续部署自动化并整合到单个生命周期中。我们参加了 GitLab 的上一次提交活动，[于 9 月在布鲁克林](/party-circuit-gitlab-commit-new-york-goes-bowling-in-the-borough/)举行，这给了我们许多关于 DevOps 实践的[有趣的故事](https://thenewstack.io/gitlab-commit-brooklyn-devops-as-a-single-application/)。因此，我们就这些新兴趋势向 Jeremiah 提问，毫无疑问，我们将在旧金山听到更多，包括 DevSecOps 和 DataOps。

[第 99 集 w/ John Jeremiah 预览 GitLab Commit 旧金山](https://thenewstack.simplecast.com/episodes/episode-99-w-john-jeremiah-a-preview-of-gitlab-commit-san-francisco)

然后，在节目的稍后部分，我们将讨论新堆栈中的其他近期故事和播客。我们谈论全球数据中心的用电量如何继续飙升，并希望获得更清洁的替代能源。还讨论了[谷歌有争议的拒绝](https://thenewstack.io/knatives-independent-future-outside-a-foundation/)将其 Kubernetes 无服务器扩展 Knative 遗留给一个独立的软件基金会，如云本地计算基金会。我们来看看 Podman， [Red Hat 的无守护进程 Docker 替代方案](https://thenewstack.io/check-out-podman-red-hats-daemon-less-docker-alternative/)，它通过更传统的 fork/exec 应用程序模型提供了许多调试和安全优势。最后，我们谈到了 Airbnb [关于微调 Kubernetes 性能](https://thenewstack.io/kubernetes-performance-troublespots-airbnbs-take/)的一次演讲。

云计算原生计算基金会和 GitLab 是新堆栈的赞助商。
