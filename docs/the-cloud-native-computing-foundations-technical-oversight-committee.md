Kubernetes、Prometheus、Envoy、container d……由[云本地计算基金会](https://www.cncf.io/)管理的项目清单相当广泛，而且似乎每个月都在增长。但是 CNCF 是如何管理其庞大的开源云原生软件项目集合的呢？它是如何决定一个项目从孵化到毕业的。这些决定中有许多是由 CNCF 技术监督委员会(TOC)做出的。我们想更多地了解 TOC 是如何运作的。

为此，我们邀请了 TOC 主席(和 Aqua Security Technology 布道者) [Liz Rice](https://www.lizrice.com/) 参加我们最新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客，来讨论 TOC 的所有事情。我们邀请了 Eduardo Silva，他是 CNCF 项目之一 Fluentd 的核心开发者，Fluentd 是一个多平台开源日志聚合器。Silva 也是 ARM 的首席工程师，他讲述了自己同意与 CNCF 合作的经历。上个月，弗伦德已经在 CNCF“毕业”，从“孵化状态”升级

“CNCF 和 TOC 对市场以及未来的趋势和挑战有着广阔的视野。这是一个真正的价值，因为你知道你可以调整你的项目来应对即将到来的挑战，”席尔瓦说。

这个采访是上个月在巴塞罗那的 Kubecon + CloudNativeCon 欧盟会议上录制的。Kubecon + CloudNativeCon 是由云原生计算基金会监督的一项活动。新的 Stack 执行主编 Joab Jackson 主持了这一集。
