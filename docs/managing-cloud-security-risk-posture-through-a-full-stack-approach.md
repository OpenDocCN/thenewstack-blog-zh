# 通过全栈方法管理云安全风险态势

> 原文：<https://thenewstack.io/managing-cloud-security-risk-posture-through-a-full-stack-approach/>

Kubernetes、容器和云原生技术为组织提供了可移植性、灵活性和提高开发人员生产力的好处，但与采用它们相关的安全风险仍然是公司的安全顾虑。在最近的[Kubernetes 安全状况报告](https://www.redhat.com/en/engage/state-kubernetes-security-s-202106210910)中，94%的受访者在过去 12 个月中至少在他们的 [Kubernetes 环境中经历过一次安全事件](https://thenewstack.io/category/kubernetes/)。

在本期 New Stack Makers 播客中， [Orca Security](https://orca.security/) 的首席执行官兼联合创始人 [Avi Shua](https://www.linkedin.com/in/avishua/) 讲述了组织如何通过应对漏洞、恶意软件和错误配置等关键风险，以及拍摄 Kubernetes 集群的快照并进行分析来增强云环境的安全性。

《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了这次播客。

[通过全栈方法管理云安全风险态势](https://thenewstack.simplecast.com/episodes/managing-cloud-security-risk-posture-through-a-full-stack-approach)

传统的安全方法无法完全保护云环境的安全，尤其是这种基础架构中涉及的数据和用户的流动性。随着越来越多的公司认识到云环境造成的攻击面的增加，他们正在应用新的方法来提供更有效的防御方法。

Shua 说:“摩擦是可能损害安全态势并阻止组织实际维护他们所需的安全卫生的主要因素。环境和警报疲劳也增加了安全团队面临的复杂问题，“组织无法真正处理数百万个警报。坦率地说，没有任何数以百万计的严重警报。确保组织安全的唯一方法是将问题放在上下文中考虑，并专注于极少数能够对组织产生实际影响的攻击链。”

在云架构中，周界的概念变得模糊。Shua 说，对身份、数据库和漏洞管理等关键领域采用单独的安全方法会产生“太多的数据点”。Shua 说:“我们真正认真解决安全问题的唯一方法是采用全栈方法，从本质上把这个问题看作一个图形和攻击链。

随着组织迁移到多云环境，这带来了复杂性。Shua 说:“复杂性是安全方面最糟糕的事情之一，因为错误会发生，攻击者会利用这些错误。”。但是，通过监控云基础设施的安全缺口，如配置错误问题和合规风险，他们可以帮助导航不断变化的云安全态势，并帮助“理解通常更复杂的问题的更大态势。”
