# 演示:扭锁安全平台如何支持无服务器

> 原文：<https://thenewstack.io/demo-how-the-twistlock-security-platform-supports-serverless/>

Twistlock 和[帕洛阿尔托网络](https://www.paloaltonetworks.com/prisma/cloud)赞助了这个演示播客。

Twistlock 继续积极地将其[安全平台](https://thenewstack.io/category/security/)扩展到大多数容器和主机环境。在过去 10 年几乎每个季度都发布新版本之后，Twistlock 的 19.07 标志着它的第 17 次发布。这一次，Twistlock 扩大了其影响范围，提供了对无服务器平台的可见性，以补充大多数多云环境。

在由 New Stack 创始人兼主编 Alex Williams 主持的演示播客中， [Twistlock](https://www.paloaltonetworks.com/prisma/cloud) 的产品营销主管 [Keith Mokris](https://www.linkedin.com/in/keithmokris) 表示，Twistlock 的持续目标是“围绕云原生基础设施不同领域的实时可见性收集大量功能。”

Mokris 说，具体到 19.07，这意味着在混合和多云环境中以新的方式可视化无服务器的能力。

“其中一件事是我们的无服务器雷达，它补充了我们今天已经为容器和主机所做的工作。因此，现在能够以一种新的方式可视化 circle 无服务器应用程序，对于我们网络安全部门来说，这是一件非常令人兴奋的事情。”

Mokris 说，无服务器功能反映了 Twistlock 以前版本的一个“关键点”，即能够以“可扩展和可控的方式”将工具的可见性范围扩展到混合和多云环境。

其他新功能包括可信映像，能够控制开发人员可以将哪些映像和代码部署到不同的环境中。Mokris 说，这真的是“安全的一个关键特征，允许他们快速进行”。

Mokris 从云本地网络防火墙的概述开始演示。“如果你以前在 Twistlock 中看到过的一件事是，我们一直强调实时可见性的概念，跨越你环境中所有容器和主机的所有网络连接，”Mokris 说，他接着展示了一个在 Kubernetes 上用 Twistlock 分析的应用程序，该应用程序带有[weaver works](https://www.weave.works/)Sock Shop，它为研究 Kubernetes 如何工作提供了一个很好的窗口。Mokris 展示了 Twistlock 如何通过观察应用程序的本机行为来自动模拟其网络拓扑。“我们可以看到前端如何与用户的服务对话，然后是用户的数据库，”Mokris 说。

在更全面的 19.07 版本中，以及在以前的扭锁版本中，使用扭锁时防火墙不会过时。“我认为我们在会议上经常会遇到的一个问题是‘这是否意味着我的外围防火墙将会消失？’Mokris 说,“这肯定不是真的——你肯定需要保护你的周边。但是，当您使用 Docker、Kubernetes 或其他微服务类型的工具时，您还需要管理和保护应用程序本身的网络层。因此，能够以可扩展的方式做到这一点，是我们在这些新架构中看到的最大挑战之一。"

Mokris 说，当有数万个容器或数百个云原生应用程序时，有必要提供网络层本身的安全管理能力。“我真的认为，能够提供从一个容器到另一个容器或跨混合环境的微分段是我们将越来越多地看到的东西，当然也是这里的一个焦点，”Mokris 说。

虽然许多组织寻求对 Docker 容器和 Kubernetes 的保护，但他们通常也有 Mokris 所描述的“最简单的用例”和“这似乎真的是一个很好的途径……本质上，只是跳过容器，在无服务器上运行他们的应用程序”, Mokris 说。“因此，对于这些应用程序来说，事件驱动的计算非常棒，而且真的很小，”Mokris 说。“我们希望从本质上把所有这些联系在一起，能够识别漏洞状态，然后提供一次性的可见性操作(对于无服务器也是如此)。”
