# Docker 扩展计划，帮助企业容器化遗留 Java。网络应用

> 原文：<https://thenewstack.io/docker-updates-legacy-application-migration-program/>

在过去的六个月里，Docker 一直在正式实现企业的现代化。NET 和 Java 应用程序在云中的容器中运行。这个项目被称为 Docker 传统应用程序现代化项目，简称 MTA。

[Docker](http://www.docker.com) 的首席运营官 [Scott Johnston](https://www.linkedin.com/in/scottcjohnston/) 说:“在这个项目中，我们帮助企业将遗留应用程序带到现代基础设施中，而无需接触源代码。“我们已经看到，在数百个 Java 和中，符合我们标准的应用程序的成功率达到了 100%。NET 应用程序。客户可以在五天或更短的时间内看到这些结果。

Johnston 补充说，一些客户能够将他们软件的发布频率提高一倍，并将总拥有成本降低 50%。

该公司在本周于哥本哈根举行的 Dockercon 欧洲会议上讨论了相关细节。

在周二的主题演讲中，特别提到了保险公司[大都会人寿](https://www.metlife.com/)。“大都会人寿在一天之内就将这种现代化模式应用到了他们的 Java 应用程序中。他们查看了自己的产品组合，发现了 600 个符合这种模式的其他应用程序，从而节省了 66%的总拥有成本。这给大都会人寿带来了数百万美元的收入。他们有超过 6000 个应用程序需要应用这种技术，”Johnston 说。

该程序目前仅限于 Java 和。尽管客户对其他语言和非 X86 平台感兴趣。约翰斯顿暗示爪哇和。NET 占企业应用程序的大多数。“我们发现了大量机会来更新他们的应用程序。Java 和。NET 将是我们重点关注的广阔市场。”

当被问及旧的大型应用程序的工作方式与现代的轻量级微服务和容器方式之间的兼容性时，Johnston 说，在更大的非集群应用程序中仍然可以节省成本。

“那些巨石柱，它们在容器里仍然工作得很好。不止好；他们的总拥有成本降低了 50 %,而且他们已经为混合云做好了准备，更加安全。Johnston 说:“Docker 在四年前开源，它与微服务一起崛起，但我们发现 Docker 可以应用于这些整体。”

Johnston 说，将一个巨大的应用程序转移到容器的好处是，它提供了一个操作基础，从那里可以将大服务削减为更小的微服务，而不会中断核心功能。

“一旦它们进入容器，客户就可以控制应用程序的各个方面，以及它们变现的速度。ROI 需要微服务吗？它需要重构吗？或者，你能抽出一两个部分，把它们变成微服务吗？剩下的部分工作得很好，因为我们将在 12 个月后淘汰它。”约翰斯顿问道。

“相比之下，其他方法会说，‘我们将带你到云和微服务，你必须重构你的应用，重新培训你的人员，并摧毁你的基础设施。’我们的不需要这个，”约翰斯顿说。

Johnston 说，无论如何，一些遗留应用程序已经成熟，可以转变为基于容器的服务。“ [Tomcat](http://tomcat.apache.org/) 或 servlet 模式非常适合放在容器中。我们发现这些是我们的客户喜欢采用的更简单的模式之一，”Johnston 说。

Docker 将在 10 月 31 日的[网络广播](https://goto.docker.com/WR2017-10-31ModernizeTraditionalAppsMTAwithDockerAvanadeandMicrosoft_LP.html?utm_campaign=MTA+Kit&utm_medium=social&utm_source=Organic&utm_content=1507834189)中更详细地解释该计划。
