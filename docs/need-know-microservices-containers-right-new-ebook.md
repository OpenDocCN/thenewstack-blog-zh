# 你需要知道的关于微服务和容器的一切都在我们的新电子书里

> 原文：<https://thenewstack.io/need-know-microservices-containers-right-new-ebook/>

作为一种架构模式，微服务在过去几年中受到了很多关注，达到了大多数从业者难以忽视的采用和传播水平。但是，尽管微服务架构因为从整体上改变了应用程序开发和生产流程而受到了广泛的赞誉，但仍有许多人认为微服务毫无新意，或者对某些人来说，它只是面向服务的架构(SOA)的一个不同名称。探索推动微服务演变成我们今天所知的模式的驱动因素非常重要，了解是什么工具和服务让它如此受欢迎也非常重要。

今天，我们发布了关于容器革命的第二本电子书，“使用 Docker &容器的[应用&微服务”](https://thenewstack.io/ebookseries/)

在本卷中，我们探索了基于容器的微服务对应用程序开发和交付的影响。我们仔细看看容器对于创建和利用微服务的过程意味着什么，以及容器技术如何在实现微服务模式中变得如此重要。

我们还探讨了微服务架构的构成，提供了一些创建和管理微服务的最佳实践，以及一些将整体应用迁移到微服务的流程。我们还为生产中的容器提供案例研究。

我们也做了大量关于开源容器生态系统的原创研究，我们渴望与我们更广泛的社区分享；本系列对开放源码社区的关注是一个持续的主题，我们正在寻求扩展，我们欢迎对我们迄今所做的工作的反馈。

我们认为这本电子书是建立我们的第一本电子书“码头和集装箱生态系统”的重要一步了解容器和微服务在应用程序开发和架构中的作用，将使我们能够在下一本电子书“Docker & Containers 的自动化和编排”中解决未来的复杂主题，其中我们将涵盖编排、服务发现、调度程序、集群管理、配置管理和跨分布式系统的自动化。

这本电子书，以及这一系列中的三本书，将我们对生态系统的分析分解为需要增加整本书深度的重点领域。我们很早就发现，关于容器生态系统还有很多要说的，这个系列可能只是我们的开始。我们一直在寻找需要更多关注和教育的新主题，我们欢迎任何关于我们下一步应该解决的领域的反馈。
