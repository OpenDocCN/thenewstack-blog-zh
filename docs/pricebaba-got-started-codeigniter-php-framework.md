# Pricebaba 如何在 CodeIgniter PHP 框架上起步

> 原文：<https://thenewstack.io/pricebaba-got-started-codeigniter-php-framework/>

当我在 2011 年计划 [Pricebaba](http://pricebaba.com) 的时候，我知道技术方面的事情会很棘手。我对 Pricebaba 的设想是，它将成为一个产品研究引擎，让印度消费者能够在当地商店以及网上商店搜索手机的供应情况。

为了让这项服务成为现实，我需要朋友们尽可能的帮助来破解它。我最好在这里澄清一下，我不是工程师，这意味着我不会写一行代码，除了我在学校学到的一些。

幸运的是，我的更倾向于技术的联合创始人 [Tirthesh Ganatra](https://twitter.com/ganatrat) 可以应对这些挑战，但作为一名大学毕业生，他也有很多东西要学。因此，当决定 Pricebaba 的 Web 应用程序堆栈时，我们感到困惑，并与许多将我们带到不同方向的意见相冲突。

我们的标准很简单。我们想使用已知的东西，我们知道规模的东西，我们将能够找到足够的大脑来建议我们。与此同时，我们想要一个雇佣工程师不会成为麻烦的平台。所以我们去了 PHP，MySQL 和 Google Analytics。(我给了我的非技术免责权？)

最困难的部分是决定我们是否应该继续使用像 [CodeIgniter](https://codeigniter.com) 这样的框架来构建 Pricebaba。CodeIgniter 的论点是，它将使开发速度更快，并解决开箱即用的基本安全问题。与此同时，反对使用框架的理由是，它会使应用程序变得更慢，并且可能无法伸缩。这是我们不得不冒的险。这就像是在自动变速器汽车和手动变速器汽车之间进行选择。

CodeIgniter 是一个 MVC(模型-视图-控制器)风格的应用程序开发框架，旨在允许个人和公司以比使用 PHP 更快的速度构建一个丰富的网站。它被设计得很灵活，只有 2MB 的内存。它需要最少的配置，并且可以支持第三方插件。

我们选择了 CodeIgniter。三年后，我们仍然主要使用 CodeIgniter，尽管现在我们的技术堆栈中出现了更多的术语。我们已经为超过 8500 万的访问者提供了服务，现在每月的页面浏览量超过 1400 万。可以肯定地说，CodeIgniter 为我们提供了很好的服务。这对网站的第一个版本来说是好的，对从那以后为我们扩展的版本来说也是好的。

当然，巨大的挑战仍有待解决。我们需要让 Pricebaba 适应移动应用的世界。做更复杂的东西，这将意味着分解和重建我们在过去几年中做的许多东西。但对于经营一家没有资金支持的小型初创公司，迄今为止大部分时间里只有几名工程师，我并不后悔选择 CodeIgniter。

使用 Zurb Foundation 作为我们的前端框架也是如此。它让我们不必从头开始构建和维护一个专用的移动网站。

是的，衰老是真实的，我们需要重新创造。但是我想花点时间表达一下，在第一天过度设计是没有必要的。当一家初创公司看到像 Pricebaba 这样的规模时，我们会做所有需要做的火箭科学。但是如果我们在第一天就尝试这样做，我们就不会走这么远。
