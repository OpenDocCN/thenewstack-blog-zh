# 背景:微服务的春天，云计算市场的未来

> 原文：<https://thenewstack.io/context-spring-for-microservices-the-future-of-the-cloud-computing-market/>

[VMware](https://www.vmware.com/company.html) 赞助了这次播客。

欢迎来到[新堆栈环境](https://thenewstack.io/podcasts/context)播客，我们将在这里讨论本周围绕云原生技术的最热门新闻。在本周的节目中，我们采访了 New Stack 创始人兼发行人 [Alex Williams](/author/alex) 和 TNS 记者兼专栏作家 [Mike Melanson](https://thenewstack.io/author/mike-melanson/) 关于在 Austin 举行的关键的 SpringOne 平台开发者大会，他们都参加了上周的会议。

TNS 的编辑和营销总监 Libby Clark 和 TNS 的执行主编 Joab Jackson 共同主持了这次播客。

[微服务的春天，云计算市场的未来](https://thenewstack.simplecast.com/episodes/spring-for-microservices-the-future-of-the-cloud-computing-market)

Pivotal 是新堆栈的赞助商，并在展会上发布了许多公告。

该公司发布了 Spring Cloud Gateway 的测试版，该公司在一篇博客文章中将其描述为“一种开发人员友好的方式来配置和路由 API 请求到应用服务。”随着测试版的发布，Spring Cloud Gateway 将获得 Pivotal 的持续维护和支持，以及 Pivotal 平台的单点登录模块的添加，这是该公司针对应用程序和 API 的身份和访问管理工具。

同样在展会上，Pivotal 和微软[预览了](https://thenewstack.io/azure-spring-cloud-offers-managed-deployment-of-spring-boot-applications/) Azure Spring Cloud，这是一个完全托管的服务，用于在微软 Azure 上部署 Spring Boot 应用。Spring Boot 基于 Pivotal 流行的 Spring 框架，是一个开源的基于 Java 的框架，用于创建微服务，它通过将运行时与应用程序打包在一起来实现。

后来在节目中，我们讨论了本周由蟑螂实验室举办的多云会议 [Escape](https://www.cockroachlabs.com/blog/announcing-escape-19-the-multi-cloud-conference/) ，杰克逊也参加了。在那里，我们在与著名的 Wedbush 财务顾问 Dan Ives 的小组讨论中了解了很多关于云市场的现状。他说，目前，大约 30%的 IT 工作负载在云中运行，但在未来几年内，这一比例将增加到 55%。随之而来的将是整个市场的巨变。第一个“早期”阶段是由亚马逊网络服务主导的，下一个阶段将是更多竞争者的竞争。每个软件和基础设施公司都看到了云是增长的源泉，并相应地关注他们的开发工作。

![](img/e3a95dac02e71c88d54c570974a82de4.png)

丹·艾夫斯，逃脱会议

云购买者已经在问更聪明的问题，也在更努力地谈判。他说，多云烘焙比赛越来越激烈了。而且它们并不总是与 AWS 一起使用。如今，10 个重型生产云工作负载中有 7 个将转向微软 Azure，而不是亚马逊 Web 服务，这与几年前完全不同。艾夫斯说，AWS 现在正在微软的圈子里竞争，这比仅仅抓住所有早期用户更难。

同样在会议上，Red Hat 的 [Dan Walsh](https://twitter.com/rhatdan) 谈到了在云操作中维护开源软件的重要性。他对当前的云计算市场领导者也没有什么好话要说。他将 AWS 比作“Hansel & Gretel”的故事，云巨人扮演女巫的角色，在狡猾地将用户锁入笼子之前(即最初的“锁入”)不断喂给用户大量糖果。

“开源和亚马逊之间正在进行一场战斗，”沃尔什直言不讳地说。
