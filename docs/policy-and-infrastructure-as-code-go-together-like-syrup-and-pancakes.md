# 作为代码的政策和基础设施就像糖浆和煎饼一样形影不离

> 原文：<https://thenewstack.io/policy-and-infrastructure-as-code-go-together-like-syrup-and-pancakes/>

许多组织需要为他们的分布式系统制定更好、更严格的基础设施策略。越来越多的错误配置凸显了这种需求，尤其是在分布式微服务和 Kubernetes 环境中。

作为代码的策略如何扩展作为代码的基础设施在最新一集的 New Stack Makers 播客中进行了讨论，这是我们的另一个“煎饼和播客”特别集。嘉宾有[迪帕克·吉里达拉戈帕](https://www.linkedin.com/in/deepakgiridharagopal),[傀儡](https://www.diagrid.io/?utm_content=inline-mention)的首席技术官； [Tiffany Jachja](https://www.linkedin.com/in/tiffanyjachja) ，Vox Media 的数据工程经理；[詹姆斯·特恩布尔](https://www.linkedin.com/in/turnbulljames)，国际知名奢侈品和艺术品拍卖商苏富比的工程副总裁；还有自称 DevOps 技术呆子的谢伊·斯图尔特。TNS 创始人兼发行人亚历克斯·威廉姆斯和 TNS 新闻总监[达里尔·塔夫脱](/author/darryl-taft/)一起主持了播客。

[政策和基础设施作为代码就像糖浆和煎饼](https://thenewstack.simplecast.com/episodes/policy-and-infrastructure-as-code-go-together-like-syrup-and-pancakes)

“在过去的一年里，我们看到了许多灾难，它们导致了安全漏洞或黑客攻击，影响非常持久。这不仅仅影响到一个特定的团队，它还影响到所有的企业和客户，以及他们客户的客户，”Jachja 说。“配置错误非常非常普遍。据报道，我认为 80%的安全攻击是由错误配置引起的，所以这实际上是人们进入 Kubernetes 或仪表板的结果。”

与管理内部和云原生环境相关的复杂性意味着 [DevOps 团队](/category/devops/)与过去的单一系统相比，需要监控的内容更多。Giridharagopal 说:“与人们过去构建应用程序和基础设施的方式相比，在所有不同的构建模块上有更多的旋钮和开关。“有更多的东西，我认为有更多的方法来配置它。它在一定程度上是分布式的、蔓延式的，我认为这些旋钮和开关在不同事物的所有不同实例中具有相同的状态变得更加重要。”

正确的策略——或作为代码的策略——有助于解决现代环境的复杂性难题，并避免错误配置。例如，特恩布尔说，政策“允许你坐在高水平上”。

特恩布尔说，制定政策涉及在整个环境中设置一系列声明或规则，如果试图访问的行为违反了政策，这些声明或规则可能会标记或拒绝对应用程序、网络或数据的访问。

策略即代码作为基础设施即代码的扩展也应该涉及 DevOps 文化。孤立的团队不再是一种选择。换句话说，人的因素仍然至关重要。斯图尔特说:“在我看来，最突出的是所有这一切中的人的因素——让更多的人掌握它，让他们能够理解或围绕一个主题合作，并邀请合适的人来参加会议。”
