# DockerCon 2019:关于端到端容器生命周期的一切

> 原文：<https://thenewstack.io/dockercon-2019-all-about-the-end-to-end-container-lifecycle/>

虽然 Kubernetes 可能在过去几年的云原生计算革命中抢了 Docker 的风头，但 Docker 正在接管让基于容器的开发真正为企业所用的工作。

在本周于三藩市举行的 [DockerCon 2019](https://www.docker.com/dockercon/) 的开幕主题演讲中，Docker 首席执行官[史蒂夫辛格](https://twitter.com/SteveSSingh)强调了 [Docker Enterprise](https://www.docker.com/products/docker-enterprise) ，该公司最新更新的容器管理包，如何帮助整个开发和部署生命周期。

Docker Enterprise 3.0 包括[多项自动化开发和部署容器化应用的进步](/docker-enterprise-3-0-simplifies-kubernetes-management/)，包括开发人员直接从桌面将应用推入多个云部署的能力，直接在桌面上进行 Kubernetes 和 Docker Swarm 的低维护设置，甚至是 Docker Enterprise 的完全托管版本。该公司的立场是，大多数 IT 商店不具备在内部运行类似 Kubernetes 的专业知识，因此 Docker 提供了一些软件包和工具来尽可能多地自动化繁重的工作。

作为主题演讲的开始，Singh 请来了一些企业用户的代表——Docker 总共有大约 750 家企业客户——来谈谈他们在应用程序容器化方面的经验。对于[嘉年华](https://www.carnival.com/)的技术副总裁托德·赫德(Todd Heard)来说，集装箱化给豪华邮轮带来的主要好处是能够通过该公司的 Medallion 奖励计划，将邮轮旅行的不同方面(如登记和船上服务)连接成一个统一的体验。“每个人都想谈论技术，但在旅游业，一切都是关于体验，”赫德说。

“我们都在云上，”保险公司 Nationwide 的云运营主管 Eric Zielinski 表示同意。该公司正在进行一项雄心勃勃的努力，希望在未来几年内将其大部分应用程序运行在云中，包括许多旧的应用程序。

遗留应用程序是 Docker 重点支持的另一个领域。在他自己的演讲中，Docker 产品高级副总裁 Scott Johnston 将企业应用分为三个阵营:绿地应用(新的云原生应用)、不再开发的遗留应用和“棕地”应用(仍在用新功能构建的遗留应用)。借助 Docker 应用程序和云本地应用程序捆绑包(CNAB)标准等工具，Johnston 承诺所有三种类型的应用程序都可以容器化。

该公司还展示了其最近与 ARM 在[的合作成果。借助 Docker Enterprise 3.0 中的预览功能，开发人员可以使用(基于 x86 的)Docker 桌面实例构建一个完整的基于 ARM 的应用程序。约翰斯顿指出，事实上，Docker Hub 中的所有官方图像都可以在 ARM 上运行，这为使用亚马逊网络服务和分组网络现在提供的低成本 ARM 服务器芯片铺平了道路。](/docker-arm-team-up-to-bring-containers-to-the-edge/)
