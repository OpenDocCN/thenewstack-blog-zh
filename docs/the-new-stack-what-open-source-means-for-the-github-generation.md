# 新的堆栈:“开源”对 GitHub 一代意味着什么

> 原文：<https://thenewstack.io/the-new-stack-what-open-source-means-for-the-github-generation/>

欢迎来到[新的堆栈环境](/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了来自[亚马逊网络服务](https://aws.amazon.com/)开源办公室的负责人[马特·阿萨伊](https://www.linkedin.com/in/mjasay/)，谈论他在新堆栈上的新系列帖子[，这些帖子记录了我们每天使用的许多最受欢迎的开源程序背后的贡献者和原创者。](/open-source-builders-an-inside-look/)

TNS 编辑和营销总监[利比·克拉克](/author/libby/)主持这一集，旁边还有 TNS 高级编辑[理查德·麦克马努斯](/author/richard/)和 TNS 执行主编[约阿布·杰克逊](/author/joab/)。

[第 123 集:“开源”对于 GitHub 一代来说意味着什么](https://thenewstack.simplecast.com/episodes/episode-123-what-open-source-means-for-the-github-generation)

在过去的几周里，AWS 的 Asay 一直在开源世界旅行——实际上——在新的堆栈上写了一系列有趣的系列文章，记录了我们每天使用的许多最受欢迎的开源程序背后的贡献者和原创者。

在这个系列中，我们遇到了十几个项目背后的开发人员，包括 [Wireshark](/wireshark-how-being-broke-inspired-a-vibrant-open-source-community/) 、 [Matplotlab](/open-source-builders-why-data-scientists-love-matplotlib/) 、 [Curl](/youre-addicted-to-curl-you-just-didnt-know-it/) 和许多其他广泛使用的工具。用 Asay 的话来说，该系列的想法是“将聚光灯照射到一系列默默服务于幕后的开源项目(及其创始人和/或主要维护者)。在这个过程中，我希望我们能够深入了解这些至关重要的项目为何以及如何能够繁荣发展如此之久。反过来，这可能会为如何最好地维持开源项目提供有用的信息。”

在这次采访中，我们询问 Asay，他从与所有这些创造者的交谈中，对项目管理和开源本身有什么样的了解。我们谈论如何加入一个开源项目，以及为什么维护者很难吸引更多的帮助(以及，在某些情况下，为什么他们可能根本不想要贡献)。议程上还有 Asay 与开源合作的悠久历史，开源许可的重要性，年轻一代开发人员如何看待“开源”的想法，以及它在世界范围内被接受的漫长道路。

“我花了 10 年时间抨击微软的机器，指责 FUD 在 SUSE 和 Linux 等方面的做法。现在，我花了同样多的时间赞扬微软对开源做出的巨大贡献。但是人们不知道那段历史。”
