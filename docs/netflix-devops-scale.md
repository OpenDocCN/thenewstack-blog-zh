# 网飞如何大规模支持 DevOps

> 原文：<https://thenewstack.io/netflix-devops-scale/>

[科技与文化如何相互影响](https://thenewstack.simplecast.com/episodes/how-technology-and-culture-influence-each-other)

在最新一集的[新堆栈制造商](https://thenewstack.io/podcasts/makers)播客中，我们采访了网飞工程工具工程总监 [Dianne Marsh](https://www.linkedin.com/in/diannemarsh/) 。她的团队负责构建工具，包括 [Spinnaker](https://www.spinnaker.io/) ，其他团队使用这些工具将他们的变更部署到云中。这也意味着她的团队处于帮助他们的开发人员提高生产力的顶峰。

她说，在网飞的故事中扮演核心角色的不仅仅是技术，还有文化。Marsh 认为技术影响文化，文化影响技术。你要么制造工具，要么采用它们。这些工具要么补充你的文化，要么挑战它。

“我们希望确保网飞的所有工程师能够像他们希望的那样快速、负责任地工作。如果我们没有向他们提供足够的关于他们正在部署什么或生态系统中正在发生什么的背景信息，那么他们可能会在服务中引发大问题。它们可能会导致服务中断，”Marsh 说。“因此，我们在我们的工具中内置了许多护栏，为我们的开发人员提供上下文，帮助他们做出正确的选择。”
