# 视频:Capital One 用于在线身份管理的 SwiftID

> 原文：<https://thenewstack.io/video-capital-ones-swift-id-online-identity-management/>

本月早些时候，银行业巨头 Capital One [推出了一项新举措](https://thenewstack.io/sxsw-capital-one-opens-apis-third-party-e-commerce-use/)，通过一套 API 和一个开发者门户网站[向第三方开放其部分银行服务。Capital One 希望超越其应用程序和网站之外的服务，](https://thenewstack.io/sxsw-video-capital-one-internal-apis-platforms-designing-developer/)[deve exchange](https://developer.capitalone.com/)。

在最初发布的三个 API 中，有一个看起来具有广泛的适用性，即 [SwiftID](https://developer.capitalone.com/products/swiftid/homepage/) 双因素认证服务。

在奥斯汀的 SXSW 会议上，新的堆栈找到了许多从事该项目的 Capital One 人员:Jamie Ashfield，他是 Capital One 的身份验证产品经理；Keith Hamburg 是 Capital One 消费者身份团队的产品经理，Margaret Mayer 是该公司消费者身份和身份信息平台的负责人。

在本播客和随附的视频中，我们将了解 Swift ID 的由来，它最初是作为一项管理 Capital One 客户身份的内部服务而开发的。“同样的认证能力可以适用于任何类型的消费者参与，”汉堡解释说。

[https://www.youtube.com/embed/t4nb1yEtJpA?feature=oembed](https://www.youtube.com/embed/t4nb1yEtJpA?feature=oembed)

作为受到严格监管的银行业的一员，Capital One 在识别用户及其使用的设备方面建立了相当多的流程，第三方应用程序开发者现在可以利用这些流程来提供自己的服务。Ashfield 解释说,“我们能够获得每个用户设备的独特的深度签名”。

该公司使用 webhooks 将客户的认可反馈给第三方应用，OAuth 用于用户授权和同意管理。该服务是免费的，开发者可以通过开发者门户了解更多信息。
