# 协调应对云中的高级威胁

> 原文：<https://thenewstack.io/orchestrating-against-advanced-threats-in-the-cloud/>

在决定如何控制云基础架构安全性时，第一步是要认识到云(以及与之相关的所有潜在利用)是一个与传统数据中心截然不同的世界。组织仅在密码劫持事件中遭受的损失就已经高达数百万美元。

这期[The New Stack Makers](/podcasts/makers)live stream 播客由 [Alex Williams](/author/alex/) 主持，他是 New Stack 的创始人兼发行人，主要关注与迁移到公共云相关的安全挑战。本次现场问答邀请了 Prisma Cloud 产品副总裁 [Ankur Shah](https://www.linkedin.com/in/ankur-shah-cloudsec) 、Palo Alto Networks 数据科学总监 Alok Tongaonkar[和 Palo Alto Networks 首席数据科学家](https://www.linkedin.com/in/gmhoward)Gaspar Modelo-Howard。

[精心策划的云端高级威胁](https://thenewstack.simplecast.com/episodes/advanced-threats-in-the-orchestrated-cloud)

在这个新的云原生世界中，“攻击面已经发生了变化，”Modelo-Howard 说。他说，因此，关键是要考虑到内部部署和云保护之间的差异是“实质性的”。

Modelo-Howard 说:“如果我是试图保护云原生环境的人，我会说，‘我怎么会受到攻击呢？’你会看到，在内部和云系统上，会有很大的不同。“因此，你必须考虑这些因素，以便有效地防止或检测这些攻击。”

IT 组织通常不会因为让传统的数据中心网络和服务器变得冗余而损失他们在传统基础架构上的投资。因此，如何同时保护传统遗留环境和基础架构环境仍然是一个问题。然而，尽管传统数据中心和云基础架构之间存在固有差异，但许多相同的数据和网络安全原则仍然适用。

Shah 说，容器可能代表“新的和酷的东西”，而“今天的大部分工作仍然是在传统的虚拟机上进行的”。相同的策略和治理适用于内部和云环境，或者适用于虚拟机或容器。“同样的原则和规则也适用，”沙阿说。

云原生和微服务环境为组织更快地部署和更新应用铺平了道路。但是，尽管如此多的云原生基础设施是可编程的，开发人员的自由和机会也带来了更大的被利用的风险。“由于一切都是可编程的，这给开发人员带来了担心安全性的过度负担——并不是每个人都有能力做到这一点，”Tongaonkar 说。“因此，我们需要能够保护开发人员不犯错误的东西，并且在出现错误的情况下，那么[需要]进行适当的监控，以便能够检测到任何导致攻击的暴露内容并进行补救。”

就像在军队服役或商业冒险中一样，对失败的分析可以产生有价值的结果。因此，如果您的云基础架构受到攻击，事后分析至关重要。Tongaonkar 推荐了图形和其他图表，他说这对“检测攻击的每一步”都很重要

Tongaonkar 说:“最终，我们的目标是在云基础设施的所有攻击媒介中‘全面提供安全保障’。”。
