# 持续交付带来的云原生挑战和机遇

> 原文：<https://thenewstack.io/cloud-native-challenges-and-opportunities-with-continuous-delivery/>

[云本土挑战与机遇不断传递](https://thenewstack.simplecast.com/episodes/cloud-native-challenges-and-opportunities-with-continuous-delivery)

云原生对不同的人意味着不同的东西。然而， [Pivotal](https://tanzu.vmware.com/) 的产品副总裁 Ian Andrews 认为持续交付是云原生运动的核心。

“在过去十年左右的时间里，一些组织认为构建和操作软件的模式是无效的。他们需要的变化之一是软件的变化率和管理，”安德鲁斯在谈到云原生范式的演变时说。

处于云原生革命前沿的人们支持像连续交付这样的事情——与传统模式形成对比的软件的频繁发布。然而，当人们接受连续交付模式时，他们面临着他们没有预料到的挑战。

他们太关注开发者了。他们选择了每两周发布一次的模型，但是他们没有对运营模型进行修改。他们没有改变他们的发布工程流程。运营团队习惯于一年两次的变化，我们现在被告知他们必须每两周发布一次软件。这在流程和理念上是一个相当激进的变化。

每个人都在协调，试图找出如何在客户开始注意到之前让代码恢复运行。

“我认为人们面临的最大挑战之一是试图在功能交付或新应用程序开发方面加快速度，但没有在整个组织中进行整体思考，”安德鲁斯说。

他还谈到了持续交付对安全性的影响。“我认为平均来说，在过去的一年里，仅在 Linux 操作系统中，我们一个月就能看到大约 10 个高或关键的 cv，”他说。运行数千台虚拟机的企业客户不仅需要更新应用程序代码，还需要更新操作系统所需的工具和技术。

安德鲁斯说:“我认为大多数人现在已经习惯了这个概念，即使他们在应用程序代码方面没有执行连续交付的实践。“我们现在正在讨论对整个堆栈都这样做。我们谈到了持续交付平台的理念。”
