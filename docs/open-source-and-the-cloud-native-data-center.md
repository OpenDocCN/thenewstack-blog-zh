# 开源和云原生数据中心

> 原文：<https://thenewstack.io/open-source-and-the-cloud-native-data-center/>

服务和应用程序中的开源组件的数量继续呈指数级增长，这种采用在软件的创建、部署和管理方面带来了许多变化。2016 年，应用程序平均有 86 个开源软件组件。

今天，根据 [Synopsys](https://www.synopsys.com/software-integrity.html?utm_content=inline-mention) 的[2021 年开源安全和风险分析报告](https://www.synopsys.com/software-integrity/resources/analyst-reports/open-source-security-risk-analysis.html)，组件的平均数量是 528。

在最新一期的 New Stack Makers 播客中，我们召集了一个小组来讨论开源的采用及其对数据中心运营的影响。

出席嘉宾有[马克·辛克尔](https://www.linkedin.com/in/markrhinkle)、 [TriggerMesh](https://triggermesh.com/?utm_content=inline-mention) 联合创始人兼首席执行官、[肖恩·奥米拉](https://www.linkedin.com/in/shaun-omeara)、[米兰蒂斯](https://www.mirantis.com/?utm_content=inline-mention)领域首席技术官； [Jeremy Tanner](https://www.linkedin.com/in/jeremytanner) ，Equinix 的开发者关系[和](https://metal.equinix.com/?utm_content=inline-mention) [Sophia Vargas](https://www.linkedin.com/in/sophia-vargas-54608220) ，谷歌开源项目办公室的研究分析师。

## 更多选择，更复杂

Equinix 发起的用于部署和管理裸机基础设施的 Tinkerbell 开源项目是 DevOps 团队对改善数据中心内部运营感兴趣的一个很好的例子。Tanner 说，通过 Tinkerbell，“开源开发者能够看到他们的项目如何在裸机上运行，无论是 Arm、x86 还是任何新的[处理器]。

他说，开源基础设施越来越受 API 驱动，特别是与 25 年前 Hinkle 开始职业生涯时相比。他说，在过去，他看到的服务器比今天多得多:“现在我们看界面，这使它变得简单而有趣，因为我们正在集成和组成基础设施，而不是以同样的方式构建它。”

Vargas 说，开源的采用和相关选择的数量也增加了数据中心的复杂性。

“既然我们有这么多选择，我们基本上可以接触到新工具的所有前沿想法，”她说。“这种讨论的一部分是选择一个固执己见的堆栈，或者选择一个经过审查的解决方案，将事情堆在一起，这是你可以相信别人会为你提供支持的事情。”

## 将数据移回内部

O'Meara 表示，越来越多的组织开始对是否所有数据中心都需要在公共云上有更细致的了解。

“越来越多的公司开始审视他们的战略，并想知道(公共云)是否是正确的道路，是否是唯一的选择，”他说。“例如，我们看到越来越多的公司来与我们讨论‘我们如何平衡内部基础设施的使用，然后在需要时存储消费者公共云？’

“人们正试图搬回本地，并改变他们看待数据中心的方式。因此，他们不再将数据中心视为存放一切的物理盒子，而是将数据中心视为工作负载的关键点。”
