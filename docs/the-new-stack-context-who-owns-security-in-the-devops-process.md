# 新的堆栈上下文:谁拥有 DevOps 流程中的安全性？

> 原文：<https://thenewstack.io/the-new-stack-context-who-owns-security-in-the-devops-process/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了[容器安全平台提供商](https://www.linkedin.com/in/talliran) [Snyk](https://snyk.io/) 的开发者倡导者和 Node.js 安全工作组的成员 Liran Tal ，讨论了在 DevOps 过程中谁应该负责安全——安全团队还是开发团队？

TNS 编辑和营销总监 Libby Clark 主持了这一集，旁边是 TNS 创始人兼出版商 Alex Williams 和 TNS 执行主编 Joab Jackson。

第 109 集:devo PS——谁应该拥有安全？

Tal 最近为我们写了一篇文章，名为“' [DevSecOps Insights 2020 ':谁真正拥有 DevOps 中的安全性](/devsecops-insights-2020-who-really-owns-security-in-devops/)”，文章总结了该公司进行的一项调查的结果[，涵盖安全性、开发和运营。该帖子包括几个令人惊讶的调查结果，即只有 14%的受访者报告说他们测试了容器映像中的已知漏洞，38%的受访者没有将自动安全扫描集成到他们的 DevOps 管道中。](https://snyk.io/wp-content/uploads/dso_2020.pdf)

正如 Tal 在文章中写道:

*当许多受访者都认为安全是快速交付软件时的一个主要问题时，这意味着我们需要提升安全性，以便快速交付安全补丁。这样做的关键在于开发人员，因为他们最终会修复应用程序源代码中的安全问题。*

我们还得到了 Tal 对将安全性纳入持续集成/持续交付(CI/CD)的看法，对开发速度的需求，以及他对 GitHub 最近收购 npm 的想法。
