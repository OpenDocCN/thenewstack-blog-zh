# Kubernetes 和云原生社区

> 原文：<https://thenewstack.io/kubernetes-and-the-cloud-native-community/>

疫情已经显著加快了对 [Kubernetes](https://thenewstack.io/what-are-cloud-native-patterns-and-how-should-you-use-them/) 和云原生环境的采用，以此来适应远程工作人员的激增和其他基础设施的限制。然而，随着疫情的开始，组织保留了对已经部署了云原生基础架构的组织的投资。他们已经意识到[云本地](https://thenewstack.io/category/cloud-native/)非常值得维持他们的投资。与此同时，Kubernetes 的采用继续保持上升趋势。然而，不用说，挑战依然存在。在这种情况下，我们来看一下与一年前相比，云原生采用的状况，尤其是此时的 Kubernetes。

在这一集的 [新栈制作者播客](/tag/the-new-stack-makers) 中，记录在[kube con+CloudNativeCon Europe 2022](https://events.linuxfoundation.org/kubecon-cloudnativecon-europe/)的地板上，我们讨论了这些主题以及 Kubernetes 的状态和社区 与[James lave rack](https://twitter.com/JamesLaverack?ref_src=twsrc%5Egoogle%7Ctwcamp%5Eserp%7Ctwgr%5Eauthor)、员工解决方案工程师 以及[Christoph Blecker](https://www.linkedin.com/in/cblecker/?originalSubdomain=ca)，现场可靠性工程师， [红帽](https://www.redhat.com/fr) ，Kubernetes 指导委员会成员。

Kubernetes 的广泛采用在很大程度上归功于它如何帮助解决扩展挑战，但随之而来的是复杂性。“Kubernetes 旨在成为一个可扩展的、弹性的容器编排平台，它做得非常好，但在云数据社区中，还有一个围绕它建立的整个生态系统，用于其他组件，这些组件在他们所做的方面是最好的，”Blecker 说。

Blecker 解释说:“整个云原生环境源自不同的项目和不同的组件，这些项目和组件共同导致了一个重要的挑战，那就是要弄清楚云原生生态系统中的哪些组件和哪些拼图是真正支持您的业务目标所需要的。”。

在许多方面，Kubernetes 的最终成功——甚至可能失败——取决于社区。例如，“试图理解我们可以做些什么来发展社区”是非常有价值的。“我们真正能做的是帮助贡献者为 Kubernetes 做出贡献，并将其打造为最好的项目。”
