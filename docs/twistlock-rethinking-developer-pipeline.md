# Twistlock 重新思考了保护超大规模应用的开发者管道

> 原文：<https://thenewstack.io/twistlock-rethinking-developer-pipeline/>

[Twistlock 关于重新思考开发者管道](https://thenewstack.simplecast.com/episodes/twistlock-on-rethinking-the-developer-pipeline)

在定义应用程序现代发展道路的事件链中(这条道路现在包括微服务、持久容器、编排器、监视器和“kubelets”)，安全部分何时开始？近年来，我们一直在谈论“将安全性融入”应用程序。但是现在，根据定义，超大规模应用程序正在成为相关功能的集合，如果你愿意的话，不再有单一的“烘焙”过程。

因此，集装箱化安全问题又回到了 2014 年开始的地方，一个安全的系统是否可以补偿不安全的通信。

容器安全平台提供商 [Twistlock](https://www.paloaltonetworks.com/prisma/cloud) 的首席技术官 [John Morello](https://www.twistlock.com/about-us/team/john-morello/) 在为我们即将推出的电子书《Kubernetes 生态系统的状态》所做的采访中，谈到了最新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客。

因为不像[ *虚拟机*的世界，你可能有一天会在那里部署 WordPress，然后你的运营团队会用 [Nessus](https://www.tenable.com/products/nessus-vulnerability-scanner?utm_source=google&utm_medium=cpc&utm_campaign=amer-en-high-velocity&utm_promoter=tenable&gclid=CjwKCAjwt8rMBRBOEiwA2F2biD45Kd2NXGektHilLo2-wLaO_Ev51C-GehV3Vej6J3zS06nYF54swhoCdHUQAvD_BwE) 或 [Qualys](https://www.qualys.com/) 或类似的工具扫描那个环境，找到一些漏洞，然后 SSH 到那个虚拟机并更新那个虚拟机中的组件……在新的容器世界中，你的开发者需要知道这一点，因为你的开发者需要重新创建易受攻击的映像。然后，他们需要部署这些新映像来替换现有的映像。"
