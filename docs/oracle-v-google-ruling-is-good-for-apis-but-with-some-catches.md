# 甲骨文诉谷歌的裁决对 API 有利——但也有一些问题

> 原文：<https://thenewstack.io/oracle-v-google-ruling-is-good-for-apis-but-with-some-catches/>

美国最高法院的裁决支持谷歌作为甲骨文对搜索引擎和云巨头的长期诉讼的被告，为依赖 API 进行工作的开发人员提供了澄清和宽慰。

然而，由于多数法院的裁决留下了 API 仍然可以受版权保护的可能性，所以仍然存在一些模糊之处。最高法院也不太倾向于在未来的裁决中扩大软件代码合法“合理使用”的范围。

自从甲骨文对谷歌提起第一起诉讼，声称谷歌侵犯了甲骨文对 Java API 的版权使用以来，该案件一直在向最高法院发展，此前做出了一系列有利于谷歌或甲骨文的判决。

在这个案例中，大法官 Stephen Breyer 在裁决中代表大多数人明确表示，谷歌为其 Android API 使用 11000 行 [Java API](https://docs.oracle.com/javase/7/docs/api/) 代码并没有侵犯甲骨文的版权。

谷歌“重新实现了一个用户界面，只需要让用户把他们积累的才能投入到一个新的和变革性的程序中去，” [Justice Breyer 写道](https://www.supremecourt.gov/opinions/20pdf/18-956_d18f.pdf)。因此，“从法律角度来看，谷歌对 Sun Java API 的复制是对该材料的合理使用。”

克拉伦斯·托马斯法官在他的反对意见中措辞特别强烈地写道:

“通过抄袭甲骨文的工作，谷歌摧毁了甲骨文的市场，并在超过 25 亿台活跃使用的设备上创建了一个移动操作系统，每年赚取数百亿美元。如果甲骨文潜在市场的这些影响有利于谷歌，那我们的合理使用分析就大错特错了。”

虽然 Breyer 法官确实为 API 相关的版权索赔以及他认为的版权所有权提供了一系列限制，其他四位法官也支持多数派意见，但他也描述了 API 的版权所有权解释如何取决于许多因素。换句话说，谷歌为其 [Android API](https://developer.android.com/reference) 使用 11，000 行 Java API 代码是合法的，但这并不意味着其创作者试图版权保护的任何和所有 API 代码的使用都没有侵犯版权所有权。

根据美国国会授权的版权法，这一原则描述了本案中的大部分适用法律，“版权保护不能延伸到‘任何想法、程序、过程、系统、操作方法、概念、原则或发现(根据法令)…”布雷耶法官写道。

由于许多因素，软件代码所有权声明也很难解释。“‘合理使用’原则是灵活的，考虑到了技术的变化。计算机程序在某种程度上不同于许多其他受版权保护的作品，因为计算机程序总是服务于功能目的，”Breyer 法官写道。由于这些差异，合理使用对计算机程序起着重要的作用，它提供了一种基于上下文的检查，使计算机程序的版权垄断保持在合法范围内

与此同时，在合理使用的法律允许范围内，适用某些限制。不同的限制适用于谷歌对 Java API 代码的使用，范围从谷歌如何使用 API 到在多大程度上依赖甲骨文拥有的代码，到“使用对版权作品的潜在市场或价值的影响”

“法院的结论是，谷歌复制 API 来重新实现用户界面，只是采取了让用户将其积累的才能用于一个新的变革性程序所需的内容，这构成了法律上对该材料的合理使用，”布雷耶法官写道。

然而，考虑到大多数开发者依赖 API 代码的现状，GitHub 的开发者政策负责人 Mike Linksvayer 认为最高法院的裁决没有任何模糊之处，特别是因为谷歌对 Sun API 的使用反映了长期的软件开发实践。

“我认为这项裁决的好处在于它没有触及软件的其他领域。“这是‘无害的’,这是现状:这些软件开发人员编写的 99.99%的内容通常都受到版权的完全保护，”Linksvayer 告诉新堆栈。“声明代码可以创造性地重复使用，你可以相对自信地认为你有一个真正强大的公平使用辩护。你不会去追，因为现在每个人都明白，合理使用(限额)非常强大。”

Linksvayer 说，法院的裁决还考虑到了对开发商和公共福利的影响。“我认为[裁决]非常有力，因为它为开发者和初创公司甚至最大的公司创造了空间，使他们能够以行业一直依赖的方式继续创新，而不会被版权巨魔捕食，”Linksvayer 说。

## 赞成和反对

微软的商业模式很大程度上依赖于专有软件许可收入，它会支持谷歌的法律主张以支持这家科技巨头，这似乎令人惊讶。然而，微软的大部分开发工作和源代码来自开源，包括其 Xbox 游戏机的 Linux 操作系统，这有助于解释其在诉讼中支持谷歌的原因。

微软的律师在法庭之友中写道，支持联邦巡回法院早先做出的有利于甲骨文的裁决将“产生的影响远远超出甲骨文和谷歌在本案中关于 Java 代码的争议。”

“虽然合理使用应该涉及‘逐案分析’……但联邦巡回法院未能采取合理使用的观点，说明功能软件代码在现实世界中的使用，因此对整个计算机行业的创新产生了深远的负面影响，”微软写道。

但是，根据最高法院的反对意见，由托马斯大法官撰写，谷歌使用甲骨文拥有的 API 代码不属于“合理使用”。

多数派的“合理使用分析与国会给予计算机代码的实质性保护完全不一致……多数派忽视了一半的相关法律文本，并扭曲了其合理使用分析，”托马斯大法官写道。“考虑到法律条文，甲骨文的代码是受版权保护的，而谷歌对受版权保护的代码的使用是不公平的。”
