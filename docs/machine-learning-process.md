我们每天都被大量的数据淹没，却没有时间或必要的人力来产生能够产生真正影响的见解。机器学习算法是数据科学家弥合这一差距的关键。整个[机器学习](/category/machine-learning/)生命周期由许多阶段组成，但最关键的是最初的步骤。记下来。

首先，对数据的深刻理解是必要的。问问你自己，我对数据有把握吗？数据是不是太有方向性了？我需要申请什么变压器？有标签还是没有标签？从一开始就了解数据在运行后将如何转变，将有助于推进这一过程。如果没有这种程度的理解，构建一个达到预期结论的算法的几率将不会对你有利。

### 第一步:清洁

这是最关键的一步，90%的时间应该花在这一步。虽然清理确实需要确保多个变量存储在一列中，或者多种类型的观察单位存储在同一个表中，但这也是为了让您的数据为训练做好准备。这是您的基础，在这种情况下，对事先发生的数据的可靠理解将派上用场，并有助于建立一个准确的模型，在现实环境中做出正确的推断。这一步将确保您的数据随着时间的推移而转换和学习。

### 第二步:构建

一旦数据得到清理，接下来就是创建适当的模型，以便机器学习算法能够从数据中做出智能推断，并考虑新出现的数据。建立一个模型很容易，而且有工具和软件可以自动完成这个过程。然而，永远不要忘记你正在构建什么。在任何时候，数据科学家都需要能够解释模型。举个例子，你在金融服务或保险等受监管行业工作。如果你的银行拒绝贷款申请，申请的个人或组织有权回来询问原因。“模型让我做的”这样的答案是不够的。你需要能够解释模型，并向任何询问的人解释数据遵循的决策过程。

### 步骤 3:实施和管理

除了清理数据，机器学习过程的第二个最重要的阶段是操作模型。如何确保该模型适合现有的 30-40 年的基础设施？这个过程非常困难，因为将两种完全不同的技术分离开来绝非易事。有软件可以帮助简化这个过程，但是建议让专业人士来帮你完成这个步骤。

我最近与一家大型金融机构的首席信息官合作，将他们的机器学习算法付诸实施。在与 IBM 合作之前，构建数据需要 3 天时间，运行数据需要 11 个月时间。通过在 z Systems 上实施更加结构化的工具，如[机器学习，该组织能够将其模型的运营时间从几个月缩短到几天。](https://dineshnirmal.wordpress.com/2017/02/20/ibm-machine-learning-for-zos-like-no-other/)

最后，这一切都是为了让你的眼睛盯着奖品。一旦模型被创建，正确的算法就就位了，持续地管理过程将确保成功。如果没有这种维护，算法将无法进化，无法从变化的数据集学习，并且容易受到未被检测到的漏洞或缺陷的影响，这些漏洞或缺陷会导致模型运行非常缓慢，使用太多内存并影响用户体验。
