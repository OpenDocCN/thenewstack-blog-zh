# infoblox:DDI 如何帮助解决网络安全和管理问题

> 原文：<https://thenewstack.io/infoblox-how-ddi-can-help-solve-network-security-and-management-ills/>

网络连接可以比作去游乐园，动态主机配置协议(DHCP)充当进入公园的门票，域名系统(DNS)是公园周围的地图。网络管理和安全提供商 [Infoblox](https://www.infoblox.com/?utm_content=inline-mention) 将这两个核心部分整合到一个平台上，使企业能够控制 IP 地址的分配以及如何管理网络的创建和移动。

Infoblox 产品营销副总裁[安东尼·詹姆斯](https://www.linkedin.com/in/anthony-james-58612b)在最新一期的 New Stack Makers 播客中解释道:“他们控制自己的 DNS，这样他们就可以更好地控制自己的流量。”该播客由 New Stack 的创始人兼发行人[亚历克斯·威廉姆斯](/author/alex/)主持。

[Infoblox:DDI 如何帮助解决网络安全和管理弊病](https://thenewstack.simplecast.com/episodes/infoblox-how-ddi-can-help-solve-network-security-and-management-ills)

Infoblox 对这种统一服务的命名是 DDI，它是 DHCP、DNS 和 IPAM (IP 地址管理——每个获得 IP 地址的设备的存储库)的简写。

“我们对 DDI 的看法是，它是任何人连接到网络，然后从网络外的网络连接到互联网等地方的基本要素，”詹姆斯说。

经常发生的情况是，当 IP 地址分离且不融合时，组织可能会陷入困境。“当(组织)建立一个网络时，通常发生的是人们认为服务是理所当然的，”詹姆斯说。

例如，组织可能会安装和配置一台微软服务器，并添加一个用于身份验证的活动目录，而“只是固有地打开 DNS 和 DHCP，”James 描述道。“这是实施这些服务的常见方式……有什么挑战吗？没有协调，”詹姆斯说。“如果您遇到安全事故，得到通知说某个 IP 地址可能受到了攻击，或者该地址被转到了恶意网站，现在您必须查看所有这些不同的服务、DNS 日志和 DHCP 日志，它们现在位于两个不同的基础架构或两套不同的基础架构上。很难搞清楚这一点。”

这些常见的 IP 地址管理和分析问题在几十年前 Infoblox 创建时首次出现。詹姆斯说，公司创始人试图帮助管理“弄清楚 IP 地址去了哪里以及如何管理 DNS”的复杂性。“然后，你把集装箱化，所以它只是导致了这个真正的大爆炸的问题来处理，”詹姆斯说。“早些时候，随着网络在应用方面变得更加虚拟，从虚拟化开始，我们说，‘看，我们有这个 DDI 平台，它将所有基础架构整合到一个平台中，让我们与这些新兴技术挂钩吧。’"
