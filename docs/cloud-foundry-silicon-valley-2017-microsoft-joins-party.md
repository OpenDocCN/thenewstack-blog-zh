# 云铸造峰会硅谷 2017:微软加入派对

> 原文：<https://thenewstack.io/cloud-foundry-silicon-valley-2017-microsoft-joins-party/>

在硅谷举行的年度用户大会上， [Cloud Foundry Foundation](https://www.cloudfoundry.org/) 引入了一个新成员:微软。

微软作为黄金会员加入了该基金会，正式确立了至少自 2015 年以来一直存在的关系，当时 Cloud Foundry 作为一项服务被引入该公司的 Azure 云平台，据微软称，这项服务的使用量在过去六个月中增长了 300%。

“我们看到如此多的客户对在 Azure 上部署 Cloud Foundry 感到兴奋，”微软合作伙伴总监 [Corey Sanders](https://twitter.com/CoreySandersWA) 在 2017 年硅谷云基金会峰会上说。Azure Cloud 提供了 PaaS 平台的 Pivotal Cloud Foundry 发行版。

“我们从客户那里听到的最重要的事情之一是，他们正在寻找一个可以在企业云上运行的企业平台。关键点是可移植性，或采用该平台并将其部署在多个云中的能力，或以混合方式部署在本地的能力，”桑德斯说。

微软最近增加了新功能，包括 MySQL 和 PostgreSQL 的托管服务。该公司还为其最近推出的 [Azure Cloud Shell](https://azure.microsoft.com/en-us/features/cloud-shell/) 提供了完整的 Cloud Foundry 命令行界面。

Cloud Foundry 是一个为企业开发人员提供运行时服务的开源平台，被许多行业的大型企业使用，包括制造业、电信和金融服务。超过 2，400 名开发人员向代码库提交了超过 51，000 份提交。它的全球用户组联盟现在拥有 70，000 多名成员。

Cloud Foundry 基金会的执行董事 Abby Kearns 在开幕主题演讲中说，Cloud Foundry 是为开发人员“编写更多代码并更快地将这些想法投入生产”而构建的。

该软件由数十家最大的 IT 企业供应商提供支持，包括思科、戴尔、EMC、惠普企业、谷歌、IBM 和 SAP。它也在大多数云平台上提供，包括谷歌云平台、亚马逊 Web 服务、IBM 的 SoftLayer 以及基于 OpenStack 和 VMware vSphere 的平台。

除了宣布微软的参与，该基金会还推出了一项名为 [Cloud Foundry 认证开发者](https://www.cloudfoundry.org/certification/)的培训计划，于 3 月宣布[。](https://thenewstack.io/cloud-foundry-launches-developer-training-certification-program/)

该组织还与由[云本地计算基金会](https://www.cncf.io/)管理的 [Kubernetes](/category/kubernetes/) 开源容器编排引擎建立了更紧密的联系。它已经接受 [Kubo](https://pivotal.io/kubo) 作为一个孵化项目，一个部署和管理 Kubernetes 的工具。Kubo 基于 [Bosh](https://bosh.io/docs) ，一个最初为 [Cloud Foundry 云部署](https://thenewstack.io/bosh-hooks-cloud-foundry-google-cloud-platform-offers-large-scale-vm-lifecycle-management)打造的云应用生命周期管理工具。

[云铸造基金会](https://www.cloudfoundry.org/)和[云本地计算基金会](https://www.cncf.io/)是新堆栈的赞助商。
