# 助记法:大数据的内存管理

> 原文：<https://thenewstack.io/mnemonic-memory-management-big-data/>

[助记符](https://mnemonic.apache.org/)是 Apache Software Foundation 最新的顶级项目之一，旨在解决大数据应用程序遇到的垃圾收集问题。

它提供了一个通用的内存持久性对象模型来解决性能问题，包括长时间暂停或交互式分析的大量资源使用。

Java 虚拟机(JVM)没有应用程序意识——用户知道程序可以在哪里停下来清理垃圾，什么时候关键会话不能，Yanping“Lynn”Wang 解释说，他是来自英特尔的项目的联合创始人。

“我们想给用户更多的灵活性和一个比 JVM 垃圾收集更好的模型，”她说。"如果你有一些关键数据，你不想被 JVM 垃圾收集所困扰，让我们来处理它。"

有了助记模型，他们需要做的就是声明他们的对象是非易失的。

基于 Java 的项目包括统一的框架、持久的对象模型和计算模型、可扩展的优化焦点，以及与 Apache Hadoop 和 Apache Spark 等大数据项目的集成。

## 硬件+软件改进

她和“Gary”Wang 一起研究如何改进大数据的内存管理，最终编写了 6000 行代码。

该软件为 Java 程序员提供了一种通过助记符来定义对象和结构的方法。她说，在测试他们在 Spark 上的工作时，他们能够将暂停时间减少一半，并将速度提高 2 到 3 倍。她说，思科和 Cloudera 等公司已经加入了 ASF 的项目，而英特尔的人工智能和客户端计算小组正在推进这项工作。

与此同时，王艳萍已离开英特尔，将工作重点放在金融交易用例上，在这些用例中，任何停顿都是不可接受的。

她说，存储继续发展，NAND 和 3D XPoint 填补了 DRAM 和硬盘驱动器之间的速度和容量差距。

“现在我们有很多非易失性存储器。在过去，有一个内存磁盘和缓存，这留下了速度差距。然后，软件开发了许多缓存算法——如果他们不进行缓存，他们想要使用的一些数据就会被交换回磁盘。那么它将需要非常长的时间来访问。现在，硬件开发人员开始使用快速内存来填补缓存和磁盘之间的空白，因此有大量内存可用。她说:“到目前为止，Spark、Hadoop 和其他软件方面的内存并没有得到有效利用。

## 就地计算

助记术是一个使用下一代 NVM 存储介质将内存计算扩展到就地计算的项目。

对于大数据计算，它将海量数据对象及其模式保存在存储上，不需要序列化/反序列化对象。

“就地计算意味着在有数据的地方进行计算，”她解释道。“你不需要移动数据。我们使用 CPU——它可以是卡上的 GPU——内存几乎可以在任何地方都有处理单元。以数据为例，它可以进行排序、分区，当一些关键工作正在进行时，处理是在数据库本身中完成的，非常接近处理单元。在过去，计算机做不到这一点，因为它太贵了，内存离缓存太远了。这也是基于硬件创新。”

实际上，记忆术是一个介于软件和硬件之间的项目，她说。

“优化是为了确保硬件的最佳利用，同时软件具有灵活性和快速使用模式。软件可以无缝运行，但本质上…我们确保硬件工作最佳，但也让软件快乐，”她说。

助记键提供了一种通过就地对象数据更新直接与本机代码通信的机制，以避免复杂的对象数据类型转换和堆栈封送。

使用 Apache 助记符，其他计算语言(如 C/C++)也可以直接访问对象。持久对象模型和持久计算模型还可能为高性能应用程序和框架带来新的无缓存和无串行化器(无串行化器和无去串行化器)架构。

她说，该项目正在继续开发该库，并致力于纯 Java 内存服务、持久对象矢量化和持久查询服务功能。

Apache 助记符为内存管理提供了一个统一的界面，”Apache 助记符提交者赵说随着大容量 nvm 的发展，它在重塑当前计算机体系结构中的存储器管理方面发挥着重要作用，以最小的成本实现了从当前基于机械的存储到基于闪存的存储的平稳过渡。"
