# 多克，这只是开始

> 原文：<https://thenewstack.io/the-new-stack-analysts-show-23-docker-its-just-the-beginning/>

当我听到 OpenStack 上个月在巴黎的热身活动中录制的这段录音时，我一直在想这个对 Docker 来说很重要的一周。

[#23: Docker，这只是个开始](https://thenewstack.simplecast.com/episodes/23-docker-its-just-the-beginning)

就像 OpenStack 一样，本周的 DockerCon 是一项社区活动。OpenStack 和 DockerCon 都提醒人们，在新兴技术市场，混乱是司空见惯的，人们将努力转移思维。我们几乎在任何技术社区都能看到这一点。容器社区也不例外。

但 Docker 的意义不亚于过去 20 年的任何技术运动。而且不仅仅是因为 Docker 技术本身。它更多的是关于容器移动的更大意义，以及它将如何影响新堆栈的下一次发展。

就重要性而言，我想到了惠普新任云计算主管 Marten Mickos 在热身时所说的话。他将原来的[灯组](https://en.wikipedia.org/wiki/LAMP_%28software_bundle%29)比作集装箱运动。在 LAMP 堆栈上运行数据库是一回事，但是当需要运行 1，000 个数据库时，真正的挑战就出现了。今天，运行 1000 个容器可能看起来是一个壮举，但很快就会有数百万和数十亿个容器跨云服务运行。瓶颈、网络问题以及规模带来的所有其他问题尚未解决。

这段录音提供了一些见解，这些见解来自一些人，比如来自 ActiveState 的 Bernard Golden，他与 Marten 讨论了 Docker 隐喻。Docker 的 Erica Windisch 和 Rackspace 的 Adrian Otto[谈论 Docker 在 OpenStack 中扮演的角色。录音的最后是云软的邓肯·约翰逊·瓦特对 Clocker 的概述，以及与英特尔](http://rackspace.com)[的比利·考克斯的讨论。我们也听到像 Ben Kepes 这样的人提出的问题，他向 Marten 提出了一个关于 Docker 在 OpenStack 中将扮演的角色的好问题。有关于 Docker 如何在 Docker 中运行以配合 OpenStack Nova 的讨论。](http://www.intel.com/content/www/us/en/software/data-center-software.html)

总的来说，这些讨论举例说明了容器与虚拟机相比的不同之处。它再次显示了对于这项新技术将如何应用于工作负载的困惑。集装箱是在上升，但现在有竞争派系，这是可以预料的。这在任何充满激情的社区都是意料之中的。

Rackspace、英特尔、云软和 ActiveState 是新堆栈的赞助商。
