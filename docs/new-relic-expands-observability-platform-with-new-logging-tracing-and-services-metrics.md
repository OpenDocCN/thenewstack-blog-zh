# 新遗迹通过新的日志记录、追踪和指标扩展了可观测性平台

> 原文：<https://thenewstack.io/new-relic-expands-observability-platform-with-new-logging-tracing-and-services-metrics/>

[新遗迹](https://newrelic.com/)赞助了这个故事。

IT 系统监控服务提供商 New Relic 正在扩展其旗舰产品 New Relic One 可观测性平台，以包括日志记录、跟踪和指标方面的新功能。

此外，该公司还为最终用户提供了在 New Relic One 上构建自己的应用程序的能力。这可以帮助组织使用可观察性数据为其他业务部门提供洞察力。该编程框架使用 React 和 GraphQL，并附带用于常见功能的预建模板。

该公司周四在纽约市举行的 [FutureStack](https://newrelic.com/futurestack) 用户大会上展示了这些新功能。

借助新的遗留日志，运营团队可以从一个可查询的视图中将日志数据与应用程序和基础架构数据相关联。那些已经在使用第三方日志管理工具的人可以摆脱它们，转而使用 New Relic 的集成功能。

同样，New Relic Traces 和 New Relic Metrics 允许用户接收来自第三方传感器的数据，将数据关联起来以建立跨环境的依赖关系，这对于调试分布式系统中难以捉摸的性能问题非常理想。

此外，New Relic 正在部署人工智能能力，New Relic AI 可以跨多种工具进行一些初步的关联、聚合和优先排序事件数据的工作。该功能对待命团队来说是一个真正的帮助，可以帮助他们更快地主动检测和解决事件。该服务还集成了 PagerDuty、Slack 和事故管理工作流程中常用的其他工具。
