# 调查发现，低代码工具改善开发人员的工作生活平衡

> 原文：<https://thenewstack.io/low-code-tools-improve-devs-work-life-balance-survey-finds/>

根据 10 月下旬发布的一项研究，头条新闻可能充斥着主要科技公司的裁员，但许多开发人员仍然坐立不安，[。](https://www.outsystems.com/1/developer-engagement-report/)

虽然 64%的被调查开发者——将近三分之二——说他们热爱他们所做的事情，但不到一半的开发者(48%)预计一年后他们会继续留在目前的工作岗位上；两年后，只有 28%的人说他们会留在原地。

不安的一个重要原因是，该研究的发现表明需要[更好的工作生活平衡。](https://thenewstack.io/the-future-of-flexible-work-in-the-tech-industry/)半数受访者表示，他们需要更好地平衡个人生活和职业生活。

根据提供低代码软件开发平台的 Outsystems 的调查，低代码和无代码工具似乎正在帮助解决这一挑战。在接受调查的报告在工作中使用低代码工具的开发人员中，71%的人说他们能够坚持每周工作 40 小时。相比之下，只有 44%的组织不使用低代码的开发人员这么说。

此外，大多数使用低代码的开发人员(63%)表示他们对自己的工资和福利很满意，相比之下，传统开发人员中只有 40%的人表示满意。

该报告的研究由埃文斯数据公司管理，调查了全球各行业的 860 名开发人员。

## 工作安全和低代码

这项研究也颠覆了传统观点，即[低代码工具会让熟练的软件开发人员变得多余:](https://thenewstack.io/why-businesses-want-to-enable-no-code-and-low-code-automation/)

*   51%接受调查的低代码开发人员表示，他们对自己的工作安全感感到满意，相比之下，传统开发人员的这一比例为 39%。
*   对于焦虑的 IT 招聘经理来说，调查提供了一些证据，表明并非所有人都渴望跳槽。46%的受访者同意“我认识的许多换了工作的人发现他们的新工作并没有好到哪里去。”
*   43%的人说经济环境会影响他们是否要换工作的决定。
