# UX 进步联盟在设计师和开发商之间架起了一座桥梁

> 原文：<https://thenewstack.io/progress-unite-ux-bridges-the-gap-between-designers-developers/>

为了弥合用户界面(UX)设计者和开发者之间的差距， [Progress Software](https://www.progress.com/) 宣布了 Unite UX 的预览版，这是一款将设计转化为前端代码的工具。

它是在与客户就他们的痛点进行深入交谈后开发的。据 Progress 客户成功高级总监[丹·莱维](https://github.com/danl3v)说，这两个职能部门之间缺乏合作的问题不断出现。

设计工具的输出范围可以从屏幕截图到开发人员必须转换为工作代码的代码片段，并且经常导致设计人员和开发人员之间的多次往返，结果经常与预期的设计不匹配。

“我们的方法是集成到现有的设计工具中，”Levy 说，他是开发者工具部门客户成功项目的负责人。

“我们不想创造另一种设计工具。设计师喜欢他们使用的工具。他们喜欢素描，他们喜欢视觉。Adobe XD 开始获得更多的追随者。我们想接入这些工具。幸运的是，这些产品的设计考虑到了这一点。”

它为 Sketch 和 Adobe XD 提供剑道 UI 库。Kendo 是 Progress 针对 JavaScript 的 UI 组件——Angular、React、Vue 等等。

另一部分是一个插件，允许他们导出屏幕，不是像他们通常做的那样作为一个图像，而是一个“元桥”格式。这是一个 JSON 文件，具有一个文件夹结构，包含导出的附带文件。

“我们喜欢称之为 Metabridge，它是真理的唯一来源，”他说。

然后，该文件被读入 UniteUX Studio，后者将其翻译成应用程序的完整编码前端。

“目前，我们正在以有角度的方式进行这项工作。我希望在我们发布的时候，React 也能得到支持。我们将 Unite UX 设计成框架不可知的，这样我们就可以添加更多的框架引擎。如果 Blazor 成为下一个大事件，或者我们对 Vue.js 引擎有足够的需求，我们就可以内置它，”他说。

现在，用户可以在这里注册一个私人预览。该产品预计将于今年晚些时候推出。没有公开测试的计划。

Levy 说:“这对于处理多个应用程序并寻求 UX 一致性的客户来说非常好。”他补充说，它解决的其他棘手问题是跨应用程序的一致性和可重用性。

“现在，如果一个企业正在开发五六个应用程序，这些应用程序通常是使用不同的技术体系构建的。但许多大型组织都希望在他们的应用程序中实现一致性，”他说。

“在消费类应用中，通常是一个应用包含不同的部分。无论使用什么工具，您都希望这些部分保持一致。使用 UniteUX，屏幕可以设计一次，然后通过输出引擎在各种不同的技术上达到像素完美。”

Metabridge 也是一个反馈回路，开发者可以在其中向设计者提出修改建议。该公司仍在研究如何将消息传递整合到该工具中，并寻找 Slack 等选项来提供更好的协作。

上周在奥兰多举行的 ProgressNEXT 2019 大会上，该公司还推出了最新版本的网络内容管理产品 [Sitefinity 12](https://www.progress.com/sitefinity-cms) 。托管的托管服务是云原生的，并与 Microsoft Azure 及其生命周期管理工具深度集成，以实现 CI/CD 流程的可见性。
