# 深度嵌套的代码、错误和分布式系统

> 原文：<https://thenewstack.io/deeply-nested-code-bugs-distributed-systems/>

[深度嵌套代码、漏洞和分布式系统](https://thenewstack.simplecast.com/episodes/deeply-nested-code-bugs-and-distributed-systems)

默认情况下，软件的本质是深度嵌套的，这使得寻找 bug 总是一项取证工作。在分布式系统中，可能只有几行代码，但可能有一长串的依赖关系。这是一个依赖链，任何地方都可能存在漏洞。

许多以前不需要解决的问题浮出了水面。“可能涉及到几十个服务 bug 可能在任何地方,[木偶](https://puppet.com/)的 [Gareth Rushgrove](https://twitter.com/garethr) 说，他在上个月的软件马戏活动中为最新版的[新堆栈制造商](https://thenewstack.io/podcasts/makers)播客与我们交谈。

连接不同的组件意味着所有这些部分必须一起工作。这就是为什么 [OpenAPI](https://readme.io/?adgroupID=38998089322&keyword=openapi&ifmobile=&gclid=EAIaIQobChMIuOH79NTX1gIVgWp-Ch3glAXSEAAYASAAEgIeovD_BwE) 和 [gRPC](https://thenewstack.io/grpc-lean-mean-communication-protocol-microservices/) 对于开发人员和工程师来说非常重要，他们正在构建这些所谓的“[脆弱系统](https://thenewstack.io/monitoring-microservices-self-healing-and-the-connection-to-antifragile-systems/)”，这些系统可以跨特定版本的 API 或多个版本的平台(如 Kubernetes)工作。所有这些松散耦合的部分都必须工作。如果要使用它们，这只是一个要求。
