# DevOps 数据收集多少就够了？

> 原文：<https://thenewstack.io/much-devops-data-collection-enough/>

[收集多少 DevOps 数据就够了？](https://thenewstack.simplecast.com/episodes/how-much-devops-data-collection-is-enough)

随着机器学习的不断发展，组织可能不再需要从服务器上收集数据的绝对策略，就性能而言，首席技术倡导者[和](https://www.linkedin.com/in/andimann) [Splunk](https://www.splunk.com/) 的 Andi Mann 指出，Splunk 提供了一个当今许多 [DevOps](/category/devops/) 专业人士正在使用的数据收集平台，在最新一期的[新堆栈制造商](https://thenewstack.io/podcasts/makers)播客中发表了讲话。

云迁移咨询公司 [nClouds](https://www.nclouds.com/) 首席执行官[JT·吉里](https://www.linkedin.com/in/jatindergiri/)参加了[2017](https://thenewstack.io/podcasts/makers)page duty 峰会。Giri 负责组织迁移到 Amazon Web Services，并将 AWS 的原生功能与他们在混合云部署中使用开源 [Kubernetes](/category/kubernetes/) 容器编排引擎所能实现的功能进行比较。这两个世界越来越相互融合，两个平台之间的差异也越来越不明显。
