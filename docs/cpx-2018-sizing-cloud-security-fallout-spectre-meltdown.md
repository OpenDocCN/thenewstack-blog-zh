# CPX 2018:评估来自 Spectre 和 Meltdown 的云安全影响

> 原文：<https://thenewstack.io/cpx-2018-sizing-cloud-security-fallout-spectre-meltdown/>

作为 Check Point 的一名经理，我花了几年时间构建网络网关和远程访问产品。当时，游戏的名称是企业网络的安全。云计算还处于起步阶段。微软 Azure 甚至还没有推出。自从我在 Check Point 任职以来，信息安全领域发生了翻天覆地的变化。

今天，网络安全的挑战已经变得越来越复杂，并且已经超越了 Check Point 首创的企业防火墙。公司必须保护移动设备上的活动，企业网络用户依赖云服务，云提供商自己也必须担心网络安全。

因此，确保云安全的挑战成为本月即将举行的[Check Point Experience](https://www.checkpoint.com/cpx/)(CPX)会议的重要议题也就不足为奇了。

多年来，CPX 已经发展成为设定网络安全市场基调的重大事件。我期待着作为 [Alcide](https://www.alcide.io/) 的代表参加会议并参展，Alcide 是我帮助创立的混合云和数据中心的安全提供商。以下是我期望在会议上突出的一些问题。

在网络安全领域，CPX 的时机再敏感不过了。

本月早些时候，我们了解了利用 CPU 架构的 Spectre 和 Meltdown，这几乎影响了近年来制造的每一款芯片。这些缺陷使得恶意行为者能够对计算机的整个内存进行未经授权的访问。缓解这些漏洞需要来自主要 CPU、操作系统和云供应商的插件。更复杂的是，已经推出的软件补丁对性能有负面影响。

无论从哪个角度看，这种处理器级的漏洞都有可能导致安全漏洞。移动设备受到 it 的影响，笔记本电脑、云客户和云基础设施提供商也是如此。

人们现在问的第一个问题是，针对该漏洞的安全补丁如何影响性能——来自各种云和服务提供商的大量报告表明，软件补丁对性能的影响非常大。

对我来说，这场危机凸显了部署提供检测和执行的安全产品的重要性——不仅在外围，而且在运行工作负载的云运营内部。我很想知道 Check Point 和整个安全社区是如何应对这种新的安全漏洞的。

每个人都会关注。

## 云中的安全性

CPX 的许多会议将关注公司越来越接受云计算所带来的安全挑战。向云的迁移迫使公司和云服务提供商努力应对可能影响双方的潜在安全漏洞。

一方面，云服务提供商必须确保他们的基础设施足够安全，不会受到攻击，并且租户不会受到影响，也不会利用基础设施来攻击邻近的租户，这也称为“旁道攻击”另一方面，云客户必须修补运行在云基础设施之上的系统和软件。

云中的安全漏洞是一个问题，影响着每一个试图实现现代化和跟上时代的企业。我相信，在 CPX 会议上，一个主要的讨论话题将是公司如何在不同的云计算环境中管理日常安全挑战。安全管理是 Check Point 的强项之一。因此，了解他们对这些实践如何向云发展，以及他们如何将其扩展到移动设备和用户的看法将会很有趣。

## 即将出现的其他话题

我预计欧盟新的[通用数据保护条例](https://www.eugdpr.org/)(GDPR)——将于今年五月生效——将会引起相当大的轰动。供应商必须确保新产品符合这些规定，就像已经上市的老产品一样。我期待着听取信息安全专业人员关于他们如何为这些变化做好准备，以及他们如何计划确保他们的客户的需求得到照顾。

回顾过去，我在检查站度过了几年美好的时光。那里的工作开始了从构建防火墙到使用分布式系统再到云系统的专业发展。现在，CPX 将是一个与 Check Point 的网络安全专家和同事重新联系的好机会，并评估网络安全的来源以及未来将如何继续发展。
