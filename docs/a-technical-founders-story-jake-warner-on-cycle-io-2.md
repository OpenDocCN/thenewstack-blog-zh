我们喜欢问技术专家关于他们的第一台计算机或者他们什么时候开始编程的。我们总能找到与工程师今天所做的事情的联系。您将在系列中听到我们提出这类问题，以便更深入地了解工程师负责整个组织时所发生的一切。我们已经听取了关于人们想从这个系列中得到什么的反馈。这里是我们收到的一些对我的推特的回复，询问对新系列的反馈。

我们采访了 Cycle.io 创始人 Jake Warner 的第一集，讲述了他如何从 10 岁时在继承的 Windows 95 机器上下载病毒到领导一家初创公司。

“你知道，我不得不向我爸爸道歉，因为我需要重新安装家里的电脑，”华纳说。“但事实是，有人仅仅通过使用一个文件就可以造成如此大的破坏，这让我开始想，哇，这比我想象的要多得多。”

华纳从来都不是一个游戏玩家。比起玩星际争霸[，他更喜欢聊天室和交谈，他更喜欢谈论这个游戏。华纳在那些聊天室遇到了更喜欢谈论游戏而不是玩游戏的人。他和一个喜欢在星际争霸主办的网络上玩游戏的团体成了朋友。孩子们一直玩的游戏。例如，他们正在学习防火墙，以便在聊天室之间相互攻击。](https://starcraft2.com/)

“正因为如此，这让我对各种防火墙和安全事物感兴趣，这导致了我对编程的兴趣，”Warner 说。“所以，我想，回到你的问题上来，它始于一场游戏，但很快就超越了它。

现在，华纳正在领导他和他的同事们从零开始建立的 Cycle。在很长一段时间里，他们将 Cycle 作为一个容器编排器进行营销。现在他们称 Cycle 为搭建平台的平台——讽刺的是类似于一个孩子在游戏中玩游戏的故事。

一段时间以来，华纳一直领导着一家他称之为容器编制者的公司。有一个企业工程师非常熟悉的 orchestrator。那是库伯内特斯。Warner 和他的团队意识到 Cycle 不同于容器编排器。那么如何改变信息呢？

知道做什么是任何一个创始人的挑战。这是我们将在技术创始人系列中探索的一个重要方面。我们希望你喜欢这些采访。请提供反馈和您的问题。它们总是无价的，是从我们采访的创始人那里获得深思熟虑的观点的一种方式。
