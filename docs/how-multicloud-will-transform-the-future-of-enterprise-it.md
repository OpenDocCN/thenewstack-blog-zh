# 多重云将如何改变企业 IT 的未来

> 原文：<https://thenewstack.io/how-multicloud-will-transform-the-future-of-enterprise-it/>

如今，与 It 相关的对话很难不以“云”这个词开始和结束。事实上，大多数最新研究表明，企业正在加速向云迁移，包括私有云和公共云。

但是云是最终的结局吗？

## 当云不代表云

首先要注意的是，关于云的讨论很少是关于单个实体的。在占主导地位的公共云领域，有 AWS、Azure、谷歌、甲骨文，如果你考虑全球参与者，名单会变得更长。

随着广泛采用的增长，很明显，随着地区指令和政策(如 GDPR)的确立，市场将会分裂。这将使云提供商争夺客户，并带来差异化的产品。

对于中等规模的企业来说，单个云似乎不太可能成为所有工作负载的绝对选择，无论是业务应用程序、数据库需求还是简单的前端零售系统支持。事实上，即使对少数企业进行抽样调查也会发现，即使在早期，大多数企业都在追求多云战略，不管他们是否知道这一点。

当想到云时，大多数人都会想到巨大的集中式资源池。虽然确实有非常大的数据中心支撑着大多数公共云，但这一愿景并非如此。

云是由云提供商管理(至少部分管理)的多个资源池的总称。这些资源位于遍布全球的数据中心。这一点非常重要，因为就工作负载而言，边缘的邻近性至关重要。

对于延迟敏感型应用程序，工作负载可能需要在更靠近用户或数据的地方运行。例如，如果用户在纽约，那么在丹佛托管工作负载是次优的。而且类似于电信和有线电视提供商，跨云也会有地域差异。正因为如此，对于分布式企业来说，所有迹象都指向多云。

在地理位置之外，随着多接入边缘计算等新技术的出现，工作负载可能会在边缘设备上以最佳方式运行。边缘设备将扩展云终结的概念，允许更丰富的选项来满足企业的物理或经济需求。

## 当过渡并不意味着放弃

该行业往往会严重高估转型的速度。迁移是复杂的，就云而言，对于大多数公司来说，可能需要很多年。

这意味着，即使一家企业全身心投入到一家云提供商身上，它也可能在至少几年内横跨新旧两个领域。即便如此，未来是多云的。

## Multicloud 和 cloud 有区别吗？

如果未来是多云的，那么考虑一下这是否会给 it 带来额外的需求是很有用的。

流程将如何跨越不同的云环境？团队是按云还是按工作量组织的？建筑策划是怎么做的？这样的例子不胜枚举，在通常各自为政的地方，多云需要团队之间更多的协调。这可能会对企业 IT 组织结构和运营模式产生深远的影响。

通常用途狭窄的工具必须在多个云环境中开放，否则企业将面临不均衡的可见性、运营性能或安全状况。这就对接口、遥测和与其他系统的集成提出了额外的要求。

甚至底层设备也发挥了作用。如果未来需要多云，那当然需要多供应商，这意味着比一些企业习惯的更加依赖开放标准。但是，如果糟糕的规划导致互操作性问题，成本可能会随着不同环境的数量而线性增长，这使得多云从一开始就是一个昂贵且注定失败的实验。

然而，衡量成功的最终标准将是基础设施的不可见性。终端用户应该不知道他们的应用和服务运行在什么基础设施上。有趣的是，这意味着执行得越好，对用户的影响就越小。有了它，情况似乎总是如此。
