# SoftlLayer 首席技术官:大数据和容器推动了人们对裸机的兴趣

> 原文：<https://thenewstack.io/softllayer-cto-big-data-containers-driving-interest-in-bare-metal/>

SoftLayer 今天宣布了其裸机服务的一些不错的新功能，为我们提供了一个与首席技术官马克·琼斯谈论裸机领域新内容的机会。

首先，新闻。用户现在可以订购裸机服务器，这些服务器将在 30 分钟内部署完毕，并按小时计费。此外，SoftLayer 还构建了包括每服务器处理器、内核、内核速度、RAM 和存储在内的四种配置，供用户选择。用户仍然可以构建自己的定制服务器，但 SoftLayer 是基于通用配置来设计这些服务器的。它计划在未来增加额外的标准配置。

琼斯说，在过去，客户可能需要等待两到四个小时才能让他们的服务器启动并运行。

SoftLayer 以早期提供裸机服务器而闻名。乍一看，该产品似乎违背了云的概念，因为它是单租户的，并且不使用虚拟化。但琼斯说，有几个原因可以解释为什么云推动了人们对裸机的持续兴趣。

他说，SoftLayer 裸机产品的核心用户群是以互联网为中心的中小企业，他们诞生于网络，他们的成功取决于他们的网络存在。“随着他们的成长，他们已经充分利用了云，无论是与我们还是与竞争对手，他们已经到了通常会遇到性能问题的地步，”他说。

答案是不断添加新的实例。不过，他说，这可能会导致实例管理方面的问题。“对于大型大数据部署或密集型工作负载，就实例数量而言，您最终可能会占用大量空间。我们已经看到许多客户将这些类型的工作负载、I/O 密集型工作负载从虚拟化公共云转移到裸机，”他说。

此外，云提供商提供的实例类型可能并不完全符合用户的需求。他说，在公共云中，“你必须调整你的工作负载或应用程序，以满足你的需求”。借助像 SoftLayer 这样的裸机产品，用户可以组合出最能满足其工作负载需求的产品包。

目前，SoftLayer 看到了希望将裸机服务器用于大数据应用程序的企业的大量需求，因为裸机可以满足许多大数据应用程序的高性能需求。

将来，一旦更多的企业将集装箱化技术用于生产，SoftLayer 可能会看到更多的需求。就目前而言，琼斯认为这还为时尚早。“我们已经与客户进行了讨论，IBM 内部也有团队在研究容器，”他说。“很多人会从简单地学习和使用公共虚拟化云中的容器开始，但当你开始谈论生产和规模时，他们会开始接受裸机。”

在虚拟化服务器上运行容器不是很高效。在裸机上运行像围绕 Docker 构建的 CoreOS 这样的操作系统更有意义。“没有虚拟机管理程序，所以你会获得更好的整体系统性能，”他说。

SoftLayer 并不是唯一提供裸机服务器的公司。 [Rackspace](https://thenewstack.io/rackspace-onmetal-a-bare-metal-service-suited-for-coreos-and-docker/ "The New Stack") ，Internap [和其他](http://www.networkworld.com/article/2188187/cloud-computing/new-bare-metal-cloud-offerings-emerging.html "Network World")提供裸机服务器。然而，包括亚马逊网络服务、Azure 和谷歌计算在内的大公司却没有。琼斯认为 SoftLayer 在提供裸机和虚拟化云服务方面都有优势。这使得企业可以在适当的时候使用裸机，同时在需要时轻松扩展到虚拟化云。
