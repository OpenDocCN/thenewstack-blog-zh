# 欧洲最大的电信公司之一的云原生之旅

> 原文：<https://thenewstack.io/one-of-europes-largest-telcos-cloud-native-journey/>

巴伦西亚——电信不一定与采用新一代技术有关。然而，德国电信通过创建和支持 Kubernetes 集群来支持其运营基础设施，在云原生环境方面进行了大量投资。

在这一集的 [新栈制造者播客](/tag/the-new-stack-makers) 中，录于地板上的[kube con+CloudNativeCon Europe 2022](https://events.linuxfoundation.org/kubecon-cloudnativecon-europe/)devo PS 工程师[Christopher Dziomba](https://www.linkedin.com/in/christopher-dziomba-5a9220153/?originalSubdomain=de)T22[Samuel Nitsche](https://www.linkedin.com/in/samuel-nitsche-457ba3158/?locale=en_US)之

[欧洲最大的电信公司之一的云原生之旅](https://thenewstack.simplecast.com/episodes/one-of-europes-largest-telcos-cloud-native-journey)

德国电信显然不是从零开始。它拥有数十年的电信基础设施和网络，这些都需要整合到 Kubernetes 的新世界中。Dziomba 说, 这涉及到很多“与其他团队的讨论”。Dziomba 说:“我们必须(与其他部门)合作，看看我们希望如何管理遗留集成，尤其是政策和流程集成。”。

事实证明，德国电信提供的许多现有服务有助于集成到分布式 Kubernetes 基础设施中。“它适合部署在像 Kubernetes 这样的东西上，”Dziomba 说。“我们还决定在德国电信内部自己建立 Kubernetes 平台，而不是购买一个。这确实促进了向云原生基础架构的迁移。”

尼彻说，这种转变也严重影响了那些“来自老路线”的供应商。“有时，确保应用程序也是真正的云原生应用，并确保它能够利用 Kubernetes 提供的所有优势是一项挑战。
