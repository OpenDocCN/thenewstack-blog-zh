# 使用机器学习优化资源管理以扩展 Kubernetes

> 原文：<https://thenewstack.io/optimizing-resource-management-using-machine-learning-to-scale-kubernetes/>

Kubernetes 擅长大规模系统，但其复杂性和透明性导致了更高的云成本、部署延迟和开发人员沮丧。随着 Kubernetes 的兴起和工作负载继续向容器化环境转移，优化资源变得越来越重要。事实上，最近的 [2021 云原生调查](https://www.cncf.io/reports/cncf-annual-survey-2021/)显示，Kubernetes 已经跨越鸿沟成为主流，96%的组织正在使用或评估该技术。

在本期 New Stack Makers 播客中，[storm forge](https://www.linkedin.com/in/mprovo/)的创始人兼首席执行官 Matt Provo 讨论了思考 Kubernetes 的新途径，包括通过自动化增强开发人员的能力来实现资源优化。他还分享了该公司最新的基于机器学习的多维优化解决方案 Optimize Live。

[Alex Williams](/author/alex/) ，New Stack 的创始人和发行人，主持了这个播客。

[优化资源管理使用机器学习来扩展 Kubernetes](https://thenewstack.simplecast.com/episodes/optimizing-resources-in-kubernetes-environments-with-machine-learning)

Provo 表示，该公司最初从哈佛大学分离出来，在实验室中开始其算法，“以找出如何利用我们的核心机器学习，将其应用于正确的问题集，以及将其产品化，并将其与正确的市场机会联系起来，如容器化工作负载的增长和 Kubernetes 的扩展”。

Provo 说，对于像 StormForge 这样诞生于云计算的公司来说，最先想到的往往是“资源管理和规模效率，尤其是在 Kubernetes 上”。随着机器学习模型大量消耗云资源，StormForge 使用自己的产品来了解和应对其客户也面临的挑战。

Provo 说，随着公司转向集装箱化架构，扩大规模的道路非常具有挑战性。Provo 说:“在我们向 Kubernetes 迁移的过程中，我们的团队发现并遇到了调整迁移到 Kubernetes 的应用程序工作负载的挑战，这是另一个棘手问题。最初专注于预生产，该公司“使用负载或性能测试作为数据输入，因为机器是连接的，并取决于输入模型的数据质量。”Provo 补充说，客户发现了场景规划等领域的价值，以及“随着黑色星期五等事件的到来，将什么部署到生产中”。

StormForge 最近在他们的平台上发布了一个模块，这些客户寻求在同一个平台上进行预生产和生产。“Optimize Live 从生产角度考虑了可观测性和遥测数据。然后，它将此作为数据源，使我们能够提供实时的资源分配建议，以及预测性的建议，”Provo 说。

凭借贝叶斯优化作为公司的知识产权，StormForge 使自己与众不同，“我们是唯一能够做到我们所说的多目标优化的人，”Provo 说。“贝叶斯优化允许我们获得无限数量的潜在参数或指标，以及它们在该应用中如何相互作用。我们不仅可以从静态的角度做到这一点，还可以从持续的角度做到这一点。

Provo 说，随着 DevOps 团队越来越多地参与帮助组织实现其目标，“Kubernetes 环境中的技能差距越来越大，在非 Kubernetes 环境中负责应用程序的人现在经常负责容器化 Kubernetes 环境中的应用程序，但没有得到他们应得和需要的培训和发展。“我们的目标是让开发人员参与到开发过程中，而不是让他们自动退出。我们非常相信开发人员增强的人工智能，允许他们在有意义的地方提供反馈，同时保持控制，”Provo 说。
