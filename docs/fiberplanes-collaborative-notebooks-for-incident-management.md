# Fiberplane 的事故管理协作笔记本

> 原文：<https://thenewstack.io/fiberplanes-collaborative-notebooks-for-incident-management/>

数据库巨头[甲骨文](https://developer.oracle.com/?utm_content=inline-mention)在 2017 年收购 wer cker 时，在其云产品组合[中增加了一个容器原生持续集成/持续交付(CI/CD)平台。自从被收购后，Wercker 的创始人](https://thenewstack.io/oracle-developing-hyperscale-cloud/) [Micha Hernandez van Leuffen](https://de.linkedin.com/in/mhernandezvanleuffen) 创办了 [Fiberplane](https://fiberplane.dev/) ，并担任首席执行官。

在最新一集的 New Stack Makers 播客中，van Leuffen 讨论了 Wercker 开发的不同方面，以及如何将其应用到他在 Fiberplane 的工作中，fiber plane 提供用于解决事件的协作笔记本。 [Alana Anderson](https://www.linkedin.com/in/alanafanderson) ，base case capital 的创始人兼管理合伙人，也从投资资本公司的角度提供了意见。

《新书库》的创始人兼发行人亚历克斯·威廉姆斯[和 TNS 编辑约阿布·杰克逊](https://thenewstack.io/author/alex/)主持了这次播客。

[Fiberplane 的事故管理协同笔记本](https://thenewstack.simplecast.com/episodes/fiberplanes-collaborative-notebooks-for-incident-management)

Wercker 是一个复杂的分布式系统，运行在 T21 的亚马逊网络服务(AWS)上，在 Kubernetes 上有 EC2 实例。“我们遇到的一个问题是，当出现停机或发生事故时，是否很难排除事故的根本原因？”范勒芬说。

“因此，我们在指标、日志和跟踪之间来回穿梭，以找出导致事件的真正原因，我总是称这是一种寻宝活动，就像你所经历的那样。这也是导致 Fiberplane 诞生的原因之一。”

当 DevOps 团队分布在世界各地的不同时区时，寻找事件根本原因的“寻宝”变得更加复杂。

“围绕一个事件进行合作总是很难，因为你需要在同一个时区，或者至少在同一时间醒来，然后一起经历这场寻宝活动，”范·勒芬说。“这是 Fiberplane 的核心。通过这种方式，Fiberplane 成为 DevOps 和 SREs 的协作笔记本平台，帮助他们调试事件并支持事件调查。”

她说，从安德森的角度来看，Fiberplane 具有“笔记本的外观和感觉”,工程师们使用起来很舒服。

“工程师能够查询完成工作所需的所有资源。安德森说:“我认为最重要的是，想象你是一名工程师，你正在进入这个‘作战室’，它实际上是 Fiberplane 的产品。

“你走进去，突然之间你就知道了所有正在发生的事情的历史，你可以看到房间里的其他人……而且，马上，你就能了解正在发生的事情，你也能对你的团队更有价值。”
