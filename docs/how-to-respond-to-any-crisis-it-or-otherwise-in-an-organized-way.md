# 如何有组织地应对任何危机(IT 或其他)

> 原文：<https://thenewstack.io/how-to-respond-to-any-crisis-it-or-otherwise-in-an-organized-way/>

[如何有条不紊地应对任何危机(IT 或其他)](https://thenewstack.simplecast.com/episodes/how-to-respond-to-any-crisis-it-or-otherwise-in-an-organized-way)

欢迎来到 [The New Stack Context](https://thenewstack.io/podcasts/context) ，在这个播客中，我们将讨论来自 [The New Stack](https://thenewstack.io) 网站的最新帖子、播客和赞助商活动。

在本周的节目中，我们采访了[page duty](https://www.pagerduty.com)的安全和事故响应高级工程师[里奇·亚当斯](https://www.linkedin.com/in/richaadams/)。几周前，PagerDuty 公开发布了组织如何建立事件响应流程的文档，包括建立指挥中心和指定事件响应负责人。[指南是基于公司自己处理危机的标准操作程序](https://response.pagerduty.com/)。

“任何人都可以使用我们指南中的信息，无论他们是否是 PagerDuty 客户。该指南是关于组织如何应对事件的——不管他们使用的是什么产品——所以我们专注于事件响应的原则和技术，而不是如何在一个工具内执行特定的操作，”亚当斯在帖子中写道。

[page duty](https://www.pagerduty.com/)是领先的数字运营管理平台，支持企业开发、IT 运营、支持和安全部门将任何信号转化为跨任何运营用例的洞察力和实时行动。

在节目的后半部分，我们采访了 Kong 的联合创始人 Marco Palladino，他谈到了所谓的“软件部落主义”的终结，因为开源将这个行业带入了一个由开发人员而不是软件堆栈供应商驱动的混合世界。

“开源对于生态系统的不同参与者来说扮演着不同的角色，但最好的角色是赋予开发者权力。在一个典型的公司中，开发人员不会做出很多软件决策，但这已经发生了很大的变化，”帕拉迪诺在文章中引用了卡夫卡的联合创始人[内哈·纳克希德](https://www.linkedin.com/in/nehanarkhede/)的话。

TNS 编辑部主任利比·克拉克主持了这一集，同时主持的还有 TNS 创始人亚历克斯·威廉姆斯和 TNS 执行主编约阿布·杰克逊。
