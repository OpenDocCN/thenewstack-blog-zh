本周编程:寻找虚拟会议栈

原文：https://thenewstack.io/this-week-in-programming-in-search-of-the-virtual-conference-stack/#0001-01-01

像你们中的许多人一样，我的第一次科技会议是很久以前的事了，从那以后我参加了一百次，如果不是更多的话。我听了无数的主题演讲，舞台上传来震耳欲聋的音乐，高管们气喘吁吁地宣扬他们的新软件将给世界带来的奇迹，为他们在舞台上现场演示的勇气而拍拍自己的背。

和你们一样，我也曾仔细看过展览大厅，那里的公司在一条线上悬挂着糖果、t恤和其他类似诱饵的东西，寻找电子邮件地址和商业机会，我也曾坐在那里看了太多没有具体目标的展板。然而，任何像你一样的会议老手都知道，你不一定能从会议中获得最多的东西，而是所谓的“走廊通道”——当你在寒冷的会议室之间徘徊时，当你拿着免费的椒盐卷饼或喝下另一杯(我的存在的祸根)会议咖啡时，那些偶然的相遇就会发生。

在我看来，我在会议中心对面的酒店酒吧里聆听或在会后活动中聊天学到的东西，可能比我在公司批准的主题演讲或小组讨论中学到的还要多……当然，也有罕见的真知灼见。

好吧，这些事情现在暂时都不存在了，而且可能还会持续一段时间，就像工作偶然变得遥远一样，整个一季的技术会议不得不迅速转变，要么取消，要么完全虚拟化。上个月，我们见证了第一批虚拟会议，这种新常态肯定有好有坏。

在赞成者一栏中，很明显，你仍然不需要为了一个主题演讲而换下睡衣，而且我们不用成千上万地周游世界，这显然对环境有益。哦，当然，你可以喝自己的咖啡，而不是前面提到的掺水的废话。

从不利的一面来看，显然没有走廊，没有偶遇，没有碰到多年未见的朋友。相反，又有一个闲置频道被添加到您不断增长的闲置频道列表中。正是你想要的，对吗？现在，面板只是一个让别人对你说话的机会，就好像你的生活中还没有足够的那样——我最不想做的事情就是坐下来盯着屏幕再看一个小时或12个小时。

这正是自由职业者DevOps和Docker顾问，以及会议老手， Bret Fisher 本周在Twitter上围绕“完美的虚拟会议技术栈”的想法提出了自己的批评

费希尔思路的一个关键点是，“人们参加会议的目的不仅仅是为了内容，而是为了联系”，我们需要的是“一种视频优先的体验，观众是节目的一部分，你可以在活动中找到你认识的人，并在学习新知识的同时赶上。”对于费舍尔来说，这将通过一系列现有的应用程序来实现，如社交视频聊天应用程序 HouseParty ，游戏玩家喜欢的直播应用程序 Twitch ，以及直播活动应用程序 CrowdCast ，这些应用程序允许互动，而不是更多的“预先录制的单向信息的网络研讨会式视频”

可以肯定的是，我们正处于虚拟会议的初期，从现在开始，每一次会议都将建立在上次会议的成功和失败的基础上。到目前为止，我们已经看到了连续运行的会议，以达到所有可能的时区，以及结合了动物穿越和抽搐的会议。接下来，微软将举办一年一度的 Microsoft Build 开发者活动，它已经做出了一些改变，将解决我们迄今为止看到的一些缺点。根据微软的Scott Hanselman的博客文章，该公司将会持续举办48小时的活动，每8个小时重复一次所有的讨论会，实际上，他们是在说让时区见鬼去吧。“不需要熬夜到凌晨2点，我们会为你做的，”他写道。

Hanselman还表示，他们将提供更短的小组，有兴趣的人可以观看深度视频，并为每个小组提供现场问答。尽管他没有具体说明如何做，但他承认，“有时会议上最好的通道是走廊通道，我们希望你在积极的环境中与志同道合的人相处，这样我们就有办法让你自我组织，进入你自己的空间来分享和学习。”微软将至少使用Fisher建议的虚拟技术会议栈的一部分，在Twitch上提供“长形式的研讨会(1-3小时)”

本周的节目中

  • 说到科技会议… 这里有一个开源科技会议的列表，这些会议已经转移到了网上，这是我这个星期看到的，其中提到了一个“缺点”,我忘了把它包括在上面的列表中——那就是“亲自参加一个会议可能会花费一大笔钱，一旦与会者为一个主要活动的入场券支付1000美元或更多，外加机票和酒店费用。”他们指出这一点与列表中的大多数会议形成对比，这些会议是完全免费的。即使是非免费的活动也只收取象征性的费用。
  • IBM的量子计算:如果玩量子计算机听起来很有趣，IBM已经宣布了其 IBM量子挑战赛，该挑战赛将于5月4日至8日举行，让你“通过使用电路解决量子计算机编程问题——从编写第一个‘Hello Quantum’电路到解决复杂的优化问题。”微软第一个在四年前推出了第一台在线量子计算机，此后已经使18个量子系统上线，并开发了开源量子软件 Qiskit ，这是一个基于Python的软件开发包，具有几个新功能。IBM量子挑战赛于美国东部时间5月4日上午9点开始，四天后结束。检查一下。

  • Twitter引入了一个新冠肺炎API端点:programmable web写道，该公司已经在与新冠肺炎的斗争中引入了一个新的端点,Twitter开发者实验室帮助开发者和研究人员更好地研究围绕新冠肺炎的全球对话端点将包括数百万条关于疫情的推文，该公司表示这些推文将使公众对话的研究成为可能，更不用说那些无数的机器人就这个话题发表意见了。ProgrammableWeb写道，该端点包括“与新冠肺炎和冠状病毒相关的实时、全保真公共推文流”，推文实时发布，同时保护私人数据并删除受保护的推文——而且没有吞吐量限制。为了防止滥用，感兴趣的开发者需要通过 Twitter新冠肺炎数据应用申请访问。
  • 跟进Go和Rust: 上周，我们查看了2019年Rust和Go的调查，考察了这些语言在哪些方面受到了阻碍。作为快速跟进，本周我们将看到对Go创造者Rob Pike的采访以及对Rust core团队成员Steve Klabnik的采访。前者侧重于“Go如何确实成为云基础设施的语言”，而后者则从一个核心成员的角度，而不仅仅是社区调查的角度，来看Rust还缺少什么功能。
