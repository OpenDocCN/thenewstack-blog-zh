麦迪逊·杰西·斯科特-克莱里

Madison 是一个过程专家、Pythonista、Gopher 和不情愿的 JavaScript 程序员。她在开源领域工作，专注于数据可视化和云开发。不工作的时候，她会写作，通常是关于软件开发、心理健康和身份认同的话题。她住在山里，在那里她照顾三只狗:一只两只德国牧羊犬和她的丈夫。

我大学毕业后的第一份工作是在一家保险公司工作，设计报价和销售工具，也正是在那里，我开始找到我的人。毫无疑问，我在大学学习音乐的时光很棒，但我觉得我属于科技。它更多的是关于软件的对话和无指导的交互，而不是我花了几年时间研究的有组织的行动。

在我离开那份工作之前，我非常坚定地认为自己是一名自豪的技术人员，尽管有点多嘴多舌。在那个时候，作为一个工作中的同性恋者出柜是可以想象的顺利:我的老板甚至参加了我的婚礼。

去 Canonical 工作是减轻压力的一种方式，也是更直接关注开源社区的一种方式，我从中获益匪浅。然而，这也是一个获得足够空间来坐下来真正解决许多身份问题的机会，因为这意味着在家工作。

> 关于出柜，特别是转变，这是要记住的第一件事:它们是连续的过程，而不是离散的事件。

我们必须记住，身份认同通常是精神病理学的:它是我们似乎只能感觉到的东西，或者至少当它让我们陷入痛苦时，当它以错误的方式被摩擦时，我们能最强烈地感觉到。我感觉到了围绕性别话题的摩擦，我必须以某种方式解决它。对我来说，这意味着探索过渡领域。毕竟，在我以前的工作中公开自己是同性恋还不错，所以也许是时候看看公开自己是跨性别者会如何工作了，而我有一个相对安全的环境来这样做。

这一切都进行得相当顺利，这是对开源界，尤其是 Canonical 的许多人的证明。我开始改名字，起初只是非正式的。对于我这个年龄的人来说，麦迪逊无疑是非常女性化的，所以这在一开始就是一个致命的泄露，我的老板问我是否在“做任何其他的改变”，我回答说我可能会及时改变。

关于出柜，特别是转变，这是要记住的第一件事:它们是连续的过程，而不是离散的事件。另一方面，尽管在这个过程中有不同的事件发生，但是除了快乐之外没有最终目标。

下一步是在我的名字被合法更改后与人力资源部进行沟通。在那一点上，我们讨论了保险和出差住宿等世俗问题，以及如何告诉团队以及如何应对可能出现的任何问题等更宏大的话题。我们进行了几次长谈，最终决定由我按照自己的进度单独联系团队成员。

要记住的第二件重要的事情是，一个积极参与的支持网络真的可以创造或破坏一次出柜或转型的经历。让理解和参与的队友和 HR 小动物帮助确保它感觉像一个对话，而不是一个指示。也就是说，每个人都有自己的舒适程度，他们参与的程度由他们自己决定。

## 继续说

尽管我有过一次一对一的成就感，但我很快意识到这将是一件持续不断的事情。留长发是一回事，但穿着裙子化着妆跑步又是另一回事。出柜是一件会反复发生的事情，不会总是得到同样随和的回应。

我必须提供的最后一件事是，人们必须记住，尽管所有的工作都是在个人层面上完成的，但过渡本质上是社会性的。不仅仅是医生的笔记，内分泌专家的拜访，或者外科会诊，而是一个发生在社会层面的过程。这发生在朋友、家人和同事身上，甚至是陌生人身上。这也同样适用于出柜。它需要交谈和分享，需要脆弱和坚强。

它在说话，而我是出了名的不守口如瓶。

当然，我没有做过任何实证研究，但我怀疑这是技术迷中的一个明确趋势，尤其是那些嵌入开源的人。也许是因为我们的产品有多少与社会互动有关，或者也许只是因为在制作我们所做的东西时需要多少社会互动。也许这只是对安静书呆子比喻的一种反叛。

有时候，我担心我做得太过了。我担心我以诚实或自我照顾的名义过度分享。这就是我开始写作和早期尝试编程的原因，那时我已经开始了大半个人生，制作公告板、留言簿和博客。撇开强迫症不谈，我对分享的强烈需求推动了我的兴趣，然后是我的教育，然后是我的事业。我从 Perl 转到 PHP，转到 Groovy，然后转到 Python 和 JavaScript，然后继续；从高中到大学再到职场。

我花了很多时间将我的课程转向技术和开源，作为一个带来社交需求的地方，通过生活的方式进行交流，并找到使社交更容易和更广泛可用的方法。我这样做是因为这是对我一生帮助最大的事情之一。

事实证明，要做的是大声说话:继续交谈，继续互动，继续寻找方法缓解围绕身份的摩擦。

说到少数群体成员，技术并不完美，远非如此。即使是自由/开源软件也不是完美的。它们是我们不完美的、社会的——甚至是怪异的——自我的镜子。我们都可以在社区中找到改进的方法，但是我们是在做好事，有好的理由。

继续说。
