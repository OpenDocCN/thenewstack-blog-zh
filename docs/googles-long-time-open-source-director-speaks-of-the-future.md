# 谷歌开源主管谈未来

> 原文：<https://thenewstack.io/googles-long-time-open-source-director-speaks-of-the-future/>

自 2003 年以来，谷歌的开源项目无疑已经走过了漫长的道路。那个时候，搜索引擎巨头仍然可以被称为一家初创公司，Android 还没有被收购，开源项目 Kubernetes、Go 和 Chromium 还在酝酿中。

也是在那时，谷歌联合创始人拉里·佩奇和谢尔盖·布林要求他们最喜欢的招聘人员去找一个“开源人”，该公司的开源总监克里斯·迪博纳(Chris DiBona)回忆道。

“他们说，‘我们现在足够大了，需要一个开源人，’”迪博纳说。

此后不久，迪博纳与招聘人员要求帮助寻找的“一个开源书呆子”共进午餐。

“他说，‘我们想招人，也许是你克里斯，’，我说，‘为什么？“你们是搜索引擎，为什么要关心我这样的人呢？””迪博纳说他说，“不，只是来面试。”我说，‘好吧，好吧。"

在加入谷歌之前，DiBona 已经是一名开源先锋，他将继续监督这家科技巨头的[开源项目](/more-organizations-report-benefits-of-open-source-programs/)，该项目将继续对 IT 行业和开源社区产生重大影响。

在这个新的 Stack Makers 播客中，DiBona 讨论了谷歌的开源政策，以及搜索引擎巨头对其开源未来的计划。

[Alex Williams](/author/alex/) ，New Stack 的创始人和发行人，主持了这个播客。

## 开源哲学的演变

从初创时期开始，谷歌就与开源在社区中扮演的角色同步呈指数级增长，这在很多方面都要归功于开源。

“当我和人们交谈时，他们会说，‘我不知道开源的故事是什么？为什么重要？”迪博纳说。他告诉他们，“很长一段时间，它是一个力量倍增器——对于一个谷歌人写的每一行代码，我有 10 或 20 行来自外部世界，对我们来说至关重要。"

虽然以前认为总是有“一种理想的单一开源方法”，但 DiBona 表示，他在谷歌任职期间，他的开源理念已经发生了变化。

迪博纳说:“我在谷歌的时候已经平静下来，我们没有一种方法——我们有开源、软件发布社区和其他所有方法。”

他补充说，“我们拥有你可能想要、需要或鄙视的所有项目、所有项目风格、所有项目管理风格，因此适应所有产品领域、所有字母实体的业务需求总是不完美的。但这是我们努力的目标。”

## 下一个大问题围绕着 Linux

迪博纳的程序每天发布大约五个项目；在与威廉姆斯的交谈中，他称自己为“官僚”，他的工作是减少发布项目的障碍——保护他的工程师“远离棘手的东西，所以他们只能是工程师。”

展望未来，他认为下一个需要谷歌开源项目——以及更广泛的开源社区——关注的主要领域是 Linux 将如何发展以满足新的需求。

他建议说，在谷歌，“由于我们暴露在这些大规模的问题中，难道我们不应该对 Linux 在未来五年的发展方向有一个预言性的洞察力吗？过去几年我一直在谈论的一件事是，Linux 需要在哪里才能继续在数据中心和移动领域蓬勃发展？”
