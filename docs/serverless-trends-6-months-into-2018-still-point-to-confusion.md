# 2018 年 6 个月的无服务器趋势仍然指向困惑

> 原文：<https://thenewstack.io/serverless-trends-6-months-into-2018-still-point-to-confusion/>

[#164:与 Symphonia.io 和 Serverless Inc.](https://thenewstack.simplecast.com/episodes/164-discussing-serverless-with-symphonia-io-and-serverless-inc) 讨论 Serverless

无服务器计算和功能即服务对企业的吸引力越来越大，但它们如何适应标准的业务环境仍有许多问题。首先，您如何在无服务器环境中管理您的依赖关系？随着时间的推移，如何跟踪和管理应用程序本身？在真实的企业用例中，各种无服务器和功能即服务产品如何处理？

为了找到这些问题以及更多问题的答案，我们与 [Symphonia.io](https://www.symphonia.io/) 的合伙人 [Mike Roberts](https://www.linkedin.com/in/mikebroberts/) 和[server less](https://serverless.com/)Inc .的创始人兼首席执行官 [Austen Collins](https://www.linkedin.com/in/austen-collins-80427a69/) 坐在了一起，他们一起了解了商业应用程序开发人员和管理员如何利用当今的无服务器技术。然而，讨论的一个大话题仍然集中在这些功能将生活在哪里:在云中，还是在 Kubernetes 中？

Kubernetes 现在很受欢迎。虽然我对直接云管理的功能即服务平台最感兴趣，但对运行在 Kubernetes 上的功能即服务平台肯定有很多兴趣。同样，我对托管的 Kubernetes 是如何发展的超级感兴趣。 [EKS](https://aws.amazon.com/eks/) ，亚马逊的 Kubernetes 托管服务在几周前推出，一旦推出完全托管的 [Amazon Fargate](https://aws.amazon.com/fargate/) 支持，甚至不必考虑有 Kubernetes 支持的 it 集群，那么超级有趣的是人们什么时候使用功能即服务，什么时候使用完全托管的容器平台？我真的认为这两者都有很大的应用空间，”罗伯茨说。

回答无服务器计算应用程序的一些问题的一种方法是创建新的标准、规范和项目来支持功能。[Cloud Native Computing Foundation 的无服务器工作组](https://github.com/cncf/wg-serverless)创建了一个名为 [CloudEvents](https://cloudevents.io/) 的新项目，以解决高效、快速、可靠地将传入数据分发到适当功能的问题。CloudEvents 项目提供了描述事件数据的单一方式，允许多个平台支持相同的事件数据流。

Collins 是 CNCF 无服务器工作组的成员，他邀请感兴趣的团体加入该组，为他们正在酝酿的各种项目做出贡献。“我们开始研究标准，尽管我们不能说 CNCF 内有‘标准’……对无服务器架构很重要的不同协调方式。我们的网站针对几个领域:事件——这是无服务器架构中的一个大概念……它比无服务器更重要。事件驱动设计和事件驱动 IT 由来已久。在未来，我们将进入无服务器计算，也就是事件驱动计算，物联网，以及构建可以实时做出决策的智能系统，我们认为事件将是一个比以往任何时候都更大的概念。因此，我们在无服务器工作组中首先致力于此，”柯林斯说。
