# 云本地市场的开源经济现实

> 原文：<https://thenewstack.io/cloud-native-markets-open-source-economic-realities/>

[#159:云原生市场的开源经济现实](https://thenewstack.simplecast.com/episodes/159-the-cloud-native-markets-open-source-economic-realities)

风险投资家和技术专家如何看待云原生技术空间是本期《新堆栈分析师》的主题。

已经到了这样一个地步，服务水平协议几乎不允许访问源代码，除非它是开源的。对于 LightStep 的联合创始人 Ben Sigelman 来说，这意味着开源代码已经成为商业的一部分，现在的问题是将代码发展成所有人都可以使用的标准技术。

Sigleman 与 RedPoint Venture 的 Astasia Myers 和 Scott Raney 一起表示，除非你对开源的作用有一个核心信念，否则没有办法攻击基础设施层市场。红点是云原生计算基金会开发的[云原生景观](https://github.com/cncf/landscape)概述的合著者。

Raney 说，任何构建下一代开放堆栈的最终用户都将使用开源技术进行构建。至于模型，基础设施软件提供商的服务市场正在兴起。使用云服务和与微服务集成的持续路径只会加速，这指向更大的经济现实。

我们不得不询问 Kubernetes，它的核心现在是如何作为容器编排的事实上的标准，以及它在市场中的作用。它指向了 Linux 内核的遗产，并且在某些方面作为其更高级原语的新继承者，这些原语一旦被构建，人们将不知道它们实际上是如何工作的。工作是在抽象中进行的，这将有助于在不了解原语和随之而来的所有复杂性的情况下构建服务。

无服务器领域的体系结构需要大量的扩充，这是应用程序工具链将继续抽象化的一个明显例子。Kubernetes 将容器作为云服务架构的原生单元，这意味着一种全新的发展趋势。反过来，这将意味着一系列新的经济现实。
