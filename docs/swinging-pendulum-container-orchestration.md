# 容器编排的钟摆

> 原文：<https://thenewstack.io/swinging-pendulum-container-orchestration/>

在本期[新堆栈制造商播客](/tag/the-new-stack-makers/)中，我们将深入探讨可插拔架构，以及思科的 [Mantl](https://blogs.cisco.com/tag/mantl) 等项目如何帮助开发人员和组织创建自选开源计算堆栈。

上个月在加拿大多伦多举行的 CloudNativeDay 2016 上，[肯·欧文斯](https://twitter.com/kenowens12)、[思科云原生平台和服务的](https://blogs.cisco.com/tag/mantl)首席技术官与[李·卡尔科特](https://www.linkedin.com/in/leecalcote)、[网络安全管理软件产品](http://www.solarwinds.com/)技术战略高级总监和 TNS 创始人亚历克斯·威廉姆斯谈论了这些问题。

[集装箱编排的摆来摆去](https://thenewstack.simplecast.com/episodes/the-swinging-pendulum-of-container-orchestration)

欧文斯指出，虽然总会有一些新技术需要应对，但重要的是人们如何利用这项技术。

“我们开始思考历史进程，认为这有点像股票市场。过去并不预示着积极的未来，但它确实预示着你可以依靠未来发生的变化，”欧文斯说。欧文斯在开发 Mantl 时考虑到了这种势头，将其设计成一组组件化的子系统。

欧文斯指出，Mantl 并没有打算取代现有的开源软件，而是花时间重新发明轮子。相反，它将当今最好的开源开发工具管理和打包成一个完整的解决方案，用于构建自己的堆栈。

运行微服务架构需要一套全新的技术。Owens 解释说，Linux 内核从来就不是高性能路由器。随着集装箱的出现，新技术，如 [FD.io](https://fd.io/) (快速数据输入/输出)，被开发出来帮助重新思考集装箱网络，

这些用于在云中工作的新工具包可能看起来很有挑战性，所以它们被采用还需要一段时间。

“这个钟摆在集中控制和集中协调之间摆动。全面了解任何环境中发生的操作，而不是将其推到边缘。作为一个行业，我们往往摇摆不定。在具有容器编排的分布式系统中，它成为一个焦点，”Calcote 说。

![cspugaww8aapxp1](img/0133e9c48e3867d2298ed5c671d1fd4e.png)

尽管对一些人来说，云原生化最终似乎是第二天性，但 Owens 强调，即使是开发人员也仍然需要帮助来理解这种新方法。

“我接触过的许多开发人员都不明白如何在 cloud native 中开发东西。他们习惯于以某种方式开发，他们并不真正确定云原生对他们来说意味着什么，”Owens 说。“许多人理解并喜欢容器，但是容器并不能帮助你成为云原生的，它们只是为你的应用程序提供了一个更好的包格式。作为一个行业，我们仍在讨论这些事情。”
