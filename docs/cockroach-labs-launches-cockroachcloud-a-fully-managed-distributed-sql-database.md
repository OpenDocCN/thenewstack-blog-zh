# 蟑螂实验室推出 CockroachCloud，这是一个完全托管的分布式 SQL 数据库

> 原文：<https://thenewstack.io/cockroach-labs-launches-cockroachcloud-a-fully-managed-distributed-sql-database/>

着眼于为全球客户群不断增长的互联网成长型公司提供服务，总部位于纽约的蟑螂实验室为其[cocroach db](https://www.cockroachlabs.com/product/)分布式 SQL 数据库、[cocroach cloud](https://www.cockroachlabs.com/product/cockroachcloud)推出了托管服务。

这项服务解决了运行全球数据库的困难，同时保持为最终用户提供即时服务所需的低延迟，[蟑螂实验室](https://www.cockroachlabs.com/)首席执行官兼联合创始人[斯潘塞·金博尔](https://www.linkedin.com/in/spencerwkimball/)在接受新堆栈采访时表示。

最初，CockroachCloud 现在处于测试版，可以在谷歌云平台和亚马逊网络服务上通过每个云各自的市场获得。当该公司开始对这项服务收费时，它将基于使用的节点数量，但随着时间的推移，将转向基于使用量的定价。

分布式、持续复制的数据库当然可以增加系统的弹性—您不希望将所有数据保存在一个位置、一个提供商甚至一个地区，以防发生大规模停机。以及异步主动/被动复制的传统方法，这种方法在出现问题时会丢失数据。但 Kimball 认为，当服务于全球用户群时，简单地复制一个数据集，甚至在几个不同的地理分布的数据中心之间连续复制是不够的。

“在许多情况下，你不希望把用户的数据放在任何地方，”他说。例如，不同的国家有不同的数据保护法。更重要的是，您希望将数据放在靠近用户的地方，以最小化延迟。例如，伦敦的某个人的数据在美国，当数据穿越海洋时，他将经历很长的等待时间。

“你必须平衡所有这些事情之间的紧张关系:你想分散数据，但同时，你想让数据靠近用户，”金博尔说。

CockroachDB 不是简单地在所有节点上复制所有数据，而是进行一致的地理复制:分区是基于地理位置的。当添加新用户时，他们会被添加到离他们的首选位置最近的分区。主拷贝保存在离客户最近的数据库实例中，然后才在其他位置复制。金博尔解释说，如果用户访问一个很远的位置，那么他们的数据可以被传输到最近的节点。

“蟑螂能够透明地完成所有这些，”金博尔说。“我们保持了这种复杂性的可控性。”

新服务本身旨在消除运行分布式数据库的所有管理难题，使用户能够扩展分布式事务数据库并在全球范围内持续复制它。管理分布式应用程序(如数据库)本身就更加困难，尤其是在全球范围内。

“启动全球集群并使其正常工作所需的自动化程度是我们现在花了一年时间做的事情。你不会希望一家公司(不是服务提供商)不得不做同样的事情。他们的时间不会花在做这样的事情上。”

在其研究中，Garter 发现云数据库系统现在是大多数企业工作负载的标准，内部部署越来越多地被归入“传统状态”该公司估计，从 2017 年到 2018 年，整体数据库管理系统市场增长了 18.4%，其中云 DBMS 占了增长的 68%。

在云领域，蟑螂将与谷歌自己的[扳手](https://cloud.google.com/spanner/)竞争，尽管它能够轻松跨越多个云提供商的数据库。金博尔认为，由于不同的需求，甚至是收购，几乎所有的公司都将不可避免地成为多云用户。

该公司周三在纽约举行的首次 ESCAPE/19 multicloud 会议上推出了这项新服务。
