# Scytale 的首席执行官 Sunil James 解释道

> 原文：<https://thenewstack.io/sunil-james-ceo-of-scytale-explains-spiffe/>

[Scytale 公司首席执行官苏尼尔·詹姆斯为您解读 SPIFFE](https://thenewstack.simplecast.com/episodes/sunil-james-ceo-of-scytale-explains-spiffe-for-you)

回到三月底的[,](https://blog.scytale.io/spiffe-joins-the-cncf-a2f0f36e4071)[云本地计算基金会](https://www.cncf.io/) [接受了](https://www.cncf.io/blog/2018/03/29/cncf-to-host-the-spiffe-project/)一个新的身份管理沙盒项目。该项目名为 [SPIFFE](https://spiffe.io/) ，代表面向所有人的安全生产身份框架。它得到了亚马逊、谷歌和红帽的支持，但该项目的核心是专门为 SPIFFE 创立的初创公司 [Scytale](https://www.scytale.io/) 。

Scytale 和 SPIFFE 是 Sunil James 的创意，他从世纪之交就进入了软件行业。詹姆斯的简历可以追溯到亚马逊和谷歌最早提供安全漏洞奖金和时间的公司之一，他对团队将企业环境迁移到快速可扩展的基于云的基础设施所经历的一些棘手问题有着长远的看法。

为此，SPIFFE 为系统提供了跨微服务和服务器进行身份认证的统一方式。使用 SPIRE (SPIFFE 运行时环境)作为处理身份的中央服务器，在端点上使用较小的代理来处理身份验证。SPIRE 只是 SPIFFE 框架的一个实现，因为 [Istio Auth](https://github.com/istio/istio/tree/master/security) 也在使用 SPIFFE。

James 说，身份对于可伸缩的环境极其重要，在这种环境中，系统会根据需求不断变化。“像 Kubernetes 这样的技术可以让你轻松地启动和关闭任意数量的服务器。这些东西是短暂的。它们存在了一段时间，它们必须与其他软件进行交互，对于一个具有一定规模的组织来说，让这些不同的软件相互认证变得非常困难。一个软件如何知道它与之交互的另一个软件实际上就是它所说的那个人？”詹姆斯问。

“这是 SPIFFE 试图解决的基本问题，是提供一种机制，不仅可以断言身份，而且允许这两者之间的加密通信，”詹姆斯说。

James 说，身份多年来一直是系统开发中讨论的话题。他说，SPIFFE 工作中的许多原则并不是新想法，而是新应用于现代环境的。

“当我们开始 Scytale 时，给我安慰的是，在我们开始 Scytale 之前，在 SPIFFE 项目中工作的是身份的概念；几代的技术专家和计算机科学家已经认识到，身份是实现和保护通信的一个非常重要的组成部分，无论是通过人类，这是大多数身份管理技术似乎关注的地方，还是在我们之前的 SPIFFE 和其他系统的情况下，围绕系统和服务，如果你愿意的话，“詹姆斯说。
