# 万事达卡的肯·欧文斯在技术和消费者之间架起了一座桥梁

> 原文：<https://thenewstack.io/mastercards-ken-owens-bridging-gap-tech-customers/>

[万事达卡公司的肯·欧文斯在技术和顾客之间搭起桥梁](https://thenewstack.simplecast.com/episodes/mastercards-ken-owens-on-bridging-the-gap-between-tech-and-customers)

在担任思科基础设施首席技术官期间， [Ken Owens](https://twitter.com/kenowens12) 是向新的、有时持怀疑态度的人群解释软件定义的基础设施的负责人。[最终，思科的其他人理解了【Owens 对他们说的话，现在该公司有了一个更加复杂的软件定义的基础设施战略。](http://www.datacenterknowledge.com/archives/2016/11/08/cisco-cto-cloud-native-group-stop-building-walls)

但现在，他在围栏的另一边担任了类似的角色，担任金融服务提供商万事达卡(Mastercard)的数字原生架构副总裁。现在，他的工作是向新的、甚至可能更加怀疑的受众阐明软件定义的基础设施的要求。在最近的洛杉矶开源峰会上，他注意到了他所认为的介绍性会议和“经验教训”讨论之间的出席不平衡，后者吸引了大部分人。

欧文斯认为，这并不是因为人们已经知道这项技术，不需要介绍。正如他在本期 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中告诉 [Scott Fulton](https://twitter.com/SMFulton3) 的那样，他观察到企业对于在任何技术上孤注一掷仍然犹豫不决，尤其是在开源领域——至少在它有更多历史背景之前。

“如果你看看我所代表的云原生空间，”Owens 说，“有数百个项目和公司在做云原生解决方案。因此，当你在企业中时，你必须问的第一个问题是，“好吧，我关心这些中的哪一个？我想更多地了解其中的哪一个？当你真的没有时间出去学习新的东西时，你会想从其他企业现在使用的子集开始。你想学习其他企业的做法。"

他认为，企业都在等着一个接一个地进行最初的尝试，包括那些我们在新堆栈中可能承认不再那么“新”的技术。其中一些企业可能已经确信将会有一个 20 世纪 80 年代或 90 年代风格的“洗牌”，留下一个或两个云提供商，也许还有一个分布式编排方法。那时，再犹豫就没有意义了，因为每个人的选择都已经为他们做出了。
