# VMware 在 Kubernetes 和 vSphere 之间架起了桥梁，推出了 Pacific Beta

> 原文：<https://thenewstack.io/vmware-bridges-kubernetes-and-vsphere-with-launch-of-pacific-beta/>

在巴塞罗那举行的 [VMworld 2019 欧洲](https://www.vmworld.com/en/europe/index.html)大会上，VMware 宣布了一项高级测试计划，为精选客户提供[项目太平洋](https://thenewstack.io/vmwares-project-pacific-integrates-vsphere-with-kubernetes/)。[该软件](https://thenewstack.io/5-things-to-know-about-vmwares-project-pacific/)旨在使管理员和开发人员能够在任何环境中使用他们已知的工具设计、部署和管理 Kubernetes 和容器基础设施。

该公司[在 8 月份的 VMworld 2019 美国大会上推出了](https://thenewstack.io/with-heptio-and-pivotal-vmware-doubles-down-on-kubernetes/) Project Pacific，旨在将 vSphere 转变为 Kubernetes 原生平台。

随着公司对 Pivotal 的收购及其自身客户群的规模，首席执行官[帕特·基尔辛格](https://www.linkedin.com/in/patgelsinger/)告诉旧金山的人群，“我们将成为 Kubernetes 的最大推动者。”

Pacific 是 VMware [Tanzu](https://blogs.vmware.com/cloudnative/2019/08/26/vmware-completes-approach-to-modern-applications/) 产品和服务组合的一部分，旨在为企业客户带来 Kubernetes。Tanzu 包括 Mission Control，这是一个单点控制，客户可以通过它管理所有的 Kubernetes 集群。

“我们将 Tanzu 视为一个全面的环境，为客户搭建起开发和运营世界之间的桥梁。这将是一个超级强大的企业级 Kubernetes 平台。Kubernetes 是这种转变的主要工具，我们现在有许多工作要做才能使它发挥作用，”Gelsinger 在 8 月份说。

然而，有人批评该公司当时展示的只是一个概念，而不是实际产品。

在宣布测试版时，该公司表示，参与需求已经超过了可用性。不过，该公司计划在今年晚些时候扩大对该项目的参与，并仍在邀请参加 vmw.re/PacificBeta 大会。

该公司还宣布了云原生主服务能力，增加了其稳定的[能力计划](https://blogs.vmware.com/partner/2019/06/master-in-msc.html)，以帮助合作伙伴展示对其技术的掌握，并向客户展示自己的优势。云原生主服务能力认证建立在二月份推出的 [VMware PKS 解决方案能力认证](https://blogs.vmware.com/partner/2019/02/vmware-pks-solution-competency-now-available.html)之上。这项新的能力认证将证明合作伙伴有能力设计一个基于 Kubernetes 的平台，该平台由云原生生态系统的补充技术提供支持。

VMware 是新体系的赞助商。
