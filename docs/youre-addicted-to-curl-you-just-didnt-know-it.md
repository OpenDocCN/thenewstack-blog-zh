你沉迷于[卷发](https://curl.haxx.se/)而不自知。是的，很容易阅读它的描述——“curl 用于命令行或脚本中传输数据”——并认为，“这听起来真的很书呆子气。我永远不会用那个。”但是你——和数十亿像你一样的其他人 —知道。每天都是。Curl 毕竟是在你的任天堂 Switch、你的 Mac、你的汽车、你的手机上运行的，还有更多的。这是你的汽车用来下载地图到导航系统的工具。这是你的游戏设备可能用来上传数据到服务器的工具。

简而言之，Curl 每天都在幕后通过数十亿台设备进行互联网数据传输。

因此，你可能会认为 curl 的开发背后有一个大企业，或者它的创始人正在花费数十亿美元来治愈新冠肺炎。唉，不！丹尼尔·斯滕伯格(Daniel Stenberg)从 1996 年开始在业余时间研究 curl，将其作为一个开源项目。虽然 2019 年他让 curl 成为他的全职工作，但他花了 21 年才达到这一点。

运行一个开源项目可能会很艰难。考虑到这一点，我采访了 Stenberg，以更好地了解 curl 的起源和他保持 curl 的持续动机。

## “一个非常小的项目”

当 Stenberg 开始用 curl 在他的 IRC 频道上自动获取和提供货币汇率时，他并没有试图改变世界。对于这个开源项目，没有数十亿用户的期望。事实上，开源还没有被“发明”。那是 1996 年，直到 1998 年 2 月“开源”这个术语才被创造出来。不，斯坦伯格只是想建立一个小工具(“这能有多难？”)作为一个“非常小的项目。”从一个现有的工具(httpget)开始，他做了一些调整并发布了后来被称为 cURL 的东西，这是一种使用多种协议传输数据的方法。需要你的车更新地图吗？你很可能在工作中使用 curl。

今天，Rust 是酷孩子们正在编写的代码，但早在 20 世纪 90 年代，Stenberg 就用 c 语言编写了 curl，除了其他优点之外，这使得它非常便携。“你基本上可以在任何操作系统或任何 CPU 上运行它；无论大小，新旧，你都可以在上面运行 curl，”他说。这被证明是 curl 和其他数据传输工具之间的区别。

当 Stenberg 用 C 语言写这个小项目时，没有人为他的麻烦付钱。这不是他的全职工作，甚至不是他的兼职工作。“这完全是枝节问题，”他说。“我这么做是为了好玩。”

当然，并不是所有关于 curl 开发的事情都很有趣。Curl 现在被用于数十亿台设备(首先是你的苹果 iPhone 或三星 Galaxy 手机)，这意味着 Stenberg 的副业可能对其他人来说是至关重要的。你可以想象，这可能会引起一些摩擦。“那些实际上向我唠叨错误报告或向我询问功能的人，是那些使用我在家里做的东西而获得报酬的人，”Stenberg 说。

并不是说 Stenberg 过分担心。虽然他承认这总是有点令人讨厌，但他强调大多数人一点也不讨厌或麻烦。

尤其是那些有贡献的人。

## 聚集在一起的团体

curl 社区中的许多人都提供了相当多的帮助。斯坦伯格说他拿回了很多好东西。多少钱？Curl 以八周为一个发布周期，每次发布都有 50 到 60 人投稿。

这是令人印象深刻的，因为该项目始于一个单独的 Stenberg 抓挠汇率“痒”。正如他所说，“从一开始，我就确信在别人的帮助下会变得好得多。”不只是更好，而是实际使用。正如 [Stenberg 所写的](https://daniel.haxx.se/blog/2016/05/31/on-billions-and-users/)，让 curl 开源让人们愿意帮忙，并添加使项目有吸引力的功能，这有助于让公司愿意使用和信任它。Stenberg 说如果这个项目没有在[开源许可](https://curl.haxx.se/docs/copyright.html)下发布，它将会在 90 年代消亡，“被遗忘和忽视”

一些最好的代码贡献已经帮助 Stenberg 扩展了 curl 的效用，远远超出了他的个人经验(和时间)所允许的范围。因此，他努力让人们更容易为这个项目做出贡献。他说，有时候，用一种更适合 curl 的方式自己写代码会更快，但他试图做更长远的考虑。“如果我帮助这个人做得对，也许他下次会做得更好，这将使我们更容易和更长期地改进项目，”Stenberg 说。

21 年后，斯坦伯格的 little side 项目已经吸引了数百名贡献者，拥有数十亿用户。你已经是这些用户中的一员了。也许是时候成为一名贡献者了？访问项目网站了解[如何成为贡献者](https://curl.haxx.se/dev/contribute.html)。
