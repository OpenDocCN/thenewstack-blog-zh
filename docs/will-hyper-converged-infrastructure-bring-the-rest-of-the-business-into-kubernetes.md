# 超融合基础设施会将其余业务引入 Kubernetes 吗？

> 原文：<https://thenewstack.io/will-hyper-converged-infrastructure-bring-the-rest-of-the-business-into-kubernetes/>

对 KubeCon+CloudNativeCon 主题演讲观众的民意调查显示，大约 80%的与会者是新人。在这一集的 New Stack Makers 播客中，TNS 出版商 Alex Williams 与 Pivotal 的 [Cornelia Davis](https://twitter.com/cdavisafc) 和戴尔科技的 [William Geller](https://www.linkedin.com/in/wgeller/) 谈论了将企业纳入 Kubernetes 的旅程，以及如何使用托管架构来克服 Kubernetes 业务流程的复杂性。

[超融合基础设施](https://thenewstack.simplecast.com/episodes/hyper-converged-infrastructure)

Pivotal 的技术副总裁戴维斯指出，MiniKube 是吸引这些人加入的主要方式之一。

“只需简单的下载和几个命令，您就可以上路了。你将把应用程序推送到 Kubernetes。您可以在自己的笔记本电脑上完成这项工作，或者他们可以通过云服务提供商来完成这项工作。”

她继续说道，“这一切都很简单。就是管用。”

这是问题的一部分。因为这种最初的体验掩盖了 Kubernetes 编排的所有真正的复杂性。

Davis 说，随着一切的扩展，您仍然需要虚拟计算、虚拟存储和虚拟网络。这时，在 MiniKube 上 30 分钟的体验变成了 30 天或 3 个月。

毫无疑问，在生产中对抗 Kubernetes 是困难的。

Geller 曾在戴尔技术公司(Dell Technologies)为 Kubernetes 从事产品营销工作，他谈到了利用复杂基础架构而不必直接管理所有这些复杂性的不同方法。他与 [VXRail](https://www.dellemc.com/en-gb/converged-infrastructure/vxrail/index.htm) 合作，这是一个超融合基础设施(HCI)包，本质上是一个充满多种数据中心管理技术的物理盒子。VXRail 与 VMware 紧密集成，支持轻松升级和生命周期管理。试图从“一个好的状态到下一个好的状态”

盖勒说，“我们试图做的是将价值线提升到基础设施之上。允许每个人专注于他们日常工作中真正重要的事情，专注于对业务真正重要的差异化任务，而不要担心下面是什么。”
