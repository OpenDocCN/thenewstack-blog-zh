# 是什么让 Wasm 与众不同

> 原文：<https://thenewstack.io/what-makes-wasm-different/>

披露:B. Cameron Gain 是 ReveCom Media Inc .的创始人和所有者，该公司为 Cosmonic 提供咨询服务。

西班牙瓦伦西亚——web assembly(Wasm)是 CNCF 项目下最热门的话题之一。在这一集的新 Stack Makers 播客中，录制于秀场的[kube con+CloudNativeCon Europe 2022](https://events.linuxfoundation.org/kubecon-cloudnativecon-europe/)，[Liam Randall](https://www.linkedin.com/in/hectaman)，首席执行官兼联合创始人 Cosmonic， [Colin Murphy，](https://www.linkedin.com/in/colin-murphy-08b3601b/) 高级软件工程师， [Adobe](https://www.adobe.com)

Wasm 的一个典型特征是它在 CPU 级别上运行，与 Java 或 Flash 没有什么不同。兰德尔说，这意味着 Wasm“可以在任何地方运行”“每个人都可以开始使用 Wasm，它的功能就像一个微型 CPU。你甚至可以把 WebAssembly 放在其他应用里面。”

Wasm 具有二进制格式(带有。wasm 文件格式)并且可以像 C 或 C++一样在 CPU 级别上运行，这意味着它是高度可移植的。“WebAssembly 确实令人兴奋，因为它给了我们两个真正令人惊叹的基本东西:一个是跨各种 CPU 和架构的可移植性，甚至是到其他地方的可移植性，比如到 web 浏览器中，”Randall 说。“它还为我们提供了一个可移植的安全模型，并且在所有这些不同的环境设置中都适用。”

[是什么让 Wasm 与众不同](https://thenewstack.simplecast.com/episodes/what-makes-wasm-different)

这种便携性使 wasm 成为边缘应用的绝佳候选。墨菲描述说，它在边缘的机器学习(ML)推理能力对于分布在许多不同应用程序中的应用程序来说尤其有前途。Wasm 也特别适合 ML edge 和其他应用程序的协作。“协作体验正是 WebAssembly 真正适合的，”他继续说道。

从许多方面来看,“WebAssembly”这个名称并不能直观地反映其含义。“WebAssembly 既不是 web 也不是 assembly——所以，这是一种有点尴尬的命名技术，但却是一种值得研究的技术，”Randall 说。“对于您的内部团队来说，通过采用这一新标准，转变他们的业务方式以节省成本和提高安全性有着不可思议的机会。”
