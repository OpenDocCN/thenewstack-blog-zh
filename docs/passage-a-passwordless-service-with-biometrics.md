# 通行:生物识别的无密码服务

> 原文：<https://thenewstack.io/passage-a-passwordless-service-with-biometrics/>

[Passage](https://passage.id/) 为网站添加设备原生生物识别授权，允许在有或没有触控 ID 的设备上实现无密码安全。

在新堆栈的演示中，Passage 的联合创始人 Cole Hecht 和 Anna Pobletts 展示了该服务如何为开发者提供生物识别服务。

Hecht 和 Pobletts 已经在产品安全领域工作多年，经常出现的问题是基于密码的安全性。但是真的没有很好的解决办法，Pobletts 说。多因素身份认证增加了安全性，但缺乏用户体验。[魔法链接](https://thenewstack.io/stytchs-api-first-approach-to-passwordless-authentication/)、[自适应 MFA](https://thenewstack.io/how-to-optimize-customer-identity-and-access-management/) 和其他技术增加了一点改进，但并没有在用户体验和安全性之间取得很好的平衡。

“然而，生物识别是我们见过的唯一一种开箱即可为您提供出色安全性和出色用户体验的选择，”Pobletts 说。

赫克特和波夫莱特的目标是什么？向开发人员提供他们自己实现起来具有挑战性的东西:具有高安全性级别和出色用户体验的无密码服务。

Pobletts 说，Passage 建立在 WebAuthn 的基础上，这是一种网络协议，允许开发者通过这些设备上的授权码将网站与浏览器和各种设备连接起来。

“所以现在什么都有可能，”她继续说道。“是指纹识别器和人脸识别之类的东西。但在未来，它可能是声音识别，或者它可能是，你知道，你的存在和类似的东西…它可能是未来的各种东西。但最终，您的设备会生成一个加密密钥对，并将私钥存储在您设备的 TPM 中。这个协议最酷的一点是，你的生物特征数据永远不会离开你的设备，这是隐私的巨大胜利。在那一段中，你的浏览器，没有人真正以任何方式看到你的指纹数据。”

“它在引擎盖下是加密安全的，顶部有通道作为平台，”波夫莱特斯说。

Pobletts 表示，WebAuthn 是为单个设备设计的。例如，开发者向一个设备认证一个指纹。但是在互联网上，用户可能有一部电话、一台平板电脑和一台电脑，这种方式就不太适用了。通道在不同的设备之间协调和配合，以提供轻松的体验。

“所以对我来说，我有一部 iPhone，我有 face ID，”Hecht 展示这项服务时说。“然后，我将自动在两台设备上登录。因此，这是一种很好的方式，让每个用户都可以访问网站，无论他们使用什么设备。”

Hecht 说，随着 Passage 的出现，用户添加的任何设备都可以添加生物特征。段落处理多设备编排。

用例？

“金融科技人喜欢它的安全属性，他们有点喜欢那种酷，闪亮的用户体验，他们希望交付给他们的最终用户，”Hecht 说。任何关注转化率的网站或企业都是一个笼统的术语。想要注册的人，试图通过注册和创建账户的人数来衡量成功的人，正在注册。“Passage 有一个非常好的故事，因为我们消除了这些转换点周围的很多摩擦。”
