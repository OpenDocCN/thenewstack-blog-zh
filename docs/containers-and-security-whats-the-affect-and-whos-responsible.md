# 集装箱和安全:有什么影响，谁负责？

> 原文：<https://thenewstack.io/containers-and-security-whats-the-affect-and-whos-responsible/>

[#142:安全与集装箱化的交汇点](https://thenewstack.simplecast.com/episodes/142-the-intersection-of-security-with-containerization)

现在，容器化和流程编排平台颠覆了端点安全等传统概念，那么现代网络中哪些点得到了安全处理，谁对它们负责呢？几年前，有人谈到让容器化的应用程序开发人员更加负责将安全性融入到他们的映像中。但在实践中——至少在某些圈子里——我们看到这项工作落到了新一代的站点可靠性工程师(SRE)身上，他们的性能管理任务现在正得到安全策略执行的补充。

[RedMonk](http://redmonk.com/) 的行业分析师 [Fintan Ryan](https://twitter.com/fintanr) 在最新一期的 [The New Stack Analysts](https://thenewstack.io/podcasts/analysts) 播客中与 TNS 出版商 [Alex Williams](https://twitter.com/alexwilliams) 交谈时说:“大规模开展 SRE 业务的是几家真正的超大规模公司。Ryan 指出，虽然 SRE 的权限通常不是安全本身，但他们在当前角色中所学到的关于他们管理的系统的知识最终会成为安全问题。

关于 SREs 在组织内产生的反馈循环，Ryan 说，采用类似 Google 的角色模型的人认为 SREs 的反馈应该只针对开发人员是错误的。“这绝对不是。这是为组织中的每个人准备的，”他说。“试图建立越来越多的组件，让所有这些反馈循环持续运行，这样人们就可以了解和分析正在发生的事情，这绝对是一件关键的事情。”

在同一次采访中，独立行业分析师[王晨曦](https://twitter.com/chenxiwang)指出[最新一批安全初创公司](https://www.google.com/search?q=security+%2B+ai+site%3Athenewstack.io&rlz=1C1CHFX_enUS665US665&oq=security+%2B+ai+site%3Athenewstack.io&aqs=chrome..69i57.13998j0j4&sourceid=chrome&ie=UTF-8)正在采取与传统的以终端为中心的思维模式截然不同的安全政策和框架。“他们正在做的事情之一，”王博士说，“是提供实时反馈回路作为他们产品的一部分。因此，您可以进行更改并实时看到更改，这体现在他们提取的数据中，并且能够非常有效和即时地获得指标并衡量效果。而这些东西在传统的安全产品中通常是不存在的。”
