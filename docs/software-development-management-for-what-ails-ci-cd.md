# 针对 CI/CD 问题的软件交付管理

> 原文：<https://thenewstack.io/software-development-management-for-what-ails-ci-cd/>

CloudBees 赞助了这个播客。

【CloudBees 的产品营销总监 Brian Dawson 当然非常了解孤岛，尤其是如何消除孤岛以支持健康的 DevOps 文化和顺畅的 CI/CD 渠道。在他过去 30 年的职业生涯中，他显然也看到了许多工具和流程的采用。但是，不幸的是，新的和强大的工具传统上为 CI/CD 的持续和密集的协作制造了更多的障碍。开源工具选择的激增无疑加剧了这一问题，尤其是对于云原生部署。

在这一集的 New Stack Makers 播客中，TNS 出版商 Alex Williams 采访了 Dawson，以了解更多关于软件开发管理(SDM)的信息。

[软件交付管理](https://thenewstack.simplecast.com/episodes/software-delivery-management-for-what-ails-ci-cd)

“随着每一项新的发展，总有一种风险，即随着一项新技术的出现，一种新的语言会成为另一个筒仓，”Dawson 说。

随着组织“从左到右”发展，通常孤立的开发运维团队之间的许多摩擦也转移到了 CI/CD..道森说:“由于行业一直专注于项目到产品的自动化、协调和转移，因此我们从提交到部署的传统组织筒仓中跨职能运营。

作为解决这些组织 DevOps 和 CI/CD 挑战的一种方式，软件交付管理(SDM)的概念已经获得了很多关注，这也是 Dawson 作为嘉宾的 New Stack Makers 播客的主题。部分 DevOps 和 CI/CD；SDM 以 GitOps 为基础，是支持更快、更健壮的测试和部署周期的一种方式。它还有助于解决上面概述的一些组织结构问题。

Dawson 说:“我们知道我们需要做什么，才能跨越现有的团队和组织筒仓，以便我们都专注于快速、重复和可靠地交付高质量的软件。”。“我们必须让人们一起工作，让这些人一起工作的一部分是确保我们能够使用一套通用的工具，或者跨这些工具连接语言、数据和信息，以便我们有一个内聚的流程，即我们从开发到 QA 再到运营，最终到客户。”

道森说，虽然 CI/CD 和 DevOps 肯定有助于减少 DevOps 团队之间“从左到右”的摩擦，但 SDM 在很大程度上可以提供行业内“从上到下”所缺少的东西。

“缩短交付时间，提高部署频率，降低变更失败率和加快恢复速度:但现在我们的挑战是——SDM 战略将帮助组织——在您的业务中扩展这一点，”道森说。

虽然 SDM 现在被 CloudBees 以及分析师和竞争对手视为“在软件业务中取得成功”的必要条件，但 CloudBees 正在开发用于软件交付管理的 [CloudBees 解决方案](https://www.previous.cloudbees.com/blog/how-cloudbees-does-software-delivery-management-drinking-our-own-champagne)，以帮助组织实施 SDM，当然，还有助于支持其 Jenkins X 和其他开发工具。

此外，SDM 平台的核心部分将是它如何“建立一个统一的记录系统”，或一个公共数据模型，它将由一个数据存储组成。其中，数据存储将为 ML 创建一个基础，并代表 GitOps 的后端。

道森说，由此产生的“应用程序框架将允许您现有的工具，跨您的各种软件团队和业务团队来扩展您的数据存储…并将其集成到 SDM 平台的更大类型的[控制平面]中”。
