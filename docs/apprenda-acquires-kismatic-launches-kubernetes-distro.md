# Apprenda 收购 Kismatic 并推出 Kubernetes 发行版

> 原文：<https://thenewstack.io/apprenda-acquires-kismatic-launches-kubernetes-distro/>

企业 PaaS 云平台提供商 [Apprenda](http://www.apprenda.com) 继续看好 [Kubernetes](/category/kubernetes/) 开源容器编排工具。该公司收购了 Kubernetes 企业支持公司 Kismatic，这是其首次收购，并推出了新的 Kubernetes 分销。

“Kismatic 是第一批真正追求企业 Kubernetes 的公司之一，”Apprenda 首席执行官兼联合创始人 [Sinclair Schuller](https://twitter.com/sschuller) 说。“他们看到了谷歌发布的这个软件，看到了所有这些功能，但他们也看到了企业使用 Kubernetes 的方式会有很大不同，所以开始围绕安全、认证和审计构建功能。”

作为发展最快的开源项目之一，Kubernetes 由 Google 开发，旨在为组织提供其内部使用的基于容器的工作流架构(最初称为[Borg](https://thenewstack.io/google-learned-borg-container-management/))。收购完成后，Apprenda 将开始为 Kubernetes 部署提供商业支持订阅。

Kismatic 创始人 [Patrick Reilly](https://twitter.com/preillyme) 将担任 Apprenda 的公司首席技术官，领导其 Kubernetes 战略。赖利还是[云本地计算基金会](https://cncf.io/)的董事会成员，该基金会[负责监管 Kubernetes 项目](https://thenewstack.io/google-launches-a-separate-foundation-around-container-orchestration/)。在 Kismatic 之前，Patrick [创立了](https://www.linkedin.com/in/preilly) Orly Atomics，[被 Mesosphere](http://techcrunch.com/2014/09/17/mesosphere-snags-orlyatomics-in-acquihire-deal/) 收购。

[Apprenda 成立于 2007 年](http://www.forbes.com/sites/janakirammsv/2016/05/19/apprenda-acquires-kismatic-to-strengthen-its-kuberenetes-offering/#79533e925e04)，最初的使命是提供企业级内部 PaaS 云服务，并[培养了支持微软的专业技术](https://thenewstack.io/apprenda-raises-24-million-makes-added-container-support-a-priority/)。Net 和 Windows Server 部署，后来还扩展到了 Enterprise Java。然而，对阿普伦达来说，对 Kubernetes 日益增长的兴趣是不可抗拒的；今年 3 月，该公司开始在其云服务中支持 Kubernetes。

展望未来，Apprenda 将继续并扩大 Kismatic 现有的业务运营。它将提供支持订阅、有保证的服务级别协议和专业服务。它还将继续开发用于将 Kubernetes 与 Apprenda 平台集成的工具，包括用于身份管理(LDAP/Active Directory)、Kerberos 身份验证和富审计的 Kubernetes 企业插件。

Kismatic 还向云原生计算基金会主办了 KubeCon 会议，但将该活动捐赠给了 CNCF。

Apprenda 是众多围绕 Kubernetes 构建企业战略的公司之一。CoreOS 提供了一个名为[constructive](https://thenewstack.io/coreos-debuts-tectonic-a-commercial-kubernetes-distro/)的商业支持发行版，Red Hat 已经将 Kubernetes [构建到其 OpenShift PaaS 平台 OpenShift](https://thenewstack.io/solving-spoiled-broth-open-source/) 中。

“CNCF 对我们的两位创始成员 Apprenda 和 Kismatic 的最新发展感到兴奋。CNCF 临时执行董事 Chris Aniszczyk 在一份声明中表示:“通过此次收购，行业将受益匪浅，公司将变得更加强大，致力于推进 CNCF 技术，并向新的云原生计算模式过渡。”。“如果公司不弥合传统 IT 和云原生技术之间的差距，他们可能会大大落后。”

CoreOS 是新堆栈的赞助商。
