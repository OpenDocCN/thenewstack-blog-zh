# Twistlock 如何保护企业容器

> 原文：<https://thenewstack.io/twistlock-helping-enterprises-better-secure-containers/>

随着企业不断转向微服务和基于容器的基础设施，许多人想知道如何最好地保护这些技术。幸运的是，一些公司开始着手改造容器技术以满足企业需求。这是容器安全平台 [Twistlock](https://www.paloaltonetworks.com/prisma/cloud) 的领域，该平台最近发布了一本关于容器 PCI 合规性的电子书。

“使用容器，您可以确切地知道容器中正在运行什么进程。Twistlock 首席执行官 [Ben Bernstein](https://www.linkedin.com/in/ben-bernstein-3427383) 在 TNS 执行主编 [Joab Jackson](https://twitter.com/Joab_Jackson) 主持的新一集 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 中说:“我们发现，基于良好的行为，容器应该做什么要容易得多。

[扭锁如何保护企业容器](https://thenewstack.simplecast.com/episodes/how-twistlock-secures-the-enterprise-container)

Twistlock 的[特性](https://www.twistlock.com/runtime/)包括环境检查、策略执行和基于角色的访问控制。“我们可以停止、启动、分析和理解任何参与者正在用容器引擎做什么。这使我们能够完成大量繁重的合规工作。我们做的另一件事是插入所有通知和所有操作系统功能，使一个进程或操作系统级别的任何东西能够了解操作系统内部发生了什么。如果我们看到任何东西，那么我们基本上可以改变并阻止它，”伯恩斯坦说。

Bernstein 指出，为了使开发人员更容易完成这一过程，可以通过使用 Rest API 将 Twistlock 控制台轻松集成到 CI/CD 管道中。

“我们认为，当开发人员做了不合规的事情时，他们应该尽快得到通知。我们不会等到有问题的时候。伯恩斯坦说:“当你将一个映像放入生产或任何环境中，而我们意识到它不兼容时，我们会让开发人员知道，这样你就可以把它推回去，说他们必须修复它。”。

最终，Twistlock 希望帮助企业和开发者创建更安全的容器化基础设施。“当开发人员做错事情时，你必须授权他们去修复它。我觉得是有一条细线的，但是你要自动执行。你不能强迫某人自愿关心安全，”他说。

[Twistlock](https://www.paloaltonetworks.com/prisma/cloud) 是新堆栈的赞助商。
