# 使用通用资源代理在 Kubernetes 上运行 Mesos 应用程序框架

> 原文：<https://thenewstack.io/running-mesos-application-frameworks-on-kubernetes-with-the-universal-resource-broker/>

[#155:使用通用资源代理](https://thenewstack.simplecast.com/episodes/155-running-mesos-application-frameworks-on-kubernetes-with-the-universal-resource-broker) 在 Kubernetes 上运行 Mesos 应用框架

在 KubeCon 和 CloudNativeCon 2017 的这场讨论中，[新堆栈分析师](https://thenewstack.io/podcasts/analysts)深入探讨了 [Univa 的](https://www.univa.com/products/navops.php) [通用资源代理](https://blogs.univa.com/2017/09/univa-releases-open-source-universal-resource-broker-for-kubernetes/) (URB)。我们了解了这种开源适配器如何允许 Kubernetes 和 [Mesos 工作负载](https://d2iq.com/)在同一基础设施上运行。TNS 创始人亚历克斯·威廉姆斯与 Univa Corporation 首席技术官兼业务开发人员[弗里茨·费尔斯特尔](https://www.linkedin.com/in/fritzferstl/)和[系统代码](https://www.linkedin.com/company/thrashing-code/)软件架构师[阿德里安·霍尔](https://www.linkedin.com/in/adron)一同加入。

“我知道很多组织希望将(他们的应用)容器化。带来模块化和隔离，而不用担心它运行在 30 年前的硬件上，”Hall 说。

Ferstl 补充说，运行 Kubernetes 而不是 Mesos 的人可能会看着它说，“那里有一些有用的东西，我可以用它工作。我可以用 URB 之类的东西来运行代码，而不是试图将代码强加到 Kubernetes 中。”

当在 Mesos 上容器化工作负载时，数据中心操作系统(DCOS)是经常选择来运行它们的环境。如果用户在 Mesos 中运行工作负载，并且希望将工作负载转移到 Kubernetes，Ferstl 解释了 URB 如何实现 Mesos API。

“我们采用了 Mesos 的 API，所以 Mesos 框架与我们的主人对话，就像我们与 Mesos 主人对话一样，”他说。"最后，Mesos 框架没有注意到."
