# Fuzzbox 的 Ramin Keene 介绍了生产实验

> 原文：<https://thenewstack.io/ramin-keene-of-fuzzbox-on-experimenting-in-production/>

该播客由 [InfluxData](https://www.influxdata.com/) 和 [KubeCon+CloudNativeCon](https://www.cncf.io/kubecon-cloudnativecon-events/) 赞助。InfluxData 提供了一个完整的开源平台，专门为指标、事件和其他基于时间的数据而构建，是一个现代的时间序列平台。KubeCon + CloudNativeCon 将于 5 月 20 日至 23 日在巴塞罗那举行，将邀请 Kubernetes、Prometheus、gRPC、Envoy、OpenTracing 等热门项目背后的主要维护者以及其他领域专家、采纳者和最终用户参加，以推动云原生计算的教育和发展。

[拉明·基恩对 Fuzzbox 进行生产实验](https://thenewstack.simplecast.com/episodes/ramin-keene-of-fuzzbox-on-experimenting-in-production)

在这一集的 New Stack Makers 播客中， [TC Currie](https://twitter.com/tccurrie) 加入了 [Ramin Keene](https://www.linkedin.com/in/raminkeene/) ，他是 [Fuzzbox 的首席执行官和创始人，](https://fuzzbox.io)这是一家全新的公司，可以安全地进行生产实验。在加州奥克兰举行的轨道发射大会上，她找到了他，他在结束演讲时问了一个问题“如果在生产中进行实验是完全安全的，你为什么不呢？

基恩说，他不得不做出一个在整个行业都很常见的转变，他过去能够了解他公司的整个堆栈，并确切知道如何修复它。但是随着复杂性的增长和增加，他现在离工程师们所做的事情已经有三个层次了。

引用 [honeycomb.io](https://honeycomb.io) 创始人 [Charity Majors](https://twitter.com/mipsytipsy) 的话，他说生产测试意味着使用“真实用户、真实流量、真实规模和真实不可预测性”进行测试

这也意味着将软件开发周期延长到发布之后。软件过去是装在磁盘上的，这对发布周期来说是一个明确的终结。但随着不断交付，这种明显的区别已经成为过去。

在最高层次上，经验会带来新知识，而测试不会。测试是关于验证、确认和整合，而实验是关于发现、学习和未知。

“你不知道你不知道什么，”基恩说。这就是实验的由来。【T2

 **这种新的思维方式的挑战之一是创造一个好的实验。来自 SRE 的工程背景，他认为他可以有一个想法并抛出一个测试，但他的数据科学家严厉批评了他，指出有多少意想不到的变化会受到他认为他正在测试的“一个”变化的影响。他说，花时间学习如何创造一个实验是很重要的。

请收听基恩关于生产实验背后的三个概念，Fuzzbox 实际上为客户提供了什么，以及患病金丝雀的危险。
