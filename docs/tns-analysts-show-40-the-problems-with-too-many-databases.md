# 数据库太多的问题

> 原文：<https://thenewstack.io/tns-analysts-show-40-the-problems-with-too-many-databases/>

Orchestrate 的首席执行官兼联合创始人安东尼·法尔科(Antony Falco)和他在波特兰的老同事、New Stack 的创始人亚历克斯·威廉姆斯(Alex Williams)一起参加了本期 TNS 分析师播客。他们讨论了查询语言的演变，被放弃的 GitHub 项目的硅墓地，以及作为服务的数据库的增长，特别是作为物联网的附件。

更多剧集，请查看新堆栈的播客部分。

[#40:数据库过多的问题](https://thenewstack.simplecast.com/episodes/40-the-problem-with-too-many-databases)

波特兰的 Orchestrate [本月被 CenturyLink Cloud](https://thenewstack.io/centurylink-acquires-orchestrate-for-its-database-as-a-service/) 收购。Alex 引用了最近[对 Foundation DB](http://techcrunch.com/2015/03/24/apple-acquires-durable-database-company-foundationdb/) 的收购以及数据库市场的其他举措，并询问 Antony 所有这些活动反映了什么。

“总的来说，市场正在整合。很久以前，当新数据库出现爆炸式增长时，人们预测这个市场将会整合。投资者说这是不可避免的，”安东尼说。

> “他们认为这些公司会进行整合。现在的情况是:查询正在整合。”

他认为，生产中有 35 到 40 个数据库，而十年前这些数据库并不存在。“人们不想在生产中运行四五个不同的数据库来构建典型的交互式应用程序，”Antony 说，“他们想简化他们的操作生活。”

“当你看到像 DataStax 这样的公司为 Cassandra 收购一个图形数据库引擎时，”他说，“他们真正做的是:他们试图把所有的查询放在一个 API 后面。”

安东尼称，Orchestrate 拥有超过 9000 名用户，“从财富 500 强到非洲学生”。那些使用 Orchestrate 构建应用程序的人平均部署略多于两种查询类型，这意味着人们现在构建的应用程序需要至少两到三个数据库。Anthony 回忆说，仅仅几年前，这样的案例还是罕见的例外，但是现在开发者和企业家正在模仿那些用这种方法成功的人。

Alex 指出，分布式架构环境的复杂性会越来越深。

“有些人希望它只是一个黑匣子，”安东尼评论道。“随着我们走向越来越多的服务，为了真正抓住世界上开发人员的想象力，你必须与他们分享它是如何构建的。我们了解到，获得参与的方法之一是拉开帷幕，向人们展示这个分布式系统是如何构建的。”

> 安东尼说:“你已经冒了风险——如果你简化并制造出设计精良、易于使用的东西，人们会认为这背后没有什么复杂的东西。”

“你必须做一点工作，让这项技术与工程师相关联。”
