# 机器人、表情符号和开源维护者——人和工具如何改变现状

> 原文：<https://thenewstack.io/bots-emojis-and-open-source-maintainers-how-people-and-tools-make-the-difference/>

[VMware](https://www.vmware.com/company.html) 赞助了这篇文章。

开源维护者今天面临着一系列不同的挑战:机器人有助于管理过载，表情符号在开源群体中占主导地位，就像它们在 20 多岁的社交圈中一样。同时，维护人员前所未有地深陷治理问题。

在这个[的新堆栈制造商](/podcasts/makers)播客中，TNS 创始人兼发行人 [Alex Williams](/author/alex/) 和 [VMware](https://www.vmware.com/company.html) 嘉宾[开源社区战略总监 Dawn Foster](https://www.linkedin.com/in/dawnfoster/) 、VMware 高级技术人员 [Nikhita Raghunath](https://www.linkedin.com/in/nikinath/) 和 VMware 高级首席软件工程师 [Michael Klishin](https://www.linkedin.com/in/michaelklishin/) 讨论了作为开源维护者、构建社区和领导者的感受。

[僵尸程序、表情符号和开源维护者哎呀！](https://thenewstack.simplecast.com/episodes/bots-emojis-and-open-source-maintainers-oh-my)

开源维护者的背景也可以提供一个窗口，当一个人参与到一个拥有开放支持开源维护者的办公室的组织中时，他可以如何为今天的挑战做准备。

例如，Foster 在 20 世纪 90 年代中期作为一名系统管理员第一次接触到开源，当时她刚刚获得了计算机科学学位。她很快意识到“开源项目的关键是社区。”

“这些年来，这对我来说是一个非常有意义的职业——我在不同的国家有朋友，我可以去几乎任何地方，找个人一起喝杯咖啡或饮料，吃顿午餐什么的。对我来说，这真的是一个社区，”福斯特说。“这是我喜欢开源的一部分。这些年来，作为这项工作的一部分，我遇到了这么多了不起的人。”

Raghunath 是 Kubernetes 指导委员会的成员，大约三年前，当她还是一名大学学生时，就开始在[谷歌代码之夏项目中实习，她很快就“真的喜欢上了这个社区”。“我非常幸运地将开源作为我日常工作的一部分，”她解释道。"而且我不用在晚上和周末工作."](https://summerofcode.withgoogle.com/)

Klishin 十几岁的时候就开始接触开源软件 Mozilla Firefox，作为一名软件工程师，他开始在乌克兰从事 Ruby 和 Ruby on Rails 项目。

“如果你想为一个项目做贡献……你可以联系维护人员，他们绝对会感谢你的贡献，”Klishin 说。“你会了解他们，谁知道这会把你带到哪里。”

虽然注意到开源社区的行为在许多情况下从“非常敌对的地方，真的不太有趣，也不太友好”到今天更加积极的氛围，但 Foster 强调仍有更多工作要做。

“包容性是我们 VMware 现在非常重视的事情，”福斯特说。“我们正在考虑如何应用成百上千的开源库，以及如何让语言更具包容性……这不是小事。”
