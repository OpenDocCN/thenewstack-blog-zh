# Meroxa 的目标是为数据管道提供简单的按钮

> 原文：<https://thenewstack.io/meroxa-aims-to-provide-the-easy-button-for-data-pipelines/>

在 Heroku 听取了客户对数据管道的挑战后， [DeVaris Brown](https://www.linkedin.com/in/devarispbrown/) 和 [Ali Hamidi](https://github.com/ahmeroxa) 决定在 Heroku 改变战略方向时解决这些棘手问题。这成为他们公司 [Meroxa](https://meroxa.com/) 的基础。

“他们在市场上留下了巨大的空白。正如您可能熟悉的那样，Heroku 基本上已经成为开发人员体验和易用性的同义词，并且是平台即服务应该如何运行的事实上的标准。对吗？人们真的向我们要 Heroku 的数据，我们听够了。这对我们来说似乎是一个巨大的机会，”布朗说，他是新公司的首席执行官，而哈米迪是首席技术官。

这家总部位于旧金山的公司成立于 2020 年初，提供了一个 PaaS 平台，包括一个[开源](https://github.com/meroxa/cli)数据平面和一个控制平面，其中[变化数据捕获服务](https://en.wikipedia.org/wiki/Change_data_capture#:~:text=In%20databases%2C%20change%20data%20capture,taken%20using%20the%20changed%20data.)与 Apache Kafka 等技术和一套规则引擎相集成，可在几分钟内自动完成重复的工程任务并构建数据管道。它可以将数据流作为 API 端点公开，或者将它指向 webhook。

“我们站在巨人的肩膀上；我们利用了很多开源软件，”哈米迪说。

数据层使用 Kafka 和 Kafka Connect 等组件作为流数据管道的核心部分，以及 Kafka 生态系统的许多部分，如 Kafka proxy，来实现其 API 生成功能。它建立在他们所贡献的 Kubernetes 之上。

Hamidi 解释说，当客户要求平台提供管道或创建连接时，他们会与控制平面进行通信，控制平面会将请求转化为指令，然后将其发送到数据平面，数据平面会支持机器实现连接。

根据 Brown 的说法，Meroxa 提供了使用单一工具集实时大规模管理分析和运营工作流的能力。

数据管道是一组组件的集合，包括资源、连接器、流、端点等，允许您将数据从一个位置移动到另一个位置。

*   资源是数据库或服务。
*   连接器是资源之间的集成。它决定了数据如何传入或传出数据流。
*   每个源连接器将产生 JSON 格式的数据记录，这些记录将进入一个流。它会自动记录数据记录中有效负载的模式，并捕获其随时间的变化。
*   使用 gRPC 和 HTTP 公开的端点允许与流进行双向通信，并提供从资源推送和拉取的灵活性。可以使用端点以编程方式访问数据记录。

然而，布朗说，移动数据对公司来说是一个赌注。

“您需要知道的只是您的数据来自哪里，您希望它去哪里，以及基本上，当它到达那里时，您希望它是什么格式？”他说。“所以现在，如果你知道东西会连接起来，你就能建造什么，对吗？

它不是点对点的产品。您可以将实时流定向到多个目的地。

“您有这个实时流可用，所以您实际上可以获取相同的流并将其指向数据湖，您可以获取相同的流并将其公开为 gRPC API 端点，您可以获取相同的流并将其指向您的图形数据库。在通常的企业案例中，你必须有专家团队来做这些事情。但是你可以通过 Moroxa 的一个工具集来做到这一点，”Brown 说。

虽然它与该领域的许多其他公司重叠，如 [Fivetran](https://fivetran.com/) 、 [Stitch](https://www.stitchdata.com/) 和 [Confluent](https://www.confluent.io/) ，但是，根据 Brown 的说法，Meroxa 对实时流和向 API 端点公开数据的关注是使其与众不同的因素之一。当竞争对手处理流程的特定部分时，Meroxa 寻求成为单一的端到端工具。

“有许多竞争对手，但他们都是与基础架构并行的产品。我们的方法是，我们真的想成为你的核心数据基础设施的一部分，”他说。“我们希望客户和企业在我们提供的基础设施之上构建应用程序、新应用程序。因为我们正在创建这个组件，它基本上从任何地方获取数据，并将其置于实时流、实时事件格式中，这样您就可以开始进入该流，提取您想要的位，并将其连接到您想要的任何地方。所以这是我们处理事情方式的核心区别，”他说。

其客户将 Meroxa 用于以下用例:

*   用于分析和仪表板可视化的实时数据仓库同步。
*   将原始记录归档到数据湖中，用于模型训练/主动学习。
*   以编程方式从自定义服务的管道中侦听和接收数据。

它还将易用性、开发者体验放在心上。它的目标是让任何软件工程师或组织内的任何工程师扮演数据工程师的角色。

哈米迪说:“这个领域挤满了公司，但还有很多很多问题没有得到很好的解决。”。“数据卫生、高质量、数据传承等领域——我认为这些都是非常有趣的问题。但是数据转换、数据处理、流处理——这些领域仍然非常复杂，很难以任何可管理的方式大规模解决。”

因此，该公司在不久的将来的主要关注领域是创建同样易于使用的数据连接，将其应用于流处理和转换。它正在开发一款名为 Functions 的产品，以提供一个抽象层，使这种处理变得非常容易。

“我们学到的一件事，真的很难，就是大规模运行 Kafka Connect 非常困难。总的来说这是一个痛苦的过程。大规模管理 Kafka connect 非常困难。写卡夫卡连接词真的很难。是的，现在有很多开源组件。但是组件的质量确实变化很大。这就是我们关注的领域，”哈米迪说。

该公司最近筹集了由 Drive Capital 牵头的 1500 万美元的首轮融资，使该公司的总融资额达到 1910 万美元。

在一篇关于为该轮融资做贡献的帖子中，Amplify 合伙人[莎拉·卡坦扎罗](https://www.linkedin.com/in/sarah-catanzaro-9770b98/)指出，数据从业者通常需要四到八个工具来将数据移入和移出数据仓库。

“对于大多数数据从业者来说，阅读[实时数据应用]很难，因为他们漫不经心地盯着半打试图解开他们迅速增长的 DAG(有向无环图)的控制台。如果他们必须花几个小时仔细阅读 Django、Singer、Airflow 和 Docker 的文档，仅仅是为了驱动一个基于批处理的 BI 仪表板，他们如何竞争？”她问。

“与其他为了简单而牺牲速度和性能的批处理工具不同，Meroxa 易于使用的流数据平台将随着数据团队的发展而增长，因为他们构建的数据密集型应用程序可以与科技巨头的应用程序相媲美。基于分布式系统和数据管理的最新技术，Meroxa 使其用户能够拥抱机器学习系统在线的未来；仪表板每秒更新一次；快得惊人的服务依赖于多个专门的数据存储，”她写道。
