# 上下文:谁负责容器生态系统？

> 原文：<https://thenewstack.io/context-whos-charge-container-ecosystem/>

技术生态系统的理想是，从中受益最多的人——大概是顾客——与维护这个系统的生产者分享他们的劳动成果。供应方和需求方彼此受益，在这样做的过程中，不时地交换角色。这是我们在 20 世纪 70 年代末的 Apple II 上首次看到的现象——第一个致力于商业计算产品的社区驱动型经济。

一个可行的生态系统主要依赖于一种共同货币的交换:仁慈。这种美德通常在没有恶意的情况下茁壮成长。

然而，对于一个生产商来说，很难在其生态系统社区的成员中保持适当的仁慈水平，同时在竞争对手面前捍卫自己的利益，这一点码头工人非常清楚。此外，每当公司的自身利益变得明显时，贡献者就会变得怀疑。

这就是为什么公司通过宣扬平台不可知论的福音来先发制人。像[云铸造](https://www.cloudfoundry.org/)这样的项目或者像[集群总部](https://clusterhq.com/)这样的公司说，我们对市场决定的一切完全开放。

正如英特尔创始人[戈登·摩尔](http://www.forbes.com/profile/gordon-moore/)发现的那样，当市场控制了生态系统的价值结构时，保持现状的唯一方法就是以如此快的速度创新，以补偿产品不断下跌的市场价值。在开源商业模式中，价值适用于支持。如果支持的价值下降太多，业务就会停滞不前。因此，每个供应商的目标变成了确保其模型是受支持的模型——通常代替所有其他模型。一开始是共同兴趣的柏拉图式座谈会，现在变成了达尔文式的生存混战。正如我母亲常说的，“自食其果。”
