# 开源社区的力量

> 原文：<https://thenewstack.io/power-community-open-source/>

开源是基于自由的。这种自由包括获取源代码的自由，合作的自由，以及最终创新的自由。在开源中，没有一个人或公司拥有一个项目。开源是一种哲学，也是一种运动，让开源茁壮成长的是围绕它成长起来的社区。

## 这需要一个社区

代码本身不会导致这种情况发生。正如谚语所说，“养育一个孩子需要一个村庄”，创建一个健康的开源项目需要一个社区。开源生态系统中的所有参与者都有机会塑造和改进软件。用户可以识别他们需要的特性，并向上游贡献代码。每个人都有机会做出改变。

如果一个开源项目周围的整个社区都积极参与，那么它就最有可能成长为一个成功的生态系统。这包括来自代码提交者、用户、文档作者、软件供应商、平台供应商和集成商的每一个人。

## 多样性让我们更强大

在开源世界中，社区不是由物理位置定义的结构。它被定义为分享态度、兴趣和目标。就其本质而言，开源社区跨越了地理、政治背景、宗教和生活经历。没有公司、国家甚至语言的界限。来自各种背景、观点各异的人都可以参与进来。

这种多样性使一个健康的社区兴旺发达。思想的多样性、经验的多样性和观点的多样性为我们提供了合作解决问题的机会，从而使我们变得更加强大。

当人们突然沉浸在这种协作文化中时，有时会很困难。整个世界经常教导人们不信任或害怕他人。许多人都有一种习得的“我们对抗他们”的心态。交流并不总是容易的，更不用说协作了。我们不会都唱着 Kumbaya 然后神奇地相处并彼此认同。

这就是为什么我们在 Cloud Foundry 社区有一个[行为准则](https://www.cloudfoundry.org/code-of-conduct/)，要求人们尊重他人。我们的核心价值观是善良。我们鼓励社区成员在相互交流时友好相处。为了表示尊重。去倾听。要包容。我们努力让社区中的每个人都感到受欢迎，不管他们的背景如何，也不管他们贡献了多少。

在 Cloud Foundry，我们鼓励包容。在我们的会议上，我们邀请来自我们社区各个方面的演讲者，确保分享不同的观点。今年，在我们即将到来的硅谷峰会上，做主题演讲的女性实际上比男性多，遗憾的是，这在这个行业并不典型。

这些社区价值不仅对项目有益；作为一个人，它们对你有好处。学会与持有不同甚至相反观点和想法的人一起工作是一个实践宽容和包容，甚至拥抱差异的机会。“恐惧需要拒绝差异，”嘉旺德鲁克法王最近在佛教[三轮车杂志](https://tricycle.org/trikedaily/how-to-combat-fear/)上说。“多样性让我们更加强大。多元化不是可以容忍的，而是值得庆祝的。我们应该带着好奇、喜悦和喜悦来迎接它。”

除了开源对商业有益之外，我认为成为开源社区的一员对你个人也有好处。学会合作，倾听他人，拥抱多样性，可以让你成为一个更好的人。当你把善良作为指导原则时，它会让你反思你使用的语言和做出的承诺。它让你更加专注。当你不再需要总是正确的时候，你甚至会学习和成长。

## 协作 R&D 加强项目

多元化是推动行业发展的关键；它帮助我们培育一个可持续发展的开放社区，并使创新蓬勃发展。协作 R&D 是开源真正推动真正价值的地方。

开源促进了项目内部的协作，也促进了其他开源项目之间的协作。例如，在 Cloud Foundry 社区中，我们与 OpenStack、云计算原生计算基金会(CNCF)、Eclipse 基金会和许多其他组织有着良好的关系。我们去年宣布的[开放服务代理 API](https://www.openservicebrokerapi.org/) 项目是一种利用行业领先技术 Cloud Foundry Service Broker API 的方式，并使其可用于其他平台和社区。在这种情况下，多个群体的集体工作有利于他们所有人的更大利益。

当一个开源项目蓬勃发展时，它会吸引用户，随着时间的推移，其开发人员的集体基础可能会比非开源项目更大。有人可能会说，由于思想的多样性，开源比其封闭的、专有的对手更有优势。随着时间的推移，随着越来越多的人成为项目社区的一部分，加速解决问题的潜力会进一步扩大。当如此多的人投入到结果和围绕它的持续创新中时，社区主导的项目或平台就有更大的成功机会。

如果你想雇佣开发人员，请注意。开发者在决定是否加入一家公司时，将开源列为他们的驱动因素之一。你的公司在开源项目中的参与程度如何？开发人员希望成为更大事物的一部分；他们希望贡献并影响技术的发展方向。
