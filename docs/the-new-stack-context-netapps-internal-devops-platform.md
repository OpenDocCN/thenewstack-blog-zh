# 新的堆栈环境:NetApp 的内部 DevOps 平台

> 原文：<https://thenewstack.io/the-new-stack-context-netapps-internal-devops-platform/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。本周，我们采访了 [NetApp](https://www.netapp.com) 的 CloudOne 开发者体验 IT 高级经理[莫汉·拉吉](https://www.linkedin.com/in/mohanraj00/)。在过去的两年中，他一直在 NetApp 构建 DevOps 平台，为需要使用 NetApp 自有工具构建云原生应用的公司应用开发团队提供云服务、自动化和 CI/CD 发布模型。

TNS 编辑和营销总监 Libby Clark 主持了这一集，旁边是 TNS 创始人兼出版商 Alex Williams 和 TNS 执行主编 Joab Jackson。

[第 108 集:Mohan Raj—NetApp 内部 DevOps 平台](https://thenewstack.simplecast.com/episodes/episode-108-mohan-raj-netapps-internal-devops-platform)

在对 Mohan 的采访中，我们深入探讨了 NetApp 决定构建自己的 DevOps 平台背后的动力。正如他在播客中解释的:

*实际上，软件开发生命周期在不同的公司、不同的团队、不同的子团队之间会有所不同……所以当我们实际构建一个平台时，我们实际上必须以一种方式来考虑它，即[我们不能]交付的东西实际上是固定不变的，但有办法让它对单个团队来说有点可定制性，以便采用起来更加容易和自然。*

我们还讨论了本周的一些热门播客和新闻故事。对亚马逊网络服务的 Rohit Gupta 的采访提供了对 DevSecOps 新兴实践的见解。我们还采访了 Docker，了解其最新的以开发者为中心的路线图。我们花了一点时间[来欣赏云原生计算基金会的开放策略代理](https://thenewstack.io/open-policy-agents-mission-to-secure-the-cloud/)，这是一个开源平台，可以跨域和云原生堆栈中的所有层实施广泛的策略。最后，我们[进入亚马逊网络服务的 Bottlerocket](/amazon-web-services-releases-a-linux-distribution-built-for-containers/) ，一个新的基于 Linux 的开源操作系统，用来托管容器。

亚马逊网络服务、云计算原生计算基金会和 NetApp 是新堆栈的赞助商。
