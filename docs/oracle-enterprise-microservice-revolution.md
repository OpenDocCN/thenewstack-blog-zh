# Oracle 如何让企业为微服务革命做好准备

> 原文：<https://thenewstack.io/oracle-enterprise-microservice-revolution/>

当搜索平台服务或软件服务来帮助扩展运营时，技术经理经常发现自己必须确定他们选择的平台是否能够在公共云中以及在私有云上的公司防火墙后正确运行。这是[甲骨文](http://www.oracle.com/)通过[甲骨文云](https://www.oracle.com/cloud/index.html)平台和基础设施服务解决的问题，该服务提供了一整套托管硬件和软件，以在公司防火墙后复制甲骨文云技术。

“我们喜欢基于公共云的基础架构的想法。在过去几年中，我们一直在构建我们的公共云，我们说让我们确保相同的堆栈，相同的基础架构可供客户使用，我们在那里像 Oracle 一样管理它。甲骨文云平台和集成产品高级副总裁 [Amit Zavery](https://www.linkedin.com/in/amitzavery) 在最新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 中说道，该节目由 TNS 执行主编 [Joab Jackson](https://www.linkedin.com/in/joab-jackson-b6b108) 发布，并于上个月在纽约市[甲骨文云日](https://www.oracle.com/cloudday/index.html)录制。

[Oracle 如何让企业为微服务革命做好准备](https://thenewstack.simplecast.com/episodes/how-oracle-is-preparing-enterprises-for-the-microservice-revolution)

“这就像在您的数据中心内有一个迷你 Oracle 公共云，而使用 Cloud Foundry，他们为您提供一个框架，但他们不运行这些东西，他们不作为服务运营，他们没有在公共云和内部运行相同的版本。对于很多云代工厂来说，公共云服务现在还不存在，”Zavery 说。

为了帮助开发人员和运营团队更好地了解甲骨文云平台及其解决方案，甲骨文将于 2017 年在全球 20 个城市进行巡演。北美的下一次活动将于 3 月 1 日在旧金山举行，欧洲的首次活动将于 4 月 20 日在伦敦举行，亚太地区的首次会议将于 5 月 10 日在新德里举行。

“这背后的想法是让开发人员使用 Oracle 公共云，其中当然会有来自 JavaScript、Kubernetes 或 Docker 以及 Oracle 演讲者的讲座。将有动手实验室，他们可以尝试服务和建立一个应用程序。此外，许多演示集中在技术方面，”扎维里说。
