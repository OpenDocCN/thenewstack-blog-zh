# 为什么云本地开源对推特和 Spotify 至关重要

> 原文：<https://thenewstack.io/why-cloud-native-open-source-is-critical-for-twitter-and-spotify/>

根据最新统计，社交媒体巨头推特拥有约 3.53 亿活跃用户，流媒体音乐服务 T2 Spotify 拥有 3.56 亿活跃听众。在这两种情况下，云本地环境的开源工具和平台都是其巨大发展的基石。

在最新一集的新堆栈制造商播客中， [Spotify](https://open.spotify.com/) 高级员工工程师 Dave Zolotusky 和 Twitter 开发者体验主管兼工程有效性经理 [Jasmine James](https://twitter.com/gojasmineee) 讨论了开源软件在各自组织中的角色。[云本地计算基金会](https://www.linkedin.com/in/katie-gamanji/?originalSubdomain=uk)[的生态系统经理凯蒂·加曼吉](https://cncf.io/?utm_content=inline-mention)和 TNS 创始人兼出版商亚历克斯·威廉姆斯共同主持了本次采访。

[为什么云本机开源对推特和 Spotify 来说至关重要](https://thenewstack.simplecast.com/episodes/why-cloud-native-open-source-is-critical-for-twitter-andspotify)

两种服务的一个共同点是: [Kubernetes](https://thenewstack.io/category/kubernetes/) 。

“开源在内部有很多创新，我能举出的一个例子——实际上是一个 CNCF 项目——就是库本内斯，”詹姆斯说。“每个人可能都熟悉这一点——我们正在经历一场大规模迁移，我正在帮助加速这场迁移，所以我想说，这可能是我现在能谈论的最大的例子。”

Zolotusky 解释说，Spotify 几年前开始迁移到 Kubernetes，今天它的大量后端操作都运行在 Kubernetes 上。“我们花了很多年时间自己构建各种类型的基础架构，包括容器编排。很明显，这些事情都没有像库本内特那样发展。所以，从这个意义上说，我们正在尽最大努力停止构建我们自己的东西，使用更多的开放源代码或者至少是行业标准，”Zolotusky 说。

开源工具和平台在两家公司如何扩展服务方面发挥着关键作用。詹姆斯说，以 Twitter 为例，“规模总是话题的焦点”。“我认为，我们总是喜欢考虑的一件事是，我们如何真正根据指标或监控或类似的事情轻松进行扩展，这样我们就不必手动进行扩展。因此，很明显，将应用程序容器化，并拥有非常专注的微服务，如果在特定时间不需要它们，可以轻松增长或停止，这是关键，”詹姆斯说。“因此，我们有一个非常好的流程来设计这些服务，并确保这一点得到考虑，我们有一个非常强大的技术设计流程，所有应用程序都要在内部通过，以确保这是我们可以不断考虑的事情。”
