# 探索 Kubernetes 的抽象

> 原文：<https://thenewstack.io/exploring-kubernetes-abstractions/>

[#158:探索库伯内特的抽象概念](https://thenewstack.simplecast.com/episodes/158-exploring-kubernetes-abstractions)

在今天这一集的[New Stack Analysts](https://thenewstack.io/podcasts/analysts)播客中，TNS 创始人 [Alex Williams](https://thenewstack.io/author/alex) ，TNS 记者 [T.C. Currie](https://thenewstack.io/author/tc/) 和[Janakiram and Associates 首席分析师 Janakiram MSV](https://www.linkedin.com/in/janakiramm/) ，以及 Heptio 首席技术官和 Kubernetes 联合创始人 [Joe Beda](https://www.linkedin.com/in/jbeda/) ，以及 Bitnami Kubernetes 技术主管 Sebastien goas guen。本周的讨论集中在 Kubernetes 开发人员可用的许多抽象，以及这些抽象如何影响大大小小的开发团队。

“我所看到的是，在 Kubernetes 之上引入另一个抽象层的全部努力，以鼓励用户、初学者，甚至大型企业 IT 团队在不了解 Kubernetes 证书的具体细节的情况下，以 Kubernetes 为目标，”MSV 说。

MSV 解释说，两个流行的播放器使得更好地运行和管理 Kubernetes 工作负载成为可能，它们是 [Helm](https://helm.sh/) 和 [Spinnaker](https://www.spinnaker.io/) 。虽然 Spinnaker 是“在虚拟机中安装的噩梦”，但 MSV 接着指出，Helm 使这一过程变得快速而简单。

“人们真的试图拥有一个针对他们特定用途的工具，”Goasguen 补充道。

在后面的讨论中，Beda 提到了 Linux 内核和 Kubernetes 本身之间的相似之处，指出集群专门化是 Kubernetes 核心的目标，用户节点根据用户的使用情况而有所不同，能够根据用户前面提到的使用情况专门化集群。
