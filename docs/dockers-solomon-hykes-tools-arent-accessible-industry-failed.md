# Docker 的 Solomon Hykes:如果工具不可用，我们的行业就失败了

所罗门·海克斯是 Docker 的创始人、CTO 兼首席架构师，也是 Docker 开源倡议的发起人。在 Docker 工作期间，他致力于为开发人员和系统管理员构建一个平台，用于构建、发布、运行和编排分布式应用程序。](http://docker.com) [](http://docker.com)

对于 [Docker](http://docker.com/) 的创始人兼首席执行官 [Solomon Hykes](https://twitter.com/solomonstre) 来说，如果整个用户社区都参与进来，IT 管道就能发挥最大的作用，开发者可以随心所欲地修改它，并将结果提交回存储库。尽管一些组织回避开源他们的代码，但 Docker 拥抱所有的贡献——不管这样的事实，一些由此产生的创造有一天会成为它的竞争对手。

在本期 [The New Stack Makers 播客](/tag/the-new-stack-makers/)中，New Stack 创始人 Alex Williams 与 Hykes 进行了交谈，听听他对规模化工作时的集群管理、协调以及 Docker 如何帮助改善开源社区的看法。这个采访来自最近出版的 TNS 电子书"[自动化&与 Docker 和 Containers](https://thenewstack.io/ebookseries/) 的编排。

“基本上，我们说的是没关系，我们都受益。Docker 的好处是，如果你要构建其他东西，而不是使用相同管道的 Docker，那么管道会在过程中变得更好。因此，Docker 会变得更好，”Hykes 说。

扩大用户群体是 Docker 公司的使命之一。

虽然总会有人有不同的想法，但目前学习编程有一个门槛，使得各种各样的人都无法学习。如果你对一个应用程序或服务有一个有趣的想法，但没有一定的编程经验，开发软件不仅令人生畏，而且是不可能的。Hykes 解释说，目前有各种各样的项目正在进行，旨在使编程更容易理解，而 Docker 希望使用于创造新事物的工具也容易理解。

“要编程，你需要工具。Hykes 说:“如果这些工具没有被设计成对所有专业水平的程序员都是可访问的和有用的，那么作为一个行业，我们就失败了。

构建软件通常可以比作创建一个庞大的、相互连接的系统网格，比如城市网格。Hykes 接着强调了 Docker 如何帮助开发人员发挥他们的作用，同时改善他们在城市基础设施中的特定建筑。
