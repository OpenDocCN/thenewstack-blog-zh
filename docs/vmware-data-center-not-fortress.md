# VMware 提议在虚拟化层保护数据中心运营

> 原文：<https://thenewstack.io/vmware-data-center-not-fortress/>

尽管外表粗糙，但数据中心可能充满潜在的安全威胁。VMware 正在制定加强数据中心系统安全性的方法，使用来自其 [NSX 虚拟化平台、](http://www.vmware.com/products/nsx/?cid=70134000001M7Iy&src=ps_56267f2b882cc&kw=vmware%20nsx&mt=e)的网络流量过滤技术以及应用层的数字签名。

“数据中心不是堡垒，”VMware 产品管理总监 Gargi Mitra Keeling 说。

甚至在五年前，加密数据中心内的流量还不是问题。“如果你能实际接触到数据中心，你是可信的，”她说。但是时代变了。存在特权用户攻击，导致组织内部的安全漏洞。Keeling 补充说，这些攻击可能来自虚拟机、移动设备、任何地方。一旦在数据中心站稳脚跟，攻击者就会发现很容易扩展到其他资源。

VMware 一直在研究如何围绕虚拟化堆栈本身的基础架构来保障数据中心的安全性，并借鉴了一个名为[微分段](http://virtualizeyournetwork.com/vmware-nsxs-top-security-benefit-micro-segmentation/)的概念。虚拟化层的微分段使用零信任防火墙隔离网络级服务之间的通信。安全策略可以在虚拟机(VM)级别设置，并在虚拟机管理程序上实施。

除了防火墙应用程序之外，虚拟化平台还可以为企业提供另一个长期追求的安全优势，即提供实现系统范围加密的基础。虽然 VMware 还没有具体的产品来轻松实现这种加密，但该公司对新的堆栈进行了讨论，希望围绕这种方法引发讨论。

## 签署网络流量

当然，加密数据和通信将使数据中心更不容易受到欺骗和其他恶意攻击。但是许多公司都在努力加密和解密数据中心内的服务链。通常情况下，它没有完成，或者没有正确完成，因为这是一个令人头疼的操作问题，尤其是在大规模运行时。

“开发人员将把 SSL(安全套接字层加密技术)放在应用层，但将使用共享证书，通常不轮换它，或使用相同的分层证书。这意味着如果一个应用程序受到威胁，任何使用 SSL 证书的应用程序也会受到威胁，”Keeling 说。

网络虚拟化可以简化这一过程。

VMware 提议改变读取数据包的方式。今天，假冒数据包很简单，而且有开源工具允许恶意用户注入数据包和欺骗 IP-Mac 层映射。如果攻击者了解映射是如何工作的，那么欺骗数据包是相当容易的。标准交换机不会验证被欺骗的数据包，而会让它通过。

借助 VMware 提出的解决方案，数据包将由 NSX 平台进行签名和验证。注入的恶意数据包不会有此标识符。数据包接收方寻找签名，如果找不到就丢弃数据包，”基林说。

例如，虚拟机管理程序 A 上的虚拟机 A 向虚拟机管理程序 B 发送一个数据包，虚拟机管理程序 B 会注意到该数据包确实来自虚拟机管理程序 A，然后该虚拟机会继续检查签名。如果数据包被注入，那么它将缺少正确的签名。

对于那些想要高级防火墙功能的组织，NSX 平台还允许第三方防火墙插件，这些插件是由 Palo Alto Networks 和 F5 等公司开发的。

微分段方法提供了一种锁定多层应用程序的所有许多元素的方法，以这种方式可以集中管理安全实施。对于 VMware 堆栈的用户来说，这种方法有助于简化保护数据中心的棘手工作。
