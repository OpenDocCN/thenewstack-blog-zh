# MongoDB 占据思想份额

> 原文：<https://thenewstack.io/mongodb-dominates-mindshare/>

编者按:趋势与使用相关吗？这是南希·戈林在这篇文章中提出的问题。

本周，围绕 MongoDB 有很多传言，谷歌和微软都在采取措施更好地支持开源的 NoSQL 数据库技术。这篇 [Flux7 博客文章](https://www.flux7.com/blog/whats-trending-with-devops-tools-a-study-using-google/ "Flux7")给了我灵感，让我看看 MongoDB 以及其他基于网络搜索的云数据库技术日益流行的情况。

虽然网络搜索与技术的实际使用并不相关，但它们确实表明了人们的兴趣，而这种兴趣很可能转化为使用。

我使用 Google Trends 绘制了一段时间内 MongoDB 和竞争技术的搜索图表。毫不奇怪，人们对 MongoDB 的兴趣正在急剧上升。

然而令人惊讶的是卡珊德拉很少出现。这可能是因为人们可能不会搜索“阿帕奇·卡珊德拉”然而，对“卡珊德拉”的图形化搜索包括了各种不相关的搜索。今天刚刚获得[6000 万美元融资](http://www.couchbase.com/press-releases/couchbase-secures-60-million-funding-capture-larger-share-16-billion-big-data-market "Couchbse")的 Couchbase，与 MongoDB 相比同样微不足道。

MongoDB 的 mindshare 领先令人印象深刻，但这并不是说其他人无法生存。云大数据工具的发展还处于初级阶段，未来几年肯定会有很多变化。

我还比较了对最流行的 Hadoop 发行版和服务的兴趣。Cloudera 吸引了最多的兴趣，稳步增长，其次是 Hortonworks。我把“Amazon EMR”和“Amazon Elastic MapReduce”都包括在内，并惊讶于这两个词的搜索量如此之少。一路走来，MapR 吸引了稳定的兴趣。

我想比较我能想到的两个最大的 NoSQL 数据库服务也可能是有趣的，因此将亚马逊的 DynamoDB 与谷歌的云数据库进行了对比。DynamoDB 获得了源源不断的兴趣，当亚马逊增加功能时，兴趣激增，而谷歌的数据存储似乎几乎没有注册。

我可以期待更多种类的数据库服务出现，来自领先的云供应商以及新兴的服务提供商。

最后，只是为了好玩，我绘制了一组基础设施服务提供商(我会包括更多，但 Google Trends 只允许您比较五个)。我很好奇“其他人”会如何比较——换句话说，除了 AWS、Azure 和谷歌之外的所有人。我随机选择了五家，排除了大型传统企业供应商以及前三名。

我对数字海洋的结果表示怀疑，主要是因为趋势线可以追溯到 2005 年。谷歌快速搜索显示，有几个名为数字海洋的组织。软蛋作为强势的二把手让我大吃一惊。如果你用 Rackspace 取代 Digital Ocean，你会看到 Rackspace 击败了其他公司，尽管与其他公司相比，它的收入相对较差。
