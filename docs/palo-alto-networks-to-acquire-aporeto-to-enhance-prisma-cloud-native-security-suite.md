# Palo Alto Networks 将收购 Aporeto，用于其不断增长的 Prisma Cloud 原生安全套件

> 原文：<https://thenewstack.io/palo-alto-networks-to-acquire-aporeto-to-enhance-prisma-cloud-native-security-suite/>

两家公司周三宣布，企业软件安全提供商帕洛阿尔托网络公司(Palo Alto Networks)已经开始以大约 1.5 亿美元的价格收购基于身份的安全机器软件提供商(T2)apore to(T3)。该收购预计将在 3 月完成，前提是通过通常的成交条件。

帕洛阿尔托最近一直在积极收购云原生安全公司。五月，它[收购了 Twistlock 和 PureSec](/cloud-native-security-consolidation-palo-alto-networks-buys-twistlock-puresec/) ，并将两家公司的技术整合到自己的 [Prisma cloud 原生安全平台](https://www.paloaltonetworks.com/prisma/cloud)。Twistlock 提供了一个平台，用于在整个应用程序生命周期中管理漏洞管理和合规性，并包括加密和扫描图像等功能，以满足安全性和合规性需求。PureSec 为无服务器工作负载提供运行时保护。

Aporeto 的微分段技术可能非常适合这个不断增长的云原生安全堆栈。公司[为 Kubernetes](/aporetos-kubernetes-security-platform-now-offers-multiregion-cluster-support-service-mesh-integration/) 开发了一个安全套件，包括多集群和多区域支持，以及多服务网格集成支持。

Aporeto 联合创始人 Dimitri Stiliadis 在解释该公司技术的博客文章[中指出，使用 IP 号码和用户认证的传统访问控制工具无法提供足够的背景来评估是否应该授予敏感工作负载的访问权限。](/trireme-brings-much-needed-context-security-ops/)

“当在两个实体之间进行通信尝试时，如果有足够的关于实体身份的可验证上下文，那么安全性可以以更高的置信度做出合理的决策。这避免了假阴性或假阳性，并对什么可以做什么不可以做进行了严格控制，”他写道。

Palo Alto Networks 成立于 2005 年，向市场推出了第一款状态检测防火墙，并在随后的几年中发展了其防火墙功能。随着越来越多的企业客户转向云，该公司一直在建立一个支持零信任安全模型的产品组合，该模型不依赖防火墙来实现安全，而是依赖于大量的用户和终端设备身份验证。在 3 月份结束的 2020 财年，该公司预计将产生高达 34.8 亿美元的收入。
