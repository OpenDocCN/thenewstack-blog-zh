# IAG 发现开源是未来最好的保障

> 原文：<https://thenewstack.io/open-source-best-insurance-future-eddie-satterly-talks-iag/>

[开源是未来最好的保险:埃迪·萨特利谈 IAG](https://thenewstack.simplecast.com/episodes/open-source-is-the-best-insurance-for-the-future-eddie-satterly-talks-about-iag)

[澳大利亚保险集团](https://www.iag.com.au/) (IAG)是澳大利亚最大的保险公司之一，当然，它也有大量的 it 足迹。IAG 集团内部运行着超过 147 个核心保险系统。其中大多数是像 AS/400 这样的中端大型机系统。

大约两年前，该公司决定改变现状。该公司任命长期高管朱莉·巴奇为首席客户官。她创办了一个名为客户实验室的组织来参与开源，这是该公司在其 140 年的历史中从未做过的事情。

作为这项计划的一部分，该公司询问 IAG 产品工程师主管 Eddie Satterly，他是否可以带着他的团队帮助客户实验室建立一个具有内部源代码程序的新文化。目标是开始采用开源方法和工具。

但是一旦他们被[开源 bug](https://github.com/iagcl) 咬了一口，事情就发生了有趣的转折。这项工作导致 IAG 实际上开源了自己的技术，[数据管道](https://github.com/iagcl/data_pipeline)，一个用 Python 进行 ETL(提取、转换和加载)的工具。

“IAG 是一家非常保守的公司……并且在高度监管的行业中运营,”Satterly 在最新一期的[新堆栈制造商播客](https://thenewstack.io/podcasts/makers)中解释道，该播客是上个月在悉尼 OpenStack 峰会上录制的。“开源他们的技术非常重要；对他们来说，这是一个巨大的变化，是一个飞跃。”

通过转向商用硬件和开源软件，IAG 已经节省了数百万美元。开源允许他们扩展，这是遗留系统不可能做到的。

萨特利讨论了他是如何帮助 IAG 改变其文化的；IAG 如何从传统基础设施转向 OpenStack 云；该公司如何使用开源机器学习技术等等。

简而言之，这只是一个开始，现在 IAG 已经享受到了 OpenStack 和开源软件的好处。IAG 正在考虑将更多的数据从旧系统迁移到新系统。该公司正在考虑将 OpenStack 用于其一些核心和关键平台。它正在寻找软件定义的存储来取代昂贵的存储区域网络。

“他们在吃饭。他们希望参与进来，因为他们看到了自己获得的性能优势。他们看到了自己获得的新能力，”萨特利说。“它将继续增长和渗透。”
