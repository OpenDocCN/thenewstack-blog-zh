# Secure seL4 微内核寻求围绕新基础构建生态系统

> 原文：<https://thenewstack.io/secure-sel4-microkernel-seeks-to-build-an-ecosystem-with-a-new-foundation/>

seL4 微内核，世界上第一个被数学证明为"[安全](https://ipads.se.sjtu.edu.cn/zh/publications/MiEuroSys19.pdf)的操作系统内核，现在有了自己的基金会，seL4 基金会，它是由 [Linux 基金会](https://www.linuxfoundation.org/)和 [Data61](https://data61.csiro.au/) 一起建立的，后者是澳大利亚[联邦科学与工业研究组织](https://www.csiro.au/) (CSIRO)的数字专家部门。

本质上，seL4 微内核是没有错误的，并且在数学上被证明是安全的，给定一些关于硬件和实现的假设。虽然 seL4 微内核自 2006 年就已经出现，但由于缺乏生态系统，它的采用还没有发挥出全部潜力。

“在过去的 10 年里，seL4 开始起飞，许多组织开始感兴趣，但现阶段真正缺少的是围绕它的生态系统，”data 61 trust worthy systems 的负责人 June Andronick 说。“我真正期待的是降低采用 seL4 的门槛，真正看到生态系统以一种人们毫不犹豫地转向 seL4 的方式发展。我们看到许多组织都在为他们的下一个产品或下一代产品寻找最佳解决方案，他们被推向 seL4 的方向，但他们希望看到长期的支持。他们希望看到它背后有一个支持它的社区。”

与此同时，seL4 基金会主席 Gernot Heiser 解释说，微内核已经在许多安全性至关重要的情况下被采用。一个例子是波音公司制造的自主直升机，它首先使用 Linux，“官方[【渗透】测试人员](https://www.imperva.com/learn/application-security/penetration-testing/)花了大约两周时间才完全拥有它。”Heiser 说，用 seL4 微内核替换 Linux，然后用 seL4 强制隔离将关键组件提取到受保护的沙箱中，这被证明是难以理解的。

与 Heiser 所说的可能“包含成千上万个错误”的 Linux 内核相比，seL4 微内核非常小，基本上没有错误，最小的时候只有不到 10，000 行代码，最大的时候有 18，000 行代码。大小因运行它的硬件而异。seL4 能够保持如此之小的部分原因是它将许多工作留给了用户，Heiser 解释说“seL4 所做的事情的一个极端观点是基本上安全地多路复用硬件资源。”

从开源的角度来看，Heiser 也认为 seL4 的这一特性很有吸引力。虽然为 Linux 内核做贡献可能很困难，并且需要遵循非常严格的过程，但是为 seL4 生态系统做贡献限制较少，因为代码是孤立的，并且存在于经过验证的内核本身之外。

“我们在基于 seL4 的系统中拥有的许多功能都是作为用户级组件运行的。像任何应用程序一样，从内核的角度来看，设备驱动程序、网络堆栈、文件系统或应用程序之间没有区别——它只是程序，这是独立开源开发的一个伟大的推动者，”Heiser 说。“在 seL4 中，因为它是一个独立的用户级组件，所以人们可以有很大的自由度来开发和贡献这些东西，我们不需要保持如此紧密的控制。”

与 Windows 和 Linux 非常相似，Heiser 还解释说 seL4 是用 C 编写的，但它的不同之处在于，它是 C 的子集，以便保持内核可证明的安全性。

“我们不得不远离一些 C 语言的特性，因为 C 语言的语义含糊不清。它没有很好的定义，所以我们必须对语言进行一些限制，但大多数限制只是常识，”海泽说。“你在 Windows 和 Linux 中看到的许多漏洞实际上是 C 语言不安全的结果——溢出的缓冲区和数组、取消引用未初始化的变量和空指针是典型的攻击媒介。原则上，我们也有这些东西，但是我们明确证明所有这些访问都是安全的。我们有证据表明，我们的内核从来不会取消对空指针的引用，也不会越界访问数组。”
