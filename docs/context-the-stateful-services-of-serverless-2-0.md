# 上下文:无服务器 2.0 的有状态服务

> 原文：<https://thenewstack.io/context-the-stateful-services-of-serverless-2-0/>

雷根赞助了这个播客。

[无服务器 2.0 的有状态服务](https://thenewstack.simplecast.com/episodes/the-stateful-services-of-serverless-2-0)

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云原生社区本周的新闻和观点。在本周的节目中，我们的话题都是关于无服务器计算的。

本周，我们采访了实时系统提供商 [Lightbend](https://www.lightbend.com/) 的创始人兼首席技术官 [Jonas Bonér](http://jonasboner.com/) ，他的公司正在做一些实验性的工作，为通用的有状态应用程序创建一个无服务器界面。Bonér 上个月在 Barcelona 的 KubeCon+CloudNativeCon 做了一个名为“[无服务器很有趣，但 FaaS 还不够](https://www.youtube.com/watch?v=J3PyYmdTsnQ)”的演讲，讨论了这个话题。

今年 2 月，Bonér 在 New Stack 上写了一篇文章[呼吁创建下一代无服务器技术，他也称之为“无服务器 2.0”或有状态无服务器。“无服务器的编程模型仍然主要局限于无状态功能——所谓的功能即服务(FaaS)模型——这限制了支持的用例范围，”他在那篇文章中写道。](https://thenewstack.io/serverless-needs-a-bolder-stateful-vision/)

在本播客中，我们将讨论从 [OpenFaaS](/openfaas-put-serverless-function-container/) 和 Kubernetes 生态系统中的其他开源项目酝酿的“无服务器 2.0”运动。我们还讨论了云原生社区对下一阶段无服务器的定义，以及无服务器 2.0 的构成。

然后在节目的后半部分，我们将讨论本周的热门播客和故事。

《新书库》本周的节目由《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持，TNS 奥克兰记者 T21·t . c .柯里为其提供协助。
