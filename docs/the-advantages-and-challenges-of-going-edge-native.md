# 走向“边缘本地”的优势和挑战

> 原文：<https://thenewstack.io/the-advantages-and-challenges-of-going-edge-native/>

随着互联网充斥着我们生活的每一个角落，对于开发者、操作工程师和雇佣他们的组织来说，它变得更加复杂。你如何减少延迟？您如何遵守您拥有虚拟存在的每个地区或国家的法规？如何将数据保存在实际使用的地方？

对于越来越多的组织来说，答案是*利用优势*。

在这一集的 New Stack Makers 播客中， [Cox Edge](https://www.coxedge.com/?utm_content=inline-mention) 的总经理 [Ron Lev](https://www.linkedin.com/in/ron-lev/) 和[Cox Edge 的产品、营销和客户成功负责人 Sheraline Barthelmy](https://www.linkedin.com/in/sheraline-barthelmy-0625b870/) ，Macrometa 的创始人兼首席执行官 [Chetan Venkatesh](https://www.linkedin.com/in/chetanvenkatesh/) 也加入了进来。三人组讨论了[边缘计算](/category/edge-iot/)的最佳用例、它可以带来的优势以及仍然存在的挑战。

播客由《新堆栈》的特写编辑希瑟·乔斯林主持。

[“边缘土著”的优势与挑战](https://thenewstack.simplecast.com/episodes/the-advantages-and-challenges-of-going-edge-native)

边缘是由物理位置靠近用户的服务器组成的——由像 Cox 及其竞争对手这样的本地互联网服务提供商提供的“最后一英里”。

Venkatesh 在本期播客中指出，对于开发者来说，它提供了“一个额外的非常有趣的新的前卫基础设施来构建和运行他们的应用程序。”

边缘的优势包括潜在的延迟减少，因为数据到达用户的距离更短。“你可以使用云运输大量产品，就像船只可以运输大量产品一样——但速度很慢，”文卡特什说。

“从 A 点到 B 点需要几天、几周、几个月。尤其是在洲际距离上，网络有点像这样。只不过我们谈论的是以秒和毫秒为单位的计时。”

他说，边缘计算很像航空旅行引入航运业——缩短旅行时间，更快地为用户服务。他称之为“云计算的亚马逊总理 T21”

Macrometa 提供了一个开发者平台，帮助开发者创建“edge 原生”应用和 API。为了确保为边缘构建的应用符合它们可能运行的地方的法规，适当的规则和策略被绑定到数据。

Barthelmy 说，边缘计算的采用有助于刺激新技术的创新，这将有助于解决开发者的问题。

她说，特别是两个用例提供了希望:零售和金融服务。“在零售业，它打开了巨大的机会，”巴塞尔米说。例如，“他们现在可以为店内购物者提供实时的上下文推荐，就像我们在网上做的一样。”

她说，在金融服务领域，边缘计算还可以帮助个性化客户体验，利用客户在线活动的数据来找出他们感兴趣的内容，并相应地定制他们的产品。

Lev 承认，商业边缘计算的时代是新的，一些挑战仍在解决中。他说，到目前为止，“优势是一种稀缺资源。”对于运营边缘数据中心的公司来说——比如为 1000 万家庭提供互联网服务的 Cox——存在规模问题。

"我们能解锁更多的位置吗？"他问道，提出了所有边缘提供商都在努力解决的问题。“我们如何在不运营网络的情况下，跨越不同的足迹进行扩张？”

Venkatesh 说，对于开发者和他们的组织来说，“挑战在于还不太了解边缘计算可以解决哪些令人惊讶的问题。”
