# 视频教程:SAP 的 OpenUI5，一个开源的 JavaScript UI 库

> 原文：<https://thenewstack.io/video-tutorial-saps-openui5-an-open-source-javascript-ui-library/>

在 OSCON 2015 之前，SAP 开发者传道者 Alexander Graebe 展示了开源的 JavaScript UI 库，由 SAP 维护，在 Apache 2.0 许可下可用。

[https://www.youtube.com/embed/BXdKitDTQDw?feature=oembed](https://www.youtube.com/embed/BXdKitDTQDw?feature=oembed)

视频

这个项目的基础是一个用于构建企业级 web 应用程序的工具，能够响应所有设备，运行在几乎任何浏览器上。Alexander 从显示 OpenUI5 的 [GitHub 页面开始，它从 2014 年 10 月就在那里了。](https://github.com/SAP/openui5)

“一开始它被称为‘SAP UI5’“它只对内部客户开放，但现在我们正在开源，”亚历山大说。我们为这个特定的库使用了开源技术，我们想回馈社区。"

OpenUI5 专注于企业级应用，所以它采用了内部化、路由、可扩展性、主题化、模板化等等。“我们正在考虑不同的用例，这些用例可能是你在启动一个非常小的网络应用时没有考虑到的，”Alexander 说。他指出，OpenUI5 还允许创建不同格式的应用程序，包括 XML、HTML、JavaScript 和 JSON。

然后他介绍了代码示例，展示了一个非常简单的 OpenUI5 应用程序，它从 GitHub API 加载数据，使数据在新的 UI5 应用程序中可访问，然后在 UI 中显示。

(示例位于:http://jsbin.com/poyozu/1/edit?html，输出)

“只是为了让你对 UI5 有一个大致的了解，”Alexander 继续说道，“它实际上只是一个包含在 HTML 文件中的 JavaScript 库，然后你可以从中创建你的 UI5 应用程序。”

他指出了关于主题化和库类型的脚本标签，特别是“placeAt”标签，亚历山大说，“这是魔法发生的地方。”

“JavaScript 库本质上像一个框架一样工作。我们告诉该库应该如何创建 UI 组件或 UI 元素，然后基于此，它会计算出它在哪个平台上运行，在哪个设备上运行，然后它会自动生成代码，并将其放入特定的 HTML 元素中。”

OpenUI5 包含双向数据绑定，也可以配置为单向数据绑定。Alexander 解释说，有了双向数据绑定，无论何时代码端发生变化，UI 也会自动更新。

他指出了基于模板的模块化列表层次结构，提到列表可能是压倒性的，“有这么多不同的列表，有时它们看起来相似，有时它们看起来不同。”

“我们试图让使用简单用例的人快速入门，但如果您有复杂的用例，您可以切换模板，然后您可以显示更详细和复杂的信息，”例如重新组装对象状态、数量和货币。

SAP 是新堆栈的赞助商。
