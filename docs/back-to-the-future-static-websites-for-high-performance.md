我们经常看到网站崩溃的报道，因为人们上网，无法访问信息，因为网站的内容管理系统(CMS)跟不上流量的激增。随着如此多的访问者突然从卫生部门、政府、诊所、公用事业服务和食品配送部门在线寻求重要信息，提高网站性能以处理不断增长的信息量变得至关重要。

三分之一的互联网网站仍然使用第一代内容管理系统，如 WordPress 和 Drupal。这些传统的 CMS 是单体系统，包括后端用户界面、插件、前端模板、级联样式表(CSS)、web 服务器和数据库。对于每个用户对网站页面的请求，服务器首先查询数据库，并将结果与来自页面标记和插件的数据相结合，以在浏览器中生成 HTML 文档。

现在是时候回归基础，回归静态网站设计了。历史上的第一个网站，由蒂姆·伯纳斯·李于 1990 年创建，是一个静态网站的例子，它只包含简单的 HTML，仅代表文本和链接。

静态站点提供内容，不需要后端数据库层。它们消耗很少的计算资源，所以加载很快，因为没有数据库查询，没有模板渲染，没有客户端-服务器请求处理。如今网站设计越来越受欢迎的一种流行模式是使用 API first 或 headless CMS (Strapi，Contentful)，非技术用户也可以使用，连接到静态站点生成器(Gatsby，Hugo，Jekyll)和/或 JavaScript 框架(React，Next.js，Node.js)。这被称为 JAMstack 架构。

WordPress 推出时的流行是由于它的 GUI 界面，让用户能够在管理文件和文档的同时设计和显示内容。当时，还没有适合非技术用户的静态站点生成器。现在，在引入这些 CMS 大约 20 年后，整体架构变得难以维护，通常很慢，并且易受黑客攻击。

我们看到了无头 CMS 的出现，它引导 API 来支持跨所有类型的设备和渠道的内容分发。Headless CMS 是 JAMstack web 开发架构的一部分，该架构正在成为现代 web 开发人员的流行解决方案，提供更好的 web 性能，也有助于更好的 SEO 排名和安全性。JAMstack 不依赖于 web 服务器，并在发出请求时立即提供静态文件。不需要查询数据库，因为文件已经被编译并提供给浏览器。

因此，似乎对高性能网站的迫切需求正把我们带回到静态网站的未来…但这一次有所不同——为开发者和非技术用户提供更大的灵活性和适应性。这是一个受欢迎的变化，由于新冠肺炎疫情，这可能在短期内非常有用，此外，我们将看到更高性能网站的长期好处，这些网站能够更好地处理不断增长的访问量。
