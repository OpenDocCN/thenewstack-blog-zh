# 机器学习的挑战以及 DevOps 和 Edge 将如何使数据科学现代化

> 原文：<https://thenewstack.io/the-challenge-of-machine-learning-and-how-devops-and-the-edge-will-modernize-data-science/>

首先是敏捷。然后是 DevOps。只不过他们只去了几个部门，主要是传统的 IT 部门。是什么阻止他们到达最极端的技术领域？据 [Dotscience](http://www.dotscience.com) 的首席执行官兼创始人卢克·马斯登称，企业级人工智能(AI)和机器学习正在受到阻碍，部分原因是其背后的数学家和统计学家仍停留在瀑布时代——仍在相互发送代码。

用于端到端机器学习数据和模型管理的协作工具 Dotscience 最近对企业进行了一项调查。由此产生的《2019 年 T4 人工智能应用开发和运营状况报告》包括 63.2%的企业报告他们在人工智能工作上的支出在 50 万至 1000 万美元之间。但 60.6%的受访者继续面临运营挑战。

> “挑战在于，如果你已经知道你的机器学习在生产中预测的事情的答案，那么你就不需要模型，因为你已经知道正确的答案。”——卢克·马斯登，网络科学。

基本上大多数企业 AI 和机器学习计划都被堵在门口。在这一集的《新堆栈制作人》中，我们将与马斯登探讨其中的原因。

他解释说，机器学习本质上比编码或编码、集成、部署和监控加数据更复杂。所有这些编码步骤加上多个版本的多组数据，加上指标，加上超参数，加上不同的模型。涌入机器学习的数据变得陈旧的速度要快得多。

“这更像是一个记录你所做的事情的问题，”马斯登解释道。

数据科学家通过来回传递文件名略有不同的 Word 文档来跟踪数据。当你刚刚开始学习机器时，马斯登并不认为这很糟糕。但是当你试图扩展一个团队的时候，它就崩溃了，在这个团队中每个人都使用不同的约定，当你在产品中有很多模型的时候，情况就更糟了。

Dotscience 希望通过数据版本和出处(数据的来源)来解决这个问题。不仅是数据的来源，还有模型的来源，这可能只是另一个过程的输出。这些都是用 Dotscience 可视化绘制的，所以你可以了解谁输入了哪个数据版本，哪个数据集正在训练哪个机器学习模型。

“如果你不能从生产中的模型可靠地说出它是根据哪些数据进行训练的，那么你就没有希望能够跟踪模型中的偏差，你就有点被蒙在鼓里了，”马斯登说。

当我们面对大量本质上有偏见的数据时，这一点至关重要。但是想象一下，当第一起无人驾驶汽车造成的死亡事故发生时。数据版本化可能是跟踪谁训练了该模型，哪些数据集进入了该模型，以及进入该模型的人类影响是什么的最佳方式。因为，[正如我们从大众](https://thenewstack.io/what-is-the-long-term-cost-of-code/)那里了解到的，即使是开发商也可以承担法律责任。

这不仅仅是技术方面的问题——尽管用[普罗米修斯](https://prometheus.io/)进行监控似乎是关键。马斯登说，与 DevOps 文化合作同样重要。如果存在模型漂移或陈旧，您仍然需要人工影响来判断建模结果是否古怪。
