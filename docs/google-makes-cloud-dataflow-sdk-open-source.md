# 谷歌将云数据流 SDK 开源

> 原文：<https://thenewstack.io/google-makes-cloud-dataflow-sdk-open-source/>

谷歌已经宣布它正在将云数据流 SDK 开源。

云数据流(Cloud Dataflow)在 6 月份刚刚推出，它将其描述为“通过实现更轻松、更可扩展的数据访问，实现大规模数据处理民主化的平台”。谷歌说,[仍是 alpha 版本](https://googlecloudplatform.blogspot.com/2014/12/google-announces-open-source-cloud-dataflow-sdk-for-java.html?m=1)，但在公司内部使用。

它旨在减轻开发人员的操作负担，同时为数据科学家、数据分析师和其他人提供更可扩展的数据访问。

SDK 将“使开发人员更容易与我们的托管服务集成，同时也为将云数据流移植到其他语言和执行环境奠定基础。”它还希望在结合基于流和批处理的处理模型方面刺激未来的创新。

云数据流由 MapReduce 和 Flume、MillWheel 等技术演化而来。底层服务是语言无关的，尽管这个 SDK 是用 Java 编写的。

据 VentureBeat 报道，谷歌希望让更多的开发者参与到自己的方法中来，即在数据到来时处理数据，而不是批量处理，并支持其谷歌云平台，与亚马逊网络服务和微软 Azure 等公司竞争。

云数据流类似于亚马逊 Kinesis，旨在减少复杂的实时数据处理之间的壁垒，并在应用程序中注入流。根据 Cloud Times 的说法，它还可以用作 ETL 替代方案，为 BI 系统中的处理准备数据。
