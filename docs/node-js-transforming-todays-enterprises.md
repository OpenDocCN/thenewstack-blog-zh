# Node.js 如何改变当今的企业

> 原文：<https://thenewstack.io/node-js-transforming-todays-enterprises/>

在今天的 [The New Stack Makers](https://thenewstack.io/podcasts/) 这一集，我们与[NodeSource](https://nodesource.com/)Solutions Architect Manager[Joe Doyle](https://github.com/JoeDoyle23)和 node source 首席技术官兼联合创始人 [Dan Shaw](https://github.com/dshaw) 坐在一起，听取更多关于当今企业如何使用 Node.js 的信息。采访记录于 2016 年 12 月在奥斯汀举行的 Node.js Interactive 2016。

“有两件事正在发生。我们需要加快步伐，每个人都变成了科技公司。是的，孤立的节点很简单，但是节点被用作解决难以置信的困难问题的工具。在企业中，你有你所珍视的遗产的祝福和诅咒，”肖说。

敏捷开发实践和大规模工作通常是携手并进的，尤其是在与微服务结合的时候。Doyle 指出，当企业最初看到在自己的基础设施中使用 Node 的其他企业取得成功时，如网飞和 PayPal，他们通常希望在自己的组织中采用相同的原则。“让我们快速前进，构建新技术，真正成长，但同时，他们有在这个平台上使用 Java 的历史，或者使用这个工具集来实现他们的。NET 应用程序。他们陷入的斗争是，“太好了。“我该如何将 Node 引入我在过去 10 年中一直使用的平台？””道尔说。

Shaw 指出，如今刚起步的公司在使用 Node 时，往往缺乏引导他们的企业向哪个方向发展的指导。Doyle 强调，虽然生态系统中的许多其他平台主要因为其开放性而支持开源，但一些使用 Node.js 的企业将社区和开源方面视为与使用它来解决公司的技术问题相比的事后想法。“Node 是最受欢迎的工具，你知道，吸引人的不是 Node，因为它是开源的，而是 Node，因为 Node 很棒，很受欢迎，并且以我期望的方式运行。”
