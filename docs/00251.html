Cloud Foundry仍然是不断变化的平台格局中的王者

原文：https://thenewstack.io/embracing-pragmatism-software-development/#0001-01-01

反对这样或那样的国王的假设是合乎逻辑的。在当今的科技生态系统中，有些领域无疑是市场领导者主导的。但是在一个如此快节奏的世界里，这个世界的国王们面对的挑战者在某些方面与他们所知的传统敌人几乎没有任何相似之处。

在某些方面很讽刺。这是代表云代工厂商不谈论“PaaS”的努力。它被视为另一个时代的术语，当时PaaS环境更加孤立，更多地作为专有软件运行。Cloud Foundry出现在这个时代的巅峰，大约在2011年，它以一种“开放”的方式强势崛起，扭转了刚刚习惯于开发开源平台的市场局面。Cloud Foundry已经成为大型企业运营中最受欢迎的“PaaS ”,并且已经成为使用基础作为开源项目基石的发展趋势的同义词。

然而，星座研究公司的副总裁兼首席分析师 Holger Mueller 在法兰克福的Cloud Foundry峰会上的讨论中声称，Cloud Foundry是“PaaS之王”


云代工厂依然称王称霸不断变化的平台格局

穆勒认为，Cloud Foundry采取了所有正确的举措。这是一个被大型供应商接受的企业平台。它拥有参与生态系统的活跃客户，平台上还有一个独立开发者黑客社区。这与半失败的公司项目不同，后者开源，很少或没有多样性，几乎没有成功的机会。

相比之下，有一些平台和服务是独特的，但也类似于Cloud Foundry。查看Pivotal的Stuart Charlton的Quora thread。他指出了不同主要平台的相同点和不同点。有Mesos，高性能调度程序。OpenShift和Cloud Foundry都是云平台。 Kubernetes ，Docker Swarm和Cloud Foundry Diego都是容器平台。

查尔顿总结得很好，他说这些不同的平台和服务是如何独立同时发展的。这是一种“选择的暴政”，在我们在container con吃了煎饼早餐后，我们越来越经常使用这个词。客户只是想弄清楚，在他们面前同时存在、竞争和独立工作的所有力量是什么。

“任何改变编程模式的东西都是一种威胁，”穆勒说。“虽然不断发展的云环境仍然是企业关注的焦点，但它的未来仍然不确定。”10%、15%、也许20%的负载已经转移到了云上，但一件大事仍将发生。问题是:谁将利用这一市场份额？”穆勒说。

与此同时，正如Mueller所说，随着越来越多的客户意识到他们必须提供云服务，我们将看到这些重叠模式继续演变。这只会加大云提供商之间的差异，并让客户能够根据服务价格、固有成本和云提供商提供的功能集做出选择。

也许这就是为什么Docker fork的故事如此引人注目。公司仍在开发他们提供云服务的策略，容器是讨论的核心，但尽管一些企业已经接受了容器，大多数企业仍在使用更传统的设置。

“这有很大的反响，因为这意味着像Cloud Foundry这样的公司将不得不提供多种构建软件的方式，”Mueller说。“实用主义总是把公司引向正确的方向，而不是宗教战争，说你必须100%这样或那样。”

云铸造基金会和 Docker 是新堆栈的赞助商。
