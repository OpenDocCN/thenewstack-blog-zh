# 谷歌发起组织保护 Istio、Angular 和其他开源项目的商标

> 原文：<https://thenewstack.io/google-launches-organization-to-protect-trademarks-for-istio-angular-and-other-open-source-projects/>

谷歌成立了一个新的组织来管理开源项目的商标。根据谷歌和 Alphabet 开源主管[克里斯·迪博纳](https://en.wikipedia.org/wiki/Chris_DiBona)的博客文章，[开放使用共享](https://openusage.org/)将帮助项目“通过商标管理和一致性测试的特定程序来维护和管理他们的项目身份”。

该公司表示，该组织为商标提供了“中立、独立的所有权”，并允许项目维护者“定义他们的身份，向下游用户提供他们产品质量的保证，并向社区中的其他人提供关于自由和公平使用品牌的确定性。”

迪博纳在一次采访中解释说，谷歌有超过 3000 个活跃的开源项目，当涉及到知识产权和开源时，它已经遇到了所有的问题，所以开放使用共享作为解决这一问题的一种方式被引入。

“我们的想法是为开源项目商标创建一个新的独立主页，并为开源项目商标的使用提供某种指导，因为如果你看看大多数开源许可证，他们不会处理、提及、否认或允许商标的使用，所以我们想解决这个问题，”迪博纳说。“我们希望围绕商标的语言与开源定义一致，可以被项目大规模采用。”

当 DiBona 写道“项目商标的使用方式不同于其代码的使用方式，因为商标是一种质量保证的方法”时，就发现了开放使用共用的核心。这包括确保有问题的代码拥有开源许可证。当商标得到适当管理时，项目维护者可以定义他们的身份，向下游用户提供他们产品质量的保证，并向社区中的其他人提供关于自由和公平使用品牌的确定性。”

成员项目将获得商标保护和管理、使用指南和一致性测试方面的支持，该组织还将接受商标教育，因为“理解和管理商标对于项目的长期可持续性至关重要，尤其是在基于开源的企业产品越来越多的情况下，”迪博纳写道。

迪博纳还表示，该项目是作为“真正的第三方”推出的，“谷歌没有多数控制权，也永远不会有多数控制权。”目前，开放使用共享资源的推出正与其董事会中的许多学术领袖以及 SADA 系统公司合作，但 IDC 的分析师 T2·阿尔·吉伦指出，名单上没有其他知名人士——尤其是在资金方面。

“顺便说一下，这个独立团体是由谷歌资助的，”吉伦说。“其他大型云供应商在哪里？为什么我们没有看到阿里巴巴或亚马逊网络服务或微软被指定为这项倡议的合作伙伴？我认为这很快就会实现，但在此之前，如果通过一个主要供应商来维持，这个实体将真正独立到什么程度？”

在发布时，开放使用共享有三个项目加入:T4 Angular web 应用框架、基于 web 的团队代码协作工具 Gerrit 和 Istio 服务网格。

## Istio 的未来

在提到最后一个项目 Istio 时，开放使用共享的声明进入了云原生社区正在进行的争论领域，即该项目是否应该由谷歌捐赠给[云原生计算基金会(CNCF)](https://www.cncf.io/) ，就像谷歌对其 Kubernetes 项目所做的那样。四月，Protocol [报道](https://www.protocol.com/google-cloud-kurian-istio-foundation)谷歌云首席执行官 Thomas Kurian 称，该公司“最终将在不久的将来将其开源项目 Istio 捐赠给一个基金会。”

迪博纳说，这确实是基金会提到，但它只专注于商标，而不是治理或任何其他方面，你会联想到 CNCF。

“我们不想成为一个营销组织，一个治理组织。我的最终目标是成为这些商标的图书管理员。我们不承担治理问题，”迪博纳说，并补充说，像 CNCF 和 Linux 基金会这样的东西已经存在，他们没有试图重新创建它们。“它们已经存在，而且很好。”

正如早先一篇关于谷歌是否会将 Istio 捐赠给一个基金会的协议文章所指出的，这里的利害关系是，在没有开放治理保证的情况下，其他公司是否会对采用或贡献项目感到舒服。然而，正如 IDC 分析师 Al Gillen 所言，这无助于维护对治理的控制 DiBona 在他的博客文章中也提到了这一点，他写道“我们希望这种新模式鼓励任何迄今为止一直站在场外的人参与这些项目。”对于吉伦来说，他仍然乐观地认为 Istio 向一个独立基金会捐款仍然是未来的一种可能性，但他并不认为这排除了这一可能性，并指出谷歌与 Kubernetes 的历史。

“你可以通过多种不同的方式将项目推向相对成熟，其中一种方式是让单个或少数商业实体实质上成为项目的治理者，推动项目向前发展，并在将项目移交给基金会之前达到一定的成熟水平。比如说 Kubernetes，谷歌就是这么做的，对吧？”吉伦说。“谷歌将 Kubernetes 逼到了某一点，然后，慢慢地，在几年的时间里，移交了与 CNCF 上的 Kubernetes 相关的治理的不同方面。但我的意思是，这个声明根本不是关于治理的。这是关于这些技术的版权和商标。”

与此同时，[关于 Istio 捐赠给一个独立基金会的传言](https://twitter.com/wm/status/1279862013565349888)一直在[流传](https://twitter.com/lizrice/status/1280103643694477313)，我们问 CNCF 首席技术官 [Chris Aniszczyk](https://www.linkedin.com/in/caniszczyk) 他对 Istio 捐赠给一个不是 CNCF 的基金会有何看法，他说“我们的社区成员对谷歌选择不将 Istio 项目捐赠给云本地计算基金会(CNCF)感到困惑，但我们很高兴帮助指导他们在任何时候重新提交他们从 2017 年开始的旧[项目提案。最后，我们的社区仍然专注于构建和支持我们的服务网格项目，如 Envoy、linkerd 和互操作性工作，如服务网格接口(SMI)。CNCF 将继续成为云原生和服务网格协作和创新的重心。”](https://github.com/cncf/toc/pull/70)

CNCF 技术监督委员会(TOC)主席利兹·赖斯(Liz Rice)进一步关注了该项目的潜在治理。

“正如 CNCF 和其他软件基金会如 ASF 和 Mozilla 所要求的，开放治理的美妙之处在于，控制权由来自多个供应商甚至最终用户的代表共享。这样，项目路线图就不会与任何一家公司的产品策略纠缠在一起。最终用户公司不希望他们所依赖的项目的生命受制于一家公司的突发奇想，这家公司可以随时选择停止投资，”赖斯在一封电子邮件中写道。“根据我与最终用户的交流经验，像 CNCF 这样的基金会让他们相信，核心的、战略性的软件组件会一直存在，并且会在未来很长一段时间内得到良好的管理和维护。供应商关心商标，但他们对在内部运行软件的最终用户来说并不重要。[……]如果 Istio 想要重新考虑，我们的大门依然敞开。”

Amazon Web Services、云本地计算基金会和 Linux 基金会是新堆栈的赞助商。
