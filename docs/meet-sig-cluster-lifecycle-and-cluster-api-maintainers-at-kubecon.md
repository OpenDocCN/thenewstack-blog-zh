# 在 KubeCon 会见 SIG 集群生命周期和集群 API 维护人员

> 原文：<https://thenewstack.io/meet-sig-cluster-lifecycle-and-cluster-api-maintainers-at-kubecon/>

KubeCon 是来自领先的开源和云原生社区的采纳者和技术人员可以见面、联系和进行热烈讨论的地方。这也是特殊利益集团(SIG)面对面讨论他们最感兴趣的技术的机会。

Kubernetes 中最完善的 SIG 之一是集群生命周期 SIG，它旨在通过使其维护如此可靠以至于没有人关注它来使集群生命周期变得乏味。当我们谈到集群生命周期时，我们指的是与 Kubernetes 集群及其组件的创建、配置、升级、降级和拆卸相关的一切。简而言之，这是用户使用 Kubernetes 的第 2 天以及任何 Kubernetes 集群运营之外的每一次旅程的开始。

这个 SIG 创建的第一个项目是`kubeadm`*，这是一个简化的安装工具和构建块，用于简化 Kubernetes 节点的安装和升级。今天，kubeadm 是最常用的节点引导程序之一，它支持生态系统中的许多工具。*

 *集群生命周期 SIG 也是集群 API (CAPI)项目的所在地，该项目提供了跨不同部署环境的机器的声明性抽象和公共控制平面配置。自发布以来，团队一直使用 CAPI 来管理他们的 Kubernetes 集群，无论他们在哪里运行。

在底特律的 KubeCon，该集团将于周三举办一场 [SIG 集群生命周期介绍](https://sched.co/182MW)。，10 月 26 日，上午 11:55–中午 12:30。这是在这个领域工作的整个社区的一个机会，来自世界各地 80 多家公司的大约 400 名开发人员在两年后终于再次相聚，看看并庆祝我们正在开发的最新和最棒的功能，并了解 SIG 路线图中的下一步内容。

参与该社区对每个人来说都是一个很好的机会，因为 kubeadm、Cluster API 和该 SIG 管理的其他项目为 Kubernetes cluster 在 VMware Tanzu Kubernetes Grid 等产品中的生命周期管理提供了可靠的基础。

如果您有兴趣深入了解集群 API，也可以参加 10 月 28 日(星期五)下午 2-3 点的[教程:kubectl Create Cluster:Production-ready Kubernetes with Cluster API](https://kccncna2022.sched.com/event/1BZDs/tutorial-kubectl-create-cluster-production-ready-kubernetes-with-cluster-api-10-killian-muldoon-shivani-singhal-yuvaraj-balaji-rao-kakaraparthi-stefan-buringer-vmware-jack-francis-microsoft)。在该讲座中，来自微软和 VMware 的一组项目维护人员将带您从`kubectl create my-cluster.yaml`和基本操作(如纵向扩展、横向扩展和升级)开始，了解如何配置和使用更高级的功能，如自动机器修复(MachineHealthCheck)和自动扩展机器。

该团队还将展示如何利用最近添加到项目中的新扩展点、SDK 运行时，以及如何使用诸如集群 API 可视化工具、`clustectl describe`等检查工具。
