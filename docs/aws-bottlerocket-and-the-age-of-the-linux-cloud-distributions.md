# AWS 的 Bottlerocket 和 Linux 云发行版的时代

> 原文：<https://thenewstack.io/aws-bottlerocket-and-the-age-of-the-linux-cloud-distributions/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了亚马逊网络服务公司负责容器和开源的副总裁[迪帕克·辛格](https://www.linkedin.com/in/dsingh/)，以及亚马逊网络服务公司负责开源的总经理[彼得·乌兰德](https://www.linkedin.com/in/ulander/)，讨论该公司最近发布的用于云计算的 Bottlerocket Linux 发行版。

TNS 编辑和营销总监[利比·克拉克](https://thenewstack.io/author/libby/)主持了这一集，旁边还有 TNS 创始人兼发行人[亚历克斯·威廉姆斯](https://thenewstack.io/author/alex/)，TNS 高级编辑[理查德·麦克马努斯](https://thenewstack.io/author/richard/)，以及 TNS 执行主编[约阿布·杰克逊](https://thenewstack.io/author/joab/)。

[第 116 集:AWS Bottlerocket 和 Linux 云发行版时代](https://thenewstack.simplecast.com/episodes/episode-116-aws-bottlerocket-and-the-age-of-the-linux-cloud-distributions)

今年早些时候，亚马逊网络服务[推出了 Bottlerocket](https://thenewstack.io/amazon-web-services-releases-a-linux-distribution-built-for-containers/) ，这是一个专门为云使用而构建的 Linux 发行版(甚至更适合 EKS)。AWS 并不是第一家这样做的公司——几年前，CoreOS 和 Red Hat 在 T21 的项目 Atomic 中采用了类似的策略。Red Hat 购买了 CoreOS 并将其整合到 OpenShift 中，CoreOS 项目也作为独立的 [Flatcar Linux](https://www.flatcar-linux.org/) 项目继续存在。所以很明显，专门为云组装的发行版的想法正在得到一些支持。因此，我们与 AWS 人员进行了交谈，以了解他们如何构建 Bottlerocket 以及它在 AWS 云中扮演的角色。

AWS 团队“学到了很多运营模式，尤其是我们的客户如何管理他们的车队。所以这让我们对 Botterocket 正在做的事情有了更多的了解，”辛格说。

然后，在稍后的节目中，我们将讨论网站上其他有趣的播客和新闻。其中包括与 Drupal 创始人 Dries Buytaert 就为什么现在是创业的最佳时机进行的坦诚讨论。我们了解一个简化运算符的新项目， [D2IQ 的 Kubernetes 通用声明运算符(KUDO)](https://thenewstack.io/kudo-automates-kubernetes-operators/) 。我们学到了博弈论，以及为什么[系统安全更像扑克而不是象棋](https://thenewstack.io/game-theory-why-system-security-is-like-poker-not-chess/)。说到游戏，我们深入研究 Sysdig 的新卡牌对抗容器。

亚马逊网络服务是新堆栈的赞助商。
