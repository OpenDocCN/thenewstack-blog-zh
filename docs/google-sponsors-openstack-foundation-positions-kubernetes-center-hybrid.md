# 谷歌将 Kubernetes 定位于混合世界的中心

> 原文：<https://thenewstack.io/google-sponsors-openstack-foundation-positions-kubernetes-center-hybrid/>

Google 将 Kubernetes 定位于企业混合云运动的核心。这在昨天谷歌宣布加入 OpenStack 的声明中显而易见。这是一个信号，谷歌将 OpenStack 视为一种工具，它需要使 Kubernetes 成为容器编排的标准。

如果是这样的话，OpenStack 的 Magnum 可能是至关重要的:它可以为企业提供一种更简单的方式来与 Kubernetes 进行交互，而不是直接处理容器编排的复杂性。Magnum 已经与谷歌的 Kubernetes 集成在一起，是一个管理应用程序容器的 API 服务。它面向希望为云用户提供自助式托管主机服务的 OpenStack 云运营商。OpenStack 表示，Magnum 本质上使 Kubernetes 和 Docker 成为 OpenStack 中的“一级资源”，可以在集群配置中的虚拟机或裸机上运行。

OpenStack 首席运营官公司的马克·科利尔在昨天的一份声明中表示，企业客户对像 [Magnum](https://wiki.openstack.org/wiki/Magnum) 这样的服务越来越感兴趣，这些服务可以利用 Kubernetes，并使云原生应用程序能够在单一平台上与传统企业工作负载一起运行。他引用了最近的行业咨询，确认该企业将 OpenStack 视为他们的容器“生产之路”。

谷歌云产品经理 Craig McLuckie 说，赞助 OpenStack 基金会是一项战略举措，[应对了当今企业计算的两种趋势。第一个趋势是 Collier 提到的:企业需要能够运行混合云基础架构环境，其中 OpenStack 处理内部组件。麦克卢奇强调的第二个趋势当然是集装箱化。McLuckie 希望看到更多熟悉容器原生应用程序设计模式的开发人员，以及结合公有云和私有云的混合环境之间的改进的互操作性。](http://googlecloudplatform.blogspot.com.es/2015/07/Containers-Private-Cloud-Google-Sponsors-OpenStack-Foundation.html)

此举发生在 Kubernetes 及其合作伙伴预计将于下周在 OSCON 发布的一系列声明之前。麦克卢奇也将在下个月的 OpenStack 硅谷活动上发表演讲。

Docker 是新堆栈的赞助商。
