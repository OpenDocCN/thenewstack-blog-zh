# 为当今高度复杂的云环境提供数据保护

> 原文：<https://thenewstack.io/data-protection-for-todays-highly-complex-cloud-native-world/>

[戴尔科技](https://www.delltechnologies.com/en-us/index.htm)赞助了这次播客。

在此[新堆栈制造商](/podcasts/makers)播客中，来自[戴尔 EMC](https://www.delltechnologies.com/en-us/index.htm) 和 [VMware](https://blogs.vmware.com/cloudnative/2020/03/19/managing-kubernetes-at-enterprise-scale-a-closer-look-at-tanzu-mission-control/) 的技术思想领袖讨论了当今高度复杂的云原生环境的数据保护动态和其他 DevOps 相关主题。

[为当今高度复杂的云原生世界提供数据保护](https://thenewstack.simplecast.com/episodes/data-protection-for-todays-highly-complex-cloud-native-world)

来自戴尔 EMC 和 [VMware](https://www.vmware.com/company.html) 的播客嘉宾有:

*   [Jennifer Aspesi](https://www.linkedin.com/in/jennifer-aspesi-3265447) ，戴尔技术公司数据保护解决方案营销高级顾问。
*   [迈克尔迈克尔](https://www.linkedin.com/in/mimichael/)，VMware 产品管理总监。
*   [Nivas Iyer](https://www.linkedin.com/in/nivasiyer) ，产品管理，走向市场和战略合作伙伴领导，戴尔技术。
*   [Tom spoon more](https://www.linkedin.com/in/tspoone)，VMware 产品线经理。

使用正确的工具，许多数据保护任务可以自动化，并且对开发人员来说基本上是透明的。换句话说，开发人员可以施展他们的魔法，同时基本上不受运营相关任务的约束。

Aspesi 说:“好消息是，现在我们有了一种方法来创建策略并保护这些工作负载，而开发人员甚至不会受到影响。”“他们可以根据需要将工作负载分配到任何地方。”

尽管如此，开发人员的角色经常延伸到以前可能是操作的领域，特别是对于数据库——反之亦然。Iyer 说:“在 Kubernetes 和云的本地世界中，这是一个开发功能，因为开发人员需要为数据库的有状态数据建立机制。”

与此同时，“运营团队也在创造代码形式的基础设施，”Iyer 说。"所以他们可以不断地改变."

开发人员最不需要的是数据保护、维护和管理任务，这会降低他们开发和部署应用程序的能力。这就是工具集和平台的选择发挥关键作用的地方。“开发者希望从有助于他们加速创新的东西开始，”迈克尔说。"他们想减少障碍，并想出如何加速软件交付管道."

Michael 说，满足开发人员的需求以减少他们部署节奏的障碍在很大程度上是“我们从 VMware 和戴尔 EMC 的角度处理事情的方式”。

[例如，VMware 为 vAdmin 提供的 Tanzu 产品和服务组合](https://tanzu.vmware.com/tanzu)“帮助客户在 Kubernetes 上以统一的方式开发和部署整个车队的应用程序，”Spoonemore 说。Spoonemore 说，DevOps 还可以“比传统方式更快速地开发和部署这些应用程序”。

当然，开源继续扮演着不可或缺的角色。“曾经有一段时间，我们采用开源项目，你围绕它建立支持，这就是业务的范围，”Spoonemore 说。“现在，我们已经有了开源软件。我的意思是，我们的旗舰产品 vSphere 内置了开源项目 Kubernetes。”
