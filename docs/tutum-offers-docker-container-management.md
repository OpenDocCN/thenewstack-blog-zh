# Tutum 希望通过 Docker 管理服务在 IaaS 和 PaaS 之间找到平衡点

> 原文：<https://thenewstack.io/tutum-offers-docker-container-management/>

Tutum 的服务旨在帮助开发人员构建、部署和管理使用 Docker 构建的应用程序，Tutum 认为它在基础设施即服务和平台即服务之间找到了平衡点。该公司正在调整其产品，以便开发者能够在任何云上使用它。

Tutum 的服务仍在测试阶段，之前需要用户用 Tutum 托管整个应用程序。“我们从用户那里得知，他们已经在 AWS 或谷歌上运行他们的应用程序了。所以我们所做的是，我们回到我们的服务，重新设计它，不包括托管，”图图姆的联合创始人兼首席执行官 Borja Burgos-Galindo 说。

Tutum 首先面向数字海洋用户，但该公司正在努力整合 AWS，并计划继续为其他云提供商提供支持。事实上，Burgos-Galindo 表示，他已经从 SoftLayer、Azure 和 VMware 那里听说了他们对与 Tutum 集成的兴趣。

Tutum 旨在简化 Docker 的使用管理。他说，这个想法是提供一个单一的平台来帮助自动化部署、扩展、链接在不同云提供商上运行的容器、实现冗余和监控部署。

特性分为三类:构建、部署和管理。Tutum 包含一个私有注册表，用户可以在其中存储私有映像和部署容器。该公司正在开发新的功能，让用户从 Dockerfile 构建，并通过 Git push 自动触发新的构建。例如，用户设置配置，以便新的开发构建自动在特定区域或特定节点集群中运行。

当开发人员准备好部署时，他们可以使用 Tutum 跨主机和云提供商链接容器。该服务自动进行负载平衡。用户通过移动 Tutum 仪表盘中的滑块来添加更多资源，从而扩展他们的应用程序。

Tutum 也有助于日常管理，但不是为了取代 New Relic 或其他监控服务而设计的。相反，Tutum 从用户的容器中收集日志和监控数据，客户可以将这些数据输入到 New Relic 等服务中，帮助他们可视化这些数据，Burgos-Galindo 说。

他认为有了 Tutum 的服务，开发者就能充分利用 IaaS 和 PaaS。他说，有了基础设施服务，你可以“灵活地调整服务，但你必须管理一切，这是一笔很大的开销”。

虽然 PaaS 负责管理方面的工作，但对于一些开发人员来说，这通常太受限制了。

Tutum 旨在让开发人员在使用基础设施服务时获得自由和灵活性，而无需担心确保他们的应用程序冗余、高度可用和可伸缩等问题。

“这是基础设施即服务和 PaaS 之间的最佳点，”他说。

首先，开发人员在 Tutum 中提交他们的云登录信息——目前用于数字海洋，但将来用于其他云提供商。这给了 Tutum 处理提供新服务的权限。

Tutum 通过一个部署在每个 Linux 节点上的小代理来工作。Burgos-Galindo 表示，无论用户是在裸机还是虚拟化服务器上构建，它都是一样的。

在之前的模式下，该公司向用户收取每个集装箱每月 4 美元的费用。它将在 11 月 1 日之前逐步取消这种做法。Tutum 尚未推出新模式的定价，用户可以在任何云上运行他们的应用程序。目前，新模型是免费使用的，布尔戈斯-加林多预计该公司将永远提供免费层。
