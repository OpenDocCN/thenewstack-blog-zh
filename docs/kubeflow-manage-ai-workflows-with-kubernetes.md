# KubeFlow:用 Kubernetes 管理人工智能工作流

> 原文：<https://thenewstack.io/kubeflow-manage-ai-workflows-with-kubernetes/>

[KubeFlow:用 Kubernetes](https://thenewstack.simplecast.com/episodes/kubeflow-manage-ai-workflows-with-kubernetes) 管理 AI 工作流

对于现代企业来说，人工智能可能正处于其宣传周期的顶峰，但对于 IT 管理员来说，它仍然是一个令人头痛的问题，需要可能完全位于正常运营范围之外的软件和流程。但是救援已经在路上了。在上个月的 Kubecon + CloudNativeCon EU 2018 上，[kube flow 联合创始人兼谷歌产品经理 David Aronchick](https://twitter.com/aronchick) 推出了第一个工作版本(0.1)的 [Kubeflow，](https://github.com/kubeflow)软件打包了最受欢迎的人工智能和机器学习工具，以便它们可以在 Kubernetes 容器编排引擎上轻松运行。

“我们不能要求人们成为所有方面的专家，”阿龙奇克在介绍这项技术的主题演讲中告诉 Kubecon 的观众。他指出，今天的人工智能工具，如 Jupyter Notebooks 或 TensorFlow，必须单独管理，而且往往需要付出很大努力(他称之为“定制”模型)。Kubeflow 可以通过提供一个可以管理这些软件的公共平台，并轻松提供给最终用户(数据科学家)来合理化这些难以驾驭的软件。

Kubeflow 被设计成可伸缩、可移植和可组合的。阿龙奇克指出，对于如何构建一个系统，“每个数据科学家都会有略微不同的看法”。Kubeflow 提供了这种所需的灵活性。软件原语被捕获为 Kubernetes 的声明性配置中的设置，允许软件跨不同平台部署。keynote 演示展示了如何在笔记本电脑和运行张量处理单元的 Google 云平台 Kubernetes 集群上运行相同的建模软件。

在 Kubecon 录制的最新一集 New Stack Makers 中，我们与 Aronchick 以及 Mesosphere 产品营销经理 [Chris Gaun](https://twitter.com/gaunetes) 和 Mesosphere 社区项目技术负责人[jrg Schad](https://twitter.com/joerg_schad)讨论了 Kubeflow，它是如何产生的，如何运行的，可以在哪里使用，以及未来的道路。
