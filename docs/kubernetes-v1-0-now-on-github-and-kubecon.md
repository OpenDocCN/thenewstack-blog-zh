# Kubernetes v1.0 现已在 GitHub 和 KubeCon 上发布！

> 原文：<https://thenewstack.io/kubernetes-v1-0-now-on-github-and-kubecon/>

Kubernetes 可以说是市场上最具优势的容器平台。它的使用越来越广泛。它背后有许多支持者，还有一个刚刚起步的社区。

昨晚，谷歌高级软件工程师布伦丹·伯恩斯(Brendan Burns)合并了 Kubernetes [V1.0.0 来掌握](https://github.com/GoogleCloudPlatform/kubernetes/pull/11103)。预期的代码合并伴随着谷歌在 OSCON 发布的大计划。

库伯内特的船现在肯定在巡航。我在 GopherCon 进行了几次对话，讨论它作为底层集群管理平台的崛起。例如，红帽 [OpenShift 3](https://www.openshift.com/products/enterprise) 现在在 Kubernetes 上运行，正如[在上个月的红帽峰会上宣布的](https://www.zdnet.com/article/red-hat-rolls-out-container-based-openshift-enterprise-3-cloud/)。IBM、惠普、Mesosphere、微软、CoreOS、VMware 和英特尔也在与谷歌合作。Kubernetes 是用 Go 编程语言编写的发展最快的项目之一，仅次于 Docker。在撰写本文时，Kubernetes 有 408 个贡献者。

Kelsey Hightower 是这个新的基础设施世界的主要思想家之一，CoreOS 的首席倡导者和产品经理，他对 Kubernetes 说，现在它已经达到了 1.0 的状态:

有很多要探索的，但更多的是，只是想提醒一下，1.0.0 版现在已经在 GitHub 上了。

## KubeCon

只有 Kubernetes 社区现在计划举办自己的独立会议才有意义。本周, [KubeCon](https://kubecon.io) 在 GopherCon 上发布，GopherCon 是 Go 编程语言社区的会议。 ~~KubeCon 定于 11 月 1 日至 4 日在丹佛万豪酒店举行~~。

**更新** : KubeCon 正在进行，但看起来日期已经确定。最终日期确定后，我们会通知您。

Kismatic 的 Patrick Reilly 和 Joseph Jacks 与 Kelsey Hightower 一起组织了这次活动。这项活动将由 GopherAcademy 的 Brian Ketelsen 和 Erik St. Martin 制作，他们是 GopherCon 会议的幕后人员。Kismatic 正在通过开源[插件](https://github.com/kismatic)和企业支持将 Kubernetes 商业化。 [CoreOS](https://coreos.com/) 是一个轻量级操作系统和平台，与 Kubernetes 有着深度的技术集成。

Jacks 在一次电子邮件采访中说, KubeCon 将为该项目的新用户和高级用户设计，让他们学习，相互合作和参与。

这种新的应用基础设施正在出现。这与一个新基础设施生态系统的成长有关，它包含的不仅仅是容器技术。 GopherCon 证明了这种新的基础设施生态系统及其与开源社区的天然关系。这是一个关于 Go 编程语言的会议，由基于 Go 开源语言的公司与容器生态系统紧密联系在一起。

[ContainerCon](http://events.linuxfoundation.org/events/containercon) ， [ContainerCamp](https://container.camp/) ， [Operability 1.0](http://www.operability.io/) ， [GoTo Conference](https://thenewstack.io/a-conference-to-discuss-developer-driven-infrastructure/) 和 [The Software Circus](http://softwarecircus.io/) 只是这个新电路中的一些会议。这些只是接下来两个月的事情。将会有更多的会议涉及到这个分布式世界不断转变成新形式的方式。一部分是机器，一部分是算法——我们的生活和工作方式将留给半自动系统，我们与这些系统互动，以保持一个日益复杂的世界的背景。随着我们寻求理解和塑造这些快速新兴技术的未来，像这样的会议只会变得更加重要，这些技术正在如此快速地改变应用程序开发和管理的概念。

CoreOS、Docker、HP、IBM 和 Red Hat 是新堆栈的赞助商。
