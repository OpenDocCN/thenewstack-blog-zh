# CNCF 选择了普罗米修斯进行监控，那么接下来呢？

> 原文：<https://thenewstack.io/harmonizing-oss-cloud-native-computing-foundation-red-hat/>

在云本地计算基金会(Cloud Native Computing Foundation)上，任何在容器中运行的技术，或者是面向微服务的应用开发方法的一部分，都可能成为该基金会的下一个托管项目。Kubernetes 是第一个 CNCF 支持的项目，[普罗米修斯](https://thenewstack.io/cloud-native-computing-foundation-prometheus-second-hosted-project/)最近获得了第二个项目的荣誉。对于那些参与 CNCF 的人来说，这不仅关系到改变云原生计算范式，还关系到开发者和企业如何协调他们的开源软件及其文化。

在这一集的《T4》中，新 Stack Makers 嵌入下面，新 Stack 创始人 Alex Williams 和联合主持人 [Lee Calcote](https://twitter.com/lcalcote) 与[Red Hat open shift](https://twitter.com/pythondj)的社区发展总监 Diane Mueller 和 [Chris Aniszczyk](https://twitter.com/cra) 进行了对话，后者在对话时是 CNCF 的临时执行董事(上周， [Dan Kohn](https://www.linkedin.com/in/dankohn) 担任了 CNCF 董事)。

[黛安·穆勒，克里斯·安尼斯克](https://thenewstack.simplecast.com/episodes/diane-mueller-chris-anisczyk-at-oscon-in-austin)

对话围绕着 CNCF 塑造开源开发的方式，以及 Red Hat 为帮助改进整个开源社区的交流所采取的步骤。

这段对话[也可以在 YouTube 上听到。](https://youtu.be/EPJMTRpI2JE)

创建一个有凝聚力的社区需要建立在经验基础上的领导力，这在一些开源项目中很难实现。幸运的是，CNCF 拥有一些最聪明的科技人才，他们为基金会的持续成功做出了贡献。这些人包括 Kubernetes 的联合技术主管兼 CNCF 技术监督委员会(TOC)成员[布莱安·葛兰特](https://lists.linux-foundation.org/pipermail/cncf-toc/2016-January/000024.html)、[weaver works 的首席执行官](https://www.mirantis.com/software/docker/kubernetes/)兼 CNCF TOC 主席代表[亚历克西斯·理查森](https://twitter.com/monadic)，以及首席技术官兼 Docker 创始人、CNCF TOC 成员[索罗门·海克斯](https://twitter.com/solomonstre)。

“看看 OpenShift，它是 Kubernetes[的上游，有很多方法可以监控 Kubernetes 集群。以一种协调的方式来做这件事是我们现在正在进行的跨社区协作所寻求的一部分。为了协调，你首先要有一个治理结构。通过进入 CNCF，我们有了一个结构，我们有一个独立选举的技术最高法院，利用他们的经验协调这些技术，”穆勒指出。](http://kubernetes.io/)

Mueller 在 Red Hat 拥有 20 年致力于社区建设的经验，不仅致力于改善 Red Hat OpenShift 内部的开源社区，还致力于改善整个生态圈。打破沟通和贡献的障碍是这一过程的重要组成部分。

Mueller 提到了 Eric Raymond 的[大教堂和集市](http://www.catb.org/esr/writings/cathedral-bazaar/)然而[再次](https://thenewstack.io/for-open-source-software-foundations-are-the-new-cathedrals/)的比喻，指出它不再适用于今天的开源方法。“下一个障碍是跨社区交流。如何让不同的上游项目相互交流，”Mueller 指出，并继续解释说，社区讨论可能会很快被关于应该关注堆栈的哪一部分或谁将从事某项工作的来回抨击淹没。

“关于我们是应该跨上游项目工作还是应该在堆栈的更下游工作，有太多的噪音和对话在进行。必须有一种方法使对话富有成效，并允许反馈循环回到从事特定用例工作的人。我们正在推广一种新的开源社区模式，专注于点对点的无障碍对话。在促进这种对话的同时，我们也要确保信息的自由流动，”穆勒说。

Aniszczyk 继续解释了云计算原生计算基础在过去的一年中取得的进展。它的重点仍然是选择有趣和令人兴奋的项目来支持，同时不知道人们如何在他们的堆栈中部署它们。

“CNCF 的目的是提供一个公共场所，人们可以从中选择并提出自己的解决方案。我们尝试邀请我们认为有趣的项目，类似于 Apache 基金会。我们不关心基础架构供应。我们不在乎你如何放弃你的机器。我们感兴趣的是你在上面运行的技术。”

码头工人、[红帽](https://www.openshift.com/)和[织工](https://www.mirantis.com/software/docker/kubernetes/)是新栈的赞助商。
