# APIcast，一个适合初创企业和小型企业的 API 网关(免费*)

> 原文：<https://thenewstack.io/apicast-an-api-gateway-suited-for-the-startup-and-small-business-its-free/>

API 管理提供商 [3Scale](http://www.3scale.net/) 发布了一项免费的云托管 API 网关服务，旨在帮助小型企业和非营利组织开始构建 API 驱动的业务。

APIcast 提供了一个简单的点击式界面，将业务的 API 端点 URL 连接到 3Scale 代理网关服务。在那里，可以使用 3Scale 的管理功能集来管理 API。APIcast 服务包括通过对用户设置速率限制来管理 API 的选项，生成报告，以及自动启动开发者门户和创建交互式 [Swagger](https://swagger.io/) 文档。Swagger 是一种易于使用的 API 的表示。

3Scale 首席执行官史蒂文·威尔莫特(Steven Willmott)表示，对于那些寻求公开数据或业务功能、测试采用、生态系统吸收以及感受可以通过 API 打开哪些渠道的企业来说，新产品是一个市场切入点。

“它绝对适合小型组织，”威尔莫特说。“人们说‘SMB ’,但我们真正指的是拥有网络业务并希望利用移动或数据开展业务的小型组织或非营利组织。”

APIcast 将免费提供给 API 每天处理不到 5 万个呼叫的企业，一旦达到这一数量，3Scale 希望讨论他们更广泛的产品范围:“我们可能会逐步扩大到付费版本，”威尔莫特说。

Bruno Pedro 是一名 API 专家，他运营着 [APIUX 博客](http://apiux.com/)，监督 API 更新注册 API Changelog，并在协作服务 [Redbooth](https://redbooth.com/) 担任平台总监。在 Redbooth，API 一直是他们 DNA 的一部分，用于支持内部创建的应用程序。但佩德罗也监督了 API 第三版的使用，该版本开始时没有外部使用，现在已经增长到每天有 500 万个调用。

对于像 Redbooth 这样使用自己的 API 的企业来说，API 调用会很快增加。但是对于那些不使用自己的 API 的人来说，比如一个获得一些动力的创业公司，它就像一个没有任何限制的黑盒子。不需要更改代码。

> 这意味着创业公司不必担心扩展他们自己的后端来管理 API 调用。

佩德罗说，当一家公司推出一款产品时，他们会关注可用性和定价等问题。用户不想担心扩展他们的 API。通过 APIcast，3Scale 可以与低端客户合作，并在他们升级到更高级的产品时与他们在一起。

Willmott 说，这项服务是为生产就绪的 API 设计的，例如，对于移动设备，开发者可以创建不同版本的应用程序，并查看每个版本产生的流量。开发者可以设置用户速率限制，而不是应用速率限制。“你可以限制每个用户，很明显，随着应用的增长，你会过渡到一种拥有自己网关的模式。但除非成功，否则你可能不想这么做。”

Willmott 说，APIcast 是一种企业可以测试哪种类型的终端产品最有可能用 API 进行构建的方法。例如，它可以连接到调用 API 的 Google 电子表格插件，甚至是利用 API 的网站小部件产品。这几乎类似于 A/B 测试:开发人员可以拥有不同版本的 API，然后在每个版本上使用控件进行测试。

APIcast 提供的 API 也与 3Scale 帐户中管理的其他 API 位于相同的接口中，包括任何属于客户付费服务的 API。

API 网关策略已经被其他提供商用作建立客户组合的一种方式。Apigee 早在 2010 年就提供了一个网关。当时它们被称为 Sonoa 系统。他们将自己的名字改为 Apigee，但此后主要将该网关用于他们的客户组合。

但总的来说，这是一种适合大量商业人士的服务，他们需要学习如何对他们的业务进行编程，这样他们就可以公开没有 API 就不可能实现的服务。
