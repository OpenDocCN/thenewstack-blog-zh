# 独立的 BaaS 提供商应该感到担忧

> 原文：<https://thenewstack.io/independent-baas-providers-should-be-worried/>

很明显，大型云提供商将开始提供更多的后端即服务功能。亚马逊网络服务、谷歌计算和 Azure 都一直在为移动开发者提供不同程度的服务，这些服务可以归入 BaaS 细分市场。

然而今天，AWS 坚定地踏入了 BaaS 的世界。虽然独立的 BaaS 提供商可能能够提供比今天推出的更多更好的功能，但毫无疑问，AWS 将为其 BaaS 功能集添加更多功能。今天的新闻给剩下的独立 BaaS 提供商带来了更大的压力。

在今天早上的纽约峰会上，AWS 宣布了几项新的移动服务。一个是 Cognito，一个管理用户身份、同步和安全的服务。该服务允许开发者通过脸书、谷歌和亚马逊登录来验证用户身份。它还可以在设备之间同步用户偏好和应用状态等数据。

Cognito 还处理一些安全功能，以便开发人员可以对从移动设备访问的 AWS 资源设置粒度权限。

亚马逊还宣布了一项新的移动分析服务。AWS Mobile 副总裁 Marco Argenti 表示:“当我们与开发者交谈时，我们听说快速、可靠的参与度指标是基于客户反馈改进应用的基石。

移动分析让开发者可以报告活跃用户、会话和留存、应用内收入和定制事件。这并不完全是实时的。它旨在从应用程序接收数据后 60 分钟内交付报告。

为了防止开发者担心亚马逊对这些数据感兴趣，Argenti 补充道:“这些数据不是由亚马逊开采或出售的。它属于开发商。”

最后，AWS 更新了其移动 SDK，旨在使开发者更容易访问各种 AWS 服务，包括今天推出的新服务以及 S3 和 DynamoDB。

独立的 BaaS 提供商提供的功能远不止这些。但是一些开发者可能会发现选择这些 AWS 移动工具比注册第三方更容易。

BaaS 市场已经非常活跃。脸书收购了 Parse，该公司在那里继续成功运营。PayPal 收购了 StackMob，并迅速关闭了这项服务。不幸的是，对于剩下的提供商，如 Cloudmine、FeedHenry 和 Kinvey，最大的云提供商似乎准备建立自己的 BaaS 功能，而不是收购独立的提供商。

对 BaaS 的球员来说，比赛还没有结束。正如金维的首席执行官斯拉维什·斯里达尔在他对 AWS 公告的评论中指出的那样，BaaS 提供商有一个明显的优势，那就是大多数不会将用户锁定在特定的云服务上。如果你走的是使用 AWS 的 BaaS 功能的路线，如果你决定放弃 AWS，转而支持 Google Compute，你会有一段更艰难的时间。

斯里达尔还指出，为什么几乎所有运行在 AWS 上的服务都应该担心 AWS 是否会决定直接与该服务竞争。“AWS 面临的挑战是，只提供计算、网络和存储不是一个长期可防御的产品集，”斯里达尔写道。这意味着它已经开始不断寻找能够推动新收入流的额外服务。BaaS 显然是它看中的一家。
