# 本周编程:Go 2.0 和令人烦恼的复杂性问题

> 原文：<https://thenewstack.io/week-programming-go-2-0-question-complexity/>

虽然 Go 1.9 尚未发布——预计本月将[发布——但在本月早些时候](https://tip.golang.org/doc/go1.9)[在 GopherCon](https://blog.golang.org/toward-go2)发布之后，大多数人都将注意力转移到了 [Go 2.0](https://dave.cheney.net/2016/10/25/introducing-go-2-0) 上。看上面的图片，[出现在 Reddit](https://www.reddit.com/r/golang/comments/6rxfjo/go_2_please_dont_make_it_happen/) 上，这是一些人对 Go 2.0 与传统 Golang 的不同之处的一种解释，传统 Golang 因其简单而为用户所熟知和喜爱。在过去的一周里，社区似乎对即将到来的变化有所了解。

围棋进化争议的核心是围棋 2.0 是否应该支持泛型，正如围棋爱好者戴夫·切尼在这里讨论的。另一篇关于这个话题的[博文](https://dr2chase.wordpress.com/2017/08/09/a-use-case-for-go-generics-in-a-go-compiler/)提供了一个 TL；DR 对这个话题做了很好的总结:“(1)许多其他语言都有泛型，(2)我们非常确定它们是有用的，(3)但是它们伴随着相关的成本和复杂性，我们不确定它们是否值得。”

这种受人尊敬的简单性是冲突的核心，但正如 Go 的联合创始人 Rob Pike 在一次谈话中提到的，简单性是复杂的:
