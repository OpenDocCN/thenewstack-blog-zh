# OpenStack 硅谷煎饼播客:Kubernetes、OpenStack 和 PaaS 2.0

> 原文：<https://thenewstack.io/ossv-pancake-breakfast-kubernetes-openstack-usher-paas-2-0/>

Kubernetes 最终会让 OpenStack 更易用吗？无服务器计算会让 OpenStack 和 Kubernetes 变得没有必要吗？[开放容器倡议](https://www.opencontainers.org/)会完成其容器运行时的 1.0 版本吗？

这些都是在 OpenStack 硅谷 2016 期间在加州山景城[计算机历史博物馆](http://www.computerhistory.org/)举行的最新 TNS Pancake 播客中解决的问题。小组嘉宾包括谷歌产品经理 [Craig McLuckie](https://www.linkedin.com/in/craigmcluckie) ，CoreOS 工程副总裁 [Mike Saparov](https://www.linkedin.com/in/msaparov) ，Mirantis 联合创始人兼首席执行官 [Alex Freedland](https://www.linkedin.com/in/alex-freedland-b4212b) ，以及 Iron.io .联合创始人兼首席执行官 [Chad Arimura。](https://www.linkedin.com/in/chadarimura) TNS 创始人 Alex Williams 主持了该小组。

[#103: OSSV 煎饼早餐、Kubernetes、OpenStack、PaaS 2.0](https://thenewstack.simplecast.com/episodes/103-ossv-pancake-breakfast-kubernetes-openstack-and-paas-2-0)

Freedland 在讨论中提到了容器是如何在 OpenStack 社区中占据一席之地的。最初，OpenStack 开发人员可能将容器视为软件开发世界的小型货车，用于将工作负载从一个地方运送到另一个地方。“您知道，无论您交付什么，无论是 OpenStack 还是 Kubernetes，都可以用类似的方式完成。我认为这是 Docker 和 Kubernetes 革命带来的巨大变化。OpenStack 主要受益于此，”Freedland 说。

无服务器计算在围绕流程编排的讨论中也有一席之地，Arimura 解释说，这有助于引领未来的 PaaS，“无服务器的概念不是没有任何服务器，而是当你开发软件时，你不会想到它们。我认为从这个角度来看，你真的在考虑 PaaS 2.0 或下一代 PaaS。”

McLuckie 强调了发展 Kubernetes 生态系统的重要性，希望任何关注 Kubernetes 的新兴组织都能在实施容器编排平台的过程中吸引用户。Saparov 接着强调了专注于 Kubernetes 的现有组织所面临的问题，“他们面临着技术挑战、人员和人才缺口挑战以及流程挑战。不管是 Kubernetes，云存储，公有云，还是 OpenStack。你必须有一种方式与他们接触，以帮助解决所有这些参数。这实际上是人员、流程和技术三者的结合。”

McLuckie 总结了讨论，强调了谷歌将 Kubernetes 带到[云本地计算基金会](https://www.cncf.io/)的三个关键原因。在公平和保持强烈的技术认同感之后，McLuckie 恳求道，“最重要的是，我想对这里的每个人说的是，我们希望创建一个对最终用户负责的基础，而不是由供应商驱动的基础。”
