# 开源领导者:Kat Marchán 将命令行引入 Npm

> 原文：<https://thenewstack.io/open-source-leaders-kat-marchan/>

下面的采访是“开源领导者”系列的一部分，在这个系列中，我们描述了开源 IT 社区中的项目领导者，以了解他们如何开发他们的软件，以及运行开源项目带来的挑战和好处。

[Kat Marchán](https://github.com/zkat) 是 [Npm 公司](https://www.npmjs.com/)的核心开发人员。，为世界上最大的软件注册中心开发命令行客户端。当 Marchán 不修复折磨终端的错误和实现新的 npm 命令行功能时，她还通过会议讨论和社区活动传播开源软件 node . js/NPM 之爱。《新书库》最近采访了她，了解这个绿发数字树妖是如何生活在开源生态系统中的。

所以十年前你在电影学校。是什么导致了从纪录片制作到 npm CLI 共同维护者的飞跃？

我在汉普郡学院的最后一年开始编写代码，在那里我学习语言学和电影制作。我正在进行一个长达一年的电影项目，这是我的学位论文。我的想法是，我会选择一些完全不同的东西，这样我就可以从这么深的杂草中理清思绪。我最终在史密斯学院学习了一门关于 Python 的 CS 入门课程。这是该地区五所大学系统的一部分，学生可以在任何一所学校上课。我喜欢它，并立即想知道为什么我没有在本科学习计算机科学！

我从电影学校毕业后就直接陷入了经济衰退。找工作非常困难，所以我通过阅读编程书籍、研究开源软件和学习编程来打发时间。我熟悉的第一种编程语言是 Common Lisp。在寻找系统管理员工作时，我把这一切都当作一种爱好，我在大学时曾以勤工俭学的方式做过这份工作。最终，我申请了一家 webdev 代理公司，他们很友好地让我成为一名初级 QA 人员，并给了我偶尔编码的机会。一路上，我又做了几份工作，从马萨诸塞州到明尼苏达州，再到佛罗里达州，最后来到了加利福尼亚。现在，我是使用最广泛的 JavaScript 包管理器 [npm CLI 的两个核心开发人员之一。](https://www.npmjs.com/package/cli)

**你是如何涉足开源的？**

在我开始写代码之前，我真的对计算机很感兴趣，整个大学期间我都在做技术支持工作。所以我第一次知道开源的存在是通过运行最新的发行版，也是通过在 IRC 上闲逛——那里有很多操作系统文化。当我真正开始编程的时候，有一个叫做 [GitHub](https://github.com/) 的新东西，每个人都告诉我去那里。我建了一个账户，开始发布我随机的个人黑客。

当我玩我的第一个编程项目，一个 [MUD](https://en.wikipedia.org/wiki/MUD) (基于文本的多用户视频游戏)时，我第一次意识到开源是多么的酷。所以我深吸一口气，把源代码放了出来——当时我就像一个完全的 OS n00b，对我来说，fork 这个词真的是一件很糟糕的事情。但是后来有人用叉子叉了它，几乎是马上，我几乎开始哭了，因为我立即想到这意味着两件坏事——一些更有经验的编码员将撕毁我的东西，我也有点觉得我正在失去对这个珍贵的个人物品的控制，这些东西一直都是我的，你知道吗？

一天后，他们提出了这个 pull 请求，对我所做的进行了重构，这很棒。我就想，“你是说其他人会来参加我的聚会，帮助我把项目做得更好？!"我被迷住了。

**你能给我们一些你当前开源主项目 [npm CLI](https://www.npmjs.com/package/cli)** 的基本项目统计数据吗？

作为 JavaScript runtime environment node . js 的默认包管理器，npm 由一个命令行客户端(也称为 npm)和一个公共包的在线数据库(称为 npm registry)组成。有两个核心开发人员，丽贝卡·图尔纳和我，他们全职维护这个项目。企业支持大概就是这样。

我们也有一些半定期的贡献者，和一些其他社区成员发送一次性补丁。我们的 AUTHORS.txt 目前有 477 行。

在一个有如此重大影响的项目上工作可能是一种紧张的思考。比如，我可以对 nmp5 安装所需的时间做一个小小的改进，当你考虑到我们的用户群时，这相当于这么多时间——实际上是一周内节省的年数，节省的工时。意识到任何一种小的混乱都会有巨大的影响也是很可怕的。800 万用户是我被告知的数字，所以如果我犯了一个只影响百分之一的错误，80，000 人将无法使用该软件。

**你参与项目**的潜在收益是多少？

我的全职工作是在一个真正可见的开源项目上工作，薪水高达六位数。对此我很高兴。

在更宏观的层面上，Npm，Inc .将开源注册表作为一项免费服务来运行，其目标是支持 JavaScript 社区的长期成功。但该公司通过在私人或企业环境中围绕包的安全使用构建工具和创建服务来赚钱。

**项目帮助解决了哪些技术和/或业务问题**？

Npm 是世界上使用最广泛的软件包管理器，它是致力于 web 开发各个层面的开发人员的核心部分。Npm 是进入由 [npm 注册表](https://docs.npmjs.com/misc/registry)上的 500，000 个包组成的庞大 JavaScript 生态系统的主要门户，这为人们开发自己问题的解决方案节省了大量时间。很有可能，如果你有一个问题，有人已经至少部分解决了它，并在注册表上免费提供。这是迄今为止所有编程语言中最大的软件包生态系统。

我们还有 Orgs 和 [npm Enterprise](https://www.npmjs.com/enterprise) 之类的东西，让开发人员更容易使用他们在开源工作中习惯的工作流来处理内部项目，并让这一切成为相对无缝的体验。

**Random Kat Marchán 仿真陈述？**

我以编写 [Lisp](https://en.wikipedia.org/wiki/Lisp_(programming_language)) 为生，这是我的第一门编程语言。那是非常罕见的。这些天来，JavaScript 对我来说感觉像是一个足够好的 Lisp——它有足够多我过去喜欢的有趣的随机垃圾。
