# 荷兰最大的银行之一是如何进入 CI/CD 的

> 原文：<https://thenewstack.io/how-one-of-the-netherlands-largest-banks-came-to-ci-cd/>

[荷兰最大的银行之一如何获得 CI/CD](https://thenewstack.simplecast.com/episodes/how-one-of-the-netherlands-largest-banks-got-ci-cd)

[Wiebe de Roos](https://nl.linkedin.com/in/wiebe-de-roos) 是 [ABN Amro](https://www.abnamro.nl/nl/prive/index.html) 的持续集成/持续部署(CI/CD)顾问和工程师，该银行是拥有 22，000 多名员工的荷兰最大银行之一，在该银行通过其 DevOps/ [DevSecOps](https://www.devsecops.org/) 计划开始向 CI/CD 转变时，他加入了该公司。管理层决定其开发团队需要进行转变的原因包括延迟交付和应用程序前端的创新不足。“团队并不真正成熟，”Roos 说，他经营着一家名为 [Business IT Nerds](https://www.businessitnerd.com/) 的咨询公司，从 CI/CD 和 DevSecOps 的角度协调业务和 IT 相关项目。

今天，Roos 表示，ABN Amro 的 CI/CD 流程由其 DevOps 和 DevSecOps 提供支持。结果:“自动化和合规性现在作为 CI/CD 流程的一部分推出，以应对市场变化，”Roos 说。“现在我们看到一个大的加速，因为集装箱，现在也被采用。”

五月底在巴塞罗纳举行的 [KubeCon + CloudNativeCon](https://events.linuxfoundation.org/events/kubecon-cloudnativecon-europe-2019/) 会议期间录制的这一期[新的 Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，and 阿姆罗的安全编码和人工智能团队负责人[张秀坤·德·斯密特](https://nl.linkedin.com/in/dominik-de-smit)加入了 Roos。他们与主持人 Alex Williams——The New Stack 的创始人兼编辑兼首席执行官——谈论了 ABN·阿姆罗的 CI/CD、DevOps 和 DevSecOps 之旅。他们还讨论了开发团队使用的工具，如那些为容器和云原生安全性提供的工具，以及它们如何在应对他们面临的现实世界开发、运营和安全挑战中发挥重要作用。

当然，其中一个主要目标是通过 CI/CD 来提高部署交付的速度，以适应文化的变化，这样所有的 DevOps 团队都可以加快软件开发的速度。Roos 说:“有许多遗留应用程序需要调整，而以简单的方式、小型应用程序和团队以及快速迭代来完成这些工作要容易得多。

Roos 说，在工具方面，Jenkins 在精简 350 到 400 个团队的开发过程中发挥了重要作用，与 Jenkins 一起工作。Roos 描述了当他加入公司时，只有几台 Jenkins 服务器在使用。“随着越来越多的团队在 CI/CD 及其管道和 Jenkins 工作方面变得成熟，我们每个月都需要添加更多的服务器。一切都基于虚拟机，添加更多服务器需要很长时间，”Roos 说。“服务器我们超负荷了，很多团队都部署在一台服务器上。因为我们有许多不同的团队，他们有不同的需求，所以会出现一大堆配置冲突。”

ABN Amro 的生产管道服务器过载现在已经成为过去。“我可以说我对此感到非常自豪:詹金斯本地已成为过去，”Roos 说。

DevOps 和 DevSecOps 文化是团队实现必要的成熟度和标准化目标的关键组成部分。“由于这种詹金斯转型，我们也可以突然提供标准管道，”斯密特说，“我的意思是，我们有一个团队来创建这些管道，我们可以真正轻松地实施，例如，在其中进行安全检查。因此，我们可以向团队交付一个管道，团队不必自行配置。”

如果每个团队都使用不同的编排器或管道，“我们将很难管理它，”Smit 说。“因此，这是我们 DevSecOps 成熟背后的一个重要驱动因素。在某些时候，你必须从标准化开始，因为否则，这真的很难。”
