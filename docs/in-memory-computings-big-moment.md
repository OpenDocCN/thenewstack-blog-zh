# 内存计算的重要时刻

> 原文：<https://thenewstack.io/in-memory-computings-big-moment/>

内存计算及其在分布式系统中如何工作的概念反映了开发人员、软件工程师以及人们传统上看待数据库及其在应用程序架构中的作用的方式所面临的新现实。

这也是一个例子，说明软件提供商在一天结束时，总是在寻找最好的硬件来运行他们的软件。它涉及微服务数据存储的进步、应用开发中的机器学习以及科学和商业用例中人工智能的出现。

[太平洋时间](https://www.periscope.tv/thenewstack/?utm_source=thenewstack&utm_medium=website&utm_campaign=InMemoryLivestream-Referral&utm_term=Intel)12 月 7 日上午 11:30，我将与[英特尔](https://www.intel.com/)设计工程师 [Christine Wang](https://www.linkedin.com/in/christine-wang-71731659) 主持一场关于内存计算的直播讨论，内容涵盖内存计算的使用和应用的基础和最新情况。英特尔已经开发了 Optane，它提供了一个持久的内存平台。它已经开发了多年，现在正在获得一些关注。我们将讨论英特尔的方法，并回答关于 Optane 的持久内存技术如何影响我们如何看待 at-scale 技术的开发、部署和管理的问题。

现在越来越清楚的是，硬件和软件需要与横向扩展体系结构进行更深入的集成。它会引出问题和讨论点:

*   新的内存功能实际上可行吗？在容器技术中使用流架构意味着对内存计算的更深层次需求的转变，数据存储可以提供 CPU 和网络之间的平衡。
*   如何设计应用程序才能最好地使用持久内存？为持久存储环境开发的应用程序才刚刚开始。例如，这些考虑归结为如何将永久存储器与 RAM 结合使用。
*   在有持久内存的上下文中，内存计算是什么意思？我们听到了更多关于内存计算在管理微服务方面的价值。区别在于节点可以管理什么，以及在考虑密度和数据总体增长时作为比较的价格。
*   如何在云本地环境中管理分布式内存技术？谁做这种工作？人们对内存架构如何与 Kubernetes 和云原生技术相适应越来越感兴趣。
*   与 Spark、Kafka 等 Apache 软件基础项目有哪些集成？
