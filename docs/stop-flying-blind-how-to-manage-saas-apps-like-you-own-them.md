想象你在飞机上。但是，你的飞行不是由驾驶舱里的飞行员控制，而是由地面上的一大群人指挥，每个人都有不同的角色。一个人操纵油门，另一个人驾驶，另一个人放下起落架。他们都必须无缝协作，这样你才能安全着陆，你不能和他们中的任何一个人说话，看事情进展如何。

很可怕，对吧？

这就是大多数组织在管理关键 SaaS 应用程序时的情况。这些应用程序日益增长的复杂性和分布式特性，加上企业不拥有应用程序基础架构或外部网络和服务依赖关系的事实，意味着他们对工作流几乎没有可见性或控制力。

但无论你是一家使用团队或 Slack 合作的初创公司，一个使用 Salesforce 管理机会的销售团队，还是一个依赖 Shopify 进行在线商店的零售商，你(和我们)都需要 SaaS 应用程序才能工作，这样我们才能工作。您的声誉、客户体验和一般功效都取决于此。

因此，当出现问题时，正如可能发生的那样，你会做些什么来解决它？你打电话给谁？你需要尽快得到答案。但是即使找到这些答案也是一个挑战。

## 传统工具无法胜任管理 SaaS 应用的任务

基于 SaaS 的生产力和协作工具为企业提供了巨大的优势。但是，它们也让负责保持关键业务 SaaS 应用程序性能一致的企业 IT 团队感到担忧。

部分问题在于 SaaS 的应用程序在设计上过于庞大。它们可以跨多个云、区域和数据中心运行，公共互联网在它们的交付基础设施中发挥着关键作用。此外，包括 DNS、secure edge 或 Secure Web Gateway (SGW)服务和第三方 API 在内的外部依赖项都需要正常工作。

此外，解决停机或性能问题的根本原因非常困难。SaaS 提供商通常很难联系到，可能会归咎于网络。传统的 NPM 工具是为您所拥有的网络而构建的，而 APM 监控技术依赖于将代码(通常是 Javascript)注入到应用程序中来测量用户行为和时间——但是当所讨论的应用程序不是您的时，这些方法就不起作用了。它们也不能提供对整个应用程序连通性的详细理解。

尽管如此，有效的 SaaS 监控解决方案对于企业来说是必备的。 [Gartner](https://www.gartner.com/en/documents/3957027) 在一份报告中预测，“零可见性将越来越被视为不可接受，因此，SaaS 监控将成为几乎所有大中型组织的 IT 运营工具集中的标准配置。”

对 SaaS 应用程序性能问题的容忍度正在下降。我们已经看到这一点，因为最近的事件使我们比以往任何时候都更依赖于 SaaS 的良好运转——想想我们现在对表现不佳的视频会议或商业通信平台的耐心有多低。

简而言之，监控 SaaS 应用基础设施至关重要，但传统方法不起作用。所以现在你可能想知道——那么有效的解决方案是什么样的呢？

## 强大的 SaaS 监控解决方案需要的关键能力

为了应对这些挑战并有效地监控 SaaS 应用程序，IT 团队需要能够做好两件事的解决方案。首先，他们需要深入了解这些动态环境，以便主动确定网络性能基准。接下来，他们应该能够触发复杂的脚本来导航用户旅程并比较页面性能，这样您就可以看到并衡量用户通过应用程序进行的基础架构之旅。

如果你没有一些活跃的东西，让你深入了解互联网、云、边缘和应用本身，你就是在大海捞针。

幸运的是，随着 SaaS 应用的增加，有效监控它们的选择也在增加。像管理自己的应用一样管理 SaaS 应用已经成为可能，这意味着您可以采取更主动的方法来优化性能，并为用户提供更好的数字体验。

这意味着你不再需要盲目飞行。这让 ITops 团队、企业和用户都松了一口气。
