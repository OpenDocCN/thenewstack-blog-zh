# 持续创新的同时降低云数据成本的 7 种方法

对于企业来说，这是一个困难的时期，它们需要在经济衰退的威胁下严格控制成本，同时仍然在技术上进行足够的投资以保持竞争力。公共云使按需扩展和缩减容量变得很容易，但对看似无限的资源的访问也使使用量和成本快速且不可预测地增加。

分析和机器学习项目尤其如此。数据湖是机器学习和流式分析的理想选择，是企业开发新产品和更好地服务客户的有力途径。但是，随着数据团队能够轻松地在云中启动新项目，必须密切管理基础设施，以确保每项资源的成本都得到优化，每一分钱都花得值。在当前的经济环境下，没有企业能够容忍浪费。

但是企业不是无能为力的。强大的财务治理实践允许数据团队控制甚至降低他们的云成本，同时仍然允许创新发生。创建适当的防护栏，防止团队使用超过他们需要的资源，并确保工作负载与正确的实例类型相匹配，以优化节省，这将大大减少浪费，同时确保满足关键的 SLA。

以下是首席信息官可以用来管理云数据湖成本的七个最佳实践。这将有助于避免不可预测的账单，并在这个不确定的时期控制支出，但仍然允许你的公司创新，并在另一方面变得更强大。

*   **班长，班长，班长。**成本管理始于准确了解哪些资源正在被使用、何时被使用以及被谁使用，并至少每天进行跟踪。通过在作业、集群和用户级别密切跟踪使用情况，您可以立即发现浪费或低效情况，并做出必要的更改。你不能管理你看不到的东西。
*   **使用异构集群。**集群中的节点可以是不同的实例类型，这取决于不同实例的工作负载和成本/可用性。例如，一个集群可以包含按需实例和 AWS Spot 实例，或者 Google [可抢占虚拟机](https://cloud.google.com/preemptible-vms)。应用工具(通常以 DIY 脚本的形式)来自动化使用，以便在满足应用程序的弹性和可用性需求的同时，使用最有价值的基础架构。
*   **主动自动缩放。**集群在不使用时不需要运行，需要时自动扩展和关闭集群将节省可观的成本。我们至少有一个客户在疫情期间闲置 15 分钟后关闭了集群，以大幅降低成本。这显然取决于应用程序的 SLA 需求，但是对于开发工作和概念验证工作来说，等待群集重启应该不成问题。
*   **测试不同的发动机。许多企业采用多种决策引擎，如 Spark、Hive 和 Presto，因为它们都适合不同的工作负载类型。在多个引擎上测试查询，看它们在哪里运行得最快。这有利于最终用户提高性能，也有利于您的公司，因为更快的查询时间通常意味着您使用的资源更少。**
*   **使用基于时间表的生命周期管理。**自动创建和销毁系统，以匹配使用模式。例如，如果集群通常在早上或高峰交易时段达到最大容量，那么可以查看正在运行的作业，并了解哪些作业可以在一天中分布。
*   **调整未充分利用的基础设施的大小。**推出新项目和应用程序时，容量要求并不总是很明确。基础架构被过度配置，当现实世界的需求变得清晰时，没有人回去改变它。将基础设施的规模调整到适当的水平。这需要仔细制定策略，因为容量仍必须考虑到预期的使用高峰。
*   教育你的用户。如果数据团队了解更大的业务需求和可用的选项，他们将尽自己的职责帮助降低成本。他们真的需要一个 r 4.4x 大型实例来进行概念验证项目吗？大概不会。与他们谈论当前的形势，以及为什么适当调整基础架构符合每个人的利益。我们的一个客户甚至和他们的团队一起做了一个练习，用彩色的 Leggo 砖块来说明不同的实例类型如何在一个集群中使用。帮助你的团队来帮助你。

云中的成本管理是关于优化利用率，同时提供财务护栏，允许团队在自助服务环境中快速移动，同时防止意外成本。这些最佳实践应该在任何组织中持续采用，但在当前的宏观经济环境下尤为重要。使用这些技巧，你将能够度过这次危机，并以良好的财务健康状态出现。
