# 思科预览面向 Kubernetes 的托管 HyperFlex 应用平台

> 原文：<https://thenewstack.io/cisco-previews-managed-hyperflex-application-platform-for-kubernetes/>

长期以来，思科一直是一个与网络硬件相关的名字，但如今，该公司正在继续向其客户(无论是物理客户还是虚拟客户)提供网络和基础设施，即将发布的三种工具被称为“闭环运营模式”

更具体地说，该公司目前正在与选定的客户测试其 [AppDynamics 客户旅程地图](https://www.appdynamics.com/blog/product/experience-journey-map/)、 [Intersight 工作负载优化器](https://blogs.cisco.com/tag/cisco-intersight-workload-optimizer)和[用于 Kubernetes](https://blogs.cisco.com/datacenter/introducing-the-hyperflex-application-platform) 的 HyperFlex 应用平台，并表示这三款产品预计将在未来几个月内全面上市。前两个产品旨在提供对应用程序性能的洞察和优化，甚至以成本等业务指标为目标。与此同时，Kubernetes 的 HyperFlex 应用平台是思科新的托管 Kubernetes 产品，它不仅提供“交钥匙”Kubernetes 平台，还提供许多其他托管服务，包括容器网络、容器存储、入口和 L7 负载平衡器、日志记录、监控、容器注册和服务网格。

思科产品管理副总裁 Kaustubh Das 解释说，该版本旨在解决最近出现的孤立应用和基础设施团队难以合作的问题。

“IT 的组织结构图在过去几年里已经发生了变化，”Das 告诉新堆栈。“应用程序团队负责运行应用程序，并将应用程序的运行状况和正常运行时间保持在一定水平。基础架构团队的职责是保持服务器、网络和云基础架构在特定级别上正常运行。这两个团队已经成为运营孤岛，当发生一些事情时，会有很多作战室，而这些不是有助于客户体验的组织孤岛。”

Das 表示，思科对这一问题的解决方案是创建“一个具有公共数据集的闭环运营模型”，使用公共术语和工具来弥合这些团队之间的差距。

他说:“使用通用数据、通用词汇和通用工具，我们能够在这些团队之间架起一座桥梁，他们也能够在驻留在公共云、本地或混合环境中的应用程序之间架起一座桥梁。”。

这三个工具组成了这个“闭环”，它主要关注基础架构、存储和计算的各个方面与团队之间的互联性。例如，AppDynamics 和 Cisco inter sight Workload Optimizer 为应用团队和基础设施团队提供了影响性能、用户体验和业务影响的各个方面的“共享视图”。Cisco inter sight Workload Optimizer 同样能够洞察混合应用架构的性能、成本和合规性，再次缩小了众多不同团队之间的差距。Das 解释说，所有这三种产品都是思科将基础设施视为代码的历史的结果。

“基础设施即代码需要一些基本原则，我们大约在 10 年前就开始了这一旅程。我们说，听着，服务器是一块铁，服务器的个性将被编码在代码和策略中，这些已经成为长期的基础。Das 表示:“我们将这种技术应用于融合基础设施和超融合基础设施，现在我们将这种技术应用到云中。“基础设施的代码驻留在云中，这为我们提供了无限的范围和无限的规模。它还为我们提供了许多遥测数据，我们可以分析并说，这个基础设施，我注意到有些事情正在出错。我可以主动补救未来的潜在问题。”
