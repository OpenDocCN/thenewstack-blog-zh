# 通过自动化简化 Kubernetes

> 原文：<https://thenewstack.io/simplifying-kubernetes-through-automation/>

巴伦西亚——管理运行容器的云虚拟机(VM)。运行[数据密集型工作负载](/category/data/)。扩展服务以应对流量高峰，但这样做不会增加您组织的云支出。 [Kubernetes (K8s)](/category/kubernetes/) 一开始看起来很简单，但它带来的挑战会让你越来越复杂。

[云原生](/category/cloud-native/)生态系统充满了旨在让开发者、数据科学家和运营工程师更容易应对这些挑战的工具。自动化越来越成为帮助团队和他们的公司更快、更安全、更有成效地工作的秘方。

在 [KubeCon + CloudNativeCon EU](https://www.cncf.io/kubecon-cloudnativecon-events/?utm_content=inline-mention) 录制的新 Stack Makers 播客特别上路版中，我们揭示了自动化帮助简化 Kubernetes 的一些方式。NetApp 邀请了来自 [Spot.io 的三位嘉宾:Apache Spark 海洋高级产品经理](https://spot.io/?utm_content=inline-mention)让-伊夫“JY”斯蒂芬[，以及 Spot.io 产品经理兼产品架构师](https://www.linkedin.com/in/jystephan/)[吉拉德·沙哈尔](https://www.linkedin.com/in/gilad-shahar-0320a589/)和[亚林·闫品](https://www.linkedin.com/in/yarin-pinyan-385b28176/)

[通过自动化简化 Kubernetes](https://thenewstack.simplecast.com/episodes/simplifying-kubernetes-through-automation)

Stephan 指出，直到最近， [Apache Spark](https://spark.apache.org/) ，用于大规模数据处理的开源统一分析引擎，还不能部署在 K8s 上。“所以所有这些普通的软件工程师都从 Kubernetes，云原生解决方案中获得了很酷的技术，”他说。“而大数据工程师，他们还停留在 10 年前的技术上。”

他说，Spot.io 让 Apache Spark 在 Kubernetes 上运行:“它对开发者更友好，更灵活，也更具成本效益。”

该公司的海洋光盘，预计将于 8 月普遍上市，旨在解决另一个库伯内特公司的问题，闫品说:金丝雀部署。

他说:“以前，如果你运行的是普通的虚拟机，没有 Kubernetes，进行 canary 部署是相当容易的，因为你必须纵向扩展一个虚拟机，然后看看新版本是否在其上运行良好，然后逐步扩展其他虚拟机。在 Kubernetes 中，这相当复杂，因为您必须处理许多 pod 和部署。"

沙哈尔说，在企业中，开发人员和 SRE 团队成员很可能服务于众多的开发人员，尽可能多地为开发人员自动化工作是必不可少的。例如，Spot.io 的工具允许用户“将配置分成几个部分”，他说，这可以让开发人员承担多少比例的配置责任，这被认为是最适合他们的用例的。

“我们试图以一种方式设计我们的解决方案，允许 devo PS[团队]一次设置好东西，并基本上为开发人员提供预烤的解决方案，”他说。"因为开发人员最清楚他们的应用程序需要什么."
