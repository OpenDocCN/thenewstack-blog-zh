# CloudNativeSecurityCon:左移陷入安全困境

> 原文：<https://thenewstack.io/cloudnativesecuritycon-shifting-left-into-security-trouble/>

西雅图:在这里的第一届 CloudNativeSecurityCon 上，好消息是云原生计算使得创建和交付软件的速度比以往任何时候都快。坏消息是，安全问题来得同样快。

在第一个主题演讲中，[云计算原生计算基金会](https://cncf.io/?utm_content=inline-mention) (CNCF)执行董事 [Priyanka Sharma](https://www.linkedin.com/in/pritianka/) 表示，尽管[容器](https://thenewstack.io/containers/)、[微服务](https://thenewstack.io/microservices-101/)和 [Kubernetes](https://thenewstack.io/kubernetes/) 已经使我们能够通过[持续集成/持续交付(CI/CD)](https://thenewstack.io/how-continuous-integration-and-continuous-delivery-ci-cd-enhances-devops/) 以创纪录的速度构建和部署程序，将工作转移到左边，但同时，它也导致“更多暴露的边缘和节点，最终会受到攻击

## 怎么办？

怎么办？Sharma 建议我们使用相同的新时代云原生工具来解决我们的安全问题。在这件事上我们没有太多选择。毕竟，“安全不是一蹴而就的任务。说到安全，没有人是一座孤岛。这是一个持续的对话，因为现在情况非常糟糕。我们什么都不做的成本非常高。”

多少钱？Sharma 说:“一个组织的平均违规成本是 380 万美元。如果看看私有云，这个数字会上升到 420 万美元。然后是公共云，超过 500 万美元。而这还只是平均成本。这不是一件好事，尤其是在我们今天面临的时代。”

为了解决这个问题，我们必须解决我们训练不足和团队间缺乏协作的问题。“孤立的团队经常在不同的国家、不同的时区使用不同的工具和策略框架工作”是导致安全漏洞的一个因素。Sharma 继续说，“我们相信安全是人的力量”，这意味着让所有人都在同一个安全页面上。

此外，“当我们作为一个知识渊博的供应商中立社区合作开发工具和流程来保护我们的系统时，我们都受益，”马莎指出。这意味着，“从业者和开发人员应该分享他们的[开发和部署](https://thenewstack.io/foundries-io-a-platform-for-iot-development-and-deployment/)专业知识。我们可以互相教导。我们都是全球实干家团队。当我们一起工作时，我们覆盖的范围比任何一个组织都要广。如果这听起来很熟悉，那应该是。这是开源开发的基本原则。

当然，现在比以前更难了。正如夏尔马指出的那样，“如今，安全确实是一个多维度的问题。”但是，如果我们齐心协力，支持社区安全工作和行业协作，我们就能保护我们的项目。但这是可以做到的。像 CloudNativeSecurityCon 这样汇集了 700 多名开发人员和安全专家的努力是一个很好的跳板。
