# 用工程管理和机器学习取代脚本

> 原文：<https://thenewstack.io/supplant-scripting-with-engineering-management-and-machine-learning/>

[驾驭](https://harness.io/ci-cd-platform-overview/)赞助本帖。

软件开发人员和工程师继续编写或运行脚本来将组件粘合到工作流中，即使这是一项耗时的任务。然而，对许多人来说，采用机器学习或其他新技术来取代这些屡试不爽的脚本可能是一个挑战。换句话说，很难说服工程师改变他们的工作方式。

在本期[的新堆栈制造商](/podcasts/makers)播客中，软件交付平台提供商布道者 Tiffany Jachja[Harness](https://harness.io/ci-cd-platform-overview/)和 Oracle Cloud 高级产品经理 [Rajsi Rana](https://www.linkedin.com/in/rajsirana) 讨论了脚本以及机器学习、CI/CD 和其他流程如何帮助引导工程文化的转变，以充分利用时间和资源。《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了这一集。

[用工程管理和机器学习取代脚本](https://thenewstack.simplecast.com/episodes/supplanting-scripting-with-engineering-management-and-machine-learning)

“我们身处一个不断变化和创新的行业，但人们仍然受习惯的驱使。因此，对于进入这个行业的人来说，当他们的确切事情自动化在当时还不存在时，很难说服他们这是未来，这是前进的方向，”拉纳说。“我认为阻力表现在多方面。它可以表现为不参与或缺乏兴趣这样简单的事情，也可以表现为更加微妙和积极的事情，例如积极反对或说，“不，这不是我真正感兴趣做的事情。”"

拉纳说，改变过时的脚本和其他习惯的解决方案包括“在组织中找到值得尊敬的人和能够真正影响变化的人”。“从自下而上的角度，而不是自上而下的角度来看，当变革是一个指令时，它通常不会很好地工作。当变革是一种集体认同时，它真的很有效——它培养了一种集体心态，并考虑到了每个人的想法、感受和观点。”

对于已经部署了 CI/CD 系统的 DevOps 团队，Harness 平台支持这些遗留系统，同时帮助组织扩展其 CI/CD。Harness 最近还收购了 [Drone](https://drone.io/) ，该公司为构建和测试软件自动化 CI，以帮助“识别痛点，并真正为不同的组织提供更好的软件交付，”Jachja 说。

“当我们谈论裁谈会时，我们实际上只需要对此持友好态度。因为无论你部署的是什么，无论是容器映像、WAR 文件还是 ML 模型，无论发生了什么，我们都要把它交到你的客户手中，”Jachja 说。“因此，我们在这里帮助您解决这个问题，因此，对我们来说，Drone 完善了我们正在构建的 CI/CD 生态系统，以帮助客户-因为，归根结底，我们在这里是一个软件交付平台。”
