# Docker 和“F”字

> 原文：<https://thenewstack.io/rumored-docker-fork-fud-moves-conversation-forward/>

在第 113 期[新堆栈分析师播客](/tag/the-new-stack-analysts/)的这一期惊喜节目中，我们快速阅读了我们故事的余波，“[一个 Docker 分叉:拆分的讨论现在在桌面上](https://thenewstack.io/docker-fork-talk-split-now-table/)”，它描述了一些知名容器生态系统玩家讨论是否分叉 Docker 代码库的传言。

这个故事在互联网上引起了[相当多](https://news.ycombinator.com/item?id=12388721)的[讨论](https://www.linkedin.com/pulse/forking-docker-daniel-riek)，所以我们邀请了 451 Research 的研究主管 Donnie Berkholz 和 Red Hat 的社区发展主管参加 [OpenShift](https://www.openshift.com/) 、 [Diane Mueller](https://twitter.com/pythondj) ，以获得关于假定分叉的更一般的背景。听着:

[#113:传闻中的 Docker Fork 和将话题向前推进的 FUD](https://thenewstack.simplecast.com/episodes/113-the-rumored-docker-fork-and-the-fud-that-moves-the-conversation-forward)

“在过去的一年里，Docker 发生了很多变化，”Berkholz 说，并指出，作为一家风险投资公司，Docker 处于协调其需求的艰难境地，以在一个不断增长和竞争激烈的集装箱社区中开拓自己的一席之地。

“他们不得不在许多方面与他们的生态系统竞争，以提升堆栈，从 Docker 本身开始，到编排层，再到管理层。伯克霍尔茨说:“他们必须努力跟上，因为他们致力于对 Docker 进行创新，他们已经有了一个庞大的群体，这些人要么与他们并行工作，要么在他们之上工作，试图增加价值，就像 Kubernetes 或 Mesos 一样。”。

她说，虽然 Red Hat 是参与谈判的公司之一，但穆勒本人并不知道有任何讨论在进行。她确实注意到，关于分叉的讨论“带来了很多关于保持社区团结的问题”, Docker 迄今为止在维护和培育社区方面做得非常出色。

“如果这是真的，我对分叉的担心是，当分叉发生时，社区最终会损失，”她认为。

Berkholz 指出，Docker 在其关于如何改进其容器的愿景中吹捧的方向有很大的不同，并将其与使用 Docker 容器支持企业客户的第三方供应商进行了比较。

Berkholz 说，第三方供应商希望“优化稳定的企业核心”，而 Docker 本身希望快速创新，并尽快为用户提供功能。

“这是任何两者都存在的分支的标志:这两个群体之间在哲学和分化方面存在一些无法调和的真正差异，”他说。

作为一名最优秀的分析师，Berkholz 发现了一篇研究开源软件分叉的研究论文。观察 224 个分支，他们发现 44%是成功的分支，两者都存活了下来。当分叉者和分叉者之间存在不可调和的分歧时，这些成功最常发生。

“没有人对多克有任何重大敌意。他们是一家公司。他们必须生存下来，做他们自己的事情，这很好。伯克霍尔茨说:“这只是为所有前进的道路找到正确的道路。

“这感觉就像是一个时刻，在许多开源项目中，这是一个自然的时刻，特别是那些像 Docker 这样有一个良好开端的项目，因为总会有一场关于谁将成为仁慈的独裁者的混战，”Mueller 说。
