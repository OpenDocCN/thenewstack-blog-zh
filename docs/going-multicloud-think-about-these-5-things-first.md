# 走向多云？首先考虑这 5 件事

我们即将迎来 2022 年，而 [Gartner 预测](https://www.gartner.com/en/doc/375973-comparing-multicloud-management-and-governance-approaches)到目前为止，75%使用 IaaS(基础设施即服务)的企业已经实施了精心设计的多云战略。现在，多云的优势对所有组织来说都很明显，从避免供应商锁定，到更好的灾难准备，甚至是有竞争力的价格，58%的人将其称为多云策略的最大优势。

然而，没有两个组织是完全一样的，尽管多元化有很多潜在的好处，但适合一个企业的可能不适合另一个企业。在进行多云飞跃之前，这里有五点需要评估。

## 1.充分利用多供应商产品

使用多种云与拥有多种云策略是有区别的。第一种方法可以简单地通过使用使用不同云基础架构的多种基于云的产品，或者通过允许特定团队采用适合其业务用例的新云供应商技术来实现。相比之下，第二种模式涉及到一个紧密结合的云战略，并且通常需要一个卓越的云中心来整合。如果没有多云战略，您的云环境就像孤岛一样存在，您看不到多云的真正优势，例如:

*   **冗余:**仅仅说“我们使用多种云，所以我们可以在发生故障时得到保护”是不够的您需要在事件响应计划中设置规则和策略，以使一个云能够接管另一个失败的地方。
*   **备份**:使用您的多云环境实现最佳实践备份，例如确保数据备份在正确的地理位置以满足合规性，或者使用一个云提供商作为另一个的备份。
*   **云工具:**根据您的研究，哪家提供商最符合您的需求？例如，如果你的内部系统严重依赖 Windows 操作系统，你可能会发现 Azure 的一些元素很有用，而 GCP 以机器学习而闻名。也许您可以通过在 AWS 上运行某些工作负载来降低成本。你甚至可以找到针对特定行业的利基云服务，比如针对医疗保健的[tres orit](https://tresorit.com/)，它可以帮助克服一些监管上的束缚和障碍。

## 2.仔细考虑安全性

多云确实不太安全，但它确实采取了不同的安全方法。同样，孤岛在这里不是你的朋友，所以用自己的云原生工具保护每个云并认为你的工作已经完成是没有意义的。您的云环境之间的差距和通信将是您的弱点，因此请为您的整个环境制定一个计划，就像您在保护一个单一的边界一样。

这从以集中的方式定义您的所有身份和访问管理开始，而不是从您的“主要”或第一个云开始并从那里扩展。考虑用户角色、最低权限，以及如何加密和监控数据访问和通信以及人工访问。别忘了监控和维护。无论您使用什么来进行更新和补丁管理，请确保它有一定的上下文，以便所有升级都与工作负载、IaaS 以及任何外部或内部依赖项保持一致。

## 3.严格控制你的费用

如果做得好，多云可以帮你省钱，但是你需要很多监督来确保你充分利用这些好处。当你注册云时，有许多听起来很有吸引力的时髦词，如弹性扩展、按需付费、敏捷性和自动化，但这些都不会自动发生。

您需要利用实例和工作负载的自动扩展和适当调整、折扣定价(如与业务相关的保留实例或现场实例)、电源调度(非关键工作负载在非高峰时间关闭)以及了解哪些是空闲的和需要完全删除的可见性。现在，将这一数字乘以适用于每个云提供商的数字。总之？如果你还没有建立云 FinOps 团队，现在就是你一直在等待的转折点。

## 4.确保你有合适的人

当谈到多云战略时，缺乏内部技能可能是一个真正的绊脚石。事实上， [85%的组织报告在云基础架构人才和专业知识方面存在不足](https://clients.451research.com/reports/100509)。到目前为止，我们谈到的每个领域，从安全性和合规性，到开发和 FinOps，都需要知道自己在做什么的人来掌舵。我们可能听起来像一个破纪录，但多重云意味着加倍努力，有更多的角色需要填补。

如果你不确定是否能吸引到合适的候选人，这并不意味着你的多云战略就此止步。考虑对现有员工进行再培训(这是一项很好的工作场所福利)，或者依靠托管服务来处理您觉得最没有信心的云扩展部分。

## 5.努力降低人工智能的复杂性

在许多情况下，您可能会发现已经有一种工具或技术可以填补您企业的重要空白，并帮助您的多云战略取得更大的成功。在基础设施层面上，这可能就像构建微服务和容器一样简单，以允许跨云和内部的可移植性。同样，出于同样的原因，开发人员也倾向于无服务器计算。

下一波解决方案将超越基础架构本身，解决监控和管理云蔓延带来的日益严峻的挑战，例如安全性和成本节约。目前，[只有 11%的公司对其云环境的可见性和监控感到满意](https://www.splunk.com/en_us/form/451-research-it-monitoring-meltdown.html)。为了确保他们不会完全被淹没，公司需要帮助。这不仅仅是找到一两个专家成员；这是关于转向更大程度地依赖人工智能来支持人类云工程师。有了这一点，企业可以在重要的环境中获得实时见解(再见警报疲劳)，并利用人工智能驱动的技术，该技术可以从多个来源获取数据，并代表他们做出明智的决策(你好，咖啡休息)。

多重云风靡一时，而且理由充分。借助强大的战略和正确的工具，您可以消除对供应商锁定的担忧，并开始受益于成本节约、灵活性和减少的漏洞。
