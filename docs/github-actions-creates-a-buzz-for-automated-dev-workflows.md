# GitHub Actions 为自动化开发工作流创造了一个热点

> 原文：<https://thenewstack.io/github-actions-creates-a-buzz-for-automated-dev-workflows/>

本周在三藩市举行的年度用户大会上，GitHub 推出了一款新产品，[【GitHub Actions】](https://github.com/actions)，它允许开发者创建小代码，这些代码可用于触发代码库服务中的事件或一组事件。

GitHub 平台负责人[萨姆·兰伯特](https://www.linkedin.com/in/isamlambert/)，[在一次采访中表示，GitHub 希望创造一个环境，让开发者能够专注于他们最感兴趣的事情，即编写代码。  “我们的问题是，‘我们如何让开发人员保持在最佳状态？’答案是除了编写代码本身之外，自动化代码生产所需的一切。](https://www.github.com)

“我们采用 CI/CD 并在从创意到生产的整个流程中应用原则，为工作流程带来灵活性，”Lambert 说。  “你可以用动作做 CI/CD，但这样就多了。”

例如，您可以将所有手动步骤包装到 can action 中，以将您的代码  推送到云服务。动作也适用于第三方软件:使用[launch darky](https://launchdarkly.com)，您可以使用一个动作来翻转功能标志，只需关闭一个问题。  稍后您可以使用单个操作在 Terraform 中批准和应用操作。

开发人员可以使用预先做好的动作，或者用任何语言编写自己的动作。  动作成为 GitHub 代码库的一部分，因此一旦编写完成，所有开发者都可以使用。

GitHub 产品设计高级总监 Max Schoening 解释说，开发人员可以创建自己的工作流程。  “你可以想链多少就链多少，可以从 GitHub 平台的任何事件触发一个工作流。”他警告说，对于 beta 来说，50 岁是有上限的，他指出，这意味着一次要把很多动作串起来。

关键是行动是简单的、可操作的和可重复的。唯一的要求就是要有一个顶级 docker 文件。  “动作让你指向别人的 repo，一个已有的 docker 文件，或者一个路径，动作就会相应的表现出来。这对于开源的可能性和生态系统来说是一个全新的难题，”微软、  的高级开发人员 Sara Drasner 在[的一篇文章中写道，这篇文章举例说明了如何编写自己的操作。](https://css-tricks.com/introducing-github-actions/)

周二的传言很快传开了。Adobe 首席开发人员 Pawan Mittal 在推特上写道“这真是太棒了。”其他人同样热情:

这就提出了这个产品将如何改变 DevOps 前景的问题。在即将到来的 [TNS 上下文播客](https://thenewstack.io/podcasts/context)中，TNS 创始人 [Alex Williams](https://twitter.com/alexwilliams) 想知道这将如何影响 CI/CD 平台提供商。  从 CloudBees 到 CircleCI 的任何人，以及市场上所有其他简化管道流程的平台，都可能受到影响。他不是唯一一个。

回到 GitHub，Lambert 很兴奋地看到开发人员将在这个游戏围栏中做什么。  “我期待着人们会做出所有我们无法预见的东西，”他说。
