# 开源安全的现状如何？别问了。

> 原文：<https://thenewstack.io/whats-the-state-of-open-source-security-dont-ask/>

德克萨斯州奥斯汀。—几乎每个组织都使用的[开源](/category/cloud-native/)软件有多安全？你可能不想知道，[根据由](https://openssf.org/blog/2022/06/21/state-of-open-source-security-2022-from-snyk-and-the-linux-foundation/)[Linux 基金会](https://training.linuxfoundation.org/training/course-catalog/?utm_content=inline-mention)和 Snyk，一家云本土网络安全公司，在基金会年度[开源峰会北美](https://events.linuxfoundation.org/open-source-summit-north-america/)上发布的调查结果，六月在这里举行。

根据这项研究，在接受调查的 500 多家组织中，41%的组织对他们使用的开源软件的安全性没有很高的信心。只有一半的参与公司说他们有解决开源问题的安全政策。

此外，与 2018 年版的调查报告相比，修复漏洞所需的天数增加了一倍多(98 天)。

这项研究是应 Linux 基金会的项目[开源安全基金会(OpenSSF)](/inside-a-150-million-plan-for-open-source-software-security/) 的要求进行的。在《新堆栈制作者之旅》的这一集里，Linux 基金会研究副总裁史蒂夫·亨德里克和 Snyk 开发者关系总监马特·贾维斯接受了 TNS 特写编辑希瑟·乔斯林的采访。

我们的客人说，尽管有令人震惊的统计数据，贾维斯告诫不要把所有的漏洞都当成四级火警。

[开源安全状况如何？别问了。](https://thenewstack.simplecast.com/episodes/whats-the-state-of-open-source-security-dont-ask)

“设定零漏洞目标可能是不现实的，因为并不是所有的漏洞都被同等对待，”贾维斯说。在您的特定环境中，一些“漏洞”不一定是风险。最好将重点放在对您的网络、应用程序和数据最关键的威胁上。

新报告中的一个亮点是:近四分之一的受访者表示，他们正在寻找资源来帮助他们保持开源软件以及所有依赖于它的软件的安全。也许与供应商更相关:62%的调查参与者表示，他们正在寻求使用更加智能的安全工具。

“从过程的角度来看，他们有很多责任，”亨德里克说。“但是他们很快就跟上了潮流，并说，我们希望供应商社区在为我们提供工具方面做得更好，这使我们的生活更加轻松。因为我认为每个人都认识到，解决安全问题需要付出比我们现在投入的更多的努力。”

## 跳上“SBOM 潮流”

亨德里克说，许多组织似乎仍然分不清他们使用的开源软件哪些是直接依赖，哪些是传递依赖。他说，澄清事情的最好方法之一是“加入 SBOM 的潮流”

[理解一个开源工具的软件材料清单，或 SBOM](/securing-the-software-supply-chain-with-a-software-bill-of-materials/) ，是“会给你对组件的深刻理解，会给你可用性，会给你信任，你会知道组件是无虚化的，”亨德里克说。

“因此，从能够处理今天到处都在发生的整个组件化问题的角度来看，这是绝对关键的。
