# 新的栈环境:开源安全的两种观点

> 原文：<https://thenewstack.io/the-new-stack-context-two-views-of-open-source-security/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。本周，我们邀请了两位嘉宾，他们讨论了保护开源软件的挑战。

首先，我们采访了哈佛商学院教授兼普查 II 项目联合主任 [Frank Nagle](https://www.hbs.edu/faculty/Pages/profile.aspx?facId=566431) ，他与 [Linux 基金会的核心基础设施倡议](https://www.coreinfrastructure.org/) (CII)合作评估开源软件组件的安全实践。这份名为“核心中的漏洞”的报告，是开源软件的初步报告和普查 II(Census II)，试图了解“现代供应链中的结构和安全复杂性，其中开源无处不在，但并不总是被理解。”我们讨论了他的发现，包括大多数开源项目管理不足的惊人状态。

我们还采访了 Neeraj Poddar，他是 Aspen Mesh 的联合创始人和工程主管，在 Istio service mesh 项目中开发了针对最近漏洞的修补程序。我们与 Poddar 讨论了当有如此多的利益相关者时，快速修复开源程序中的 bug 所面临的挑战，以及在准备好发布公告之前，应该如何封锁这项工作。

[第 105 集:开源安全的两种观点](https://thenewstack.simplecast.com/episodes/episode-105-two-views-of-open-source-security)

我们也讨论一些更相关的播客和最近的新闻故事。我们宣布播客同伴 [TNS 分析师](https://thenewstack.io/podcasts/analysts/)的回归，以及在那里关于[Kubernetes](/improving-developer-happiness-on-kubernetes-but-first-who-does-configuration/)开发者体验的对话。我们还热烈讨论了卡耐基梅隆大学的一名研究人员，他[揭穿了 10x 程序员](/the-veracity-and-relevance-of-the-10x-programmer/)的神话。我们深入研究了 Platform9、Rancher Labs 等公司和 Google Cloud 等服务如何让 Kubernetes 变得更容易使用。最后，在人工智能领域，我们来看看一个深度神经网络重建了隐藏在毕加索画作中的[神秘图像。](/deep-neural-network-ai-reconstructs-mysterious-image-hidden-in-picasso-painting/)

TNS 编辑和营销总监 Libby Clark 主持了这一集，同时主持的还有创始人兼出版商 Alex Williams 和 TNS 执行主编 Joab Jackson。

Aspen Mesh 和 Linux 基金会是新堆栈的赞助商。
