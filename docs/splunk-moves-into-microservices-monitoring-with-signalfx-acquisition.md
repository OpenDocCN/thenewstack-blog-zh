# Splunk 通过收购 SignalFX 进军微服务监控领域

> 原文：<https://thenewstack.io/splunk-moves-into-microservices-monitoring-with-signalfx-acquisition/>

据两家公司称，应用性能管理(APM)软件提供商 [Splunk](https://www.splunk.com/) 正在以大约 10.5 亿美元的价格收购 [SignalFX](https://www.signalfx.com/) ，希望扩展到困难的领域[微服务可观察性](/observability-a-3-year-retrospective/)。

SignalFx 提供托管服务，提供实时监控和指标，可应用于新兴市场的[云基础设施](/category/cloud-services/)和[微服务](/category/microservices/)，以及常规应用。Splunk 开始提供一个流行的设备来搜索性能问题的日志，尽管扩展到了全系列的 APM 服务，包括针对 AIOps 和 IT 运营管理(ITOM)的服务。

Splunk 总裁兼首席执行官道格·梅里特在一份声明中表示:“数据推动了现代业务，收购 SignalFx 无疑将 Splunk 置于大规模监控和可观察性领域的领导者地位。”。据 Splunk 称，合并后的 Splunk 和 SignalFx 数据平台将提供单一界面，实时监控和观察这些系统产生的所有数据。

SignalFx 创始人兼首席执行官 [Karthik Rau](https://www.linkedin.com/in/krrau/) 在一份声明中补充道:“随着世界继续向复杂的云优先架构发展，Splunk 和 SignalFX 是实时监控和观察云原生基础设施和应用所需的新方法，无论是通过日志、指标还是跟踪。

此次收购是对 Docker、Kubernetes、serverless 和其他云原生技术快速增长的生态系统的直接回应。IT 分析公司 Garter [预测](https://www.gartner.com/en/documents/3902966/best-practices-for-running-containers-and-kubernetes-in-)到 2022 年，超过 75%的组织将在生产中运行容器化的应用程序。在 Splunk 看来，这些技术在确保高可用性和无缝运营方面为 IT 专业人员和开发人员带来了新的挑战。

“在传统的企业架构中，你有一个整体，它运行在一台服务器上。Rau 在 2018 年的播客采访中告诉新堆栈[。“今天的体系结构越来越分散，您有虚拟机、容器、分布式体系结构和分布式数据库，它们可能运行在数十或数百个节点上。因此，一个实例上发生的事情不如整个实例集合上发生的事情有趣，因为性能对整个服务至关重要。”](/signalfx-cto-microservices-must-make-radical-performance-gains/)

该交易预计将在 2020 年下半年完成，取决于惯常的成交条件和监管审查。
