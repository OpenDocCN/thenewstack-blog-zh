如今，与 DevOps 社区中的任何人交谈，“云原生”这几个词会一次又一次地出现。如果你是持续集成和持续交付的新手，你可能听说过“构建云原生应用”这句话但说真的，这意味着什么？这只是我们已经在做的事情的另一个时髦说法吗？

最基本的，云原生应用包含关键元素。DevOps 团队采用微服务设计方法，他们在容器上编排这些服务。为了交付这些应用和多种微服务，DevOps 利用自动化和连续交付。它们构建在云之上，为云开发并部署到云。结果呢？云原生应用由支持敏捷性的容错块组成，可以快速交付和迭代。

您还可以看到云原生计算基金会([【CNCF】](https://www.cncf.io/))如何定义云原生应用:

*   云原生技术使组织能够在现代动态环境(如公共云、私有云和混合云)中构建和运行可扩展的应用程序；
*   容器、服务网格、微服务、不可变基础设施和声明式 API 就是这种方法的例子；
*   这些技术使得松散耦合的系统具有弹性、可管理性和可观察性。与强大的自动化相结合，它们允许工程师以最少的劳动频繁地、可预测地做出高影响力的改变。

## 需要什么

那么，为什么所有这些如此重要呢？作为一名开发人员，如果您想要构建云原生应用程序，您需要了解它们的基础知识。为了指导你，有几个必备的:

*   **弹性。**云原生 app 需要保证服务，无论如何。开发人员需要为失败设计应用程序。假设最坏的情况；
*   **可重用服务。**架构服务，以便它们可以跨应用程序和由其他服务使用。这也需要你有接口契约和发现机制；
*   **可扩展性。**为了有效运行，水平可扩展性至关重要。确保底层计算、存储、数据库和网络资源以及应用程序逻辑。支持扩展到多个服务实例以满足需求。

云原生应用利用了云技术的世界，因此它们是专门为在云环境中工作而设计的。可扩展性和弹性的内在优势使得构建云原生应用成为对开发人员有吸引力的提议。

在 CI/CD 领域，构建云原生应用程序可能是一个需要多种工具的紧张过程。其中，他们需要一种 DevOps 方法，在这种方法中，开发人员和运营人员的需求是一致且互补的。开发人员和运营人员进行协作，或者管道、基础设施和工具支持全栈开发，在这种开发中，开发人员可以编码、构建、测试和部署到生产环境中，而无需跨职能依赖。

该技术还需要微服务。单个微服务往往比传统应用程序更小，更易于管理，而跨服务的通信以及管理服务的生命周期可能是复杂而具有挑战性的。

既然您已经了解了热门话题是什么，并且想要开始开发云原生应用程序，那么该如何开始呢？如果你是个人或者小团队的一部分，最明显的地方就是从 [詹金斯 X](https://www.cloudbees.com/jenkins/jenkins-x) 开始。Jenkins X 是由包括 [James Strachan](https://github.com/jstrachan) 在内的许多 CloudBees 团队成员开发和引入的 Jenkins 子项目，是开发人员在 Kubernetes 环境中启动和运行构建云原生应用的一种方式。
