# 收购 Red Hat 将是 IBM 在云计算领域未来的关键

> 原文：<https://thenewstack.io/red-hat-acquisition-will-be-the-key-to-ibms-future-in-the-cloud/>

IBM 认为自己是人工智能领域的先驱——想想沃森赢得了 Jeopardy 和深蓝击败了加里·卡斯帕罗夫——以及云，它在十年前[开始](https://www-03.ibm.com/press/us/en/pressrelease/22613.wss)推广云的深蓝广告自此主导了电视广告。

但就像 IBM 未能利用的许多创新一样，它很快就落后了。Watson AI 变成了恐龙，亚马逊、谷歌、微软在云端从 IBM 身边呼啸而过。

IBM 必须做一些事情来获得相关性，特别是在云领域，它认为这是一个“战略要务”，或者说是其未来的支柱。周日宣布的 340 亿美元收购 Red Hat 的计划就是一个例子，这是 IBM 有史以来最大的一次收购，并为该公司的云，人工智能和硬件系统提供了一个后端，这些系统随着 Power 芯片和 Z14 大型机的出现而反弹。

通过 Red Hat，IBM 获得了世界第二大服务器操作系统，以及一些用于各种云实现的高级后端技术。除了同名的企业 Linux 操作系统，IBM 还将拥有 Red Hat 的 OpenStack 平台和云套件，其中包括 Docker 和 Kubernetes 的应用开发工具。Red Hat 中间件也在 AWS、谷歌和微软 Azure 中使用，IBM 将间接接管这些产品。IBM 现在还将拥有 CoreOS，该公司于 2018 年初被红帽以 2.5 亿美元收购。

客户可能会选择完全远离云服务中的 Red Hat 中间件，但 IBM 正在采取措施确保客户没有理由担心。红帽将是一个独立的单位，IBM 将继续支持红帽在谷歌云、AWS 和微软 Azure 中的中间件。IBM 保证将保留红帽的开源精神、产品组合和“独特的开发文化”

但 IBM 在以前的云收购中有着好坏参半的记录，最突出的例子是 SoftLayer，一家云基础设施提供商，于 2013 年被收购，但其服务——后来被更名为 Bluemix——最终被扼杀并整合到现有的云、存储和其他产品中。2018 年第三财季，IBM 技术服务和云平台部门的收入为 83 亿美元，同比下降 2%。

Red Hat 还满足了该公司收购一个重要的 Linux 发行版的需求，它可以称之为自己的发行版，这可能是其在服务器和云中所有硬件和软件产品的默认操作系统。IBM 一直试图说服人们放弃其硬件中的 Unix 操作系统，转而使用 Linux，并为此投入了数十亿美元。IBM 已经与 Red Hat 在服务器和大型机领域合作了几十年，但拥有操作系统更好。

IBM 不得不继续捍卫 AIX 的未来——它为大型机和基于 Power 的服务器开发的基于 Unix 的老化操作系统——但是毫不避讳地说 Linux 是它硬件的未来。它的 Power 芯片在设计时考虑到了 Linux，IBM 为其大型机硬件提供了 Red Hat 和 SUSE，随着最近的 Z14，大型机硬件正在复兴。红帽可能会取代 IBM 自己的 LinuxONE，这是去年为大型机推出的。流行的基于 Unix 的 IBM Z 大型机的 zOS 将继续存在，尽管 Linux 是首选。

现在的问题是 IBM 如何将这些整合在一起。该公司正试图将自己从一家老古董转变为一家以千禧一代为推动力的预科生公司。将 Red Hat 作为一个独立的部门是明智的，尤其是考虑到将过去的收购整合到 IBM 的传统文化中的难度。如果 IBM 继续保持 Red Hat 的独立性——就像它成功收购了 Weather Channel 的资产一样——这次收购可能会推动其硬件和软件策略的重大变化。

IBM 没有回复对其整合策略和 Red Hat 产品计划的评论请求。
