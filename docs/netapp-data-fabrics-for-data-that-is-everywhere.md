# NetApp:无处不在的数据的数据结构

> 原文：<https://thenewstack.io/netapp-data-fabrics-for-data-that-is-everywhere/>

NetApp 赞助了这次播客。

就在不久前，数据结构还主要被归入组织为数据管理制定的各种策略中。这个概念在很大程度上局限于确保数据在它应该在的地方，并存储在正确的位置。时至今日，数据结构通常与任务关键型分析紧密相关，通常跨越多云和内部环境的混合环境。当然，DevOps 团队越来越依赖数据结构来帮助支持 DevOps 运营和文化。

在本期[新堆栈制造商](https://thenewstack.io/podcasts/makers)播客中， [NetApp 云数据服务](https://cloud.netapp.com/our-cloud-story)产品经理 Neil Stanley 和[NetApp 云解决方案高级产品营销经理 Jason Blosil](https://www.linkedin.com/in/jasonblosil) 讲述了数据结构在当今的意义，以及 NetApp 如何调整其架构方法以应对现代 DevOps 挑战。

[NetApp:无处不在的数据的数据结构](https://thenewstack.simplecast.com/episodes/netapp-data-fabrics-for-data-that-is-everywhere)

Blosil 说，传统上，数据结构通常是在孤立的环境中实现的，即使已经努力在数据中心内共享基础设施。“我认为我们已经看到的是，我们开始看到一个混合云世界或云驱动的世界，那里有发生同样事情的风险，”Blosil 说。

Blosil 说，通常今天，一个组织可能会在亚马逊和谷歌云服务、内部和私有云中进行部署。因此，现在你有一个情况，我又有筒仓，但他们是在不同的云和本地的一些数据的形式。因此，数据结构实际上是解决这个问题的一种尝试。因此，您可以说，“我如何管理整个环境和整个 it 环境，使我能够以更易于使用、消费和部署的方式简化编排和部署应用程序，而不是再次重复我的孤岛式体系结构，在它自己的空间中有一堆数据和应用程序，我不必独立于其他应用程序进行管理。"

NetApp 对数据结构的部分强调是“数据引力”在云服务中所起的作用，在云服务中，数据存在于不同的云服务中，而客户的架构中也有数据。

“NetApp 一直试图通过我们的技术来实现最高效的数据存储方式，真正有助于降低数据的重要性，并使其更具移动性，”Stanley 说。“因此，围绕数据结构的治理理念不仅仅是在公共或私有云内拥有一个数据结构，它实际上与组成数据结构的那些组件的更大概念更相关。”
