# Kubernetes 部署和安全模式电子书现已推出

> 原文：<https://thenewstack.io/kubernetes-deployment-security-patterns-ebook-now-available/>

今天，新书库发布了我们 Kubernetes 生态系统系列的第二本电子书，“Kubernetes 部署安全模式”在本书中，我们的重点是告知部署和管理容器化应用程序所需的集群的操作团队。随着越来越多的工作负载转移到基于 Kubernetes 的容器部署，IT 运营团队需要一个了解这一庞大而复杂的技术的视角。

“Kubernetes 部署和安全模式”有三个全面的章节，通过关注您的组织对安全性、存储、日志记录和负载平衡等因素的重视，帮助运营团队评估 Kubernetes。这些因素有助于定义公司如何管理工作负载，无论是基于云服务、内部部署还是通过多云方法。该书的亮点包括:

*   ### 1)关于组织和部署规模如何决定 K8s 挑战的深入数据分析。

一个组织的规模和他们在哪里运行他们的容器工作负载在很多方面决定了他们部署 Kubernetes 的可能性以及他们在部署时将面临的挑战。正如我们在两项调查的结果中看到的，一项由云计算原生计算基金会进行，另一项由新堆栈进行，大型组织在生产中运行更多的容器，因此更有可能部署 Kubernetes。(您可以在 Google 电子表格中访问该分析的原始数据集。)在拥有 1，000 名或更多员工的大型组织中，负责运行容器集群的运营团队最关心安全性、网络和复杂性，而中小型组织则更关心存储和监控。

*   ### 2)部署环境对比。

在概述了任何 Kubernetes 生产部署中的关键构建模块之后，第 2 章讨论了用户责任如何在自托管环境、托管提供商和公共云中变化。同样，评估这些信息的最佳视角是您组织的特定需求。一个方便的表格比较和对比了各种选项，并显示自托管 Kubernetes 部署提供了最大的灵活性，但成本非常高。当部署外包给托管提供商时，定制的灵活性就会下降。这种失去控制的情况相应降低了总拥有成本(TCO)。用于托管 Kubernetes、CaaS 和 PaaS 的各种工具也可以简化部署。第 2 章介绍了这些，以及新兴的部署模式，如无服务器。

*   ### 3)适用于所有 K8s 部署的安全模式。

所有 Kubernetes 部署的一个共同点是需要理解和集成新的、特定于 Kubernetes 的安全模式。第 3 章讨论了适用于 Kubernetes 部署的威胁模型和 Kubernetes 的各种安全考虑，包括容器安全、部署安全和网络安全。这些考虑跨越了底层节点、容器映像和注册表。务必要特别注意处理容器映像上的关键漏洞，并对整个 DevOps 管道进行漏洞评估。管理注册表是另一个需要重点关注的关键领域，以确保适当的控制，从而只允许部署经过批准的映像。It 团队有责任确保不会从互联网上下载图像供组织内部使用。
