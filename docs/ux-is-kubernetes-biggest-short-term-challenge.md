# UX 是 Kubernetes 最大的短期挑战

> 原文：<https://thenewstack.io/ux-is-kubernetes-biggest-short-term-challenge/>

根据新堆栈在 KubeCon 和 CloudNativeCon 的[覆盖面](https://thenewstack.io/tns-at-kccnc-2019-san-diego/)之前对 100 多人的调查，用户体验(UX)是 Kubernetes 社区需要关注的地方。总体而言，19%的人认为管理员和操作员体验是核心 Kubernetes 项目在下一年需要解决的首要问题，另有 17%的人提到了开发人员体验。难怪 Kubernetes 相关的营销往往是为了让 DevOps 的角色更容易。

尽管有人抱怨设置和运行 Kubernetes 的复杂性，但它已经获得了广泛的采用。尽管大量公司涌入提供 Kubernetes 服务和产品，厨师联合创始人[亚当雅各布](https://twitter.com/adamhjk/status/1194701449625686016)认为这是“从用户体验的角度来看，热垃圾。”他认为，在米兰蒂斯[收购 Docker Enterprise](https://thenewstack.io/mirantis-acquires-docker-enterprise/)之后，Docker 公司的剩余部分应该“重组，回归到在集装箱顶部提供最好的开发者 UX”

其他挑战更具技术性。13%的人认为多租户问题需要解决。另有 10%的人认为存储需要改进，尽管这一发现可能受到我们最近出版的一本名为“[面向开发人员的云原生存储新方法”的电子书的影响。](https://thenewstack.io/ebooks/storage/state-of-state-cloud-native-storage-for-developers/)”

Kubernetes 的老员工仍然关心如何简化集群管理。与此同时，越来越多的开发人员利用 Kubernetes 中的这些集群来协调工作负载。从长远来看，可能需要改进开发人员的工作流程，以获得新一轮的容器采用。
