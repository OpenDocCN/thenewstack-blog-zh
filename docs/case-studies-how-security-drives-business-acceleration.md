麦肯锡的上述引用体现了为什么企业正成群结队地向云迁移的任何原因。简而言之，云本身正在超大规模增长，因为 cxo 需要它。在 2019 年的[报告](https://www.mckinsey.com/~/media/McKinsey/Business%20Functions/McKinsey%20Digital/Our%20Insights/Unlocking%20business%20acceleration%20in%20a%20hybrid%20cloud%20world/Unlocking-business-acceleration-in-a-hybrid-cloud-world-Summary.ashx)中，麦肯锡发现首席执行官的两大优先事项是收入加速、提高敏捷性和更快的上市时间。这些发现与安全专业人员高度相关，因为它们强调了不仅需要现代化工具，更重要的是，需要现代化安全团队的结构。

作为帕洛阿尔托网络公司公共云的首席安全官，我获得了与世界各地的安全团队合作的绝佳机会。为了揭示一些最大的组织如何构建他们的云安全团队来响应 CxO 需求，我策划了(并匿名)两个案例研究。

## 案例研究#1:从目标开始

一名在全球动物健康公司工作的安全经理负责保护 Azure 和 AWS 云中云基础设施的快速构建。他最初从他的团队中指派了一名安全系统管理员作为云领导，但很快意识到系统管理员超出了他的能力范围。

安全经理说，在他们每周的项目审查中，他和系统管理员就系统管理员如何与他在云业务办公室的同事互动进行了多次交谈。安全经理会给系统管理员一些建议，告诉他如何更好地倡导安全，并确保事情安全地进行。尽管他意识到由于他没有花时间预先记录从安全角度看成功是什么样子，他没有把系统管理员放在成功的位置上。

在接下来的一周里，安全经理清空了他的日程表，并与系统管理员密切合作，记录他们的云安全策略。

安全经理说，他们为每个领域创建了成功指标和里程碑。结果对他们两人以及最终参与进来的团队其他成员来说都是 100%清楚的。又过了两个星期，另一件事变得明显了:他在内部没有知识和技能来跟上他们的开发和 IT 团队的步伐。两人与一个值得信赖的合作伙伴合作，引入短期资源来填补关键的技能缺口。

回顾项目的前三个月，安全经理意识到，如果他花时间深入了解项目需求，他就可以从一开始就提供帮助。他说，他完全低估了转变云安全方式所需的知识和时间。他认为他们可以利用大部分现有的流程和安全工具，但事实并非如此。最终，他们选择了[云本地安全平台](https://www.paloaltonetworks.com/cyberpedia/what-is-a-cloud-native-security-platform) (CNSP)。

## 案例研究#2:安全之外的协作

一家全球专业服务公司的云安全总监负责组建云安全团队，建立全球云安全标准，并与 IT 部门合作部署新的安全控制措施。她面临的最大挑战是试图找出如何在组建新团队的同时自己完成所有这些工作。

她过去作为安全架构师做过许多云安全项目，但这是她第一次从头到尾拥有它。她说，她真的觉得自己与安全团队的其他成员以及 IT 部门隔绝了。尽管她向 CISO 报告，但除了“确保云安全”之外，她没有从他那里得到任何指示

她知道她不能独自完成所有的工作，因为她需要建立一个云安全团队。云安全主管必须快速执行，而不是像架构师一样思考。

在接下来的一个月里，她安排了多次与安全、IT、风险、合规、财务和法律部门同事的一对一会谈。她从这些谈话中收集到的东西是无价的。她不仅建立了关系和盟友，还能够将拥有几个共同目标的团队聚集在一起:业务加速、管理风险、控制云支出和法规遵从性。有了这个对需求的新观点，她开始制定她的策略、里程碑和资源需求。

她说，一旦她记录了项目的这些重要元素，她就和她的经理(CISO)见面，并用四张幻灯片展示了她的战略。在根据他的反馈做了一些调整后，她获得了继续进行的许可。

在接下来的六个月里，她雇佣了三名全职员工和两名合同工。她还与她上任第一个月遇到的同事举行了两周一次的简报会。最后，安全主管说，云安全团队的组建和安全标准的实施是成功的，因为她有一个包括来自安全部门外部的反馈的计划。如果她试图孤立地做这件事，人们只能想象结果将远远不是最佳的。

## 团队组成是业务加速的关键

正如以上两个案例研究所示，如何构建您的云安全团队对于实现业务加速至关重要。[麦肯锡指出](https://www.mckinsey.com/~/media/McKinsey/Business%20Functions/McKinsey%20Digital/Our%20Insights/Unlocking%20business%20acceleration%20in%20a%20hybrid%20cloud%20world/Unlocking-business-acceleration-in-a-hybrid-cloud-world-Summary.ashx)52%的首席信息官认为安全要求和合规性约束是他们实现敏捷性目标的最大挑战。面对这一挑战，安全团队处于非常有利的位置。有了正确的组织结构，他们不仅可以缓解这些 CxO 挑战，还可以提前预测和解决这些挑战。
