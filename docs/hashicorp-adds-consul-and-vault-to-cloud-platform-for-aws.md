# HashiCorp 将 Consul 和 Vault 添加到 AWS 云平台

> 原文：<https://thenewstack.io/hashicorp-adds-consul-and-vault-to-cloud-platform-for-aws/>

基础设施软件和服务提供商 HashiCorp 继续其使命，使其整个平台和工具组合在云上可用，最新的举措是 hashi corp[consult](https://www.consul.io/)最近的全面可用，以及其受欢迎的 [Vault](https://www.hashicorp.com/products/vault) secrets 安全管理工具的测试版，现可在亚马逊网络服务(AWS)上使用。

在“HCP 领事”的命名下，哈希公司的云服务网络和服务网格产品也是哈希公司云平台的第一个普遍可用的服务，标志着哈希公司云平台雄心的重要一步。

“在云基础架构上运行应用程序有助于组织实现业务转型，但也带来了新的挑战，从基础架构复杂性、技能短缺到新型安全威胁。产品营销副总裁 [Amith Nair](https://www.linkedin.com/in/amith-nair-11b793) 、 [HashiCorp](https://www.hashicorp.com/?utm_content=inline-mention) 告诉 New Stack。“我们的客户和社区一直要求我们提供能够帮助他们解决这些挑战的工具。HashiCorp 云平台是专为应对这些挑战而打造的产品。”

HashiCorp 去年介绍了 HCP 版本的 Vault 将如何允许组织采用更灵活的定价模式，而不是支付固定费用直接下载、安装和管理 Vault 的传统方式。

该公司宣布，HCP 领事和 HCP 金库现在都可以在亚马逊网络服务上使用，此前该公司承诺将 HCP 提供给 Azure 和 GCP(没有透露确切的时间表)。HashiCorp 还计划最终在 HCP 上提供其所有 HashiCorp 产品，包括其最近宣布的 [Boundary](http://boundaryproject.io) ，这是一个基于身份的访问管理的开源项目。

奈尔说，HCP 设计的目标是“大幅削减我们客户的运营负担，因为它是由哈希公司的专家管理的”。

“随着第一批 HCP 服务——HCP 领事和 HCP 金库的推出，个人和团队现在可以专注于安全地构建云原生应用程序并将关键工作负载迁移到云上，”奈尔说。“他们可以用更少的资源更快地向云迁移。”

虽然服务网格越来越被视为管理复杂的云原生环境的必要条件，但服务网格的管理本身也可能带来复杂的运营相关挑战。HashiCorp 的 HCP 平台旨在应对这些挑战，为组织提供一种针对云基础架构的自助式替代方案，这些组织可能还希望能够更多地关注软件开发和部署，减少需要管理的运营相关任务。

“与其他依赖于第三方插件的自我管理解决方案不同，HCP 领事由哈希公司的专家构建和运行，以降低提供服务发现或运行服务网格的操作复杂性，”奈尔说。“现在，个人可以按需使用 Consul，企业可以将 Consul 作为共享服务使用，而无需自己维护它。”

Nair 说，与 HCP 领事类似，HCP 保险库旨在允许组织利用保险库，“没有管理费用、人工维护和相关成本”。“他们可以从云环境中获得秘密管理和加密的好处，而无需设置和管理新工具，”Nair 说。
