在[云铸造基金会](https://www.cloudfoundry.org/)，我们对每天使用和构建云铸造平台的开发人员做出承诺。我们实现这一目标的一部分是支持各种技术，以便 Cloud Foundry 能够继续成为适应性最强、最具发展性的云原生平台。

对于我们，我们的成员和贡献者，这意味着我们接受互操作性。我们的云应用平台不仅由我们开发的技术定义，还由在云铸造社区之外启动的各种创新性开源项目定义。

> Cloud Foundry 仍然专注于提供多平台、多云和多语言的强大资源。

随着技术的成熟，我们会定期将它们添加到平台中。这种方法得到了我们许多企业最终用户的赞赏，他们希望利用最新的云原生技术，但对建立一个专注于识别、跟踪和维护这些技术的团队不感兴趣。对于担心被锁定在不再满足其需求的封闭或限制性平台上的 It 领导者来说，这也是一个受欢迎的变化。

## 少担心，多发展

Cloud Foundry 是开发者最好的平台。当我们考虑基金会和平台如何继续发展时，我们每天都会牢记这一愿景。我们认为，云原生平台应该为开发人员配备尖端技术，以及一个不断迭代和尽可能多地自动化任务的框架。

> 开发人员有很多问题需要筛选。担心将各种技术结合在一起不应该是其中之一。

不受约束地访问中央框架对开发者来说是一个巨大的好处。通过不断发展和壮大，Cloud Foundry 使得利用新技术变得简单。

我知道这是一种信念的飞跃，就像第一次踏入一辆自动驾驶的汽车。突然之间，您无法选择应用程序的部署位置，也无法选择如何动态地进行小的配置和故障排除。

但就像无人驾驶汽车如何自动驾驶一样，我们正在将人类排除在等式之外。这不仅使 it 更加安全可靠，还降低了组织面临数据泄露和人为错误的风险。最重要的是，它允许开发人员在更短的时间内完成更大规模的目标。

## 为什么互操作性至关重要

如果您的组织使用内部构建的解决方案运行自己的环境，这些挑战将直接落在业务上。谁在维持运营？在哪里管理？这些挑战会给最成熟、最有远见的企业带来不必要的压力。

当 IT 领导者更习惯于在容器中运行他们的应用程序，更少担心幕后发生的事情时，他们就可以自由灵活地将精力集中在构建出色的应用程序上。这就是我们最终在这里的目的，对吗？企业渴望并依靠应用程序发展，这些应用程序通过保持领先地位来改进流程并提高他们的成功能力。

为什么不让我们消除这种担忧呢？我们 Cloud Foundry 基金会密切关注云原生技术的发展，Cloud Foundry 是一个在关键更新可用时负责更新的平台。当一个巨大的安全漏洞被发现时，就像最近的 [runC](https://www.openwall.com/lists/oss-security/2019/02/11/2) 的情况一样，Cloud Foundry 将在几分钟内让[对其进行修补和更新——当然，没有停机时间。](https://twitter.com/rseroter/status/1095390734318997504)

整合所有这些技术的价值不仅仅在于可访问性；它是关于减少理解和维护新的(和快速发展的)技术带来的担忧。安全漏洞是已知的。Cloud Foundry 让您高枕无忧，因为我们支持您。
