# 现成的黑客:更多的 Linux 命令行命令

> 原文：<https://thenewstack.io/off-the-shelf-hacker-more-linux-command-line-commands/>

树莓派是为学习如何编程而发明的。你可以加载几乎任何你能想到的语言，而且相对较低的主板价格使这项技术人人都可以使用。接上像样的电源、键盘和鼠标，再用一根电缆连上 HDMI 大屏幕，你就可以开始工作了。

作为现成的黑客，我们将机制、硬件、软件、技术、流程、美学和工具整合到物理计算原型和有趣的项目中。因此，我们的编程需求适度地局限于相当简单的面向用户的程序和微控制器固件。强大的应用程序、可移植性和工具对我们来说很重要。

Linux 是每个 Pi 内置的强大工具箱。打开盖子，就可以看到基于文本的命令行，这是 Pi 上最有用的工具之一。

几周前，我开始了一个新的[系列故事](https://thenewstack.io/off-the-shelf-hacker-embrace-the-linux-command-line/)，介绍命令行。我们涵盖了 **ls** 、 **cd** 、 **pwd** 和 **cat** 。本周我们将会看到更多的命令，以及使命令行如此有效的一点点设计理念。

## 正文飞快地过去了

您可能还记得，cat 命令将一个文件打印到屏幕上。不幸的是，较大的文件滚动得太快，你根本无法阅读它们。如果能够一次读取一页，并在文件中上下移动，岂不是很棒？为什么不用[更多的](http://man7.org/linux/man-pages/man1/more.1.html)命令？

在命令行输入“**more”**和文件名，你的愿望就会实现。使用空格键前进到下一页。想后退一页吗？键入字母“b”。

请注意，使用空格键和“b”是组织命令行为的一种非常合理的方式。这是如何做事的[“Unix 哲学”](https://en.wikipedia.org/wiki/Unix_philosophy)的一部分。Unix 最初是基于终端的系统，使用键盘进行输入。没有图形用户界面，没有高分辨率显示器，甚至没有鼠标。那时候编程是非常乏味的，所以任何可以加速输入代码、测试和调试的方法都有助于工作流程。“b”隐喻的意思是“返回”，也在空格键的正上方，可以很容易地从标准的家用 QWERTY 键盘手的位置键入。如果你能触摸时间的话。Linux 继承了许多老派的 Unix 传统，因为它们有意义并且能够正常工作。

我在初中学会了触摸打字，甚至从未想过几年后我会作为一名 it 专业人士和技术作家广泛使用它。永远不要忽视那些对未来可能的技术机会的强烈预感，即使你还没有把任何点联系起来。

不管是在 Pi 上还是在 Linux 笔记本上，我的命令行上经常使用 cat 和更多的东西。

## 回收命令

另一种提高命令行效率的方法是简单地减少输入。

假设您有一个复杂的命令行，带有一堆选项和一个相当长的文件名。也许您需要多次运行该命令。每次重新输入相同的命令是极其乏味的，而且几乎肯定会出错。

当您输入命令时，Linux 会捕获您的命令，并将它们存储在一个顺序队列中。在空白命令行中，只需按向上箭头键，一次一个地返回命令。出现所需命令时，键入“Enter”键执行该命令。一旦您在队列中，您可以使用向下箭头键移回到您最近的命令行。通过使用 control-C 组合键，完全脱离队列并为新命令做好准备。然后，键入您的新命令。自然，它将作为最新的一个被推入命令行队列。

顺便说一下，您还可以使用左右箭头键以及 delete 和 insert 键来编辑调用的命令。当您想要更改几个字母或数字并运行该命令时，这一功能非常方便。

## 回忆历史

命令行队列也可以通过[历史](https://www.geeksforgeeks.org/history-command-in-linux-with-examples/)命令访问。不要用向上箭头在 200 个命令中后退，在命令行中键入“history ”,然后在终端窗口中使用滚动条或鼠标滚轮来找到您想要的命令。点按“终端”窗口顶部的最大化图标，以获得以前命令的完整显示。

当您找到想要运行的命令行时，请查看屏幕的左边缘并记下行号。打一个"**！**"字符后面紧跟着要执行命令的行号。这可能看起来工作量很大，但是一旦你习惯了这个过程，它真的很有效率。

历史的一个奇怪之处在于，只要你输入“**！**"和行号后跟回车键，该命令将立即执行。
