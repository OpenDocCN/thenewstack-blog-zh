# 自动化可追溯到 100 年前的基础设施

> 原文：<https://thenewstack.io/automating-infrastructure-that-dates-back-100-years/>

大规模部署需要大规模扩展能力和强大的基础设施来服务数千万甚至数亿用户。云原生环境提供了根据需要进行扩展的可靠性和性能，通常适用于多种内部和多云环境。在许多方面，电信公司通常需要这些聚合的扩展和计算能力，而分布式功能 Kubernetes 和 microservices 提供了这些功能，既可以向数百万客户提供超大规模的语音、数据和媒体应用，又可以保持对通常已有几十年历史的基础设施的运营控制。

在新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中， [Bill Mulligan](https://www.linkedin.com/in/bamulligan/?originalSubdomain=de) 在帮助客户通过 Kubernetes 和运营商为 [Loodse](https://www.loodse.com/) 实现 IT 运营自动化方面发挥了关键作用，他讨论了自己的背景——从威斯康星大学麦迪逊分校(University of Wisconsin-Madison)经过牛津大学(University of Oxford)到他今天在柏林的生活——以及云原生技术在支持电信公司和其他 Loodse 客户方面发挥的关键作用。

[自动化可追溯到 100 年前的基础设施——与 Loodse 的 Bill Mulligan 合作](https://thenewstack.simplecast.com/episodes/automating-infrastructure-that-dates-back-100-years-w-bill-mulligan-of-loodse)

Mulligan 说,[牛津互联网研究所](https://www.oii.ox.ac.uk/)是“一个非常酷的机构，我认为它确实是跨学科的，这是我在世界上其他地方几乎没有见过的。”在那里，Mulligan 写了他关于科学家如何将人工智能用于癌症研究的论文。“所以，我的科学背景是一种结合人工智能的新互联网技术，”穆里根说。“所以，牛津是一个非常酷的地方，在那里，我可以重新定义自己，投入到我真正感兴趣的事情中。”

穆里根的第一份工作是在柏林一家现已倒闭的初创公司，在那里，他对 Kubernetes 的介绍无疑是一次火的洗礼。在他的第一天，Mulligan 被要求建立一个 Kubernetes 集群。“我当时就想，‘那么，库伯内特斯——我们现在要去某个希腊岛屿，这是第一次公司度假吗，还是我错过了什么，’”穆里根说。“八个小时后，有了一台 Windows PC、一点点 putty 和亚马逊网络服务(AWS)，我实际上有了一个正常工作的 Kubernetes 集群。所以，我真的真的非常非常艰难地学会了 Kubernetes。”

Mulligan 将 Loodse 描述为一家 Kubernetes 和云本地公司。“我们真正关注的是帮助公司自动化他们的 IT 运营，这样他们就可以通过软件而不是人来扩展，”Mulligan 说。“我们这样做的方式是使用 Kubernetes 和其他云原生技术。”

Mulligan 说，电信公司也是 Loodse 业务的一大部分，因为电信公司越来越多地将其运营和基础设施转移到云原生平台。这一举措通常被认为是必不可少的，特别是对于传统的电话运营商来说，他们必须想出如何过渡他们的基础设施，以支持远远超出传统语音和电话服务的业务。成功的电信公司通常会提供广泛的数据密集型通信服务，除了语音应用之外，还包括广泛的视频、网络基础设施和媒体产品。

然而，关键问题包括保持与可追溯到 100 年前的基础设施的互操作性，以及实现文化和思维方式的改变。“对于我们遇到的每一个客户，最大的挑战不是技术，而是人们愿意学习新事物。最大的挑战是真正改变流程和实际文化，”穆里根说。

Mulligan 说，当然，CI/CD 作为一种全新部署很容易做到。然而，对于电信公司来说，“你可能需要集成到 10 年、50 年或 100 年前的产品中，你需要能够做到这一点，”Mulligan 说。

“所以，不仅仅是他们自己，还有很多组织动力，我认为他们现在真的很难克服，”Mulligan 说。
