# 总部位于 Jenkins 的 CloudBees 收购 Codeship 以充实 CI/CD 产品组合

> 原文：<https://thenewstack.io/jenkins-based-cloudbees-acquires-codeship-fill-ci-cd-portfolio/>

两家公司周二宣布，为了充实其持续集成和交付(CI/CD)工具和服务的组合，基于 Jenkins 的提供商 [CloudBees](https://www.cloudbees.com/) 收购了基于云的 CI/CD 供应商 [Codeship](https://codeship.com/features/pro) 。

Codeship 联合创始人 [Moritz Plassnig](https://twitter.com/moritzplassnig) 在关于收购的博客文章[中指出，加入每家公司的服务系列非常有意义:Codeship 提供简单、自以为是的基于云的 CI/CD 平台即服务，而 CloudBees 的产品，如](https://rollout.io/blog/codeship-acquired-by-cloudbees/)[cloud bees Jenkins Enterprise](https://www.cloudbees.com/products/cloudbees-jenkins-enterprise)，更加灵活和可定制，针对更大的内部环境或私有云企业使用。

“通过合作，我们可以为整个市场、所有垂直行业、所有类型的应用和所有类型的技术提供服务。我们的目标是通过提供一系列产品来满足所有需求，从而为所有人提供一流的 CI/CD，”他写道。

Codeship 服务的现有订户将不会看到中断，Codeship Basic、Codeship Pro 和 Codeship Enterprise 将继续提供。不会强制过渡到 CloudBees 或 Jenkins 产品——Codeship 用户不会被迫使用 Jenkins，如果他们选择不使用的话。两家公司指出，随着时间的推移，CloudBees 和 Codeship 平台将合并为一个统一的产品，涵盖一些公司和大型企业的云和内部部署。

“通过联合我们的力量和投资组合，我们正在形成一个 CI/CD 发电站，能够满足所有需求:从一个人的团队到拥有数万名开发人员的公司，”CloudBees 首席执行官[Sacha laboury](https://www.cloudbees.com/team/sacha-labourey)在 CloudBees 博客上写道[。](https://www.cloudbees.com/blog/codeship-and-cloudbees-join-forces)
