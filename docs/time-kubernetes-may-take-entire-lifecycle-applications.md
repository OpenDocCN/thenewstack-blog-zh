# 随着时间的推移，Kubernetes 可能会接管应用程序的整个生命周期

> 原文：<https://thenewstack.io/time-kubernetes-may-take-entire-lifecycle-applications/>

[随着时间的推移，Kubernetes 可能会接管应用的整个生命周期](https://thenewstack.simplecast.com/episodes/over-time-kubernetes-may-take-over-the-entire-lifecycle-of-applications)

CoreOS 从一开始就处于集装箱化的生态系统中。事实上，它是第一家围绕 Kubernetes 开源容器编排软件引入企业产品(构造)的公司。如今，Kubernetes 在新兴的云原生环境中无处不在。所有三个主要的云提供商——亚马逊网络服务、[谷歌计算引擎](https://cloud.google.com/kubernetes-engine)和[微软 Azure](https://azure.microsoft.com/en-us/?v=17.14)——都支持 Kubernetes。甚至 [Docker](https://thenewstack.io/docker-fully-embraces-kubernetes/) 和 [Cloud Foundry](https://thenewstack.io/kubo-becomes-cloud-foundrys-container-runtime-default-kubernetes-deployment/) 都在用。

是什么让 Kubernetes 独一无二？在去年 12 月在 Kubecon + CloudNativeCon 录制的最新一期 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，我们采访了首席技术官兼 [CoreOS](https://coreos.com/) 联合创始人布兰登·飞利浦[(该采访录制于](https://twitter.com/BrandonPhilips)[红帽今年早些时候收购 CoreOS](https://thenewstack.io/docker-acquiring-coreos-red-hat-aims-kubernetes-company/) 之前)。

传统的开源技术创造了占主导地位的闭源技术的商品化替代品。Kubernetes 是个例外。“Linux 是 UNIX 的竞争对手，Apache web server 是微软 IIS 的竞争对手。Kubernetes 在和什么竞争？我们没有与之竞争的闭源产品，”飞利浦说。

从本质上来说，Kubernetes 是在与人们创建的所有脚本竞争，这些脚本是为了将他们在笔记本电脑上开发的源代码部署到生产中。有成千上万种方法可以做到这一点——CI/CD 工作流、bash 脚本、配置管理工具等等。获得生产所需的所有材料总是一个痛苦的数周开发周期。这就是 Kubernetes 来救援的地方。它通过围绕这个问题创建 API 来缩短这个周期。它提供了一致性和可重复性，可适用于任何模型。

“这是 Kubernetes 的重点，”Philips 说，“随着时间的推移，我们可能会接管应用程序的整个生命周期——从创意到监控到应用程序工作流，再到该应用程序的最终停用。我们已经看到用户在扩展 Kubernetes APIs。”据飞利浦公司称，这种情况将会持续下去。“天空是我们构建抽象概念的极限，”飞利浦说。“观察它在未来几年如何演变将是一件有趣的事情。”
