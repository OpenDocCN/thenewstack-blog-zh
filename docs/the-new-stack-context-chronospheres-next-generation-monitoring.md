# 新的堆栈环境:超时空领域的下一代监控

> 原文：<https://thenewstack.io/the-new-stack-context-chronospheres-next-generation-monitoring/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。本周我们采访了 [Rob Skillington](https://www.linkedin.com/in/robskillington/) ，他是 [Chronosphere](https://chronosphere.io/) 的联合创始人兼首席技术官，这是一家去年底脱离秘密行动的监控公司，围绕开源度量平台而建立，Skillington 和 chrono sphere CEO[Martin Mao](https://twitter.com/martin_c_mao?lang=en)在优步帮助开发了该平台。

TNS 编辑和营销总监 Libby Clark 主持了这一集，旁边是 TNS 创始人兼出版商 Alex Williams 和 TNS 执行主编 Joab Jackson。

[第 107 集:罗布·斯基林顿——超时空的下一代监控](https://thenewstack.simplecast.com/episodes/episode-107-rob-skillington-chronospheres-next-generation-monitoring)

复杂性最终导致这家乘车共享公司建立了自己的指标平台，名为 M3，并于 2018 年开源。该项目的两位技术负责人斯基林顿将这项技术推进到了超时空领域。

在本周发布在新堆栈上的一篇文章中，现为 Chronosphere CEO 的毛解释了为什么复杂性最终促使创建了自己的监控平台:

“优步在不断发展，我们不得不扩展基础架构，但与此同时，监控的复杂性也成倍增加。这些架构变化会带来什么，因为现在有这么多更小的组件。监控端会产生更多的数据。这些数据中有很多是高基数或高维数据。我们发现，我们使用的开源工具没有一个能够真正扩展到所有新的数据量和现在产生的高维数据。”

我们向 Skillington 询问了他们在大规模测量中遇到的问题，以及该公司的新方法和如何将其用于其他业务。

然后，在节目的后面，我们讨论其他 TNS 播客和感兴趣的博客帖子。我们的姐妹播客 [The New Stack Makers](https://thenewstack.io/podcasts/makers) ，专题报道[与 NS1 的 Kris Beevers](/ns1-builds-on-dns-to-speed-traffic-management/) 就增强的 DNS 服务在优化流量吞吐量方面的作用进行了精彩的交谈。我们还讨论了为什么[的可观察性需要保持怪异](/why-observability-needs-to-stay-weird)， [Container Solutions 的新工具 Flux，](/gitops-made-simple-with-flux/)将房子的“操作”端连接到 GitOps，以及云原生计算基金会的[艰难决定](/cncf-reschedules-kubeconcloudnativecon-europe-2020-amid-coronavirus-concerns/)由于新冠肺炎而推迟 KubeCon+CloudNativeCon Europe。

云原生计算基金会、KubeCon+CloudNativeCon 和 LightStep 是新堆栈的赞助商。
