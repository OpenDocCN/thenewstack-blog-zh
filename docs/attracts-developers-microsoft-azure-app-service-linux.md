# 为什么微软 Azure 中到处都是容器

> 原文：<https://thenewstack.io/attracts-developers-microsoft-azure-app-service-linux/>

[为什么微软 Azure 里到处都是容器](https://thenewstack.simplecast.com/episodes/why-containers-are-everywhere-in-microsoft-azure)

微软 Azure[的计算主管 Corey Sanders](https://azure.microsoft.com/en-us/?v=17.14)称，Azure 的技术基础设施中到处都是容器。开发者希望使用容器来构建新的应用，而不是虚拟机。微软自己也在使用容器作为在 Azure 中管理自己服务的重要基础设施，并使用开源技术来做这件事。

在最新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，桑德斯从他在新泽西的办公室加入了我们，谈论集装箱和 Azure，他承认他可以讨论几个小时。

他解释说，容器是应用程序部署的一个根本性转变，反映了速度、可移植性和灵活性的一个新维度。容器支持快速部署、跨所有托管环境的可移植性以及灵活地上下旋转组件。桑德斯说，大多数开发者不想使用虚拟机。对于开发人员来说，使用虚拟机意味着要做诸如移动磁盘之类的任务，而他们不想这么做。相反，开发人员想要容器附带的抽象。

桑德斯对集装箱前景有着广阔的视角。在这次采访中，他分享了他对容器技术及其在 Azure 中的使用的看法。他讲述了微软对开源立场的更大转变意味着 Azure 在支持 Linux 和所有运行于其上的编程语言方面有多灵活。

例如， [Azure Container Service](https://azure.microsoft.com/en-us/services/container-service/) 现在[支持三大容器编排引擎](https://thenewstack.io/new-updates-making-azure-container-service-applications-run-not-containers-run/)，包括 Kubernetes、Docker Swarm 和 Mesosphere 的 DC/OS。内部 [Azure 服务架构](https://azure.microsoft.com/en-us/services/service-fabric/)为[微服务](/category/microservices/)提供容器原生解决方案。现在，在预览模式下，Service Fabric 可以在内部或其他云中使用——展示了使容器对构建微服务应用程序的开发人员如此有利的可移植性。
