# 服务网格如何放大商业价值

> 原文：<https://thenewstack.io/how-a-service-mesh-amplifies-business-value/>

Aspen Mesh 赞助了这个播客，作为讨论服务网格如何帮助开发人员的一系列采访的一部分。听听之前关于[云迁移的门户](https://thenewstack.io/when-you-need-or-dont-need-service-mesh/)和[如何构建 Istio 以提高工程效率](https://thenewstack.io/how-istio-is-built-to-boost-engineering-efficiency/)的故事。

服务网格应该越来越多地提供的一个关键功能是帮助开发运维团队更好地观察哪些事件导致了应用部署和管理问题。他们还应该帮助确定哪个团队可以采取适当的行动。

在[的最后一集，新堆栈制作者](/podcasts/makers)三集播客系列，其中包括 [Aspen Mesh](https://aspenmesh.io/) 、[新堆栈的创始人兼发行人 Alex Williams](/author/alex/) 和 TNS 记者 [B. Cameron Gain](/author/bruce-gain/) ，与受邀者讨论服务网格如何帮助开发人员避免管理复杂的云原生和遗留环境的痛苦，以及如何将它们转化为成本节约。在特邀嘉宾[Aspen Mesh 副总裁兼总经理 Shawn Wormke](https://www.linkedin.com/in/shawnwormke) 和[开源社区总监 Tracy Miranda](https://www.linkedin.com/in/tracymiranda) 、 [CloudBees](https://www.cloudbees.com/) 的参与下，他们还讨论了服务 Mesh 在帮助实现业务目标方面可以做什么和不可以做什么，以及未来的发展前景。

[一个服务网如何放大商业价值](https://thenewstack.simplecast.com/episodes/how-a-service-mesh-amplifies-business-value)

沃姆克说:“我们从客户那里看到的是，他们希望将[维护工作]转移到应用程序之下，让应用程序所有者真正专注于商业价值代码。“他们还希望让运营团队(DevOps 的‘运营’部分)真正致力于为他们提供在大型企业环境中运行这些东西所需的工具和通用基础架构。”

已经证明了服务网格的使用是采用微服务和容器化环境的必要要求。例如，在将运营从传统内部环境转移到云原生基础架构时，服务网格无疑是一个重要工具。然而，服务网格并不一定能帮助所有组织向容器化环境转变。

米兰达说:“我认为，根据服务的数量，存在一个[服务网格]有意义的阈值。“如果你从一个非常简单的架构开始，并且你不想协调太多事情，也许服务网格增加了你不需要的复杂性。但用不了多久，你就可以拥有一个重要的系统，可以利用不同的功能。”

虽然 Kubernetes 的基础设施越来越受到支持和标准化，但开发人员的体验仍然会给云原生部署的上游开发带来挑战。以这种方式支持开发人员体验，代表了网格服务可以提供的另一个有用的功能。

Wormke 说:“大多数开发人员都深陷于代码中，并专注于尽可能生产最高质量的软件，但这并不总是转化为良好的最终用户体验，也不总是转化为良好的可管理产品。”。“因此，我们所做的很大一部分工作就是努力在这种环境中代表我们的企业和服务提供商客户，并确保他们努力做出至少是明智的选择。我们还帮助客户避免将这些大型部署放在他们无法恢复的地方。”

沃姆克说，因此“像阿斯彭网格这样的公司”有机会。“我们所做的很大一部分是专注于如何使用[软件工具]和处理 Istio 部件本身的生命周期管理，以及这项技术如何在大型组织中工作，从而使企业取得成功。”

通过这种方式，服务网格为用户组织提供了许多强大的功能，一旦他们通过了学习曲线，就可以利用这些功能。

“我认为，一旦人们开始欣赏你从一开始看似复杂的成本中获得的好处，并且随着工具变得更容易使用，我认为你想在你的系统中拥有它将变得不言而喻，”他说。

服务网格也是一种相对较新的技术——许多新的和潜在的采用者可能没有意识到这一点——尽管他们已经证明了他们通常是值得投资的。

米兰达说:“我们只需要一种简单的方法将[服务网格]送到人们手中，并帮助他们避开陷阱，这样他们就可以获得所有真正的魔力，一旦你得到了这种编排和所有这些东西，你就可以开始做这些事情了。你可以开始做一些非常聪明的事情。"

对于这篇播客，Aspen Mesh 拒绝对谷歌将 Istio 商标转让给开放用户共享的计划发表评论。
