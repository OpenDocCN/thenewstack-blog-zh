# 监控问题阻碍了混合、多云部署

> 原文：<https://thenewstack.io/monitoring-concerns-hamper-hybrid-multicloud-deployments/>

在过去十年中，安全性和缺乏控制是不使用公共云提供商的主要原因。尽管存在许多合理的担忧，但最近两项由供应商发起的调查显示，云提供商的能力通常不是增加采用的关键挑战。相反，人们更担心的是，现有的监控系统没有经过优化，无法同时跟踪内部和云环境。

根据由 [Cavirin](https://cavirin.com/) 发起的[网络安全内部人士](https://www.cybersecurity-insiders.com/)社区的一项以 AWS 为中心的[调查](https://www.cavirin.com/resources/analyst-report-2019-aws-cloud-security-report)，云平台的错误配置被认为是公共云中最大的安全威胁。然而，人们担心的不是 AWS 本身，因为 83%的人认为该公司提供了足够的原生云安全控制和服务。相反，担心的往往是他们自己组织的局限性和由其他方控制的不安全的接口/API。另一个大问题是 AWS 工具正在被使用，只是没有有效地使用。因此，虽然 71%的 AWS 客户表示他们使用 AWS 身份&访问管理，但在使用 AWS 进行身份和访问控制的客户中，只有 42%的人认为他们的组织有效地使用了这些工具。

另一个问题是，人们已经习惯了遗留工具所提供的强大功能。例如，Cavirin 研究发现，只有 15%的传统网络安全工具或设备在云环境中拥有所有功能。虽然缺乏某些功能可能是个问题，但很多时候，许多功能实际上是为每个公司的环境定制的。

网络监控工具提供商[是德科技](https://www.keysight.com/us/en/home.html)也[报告了](https://about.keysight.com/en/newsroom/pr/2019/25mar-nr19044-keysight-cloud-monitoring-survey.pdf)新旧系统整合的问题。它对 388 名 IT 专业人员的调查发现，62%的人担心云流量向其内部监控解决方案的安全交付。IT 专业人员喜欢他们已经拥有的设置，这恰好是以一个内部工具为中心。事实上，82%的受访者认为他们可以完整、及时地访问内部环境中的网络数据包，而只有 15%的人对公共云有类似的感觉。这种缺乏可见性的情况令人沮丧，因为 32%的受访者担心访问流经容器的网络数据包。然而，两倍多的受访者关注数据和应用流量的更全面可见性。

总体而言，87%的受访者担心公共云中的可见性会阻止他们防范安全威胁。55%的受访者认为他们的公共云提供商提供的数据不足以监控公共云中发生的事情，但这也意味着大约一半的受访者认为数据缺乏不是与他们的监控挑战相关的因素。

供应商消息通常将对云的普遍担忧与具体的具体威胁和挑战(AWS、Azure、Google Cloud 等)混为一谈。也许公司需要新的服务来管理多云环境。或者，遗留软件可以将现有系统与来自云提供商的数据集成在一起。无论发生什么，请记住，许多挑战更多的是关于集成多个 IT 系统，而不是对云计算的控诉。
