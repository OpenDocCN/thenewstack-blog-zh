# CloudFabrix 如何将企业治理引入云

> 原文：<https://thenewstack.io/cloudfabrix-bringing-enterprise-cloud/>

随着企业继续跨云进行纵向和横向扩展，其基础架构可能会变得难以管理。了解从何处提取基础架构层对运营团队非常有益，因为这在处理策略管理和创建时提供了更多的控制。

在本期 [The New Stack Makers](/tag/the-new-stack-makers/) 播客中，我们采访了 [CloudFabrix](https://www.cloudfabrix.com/) 首席产品官 [Bhaskar Krishnamsetty](https://www.linkedin.com/in/bhaskarkrishnamsetty) 和首席技术官 [Raju Penmetsa](https://www.linkedin.com/in/rpenmetsa) ，了解 CloudFabrix 如何帮助企业管理整个应用堆栈。TNS 主持人[凯尔·麦克唐纳](https://twitter.com/kylemacdonald)和[李·卡尔科特](https://www.linkedin.com/in/leecalcote)—[网络安全管理软件产品](http://www.solarwinds.com/)的云和容器技术负责人——风险投资公司 [RedPoint](http://www.redpoint.com/) 的合伙人[斯科特·雷尼](https://www.linkedin.com/in/sraney)也加入其中。

[CloudFabrix 如何将企业带入云端](https://thenewstack.simplecast.com/episodes/how-cloudfabrix-is-bringing-the-enterprise-into-the-cloud)

采访录制于 2016 年云铸造峰会。也可以在 [YouTube](https://www.youtube.com/watch?v=UirTqkQGnys) 上听这一集。

[CloudFabrix](https://www.cloudfabrix.com) 始于 2009 年，最初是一个基础设施管理平台，最终发展为提供各种端到端治理、管理和分析基础设施的服务。他们的目标是不仅为开发人员，也为运营团队提供治理控制。“有了 Docker 和 Docker 技术，可移植性变得更加容易，但来自企业的最大障碍仍然是治理、安全和控制，”Krishnamsetty 说。

“这就是我们需要将这些东西结合在一起的地方，通过在应用层而不是底层进行抽象。抽象程度越高，你就有更好的控制。抽象程度越低，你的控制就越少，”Krishnamsetty 解释道。“更高抽象层的一个挑战是，如果做得不好，平台就变得更专有。这是 PaaS 生态系统目前面临的最大挑战，”雷尼补充道。

最终，今天的许多业务关键型企业基础架构还没有达到需要的水平。Penmetsa 恳求说，这一事实应该成为当今面向企业的技术解决方案的主要焦点。“如果你真的看一看企业，70-80%是业务关键型、传统型的。NET，基于 Java，Oracle SAP，对吧？他们在设计时并没有考虑到云。我们需要找出适合他们转型的平台或技术。”

CloudFabrix 和 T2 云铸造公司是新堆栈的赞助商。
