# 从整体服务到采用 Iron.io 的微服务

> 原文：<https://thenewstack.io/from-monolith-to-microservices-2/>

在本期 [The New Stack Makers](/tag/the-new-stack-makers/) 播客中，我们将探索无服务器软件提供商 [Iron.io](https://www.iron.io/) 如何帮助组织将他们的单片应用程序重建为微服务，以及该公司如何帮助流行的社交饮酒应用程序[un tapd](https://untappd.com/)进入无服务器生态系统。New Stack 创始人 Alex Williams 在 2016 年 DockerCon 展会上与 Iron.io 联合创始人&首席执行官 [Chad Arimura](https://www.linkedin.com/in/chadarimura) 和 Iron.io 销售工程师 [Doug Colburn](https://www.linkedin.com/in/douglascoburn) 就这些问题进行了简短的讨论。

[从铁板一块到微服务. io](https://thenewstack.simplecast.com/episodes/from-monolith-to-microservices-with-iron-io)

这段对话[也可以在 YouTube](https://www.youtube.com/watch?v=ZEkCnfSQHgQ) 上听到。

Iron.io 最近宣布了与 Mesosphere 的战略合作伙伴关系，希望开创一个无服务器计算的新时代，通过提供一个执行代码块的平台，让最终用户不必维护服务器。Arimura 指出，当该公司在 2011 年成立时，是 LXC 集装箱公司负责作业和批处理。

Arimura 指出，尽管这种情况很快发生了变化，“当 Docker 出现时，它是最合适的，因为它标准化了包含和发布代码的方式”。Iron.io 的开发人员将工作流容器化，将单个虚拟机分开，这样他们就可以让多个用户大规模运行他们的应用程序，而不会相互影响。

将开发人员从如何构建和维护基础设施中抽象出来是最近的热门话题。在 Iron.io，正是这种远离基础设施内部工作的抽象，科尔本解释说这是无服务器计算的好处之一。

“通过从开发人员那里抽象出基础架构，他们可以更轻松地工作。无论是创建任务还是有效载荷。从微服务的角度来看，它允许他们创建一个容器来完成一项任务，并且做得很好。它正在从整体服务转向微服务，这种抽象现在已经走得更远了。开发人员可以专注于，‘这是我的一点代码，它在自己的发布周期中完成这些，现在我不必担心基础设施了。"

有村后来提供了最后一点:“新的堆栈正在形成。这就是这个蓬勃发展的生态系统的伟大之处，一组开发人员可以选择一套技术，让他们进入这个没有服务器的世界”

Iron.io 和 [Mesosphere](https://d2iq.com/) 是新堆栈的赞助商。
