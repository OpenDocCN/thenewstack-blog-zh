为了创造新的收入流、管理客户关系和提供卓越的用户体验，组织正感受到越来越大的压力来加快其数字化转型计划。因此，他们越来越多地投资于云原生应用、容器和基于微服务的架构。

集装箱尤其受欢迎。2020 年云本地计算基金会的一项调查发现 [84%的组织目前在生产中使用容器](https://www.cncf.io/blog/2020/03/04/2019-cncf-survey-results-are-here-deployments-are-growing-in-size-and-speed-as-cloud-native-adoption-becomes-mainstream/)，Kubernetes——被 78%部署容器的企业使用——成为管理容器的事实上的解决方案。Kubernetes 和 containers 的动态性质，以及它们可以上下旋转的速度，也意味着随着企业加快数字化转型，它们也在加速进入一个极其复杂的 IT 环境。由于传统监控的局限性，这种复杂程度导致对整个技术体系的观察和洞察很少。这影响到每一个人，从 ITOps 到应用开发者，再到 Kubernetes 平台运营商本身。

Kubernetes 的数字化转型必须与人工智能驱动的基础设施监控方法齐头并进。没有人工智能，你就无法捕捉到你的 Kubernetes 环境和支撑它的基础设施中发生的事情的全貌。

## **人工可观测性不能满足 Kubernetes 复杂性的需要**

基于 Kubernetes 的云平台，如谷歌 Kubernetes 引擎(GKE)、亚马逊弹性 Kubernetes 服务(EKS)和微软 Azure Kubernetes 服务(AKS)，反映了动态容器和微服务集成到组织的多云环境中的速度有多快。问题是，那些相同的容器和微服务可以在几秒钟内来来去去。这对于构建更加敏捷和灵活的 IT 环境来说非常好，但是对于可观察性来说却是一个噩梦。

试图维护容器、微服务和 Kubernetes 的手动可观察性和配置是费时费力的。IT 团队承受不起这种代价，尤其是现在，他们面临着做更多事情、更快交付成果的任务，而资源更少、出错空间更小。手动观察不仅消耗了 IT 生产力，还遗漏了 Kubernetes 环境的基础设施层:容器、pod、节点、集群以及这些组件提供的丰富的数字业务分析。缺乏对这些基础架构组件和相互依赖关系的可观察性，使得更难全面了解您的环境，从而更难优化成本和消除性能下降。

所有这些都导致了更复杂的环境，几乎没有可观察性和系统异常，这些异常可能会不受控制地运行，因为没有办法找到并修复它们的根本原因。在人们更快地根据数字服务的速度和质量来评判组织的时候，这一切都可能对企业产生严重的连锁效应。不仅能够在应用、微服务和 Kubernetes 上，而且能够在它们运行的基础设施上部署完整的可观察性，这对于全面了解您的环境并相应地优化性能和可用性至关重要。

## **利用人工智能辅助提高 Kubernetes 基础设施的可观测性**

动态环境需要动态解决方案。ITOps 团队、应用程序开发人员和 Kubernetes 平台运营商都需要能够自我发现和自动检测其云环境中的变化，并实时捕获可观察性数据，以便跟上生产中数千个容器和微服务的速度。

正如持续自动化和人工智能辅助已经在很大程度上改变了 IT 环境的运行方式一样，它们对于提供快速、自动的代码级洞察也是必不可少的，这种洞察是在 Kubernetes 中运行的云原生应用程序、容器和微服务之间获得高级可观察性所必需的。AI-assistance 将 IT 的姿态从被动转变为主动，并通过消除单调乏味的手动工作的无用动作，将他们释放出来进行更具创新性的业务增值任务，从而成倍提高团队的生产力。

这种人工智能驱动的基础设施监控方法可以更快、更容易地实现深入 Kubernetes 基础设施层的高级可观察性，因此团队不仅可以看到每个容器、pod、节点、集群和微服务，还可以看到它们对真实用户(他们的客户)和业务的影响。

对于团队来说，手动获取这种级别的洞察力是极其困难和耗时的。人工智能辅助可以快速、自动、无痛苦地获得高级可观察性，并从根本上解决问题，优化成本、容量和工作负载，即使一切正常。

## **Kubernetes 基础设施的完全可观察性=更成功的数字化转型**

IT 团队正努力提供一流的用户体验，同时以前所未有的速度提升数字服务能力。组织无法承受需要手动配置、脚本、特殊附加组件的可观察性方法——任何会使在 Kubernetes 环境中实现可观察性变得更慢、更难、更昂贵的方法。他们也不能因为缺乏可观察性而让潜在的问题未被发现或未被解决，并使用户体验处于风险之中。

这是一个可以解决的问题。企业需要投资于人工智能驱动的高级可观测性，挖掘优化容器、微服务和 Kubernetes 环境所需的基础设施数据；识别并自动修复异常情况；并对其技术堆栈中运行的数十亿个相互依赖项提供始终准确的洞察。

当你没有这些洞察力时，你就不能为你的用户、你的业务或你自己的团队交付成果。
