# TNS 背景:Grafana Loki 和 KubeCon 外卖

> 原文：<https://thenewstack.io/tns-context-grafana-loki-and-kubecon-takeaways/>

[格拉法纳洛基和库比康外卖](https://thenewstack.simplecast.com/episodes/grafana-loki-and-kubecon-takeaways)

本周在 Context 上，我们与 Grafana Labs 的产品副总裁 Tom Wilkie 进行了交谈，Grafana Labs 刚刚为 Kubernetes 发布了一款名为 [Loki](https://grafana.com/loki) 的新开源日志聚合工具。与其他日志聚合工具不同，Loki 不索引日志的全文，而是只索引元数据，并使用与 Prometheus 相同的服务发现技术。

“许多系统没有元数据，它们在查找日志内容方面确实很强大，但找到这些日志来自哪里、何时生成，可能是什么主机和软件的什么版本，这才是我们真正关注的地方。就像你能对付普罗米修斯一样，”威尔基说。

洛基已经吸引了社区的注意力，在宣布的第二天就出现在黑客新闻的第一名，并在 GitHub 上获得了超过 2000 颗星[。在本播客中，Wilkie 与西雅图 KubeCon + CloudNativeCon 的创始人兼主编 Alex Williams 和 New Stack 的编辑总监 Libby Clark 进行了对话。](https://github.com/grafana/loki)

“我们认为这是非常强大的，因为它将更容易，更具成本效益地运行，比弹性集群更容易操作，”威尔基说。

在稍后的节目中，我们将展示我们从 KubeCon 和 CloudNativeCon 获得的信息。新的 Stack 直播了我们的播客，举办了许多煎饼早餐会，并发布了许多新闻:
