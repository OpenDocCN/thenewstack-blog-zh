# 反思对云安全的信任

> 原文：<https://thenewstack.io/rethinking-trust-in-cloud-security/>

从云安全提供商到开源软件，信任已经成为一个组织安全的基础。但是随着云原生技术的兴起，构建应用程序的新方法正在挑战传统的安全方法。不断变化的云原生环境要求跨技术堆栈的更广泛的安全覆盖，以及对环境的更多上下文感知。那么，商业企业和政府中的 DevOps 和 InfoSec 团队应该如何反思他们的安全方法呢？

在这一集的 New Stack Makers 播客中，[三一网络公司总裁汤姆·博塞特](https://www.linkedin.com/in/tombossert/)(也是两位总统的前国土安全顾问)； [Patrick Hylant](https://www.linkedin.com/in/patrick-hylant-406a49126/) ，VMware 客户经理；和[王晨曦](https://www.linkedin.com/in/chenxiwang88)，执行普通合伙人，雨资本讨论如何企业和美国政府可以适应不断变化的威胁格局，包括新的倡议和教训，可以适用于这种高风险的环境。

《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了这次播客。[Armory 首席执行官 Jim Douglas](https://www.linkedin.com/in/jidouglas/) 也参加了此次直播活动。

[反思对云安全的信任](https://thenewstack.simplecast.com/episodes/rethinking-trust-in-cloud-security)

随着 IT 环境的现代化，围绕网络安全和新技术的挑战浮出水面，但这次发生的变化是，“了解软件和硬件的来源，以及在这一过程中谁接触了它们。这种情况已经很久没有出现了，尤其是在开源领域，”道格拉斯说。

“开源还不错。其实开源就好。有时它会得到一个坏名声，”博塞特说。但当涉及到网络安全时，问题是“大多数运营商需要更好的工具来获得更多的上下文可见性，”Bossert 补充道。他说，通过开源获得它们可能比传统投资更快。

“如果你不利用已经开发的开源库，你将无法竞争，”王说。“开源正在激增。我们需要知道他们在哪里；我们需要知道哪个软件使用哪个库。而是到达那里的一个方法，”王补充道。

但是随着开源成为主流，合作伙伴和对手之间的信任线将会改变。“我们将开始看到地缘政治的分裂蔓延到互联网的分裂。不同信任圈之间的规则会给这种模式带来问题，”海兰特说。

尽管拜登政府的网络安全行政命令改善软件供应链，“我们在行业中没有看到我认为我们应该在 SBOMs 上做的那么多要求。只有大约 15%的软件提供商听说过它，”王说。“但即使我所有的产品都和 SBOM 在一起，我也不知道它们在哪里。我不知道如何管理或阅读它们，”王补充说。

Bossert 说，最终，尽管跨云原生环境的安全战略随着地缘政治风险而变化，“我们需要以创新的名义激励一些冒险行为”。
