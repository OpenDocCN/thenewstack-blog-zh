# Equinix:在数据中心内部构建云连接点

> 原文：<https://thenewstack.io/equinix-building-cloud-connection-points-inside-data-center/>

到目前为止，我们还没有过多地讨论过 [Equinix](http://www.equinix.com/) 。但是，世界上市场份额最大的协同定位提供商与大规模部署应用程序有什么关系呢？

Equinix 对我们今天的空间很重要，因为它为企业提供了服务器和云服务提供商之间的直接连接点。它的云交换计划为企业数据中心提供了到他们使用的平台的直接路径——有点像内容交付网络，但用于访问服务而不是相反的方向，即服务内容。其[性能中心和数据中心服务](http://www.datacenterknowledge.com/archives/2016/04/05/new-equinix-data-hub-broadens-enterprise-colo-options/)支持私人访问公共连接和公共存储，使亚马逊网络服务的 S3 存储服务、容器引擎和分析可用于各种行业，否则这些行业将因合规问题而被禁止使用。

因此，如果你正在考虑基础设施即代码，那么 Equinix 会让你突然觉得基础设施变得更加丰富。

“Equinix 是一家传统的点对点网络公司，”Equinix 全球解决方案营销总监[劳伦·库尼](https://www.linkedin.com/in/laurencooney/)在接受 TNS 创始人[亚历克斯·威廉姆斯](https://twitter.com/alexwilliams)采访时解释道，“数据中心将是所有流量的来源。他们会交换容量，以及类似的东西。

“现在，Equinix 真正关注的是数字边缘和面向互联的架构，”库尼继续说道。她将这种面向互联的架构(IOA)描述为类似于面向服务的架构(SOA)。“基本上，它允许用户创建一个到公共云的直接私有连接，连接到我们数据中心内部的多个公共云，跨越 21 个城市的 44 个不同区域。”

在“Equinix 如何构建数字边缘”中了解更多有关 Equinix 如何组装其更软、更抽象的数据中心连接策略的信息，这是最新一集的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客，在圣克拉拉举行的上一届 Cloud Foundry 峰会上录制。
