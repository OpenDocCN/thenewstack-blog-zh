[# 18:SaaS 与 PaaS 整合的涟漪效应](https://thenewstack.simplecast.com/episodes/18-the-ripple-effects-of-saas-and-paas-integration)

SaaS 和 PaaS 的集成产生了连锁反应，影响了商业、消费者和开发者。但是，“对这个问题的思考很久以前就开始了，”Tal 说。“有两个趋同的趋势:一个是，正如我们所知，基础设施即服务供应商正试图转向平台，显然软件即服务供应商也在转向平台，因此在这中间有一个会议，特性和功能，更重要的是，SaaS 提供商提供的用户和实际数据可以成为使用这些 SaaS 应用程序作为平台的定制 SaaS 的基础。”

对话正在发生变化，因为环境正在扩大，包括 Salesforce 等平台，用户在这些平台上积累了大量自己的数据。Alex 发现人们对开发技术的兴趣已经增长到了如此程度，以至于许多人都在尝试使用这些连接工具来构建自己的应用程序。这类新的公民开发者仍然需要简单的可视化工具，而不是编写代码。

“我实际看到的是，IT 或 DevOps(或您现在想称之为该组织的任何名称)的整合，我们称之为消费化或部门化的东西实际上正在回归，因为业务部门需要额外的功能，IT 可以通过 DevOps 提供这些功能。”

Donnie 指出，随着这些技术的成熟，它们开始变得越来越难以管理，最终业务部门会希望将管理工作外包给 IT 部门。

Tal 说，人们仍然期望从头开始构建，但现在通过采用 SaaS 应用程序，正在创造一种“新遗产”，这些应用程序本身正在成为平台。

唐尼认为，挑战在于分裂。“将会有围绕多个不同的 SaaS 提供商构建的多个平台，这些提供商将提供他们自己的 API 集，以及他们自己的服务。”

“最大的问题之一将是以一种不会损失太多的方式翻译不同平台上的所有隐喻。”

Tal 和 Donnie 以及许多其他人看到了更好的前进方向，但也看到了开发的本质必将在未来的日子里导致大量的定制、不同的工作流和专有平台。
