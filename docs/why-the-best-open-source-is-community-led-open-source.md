# 为什么最好的开源是社区主导的开源

> 原文：<https://thenewstack.io/why-the-best-open-source-is-community-led-open-source/>

亚马逊网络服务(AWS) 赞助了这篇文章。

 [马特·阿萨伊

Matt 参与了开源及其支持的所有领域(云、机器学习、数据基础设施、移动等。)近二十年来，为各种开源公司工作，并定期为 InfoWorld 和 TechRepublic 撰写文章。你可以在推特上关注他(@mjasay)。](https://www.linkedin.com/in/mjasay/) 

几十年来，开源软件的重要性一直在增长，但当大型网络公司开始认真贡献代码时，它达到了逃逸速度。突然之间，世界上充斥着金钱不需要购买的最具创新性的基础设施。像雅虎这样的公司！(Apache Hadoop)、脸书(Apache Cassandra)、LinkedIn (Apache Kafka)和谷歌(Kubernetes)发布了非常有前途的软件，邀请全球社区帮助他们改进他们的创作。

业界回应。虽然有时会出现一个占主导地位的供应商来维护一个特定的项目(例如 Apache Cassandra 的 DataStax)，但是对这些和其他类似项目的贡献往往来自各种公司和个人开发人员。以[库伯内特](https://kubernetes.io/)为例。虽然谷歌创立了这个项目，[来自 2000 多家公司的 35000 多人为这个项目贡献了](https://k8s.devstats.cncf.io/d/9/companies-table?orgId=1)，这个项目现在运行了近 200 万行代码。或许最有希望的是，这些贡献者并不都是技术供应商(例如，全球市值最高的金融科技公司蚂蚁金服和中国大型团购网站美团点评都位列贡献者前 20 名)——这意味着客户越来越多地塑造开源软件来满足他们的需求。

如果你担心开源的可持续性，这些由社区主导的项目给你带来了希望。当然，虽然有像 MySQL 这样完全或主要依赖于单一供应商的优秀开源项目(MySQL 的例子是 Oracle ),但当由社区维护时，开源是最具创新性和可持续性的。社区主导的开源开发促进了富足，为所有人创造了更大的“馅饼”，包括单个供应商更大的收入潜力，而不是单个供应商希望为自己囤积相对较小的“馅饼”的好处。

想想 Linux，这个开源操作系统驱动着世界上最快的超级计算机，我们访问的大多数网站，甚至我们的手机。

## 向 Linux 学习

Linux 一直是一个社区事件，但它并不一定注定如此。Linux 创始人 Linus Torvalds 最初在 Transmeta 工作，这是一家早期的无晶圆厂半导体公司，从 1997 年到 2003 年，他加入了开源开发实验室(OSDL)，后来成为 Linux 基金会。托瓦尔兹在 Linux 基金会的工作确保了中立性，这将继续有利于所有的 Linux 供应商和用户，包括 Red Hat，一个最初通过销售 Linux 支持而出名的开源供应商。

红帽，虽然对许多人来说是 Linux 的同义词，但从未主导过它的发展。根据 5.5 内核的[最新数据，Red Hat 只贡献了 6.6%的 Linux 代码(以变更集衡量)。在 Red Hat 被 IBM](https://lwn.net/Articles/810639/) 收购[之前的最后一个完整财年，这 6.6%的贡献转化为 34 亿美元的收入。说到 IBM，如果我们把它的代码贡献加进去，Red Hat/IBM 的代码贡献加起来占所有 Linux 内核贡献的 9.8%。这反过来又让 IBM 从基于 Linux 的硬件(更不用说咨询服务)中获得了数十亿美元的相关收入。重要的是，如果 Red Hat 神奇地拥有所有的 Linux 开发，Linux 的价值将急剧下降。Linux 之所以如此重要，是因为它是一个丰富的社区资源。](https://www.cnbc.com/2019/07/09/ibm-closes-its-34-billion-acquisition-of-red-hat.html)

并不是只有红帽公司(IBM)从这种富足中获益。甲骨文、HPE 和一系列其他企业供应商通过销售围绕 Linux 的硬件、服务或软件获得了自己的数十亿美元，AWS、微软、阿里巴巴和谷歌等云供应商也是如此。许多其他公司(如脸书、网飞等。)构建并受益于 Linux，为客户创造数十亿美元的价值。至关重要的是，如果一家公司拥有 Linux，那几十亿美元可能会蒸发掉；该公司将获得所有的价值，而这一价值将大大减少。[根据一些估计](https://www.zdnet.com/article/can-the-internet-exist-without-linux/)，我们所知道的互联网如果没有一个免费的、社区驱动的 Linux 根本就不可能存在。

一家公司拥有一个操作系统并从中独享利润的时代已经结束。这部分是因为我们发现，例如，对于所有人来说，更好和更有利可图的是 Linux 社区创造一个由许多人分享的巨大且不断增长的馅饼，而不是由一个人消费的相对较小的零和馅饼。

## 开源的“阴云”未来:Kubernetes、鞭炮等等

这种关于馅饼的谈论听起来像乌托邦吗，就像那种只有当开源软件从像托沃兹这样有才华的个人的头脑中出现时才能工作的东西？库伯内特斯地区的人们可不这么想。它也不符合一批新的面向云的开源项目，如亚马逊贡献的[鞭炮](https://aws.amazon.com/blogs/aws/firecracker-lightweight-virtualization-for-serverless-computing/)和 Lyft 创建的[特使](https://www.envoyproxy.io/)，它们是该行业一些最具创新性服务的基础。

Kubernetes 不一定是这场面向云的开源革命的始作俑者，但它无疑有助于这场革命的发展。Kubernetes 于 2014 年推出，为行业提供了分布式系统的共同聚集点或标准。这只是一个面向社区的项目，其中 Red Hat、VMware、AWS 和其他公司也参与进来，让 Kubernetes 变得更大更好。正如 Redmonk 分析师[史蒂夫·奥格雷迪](https://twitter.com/sogrady/status/1235265398477598725)所说，“在一个 Kubernetes 不开源的世界里，它是一个利基产品……”。

无服务器公司(Serverless Inc .)的开发人员关系主管 Jared Short 写道，以类似的方式，鞭炮为开发人员提供了一种新的方式来“构建容器编排平台或运行大量容器，并需要以亚秒级的延迟来完成这些工作”。bracket 位于 AWS Lambda 和 AWS Fargate 的核心，并为其他人提供构建类似服务的能力。和 Kubernetes 一样，随着 Weaveworks 的开发者和许多其他人扩展它的用途，鞭炮越来越成为一种社区努力。

这也不仅仅与技术供应商有关。

Lyft 开发了 Envoy，这是一个为云原生应用程序设计的开源边缘和服务代理，并将其治理转移到了云原生计算基金会下。Envoy 已经找到了进入大量商业产品的途径，包括 HashiCorp 的 Consul 和 T2 的 AWS 应用 Mesh。Apache Hadoop 在 Yahoo！，但一旦开源，就点燃了大数据革命。Apache Cassandra 在脸书是一个很好的项目，但它已经成为苹果、网飞和其他公司令人难以置信的强大数据基础设施的支柱，即使这些公司做出了贡献，使它变得更加强大。

在这些和其他情况下，最好的开源软件往往是面向社区的，鼓励开发人员贡献他们的专业知识，使一个项目比任何一个组织都要好得多。

## 开源是丰富，而不是匮乏

当然，并不是所有人都同意。一些创业公司已经转向专有许可，以保护开源代码不被用作…开源代码。这是可以理解的，但在我看来，这是目光短浅的。在一个因广泛参与而变得更大、更重要的市场中，包括这些供应商在内的所有人都有更多的机会，无论是在软件开发方面还是在为客户运营软件方面。

纯谷歌的 Kubernetes 很有趣，但不会改变市场。由 Red Hat 控制的 Linux 也是如此——这样的操作系统永远不会超过微软的 Windows、UNIX 和其他所有服务器操作系统。

这是开源的承诺:丰富，而不是匮乏。作为一个行业，我们应该小心不要将这种承诺浪费在零和许可游戏上，这种游戏会让每个人——客户和供应商——为不断缩小的贫困馅饼而争吵。

CNCF、哈希公司、Linux 基金会、Red Hat 和 VMware 是这个新堆栈的赞助商。
