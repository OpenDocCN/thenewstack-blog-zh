# Metrika 为 Web3 带来网络监控和警报

> 原文：<https://thenewstack.io/metrika-brings-network-monitoring-and-alerting-to-web3/>

虽然 Web 2.0 的集中式云托管基础设施和 Web3 常见的分散式节点基础设施之间存在许多差异，但它们都有监控网络健康的共同需求。最近的 Solana 断电是 2022 年区块链的第五次断电，凸显了快速识别问题的需要。对于这种类型的 TCP/IP 网络监控，有许多成熟的解决方案，但像 Web3 的许多其他方面一样，监控工具仍在不断发展。

[Metrika](https://www.metrika.co/about/) 正在构建工具来填补区块链网络空间中的这一空白。该公司提供分析区块链运行状况的工具，并为节点提供一致的性能指标。这种支持最初是为 Algorand 和 Axelar 区块链推出的。

今天，Metrika 正在推出一些额外的功能来支持 Web3 社区，对影响网络和节点健康的事件进行监控和警报。

## 社区警报

对于许多 Web3 协议和项目来说，Discord 和 Twitter 是当今了解中断的常见地方。Metrika 正在添加一项名为“社区警报”的功能，通过在区块链健康受到影响时向社区成员提供实时警报，以更主动、更可扩展的方式解决这一问题。如果您曾经轮流值班，那么这些警报机制看起来会很熟悉。

在接受 New Stack 采访时，Metrika 创始人兼首席执行官 [Nikos Andrikogiannopoulos](https://twitter.com/andriko) 表示，“社区警报通过电子邮件、PagerDuty、Slack 和 OpsGenie 发送。更多集成即将发布。我们的目标是让 Web3 insights 可用于传统的 Web2 监控工具，以便用户可以在他们用于传统基础设施的任何平台上获得通知。”

有些警报事件是可配置的，有些是基于机器学习和异常检测的，不需要配置。Metrika 计划在未来允许创建自定义警报逻辑。

## 哪些类型的数据浮出水面？

Metrika 还启用了一组“社区网络仪表板”，该仪表板将执行类似于通过其他服务提供的正常运行时间仪表板的功能-除了使用 Web3 镜头。呈现的数据可能特定于网络中的单个节点，包括 CPU、内存和网络使用数据等，以及特定于协议的参与性能。仪表板也支持网络数据。Andrikogiannopoulos 说，“网络特定数据包括共识参与、性能和产生的块生产时间、网络延迟、API 延迟响应和异常网络活动。”

## Metrika 社区监控工具的定价

当我问 Andrikogiannopoulos 关于定价的问题时，他回答说:“我们提供我们平台的社区层作为公共产品，作为协议基金会支持的公共基础设施，免费提供给每个人。高级层面向协议团队、企业和金融客户。”

对于我们这些将开源产品作为开发过程一部分的人来说，这听起来像是一个熟悉的模型。

随着开发人员开始构建更多的生产工作负载并扩展到更多的用户，监控网络和节点健康状况是 Web3 领域中一个相当明显的需求。拥有 Metrika 的警报和网络监控等工具是在出现问题时获得快速响应所需的可见性的关键一步。像很多 Web3 项目一样，你可以在 [Discord](https://discord.gg/3tczKjK3ST) 上得到团队的支持。
