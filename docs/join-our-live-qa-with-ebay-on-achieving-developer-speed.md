# 加入我们与易贝关于实现开发者速度的实时问答

> 原文：<https://thenewstack.io/join-our-live-qa-with-ebay-on-achieving-developer-speed/>

作为一家互联网规模的电子商务公司，易贝是可观察性和渐进交付等现代开发实践的早期采用者之一。今天，工程团队继续通过实践进行创新，使他们的开发人员能够快速可靠地部署到生产环境中。

API 开发和管理 [取得了特别的成功](https://thenewstack.io/how-ebays-buy-apis-hit-5-billion-in-gross-merchandise-bought/) 并在 2020 年底达到了一个里程碑。易贝的 Buy APIs 允许第三方网站从互联网上的任何地方下单，累计 GMB 超过 50 亿美元(商品购买总额)。

他们如何支持开发速度并继续大规模创新？通过易贝的工程领导了解更多信息，并提出您的问题！太平洋时间 2021 年 4 月 1 日上午 11 点，New Stack 创始人兼发行人 Alex Williams 将与我们来自易贝的小组成员:“买家体验工程”副总裁 Lakshimi Duraivenkatesh 一起参加现场播客讨论**;马克·温伯格，核心产品工程副总裁；首席架构师兼工程副总裁 Randy Shoup。**

我们将讨论:

*   当今的易贝，其基础设施和应用架构。
*   提高易贝开发团队的速度。
*   扩展 API 交付和管理。
*   优化敏感服务的延迟。
