# 转型可观察性之旅

> 原文：<https://thenewstack.io/the-journey-to-transformational-observability/>

这是 10 月 24-28 日 KubeCon + CloudNativeCon 系列文章的一部分。

虽然数据以不断增长的速度增长，但 IT 预算却跟不上增长的步伐。随着数据管理成本的不断增加，企业从可观察性数据计划中获得的价值并没有相应增加。

尽管希望利用数据做出决策的用户数量有所增加，但数据的访问和可用性仍然是一个挑战。随着数据源的增加，对数据互操作性和相关性的需求也在增加，以便用户可以访问、统一和使用来自不同系统和功能组的数据。

如果没有适当的流程和技术，组织无法充分利用可观测性数据，并且可能会影响风险管理和他们可以向客户提供的服务质量。

企业正在寻找方法来增加他们从数据投资中获得的价值，同时管理螺旋上升的成本。他们正在寻找让更多用户能够更好地访问和控制数据的方法。

依靠控制是不够的，例如将数据路由到可观察性工具和低成本存储来帮助管理成本。除了管理和控制数据，企业还寻求更多的价值和实时可操作性。有了更好的控制加上通过丰富和关联增加的数据，企业可以更快地对问题做出响应，并发现改善客户体验和推出新产品的机会。

通往真正可观察性的道路是一个旅程，了解您所处的阶段有助于确定人员、流程和技术差距，从而从您的数据投资中获得更多商业价值。企业可以使用以下四个阶段来评估他们的实践和工具，并向客户提供更好的产品和服务。

## **阶段 1:临时日志监控**

在这个阶段，组织承认对可观察性的需求，但是他们的关注点仅限于记录和监控单个系统和应用程序。数据源可能包括一次性的服务器或应用程序，目的地通常是日志管理工具或像 AWS S3 这样价格合理的存储设备。日志由单个用户收集和汇总，用于特定用途。

不同的组织可能会寻找供个人使用的工具，或者采用开源技术来构建他们的日志管理解决方案。主要用例是日志聚合和手动分析来识别问题。

基本的解析和排除功能有助于识别问题和管理数据成本。可观察性工具很可能属于运营团队，他们必须致力于帮助其他团队访问他们需要的数据。

## **第二阶段:计划控制**

数据控制能力增强的组织拥有更好的流程和工具，能够实现更全面的数据访问和更好的控制。全公司都认识到了可观察性的需求，企业解决方案取代了一次性的日志管理工具。

数据摄取来自各种云基础设施、Kubernetes 或 Kafka 等流媒体平台。目的地包括各种消息队列、可观察性和安全平台。许多用户，包括开发人员、ITOps、DevSecOps 和 SREs，都使用该解决方案进行日志管理和分析。

诸如重新映射、聚合、过滤和数据采样等功能有助于用户更好地控制数据。复杂的访问控制和路由满足了组织中广泛用户的数据需求。

## **阶段 3:针对可操作性进行优化**

在这个阶段，企业可以更好地控制他们的数据，并从他们的可观察性投资中获取更高的价值。可观察性流程和基础设施经过优化，可获得切实可行的见解，从而提高 MTTD/R 和产品上市速度。

组织在丰富数据以获得更好的背景以及关联跨领域数据以揭示更深刻的见解方面非常成熟。他们利用 OpenTelemetry 获得额外的上下文。

此外，丰富和关联是在数据仍处于运动状态时发生的，而不是在数据到达目的地(如数据湖)时等待分析。因此，这些组织可以在模式成为安全问题或客户投诉之前主动识别它们。

## **第四阶段:转型**

处于转型可观察性阶段的组织已经超越了在数据孤岛中工作，并拥有先进的 DevOps、DevSecOps 和 DataOps 方法。他们应用可观察性原则和技术来转变他们的业务。他们将数据视为从多个来源捕获的企业资产，包括 CRM 和客户数据平台等业务应用程序，并使用洞察力来激活安全或面向客户的应用程序中的工作流。

实现转换可观性需要实时跨域关联和智能数据处理。DataOps 和 DevOps 联手充分利用企业数据。洞察变得可行，并用于通过提供卓越的客户体验和产品来获得竞争优势。

向数字化的转变正在加速，数据是一项关键的竞争优势。随着企业管理复杂的供应链并交付跨越数十种应用和系统的复杂客户体验，对数据访问、控制、可操作性和安全性的需求也在增加。

客户期望迫使公司做出更快的响应，而不断增加的安全威胁和法规要求组织更加警惕和主动。在这种市场动态中，组织必须实施必要的流程和技术，以帮助他们使用可观察性数据来提高效率、合规性和竞争力。

了解组织在其可观察性旅程中所处的位置将允许他们评估其差距，并制定与公司业务目标一致的路径。
