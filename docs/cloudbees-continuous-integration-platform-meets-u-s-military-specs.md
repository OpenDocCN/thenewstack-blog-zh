# CloudBees 持续集成平台符合美国军方规范

> 原文：<https://thenewstack.io/cloudbees-continuous-integration-platform-meets-u-s-military-specs/>

一个“加固”版本的 [CloudBees 的](https://www.cloudbees.com/) Jenkins 持续集成(CI)平台已经满足了空军的[现场证书(CtF)](https://apnews.com/dff493fa099c46e49709abae52a7a8b9) 的要求，允许该软件用于安全敏感的美国国防部(DoD)项目。具有关键安全需求的组织也可以考虑使用该版本的软件。

美国空军的[平台一号](https://software.af.mil/dsop/services/)项目管理认证项目。

CloudBees CI 的硬化 Docker 容器映像以前称为 CloudBees Core，存储在国防部集中式工件存储库(DCAR)中。国防部内部的开发团队或具有许可的外部民间承包商可以访问 DCAR 之外的加固码头集装箱图像。可以理解的是，与 Docker 容器映像相关联的库或组件被持续地扫描并监控安全漏洞。例如，如果一个团队使用一个库来执行 CloudBees CI 主机和代理之间的 http 通信，CloudBees CI 中的功能可以确保两端使用安全的端口和协议，CloudBees 说。

“至于安全的 Jenkins 是否会自动产生更安全的代码，你会希望它会根据应用的标准来审核代码，”Clive long bottom and Associates 的分析师说然而，我自己不会依赖它——我仍然希望确保在这样一个环境中的所有开发人员将军事安全理解为一个概念和一个实际的环境。"

隆巴顿说，国防部有非常具体的安全要求，也“不一定与一个标准组织需要或想要的相同”。“因此，如果国防部的安全版本比更‘基本’的版本更贵，我会选择基本的，”隆巴顿说。“只有那些需求与军事环境相同的组织，如石油和天然气勘探或制药开发，才会真正需要这些能力。”

空军首席软件官兼国防部企业开发项目联合负责人 Nicolas chai llan[表示，CloudBees 的 CI 平台符合国防部的容器规范，也是国防部提升美国军事和政府能力以提高其软件开发能力以及支持国防部开发项目流程的任务的一部分。](https://www.linkedin.com/in/nicolaschaillan)

在最近由 [Brian Dawson](https://www.linkedin.com/in/bvdawson) 主持的播客中，Chaillan 表示，国防部发展战略任务的一个核心部分是——反映私营部门的一个普遍目标——消除发展战略管道中的结构孤岛。

“我认为我们在采用 DevSecOps 方面相当成功的原因是，我们真正建立了这个集中化的团队，并使之成为一种集中化的企业服务。我认为不这样做的人，让每个团队做他们需要做的事情，只会减慢他们的速度，”Chaillan 说。“我认为您需要能够为 DevSecOps 带来这种企业服务概念，同时不要一刀切，以便提供选项，不要太多，也不要太少……这很难理解。”
