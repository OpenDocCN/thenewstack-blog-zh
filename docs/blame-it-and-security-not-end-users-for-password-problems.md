# 将密码问题归咎于 IT 和安全部门，而不是最终用户

> 原文：<https://thenewstack.io/blame-it-and-security-not-end-users-for-password-problems/>

最终用户没有最佳的安全实践，但去年向远程工作的过渡提醒我们，技术人员在大声抱怨之前应该照照镜子。

BitWarden 报告称，53%的 IT 决策者通过电子邮件共享密码，比一年前的 39%有了大幅增长。在信息安全专业人士中，对安全的冷漠尤其令人担忧，在 *[Passw@rd 的 2021](https://1passwordstatic.com/files/resources/2021-state-of-secure-access-report.pdf)* 访问状态报告中，84%的人感到筋疲力尽。在那次调查中，将近一半(44%)的信息安全专家认为安全规则和政策“不值得争论”

职业倦怠导致工作流动性，一旦员工离开公司，这将成为更大的安全问题。在 [Teleport](https://goteleport.com/?utm_content=inline-mention) 的 [基础设施访问状态 和安全报告](https://goteleport.com/pdf/state-of-infrastructure-access-2021.pdf) 中，30%的受访者不到 75%的人认为前员工的所有访问密钥都已被撤销。

## 访问管理，然后是无密码，然后是零信任

用另一种身份验证方法补充密码有所帮助，但不能解决一些人集中访问基础架构的根本问题。70%的 Teleport 研究仍然允许使用密码访问基础设施。尽管 VPN 已经失宠，但在这项研究中，仍有一半人使用 VPN，硬件安全令牌也是如此。一次性电子邮件链接和短暂的基于身份的证书不太常见。

Teleport 将[基于证书的认证](https://goteleport.com/how-it-works/certificate-based-authentication-ssh-kubernetes/)视为被称为[无密码](https://goteleport.com/blog/passwordless-infrastructure/)基础设施的基础。几乎一半不使用短期证书的人认为正在使用的认证系统已经足够了。大多数人不认为解决方案太复杂，因此零信任的企业计划可能会开花结果。

零信任得到了很多关注，大多数人的路线图上都有它。Dimensional Research 进行的 2021 年 IT 安全专家调查发现，51%的人已经开始实施零信任。同一项研究还报告称，四分之三的人认为零信任至少对其组织的安全态势非常重要。

对基础设施无密码访问是访问管理的一个组成部分。零信任需要有效的访问管理系统。优先考虑零信任的决策者也应该实际考虑管理他们现有系统的策略和访问管理所需的阻塞和处理。
