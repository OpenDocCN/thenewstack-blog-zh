新的一年会给云带来什么？

我们问自己，每年年底，有时感觉像是在瞎猜——在这么短的时间内，技术在主流中发展得如此之快，以至于谁能确定接下来会发生什么？

话虽如此，现在云世界正在发生一场运动，预计将在 2018 年大爆发:采用 [Kubernetes](/category/kubernetes/) 开源容器编排引擎。更具体地说，2018 年将带来一系列挑战，Kubernetes 社区将奋起迎接并优雅地解决这些挑战，或者用鲜血、汗水和泪水来奋斗。

Kubernetes 赢得了集装箱战争。接下来是成长的烦恼。

在这一点上，可以肯定地说，Kubernetes 已经有效地赢得了容器编排之战，一旦 Docker Swarm、Mesos 和 Pivotal 等竞争对手宣布他们将为他们的平台添加 Kubernetes 支持，这一发展就几乎成为官方发展。如果 2017 年是行业团结在单一集装箱化解决方案背后的一年，那么过去的 12 个月已经清楚地表明:这个单一的解决方案就是 Kubernetes。

但是，赢得战争不是一切。战利品属于胜利者，在这种情况下，这些战利品带来了一系列新的成长烦恼，如果 Kubernetes 想要保持胜利者的地位，它现在需要处理这些烦恼。毕竟，就在不久前，OpenStack 还被类似地称为“赢了”——一旦企业用户开始在生产中大规模操作它，并不得不学会应对集成复杂软件的所有挑战，这场胜利就变得不那么崇高了。

> 这必须是一项团队运动，每个人都被激励去做正确的事情，而不是以牺牲更大的利益为代价去追求自己的个人道路。

为此，2018 年可能是 Kubernetes 社区成败的一年，他们需要从其他社区项目中吸取经验教训。尽管 Kubernetes 很受欢迎，但它仍处于生命周期的早期，尚未面临被具有不同需求的高要求企业用于生产的一些挑战。

### Kubernetes 在 2018 年必须实现的三个目标

以下是 Kubernetes 社区必须认真对待的成长中的烦恼，如果他们想在未来一年真正保持容器编排领域的领先地位的话:

*   **明确目的:**最终目标是什么？实现这一目标的途径是什么？有一个设定的目标可以统一社区中的支持和兴趣，因为每个人都知道他们前进的方向是什么、如何以及为什么。类似地，未能让所有人都清楚地理解这一目标可能会让 Kubernetes 社区陷入混乱和混乱。
*   **建立/保持强有力的领导**:强有力的领导者可以推动社区实现最终目标——但是这些领导者是谁呢？仅有一个目标是不够的；还必须有坚强的个性，能够凝聚实现这一目标所需的支持，并让所有人都关注眼前的任务。如果没有人推动社区朝着目标前进，那么目标就不那么重要了。
*   **实施良好的社区管理**:社区内部可能会出现分歧；这是创建社区的自然组成部分。但是，必须实现微妙的平衡，在这种情况下，建设性的讨论和分歧是受欢迎的，但不要达到具有破坏性或过度受企业供应商利益影响的程度。需要一个强大的社区管理存在来缓和那些不可避免的分歧，并让更大的社区站在你这边。

OpenStack 在社区管理方面取得了成功[,但从一开始就很难勾勒出一个清晰的目标](https://thenewstack.io/7-ways-kubenetes-avoids-openstack-like-hype-cycle/),让社区能够同意并支持，然后实施强有力的技术领导来推动社区实现该目标。OpenStack 正在解决这些问题，但这需要时间，Kubernetes 社区在进入 2018 年时应该注意这一点。

### 让团队专注于任务

在管理这些成长烦恼的同时，也防止了所谓的“简历驱动型开发人员”的出现——换句话说，社区内的开发人员更专注于填写简历和成名，而不是投资推动社区项目。这必须是一项团队运动，每个人都被激励去做正确的事情，而不是以牺牲更大的利益为代价去追求自己的个人道路。强有力的技术领导和社区管理至关重要——否则 Kubernetes 将成为下一个反面教材。

预测云的未来是一项棘手的任务，因为它已经证明自己在如此短的时间内发展如此之快。而且，我不会说 2018 年将是“云之年”，主要是因为我们已经过了那个点！但是，如果 2018 年将是任何人的一年，那将是库伯内特之年。随着 2017 年的临近，所有迹象都表明 Kubernetes 能够很好地理解和应对未来的挑战。

在 Canonical，我们肯定是这样认为的，这就是为什么[我们继续投资 Kubernetes](https://www.ubuntu.com/kubernetes) 并扩大我们的客户群。新的一年将提供新的技术、新的机会和新的项目——所以对于我们这些在这个领域忙碌的人来说，前方是有趣的时光。

管理 Kubernetes 的[云本地计算基金会](https://www.cncf.io/)和 [OpenStack 基金会](https://www.openstack.org/)是新堆栈的赞助商。
