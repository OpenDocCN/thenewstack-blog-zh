# 一个项目如何从云计算基础毕业

> 原文：<https://thenewstack.io/how-a-project-graduates-from-the-cloud-native-computing-foundation/>

KubeCon + CloudNativeCon 赞助了这个播客。

[一个项目如何从 CNCF](https://thenewstack.simplecast.com/episodes/how-a-project-graduates-from-the-cncf) 毕业

最初由打车公司 Lyft 在三年多前创建的 [Envoy](https://envoy.com/) 是一个面向微服务网状架构的网络代理，大约在两年半前开源。在 Envoy 1.0 发布大约一年后，Envoy 被[云本地计算基金会(CNCF)](https://www.cncf.io/) 接受为孵化项目。

在 kube con+CloudNativeCon North America 2018 的播客中，由 New Stack 创始人兼主编 Alex Williams 主持，[CNCF 生态系统总监 Cheryl J Hung](https://www.linkedin.com/in/cheryljhung/?originalSubdomain=uk) 和[Lyft](https://www.linkedin.com/in/mattklein123)高级软件工程师 Matt Klein 讨论了 Envoy 作为 CNCF 项目如何受益，以及 Envoy 的生命周期如何演变。

他们详细描述了当一个 CNCF 项目“毕业”时会发生什么，以及用户最终如何从这个过程中受益。

“总的来说，这对我们来说是一次很好的经历——CNCF 对帮助我们发展项目非常有帮助，不仅仅是从事件的角度，还包括管理和所有这些类型的事情，”Klein 说。“因此，特使自开源以来，在过去两年中有了相当惊人的增长，我认为通过自然的项目增长和演变，特别是在 2018 年，很明显是时候考虑毕业标准了。”

在特使与 CNCF 合作，以确定解决方案是否符合 CNCF 的标准，并最终对其资格进行表决后，特使获得了毕业地位。克莱恩说:“我认为[毕业过程]是该项目的成熟以及更大的行业对其向前发展的信心的一个很好的证明。”。特使开始作为一个孵化项目。“我认为，这主要是因为我们在 Lyft 已经有了非常广泛的生产用途。因此，我认为，我们相当有信心，它有一个基本的稳定水平，使它可以被其他可能想拿起它的用户接受，”克莱恩说。“他们开始孵化的原因显然是 Lyft 当时是唯一的用户。

Hung 说，CNCF 定义的标准包括采用率、寿命以及开源项目是否可以用来构建产品。像每个 CNCF 项目一样，特使也有“相关的成熟度等级”洪说:“因此，成熟度水平是在毕业水平上表明这是经过战斗考验的东西，具有良好的采用和牵引力，”洪说。

一个项目是否符合标准主要由 CNCF 技术监督委员会(TOC)管理。Hung 说，TOC 由社区技术专家和布道者组成，他们被选举出来并判断生产用例是否足够成熟。

目前，Envoy 有来自五个不同组织的 10 或 11 名维护人员。此前，当 Lyft 首次开源该项目时，“Lyft 只有编写代码的人，实际上，我自己编写了大部分代码，”Klein 说。“如果你真的看看过去几年的数据——有一个很好的演示，其中一个维护者对项目的贡献进行了分析——Lyft 的贡献实际上已经下降了，”Klein 说。“从谷歌开始有一条曲线，现在我们从所有类型的其他玩家开始有一条曲线上升。所以，它确实经历了三个阶段:从只有 Lyft 的项目，Lyft-Google 的项目，到现在 Lyft、Google 和一大堆其他人的项目。”

这个过程反映了 CNCF 如何“寻找一个可持续的维护系统，这样，你知道，如果一个人或一家公司退出，这并不意味着这个项目会消失。”克莱因说:“我认为，从特使的角度来看，我们显然已经做到了这一点。”。"我认为这个项目现在最终可以自给自足了。"
