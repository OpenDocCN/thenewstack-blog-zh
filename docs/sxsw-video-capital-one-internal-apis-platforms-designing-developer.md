# SXSW 视频:内部 API、平台和为开发者设计的第一资本

> 原文：<https://thenewstack.io/sxsw-video-capital-one-internal-apis-platforms-designing-developer/>

本周早些时候，Capital One 在新推出的开发者门户网站 [Capital One DevExchange](https://developer.capitalone.com/) 中开放了一些 API。该公司在本周和下周在得克萨斯州奥斯汀举行的 [SXSW 互动](https://thenewstack.io/sxsw-capital-one-opens-apis-third-party-e-commerce-use/)上首次推出了新的开发者交流。

在这个现场视频讨论中，Capital One 的 Naveed Anwar、Dave Goldberg 和 Harrison Williams 讲述了 Capital One 如何在内部使用 API 及其 API 生命周期管理系统的开发。我们还讨论了 Capital One 的内部平台、工具开发以及对其中间件环境的影响。

Capital One 有很多关于公司如何在为外部开发者社区做任何事情之前建立内部基础设施的内容可以分享。例如，从 SOAP 到 RESTful APIs 的转变，以及采用现代编程语言和构建新工具，同时维护旧的中间件环境。

Capital One 的 API 生命周期管理平台用于规划、设计、生产和发布。任何团队都可以使用公司开发的 API 生命周期管理过程来开发新的服务。在 Capital One DevExchange 的案例中，[公司开放了三个 API](https://thenewstack.io/sxsw-capital-one-opens-apis-third-party-e-commerce-use/):

*   **SwiftID** ，基于 Capital One 的数字客户账户提供双因素认证服务。
*   **奖励**，提供 Capital One 客户奖励信息，如里程、积分和客户获得的现金奖励。
*   **信用优惠**，其中向 Capital One 金融服务返回一组资格预审的定制优惠，如信用卡。

戈德堡解释说，三年前，该公司将 API 提升到了不仅仅是技术问题的高度。它变成了一项设计工作，重点转向开发人员如何使用 API，以便它们可以重用，并且长期来看更有效。该流程使其能够推出 Capital One DevExchange。

欢迎来到 Antone SXSW 的 Capital One House，在这里，新堆栈将采访技术专家和开发人员，讨论 API、平台以及在新堆栈经济中构建应用的新方式。

Capital One 的 API 平台总监哈里森·威廉姆斯(Harrison Williams)表示，Capital One 在开始其旅程时拥有 9000 个中间件服务。但是与中间件有很多重叠，它们没有被很好地记录，并且很难发现。他们应用产品管理实践，通过开发他们的 API，他们开发工具，使用现代 API 管理技术，如 [Swagger](https://swagger.io/) 。

我们的目标是获取工具，并在有意义的时候让内部和公众都可以访问它们。他们计划在接下来的几个月中引入更多的工具，使 API 生命周期对于那些开始自己的内部开发开放 API 文化之旅的外部客户来说更加容易。
