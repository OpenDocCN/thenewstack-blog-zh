# 阿蒙·达德加尔谈哈希公司的实践方法

> 原文：<https://thenewstack.io/armon-dadgar-on-hashicorps-practitioner-approach/>

艾蒙·达德加尔和 T2【米切尔·桥本是长期的开源实践者。大约十年前，当他们创办 [HashiCorp](https://www.hashicorp.com/?utm_content=inline-mention) 时，他们就把对实践者的关注作为他们方法的核心。今天，哈希公司是一家上市公司。

在创办 HashiCorp 之前，Dadgar 和 Hashimoto 是华盛顿大学的学生。在大学期间和之后，他们开始接触开源软件，学习如何在开源软件中构建软件。

HashiCorp 的业务是这两个开源社区从业者的产物，HashiCorp 的联合创始人兼首席技术官 Dadgar 在本月早些时候洛杉矶 HashiConf 全球会议上接受采访时说。

Dadgar 说，他们两人都想重新创建异步协作，他们非常喜欢他们作为实践者参与的开源项目。他们知道他们不想要官僚主义或难以遵循的路线图。

Dadgar 引用 Terraform 作为他们方法的例子。 [Terraform](https://thenewstack.io/terraform-cloud-now-offers-less-code-and-no-code-options/) 是 HashiCorp 的开源、基础设施代码、软件工具，反映了该公司在提供良好用户体验的同时控制其核心的模式。这种体验超越了社区开发，进入了应用程序架构本身。

“如果你是一个周末战士，你想贡献一些东西，你不会去阅读这个庞大复杂的代码库来理解它是如何工作的，只是做一个集成，”Dadgar 说。"因此，我们为 Terraform 建立了一个非常特殊的集成表面区域."

Dadgar 说，集成大约有 200 行代码。他们称集成为他们的核心加插件模型，带有一个说明性的支架，如何集成的例子，以及 SDK。他们整合的“黄金路径”是该公司如何开发一个如今拥有约 2，500 家提供商的计划。

HashiCorp 开源模型依赖于它的核心和插件模型。在 Twitter 上，一个人问为什么 HashiCorp 不是一家专有公司。在我们的采访中被问到这个问题时，Dadgar 提到了 HashiCorp 的开源方法。

“哦，这是一个有趣的问题，”达德加尔说。“你知道，我认为这是一个很难扩大规模的公司。我的意思是，如果你看一下像一个 TerraForm 社区或金库——有成千上万的贡献者。这就解决了整合问题。对吗？因此，如果你说，我们是专有的，嘿，需要多少工程师来构建 2000 个 Terraform 集成？会比现在多很多人。所以我认为从根本上来说，开源帮助你解决的是这样一个事实，你知道，现代基础设施有这种真正广泛的集成表面。我认为，作为一家自营企业，你无法解决这个问题。”

“我不认为我们能够实现如此广泛的整合。我们也许可以覆盖核心云提供商。但是你会有 50 个平台提供商，而不是 2500 个。”
