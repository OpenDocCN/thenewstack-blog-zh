# CDN 提供商与部署 Jamstack 的 Web 开发人员竞争超大规模云

> 原文：<https://thenewstack.io/cdn-providers-rival-hyperscale-clouds-for-web-developers-deploying-jamstack/>

技术专家、投资者和记者继续对下一代网络发展可能带来的[伟大的解绑](https://thenewstack.io/the-great-unbundling-jamstack-and-the-future-of-the-web/)或[解绑](https://www.protocol.com/amp/jamstack-web-development-edge-2653308517)的可能性保持敬畏。他们还应该注意内容交付网络(CDN)提供商如何包装边缘和功能相关的服务，以与云计算巨头竞争。

与 2018 年和 2020 年的无服务器研究相比，最近发布的[“2021 年 Jamstack 状况”](https://kontent.ai/resources/state-of-jamstack-2021-report)报告表明，面向网络开发者的平台实际上已经获得了牵引力。这让我们相信，接受调查的开发人员不仅仅是早期采用者，而是愿意与全新的服务提供商合作。

该研究询问了 529 名 Jamstack 开发人员，他们通常在哪里托管和部署他们的项目，与 2020 年的调查相比，只有 29%的人提到了微软 Azure，而亚马逊网络服务从 34%下降到 25%。出于某种原因，谷歌云没有被问及。与此同时， [Cloudflare](https://www.cloudflare.com/) 在被添加到最新版本的调查问卷中时，获得了 27%的支持率。所有这些受访者可能都没有使用针对 Jamstack 用例的 [Cloudflare Pages](https://pages.cloudflare.com/) ，因为 DNS 和 CDN 只是该公司 36 个产品阵容中较为突出的几项服务。

在 14%使用的混合中看到 Akamai 并不奇怪。长期的 CDN 提供商[继续创新](https://thenewstack.io/akamai-brings-key-value-data-to-the-edge-adds-api-acceleration/)，提供无服务器边缘计算产品和能力，以加速作为 Jamstack 应用核心的 API 请求。

[盖茨比云](https://www.gatsbyjs.com/products/cloud/)、[数字海洋](https://www.digitalocean.com/)和 Netlify 各有 21%被认为是部署和托管 Jamstack 项目的地方。Gatsby Cloud 和 Netlify 也将自己推销为 web 开发人员的平台，开发人员可以在那里构建他们的应用程序，将它们部署到边缘位置，然后在执行功能或应用程序工作流的其他部分的第三方的帮助下运行它们。

Jamstack、 [edge](https://thenewstack.io/category/edge-iot/) 和[无服务器](https://thenewstack.io/category/serverless/)架构认为，将数据、业务逻辑和计算放在离应用的最终用户更近的地方，将带来性能提升和成本降低等好处。也许 AWS 和 Azure 可以基于它们的大小和规模提供这些，但是它们应该提供什么类型的开发者体验呢？它应该面向前端、后端、全栈、边缘还是物联网开发者？这完全取决于你对非捆绑网络的看法。
