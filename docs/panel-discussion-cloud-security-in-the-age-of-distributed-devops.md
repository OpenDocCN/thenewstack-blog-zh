# 小组讨论:分布式开发运维时代的云安全

由于新冠肺炎全球疫情，许多 IT 系统正面临前所未有的工作负载，每天都达到使用水平，而这通常只发生在一年中最繁忙的日子。好消息是，云原生方法已经在大大小小的企业中迅速流行起来，以帮助满足这些突然的需求。并且必须在这些新兴的云原生系统中构建适当的安全预防措施。

当然，由于新冠肺炎全球疫情的爆发，我们大多数人的工作都发生了变化。但阿伦德指出，这可能是一个很好的机会，让组织重新思考他们如何处理业务连续性和弹性。那些已经踏上数字化之旅的人比那些刚刚起步的人更好地度过了这场危机。现在是关注数字创新的大好时机。

事实上，如果说有什么不同的话，那就是创新正在加速，Morello 表示同意。虽然没有亲自互动的能力，但支持数字化转型的工具——Kubernetes、containers——可以帮助人们更有效地运作。

云本地计算基金会最近的一项调查发现，采用云本地技术的主要原因是提高运行服务的可用性，提高服务的可扩展性和更快扩展的能力。Hung 补充说，这些正是目前组织所需要的特征。

这就是安全如此重要的原因，阿伦德说。企业需要确保其人员和数据的安全，并且企业可以在大规模远程环境中安全运营。

Morello 提供了如何做到这一点的技巧。他建议工具、数据和知识，您的安全工具不会被锁在小仓库中。组织应该使它们可用，并对所有其他正在使用的框架和工作流开放。

他说:[零信任网络](https://thenewstack.io/how-to-start-applying-googles-zero-trust-model/)的理念在这里很重要。这实际上是为了保护应用程序的各个组件，并能够限制爆炸半径。因此，如果有一个妥协，它是非常有限的。

阿伦德建议说，这也是开始考虑 T2 发展合作的好时机。然而，这需要一些计划。她建议在项目开始前的一开始就进行安全风险评估。确保端到端地测试这些概念，并接受你不能解决第一个版本中的所有问题。

Morello 对迁移到 DevSecOps 提出了一些建议:自动化生命周期中所有的接触点，这些接触点现在都是手工操作的。此外，尽可能在生命周期的早期和整个生命周期中引入预防性的方面。
