# 集装箱，Kubernetes 在医疗人员配备方面的燃料创新

> 原文：<https://thenewstack.io/containers-kubernetes-fuel-innovation-in-medical-staffing/>

向数字化的转变对任何行业都至关重要，尤其是医疗保健和为其服务的企业。对于在快速变化的世界中提供服务于基本前线提供商的应用程序的组织来说，基于容器和 Kubernetes 的 IT 策略正日益成为游戏规则的改变者。

例如，为了管理人员需求，医院依赖数字平台和人才管理工具。他们越来越希望采用数字门户来降低复杂性，并更轻松地访问员工数据，如许可证和雇佣合同。这一点至关重要，因为他们正在为空缺职位寻找具备合适技能的人。

在过去的两年里，疫情给医疗机构带来了更大的压力，医疗机构一直面临着严重的人员短缺。医疗人员行业——无论是临时职位还是永久职位——正在发展其高接触、礼宾式的服务，以满足医院和医疗专业人员在数字时代的需求。对于人力资源行业的创新 IT 团队来说，加速新应用的交付意味着致力于云原生方法和基于 Kubernetes 的标准化。

## **便携性=为未来做好准备**

现在，容器和 Kubernetes 是一种更加成熟的技术——更容易使用，便于携带，因此非常适合需求的发展。例如，对于总部位于犹他州的 CHG 医疗保健公司来说，这种便携性是其为医疗保健提供商构建一套数字解决方案的战略的重要组成部分。

CHG 帮助医疗机构找到他们需要的员工和专业知识。它提供了美国 30%以上的临时医疗雇员。专注于云原生开发使其能够获取部署在 AWS 上的开发人员代码，进行一些调整，然后将其应用于 Azure、GCP 甚至内部部署——只要有需要。

CHG 团队专注于 DevOps 方法和使用 HashiCorp 的 Terraform 作为代码的基础设施，这使其能够在为业务需求开发应用程序时快速行动，从而加快上市时间。它始于云中的 AWS，在成为多云的过程中支持灵活的应用部署。

## **存储层的灵活性**

像云原生堆栈中的任何元素一样灵活的存储层是在云中运营服务和管理成本的核心。优化和自动化存储管理提供了许多好处，从更快的软件升级到快速启动新集群的能力。

虽然许多组织报告了 Kubernetes 的成本管理问题，称难以分配和过度配置资源，但 CHG 在使用 Portworx 的持久存储将 Kafka 事件流数据从托管云服务迁移到 Kubernetes 后，每月可收回多达 5000 美元。

由于备份和恢复是首要问题，因此在恢复关键应用程序数据时完全降低风险意味着选择一个工具集，该工具集可以灵活地为任何环境(从 Kubernetes 到云或本地)中的工作选择合适的工具。例如，CHG 利用容器快照轻松自信地支持迁移。

## **前进的基础**

CHG 工程总监 Reed Glauser 表示，it 领导者构建基础架构以确保他们的团队了解到目前为止已经交付了什么，这一点非常重要。分享这种理解意味着其他人可以订阅它并做出贡献。

基于云的部署和云原生应用开发和基础设施的美妙之处在于，它非常灵活，并将控制权交给了用户。云原生技术加快了创新的步伐，因为基础设施不再碍事。自动化应用程序开发和部署可按需扩展，支持用户和应用程序的快速增长。
