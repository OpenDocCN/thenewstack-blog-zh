# 图表数据库如何改变我们与数据的关系

> 原文：<https://thenewstack.io/how-graph-databases-are-changing-our-relationships-with-data/>

[图形数据库如何改变我们与数据的关系](https://thenewstack.simplecast.com/episodes/how-graph-databases-are-changing-our-relationships-with-data)

![](img/3c6c911ee8fb7db26979e297f81c8fae.png)

贾斯汀比伯和我们的关系有什么关系呢？如果不说别的，他在 Twitter 上惊人的 1.058 亿粉丝(在这次对话的时候)对数据科学家来说是一个真正的挑战。如果你想知道 Bieb 的集合与他和其他人之间的关系，你永远无法用实体优先的关系模型来实现——你知道，列、行和电子表格，或者像维基百科所说的，“一个简单的关系数据库实现，表中的每一行代表一个实体类型的一个实例，表中的每个字段代表一个属性类型。”如果您真的想开始理解 Beib 的广泛支持者背后的原因，您需要首先组织数据关系。

这只是现实世界中的一个例子[Denise Gosnell](https://www.linkedin.com/in/denisekgosnell/)博士在最新一集的 [The New Stack Makers](/podcasts/makers) 播客中提供的例子，关于图形数据库的重要性及其背后的技术正在改变我们思考和组织数据的方式。 [DataStax](https://www.datastax.com/) 的全球图形实践负责人，也是图论、图形算法、图形数据库以及图形数据在所有行业垂直领域的应用方面的世界顶尖研究人员之一。

[data tax Enterprise(DSE)](https://www.datastax.com/resources/tutorials/what-is-enterprise)构建在开源 [Apache Cassandra](http://cassandra.apache.org/) 之上，从这些不同的数据源中查询信息，并利用 DSE 图来绘制这些数据之间的关系。然后，[随着边缘计算和 5G 越来越受欢迎](https://thenewstack.io/how-edge-computing-will-deliver-on-the-promise-of-5g/)，图表将能够加速这些见解，并将它们与实时机器学习和自动化相结合。

许多企业都在使用图计算，但是没有真正理解如何使用它。戈斯内尔指出的第一个错误是，人们在没有真正理解数据之间的关系的情况下就开始实现 graph。她举了一个从你家步行到杂货店的例子。当然，谷歌会给你一两条路，但是要到达那里，你会遇到更多的转弯和捷径。不理解这种复杂程度会导致企业无法正确地对数据和关系进行建模。

戈斯内尔说，正确的图形数据库是“大小合适的工具，用于表示、存储和查询数据中存在的关系。”

大多数工具都是为存储和组织实体(概念或对象)而构建的，但 DSE Graph 等工具的出现是因为我们很快意识到大数据的价值不在于大数据，而在于这些数据之间的关系。
