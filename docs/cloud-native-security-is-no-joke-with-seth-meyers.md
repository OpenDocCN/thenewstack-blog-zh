# 云原生安全不是和塞斯·梅耶斯开玩笑的

> 原文：<https://thenewstack.io/cloud-native-security-is-no-joke-with-seth-meyers/>

本期的 [The New Stack Makers](/podcasts/makers) 播客将在 Palo Alto Networks 的云原生安全虚拟活动中展示演讲者。开场不是别人，正是美国艾美奖获奖喜剧演员[塞斯·梅耶斯](https://www.imdb.com/name/nm1024878/)，他出演了《[塞斯·梅耶斯深夜秀](https://www.nbc.com/late-night-with-seth-meyers)》和《[周六夜现场](https://www.nbc.com/saturday-night-live)》(SNL)，名声大噪，他采访了[帕洛阿尔托网络](https://www.paloaltonetworks.com/cloud-security?utm_content=inline-mention)创始人兼首席技术官尼尔·祖克。

Meyers 在会议开始时宣称“就像 Nir Zuk 一样，我也是一名网络安全专家。”他还说他不想“过分吹嘘”自己的成就，但他说用你母亲的娘家姓来恢复密码是他的主意。

Meyers 然后问 Zuk，cloud native 对组织意味着什么，以及它对安全管理的影响。

对于 Zuk 来说，首先要明白云原生环境和平台首先是为云设计的。

“当我们从内部架构迁移到云时，情况必须改变。如果我们只是做所谓的“提升和转移”，也就是说将您现有的任何东西都转移到云中，这是行不通的，当然，它没有利用云中的任何东西，这需要大量的资金等等，”Zuk 说。“因此，为了获得云的所有优势，您必须成为云原生用户。这既适用于要移动的应用程序，也适用于这些应用程序的安全性。”

[塞斯·梅耶斯和客人如何学习云原生安全不是闹着玩的](https://thenewstack.simplecast.com/episodes/how-seth-meyers-and-guests-learn-cloud-native-security-is-no-joke)

虽然有许多明显的理由将网络安全原则应用到云原生环境，但组织也通常会很快意识到云安全不同于传统的数据中心安全。Zuk 说，如何部署安全并将其与自动化 CI/CD 管道和其他 DevOps 流程集成，这些都是为了“网络安全本身成为云原生的”而必须解决的挑战。

云提供商也分担数据保护的责任。“我认为，最初，对于谁负责云中的安全存在一些困惑:例如，是我的云提供商…负责我的安全，还是我负责我的安全？”祖克说。“我认为现在很清楚，这是一项共同的责任，云提供商负责基础架构的安全性，应用程序所有者负责保护基础架构以外的一切，包括操作系统、应用程序、网络等。”

另一个考虑是“你把网络安全放在哪里？”祖克说。“你希望使用云的语言，而不是传统数据中心的语言，”Zuk 说。“您不希望使用 IP 地址和端口号等，您希望使用工作负载、应用程序以及您在云环境中使用的其他术语，而不是传统的数据中心。”

与此同时，尽管发生了许多广为人知的安全漏洞，当然包括[网络安全管理软件产品安全攻击](https://thenewstack.io/solarwinds-the-worlds-biggest-security-failure-and-open-sources-better-answer/)，但仍有理由对云原生安全实践保持乐观。

Zuk 表示:“我们看到越来越多的客户在做正确的事情，即采用他们在传统数据中心拥有的相同功能，并找到云原生方式在云安全中实现这些功能。
