# 用 Kafka 和 Confluent 实现流式数据传输的新时代

> 原文：<https://thenewstack.io/travelling-the-river-of-data-with-confluent/>

由 Apache Kafka 分布式流媒体平台的创造者推出的 [Confluent](https://www.confluent.io) 不仅维护 Kafka，还肩负着教导世界数据驱动的应用生态系统可以为商业做些什么的使命。在今天这一集的[中，新的堆栈制造商](https://thenewstack.io/podcasts/Makers)，融合的首席执行官兼联合创始人 [Jay Kreps](https://www.linkedin.com/in/jaykreps) 接受了 TNS 副播客制作人 [Kiran Oliver](https://thenewstack.io/author/ckoliver/) 的采访，讨论 Kafka 多年来的发展，刚刚开始使用数据流的企业可以从该平台获得什么，以及开发人员如何解决在处理恒定数据流时通常面临的一些挑战。

[流式数据的新一天，与卡夫卡和](https://thenewstack.simplecast.com/episodes/a-new-day-for-streaming-data-with-kafka-and-confluent) 汇合

“谈论流媒体几乎是一个未知的领域。当我们最初把卡夫卡作为开源软件发布的时候，即使在那时，我也不认为人们知道该怎么做，”克雷普斯说。

刚刚开始使用 Kafka 的公司应该考虑的一部分是，他们如何将数据从 A 点传输到 b 点。这是 Kafka 的一个亮点。在过去，数据通常每天大批量处理一次，类似于银行每天处理交易。

“在高层次上，卡夫卡所做的是让你将数据作为一种流来处理。你可以把它想象成一条流动的数据之河。云或内部迁移的结果是他们的数据分散在所有这些地方。第一个用例是在应该在的地方获取数据。将数据传输到它应该在的地方，”克雷普斯说。创建不仅能处理和接收数据，还能对数据做出反应和适应的应用程序，是小企业在开始处理数据流时可以考虑的另一件事。Kreps 强调了一个使用案例，零售巨头沃尔玛在自己的库存系统中使用了 Confluent，使其能够更好地跟踪库存水平，并确保商店级商品计数准确。

在当今的现代世界中，例如，如果一名旅行者正在用手机与他人玩游戏，然后在飞行途中失去了连接，那么他们现在可能已有几个小时的数据会发生什么情况？“唯一重要的时间是数据在你的手机上实时出现的时间，而不是它到达系统的时间，”克雷普斯指出，并补充说，“你必须解决的另一个问题是，当你改变代码时你该怎么办？你所做的基本上就是倒带时间，然后快进，在同一数据流上快速处理。通过这样做，您可以使用相同的系统和完全相同的代码。这种能够倒带和回放时间的想法是事情结束的方式，因为它非常简单。”

最终，Kreps 希望数据流将带来应用程序开发和数据处理的未来。“我们的目标是让这成为构建应用程序的默认方式。我认为我们现在才刚刚到达那个点，用这种方式构建应用程序更容易。这不仅更快，也更容易。”
