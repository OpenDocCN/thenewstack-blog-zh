# Apache 软件基金会在太多的成功中挣扎

> 原文：<https://thenewstack.io/apache-software-foundation-struggles-weight-much-success/>

Apache 软件基金会(Apache Software Foundation)正在快速发展，如果说有什么不同的话，那就是发展得太快了，Apache 总裁(T2)萨姆·鲁比(T3)周二在 ApacheCon North America 告诉与会者。

他还报告说，该组织正在运行赤字，必须找出一种模式来实现可持续发展。

他说，该组织之所以成功，是因为它在软件开发方面的多样性、独立性和开放性的理念。没有一个贡献者或公司的投入比任何其他人的投入更有价值。它有一个严格的供应商中立政策。

要做到这一点并不容易，这是 Ruby 一直坚持的观点。

## 一路跌跌撞撞

他指出，有时一个主要贡献者离开了，然后回来期望在一个项目中有一个特殊的地位，但这不是它的工作方式。

或者一个公司想要吹捧它的一个员工是项目管理委员会(PMC)“领导”除了 PMC 没有领导。

该组织在其 [18 年的历史中](https://blogs.apache.org/foundation/entry/the-apache-software-foundation-announces6)，一直[强调项目连续性的重要性](https://www.apache.org/foundation/how-it-works.html)不管主要贡献者是否离开。在另一个主题演讲中，IBM 研究员兼沃森数据平台首席技术官 Adam Kocoloski 谈到了这种心态对后来被 IBM 收购的 CouchDB 供应商 Cloudant 的重要性。

在采用方面，该组织取得了巨大的成功，领导了一些最广泛使用的开源软件，包括同名的 Apache Web 服务器、Hadoop 大数据处理平台和 Cassandra NoSQL 数据库，以及许多其他项目。该组织有 5 822 名承诺者、684 名成员、181 个项目管理委员会、57 个 podlings(拟议项目)和 96 个各种团体。

然而，Ruby 在后来的采访中说，该组织必须在决策方面变得更加有效。他说，当他去年成为总统时，没有建立预算的机制。

“是的，我们一直凭直觉飞行，”他说。

他加入董事会已经 13 年了，董事会经常在没有做出最终决定的情况下结束会议。他说，实际上，该组织需要一位首席执行官，这是一个长期由委员会统治的组织所厌恶的。

“过去，我们的想法是把所有人召集到一个房间里做决定。但是，当我们只有 25 个人时奏效的方法，在我们有几千人时就不奏效了，”他说。

他说，他一直在对董事会没有明确说明的事情做出决定。

与此同时，该组织最近公布的年度报告显示，收入刚刚超过 120 万美元，赤字约为 13 万美元。他告诉听众，该组织有大约一年的储备。

鲁比说，他第一次要求那些负责特定方面的人——基础设施、筹资、营销——对未来五年做最好、中等和最坏的预测。按照目前的增长速度，根据最坏的情况，五年后，该集团可能会亏空 933，000 美元。

因此，该集团要么削减开支，要么增加赞助。董事会两个月前在华盛顿特区开会，周二又开会讨论该做什么。他说，一致意见是重点增加赞助商，主要是通过新的宣传资料来更好地宣传为组织做出贡献的价值。
