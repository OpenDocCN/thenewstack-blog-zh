# Puppet 希望将容器技术连接到企业

> 原文：<https://thenewstack.io/puppet-wants-bridge-container-technologies-enterprise/>

Puppet 的前身是 Puppet Labs，它发布了新的模块，以支持 Docker container technologies、Apache Mesos 和 Kubernetes 到其配置管理软件中，作为拥抱新技术的新计划的一部分，该计划名为 [BlueShift](https://puppet.com/product/managed-technology/blueshift) 。

“我们的客户和社区要求我们帮助他们理解下一批最新技术。企业可能知道 Docker、Kubernetes、Mesos 和其他一些软件的名字，但是他们不清楚如何利用它们，尤其是在他们当前的基础设施中。所以实际上，他们希望我们以一种安全、可重复的方式成为他们传统基础设施和这些新工具之间的桥梁，”Puppet 产品营销高级总监 Tim Zonca 说。

Zonca 解释说，Blueshift 不是一个产品，而是“内部 Puppet engineering、我们的社区、技术公司及其社区之间的合作，以提供一种方式来利用 Puppet 作为提供和管理新技术的通用语言”。

IT 环境确实在快速变化，对当今基础设施至关重要的技术可能会被新技术取代。[皮特·莱文](http://peter.a16z.com/about/)，安德森·霍洛维茨律师事务所的普通合伙人，在上个月的开放网络峰会上分享了一些有趣的见解。他举例说，云计算不会永远存在。它将会改变，它将会变形，在他看来，这个行业将会从集中走向分散，然后再回到集中。在不久的将来，云计算模式将会分解，回到分布式计算的世界。

这将发生在每一个领域。今天最热门的技术明天会被新的技术取代。“今天，这意味着像码头工人，DCOS 或 Kubernetes 未来可能是无服务器计算或人工智能驱动的自动化或其他什么，”木偶公司的高级软件工程师加雷思·拉什格罗夫说。

明天已经来临。“许多组织已经开始关注容器和下一代操作系统，”Zonca 说。“无论他们是否希望使用这种新软件来加速开发，或者他们是否试图确定在生产中更广泛地使用这种技术的正确方法，我们都使这种新软件更易于使用，并提供一种标准和一致的方式来交付和管理它。”

Puppet 吸引了两类用户:这些技术的新手；以及已经在使用它们的用户。

Rushgrove 解释说，对于 Docker 和 Mesos，Puppet 有两个用例。一个是安装和管理底层软件，另一个是与该软件提供的 API 进行交互。“为此，Puppet 发布了[一个用于管理 Docker 恶魔和容器的模块](https://forge.puppet.com/puppetlabs/docker_platform)。另一个新模块， [docker_ucp](https://forge.puppet.com/puppetlabs/docker_ucp) ，帮助 docker 通用控制平面在所有 Docker 主机上运行。

还有用于[部署 Kubernetes](https://forge.puppet.com/garethr/kubernetes) 和用于 Consul 的[的模块，Consul](https://forge.puppet.com/KyleAnderson/consul/readme)是 [Hashicorp](https://thenewstack.io/hashicorp-containers-should-be-managed-just-like-vms-so-where-does-that-leave-pets-and-cattle/) 的开源工具，用于发现网络上的服务。

关于 Mesos，Puppet [正在开发由 Puppet 社区编写的一系列模块，帮助用户使用 Puppet 安装 Mesos 和 Mesos 框架。“Puppet 和 Mesosphere 还在合作开发一个模块来安装和配置 meso sphere[数据中心操作系统]，”Zonca 说。](https://puppet.com/blog/using-puppet-mesos)

“如果你想一想 Puppet 迄今为止的成功之处，那就是它将基础设施的底层细节从你想用它做的事情中抽象出来，”他说。“凭借 Puppet 的经验和在企业中的采用，再加上我们在代码和开源基础设施方面的路线，我们觉得自己有能力尝试推动这一对话，并有望加快创新技术的采用，”

Docker 是新堆栈的赞助商。
