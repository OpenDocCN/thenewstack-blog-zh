# 说服企业为开源做出贡献

> 原文：<https://thenewstack.io/open-source-contribution-key/>

尽管开源越来越多地被企业使用，但是开发人员仍然经常发现他们自己必须解释为什么加入开源或者为开源项目做贡献是有益的。

在这一集的[中，新的堆栈制造商](https://thenewstack.io/podcasts/)嵌入在下面，我们来看看开源社区在整个行业中的发展方式，为什么今天的企业应该对开源社区做出贡献，以及开发人员如何与他们的首席技术官合作，以确保他们已经加入的开源项目得到长期维护。新堆栈创始人 [Alex Williams](https://twitter.com/alexwilliams) 与 Capital One 董事&平台工程研究员 [Tapabrata Pal](https://twitter.com/topopal) 、思科云服务首席技术官 [Ken Owens](https://twitter.com/kenowens12) 和专门研究云技术的思科解决方案架构师 [Ann Wallace](https://twitter.com/annnwallace) 坐下来，听取他们的想法。

[为企业带来开源:贡献是关键](https://thenewstack.simplecast.com/episodes/bringing-open-source-to-the-enterprise-contribution-is-key)

“我认为到目前为止发生的事情是，一个企业一直在使用开源，但不知道他们正在使用开源。然后他们说，与其忽视和使用开源，不如承认(我们正在使用)开源。这就引出了一个想法，既然我们在使用开源软件，那就让我们也做出贡献吧，”Pal 说。

无论是作为开发人员还是作为企业来使用开源软件，长期保持开源社区及其项目的整体健康都是至关重要的。Owens 指出，思科对开源的利用采取了不同的方式，因为该公司并不试图通过开源赚钱来实现价值主张。相反，他们正专注于用开源软件解决整个行业的问题。

“我们将一个项目的成功看做是组织之外的其他人对它的贡献。我们正试图解决整个行业更广泛的社区问题，我们觉得开源是实现这一目标的途径，”Owens 解释道。Wallace 指出，虽然一些组织可能很难弄清楚如何在保留其知识产权的同时为开源项目做出贡献，但这是可以做到的。

随着讨论的继续，Pal 为在其堆栈中使用开源的开发人员和企业提供了一条明智的建议:“这不仅仅是开源一个特定的项目，而是背后的持续支持，以及背后的社区建设。”
