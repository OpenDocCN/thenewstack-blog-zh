# KubeCon 预览版:VMware 的 Kubernetes 和社区

> 原文：<https://thenewstack.io/kubecon-preview-kubernetes-and-community-at-vmware/>

[VMware](https://www.vmware.com/company.html) 赞助了这次播客。

在过去几年中，VMware 的公司结构确实发生了一些变化，但今天，VMware 的业务举措对开发人员来说尤其重要。例如，在 8 月份与[达成收购 Pivotal](/with-heptio-and-pivotal-vmware-doubles-down-on-kubernetes/) 的协议后，VMware 继续努力调整其开源贡献和服务，以满足社区不断发展的需求，并更加重视 Kubernetes。这意味着 VMware 越来越多地参与为 Kubernetes 以及其他开源项目提供和创建开发人员所需的工具和平台。

在本期的新 Stack Makers 播客中:Kubernetes、开源、开发者体验和新技术。还讨论了 VMware 在即将举行的 [KubeCon + CloudNativeCon](https://events19.linuxfoundation.org/events/kubecon-cloudnativecon-north-america-2019/) 大会上的角色。现场讨论这些主题的 VMware 嘉宾包括:

这三位嘉宾都积极参与开源社区，特别是为 DevOps 工程师改善 Kubernetes 体验。Pepper 说，最终目标是帮助 Kubernetes 开发者创造“另一个可用性角度”，这将是 Kubernetes 贡献者峰会在 KubeCon+ CloudNativeCon 上的一个主要议题。Pepper 说，这个想法与其说是帮助 Kubernetes 代码或其集群变得可用，不如说是帮助确保项目是可用的。

来宾们在开源社区中的职业道路和担任 VMware 高管的职业道路也非常不同。然而，它们也反映了构成开源社区的那些人的不同背景和才能，他们中的许多人将会见面并利用 KubeCon + Cloud NativeCon 的协作环境。

例如，Liles 是 KubeCon 的联合主持人，也是一名 [K8s 工程师](https://www.linkedin.com/jobs/view/cloud-k8s-engineer-at-vmware-1211815524)，他在 VMware 收购 Heptio 时加入了 VMware。

“我可以诚实地说:我实际上参与了云的每一个方面，从它开始到今天。，”莉莉斯说。“更重要的是，让我留在这里的不仅仅是为客户提供更多的东西，而是真正创造工具来帮助开发人员更好地做他们想做的任何事情。”

然而，Pepper，同时也是 [SIG 运行运营和管理 SIG](https://github.com/kubernetes/community/tree/master/sig-release) 流程的联合主席，以及 Kubernetes LTS 工作组的联合组织者，表示他开始从 Liles 的“相反角度”参与基于云的应用程序的开发。“即使仅仅是在三四年前，如果你问我，我也会说，‘不，我永远不会在云上工作，这是一个很高的问题，’”佩珀说。

今天，Pepper 大量参与了 Kubernetes 的开发，并将 Kubernetes 平台与生产管道相集成。“当你运行一个集群时，你有一大堆其他相关的东西，所有这些都需要以某种方式集成，并作为发布过程的一部分进行联合测试，以真正实现质量，”Pepper 说。

德鲁是 Kubernetes 的 [SIG 可用性](https://github.com/kubernetes/community/tree/master/sig-usability##targetText=The%20scope%20of%20SIG%20usability,of%20different%20backgrounds%20and%20abilities.)和[多租户](https://github.com/kubernetes-sigs/multi-tenancy)工作组的联合主席，她说 VMware 在 Chef Software 为一个名为 [Habitat](https://www.habitat.sh) 的开源项目工作了几年后招募了她，该项目专注于应用程序自动化。Drew 说，当公司开始投资太平洋项目时，VMware 找到了她。Drew 说:“当我开始了解 VMware 对他们正在用 Kubernetes 和 vSphere 做什么的愿景，以及他们如何转向开源时，这听起来像是一个令人难以置信的令人兴奋的机会。

Drew 今天在 VMware 和开源社区的工作反映了整个 VMware 团队的协作努力和目标。

例如，在 SIG 会议期间，Drew 说:“我们正在考虑是否真正了解谁在尝试使用 Kubernetes，以及他们需要从 Kubernetes 获得什么。我认为很多公司一直在做他们自己的孤立研究，但这里的想法是作为一个社区一起工作，真正了解 Kubernetes 的用户。”

KubeCon + CloudNativeCon 和 Pivotal 是新堆栈的赞助商。
