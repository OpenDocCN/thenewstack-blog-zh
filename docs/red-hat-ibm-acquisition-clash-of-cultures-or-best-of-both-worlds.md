# 红帽-IBM 收购:文化冲突还是两全其美？

> 原文：<https://thenewstack.io/red-hat-ibm-acquisition-clash-of-cultures-or-best-of-both-worlds/>

[#169:红帽-IBM 收购——文化冲突还是两全其美？](https://thenewstack.simplecast.com/episodes/169-red-hat-ibm-acquisition-clash-of-cultures-or-best-of-both-worlds)

红帽被 IBM 收购是今年最大的新闻，让微软收购 GitHub 相形见绌。但此次收购之所以引人注目，原因有很多，其中之一是这是历史上第三大 IT 收购，仅次于 Broadcom 和戴尔-EMC。

New Stack 创始人兼总编辑亚历克斯·威廉姆斯与 Spark Labs 创始人兼首席执行官[劳伦·库尼](https://www.linkedin.com/in/laurencooney)、WSO2 首席执行官[泰勒·朱厄尔](https://www.linkedin.com/in/tylerjewell/)以及云原生计算基金会首席技术官兼首席技术官[克里斯·阿尼斯奇克](https://www.linkedin.com/in/caniszczyk/)一起讨论了此次收购的影响。

讨论的核心要点包括对市场的影响、对红帽开源贡献的影响、对红帽内部文化的影响以及两家公司的产品团队在争夺相同客户时可能发生的冲突。当公司将两种不同的文化融合在一起时，事情可能会出错。

## 文化冲突

两家公司的文化差异很大。“红帽是一家工程公司，而 IBM 是一家销售公司，”库尼说。

与红帽和 IBM 竞争的朱厄尔怀疑 IBM 的文化可能会对红帽工程师产生一些负面影响。“红帽有一项员工政策，允许他们为任何上游项目做出贡献，即使与红帽的商业利益有冲突，”他说，“但 IBM 不允许这样做。”

Aniszczyk 同意了。作为一个过去在两家公司都工作过的人，他可以清楚地看到文化差异。他说，许多从 IBM 来到红帽公司的人都很关心这次收购。

Aniszczyk 说:“这真的取决于 IBM 如何处理这次收购来安抚这些员工，否则随着时间的推移，他们将很难留住人才。”

它不仅仅是能够为开源做出贡献。两家公司的工作文化也不同。虽然 Red Hat 鼓励远程工作，但 IBM 一直在避免这种做法，让人们回到办公室。“所以会有一些文化冲突，”Aniszczyk 说。

在一次新闻发布会上，IBM 保证 Red Hat 将继续运营，保持所有地点不变。

可能发生冲突的不仅仅是两家公司的工程文化；销售团队也可能会陷入冲突。这两家公司有相互竞争的产品组合，他们将面临合作和整合这些产品的挑战。

“你真的不希望 IBM 和 Red Hat 的产品经理争夺同一个客户，”库尼说。

库尼看到的另一个大挑战是，IBM 受到其红利的驱动，在某些时候，他们将不得不做出艰难的决定，这可能与文化原则有一些冲突。

## 云中的一线希望

然而，并不是一切都是黑暗和灰暗的。红帽对 IBM 来说就像 Next 对苹果一样。一些专家预测 IBM 将被红帽公司反向并购。其他人认为，在 Red Hat 中，IBM 将找到理解开源文化的未来领导者；一个围绕开源创建了数十亿美元公司的领导者；一个了解开源商业的领导者。是的，我们正在谈论吉姆·怀特赫斯特，红帽公司的首席执行官。

在吸收红帽人才和产品，并带来受益于红帽贡献的大规模开源社区时，怀特赫斯特也可以成为恢复 IBM 所需的信任、透明和信心的催化剂。

库尼和朱厄尔认为，IBM 和 Red Hat 可能会推迟完成收购，以便给员工足够的时间来安顿下来，了解这次收购的真正影响。尽管我们在收购时听到了各种各样的声音，但我们还没有看到大量的人离开 GitHub。

两家公司，尤其是红帽公司，似乎都意识到了员工的恐惧，他们会小心翼翼地保持这种信任。与这种担心相反，红帽的文化很有可能会改变 IBM 的文化。我们已经看到了像微软这样的公司的转变。IBM 也不能幸免于这些变化。也许 IBM 需要 Red Hat 的领导地位，就像它需要 Red Hat 的云原生技术一样。

无论是哪种情况，正如 Aniszczyk 所言，IBM 从这次收购中获得的比红帽公司更多。

Aniszczyk 说:“一方面，我们有 IBM 庞大的销售渠道，这使它成为最好的销售公司之一。“另一方面，Red Hat 是最好的工程公司之一，尤其是在文化方面。他们拥有最现代的工程文化之一。它将两个世界的精华结合在一起。我认为在正确的领导下转变新的实体将是理想的，”Aniszczyk 说。“但我们会看到结果如何。球在他们的球场上，全世界都在看着。”
