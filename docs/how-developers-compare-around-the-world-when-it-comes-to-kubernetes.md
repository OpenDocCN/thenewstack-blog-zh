# 谈到 Kubernetes，世界各地的开发人员是如何比较的

> 原文：<https://thenewstack.io/how-developers-compare-around-the-world-when-it-comes-to-kubernetes/>

[Pivotal](https://tanzu.vmware.com/) 赞助了这个播客。

来自 Pivotal 的高管参加了在 [SpringOne Platform](https://springoneplatform.io/) 会议上录制的最新一集 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客，讨论全球开发者体验。他们特别适合就开发商作为全球客户所面临的问题提供意见，因为每位高管分别在美国、欧洲和亚洲工作。他们描述的通用线程体验还涵盖了在 Kubernetes 上部署软件的感受以及 CI/CD 最佳实践和工具。

来自 Pivotal 的播客嘉宾包括:

*   [Lionel Lim](https://www.linkedin.com/in/limlionel/?originalSubdomain=sg) ，亚太及日本区副总裁兼董事总经理。
*   [Chad Sakac](https://ca.linkedin.com/in/chadsakac) ，美洲销售。
*   [巴斯·莱门斯](https://www.linkedin.com/in/lemmens/?originalSubdomain=nl)，EMEA 副总裁兼总经理。

Sakac 称，软件团队通常分为三类，即 Pivotal 的 [Pivotal 容器服务](https://pivotal.io/platform/pivotal-container-service)中的“PKS 老兄”。有些人“迷失和困惑，不知道什么方向是向上的”和“到处都有流行语朝他们飞来。”

“他们是呆伯特漫画中头发尖尖的老板的活生生的例子，他们说‘嘿，为什么不是所有东西都是云原生的，容器和 Kubernetes，”萨卡克说。然后，在另一端，你会发现像我一样完全是个书呆子的人…对技术着迷，他们到处进行科学实验，最终，他们迷失在自己的道路上。"

第三类客户包括较少关注技术的开发人员，他们希望使用云原生应用程序开发来交付特定的业务成果。“这些人找到了正确的中间道路，”萨卡克说。

在亚洲，Lim 说他在开发商中观察到了同样的现象。该地区客户的一个主要潜流是了解他们的组织如何在当今“贸易越来越受限的时代”更好地竞争 Lim 说，他们关心的是“他们在全球竞争中能够(实现)的各种商业成果”。

莱门斯表示，EMEA 地区的客户也有类似的担忧。“我们得到的最多的问题是假设技术正在变化，假设云原生非常重要:你如何帮助我们利用这一点并在这方面做得更好？”莱门斯说。

具体到欧洲，莱门斯说“高管们不太懂技术。首席信息官们更专注于管理他们的业务和结果，而不是谈论 Kubernetes，”莱门斯说。

三大洲的工作文化显然存在差异。Lim 说，世界上一半的开发者都在亚太地区，在这个地区，一个关键的挑战是与等级森严的社会和公司结构合作。然而，Lim 的具体挑战仍然涉及全球开发运维团队面临的共同经历，例如打破阻碍开发运维及 CI/CD 的孤岛。

在“突破功能孤岛”的同时，Lim 还说“我们取得了很大的成功。我们看到越来越多的牵引力。”
