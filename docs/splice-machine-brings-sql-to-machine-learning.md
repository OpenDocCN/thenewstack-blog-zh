# Splice Machine 将 SQL 引入机器学习

> 原文：<https://thenewstack.io/splice-machine-brings-sql-to-machine-learning/>

[拼接机让商家现代化并保留定制的程序](https://thenewstack.simplecast.com/episodes/splice-machine-allows-businesses-to-modernize-and-keep-customized-programs)

该公司首席执行官 [Monte Zweben](https://www.linkedin.com/in/mzweben/) 说 [Splice Machine](https://www.splicemachine.com) 希望将 SQL 带入机器学习(ML)的世界。

在企业中，数据通常驻留在三个独立的平台上:事务数据库、数据仓库和机器学习(ML)环境。

大多数人工智能(AI)应用程序都是建立在已经加载到 ML 环境中进行数据挖掘的历史数据之上的。所以你要从后视镜里看看已经发生了什么，他说。如果您想立即做出决策，您需要一个可操作的数据存储，比如传统的数据库管理系统，或者为了获得更高的可伸缩性，需要一个 NoSQL 数据存储。最后，ML 带来了自己的算法、数据科学工作台和建模工作流程。

目前，由开发人员将所有这些工作负载整合在一起。

Splice Machine 解决了所有这些问题，并使用 SQL 作为其基础。其人工智能数据平台无缝地将所有这些组件放在一起，用一个可扩展到 Pb 的平台取代旧的数据平台，允许通过 ML 平台进行即时分析，并使数据立即可供最终用户使用。

这使得企业能够将他们花费数千美元购买的专门构建的应用程序现代化。Zweben 说，这些定制通常是其市场中的关键区别，可以理解的是，他们不愿放弃这种定制来实现现代化。但这就像他们的脚在水泥里，因为那些高度定制的应用程序不能被带入新的技术堆栈。

Splice Machine operational AI platform 允许客户保留其所有定制，消除繁琐且耗时的提取、转换和加载(ETL)流程，将 ML 组件添加到其数据分析中，并一举获得现代化堆栈的优势。

例如，Splice Machine 一直在与一家保险公司合作，该公司已经建立了用于欺诈检测的 ML 算法。但是，当索赔数据从业务系统中提取出来、通过 ETL 流程运行并加载到 ML 模型中时，它已经更改了很多次，以至于数据在 ML 位置已经过时，因此欺诈检测分数已经过时，因此不准确。

通过将 Splice Machine 添加到堆栈中，该公司能够运营那些现有的基于 Spark 的业务模型，并将它们直接注入到业务系统中，从而实现实时欺诈检测。

“这是拥有一个可操作的人工智能数据平台的缩影，”兹韦本说。它可以运行具有操作能力的业务系统，但也具有驻留在同一平台上的分析能力和 ML 模型。

请收听，了解更多关于这一可操作人工智能平台的商业用例、现代化之旅的步骤、Zweben 从 NASA 开始的 20 年人工智能职业生涯，以及这些如何影响了他对 Splice Machine 的构建。
