# 易贝如何提高开发速度

> 原文：<https://thenewstack.io/how-ebay-works-for-developer-speed/>

[The New Stack](/podcasts/makers) 最近的直播播客报道了很多关于[易贝](https://tech.ebayinc.com/?utm_content=inline-mention)连续五次尝试重组其 IT 架构的内容。

由新堆栈的创始人和发行人 [Alex Williams](/author/alex/) 主持，该录音邀请了易贝嘉宾[副总裁、工程和首席架构师 Randy Shoup](https://www.linkedin.com/in/randyshoup/)； [Mark Weinberg](https://www.linkedin.com/in/mark-weinberg-65a35950/) ，核心产品工程副总裁；买方体验工程副总裁 Lakshimi Duraivenkatesh 提供了他们对煎饼的见解和经验教训。

[https://www.youtube.com/embed/kzdTsEd3NbM?feature=oembed](https://www.youtube.com/embed/kzdTsEd3NbM?feature=oembed)

电子商务巨头易贝一直面临着许多挑战，因为它重新设计了其架构，除其他外，增加了每天部署和软件更新的频率，以最终改善其在线客户的用户体验。“我们决定要做的是把重点放在软件交付点上，”Shoup 说。“我们需要能够快速、重复和可靠地交付软件，以完成我们希望进行的任何上游架构改进。”

温伯格解释说，易贝的 DevOps 团队保持“着眼于小胜利和长期能力的总体方法”。

温伯格说:“如果我们可以为一名工程师在一项任务上节省 10 分钟，乘以该工程师一天做多次该任务，乘以一年中的每一天，再乘以整个组织中的每一名工程师，这实际上是一个相当大的胜利。”“但还有一些我们知道需要实现的长期功能，如重新架构我们的一些代码，这些是更大的变化，具有更大的影响和更大的风险，但我们知道我们需要这样做。”

[易贝是如何为开发商提速的](https://thenewstack.simplecast.com/episodes/how-ebay-is-working-for-developer-speed)

DevOps 团队文化，更具体地说，一直在寻找包含共享目标、协作和打破孤岛的方法。“易贝的所有团队需要朝着一个共同的目标努力，这一点至关重要。我们清楚地看到一场正在发生的文化变革，人们正在考虑端到端，他们有意识地打破孤岛，”Duraivenkatesh 说。“这不容易，但人们都在谈论这件事。而且，我们也看到每个人都在想，‘好吧，我想为这个特别的计划取得什么样的结果？以及我们如何共同努力实现这一目标。所以发生了很多变化。"
