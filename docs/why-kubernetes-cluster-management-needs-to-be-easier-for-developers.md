# 为什么 Kubernetes 集群管理对开发者来说需要更简单

> 原文：<https://thenewstack.io/why-kubernetes-cluster-management-needs-to-be-easier-for-developers/>

Kubernetes 无疑是云本地计算的首选平台，但广泛采用的容器编排器仍然难以配置和管理。用于配置、策略和审计管理、安全、监控和一系列其他操作相关任务的工具和平台正在出现。它们有助于简化 Kubernetes 集群的管理，同时减少手动流程。

这是为 [KubeCon + CloudNativeCon 北美](https://events.linuxfoundation.org/kubecon-cloudnativecon-north-america/)录制的 [The New Stack Makers](/podcasts/makers) 播客，主要介绍 IT 基础设施管理软件提供商 [Puppet](https://puppet.com/) 的中继技术产品经理 [Eric Sorenson](https://www.linkedin.com/in/ericsorenson) 和 Red Hat 的混合云管理总经理兼工程副总裁 [Dave Lindquist](https://www.linkedin.com/in/davelindquist) 讨论 Kubernetes 集群配置管理的状态。TNS 记者 B. Cameron Gain 主持了这一集。

[为什么 K8 的集群管理预计不会很快变得乏味](https://thenewstack.simplecast.com/episodes/why-k8-cluster-management-is-not-expected-to-become-boring-anytime-soon)

对于许多开发人员来说，集群管理总有一天会成为一个透明的过程，这样他们就可以更好地专注于创建和配置应用程序。

索伦森说:“该平台的目标是最终消失，让业务开发人员能够专注于为企业提供价值。”“整个团队可以把平凡的部分去掉，让它淹没在意识层面以下，这样他们就不会意识到这一点——(但)我确实认为我们离这一点还有很长的路要走。”

尽管在许多方面，开发人员越了解应用程序将在其中运行的集群的计算结构，他们就越能找到改进应用程序运行方式的方法，并最终为最终用户提供价值。

Sorenson 说:“您不仅需要了解事情如何运行以及它们在哪里运行的实施细节，作为一个应用程序团队，您对此了解得越多，您就越能利用这些功能。”。“我认为仍有许多移动部件没有完全凝结和固化成拨号音级别的可靠性和简单性。”

“当我们有了跨云平台的一致性级别，然后我们有了这种级别的一致性和服务，以及通过其生命周期管理集群的简单、自动化的方法，以及合规性和安全性，然后开发团队可以专注于持续集成和持续交付—他们可以专注于可能滚动更新的部署模型；他们可以轻松地进行更新和升级；他们可以轻松了解应用程序的使用情况，”Lindquist 说。
