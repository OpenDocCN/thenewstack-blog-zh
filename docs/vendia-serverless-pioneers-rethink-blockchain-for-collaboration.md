# Vendia:无服务器先锋重新思考区块链协作

> 原文：<https://thenewstack.io/vendia-serverless-pioneers-rethink-blockchain-for-collaboration/>

大约三年前，Shruthi Rao 在与 [Tim Wagner](https://twitter.com/timallenwagner?ref_src=twsrc%5Egoogle%7Ctwcamp%5Eserp%7Ctwgr%5Eauthor) 面试成为亚马逊网络服务(AWS)的无服务器业务开发主管时，她问道，“为什么没有人在区块链使用无服务器？”

这引发了一场对话，并演变成一家新的创业公司 Vendia，基于这两种技术的维恩图。

瓦格纳是无服务器的创始人之一，并为 AWS Lambda 撰写了最初的商业计划。两人之前都在加密货币交易所与区块链合作，饶在 AWS 创办并领导了区块链团队。

Vendia 刚刚宣布了一轮 510 万美元的超额认购种子投资和其初始技术 Share，作为预览。

“我会见了 1，092 名独立客户，听到他们说，‘看，我们有多个合作伙伴……有些在 AWS 上，有些在其他云上。我们已经进行了大量的物联网和边缘设备投资，产生了大量的数据…人工智能、人工智能和分析投资。我们正在尝试使用区块链，这样我们就可以从多个地方获得这些数据，并集中到一个地方，这样我们就可以对其进行一些智能分析，”Rao 说，并解释说他们希望进行自主预测、实时供应链管理等类似的工作。

但是在与第 500 个客户会面后，很明显，企业并没有像最初宣传的那样使用区块链来处理信任问题，而是在一个地方收集数据，以便用它做一些事情。

“有很多客户流失，不仅仅是我们；这是为了整个区块链产业，”她说。

“大多数区块链的 POC 和 pilots 都死在了葡萄藤上。他们从未投入生产。它是昂贵的。对于一个三人区块链来说，需要 14 个多月的时间，需要昂贵的顾问。…所以我们意识到，企业正试图利用区块链解决一个完全不同的问题，而不是“这个不存在的信任问题。”"

Vendia 旨在结合这两种技术，同时解决各自的局限性。

最近在 ANZ 举行的 AWS 无服务器社区日上，Wagner 概述了这些挑战。虽然无服务器是高度可扩展的，根据使用情况计费，并为用户消除了服务器的麻烦，但每一行代码都假设了单一所有者模型，对共享没有帮助。它是供应商专有的，基于单个帐户和单个地区。此外，它没有内置的国家支持。

与此同时，区块链基本上是一个点对点的共享平台，在每个节点上都有一个完全复制、完全有序的数据库。它提供了一个防篡改的拷贝，保证在每台机器上都是相同的。单个区块链节点仅限于一台设备或一台服务器。然而，区块链不能扩大规模。一旦你达到了 CPU、内存或网络带宽的极限，那就完了。它变成了一个复杂的 DIY 项目。它可能很慢，并且需要使用难学的语言。

“不幸的是，你必须选择一个:一个没有能力扩展的复制数据模型或一个只能在单个帐户中运行的高度可扩展的服务，”瓦格纳在他的演讲中说。

## **多账户、地区、云**

富国银行负责企业发展和新举措的高级副总裁 Guru Charan 表示:“区块链根本无法支持我们不断增长的交易量，也无法提供金融机构所需的安全保证。“Vendia 将区块链概念重新构想为一个云原生、模式驱动的解决方案，易于使用，没有供应商限制。我们对 Vendia 在分布式应用方面的承诺感到非常兴奋，无需重新培训我们的开发人员或增加运营负担。”

瓦格纳在接受采访时说，Vendia 使与在不同云平台上工作的其他部门或公司合作变得容易。Vendia 的每一项实施都跨越客户、地区和云。它提供了一个不可变的、完全有序的、防篡改的分类帐，以及应用程序运行的每个云、帐户和区域中的应用程序状态的复制副本，以及开箱即用的移动和 web 支持。

你可以把它看作一个编译器，它接受一个 JSON 模式，并在后台把它变成一个无服务器组件的集合。

他们有两三块。最低层是复制数据模型——一个完全有序、完全一致的事务数据库，具有 ACID 语义，将在所有协同工作的节点上维护该数据。

“当两家公司互操作时，他们通常不在同一个云上，他们也没有做出相同的技术选择，”他指出。

这些节点可能是两家试图共享汽车零部件信息或追踪解决方案的不同公司。或者它可能只是一个公司拥有的应用程序的两个不同部分。也许其中一个是摄取物联网信息，比如在 Azure 上，但随后希望将其存储在数据库中或在 AWS 中用代码操作它。

然后是数据复制，区块链确保一致性，确保所有的更改所有的事务都在所有部分正确、安全和一致地分布和输入。

“我们使用 GraphQL 集成中心进行设计。因此，这些应用程序中的每一个都内置了移动和 web 支持，内置了大文件和 blob 支持，内置了事务复制数据库。因此，当客户开始使用 Lambda 或 API 网关时，一些最难构建的部分已经准备好，可以开箱即用，因此他们不必重新发明轮子，”Wagner 说。

虽然预览版仅适用于 AWS，但计划中的夏末正式上市将是一个多云解决方案。

“我们不要求客户了解或理解无服务器。但是，我们希望该实施战略的产品优势能够通过一路向下扩展到完全关闭来体现，这样就没有租赁成本，而且还能够根据客户的需要纵向扩展。”

## **分享控制权**

他说，该公司的设计合作伙伴一直吹嘘能够移动千兆字节的数据，并拥有由数千人组成的参与者团体，而不是数百人一起工作。Vendia 从未集中或共享任何客户数据。公司可以以可控的方式共享数据，而不必相互信任。他们只需要信任他们的云提供商。

两家汽车制造商是设计合作伙伴，致力于跟踪汽车中的 30，000 到 35，000 个零件。Roa 解释说，这涉及到结合来自零件制造商、国家公路安全管理局以及经销商和服务中心的数据，其中 PC 可能是他们 IT 部门的范围。

“因此，我们创建了一个虚拟数据湖解决方案，您可以放入该解决方案，您可以访问他们的所有数据，而不会将数据透露给可能不需要看到这些数据的各方。因此，他们能够在不共享实际数据的情况下获取情报，”她说。这些汽车制造商已经在市场上尝试了许多基于区块链和 API 的解决方案。但是使用 Vendia，他们能够在两周半内完成概念验证。

“对我们来说非常重要的一件事是，当 Tim 推出 Lambda 时，它真的降低了任何开发者、任何公司的准入门槛。他们不必财大气粗，也可以尝试一些东西，看看是否有什么东西能改变他们的商业模式。因此，这种低准入门槛确实在这个领域催生了许多不同的公司和不同的行业，”她说。“我们想在 Vendia 身上复制这一点。”
