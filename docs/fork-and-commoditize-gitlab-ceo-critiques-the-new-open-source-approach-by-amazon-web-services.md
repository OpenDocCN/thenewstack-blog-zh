# “分叉和商品化”——git lab 首席执行官谈超级云的威胁

> 原文：<https://thenewstack.io/fork-and-commoditize-gitlab-ceo-critiques-the-new-open-source-approach-by-amazon-web-services/>

GitLab 是重新推出的@ Scale 播客的赞助商。

[【分叉与商品化】— GitLab CEO 谈超级云的威胁](https://thenewstack.simplecast.com/episodes/fork-and-commoditize-gitlab-ceo-on-the-threat-of-the-hyper-clouds)

GitLab 从许多方面展示了开源世界的可能性。从零开始创建，目标是解决一个问题——在这种情况下，如何为开发者提供一个协作软件平台；GitLab 体现了开源社区的思想和精神。

但是，在快速变化的软件开发世界中，随着科技巨头谷歌、亚马逊和微软寻求以无与伦比的资本资源重新定义软件的制作和分发方式，开源企业面临着许多挑战和威胁——其中一些挑战和威胁对许多项目来说几乎是生死攸关的。

在重新推出的 New Stack @ Scale 系列的第一集播客中，由 New Stack 的创始人兼主编 Alex Williams 主持，并在开源领导力峰会期间录制；GitLab 的联合创始人兼首席执行官 Sid Sijbrandij 讨论了当今开源开发的现状。他还讨论了新出现的威胁，如超云模型带来的威胁，以及许多人所说的 Sijbrandij 描述为亚马逊 AWS“分叉和商品化”策略的威胁。

事实上，作为亚马逊如何“分叉和商品化”一个项目的例子，Sijbrandij 使用了 [Elasticsearch](https://en.wikipedia.org/wiki/Elasticsearch) ，这是一个开放的核心模型，既有专有代码，也有开放源代码。“亚马逊所做的就是将它分叉并商品化。他们提供了另一种发行版，在这种发行版中，他们采用了许多人们付费购买的功能，比如加密，并且他们把这些功能开源了。“他们直接追求人们为 Elasticsearch 支付的价值，我认为这是一个非常有趣的发展，我们将看到这将如何发展。但很明显，商业开源公司现在很容易受到这些分叉的影响，并通过超级云进行商品化。”

虽然 AWS 否认其“分叉”Elasticsearch，但开源社区中的许多人大声疾呼这是犯规的。AWS 还被指控为 Java 采用了 [OpenJDK](https://openjdk.java.net/) ，并试图提供一个带有 [Corretto](https://aws.amazon.com/corretto/) 的替代产品。

Sijbrandij 说，专有特性曾经是“商业开源公司的生命线，现在它们是开源的”。Sijbrandij 说:“我认为这对他们的客户很好，对用户很好，对开源也很好，但作为商业开源软件公司，我们如何处理这个问题将会很有趣。”

与此同时，商业开源软件公司仍然是“开源项目的管理者，但也在围绕它建立业务，”Sijbrandij 说。“他们中的大多数都像我们一样倾向于风险投资。因此，一方面，他们希望看到开源软件的增长，另一方面，他们也需要一个流动性事件。”

GitLab 的叙述作为一个开源社区的管理者，再合适不过了。它的起源可以追溯到联合创始人、现在的工程研究员 Dmitriy Zaporozhets 住在一个没有自来水的房子里。但是，除了每天早上必须醒来并从井里打水之外，他还缺少他当时作为开发人员的全职工作所需的协作软件。

“所以，德米特里说，‘我要解决这个问题’，但他们没有钱。他说，‘好吧，我自己做就是了，’”Sijbrandij 说。"然后他出去创建了 GitLab，不到一年，就有 300 人加入了这个项目。"

Zaporozhets 创建了这个平台后，Sijbrandij 在一年后加入了进来。注意到 SaaS 平台的必要性以及 GitLab 只能下载。扎波罗热茨当时对自己说:“SaaS 是未来，所以我要创办 GitLab.com。”

2012 年，在一盘煎饼上，Sijbrandij 很快回答了第一批用户的问题

Zaporozhets 说:“我让我现在的妻子把煎饼吃完。"我上了楼，坐在那里吃着东西，回答问题，整整一个晚上."

GitLab 作为一个领先的协作软件开发平台的故事已经成为历史。
