 [Doug Davis，IBM

Doug Davis 在 IBM 的云&开源/标准部门工作，专注于增强、推广和集成基于云的开源技术到 IBM 的云产品中。在他的职业生涯中，他参与了许多其他开源项目和标准，如 CloudFoundry、Apache Axis、CIMI 以及大多数与 SOAP 相关的规范。他在 IBM 参与 W3C、OASIS 和 DMTF 等标准团体的过程中发挥了关键作用。](https://developer.ibm.com/opentech/author/dug/) [](https://developer.ibm.com/opentech/author/dug/)

有时，当一家像 IBM 这样规模的公司宣布它将为一个开源项目做出贡献时，怀疑就会出现。怀疑者会很快宣布，该公司将只致力于改善其底线的问题，或者它只是推动其议程的项目的一部分。

在某些情况下，这碰巧是真的，但这不是 IBM 的情况，IBM 高级技术人员 Doug Davis 在下面嵌入的新一集 [The New Stack Makers](/tag/the-new-stack-makers/) 播客中断言。戴维斯与新堆栈创始人亚历克斯·威廉姆斯就新 TNS 电子书“用 Docker &容器进行[自动化&编排”进行了交谈。](https://thenewstack.io/ebookseries/)

“不管是大事还是小事，都没关系。我们会在社区支持和需要的地方提供帮助。因为我们希望确保社区将 IBM 视为推动事情发展的贡献者。我们不是来推动 IBM 的议程的，”戴维斯说。

在使用容器时避免供应商锁定是 IBM 希望帮助其用户避免的事情，不管他们选择什么样的容器平台。设定社区范围的标准是 IBM 希望看到在各种层面上完成的事情，Davis 指出这目前正在由云本地计算基金会进行开发。

“我们在这些社区中寻求的是在基础设施、核心功能集以及理想情况下的一些规范以及共享参考实施方面的普遍共识，”Davis 说。

Davis 还谈到了在生产中使用容器的挑战，在生产中，开发人员通常将容器视为微型虚拟机。然而，最大限度地利用容器需要开发人员改变对工具的看法，例如与标准的 VM 设置相反的 Docker T8:“停止从机器的角度考虑问题，开始从过程的角度考虑问题。不要想，“好吧，我需要安装 WebSphere 和 MQ 以及所有这些东西来完成我的工作。更应该把它想成，“我们需要运行什么过程来完成我的工作？”
