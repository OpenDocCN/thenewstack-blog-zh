# GitLab v10 与 Kubernetes 集成

> 原文：<https://thenewstack.io/gitlab-v10-integrates-kubernetes/>

自从 [GitLab](https://about.gitlab.com/) 发布其以开发者为中心的集成应用开发平台 9.0 版本以来，已经过去了 6 个月，下载量达到了 200 万次。自那以后，该公司一直很忙，对 VMWare、索尼和 Ticketmaster 等客户公司的近 1000 名用户进行了调查，以了解他们的开发人员需要哪些功能来支持迄今为止最具企业价值的 GitLab 版本。

今天，结果出来了:GitLab 发布了 10.0 版本，增加了世界各地开发人员所要求的新特性和新功能。

GitLab 首席执行官 [Sid Sijbrandij](https://twitter.com/sytses) 表示:“GitLab 的这一版本提供了充分利用 DevOps 优势的能力，特别是基于 CI/CD 和 Kubernetes 的应用开发。他指出，该行业越来越多地采用云原生能力，利用市场上的 [Kubernetes](/category/kubernetes/) 开源容器编排软件，导致对自动化流程的需求不断增加。GitLab 10.0 提供了企业版和社区版来满足这一需求。

通过扩展 Kubernetes 功能，GitLab 10.0 中的云原生功能得到了提升。通过这一最新的集成，部署到 Kubernetes 被设计为一个无缝的过程，利用 GitLab CI 快速配置、部署和利用集群，而不管服务器可能在哪里运行。作为其成为 Kubernetes 和云原生软件开发工具的使命的一部分，GitLab 还与[云原生计算基金会](https://www.cncf.io/) (CNCF)联手。今年早些时候，GitLab 10.0 被 CNCF 评为 [30 个速度最快的开源项目](https://about.gitlab.com/2017/07/06/gitlab-top-30-highest-velocity-open-source/)和十大开源项目之一，进一步推动了该公司帮助企业获得云原生应用开发全部优势的承诺。

这一最新版本将 GitLabs Auto DevOps 与 Kubernetes 一起运行，同时提供改进和增强的 CI/CD 功能，以创建最简单、最高效的方式来自动化安全、灵活的连续交付管道。

其他新功能选项包括:

*   [Auto DevOps](https://gitlab.com/gitlab-org/gitlab-ee/issues/2517) :该特性默认自动配置软件开发生命周期，提供开箱即用的模板来自动启用和配置构建、测试、代码质量、审查应用、部署和监控。
*   [新的用户界面](https://about.gitlab.com/2017/09/13/unveiling-gitlabs-new-navigation/):更新的用户界面提供了更简单的导航，以及新的测试和监控功能。用于缩短周期时间的生产力特性包括增强的子组、部署板和 Prometheus、在对象存储库中存储文件的能力，以及对 Slack 和 JIRA 的增强集成支持。
*   扩展的 Kubernetes 功能:GitLab CI/CD 通过提供一种快速的方法来配置、部署和监控 Kubernetes 内部的应用程序，使部署到 Kubernetes 成为一个无缝的过程，无论 GitLab 是安装在集群内部还是外部。
*   问题板:组级别的问题板允许更好的协作，使跨多个项目工作的团队能够在一个视图中管理单个组中所有项目的问题。列表、标签和里程碑都在组级别的板上管理，允许用户宏观地关注组抽象。

10.0 版本的特性旨在加速工作流程的同时，使开发人员的生活更加轻松，并使强大的技术更加易于使用。Sijbrandij 说:“以 Auto DevOps 为例，开发人员可能会同时处理许多项目，为每个项目设置工具是没有意义的。Auto DevOps 更进一步，在默认情况下启用功能，完全取消了配置步骤，并提供零点击方法

他总结道，这使得经验丰富的开发人员能够节省时间，同时还能让该领域的新手更容易上手。鉴于全行业三分之二的自托管组织使用 GitLab，这些好处将带来广泛的回报。

简单地说，Sijbrandij 说，“我们正在为企业开发者提供更快地开发优秀代码所必需的工具。”

[云本地计算基金会](https://www.cncf.io/)是新堆栈的赞助商。
