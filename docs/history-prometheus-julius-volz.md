# 普罗米修斯在库比肯有自己的一天，证明了它在生态系统中的崛起

> 原文：<https://thenewstack.io/history-prometheus-julius-volz/>

由谷歌和 Docker 等公司部署的 [Prometheus](https://prometheus.io/) 是一种开源监控服务和时间序列数据库，越来越多地用于微服务监控。

在最近对[的采访中，新堆栈制造商](/tag/the-new-stack-makers/)播客、普罗米修斯创造者和基础设施工程师[朱利叶斯·沃尔茨](https://www.linkedin.com/in/julius-volz-25793713)讨论了该软件的方向，这是继 [Kubernetes](/category/kubernetes/) 之后[云原生计算基金会](https://www.cncf.io/) (CNCF)已经采用的第二个项目[。](https://thenewstack.io/cloud-native-computing-foundation-prometheus-second-hosted-project/)

“普罗米修斯日”将会引起人们的特别关注，这是 CNCF 帮助组织的，同时还有 11 月 8 日和 9 日在西雅图同时举行的“库比康日”和“云竞赛日”。

“将有来自世界各地的演讲者谈论 Prometheus 的内部工作方式，合作开发人员谈论新功能。此外，还会有用户提供一些案例研究体验，”沃尔茨说。

[普罗米修斯在库比肯有了自己的一天，这证明了它在生态系统中的崛起](https://thenewstack.simplecast.com/episodes/prometheus-gets-its-own-day-at-kubecon-a-testament-to-its-rise-in-the-ecosystem)

Prometheus 的崛起伴随着向动态云环境的转变以及对更好地理解分布式服务的需求。 [Line](https://line.me/en-US/) ，一个类似 WhatsApp 的日本服务，正在内部使用 Prometheus。沃尔茨说，谷歌已经在开源项目中加入了原生的普罗米修斯度量标准，Docker 也在自己的项目中加入了普罗米修斯度量标准。

最初[由 SoundCloud 工程师](https://thenewstack.io/prometheus-microservice-monitor-reports-production/)建造，并模仿谷歌内部的 BorgMon 监控工具，Prometheus 在本质上已经相当成熟。例如，它现在正在扩展，以集成长期存储:正在开发的是 Prometheus 背后的任何长期存储环境的本机读/写集成，将它的使用扩展到本地存储之外。这可能类似于 NoSQL 数据库，如 [InfluxDB](https://www.influxdata.com/) 或消息队列，如 Apache Kafka。

“我们已经有了一种将样本写入远程端点的方法，但我们还希望有一种回读样本的方法，”沃尔茨说。

Prometheus 越来越受欢迎的部分原因可能是它使用了一种[多维数据模型](https://thenewstack.io/contributing-prometheus-history-alertmanager/)和相关的查询语言。它正被那些喜欢使用动态云环境工具的用户所采用。Prometheus 的查询语言引起了系统诊断和监控领域其他开放项目的兴趣。

随着社区在使用云中数据方面的不断发展，沃尔茨指出，在当今的系统中，人们越来越多地意识到并采用监控工具包。“我认为我们是这类系统新浪潮的一部分，”沃尔茨说。

沃尔茨指出，CNCF 没有插手并试图微观管理普罗米修斯，而是相对不干涉该项目的日常工作。“好的一面是，他们基本上允许我们做之前计划好的事情。他们并没有试图在很大程度上影响。主要是试图提供帮助，否则，他们主要是在逃避，如果你已经在开源项目中建立了流程，这真的很好。”
