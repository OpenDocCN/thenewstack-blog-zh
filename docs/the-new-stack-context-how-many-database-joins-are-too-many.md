# 新的堆栈上下文:多少个数据库连接是太多了？

> 原文：<https://thenewstack.io/the-new-stack-context-how-many-database-joins-are-too-many/>

欢迎来到[新的堆栈环境](/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了[丹尼斯·戈斯内尔](https://www.linkedin.com/in/denisekgosnell/)，他是 [Datastax](https://www.datastax.com/) 的首席数据官，也是奥赖利的书[图表数据从业者指南](https://www.oreilly.com/library/view/the-practitioners-guide/9781492044062/)的合著者她还亲切地为我们写了一篇文章，解释为什么图形数据库在企业中获得了发展。

TNS 编辑和营销总监 [Libby Clark](/author/libby/) 主持这一集，旁边还有 TNS 高级编辑 [Richard MacManus](/author/richard/) 和 TNS 执行主编 [Joab Jackson](/author/joab/) 。

[第 126 集:Denise Gosnell，DataStax——多少个数据库连接才算多？](https://thenewstack.simplecast.com/episodes/episode-126-denise-gosnell-datastax-how-many-database-joins-are-too-many)

图形数据库系统不同于标准的关系(SQL)系统，因为它们被[设计成更容易捕捉不同实体之间的关系。“当你查看数据库时，图形数据库允许你通过使用关系更有效地对数据建模，”戈斯内尔说。](/illuminating-the-anonymous-with-neo4js-graph-database/)

您可以通过一系列独立表的数据库连接来捕获关系信息，但是最终，这种方法的复杂性会使其望而却步。“当你看到在应用程序中使用它或维护代码或更新边缘的完整端到端复杂性时，图形数据库将使应用程序的整个生命周期和维护变得更加容易，”她说。

在我们的采访中，我们还“询问”了戈斯内尔:

*   图形技术的定义。
*   图和机器学习的交集。
*   与数据湖或数据仓库相比，使用图形系统的优势。
*   关于图形查询模型的更多信息。
*   用例。
*   使用 GraphQL 的图形数据库的前景。
