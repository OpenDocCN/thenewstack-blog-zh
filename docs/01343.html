40多岁的人要求音乐推荐，Redditors指出算法

原文：https://thenewstack.io/40-something-dude-asks-for-music-recommendation-redditors-point-to-the-algorithms/#0001-01-01

克里斯·丹西在推特上指出了这一点。

下面是人们对所说的的一个例子:

我要说的是，你最大的缺失不是某个艺术家或流派，而是你发现好音乐的媒介。我不知道你的品味是什么，也不了解你，我不能推荐任何艺术家，但我能做的是向pandora和spotify推荐应用程序。在过去的十年里，对音乐产业影响最大的不是一些崭露头角的艺术家，而是我们发现他们的方式。音乐比以往任何时候都更容易获得，它正等着你去发现你喜欢的东西。

几乎所有的回复都带有一些“服务推荐”，这能告诉你一些东西。这些服务都使用复杂的算法来推荐音乐，你知道吗？他们工作。

音乐服务调整了他们的算法，所以我们甚至不再费心推荐音乐了。

阅读Spotify实验室的博客，这一点显而易见。大约六个月前，Spotify发布了它的 web API 。它现在有40个端点。从现在起一年后检查，我的猜测是会有两个或更多的端点被ping通。

这篇文章是写给想用Spotify服务的数据开发一个应用的开发者的。API定义了这些服务。它们被利用得越多，服务就能分析得越多，从而提出更好的建议。随着越来越多的开发人员进入这个世界，预计会有更复杂的系统来发现人类用他们单一的人脑无法完成的事情。

此外，更少的人工建议需要以应用程序为中心的基础架构，而越来越多的基础架构最好留给机器来管理。只要看一下 Soundcloud的普罗米修斯服务，你就能了解情况了。

对此该怎么说？人类仍在这里，但选择将音乐推荐留给算法。我们只是为服务提供数据的节点。

不过，这并不完全正确。所有被认为是歌曲是我最喜欢的音乐播客之一。他们加入了美妙的音乐元素，但我想知道他们的选择是如何受到算法的影响的，比如我们这位40多岁的老兄用来发现他真正喜欢的音乐的算法。

丹西，比任何人都更有联系，有更深的视角。用API和传感器连接自己让他的生活变得更好。他控制着它。这是他的一次演讲，很好地为人类和机器智能能力带来的复杂性提供了背景。
