# 对于开源来说，软件基金会是新的教堂

> 原文：<https://thenewstack.io/for-open-source-software-foundations-are-the-new-cathedrals/>

Red Hat [OpenShift](https://commons.openshift.org/) 的社区开发总监 Diane Mueller ，已经在开源社区工作了 20 年，亲身经历了开源软件开发的兴起。有了这些丰富的经验，我们对如何改进开源社区的合作方式有了独特的看法。这包括内部讨论和外部交流。像 RedHat [OpenShift Commons](https://commons.openshift.org/) 这样的倡议也有助于为开源开发者创造更好的工作环境，因为他们的社区和项目在不断发展。

在这一集的[新堆栈制作者](/tag/the-new-stack-makers/)中，新堆栈的创始人 Alex Williams 与 Mueller 谈论了她在 Red Hat open shift(Red Hat 的开源 PaaS 平台)中的角色。Mueller 讨论了目前在 OpenShift Commons 上合作的越来越多的个人和组织，OSS 的未来可能会是什么样子，以及基金会、组织和开发者如何在欢迎其他人的同时加强他们的社区。

[黛安·穆勒带着红帽在奥斯丁的 OSCON 接受采访](https://thenewstack.simplecast.com/episodes/diane-mueller-with-red-hat-interview-at-oscon-in-austin)

对话[也可以在 YouTube 上听到](https://youtu.be/UoPAWGs3zhU)。

“在试图构建 OpenShift Commons 时，我们创造了一个新的隐喻，人们认为这是受 Eric Raymond 的书*[The Cathedral and The Bazaar](http://www.catb.org/esr/writings/cathedral-bazaar/cathedral-bazaar/index.html#catbmain)启发的开源社区模型。*在 Raymond 看来，大教堂是单个实体如何创建专有软件的模型，而集市是开源软件如何集体开发的模型。Mueller 解释说，基金会正在成为新的大教堂，这是开源软件持续商业化带来的一种模式。

Muller 指出，Raymond 在 1999 年称之为集市的东西现在更为人所知，它给 OSS 世界带来了挑战。“每个云和供应商都在这些市场中，不再是个人。在一个基金会驱动的世界里，个人没有发言权。话虽如此，基金会在孵化项目、提供治理和指导方面非常出色，基金会也有自己的角色。”

然而，在开源社区中创建新软件并非没有障碍。这就是 Red Hat OpenShift 的用武之地。众所周知，开源社区被认为有不必要的高门槛，而如果新贡献者选择的项目是他们第一次涉足编程，他们可能会遇到敌意。

要求看门人和基金会首脑在参与社区讨论时拥有最终发言权是 Mueller 开始改进的事情，他改变了 OpenShift 处理开源社区模型的方式，同时详细说明了其他开源社区可以做些什么来改善他们自己的，“让每个人都在一个公平的竞争环境中开始。这意味着没有基础费用。在 OpenShift Commons 中，有一个供每个人加入的共享竞技场，能够无障碍地与来自生态系统另一方面的人进行交流和交谈，”穆勒指出。
