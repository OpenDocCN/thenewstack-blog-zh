# 云原生 Live: Twistlock 的虚拟会议

> 原文：<https://thenewstack.io/cloud-native-live-twistlocks-virtual-conference/>

[云原生直播:扭锁的虚拟大会](https://thenewstack.simplecast.com/episodes/cloud-native-live-twistlocks-virtual-conference)

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，这是一个每周一次的播客，我们在这里讨论可扩展 IT 社区中最热门的新闻和观点。

在本周的节目中，我们采访了云原生安全提供商 [Twistlock](https://www.paloaltonetworks.com/prisma/cloud) 的宣传主管 [Sonya Koptyev](https://www.linkedin.com/in/sonyakoptyev/) ，该提供商将于 1 月 29 日周二举办一场名为 Cloud Native Live 的网络研讨会/会议。在这场为期一天的活动中，有许多优秀的演讲者。在主要主题演讲中，Pivotal 的 [Dormain Drewitz](https://twitter.com/DormainDrewitz) 将分享成功采用数字化转型最佳原则的企业的秘密。CI/CD 专家 [Brice Fernandes](https://twitter.com/fractallambda?lang=en) 将解释 [GitOps](https://thenewstack.io/gitops-git-push-all-the-things/) 模型，确定用于这一新兴实践的最佳实践和工具。其他发言人包括 GitLab 云原生联盟主任 Priyanka Sharma 和云原生计算基金会执行主任 Dan Kohn。

除了虚拟会议，我们还谈到了 DevSecOps，这是[本周在 Twistlock 的新堆栈上发表的一篇博文](https://thenewstack.io/from-agile-to-devsecops/)的主题。

TNS 创始人亚历克斯·威廉姆斯主持了这次播客，并得到了 TNS 执行主编乔布·杰克逊的协助

## 本周讨论的故事

*   [美国国防部如何检测“敏捷 BS”](https://thenewstack.io/the-u-s-department-of-defense-on-how-to-detect-agile-bs/):敏捷是最常见的开发方法之一，也是最常被误用的方法之一。因此，毫不奇怪，在线程序员对美国国防部(DoD)发布的一份信息丰富的新文档反应强烈，该文档为经理们提供了如何微调项目的建议。国防部的“检测敏捷 BS”指出了一个项目没有真正使用敏捷开发方法的六个迹象——例如，缺乏协作，甚至最终用户“在整个开发过程中没有行动”
*   [从敏捷到开发团队](https://thenewstack.io/from-agile-to-devsecops/):当敏捷软件交付方法在 21 世纪初出现时，它是创新的。但那是近二十年前的事了。如果您的组织今天仍然主要依赖于敏捷方法，那么它就错过了重要的效率，并且应该专注于升级到 DevOps 实践。
*   [如何在保持运营的同时重写您的基岩应用](https://thenewstack.io/how-to-rewrite-your-bedrock-application-while-remaining-operational/):2016 年 1 月，弹性计费服务提供商 Chargify 决定进行代码重写。该应用程序已有六年历史。该公司的 IT 主管认为他们已经学到了足够多的东西，并且受到了现有架构的足够多的限制，因此通过重新开始，他们实际上会走在前面。这是他们学到的教训。
*   [数据库运营商为 Kubernetes 带来有状态的工作负载](https://thenewstack.io/databases-operators-bring-stateful-workloads-to-kubernetes/):去年，Red Hat 正式推出了 Operator Framework，这是一种为特定应用定制 Kubernetes 的 API 的方式。现在我们开始看到更多的独立软件厂商(ISV)使用 Operator 框架来包装自己的应用。特别是，数据库管理系统提供商正在使用该技术来解决与 Kubernetes 合作的最困难的问题之一，即管理有状态的应用程序。
*   [Kubernetes 和虚拟机的回归](/kubernetes-and-the-return-of-the-virtual-machines/):在本播客中，我们将与 Pivotal 首席技术专家 Paul Czarkowski 和 Red Hat 云平台副总裁 Joe Fernandes 讨论，进一步了解在 Kubernetes 环境中使用虚拟机的吸引力。

云计算原生计算基金会、Pivotal、Red Hat 和 Twistlock 是新堆栈的赞助商。
