# Docker 让开发者可以轻松构建和部署应用

> 原文：<https://thenewstack.io/docker-makes-it-easy-for-a-developer-to-build-and-deploy-apps/>

[Docker 让开发者轻松构建和部署应用](https://thenewstack.simplecast.com/episodes/docker-makes-it-easy-for-a-developer-to-build-and-deploy-apps)

最近， [Docker](https://www.docker.com/) 发布了 Mac 版[Docker](https://docs.docker.com/docker-for-mac/install/)和 Windows 版[Docker](https://www.docker.com/docker-windows)的新版本，开发者可以使用模板创建一个应用，并在一分钟内运行。现在，构建和部署应用程序就像创建 Microsoft Office 文档一样简单。

这并不奇怪。Docker 对 IT 世界最大的贡献之一就是让容器变得非常容易使用。由于桌面上的 Docker，开发人员可以在他们的本地机器上工作，构建应用程序并在生产中运行它，没有任何复杂性。

在本期[新堆栈制造商](https://thenewstack.io/podcasts/makers)播客中，我们与 Docker Inc .的产品经理 [Gareth Rushgrove](https://www.linkedin.com/in/garethrushgrove) 坐下来谈论这款工具。Rushgrove 表示，他的重点更多的是开发者桌面，而不是服务器和云。“我关心个人开发人员的笔记本电脑和台式机。Rushgrove 说:“我们致力于为在本地使用 Docker 创造最佳的开发者体验。

Docker 投入了时间和资源来创建与运行在开发人员机器上的操作系统的无缝集成。

“我们做了大量工作来集成本机虚拟机管理程序、本机网络堆栈和本地文件系统。这些不是你在服务器领域看到的问题，但对开发人员的体验至关重要，”Rushgrove 补充道。

由于 macOS 和 Windows 不附带 Linux 内核，Docker 创建了 Linux 工具包来帮助 Docker 团队构建这些平台。去年，Docker 开源了这项工作，因此任何人都可以继续为他们需要的平台添加支持。

在 [Linux 套件](https://github.com/linuxkit)之前，用户不得不求助于虚拟机，这可能是非常资源密集型的，并且它们不提供人们期望的深度集成。有利于 Rushgrove 的是，他曾作为运营团队的一部分工作过，所以他知道痛点，并带来这种经验来进一步改善开发人员的体验。
