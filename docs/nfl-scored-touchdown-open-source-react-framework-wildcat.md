# NFL 如何通过 React 和 Node.js 获得用户界面触地得分

> 原文：<https://thenewstack.io/nfl-scored-touchdown-open-source-react-framework-wildcat/>

[国家足球联盟](http://www.nfl.com/)是当今使用 [Node.js](https://nodejs.org/en/) 服务器端 JavaScript 运行时的最大组织之一。在最新一集的[新堆栈制造商](https://thenewstack.io/podcasts/Makers)播客中，我们采访了 NFL 软件工程师[James Hush](https://github.com/jamsea)React 和 Node.js 如何塑造了公司，以及 NFL 的开源“固执己见”React 平台 [Wildcat](https://github.com/nfl/react-wildcat) 。采访在 Node.js Interactive 2016 举行，由 TNS 创始人亚历克斯·威廉姆斯和 TNS 撰稿人[达里尔·塔夫特](https://www.linkedin.com/in/darryltaft)主持。

NFL 工程团队对 React 的第一次尝试是为 NFL PlayStation 4 应用程序创建一个概念证明。Hush 指出，在意识到完成这一过程是多么简单之后，该团队“在过去两年里全力以赴，在 React 上下注，并且获得了巨大的回报。”

Hush 说:“React 最大的特点之一就是能够在服务器上渲染。“这就是我们使用 Node.js 的原因，因为我们可以在服务器上渲染 react，然后将已经预渲染的所有内容发送给 web 浏览器。”

当被问及该公司今天面对 [Angular 2](https://thenewstack.io/google-preps-angular-2-final-release/) 等技术时是否还会选择 React 时，Hush 毫不犹豫地回答 NFL 的选择还是会是 React。该公司确实使用 Angular.js 框架[量角器](http://www.protractortest.org/#/)进行端到端测试，“我们关注的一件大事是自动化测试，特别是使用量角器，它允许你用 JavaScript 编写 selenium 测试。我们有一系列自动化的用户界面测试，检查每一个页面和按钮，甚至当你调整一个页面到手机。“每次我们改变哪怕一行代码，我们都会在 20 台不同的机器上并行运行这些测试，”Hush 说，并补充说，“今年无论如何，我们在游戏期间都没有在网站上出现过一个致命问题。”
