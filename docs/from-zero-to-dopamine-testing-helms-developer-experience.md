# “从零到多巴胺”:测试 Helm 的开发者体验

> 原文：<https://thenewstack.io/from-zero-to-dopamine-testing-helms-developer-experience/>

[KubeCon+CloudNativeCon](https://www.cncf.io/kubecon-cloudnativecon-events/) 赞助了这个播客。

[微软高级软件工程师 Michelle Noorali](https://www.linkedin.com/in/michelle-noorali/) ，在[掌舵](https://helm.sh/)的最初几天，她与 [Matt Butcher](https://www.linkedin.com/in/mattbutcher) 在 Deis 工作时，在白板上写了一份声明，该公司于 2017 年被微软收购。“五分钟内从零到多巴胺”仍然是微软首席软件开发工程师 Butcher 和他的团队用来衡量他们如何为用于启动 Kubernetes 的流行软件包管理器构建开发人员体验的短语。

在本期的[播客中，新的 Stack Makers](/podcasts/makers) podcast、Butcher 和三星的高级员工工程师 Matt Farina 讨论了 Helm 的更新如何帮助改善整体 Kubernetes 体验，并在这样一个大型社区中平衡可用性，以提供最佳的开发人员体验。《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了这一集。

[【从零到多巴胺】:测试头盔开发者体验](https://thenewstack.simplecast.com/episodes/from-zero-to-dopamine-testing-helms-developer-experience)

从很多方面来说，Helm 发展背后的理念自创建以来并没有太大变化。事实上，Butcher 认为 Norrali 的“五分钟内从零到多巴胺”的陈述一直是指导原则。

“在项目早期，我们对三类用户进行了优先排序，并意识到我们必须在哪些方面做出选择，这些选择会影响我们为谁提供最佳服务。“从第一天开始，‘零到首次安装’就是我们最重要的故事，”Butcher 解释道。“接下来，‘日常包管理’是第二大亮点。”

然而，Helm 可能并不总是能够适应 Butcher 所描述的新兴的“前沿”用例，即“启用新的使用模式”

“我们意识到，如果我们追求第三类东西，那么很大程度上会对前两类产生负面影响，”布彻说，“所以，我们真的试图说出我们想要什么。”

Butcher 说，当一个组织刚刚开始采用 Kubernetes 时，他的团队“爱听”发生的故事。

“在这些情况下，我们希望听到一个用户说，‘我刚开始使用 Kubernetes，我第一次安装了 WordPress，并让它运行起来，现在我可以看看我安装了什么，并从那里了解 Kubernetes 是如何工作的，’”Butcher 说。“我们喜欢听这个故事，它每次都会让我们开心。我们还可以从与社区的互动中看到的使用模式，从每月 200 万的下载量中看出，我们真的抓住了第二个故事，这很好，因为这就是 Kubernetes 目前的真实情况。”

这样，第二个 Helm 的日常操作使用故事就覆盖了 80%的用例。这涉及 Helm 如何提供操作员管理集群所需的核心工具集。“所以我们很高兴知道我们真的抓住了这个组织的大部分，”布彻说。

如今，用户在采用 Helm 时面临的一些挑战包括平台的使用模式和可用选项的复杂性如何增加。

“现在，有太多的东西，要全部学会很复杂。我认为，在早期，赫尔姆帮助人们用一种方式解决问题，”法里纳说。“现在，Helm 允许人们创建图表，对其进行改进，并添加许多内容，这样，安装图表并在群集中使用它的人就不必了解所有这些内容，仍然可以获得好处。”

[云原生计算基金会](https://www.cncf.io/)也为 Helm 的采用和发展提供了一个坚实而安全的框架。与此同时，Helm 和 Flux GitOps 运营商是 CNCF 终端用户社区向[最推荐的云原生操作 CD 工具](/flux-helm-top-cloud-native-user-recommendations-for-continuous-delivery-technologies/)。因此，CNCF 在该项目的成功中发挥了作用。

“我向 CNCF 致敬，尤其是那些每天为项目工作的员工。所以，如果你在消费一个项目，特别是如果你是一个企业，你想知道一些东西是稳定的，它是坚实的，你可以信任它，”法里纳说。“因此，CNCF 一直在努力将需求落实到位，并确保项目符合需求，这样你就知道它们在哪里，然后你就可以将它与你的信任级别联系起来。”
