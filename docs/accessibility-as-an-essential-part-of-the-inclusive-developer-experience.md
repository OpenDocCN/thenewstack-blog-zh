# 可访问性是包容性开发者体验的重要组成部分

> 原文：<https://thenewstack.io/accessibility-as-an-essential-part-of-the-inclusive-developer-experience/>

[将可访问性作为包容性开发者体验的重要组成部分](https://thenewstack.simplecast.com/episodes/accessibility-as-an-essential-part-of-the-inclusive-developer-experience)

[![](img/36e756062123566514537aa0901725fd.png)](https://twitter.com/raqueldesigns/) 我们已经写了一些关于[构建应用程序和网站的可访问性](https://thenewstack.io/html-css-and-the-path-to-accessible-web-design/)——尽管总是有更多的东西要学习和做——但是在开发人员的世界里，可访问性和包容性又如何呢？当您的客户是开发人员时，您如何最好地服务于您的全部客户群？包容性设计如何成为开发者体验的重要部分？这些是 Pivotal Labs 的高级设计师 Raquel Breternitz 在本期新堆栈制造商播客中试图回答的问题。

为什么包容性对 Breternitz 如此重要？

“我们不仅仅要确保有人能够到达那里，并在他们到达时能够理解，但我们也必须找到人们被排斥的方式，并打破这些障碍，”她在后续对话中告诉新的堆栈。

Breternitz 认为，以用户为中心的设计必须推动任何数字化转型。引用一句古老的建筑智慧，“你可以在制图桌上用橡皮擦，或者在建筑工地上用大锤。”

她继续说，设计师经常追求某种外观，也许是一个更干净的设计，然后过度打乱设计，使其不容易理解，因此对每个人都不太有用。这在所谓的干净设计实践中很常见，用户需要将鼠标悬停在上面才能查看信息，但对于屏幕阅读器来说这是不可读的，并且通常所有用户都完全忽略了这些信息。此外，对设计师来说，这更具挑战性，也更费时，这使得为其设计的公司成本更高。Breternitz 还提到了“美丽”的苍白设计的趋势，这对每个人来说都很难阅读。

对她来说，可及性是描述性的和有意义的。尽可能多的人能够使用你的网站或工具。这意味着设计师的工作是清晰地交流，传达清晰的概念能力。

这也贯穿到文档中。Breternitz 说，许多文件作者想用巨大的图表来解释重要的概念。这对于屏幕阅读器来说通常是不容易理解的，那些不是视觉学习者的人将会被遗忘。每个设计触及用户或开发者体验的东西的人都应该强迫自己叙述他们正在创造的东西。你能有多少种解释？然后把它写下来，包括在文档、笔记或替代文本中。请记住，可访问性和包容性是为了确保不只是专家在使用您的工具，初学者最有可能从您的文档中受益，所以不要想当然地接受任何解释，尤其是入职时。
