# 后 DevOps 时代的即时数据归结为状态

> 原文：<https://thenewstack.io/instant-data-in-a-post-devops-world-comes-down-to-state/>

[VMware](https://www.vmware.com/company.html) 赞助了这次播客。

在可预见的未来，有状态数据管理和存储将始终存在于 IT 基础架构中。但是随着容器和 Kubernetes 的出现，无状态和多平台环境的存储管理将会改变。无论您的组织是否雇佣了分布在世界各地的数百名开发人员，他们经常开发无状态容器，或者您的组织是否有分布在多云和本地遗留环境中的数据，都是如此。在任何情况下，所有这些数据都必须以有状态的形式存储在某个地方。

说到底，“状态”真正的意思是“数据比容器更持久”， [Portworx](https://portworx.com/) 的首席执行官兼联合创始人 [Murli Thirumale](https://www.linkedin.com/in/murlithirumale) 说。然而，在这个无状态数据的时代，一种新的存储方式出现了。

在本期的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，Thirumale 描述了存储管理的新时代，以及 Portworx 作为容器和多平台环境的有状态存储提供商在这一背景下所扮演的角色，该播客于 9 月在旧金山的 [TC Sessions: Enterprise](https://tcrn.ch/2Y3wJGI) 中录制。

[后 DevOps 世界的即时数据归结为状态](https://thenewstack.simplecast.com/episodes/instant-data-in-a-post-devops-world-comes-down-to-state)

与今天相比，存储在过去无疑是乏味的，因为现在有如此多的组织需要存储和分析来自多种多样且日益容器化的来源的数据。

“数据管理实际上是管理跨多个应用程序和节点、跨多个用户以及跨这些应用程序的多个实例的数据。数据管理是容器带来的爆炸式增长的新领域，”Thirumale 说。"容器不仅仅是驱动因素之一，你现在有了一个分布式的世界."

因此，今天的 IT 环境通常是非常多云和多环境的，“人们同时在多个地理位置运行相同的应用程序，你会期望它将在全球范围内推广，”Thirumale 说。

Thirumale 说，一个“后 DevOps 世界已经出现”，在这个世界中，数据可以从世界各地的应用程序中即时获得。“管理所有这些成为一个特殊的挑战，将超过仅仅存储数据，”Thirumale 说。“因此，通过像 Portworx 这样的解决方案，我们的产品可以精确地为容器化部署提供所有这些功能。”

例如，Portworx 提供的存储解决方案满足了采用 Kubernetes 环境的组织的修改需求。Thirumale 说，“社区已经创建了扩展社区，以解决 Kubernetes 没有保证的所有问题，”如驱动程序量。“Portworx 是与 Kubernetes CSI 接口一起工作的产品之一，它提供了使有状态应用程序与 Kubernetes 一起工作的所有功能。”

Portworx 是新堆栈的赞助商。
