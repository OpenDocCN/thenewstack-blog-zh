# 阴影带来的问题是:它不会留下阴影

> 原文：<https://thenewstack.io/tns-analysts-show-99-evaluating-cloud-management-platforms-chefconf/>

对 [IT 管理](https://thenewstack.io/tns-research-devops-factions-software-development-world/)的“旧方式”的不满会给企业带来灾难性的后果。“人们不会掏出信用卡去 AWS，因为他们想去。这是因为他们对其中的“标准程序”感到失望。如果它给了他们同样的能力，用户会很高兴采用它，”CloudBolt 首席执行官， [Jon Mittelhauser](https://www.linkedin.com/in/jonmittelhauser) 。

在这一集的[中，下面嵌入的新堆栈分析师](https://thenewstack.io/podcasts/)将探讨云计算驱动的“影子 IT”问题，以及 [CloudBolt](https://www.cloudbolt.io/) 希望如何通过其为最终用户带来管理员级别的功能和控制的方法来改变这一问题，从而让 IT 管理员能够将注意力集中在重要的地方。我们采访了 Mittelhauser，以及希捷建筑师和工程经理 [Yogi Porla](https://www.linkedin.com/in/yogiporla) 和博思艾伦汉密尔顿首席技术专家 [Nirmal Mehta](https://www.linkedin.com/in/nirmalkmehta) 。该节目由 TNS 技术顾问 [Lee Calcote](https://twitter.com/lcalcote) 和 TNS 执行主编 [Joab Jackson](https://www.twitter.com/Joab_Jackson) 主持。

[#99:阴影的问题——它没有留下阴影](https://thenewstack.simplecast.com/episodes/99-the-problem-with-shadow-it-it-leaves-no-shadow)

Mehta 强调了这样一个例子，一位首席技术官面对一个业务团队，该业务团队使用自己的信用卡购买 AWS 工具、GitHub 帐户和其他他们需要的工具，因为他们的组织的 it 团队通过安全措施阻止他们访问这些工具，这种做法通常被称为“影子 IT”。因此，梅塔指出,“影子实际上并没有投射出影子，这就是问题所在。该公司的内部 IT 团队找到我们说，“我们现在有一个更糟糕的问题，因为我们知道人们正在这样做，但我们对它没有任何可见性，”如果你建立一个大坝，水将四处流动。”

Mittelhauser 提供了 CloudBolt 的简要概述，指出其核心原则之一是最终将管理员的权力交给用户，而不是 IT 管理员。

![containercon_use](img/87828b392aaebc0d708c601f07dfcde2.png)

“归根结底，最终用户自助配置才是最重要的。通常在企业中，如果您需要一台新服务器，您必须等待两个星期，等待有人来提供它，有人来设置网络，并设置磁盘。通过自动化和使用云管理，这可以得到简化。我们为最终用户提供了一个简单的对话框，他们可以在其中管理自己的功能。因此，管理员花时间在他们应该做的事情上，”Mittelhauser 说。
