# 微软的 Mark Russinovich 对微服务的定义

> 原文：<https://thenewstack.io/microsofts-mark-russinovich-on-defining-microservices/>

[云原生世界中的微服务](https://thenewstack.simplecast.com/episodes/microservices-in-the-cloud-native-world)

你有没有尝试过给别人解释微服务？根据上下文的不同，人们会有不同的解释。但是当谈到建立微服务和云原生计算之间的关系时，没有人比微软 Azure 的首席技术官马克·鲁斯诺维奇在最新一期的播客中解释得更好了。

根据 Russinovich 的说法，了解微服务基础设施的最佳方式是查看客户端-服务器时代的上一代架构。巨石时代。通常有三层:应用程序(前端)；中间层(业务逻辑)和后端的第三层，比如数据库服务器。

这种模式让一切都慢了下来。你不能创新，因为所有三个层次都需要彼此同步。通过拥抱云来实现敏捷意味着按需创建计算资源。它打破了旧的模式。您最终会发现应用程序架构的交付模型和底层平台所能提供的不匹配。

微服务作为一种将整块石头分解成组成部分的方式出现了。现在，不同的开发人员或开发团队可以处理这些组件，并独立于正在处理大型应用程序的其他微服务的其他团队发布它们。

突然之间，组织的应用程序架构中的敏捷性级别可以与底层基础架构提供的敏捷性优势相匹配。

“事实是，在许多复杂的应用中，如果你不这样做，那么你就不会有竞争力，”Russinovich 说。“因此，微服务方法受到了两方面的推动，一是试图利用基础架构，二是保持竞争力。”

然而，他也明确表示，微服务不是银弹，它不是所有情况下的答案。但是在有多个团队的复杂项目中，以及不同部分以不同的速度发展和扩展时，它肯定是非常有效的。
