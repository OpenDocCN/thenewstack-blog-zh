# 评估无服务器安全的当前状态

> 原文：<https://thenewstack.io/assessing-state-serverless-security/>

安全性几乎总是用户提到的一个主要痛点，他们被调查的环境或技术似乎真的无关紧要。每当我们开始关注新技术或新生态系统时，安全性总是我们最关心的事情之一。但是当谈到[无服务器技术](/category/serverless/)时，从安全的角度来看，它真的与围绕微服务和云提供商的经验教训有什么不同吗？无服务器社区对少数集中式工具的关注实际上改善了早期安全性的前景吗，或者它只是加剧了那些问题？

在本期[新堆栈分析师](https://thenewstack.io/podcasts/analysts)播客中，我们与作家/分析师[马克·博伊德](https://www.linkedin.com/in/mark-boyd-28386936/?ppe=1)，安全公司 [Snyk 的](https://snyk.io/)首席执行官[盖伊·波德贾尼](https://www.linkedin.com/in/guypo/)，以及[保罗·约翰斯顿](https://www.linkedin.com/in/padajo)，环岛实验室[的创始人](http://roundaboutlabs.com/)谈论了无服务器安全的现状。在我们的讨论中，我们谈到了用于保护无服务器环境的工具，以及用于提供固有的、一致的安全方法的方法。

专家组一致认为，有许多传统的安全问题仍然存在，特别是在云提供商的最佳实践方面，但在无服务器允许团队进行试验的速度方面，仍然存在新的安全威胁。我们讨论了无服务器如何将安全问题从基础设施级别转移到高级目标，如应用程序安全、数据安全和网络安全。最后，我们还讨论了随着无服务器应用的普及，无服务器安全的状态将会如何变化。
