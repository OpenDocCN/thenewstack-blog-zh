# NGINX，一个用于微服务架构的开源 Web 服务器

> 原文：<https://thenewstack.io/exploring-bright-future-of-nginx-plus/>

快速增长的 web 服务器之一，[尤其是高容量站点](https://news.netcraft.com/archives/2016/09/19/september-2016-web-server-survey.html)， [NGINX](https://www.nginx.com/) 可能是微服务架构的天然选择。

在 OpenStack Barcelona 录制的这一集 [The New Stack Makers](https://thenewstack.io/podcasts/) 播客中，NGINX 首席执行官 [Gus Robertson](https://www.linkedin.com/in/gusrobertson) 、NGINX 产品负责人 [Owen Garrett](https://www.linkedin.com/in/owengarrett) 和 NGINX 首席营销官 [Peter Guagenti](https://www.linkedin.com/in/peterguagenti) [讨论了](https://youtu.be/3ZM2HyrQPn4)该公司最新发布的商用 [NGINX Plus R11](https://www.nginx.com/blog/nginx-plus-r11-released/) ，为什么 NGINX 对微服务如此有利，以及 NGINX 在 OpenStack 会议上首先做了什么

[NGINX，微服务开源 Web 服务器](https://thenewstack.simplecast.com/episodes/nginx-an-open-source-web-server-for-microservices)

T21 R11 版本的一个新特性是支持第三方模块的能力。“有一个模块开发人员和专家社区，他们为我们的开源产品创建扩展、代码模块和代码插件，我们的许多用户依赖这些扩展来实现增值功能。我们希望确保我们的商业客户也可以使用这些第三方扩展，”Garrett 说。

Garrett 指出，使用 C 语言，用户可以编写扩展，应用于 NGINX API，发布和分发他们的扩展，只要最适合他们。

当将开源扩展引入运行 NGINX+的商业基础设施时，用户可能会担心第三方扩展的质量。为了解决这个问题，NGINX 使用了一个排名系统，并运行自己的测试来确保所有提交的扩展都符合它的标准。

虽然我们认为 NGINX 是位于 Web 应用程序前面的 Web 服务器，但该软件实际上也可以在连接服务方面发挥关键作用。

“随着微服务的兴起，我们的角色也越来越重要，”Guagenti 解释道。以[网飞](https://thenewstack.io/netflix-uses-node-js-power-user-interface/)为例，它是一个成功的微服务架构的模型。NGINX 软件在这一操作中起到了关键作用。

“在过去的 12-6 个月里，大多数人都在说，‘我正在了解微服务’，没有一个客户真正使用微服务架构投入生产。在过去的 6 个月里，这种情况发生了变化。
