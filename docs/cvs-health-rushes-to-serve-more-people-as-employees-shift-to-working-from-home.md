# 随着员工转向在家工作，CVS Health 急于为更多人提供服务

> 原文：<https://thenewstack.io/cvs-health-rushes-to-serve-more-people-as-employees-shift-to-working-from-home/>

作为美国最大的连锁药店和 Aetna health insurance 的所有者， [CVS Health](https://cvshealth.com/) 已经刻不容缓地准备全球新冠肺炎疫情。该组织不得不将其为期两年的 Office 365 团队部署缩短到不到三个月，以便为该组织现在在家工作的 40，000 名企业员工中的 98%做好准备(此外还有在商店工作的 60，000 名一线员工)。

除了这一挑战之外，还有公司的飞速发展——CVS 在过去十年中收购了 10 家大公司，并且在不到一个月的时间里,[提供了 50，000 个新工作岗位。](https://www.bloomberg.com/news/articles/2020-03-23/cvs-health-to-hire-50-000-in-response-to-coronavirus-demand)

“幸运的是，我们已经做好了准备，”首席技术官办公室的 CVS Health 的 API 集成和数字生态系统负责人梅丽莎·珀森-阿什福斯(Melissa Person-Ashforth )通过变焦电话说道。

Person-Ashforth 最重要的职责是管理整个 CVS-Health 平台的企业 API 集成和生态系统。她管理着由架构师、开发人员、质量保证、生产和支持组成的团队，所有人都对其数据聚合服务的正常运行时间负责。

Person-Ashforth 说，该公司非常重视维护服务水平协议(SLA)的目标，毫秒内的响应速度和每秒高达一百万的 TPS 供应商协议。

她说，我们有过延迟，我和我的团队正在向首席信息官推销升级，她一直在寻找机会进行改进和现代化。其中一个策略是向混合云迈出更大的步伐。目前。CVS 有 25%是基于云的，它正试图快速过渡到至少一半的业务。

Person-Ashforth 表示，CVS 处于持续的业务连续性模式，因为在危机期间不可持续根本不是一个选项。即使在危机模式下，公司也必须有一个备份。

当美国电话电报公司的一条线路出现故障，影响了 CVS 的思科 Webex 部署时，会议从视频会议转移到了他们的公司手机上。

这项工作极具挑战性，因为所有客户数据都要遵守针对个人健康信息(PHI)、个人身份信息(PII)、健康保险便携性和责任法案(HIPAA)、欧盟通用数据保护法规(GDPR)和加州消费者隐私法案(CCPA)的严格法规和强制性合规培训。这些法规、合规性和安全性要求也适用于他们为之提供 API 的第三方。

## 远程办公

她说“我们是一家医疗保健公司，我们必须积极主动”，这就是为什么 CVS Health 在近两个月前通过转换到 remote-first 树立了榜样，远远早于许多美国公司。

她说，这是一个有孩子在家工作的时代，弹性工作制和轮班制是必不可少的，并且相信人们会在预算和规格范围内完成项目。

“虽然不是，但我觉得一切如常。我们已经配备了笔记本电脑，能够在另一个地点工作。我的商务旅行从 90%变成了 0%，”Person-Ashforth 说。

当她意识到自己在家工作的时间比平时长——在正常上下班的时间工作——她开始设定一些界限。尤其是现在，他们可能有西海岸的客户需要他们工作到很晚，但她鼓励她的团队不要开会，不要从早上 7 点工作到晚上 10 点
