# Ben Golub:从存储到 Docker，再回到存储，但现在是区块链

> 原文：<https://thenewstack.io/ben-golub-from-storage-to-docker-and-back-to-storage-but-now-with-blockchain/>

[Ben Golub:从存储到 Docker 再回到存储但现在有了区块链](https://thenewstack.simplecast.com/episodes/ben-golub-from-storage-to-docker-and-back-to-storage-but-now-with-blockchain)

当我在波特兰的 Doug Fir Lounge 会见 Ben Golub 和 Solomon Hykes 时，已经是 2013 年了，OSCON 一片繁忙。Golub 找到了一个家，由 Docker 担任首席执行官，此前他成功地领导了 Gluster，这家存储公司被卖给了 Red Hat。

如今，Golub 现在是 Storj and Hykes 的临时首席执行官，几周前刚刚离开 Docker。他们的离开是另一天的讨论。但他们率先打开了市场，自 2013 年以来，我们确实知道容器已被广泛采用。现在的问题是关于面向应用程序的架构以及它们以一种不断分布的方式运行的资源。这个话题超越了关于容器的讨论，进入了面向应用程序基础设施和数据架构对构建新的预测建模系统的影响的话题。这些是事件驱动的平台，运行在为工作负载配置的资源上，主要是在 Kubernetes 或 Kubernetes 风格的架构上运行的容器。在许多方面，五年来最大的转变是对机器本身的抽象，这是我和 Golub、Hykes 在 Doug Fir 吃午饭时即将发生的。

Storj 反映了从机器中提取数据的概念思维的转变。它解决了对未使用资源的需求。网络分布越来越广，但人们设备上未使用的资源的利用大多尚未开发。Stor 向开发人员和服务提供商提供可从设备中获取的资源。Golub 会说，区别在于区块链如何集成到平台中。

“这是磁盘驱动器的 Airbnb，”Golub 说。

作为一家新创业公司的首席执行官，与 Golub 交谈很有趣。在 OSCON 夏季秀上，我们见到了本，有种能量。也许是温暖的阳光，在波特兰骑着自行车和镇上这么多的极客在一起。但是还有别的东西。随着 API 的出现，应用程序架构的思想发生了转变，出现了新的平台来推动云资源的使用，尤其是 Amazon Web Services。关于这一切有一个难题——资源如何适应具有弹性和可伸缩性的应用程序架构？拼图的碎片显示出图案——有线开始发展。但这仍然不容易。在云资源上搭建灯栈？不用了，谢谢——有些东西可能会与众不同，开发者会采用的。他们拥有所有这些可用的资源，如何更方便地使用它们呢？

现在我们知道从那时起事情发生了怎样的变化。但是看看我们现在。资源总是被分配的。Golub 又回到了存储领域。
