# Kubernetes 认证如何确保互操作性

> 原文：<https://thenewstack.io/how-kubernetes-certification-works-to-ensure-interoperability/>

[Kubernetes 认证如何确保互操作性](https://thenewstack.simplecast.com/episodes/how-kubernetes-certification-works-to-ensure-interoperability)

[云本地计算基金会](https://www.linkedin.com/in/dankohn/)的执行董事 Dan Kohn 称，新的 [Kubernetes 服务提供商认证计划](https://www.cncf.io/certification/kcsp/)的推出是该基金会围绕[开源容器编排引擎](/category/kubernetes/)做出的最重要的宣布。

在 KubeCon + CloudNativeCon 2017 新一集的新堆栈制造商中，我们将从 Kohn 和谷歌产品经理[William Dennis](https://www.linkedin.com/in/williamdenniss/)那里了解更多关于该计划如何帮助确保互操作性以及为什么它如此重要。

Denniss 指出，避免供应商锁定是人们希望使用 Kubernetes 的一个关键原因，重点是从他们正在使用的基础设施中抽象出来。他继续解释说，一致性计划是人们证明他们的工作负载在 Kubernetes 供应商之间可互操作的一种方式。“该计划的目标是提供可预测性，即如果我的工作负载在经过认证的 Kubernetes 上运行，它应该可以在任何 Kubernetes 提供商上运行。”

Denniss 接着强调，该认证还附带升级保证，“Kubernetes 的一个特点是，它的发展速度非常快[……]我们不希望全世界都停留在 Kubernetes 1.5 上，而我们正在发布 1.9。一致性计划要求人们保持最新状态，因此用户可以有选择地利用社区正在开发的所有新功能。”

Kohn 继续解释说，到目前为止，已有 42 家厂商和软件发行商注册了认证平台，目前正在完成认证。Kohn 指出，每家公司都进行自我认证，“任何用户都可以随后来运行这些完全相同的测试，并确认该平台符合他们的需求。”

“这是不寻常的，因为验证几乎是众包，”他补充说。

在最近发布的 Kubernetes 1.9 之后，cert 包括了新的一致性测试，要求公司保持其 Kubernetes 版本最新。“我们觉得这是运营 Kubernetes 的一个非常重要的部分，因为它是最新的。我们到达的地方是在 Kubernetes 发布后的一年内认证是有效的。我们允许对当前和以前版本的 Kubernetes 进行认证，”Denniss 补充道。
