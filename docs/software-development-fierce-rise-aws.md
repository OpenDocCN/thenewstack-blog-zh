对亚马逊网络服务公司(AWS)成长为一家年收入 100 亿美元的公司的报道很少伴随着对为什么的连贯、全面的解释。为什么亚马逊凭借其公共基础设施即服务(IaaS)迅速崛起，为什么它如此迅速地从如此多的参与者(硬件、软件和服务)那里获得如此多的支出？

围绕 AWS 崛起的媒体要么倾向于跳过对公司为什么转向公共 IaaS 的任何解释，要么倾向于关注树而不是森林:像云爆炸(基本上没有人会这样做)或精细定价之类的事情。

亚马逊网络服务的崛起是关于软件的崛起。马克·安德森观察到软件正在吞噬世界，这是一个自然的结果，明白这一点的技术公司将会是最成功的。

> 如果说软件正在蚕食世界(公司越来越成为软件公司)，那么公司的前进进度直接关系到他们有效开发软件的能力。

这是一个相当简单的逻辑路径:一个公司是否会成功，以及它会有多成功，直接关系到它能多有效地开发和交付软件。

今天，成功的业务执行的很大一部分是围绕软件开发的执行。而且软件开发不容易做对。以我的经验来看，绝大多数公司开发软件的方式，专家知道会导致次优结果，然而这些公司中的大多数对此一无所知。

这与缺乏有效的技术继续教育有关，优秀人才供不应求，技术创新超出了大多数公司利用技术的能力。

亚马逊网络服务的公共 IaaS 为软件开发提供了软件即服务(SaaS)为传统业务流程提供的东西。

为什么 Salesforce 扰乱 Act！？谷歌文档是如何颠覆微软 Office 的？为什么我们大多数人更喜欢基于网络和移动应用的软件，而不是桌面软件？因为 SaaS 只是更好的软件:错误修复得更快，新功能出现得更快，我们可以按使用付费，而不是预先进行大量的资本投资。

在 Amazon Web Services 出现之前，软件开发有时意味着 IT 部门要等待数周或数月才能提供开发机器。由于手动 IT 和 QA 流程，只能每季度部署更新。由于这些缓慢的过程，基于软件的业务增长被延迟。AWS 的兴起可以追溯到大量这样的对话:

**业务**:“为什么制作这个简单的新程序要花我们九个月的时间？”
**软件开发**:“嗯，开发只需要三个月，但是准备好用于开发的机器，然后进行测试，然后部署到生产环境中需要另外六个月。”
**商务**:“有没有办法让它走得更快？”
**软件开发**:“嗯，我们可以绕过 IT 部门，从 AWS 购买，然后让我们需要的所有机器在几分钟内启动并运行。”
**生意**:“那要多少钱？”
**软件开发**:“每个月几百块钱”
**业务**:“这是我的公司卡。去吧。”

今天，亚马逊从传统的软件开发和软件部署过程中消除了如此多的摩擦，同时仍然为基本上每个公司提供有效部署软件所需的所有元素(以至于它变得越来越[复杂](https://thenewstack.io/amazon-web-services-isnt-winning-problems-poses/)和有问题的)。对于越来越多的公司来说，减少开发和部署软件的时间直接转化为更高的增长，这些公司正在使用他们的软件来获取更多的收入。
