# 为什么框架定义了 Java 的云原生未来

> 原文：<https://thenewstack.io/why-frameworks-define-javas-cloud-native-future/>

datastass 赞助了这个播客。

Java 仍然是 T2 最受欢迎和信任的编程语言之一，但是它不一定适合构建云本地应用。

虽然 Java 的优雅和多功能性体现在它只需编写一次就可以在任何地方运行，但它是在容器化和 Kubernetes 鼓励的分布式计算模型之前开发的。换句话说，Java 不是 Kubernetes 的 [Golang](https://golang.org/) 。然而……

[数据税:为什么框架定义 Java 的云原生未来](https://thenewstack.simplecast.com/episodes/datastax-why-frameworks-define-javas-cloud-native-future)

框架很可能成为 Java 的 Kubernetes 困境的解决方案。在本期[中，新的堆栈制作者](/podcasts/makers)播客、 [DataStax](https://bit.ly/3at7wfH) 的[先锋架构师 Alice Lottini](https://uk.linkedin.com/in/alicelottini) 和[开源 DataStax 高级产品经理 Christopher Splinter](https://www.linkedin.com/in/csplinter/) 讨论了框架如何让 Java 仍然能够创建在云原生环境中运行得更好的应用程序，以及它们如何代表了这种有着 25 年历史的编程语言的新身份。《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了这一集。

由于 Java 的健壮性、多功能性和整体优雅性，它在今天仍然具有相关性，但这种语言本身并不特别适应短暂的本质和与云原生环境中的容器化平台相关的其他约束。

Lottini 说，Java 需要变得更有效率，以满足应用程序的需求，这些应用程序快速上下旋转，并且“可以很容易地共存，并通过共享它们来最大限度地利用可用资源”。

事实上，Java 在很大程度上被设计为“在任何地方运行”，它使用一个调整时间的模型来适应它运行的不同平台，洛蒂尼解释说，Java 的这一特性“在其他环境中确实很好，但它对云环境来说却是一个障碍。”

Lottini 说，因此，一个适用于云原生应用的 Java 框架将需要转向编译时准备，以使应用启动阶段“尽可能轻”。“我认为这是我们需要采取的方向，有一些框架正在朝着这个方向发展，并正在努力解决这种问题，”洛蒂尼说。

好消息是正在开发的框架提供了更快的启动时间。一些更有希望的选择包括夸尔库斯、T2、微型机器人等等。“我们看到 Java 生态系统中出现的所有这些不同的东西开始解决这些问题，特别是在 JVM 范围内——为用 Java 构建应用程序的人们。斯普林特说:“我认为我们才刚刚开始做很多这样的事情。

正在开发的框架也是最近才创建的。鉴于开源社区解决问题的速度，Java 缺乏快速启动时间的速度很可能只是一个脚注，随着框架的改进，它可能会成为未来云原生应用程序的一个角色。

Splinter 说，对于 Java 框架来说,“我认为我们处于成熟的前沿”。“随着越来越多的用户越来越关心启动时间和资源分配，以及真正成为容器原生的，我们将看到更多的采用。”
