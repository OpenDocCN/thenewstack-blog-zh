# 打基础:如何定位一个开源项目

> 原文：<https://thenewstack.io/laying-the-groundwork-how-to-position-an-open-source-project/>

开源项目最吸引人的特点是挖掘合作者的潜力。但是，当寻找用户并围绕它建立社区需要项目从数百万人中脱颖而出时，你如何建立一个计划来赚钱呢？

在本播客中， [Emily Omier，](https://www.emilyomier.com/)一位定位顾问，与创业公司合作，在[云原生](https://thenewstack.io/category/cloud-native/) / [Kubernetes](https://thenewstack.io/category/kubernetes/) 生态系统中寻找正确的定位，讨论如何通过为你的开源创业公司找到正确的市场类别来发展你的项目。

《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了这次播客。

[打基础:如何定位一个开源项目](https://thenewstack.simplecast.com/episodes/laying-the-groundwork-how-to-position-an-open-source-project)

“最终，人们希望围绕一个项目建立一个充满活力、可持续发展的社区。这就是开源的全部意义，”奥米尔说。“要做到这一点，你需要确保你吸引了合适的用户，他们会以最好的方式使用你的项目，并从中获得最大收益。”

虽然开源模型鼓励开放协作，但项目经常会被误解，这导致多个用户从中发现价值，但同时，它也可能导致目标感淡化。“你真正想要的是一群试图去同一个地方的人；他们试图达到同样的目的。为了建立这种关系，你必须清楚地告诉他们你想去哪里。

能够将你的开源项目货币化就是了解你的目标用户。“当一个问题与主题无关时，这就表明你有问题。只是说明提问者理解错了什么。然后当它和其他东西比较时…比如‘这和 x 竞争吗？’或者‘它和 x 是互补的吗？’如果这些问题有意义，那很好。如果它们没有意义，或者人们似乎把你和一些不切实际的事情相提并论，那就是你没有正确沟通的迹象。"

了解目标用户的痛点、他们期望的结果和他们正在使用的其他技术，以及注意他们可能遇到的限制，可以帮助项目取得最大的成功。“在开源环境中，你想知道，是什么让他们认为今天是我需要去找一个我可以下载来完成的项目的日子？…是什么促使他们停下手头的工作，去寻找解决方案？”奥米耶说。

所有这些都会带来结果。“他们不会对你如何解决技术问题感兴趣，除非他们知道你解决的是什么问题，”奥米耶说。“但不要关注你的项目的特点……要关注这个项目为什么重要，以及它如何让用户的生活变得更容易。”
