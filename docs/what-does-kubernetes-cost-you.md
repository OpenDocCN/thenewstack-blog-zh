# Kubernetes 花了你多少钱？

> 原文：<https://thenewstack.io/what-does-kubernetes-cost-you/>

在这一集的开源峰会上, [KubeCost](https://www.kubecost.com) 的联合创始人兼首席执行官 [Webb Brown](https://www.linkedin.com/in/webbbrown) 与新堆栈讨论了公开 Kubernetes 实际成本的问题。

无论我们谈论的是一般的云成本还是与 Kubernetes 具体相关的成本，团队抱怨的问题是缺乏可见性。这是对 AWS 的老生常谈的抱怨，但一旦 Kubernetes 出现，事情就变得更加复杂了。“现在一切都是分布式的，一切都是共享的，”布朗说。“理解和分解这些成本变得更加困难。事情会变得更加动态。”吊舱上下旋转的能力是 Kubernetes 的一个关键优势，带来了弹性，但这也使人们更难理解运行一个特定功能的成本。

[Kubernetes 花了你多少钱？](https://thenewstack.simplecast.com/episodes/what-does-kubernetes-cost-you)

成本也不仅仅是钱的问题。即使有无限的资金，查看成本信息也可以提供关于性能问题、可靠性或可用性的重要信息。“我们的创始团队在谷歌从事基础设施监控工作，我们认为成本是这个等式中非常重要的一部分，但只是等式的一部分，也就是你真正关注的是性能和成本之间的关系，”布朗说。“即使有无限的预算，您仍然会关心资源和配置，因为这确实会影响服务的可靠性和可用性。”
