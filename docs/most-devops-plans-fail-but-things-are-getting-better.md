# 大多数 DevOps 计划都失败了，但情况正在好转

> 原文：<https://thenewstack.io/most-devops-plans-fail-but-things-are-getting-better/>

有很多关于提高应用程序发布节奏的讨论，但事实是大多数组织还没有想出如何更快地部署应用程序。根据分析公司 Gartner 的数据，到 2023 年，90%的开发运维计划将无法完全满足预期。

在 New Stack Makers 播客的这一期早餐节目中，我们讨论了今天的 [DevOps](https://thenewstack.io/category/devops/) 的挣扎和挑战。还涵盖了潜在的解决方案，例如 DevOps 团队如何转向自助服务开发人员平台来实现他们的云部署目标。

[Cody De Arkland](https://www.linkedin.com/in/codydearkland) ，首席技术营销工程师[launch darky](https://launchdarkly.com/?utm_content=inline-mention)；分析公司 RedMonk 的高级分析师雷切尔·斯蒂芬斯(Rachel Stephens)；[GitOps 解决方案提供商兼 Flux creator Weaveworks 首席运营官 Steve George](https://uk.linkedin.com/in/stevegeorge)；以及 Armory 总裁兼首席运营官 Margaret Francis 都参与了此次讨论。

《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了本期播客。

[大多数 DevOps 计划失败，但情况正在好转](https://thenewstack.simplecast.com/episodes/most-devops-plans-fail-but-things-are-getting-better)

理想情况下，开发人员应该只需要创建应用程序，并在将代码提交给 GitHub 时与 Jenkins 一起进行部署。对于在 Kubernetes 上运行的应用程序，开发人员必须管理的 YAML 文件和其他配置应该是无缝的。然而，这个过程仍然非常复杂。

“人们真的很难应对整个开发人员工具链的碎片化，所以这可能是一个小小的开发人员体验差距——我试图让一些东西写在你的笔记本电脑上，一直到生产，为了实现这一点，必须集成的工具和东西的数量对许多公司来说真的令人生畏，”斯蒂芬斯说。"所以，我认为一方面工具是绝对必要的."

DevOps 缺乏标准化是“DevOps 有点混乱”的另一个例子，Francis 说。“但我认为，我看到的是当一个团队进入一个凹槽时，就像他们实际上能够标准化一个工具链和一套实践，”Francis 说。“他们确实获得了速度优势和一致性优势，这正是他们当初选择这些工具时所寻求的。”

De Arkland 说，基础设施也在不断改进，以根据性能规模的需求进行扩展和收缩。“我认为我们正处于一个不断发生事情的时代，关键是我们如何以最小的风险做到这一点？”德·阿克兰问道。“我们如何让团队更快地做到这一点？当出现问题时，我们如何让人们少接到凌晨 2 点的电话，但我们在不断改进我们所做的事情。”

然而，在可预见的未来，如何管理与当今开发运维工具链相关的复杂性预计仍将是一项挑战。“事实上，我们现在比几年前更不知道如何管理事情，”乔治说。“我们现在运行数百项服务，这些服务放在一起创建的服务只会增加复杂性。”
