虽然有传言称企业放弃云迁移并回归本地策略(这种现象被称为“云回归”)，但这一传言实际上与事实相去甚远。

根据上个月发布的 Gartner research[新报告](https://www.gartner.com/en/newsroom/press-releases/2019-04-02-gartner-forecasts-worldwide-public-cloud-revenue-to-g)，全球公共云服务市场将从 2018 年的 1824 亿美元增长到 2022 年的 3312 亿美元。

许多企业并没有完全放弃云基础设施，而是退后一步，检查存储在公共云中的工作负载，最多将 5%到 10%的工作负载转移回去。通常，这只是暂时的，因为企业会重新配置应用程序，以便在云中更好地运行。

虽然云的采用并没有放缓，但企业应该重新思考如何进行过渡。需要后退一步重新配置应用程序通常是因为公司匆忙进入云基础架构，而没有花时间规划对他们来说至关重要的应用程序和工作负载。因此，这就引出了一个问题，是时候停止谈论云回归，开始讨论更有效的方法来过渡到云基础架构了吗？

## 挖掘神话

企业并没有放弃云，但是如果他们暂停了，他们这样做的原因是什么？

成本和获取 IT 人才是常见的罪魁祸首，因为这两方面的需求通常比内部 IT 战略需要更多的资源。然而，采用云迁移的 IT 领导者应该关注长期收益而不是短期支出。是的，前期成本可能会导致价格冲击，但从长远来看，它们会带来回报。

在正确的帮助下，组织应该能够管理这些方面，同时还能够利用云可以提供的优势，特别是当我们进入互操作性时代时，在这个时代，多个数据集必须过滤到不同的基于云的工具中，才能产生所需的洞察力。

目前还不清楚有多少组织正在“遣返”或退出他们的云操作。但是，解决常见云难题的方法不是退回到内部部署战略，而是与了解他们需求的合作伙伴一起应对这些挑战。

## 钉钉云迁移第一时间

从云战略中获得最大收益归结于迁移。

定义云模型应该是一次性事件，这就是为什么与合作伙伴合作是最有意义的，合作伙伴是一位反复经历该过程的专家，他知道会发生什么，并且知道如何让您的企业完全得到它想要的。

定义云模型是一项非常技术性的任务。对于这个长期的任务，我建议向专家寻求指导。与此同时，许多企业面临着将工作负载迁移到云的迫切需求，因为他们寻求与数字化转型的竞争对手竞争。企业应该与第三方专家联系，帮助他们在云环境中获得正确的解决方案，而不是消除错误并离开云环境。

为了说明混合云策略可以带来的好处，让我们考虑一个场景。一所大型公立大学希望更好地利用其非结构化数据，以了解是什么推动了学生保留率。它拥有大量关于 It 学生的数据，包括每个学生获得的经济资助、他们的专业、他们是否住校等等，但这些数据分散在大学的数据存储中。

该大学希望将这些数据转化为可操作的见解，从而提高学生保留率，并在同类学校中更具竞争力。但是，如果没有适当的专业知识来统一所有这些数据，该大学将花费大量的精力和时间，却无法实现可持续的战略。

但是，托管服务合作伙伴可以帮助该大学与合适的云提供商建立联系，并连接其内部数据，以获得实时更新的丰富见解。

云提供了许多长期好处，但一些公司让前期成本和技术需求等因素阻碍了他们的发展和创新。通过与第三方云专家合作，组织可以确保他们的云迁移一步到位，并定义一个可持续、经济实惠的战略来支持他们雄心勃勃的业务目标。
