# Trilio Data 是一个用于 OpenStack 数据备份和恢复的时间机器

> 原文：<https://thenewstack.io/trilio-data-time-machine-openstack-data-backup-recovery/>

[Trilio 数据，时间机器 For OpenStack 数据](https://thenewstack.simplecast.com/episodes/trilio-data-a-time-machine-for-openstack-data)

随着 [OpenStack](https://www.openstack.org/) 投入生产，用户将需要某种数据保护和备份。谁在为这些客户服务？我们采访了提供 OpenStack 备份和恢复解决方案的 Trilio Data 的首席执行官 David Safaii。

尽管在整个 IT 市场中有许多公司提供数据保护和恢复，但据 Safaii 称，Trilio Data 的与众不同之处在于，它是针对 [OpenStack](https://www.openstack.org/) 的唯一原生数据保护解决方案。

该公司由曾在 EMC 工作的 Balcha Muralidhar 创建。Muralidhar 是第一个 OpenStack 备份即服务规范 [Raksha](https://wiki.openstack.org/wiki/Raksha) 的合著者之一。Muralidhar 将其产品化，并围绕该规范创建了一个闭源产品。

萨法伊说:“因为它是为云而生的，所以它符合云的特性。”“它满足了云的所有要求—可扩展、自助服务、多租户，并能跟上客户云的扩展和收缩。它与客户一起成长。”

萨法伊说，基于传统的数据保护解决方案拥有几十年前编写的代码。它需要硬件。它需要代理人。相比之下，Trilio 数据是软件定义的。它是可下载的，可以很容易地成为客户基础设施的一部分，无需任何代理或黑盒。

“这一点至关重要，因为当你考虑云时，你必须考虑管理、扩展和收缩它。Safaii 在最近的 OpenStack Summit Sydney 2017 会议上录制的最新版本的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客的采访中表示:“管理代理以恢复环境成为另一个难题。此外，传统解决方案无法提供客户如今所需的真正的多租户自助服务功能

Trilio 为任何类型的数据提供保护，从交易数据和测试开发环境到网络功能虚拟化领域的银行和电信网络；它甚至可以拯救运行在 OpenStack cloud 中的 WordPress。

“有很多不同类型的数据，我们确实对应用程序进行了保护，”萨法伊说。“我们希望人们有能力在需要时或出现腐败时恢复元气。倒回时钟。如果你愿意的话，这就像一台时间机器。”
