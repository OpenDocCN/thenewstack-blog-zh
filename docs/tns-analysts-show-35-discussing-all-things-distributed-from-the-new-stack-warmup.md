# 讨论新堆栈预热中分发的所有内容

> 原文：<https://thenewstack.io/tns-analysts-show-35-discussing-all-things-distributed-from-the-new-stack-warmup/>

在 3 月 3 日西雅图的 2015 年第一次热身中，我们谈到了向新的分布式环境的过渡，以及平台提供商、应用开发者、运营专家和企业正在适应的方式。在这一集的 New Stack Analysts 播客中，主持人 Alex Williams 加入了四位优秀的小组成员，他们都在管理和分析分布式系统的第一线:

*   **Avi Cavale**,[船运公司](https://www.shippable.com/)的联合创始人兼首席执行官，
*   **希瑟·麦克维**，巴绍[工程副总裁](http://basho.com/)，
*   **[Richard se roter](https://twitter.com/rseroter)**century link Cloud 产品管理总监，
*   还有， **[Kit Merker](https://twitter.com/KitMerker)** ，谷歌的产品经理。

更多“新股票分析师”的剧集，请查看播客部分。

[#35:讨论所有从新栈分配的东西](https://thenewstack.simplecast.com/episodes/35-discussing-all-things-distributed-from-the-new-stack-warmup)

“我们用容器解决的问题是，”Avi 说，“我如何在一台超高速运转的机器上获得一些进程，并保持它的独立性？”"

“尽管已经解决了这个难题的一部分，但其他的低效问题正在被放大，”Avi 说，并提到了几个问题，包括“编排…‘我如何管理所有这些容器的所有混乱？’"

关于容器，Heather 说，“他们允许你将其分解为微服务，这很好，但从传统的角度来看，这非常类似于计算中的图像服务。需要很好地解决的一个关键问题是，“我如何保持这些容器与时俱进，不得不一直重新构建它们？”"

Kit 说，“可能有一个迷因或神话，关于‘容器是新的虚拟机，从虚拟机转移到容器’”。“我们(在谷歌)确实有一点不同的想法，那就是容器是一种充分利用虚拟机基础设施的方法。”

Kit 说:“如果你使用容器作为减少开销的一种方式——通过共享操作系统，这是一个方面——第二个是通过堆叠它们，能够利用更多的 CPU 和内存资源——这是一种获得更好利用率的方式。“当您纵向扩展或缩减时，或者当您改变这些虚拟机上不同工作负载的组合时，您可以从中获得更多。”

“复杂性正在发生变化，”Richard 说，“正如你所看到的，微服务正在从这些‘智能’中间件解决方案向哑管道和智能端点转变，现在甚至我的不同微服务都必须更加智能地考虑‘我如何进行身份认证；我如何集中登录；“我如何确保它能够有效地水平扩展？”所以，有了这些你以前可能不太担心的事情:‘现在我的终点必须更聪明；“现在，这个东西必须扩展，支持持续集成和持续交付”，这些东西以前可能是理所当然的，现在复杂性可能会转移到我不能想当然的服务级别。"

“我们没有变得更简单，”理查德说。“我们不会在一年后回过头来说‘做分布式系统更容易。’"
