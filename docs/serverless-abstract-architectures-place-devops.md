# 无服务器和抽象架构在 DevOps 中有一席之地吗？

> 原文：<https://thenewstack.io/serverless-abstract-architectures-place-devops/>

[无服务器和抽象架构在 DevOps 中有一席之地吗？](https://thenewstack.simplecast.com/episodes/do-serverless-and-abstract-architectures-have-a-place-in-devops)

也许没有通往 [DevOps](/category/devops/) 涅槃的单一路径，也没有实现持续集成的单一方式。这是[devo PS Research and Assessment(DORA)](https://twitter.com/nicolefv)的创始人兼首席执行官 Nicole Forsgren 在旧金山 2017 年 PagerDuty Summit 上录制的 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客的一次广泛对话中得出的令人瞠目结舌的结论。

她说，像无服务器架构这样的技术可以很好地从软件开发任务中抽象出操作细节。但福斯格伦问道，如果这一切最终都为最终用户和客户带来了积极的价值，那么谁会关心 DevOps 是如何完成的呢？由[傀儡](https://puppet.com/)每年出版的[DevOps 报告](https://puppet.com/resources/whitepaper/state-of-devops-report)的共同作者甚至建议，对于任何两个组织，devo PS 可能没有一个状态。

“我们的许多工作都应该手动完成。如果我是一名程序员，我应该做新颖、创新、有趣、有创意、增值的工作，”福斯格伦说。“但是那些死记硬背、可预测的事情……让我们尽可能地实现自动化，这样我就可以腾出时间来为我的组织增加价值。”
