# Glassdoor:不要为了新功能而牺牲性能

> 原文：<https://thenewstack.io/glassdoor-dont-sacrifice-performance-for-new-features/>

欢迎来到[新的堆栈制造商](/podcasts/makers):攀登新的高度，这是一系列对工程经理的采访，他们谈论了他们面临的问题和他们寻求的解决方案，由嘉宾主持 [Scalyr](https://www.scalyr.com/) 首席执行官[克里斯汀·赫克特主持。](https://www.linkedin.com/in/christineheckart/)

当 Bhawna Singh 开始担任 Glassdoor 的首席技术官和工程高级副总裁时，她有两项任务:在旧金山开设一个办公室，以利用该地区的人才库，并重建工作结果的垂直搜索。Glassdoor 是一个工作和招聘网站，它提供的服务允许人们查看信息，如公司评论、工资评论和潜在雇主提供的福利。

[攀登新高度 EP # 7–玻璃门:性能至关重要](https://thenewstack.simplecast.com/episodes/scaling-new-heights-ep-7-glassdoor-performance-matters)

为了提高搜索质量，Heckart 必须设定团队可以信任的指标。当团队将精力集中在构建平台的战术方面时，性能挑战浮出了水面。他们调整了系统的质量；构建部署基础架构并添加机器学习模型。但是这项工作使系统变得更重，性能更差。

此时，该网站的流量越来越大，性能问题也越来越严重，这使得它更难扩展。质量改进被拉回以关注性能——这是必要的，因为做这项工作的团队已经相当小了。

然后，该团队将负载和性能测试特性构建到他们的部署管道中，以确保他们正在比较每个版本的性能。此外，他们回到了最初的质量改进路线图。

工作得到了回报。随着基础设施的完善，性能确实有所提高，工作也进入了下一个阶段:离开整块石头。
