# 本周新闻:Oracle 的自主数据库 InfluxData 的适时数据库

> 原文：<https://thenewstack.io/week-news-oracles-autonomous-database-influxdatas-timely-one/>

[本周要闻:Oracle 的自主数据库，InfluxData 的适时一期](https://thenewstack.simplecast.com/episodes/this-week-in-news-oracles-autonomous-database-influxdatas-timely-one)

大家好，欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)播客，在这里，我们回顾了本周云原生技术和大规模应用程序开发方面的最热门新闻，并展望了我们预计将在未来几周获得更多关注的主题。

本周我们忙于报道纽约的事件，从周一甲骨文的云日活动[开始，在那里我们了解到了云提供商自主服务的最新情况。周二，我们还参加了时间序列数据库提供商 InfluxDB 的用户会议。](https://thenewstack.io/oracle-promises-self-securing-self-repairing-autonomous-cloud-services/)

我们还将讨论 Twistlock 的 John Morello 本周提供给我们的一篇有趣的文章，这篇文章解决了 Twistlock 的许多客户面临的一个问题。他们想知道如何在不同的云原生技术之间进行选择，以进行工作负载部署和管理。

为了使决策更容易，Morello 将云原生技术放在一个连续体上，从左边的虚拟机(VM)到中间的容器，再到右边的无服务器。您做出的选择最终取决于每种工具的优势和权衡，Twistlock 发现许多客户至少同时使用其中的一些工具。
