# 现在钻取顶级 Apache 项目

> 原文：<https://thenewstack.io/drill-now-a-top-level-apache-project/>

Apache Software Foundation 已经将 Drill 提升到一个顶级项目，类似于 Hadoop、Spark 和 httpd。Drill 于 2012 年 8 月进入 Apache 孵化器，并于去年 8 月首次向公众发布。它的团队，包括 MapR、Intuit、Mesosphere 和 Hortonworks 的工程师，已经发布了每月更新。

当很明显存在一个大的社区，并且项目已经存在时，项目就被提升到顶级状态。Apache Drill 项目管理委员会的成员 Tomer Shiran 解释说，这吸引了更多的用户和贡献者。

他将这一新地位称为“Drill 的一个重要里程碑”，并补充道，“我们很高兴 Drill 确实将成为 Hadoop 应用程序开发人员和 BI 分析师等的游戏规则改变者。”

升级后的状态让这个项目更加引人注目——它现在在 drill.apache.org[有自己的网站，Apache 的新闻团队也参与其中。与此同时，Drill 已经成为一个自主的项目，可以自己决定发布、流程等。](http://drill.apache.org)

Shiran 领导 MapR 的产品管理团队，该团队已将 Drill 集成到其大数据平台中。他在[的一篇博客文章](http://drill.apache.org/blog/2014/12/02/drill-top-level-project/)中解释说，Drill 是出于更快应用程序开发的需要而发展起来的，以多种形式使用来自无数来源的大量数据。

Drill 被设计为用于多个数据源的无模式 SQL 查询引擎。包括 JSON、Parquet 和 HBase。它不仅允许在 Apache Hadoop 上快速开发应用程序，还使企业 BI 分析师能够自己探索数据——将 It 人员从为他们构建数据中解放出来。

“支持更多业务部门[使用我们的 Hadoop 大数据](http://venturebeat.com/2014/09/16/mapr-redefines-sql-on-hadoop-with-apache-drill/)并从单个 ANSI SQL 接口关联我们的其他大数据源将会非常强大，并且几乎不需要重新调整。托管安全服务公司 Solutionary 的软件开发总监斯科特·罗斯曼(Scott Russman)在 Drill 的首次发布会上表示:“这将减少定制应用程序的工作，提高整个组织的知识共享效率。

惠普、SAP 和 NuoDB 是最近推出基于 Hadoop 的 SQL 产品的公司之一。但是，Drill 让您无需 ETL 或先创建模式就可以分析 Hadoop 数据；它动态生成模式，并保持文件的原始格式，而不是在将文件加载到数据库系统之前将其转换为表格或预先指定的格式。

与 Hive 和 Impala 等技术不同，它不仅限于 Hadoop。它支持存储插件，包括本地文件系统，HDFS，MongoDB，HBase，其他正在开发中。单个查询可以从多个来源收集数据。

至于 Drill 的未来，“我们将推出许多新功能。1.0 将在 Q1 发布。像 Cassandra 和 RDBMS 这样的新数据源正在增加。我们还增加了更高级的 SQL 功能。我们现在正与一些更成熟的用户密切合作，解决像稀疏 JSON 数据这样的新用例，”Shiran 说。
