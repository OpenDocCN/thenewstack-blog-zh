# IBM 要想在云计算领域真正竞争必须做些什么

> 原文：<https://thenewstack.io/what-ibm-must-do-to-really-compete-in-the-cloud/>

亚马逊网络服务是云 1.0 战争中无可争议的赢家。但是，谁会赢得云 2.0，我指的是混合云和多云世界？谷歌和微软显然在竞争中，但被亚马逊和云计算 1.0 抢先的主要 IT 平台也是如此:IBM、VMware、思科、HPE、戴尔等。

在新堆栈的一系列文章中，我将探讨每个主要参与者需要做些什么才能在云 2.0 世界中竞争并获胜。让我们从 IBM 开始，它的[刚刚发布了超过预期的收益](https://finance.yahoo.com/news/ibm-earnings-beat-estimates-cloud-200812153.html)——部分归功于其云业务的增长。

IBM 有公共云，但在与 AWS、微软和谷歌的云战争中，它的公共云并不是真正重要的东西。相反，IBM 有一个秘密武器，不需要同样大规模的资本支出:软件。

第一代云完全是关于构建基础设施，而第二代云是关于管理基础设施内和跨基础设施的应用程序的软件。企业真正关心的是应用，而不是基础设施。这意味着现在企业的真正价值在应用管理层，而不是基础架构层。

虽然 IBM 有公共云，但其更相关的资产是软件相关的:沃森人工智能产品、区块链解决方案、Cognos，当然还有紧随其收购红帽之后的新内容。特别是通过收购 Red Hat 及其软件驱动的应用管理堆栈 OpenShift，IBM 现在可以在云战争中超越其重量，并从客户那里获得越来越大的价值份额，这些客户通常会在大型云提供商而不是 IBM 那里花费数十亿美元。

OpenShift 结合了 IBM 可以用来赢得云战争的三项资产:

1.  高利润软件(相对于低利润基础设施)。
2.  基于 Kubernetes 的复杂应用管理平台。
3.  提供真正的混合和多云操作的能力。

由 OpenShift 推动的这一组合允许企业在任何环境中轻松、快速且经济高效地运行应用程序。虽然云提供商通过降价来吸引客户，有效地将他们自己的服务商品化，但 IBM 可以利用这一点，在其他公共云的基础上提供一个平台，允许企业在他们决定运行应用的任何地方管理他们的应用。

这并不意味着 AWS、微软或谷歌将温和地进入那个美好的夜晚，但如果 IBM 要与主要的云提供商竞争，它必须以提供多云应用管理平台为基础。事实上，就连谷歌也在用谷歌 Anthos 做同样的事情，这表明它也意识到仅仅有基础设施是不够的。

我对 IBM 的 Red Hat 收购和 OpenShift 战略给予 A+评级，但目前执行情况为 C- on。随着时间的推移，这种糟糕的执行可能会让 IBM 损失数十亿美元的收入和股东价值。IBM-Red Hat 执行失误的一个例子是[未能看到数据管理在企业](https://www.forbes.com/sites/forbestechcouncil/2020/07/07/why-the-big-cloud-providers-are-falling-short-on-hybrid/#59eecf6578a4)中的重要性，以及如何不为运行在 OpenShift 上的应用程序提供完整的数据管理平台，大规模的采用就不会到来。

根据[GigaOm Research Radar for Kubernetes Storage](https://gigaom.com/report/gigaom-radar-for-data-storage-for-kubernetes/)的报告，RedHat 被列为“特色剧”在这种情况下，与“平台玩法”相反，功能玩法意味着该产品仅专注于存储，而不是更广泛的数据服务平台，如备份、灾难恢复、安全等。(可通过其他 Red Hat 和 IBM 服务获得)。此外，虽然路线图会随着时间的推移而变化和发展，但事实上，今天的 Red Hat OpenShift 容器存储仅限于仅由[9 台服务器和 108TB 可用存储](https://access.redhat.com/documentation/en-us/red_hat_openshift_container_storage/4.4/html/planning_your_deployment/supported-configurations_rhocs#sizing_and_scaling_for_dynamic_storage)组成的集群，这表明 OpenShift 正在从一个非常成功的 SMB 公司的角度接近企业销售大联盟，而不是一个能够与公共云提供商竞争的公司。

无论是通过自己构建，还是与不断增长的云原生存储提供商生态系统合作，OpenShift 都需要能够响应世界上最大的零售商、金融服务集团、医疗保健和制药公司的愿望，这些公司希望在 OpenShift 上运行他们的所有应用程序，而不仅仅是不需要超过 36TB 存储的微小子集。我相信 IBM 能做到。我敢打赌，AWS、谷歌和微软都希望他们不能。
