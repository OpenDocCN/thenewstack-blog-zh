# 亚马逊网络服务和 Kubernetes:现在怎么办？

> 原文：<https://thenewstack.io/aws-kubernetes-now/>

[#153: AWS 和 Kubernetes:现在怎么办？](https://thenewstack.simplecast.com/episodes/153-aws-and-kubernetes-now-what)

随着亚马逊在上个月的 AWS re:Invent 上发布了许多公告，包括其针对 Kubernetes 的新亚马逊弹性容器服务( [EKS](https://aws.amazon.com/eks/) )，许多人想知道亚马逊将如何在 Kubernetes 社区中定位自己，以及其 EKS 和全方位服务 [Fargate](https://aws.amazon.com/fargate/) 托管容器产品将如何影响亚马逊弹性容器服务(ECS)和整个 [Kubernetes](/category/kubernetes/) 社区。

开发人员注意到，Fargate 几乎否定了 EKS 的存在，这就向我们的嘉宾提出了关于最新一集[新堆栈分析师](https://thenewstack.io/podcasts/analysts)的问题，我们邀请到了 Adobe 开发生态系统负责人 [Matt Asay](https://www.linkedin.com/in/mjasay/) 和负责基础设施、应用平台和开发的首席研究顾问 Krishnan subra manian[。](https://www.linkedin.com/in/krishnansubramanian/)

问题是，当 ECS 做得这么好的时候，为什么要提供 EKS 呢？Asay 继续解释道，答案很简单:“为什么不提供这项服务呢？如果它战胜了 ECS，那太好了。他们在那个游戏里。如果没有，ECS 继续增长，可能不会公开边缘化 Kubernetes，但会有效地边缘化 Kubernetes。”

Subramanian 在前面的讨论中也提到了这样一个事实:“当亚马逊领先时，他们会坚持己见，要么按我的方式走，要么就走高速公路。当他们落后时，他们提供选择，为客户提供选择。”
