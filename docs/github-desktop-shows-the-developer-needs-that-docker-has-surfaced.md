# GitHub 桌面展示了 Docker 浮出水面的开发者需求

> 原文：<https://thenewstack.io/github-desktop-shows-the-developer-needs-that-docker-has-surfaced/>

GitHub 发布了 [GitHub Desktop](https://desktop.github.com/) ,让开发人员能够从笔记本电脑上编写代码，并将提交的更改直接推送到他们在 GitHub 网站和 GitHub Enterprise 上的 repos。

此举承认了新的编码环境和开发人员的需求，自从 Docker 和容器生态系统运动的发展以来，这些需求已经成为规范。Docker 和容器技术的最大优势之一是，它为开发人员提供了从个人编码项目到 QA 和登台、测试、部署和生产的单一事实来源。当移动到更大的应用程序架构设置时，个人笔记本电脑上的应用程序开发环境与整个团队或企业系统上的应用程序开发环境是相同的。

这也是对 GitHub 自身不断扩大的影响力的日益认可。像波因特网站上 Melody Kramer 的记者 GitHub 教程这样的文章越来越受欢迎，因为它们展示了开发工具对非开发人员的影响，就像[非开发人员可以利用容器技术](https://thenewstack.io/containers-non-developers-shifting-conversation-wider-business-focus/)一样。虽然大多数开发人员可能习惯使用命令行从 Mac(或 Windows)上访问他们的 Git repos，但是桌面版本意味着不仅开发人员可以从他们的笔记本电脑上使用用户界面访问他们的 repos，而且——可能更重要的是——越来越多地使用 GitHub 平台的非开发人员也可以使用。

现在，GitHub 正在模仿这种体验，让开发人员能够在桌面(Mac 或 Windows)版本的 GitHub 上工作，并访问他们将通过 web 门户访问的一些相同的功能。开发人员可以在他们的 GitHub 帐户中克隆或创建新的项目回购，创建和切换分支，并查看不同版本的未提交的更改。新的桌面工具还提供了 GitHub.com 的可视化元素，比如查看提交图和浏览本地分支。

开发人员还可以创建拉请求，并将更改合并回主分支。

理想情况下，新的桌面工具适合处于开发到部署工作流程中的开发人员，但对于那些从事开源项目或使用 GitHub 监控现有应用程序中的错误、功能请求或其他问题的开发人员来说，桌面还不是答案。用户无法通过 GitHub Desktop 查看问题，也无法添加新问题或回复现有问题，因此对于这些更基于团队或面向社区的开发任务，浏览器或命令行仍然是必不可少的。

GitHub Desktop for Mac 的一个很酷、像复活节彩蛋的方面是给[发行说明](https://desktop.github.com/release-notes/mac/)的有趣标题。改进、添加或修复的功能列表显示了它们的相关版本号(新桌面现在是 3.0 版)。我猜想，每一组变化都有一个简短的标题，表面上是从一些随机的网络迷因中摘取的。《牢不可破的童菲·施米特》中的经典台词“重生的巨魔，杰瑞米”获得了认可，还有托比亚斯·芬克在《发展受阻》中最令人难忘的台词的标签友好版本这是吸引用户阅读枯燥的变更日志的一个很好的方式，并且给回顾版本更新中的变化带来了一种古怪的乐趣。

Docker 是新堆栈的赞助商。
