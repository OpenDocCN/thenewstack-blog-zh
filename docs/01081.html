OpenFaaS创始人谈开源社区资助模式的成功

在最新一集的 The New Stack Makers 播客中，我们与 OpenFaaS 的创建者 Alex Ellis 坐在一起，讨论他的开源无服务器平台的最新更新，这个新概念“无服务器2.0 意味着什么，以及支持开源项目的困难。

Ellis几年前创建了OpenFaaS，希望将Amazon Web Services的function-as-a-service提供的功能扩展到任何容器化的计算环境，这一功能后来被其他无服务器产品效仿。这个想法是，你不仅可以把你的应用程序打包，还可以把所有的支持库和依赖项打包到一个容器中，这样它们就可以无服务器运行了。您不局限于一个云提供商或一个语言运行时。“我想找到一种方式，将我对Docker的热爱和对编码的热爱结合起来，”Ellis说。

有了无服务器2.0，“你可以运行任何代码，无论是二进制还是HTTP服务器，以任何你喜欢的方式——你的笔记本电脑，内部，在OpenShift上，在云中，”埃利斯说。Kubernetes提供了一个共同的基础。

虽然Ellis自己创建了OpenFaaS，但他已经围绕该软件建立了一个健康的贡献者社区，以及一个非常健壮的最终用户社区，包括许多大公司。他承认“让飞轮转动起来”需要大量的工作，尽管通过志愿者捐赠机制只找到了有限的支持，如GitHub赞助商或Linux基金会的社区桥。

“它们都基于同一个模型:如果我们创造一种让你的支出透明化的方式，人们就会给你钱。我认为事实并非如此，”埃利斯说，称这些努力在大多数情况下是“失败的模式”。

(自从接受我们的采访后，埃利斯启动了一个内部人士计划，每当赞助商和支持者在GitHub或Patreon上赞助他时，都会给他们带来附加值。他说，这已经比捐赠模式成功得多。这从5美元开始，上升到主页赞助水平。内部人员将定期收到“内部人员更新”，涵盖Alex的所有OSS工作、博客、视频、行业更新，以及对OpenFaaS新闻和功能的早期访问。)
