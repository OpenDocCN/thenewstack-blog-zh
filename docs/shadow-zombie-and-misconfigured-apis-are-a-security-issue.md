# 影子、僵尸和错误配置的 API 是一个安全问题

> 原文：<https://thenewstack.io/shadow-zombie-and-misconfigured-apis-are-a-security-issue/>

影子 API 是一个不是每个人都认真对待的问题。只有 11%的受访者对 Salt Security 的 API Security Q3 2021 的[状态非常担心影子或未知 API 的安全风险。相比之下，42%的人非常担心僵尸 API，它们被认为是关闭的，但却可能成为帐户接管的目标。](https://salt.security/api-security-trends)

解决影子 API 的第一步是记录您的组织拥有的影子 API。与六个月前第一次进行调查时相比，至少对自己的原料药库存有一定信心的受访者比例从 53%增加到 62%。

Salt Security 的客户数据表明，受访者明显低估了现有影子 API 的数量。每个人都知道[错误配置的 API](https://thenewstack.io/cloud-engineers-try-policy-as-code-to-cure-misconfiguration-woes/)对于云安全来说是一个大的并且[越来越大的](https://thenewstack.io/misconfiguration-worries-grow/)问题。每一个未说明的 API 都会增加这个问题，但是没有正确记录的 API 也会增加这个问题。

也许今后最容易做的事情就是提醒大家，僵尸 API 实际上只是影子 API 的一种。更难的是让成年人承认他们害怕自己的影子。
