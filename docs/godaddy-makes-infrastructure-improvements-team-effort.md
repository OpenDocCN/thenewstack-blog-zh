# GoDaddy 如何通过 OpenStack 和微服务简化配置

> 原文：<https://thenewstack.io/godaddy-makes-infrastructure-improvements-team-effort/>

将服务器部署到生产环境中并组织一个结构良好的流程编排是一项挑战。在 GoDaddy，将服务器部署到生产中的过程曾经是一个漫长的手动过程，主要由支持票证驱动。通过利用持续集成管道、编排工具和过渡到微服务，GoDaddy 大大缩短了上市时间，开发人员可以在不到一分钟的时间内将服务器部署到生产中。

在本期 [The New Stack Analysts](/tag/the-new-stack-analysts/) 播客中，New Stack 创始人 Alex Williams 和共同主持人 Scott M. Fulton III 将了解 GoDaddy 向自动化程度更高的生产流水线的转变。OpenStack 工程的高级总监[沙希达·尼扎尔](https://www.linkedin.com/in/shaheeda-nizar-6674532)也加入了他们的行列；[乔希·哈洛](https://github.com/harlowja)，GoDaddy OpenStack 技术负责人；还有 [Kris Lindgren](https://www.linkedin.com/in/krislindgren) ，GoDaddy 高级 Linux 系统工程师。

林德格伦指出，在 GoDaddy 实现自动化之前，在该公司安装服务器是非常困难的。“有很多人在等待其他团队。一切都是开一张服务单，让其他人等着，催促他们把事情做完。一切都是手动的。林德格伦说:“没有 API 或 ui 来与许多服务进行交互。GoDaddy 的团队知道有些事情必须改变。

一些事情导致了 OpenStack 的出现。最初，GoDaddy 的基础设施不能在 OpenStack 上运行，所以他们很快转向与开源社区合作，这是一个在雅虎非常成功的策略。切换后，GoDaddy 团队试图通过创建更好的自动化生产流水线工具来解决问题。

在实现可靠的编排设置的整个持续挑战中，GoDaddy 一直面临着挑战。尼扎尔强调，这项任务需要团队努力，将多个部门聚集在一起，努力确定并消除当前流程中不必要的步骤。她指出:“围绕部署流程和已完成的常见工作，仍有一些工具链需要整合。

在许多公司已经构建了单片应用程序的地方，GoDaddy 已经开始将其架构转变为具有一系列微服务的分布式系统。“它建立在无共享架构上，因此部署简单且有弹性，”尼扎尔说，并指出这来之不易。

所涉及的工作是一个漫长而具有挑战性的转换，在创建每个服务时都包含了深入的技术工作。尼扎尔进一步解释说，尽管基础设施已经完工，但还有很多工作要做。改善其基础设施的下一步包括用正确的架构建立 GoDaddy 的产品团队，然后让团队成员开始为他们的工作流构建微服务。

公司很少提到过渡到微服务、统一团队以完成共同目标以及构建新基础设施所涉及的工作。考虑采用微服务的公司可能会感到沮丧，因为他们的设置是一个挑战，而不是他们可能期待的“点击完成”设置。幸运的是，尼扎尔、哈洛和林德格伦在本期节目中解释了这种努力是如何得到回报的。

GoDaddy 是新堆栈的赞助商。
