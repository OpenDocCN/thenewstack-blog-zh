# 成熟的容器技术环境中的{code}使命

> 原文：<https://thenewstack.io/code-mission-maturing-container-technology-landscape/>

[{ code }仓储项目](https://thenewstack.simplecast.com/episodes/the-container-landscapes-maturity-evident-in-code-storage-projects) 集装箱景观的成熟

集装箱行业正在走向成熟。伴随着这种成熟而来的是新的挑战，即公司如何在运营环境中考虑状态、持久数据和应用程序架构。

人们如何在生产中使用容器将在很大程度上取决于他们的网络和存储技术如何满足云原生计算的新需求。今天的许多容器技术都是作为开源项目构建的。正是在这些开源社区中，像 [REX-Ray](https://github.com/rexray/rexray) 这样的项目得到了开发。REX-Ray 是一个容器存储编排引擎，支持云原生工作负载的持久性，提供与各种环境(包括戴尔、亚马逊和 Red Hat Ceph)的存储集成。它还与容器存储接口、Kubernetes 和 OpenShift、Docker 以及 Apache Mesos 互操作。

REX-Ray 由戴尔技术公司开源，由该公司的开源研发部门{code}管理。我们[在软件马戏团采访了](https://www.periscope.tv/thenewstack/1yoKMpVMdmoxQ)克林特·麦显杰 , {code}的主管，并询问了容器技术的成熟及其与{code}的契合度，以及它的推广和它为使 REX-Ray 与不同存储环境集成所做的工作。
