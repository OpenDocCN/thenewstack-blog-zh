将分析嵌入到您的商业软件应用程序中可以带来巨大的价值。它为业务用户提供了对其应用环境中的分析的访问。但是实践带来了挑战:产品所有者不仅需要特别注意 UI 定制，还需要特别注意架构集成。一个架构良好的集成可确保一个安全、高性能的环境，为用户提供无缝体验，以便在您的应用程序中始终提供分析功能。

考虑到这些关键要求，产品所有者应该特别注意大量的现代特性。以下是将分析嵌入另一个产品或应用程序时要考虑的一些最重要的因素:

## 轻松设置，无需客户端编码

设置您的数据分析业务流程应该很简单。您的客户无需安装任何客户端应用程序、插件或购买新设备即可开始使用。同样重要的是，当硬件或操作系统配置发生变化时，产品所有者不应担心潜在的升级或迁移。

鉴于此，标准支持应该包括单点登录和 RESTful 架构、各种 API 和前端技术，如 [JavaScript](/tag/javascript/) 。这确保了对安全性、web 应用程序和数据连接的一系列支持。

## 无需培训即可使用

最终，您的客户将会参与到您产品的分析方面，提供自助式用户界面和引导式报告创建对于最大限度地提高便利性至关重要。

嵌入式分析通过提供上下文洞察实现了更高的可用性，因为分析是从您的应用程序内部提供的。此外，解决方案本身可以利用大量可用性概念来帮助完成从数据到洞察的任务。

例如，图表创建应该具有诸如一次点击图表转换这样的快速动作，允许用户快速原型化他们的最佳可视化方式。此外，响应式设计支持各种设备外形的可视化。

考虑到这一供应以及无缝集成的分析环境，您的客户应该无需培训即可使用您的分析功能。产品所有者应该警惕那些需要冗长的现场或网络培训课程才能使用其产品的供应商。如果用户已经接受了核心应用程序的培训，他们不应该需要全面的培训来使用面向分析的一面。

## 可定制的独特体验

如上所述，产品负责人通常需要提供特定环境下的分析。这方面的考虑包括白标、语言本地化或分析解决方案如何出现在应用程序中。当需求可能发生变化时，提供静态分析界面、纯英文解决方案或在定制和集成功能方面的肤浅选项的解决方案将会限制您的应用程序。

为此，产品所有者应该寻求一种分析解决方案，允许他们采取细粒度的方法来决定嵌入哪些单独的功能。此外，这种方法应该允许控制特性的行为，以及在应用程序的某些区域公开功能和控制选项的选项。

## 最后的想法

产品所有者知道，为了支持收入增长，集成分析对于增加产品价值至关重要。归根结底，集成对于通过应用程序内部的统一分析工作流提供上下文洞察至关重要。

客户将真正喜欢在方便、易用且提供高级功能的环境中工作。此外，良好集成的应用程序减轻了开发团队的工作负担，因为对于您的客户来说，一切都更加同步和简化。具体来说，R&D 由分析提供商提供支持，该提供商将持续改进您的分析产品，而不会分散您的开发团队在核心应用程序上的工作。
