# 建立采用 Kubernetes 的最佳实践

> 原文：<https://thenewstack.io/establishing-best-practices-adopting-kubernetes/>

为新兴技术建立最佳实践是一项很多人会认为不可能完成的任务。但是，对你来说，识别团队是如何持续解决问题的仍然很重要，即使他们周围的技术在变化。Kubernetes 有许多最佳实践，它们的出现可能为评估 Kubernetes 的未来提供关键。

在本期《新堆栈制造商》中，我们在 [ReactiveOps](https://www.reactiveops.com/) 采访了首席技术官[贾斯汀·蒙德](https://www.linkedin.com/in/justinmound/)和现场可靠性工程师[埃里克·霍尔](https://www.linkedin.com/in/eric-hole-519a468/)。根据他们与采用和利用该平台的客户的互动，我们询问了他们认为围绕 Kubernetes 出现的最佳实践。虽然这些最佳实践中的许多在 Kubernetes 社区中得到了巩固，但是仍然有许多棘手的问题，比如存储持久性和秘密管理。我们讨论了 ReactiveOps 如何解决客户的一些问题，以及随着 Kubernetes 生态系统中工具的发展，他们期望解决哪些问题。
