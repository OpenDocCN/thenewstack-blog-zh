# 从 IT 管理员到 Kubernetes 管理员

> 原文：<https://thenewstack.io/from-it-administrator-to-kubernetes-administrator/>

[戴尔技术](https://www.delltechnologies.com/en-us/index.htm)赞助了该播客，该播客于 11 月 18 日至 21 日在圣地亚哥举行的 [KubeCon + CloudNativeCon 北美 2019](https://events.linuxfoundation.org/events/kubecon-cloudnativecon-north-america-2019/) 上录制。

我们正在见证数字化转型的下一个阶段。在第一阶段，瀑布团队和遗留架构被分成越来越小的小组，以使组织能够快速响应客户需求。现在，随着 Kubernetes 和新兴最佳实践的广泛采用，整个业务都在发生变化，角色也在演变。

我们如何确保每个人都参与下一步？

今年 11 月，New Stack 创始人兼出版商 Alex Williams 与戴尔 EMC 高级营销顾问 [Bob Ganley](https://www.linkedin.com/in/ganleybob/) 和 Pivotal 专业销售和合作伙伴总经理 [Craig Steel](https://www.linkedin.com/in/crsteel/) 在圣地亚哥的 KubeCon + CloudNativeCon 坐下来讨论这个问题。

[从 IT 管理员到 Kubernetes 管理员](https://thenewstack.simplecast.com/episodes/from-it-admin-to-kubernetes-admin)

“我们看到的是，现有的传统虚拟化应用程序有许多相关的基础架构。人们知道如何经营它们，而且在技能方面也有大量投资。现在，随着组织开始关注原生云，他们如何利用现有投资并将其扩展到运行新一代应用程序？”甘利说。

他继续说，转型不再只是一个预算项目。它涵盖了整个业务—从首席管理人员到 It 和运营人员，再到实施这些计划的开发人员。要做到这一点，他们必须致力于在整个组织中进行这些变革所需的文化转变。这一切的基础设施支柱——我们可以很快接受这一点——将是 Kubernetes。

Steel 说，Kubernetes 充当了一个底层编排层，任何人都可以在其上编写 API。他在采访中分享道，在过去五年左右的时间里，开发人员已经成为某种“国王的创造者”，他们颠覆 IT，快速采用云原生流程和服务，如 Kubernetes。现在是时候让企业的 IT 和运营部门作为下一阶段数字化转型的主要利益相关方重返谈判桌了。

因为，正如威廉姆斯所说，“采用 Kubernetes 是一回事，管理它完全是另一回事。”

## 云作为一种运营模式

Ganley 和戴尔 EMC 团队不再将云视为一个地方，而是一种运营模式。他提出了他们建议组织权衡的三件事:

*   物理定律:也许您需要应用程序的某个部分具有低延迟，这推动了局部性。
*   经济法则:也许您有一个需要弹性的短期工作负载，这推动了公共云的使用。
*   土地法则:或者数据存放在哪里是最重要的。

每个应用程序在这三者之间都有不同的平衡。这导致了多云方法越来越受欢迎，这种方法本质上要求灵活性，以及使您能够控制这种灵活性的工具。

然而，通常在灵活性和安全性之间有一个折衷。正如甘利指出的，“安全始于:你是谁，你能访问什么？”工具越多，就越模糊。在这一集中，他提供了戴尔技术堆栈作为管理这种微妙平衡的方法。

同样，在 Pivotal，他们专注于通过您为应用程序构建的管道将业务推向 CI/CD，这反过来使您的应用程序更像一种产品。然后，产品就是你不断努力使其“更好、更快、更强、可扩展”的东西

随着我们从 IT 管理员过渡到 Kubernetes 管理员，从项目过渡到产品，这一集深入探讨了许多来自 IT 领域的最佳实践和模式。
