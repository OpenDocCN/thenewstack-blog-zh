# Datadog 的数据科学将监控从黑暗中拉出

> 原文：<https://thenewstack.io/datadog-bringing-monitoring-darkness-light/>

监控的未来正在快速变化，因为越来越多的企业开始看到以[算法和编程方式](https://thenewstack.io/datadog-monitors-scalable-systems/)查看警报的价值，因为他们很难确定图表上的峰值或谷值是好是坏。

在这一集的[中，New Stack Makers](/tag/the-new-stack-makers/)embedded below，New Stack 创始人 Alex Williams 与基础设施监控工具提供商 [Datadog](https://www.datadoghq.com/) 的技术社区和传道部总监 [Ilan Rabinovitch](https://twitter.com/irabinovitch) 在[OSCON·奥斯丁](https://thenewstack.io/preview-the-new-stacks-ebooks-about-docker-and-containers-launching-at-oscon-on-github/)会面，详细了解 Datadog 如何帮助开发人员使用基于微服务的基础设施，开发中容器的兴起，以及收集指标的方式在经历了十年的黑暗之后重新受到欢迎。

[TNS 采访:伊兰·拉宾诺维奇在奥斯丁的 OSCON](https://thenewstack.simplecast.com/episodes/tns-interviews-ilan-rabinovitch-at-oscon-in-austin)

这段对话还可以在 YouTube 上观看。

随着公司试图过滤信噪比，数据科学越来越多地应用于运营。Datadog 本身利用离群点检测和聚类算法来帮助。

“事实证明，基于标记的原语、使用标记将警报和仪表板构建为带有谓词的查询为您提供了相当多的功能。监控是一个复杂的大数据问题，我们可以帮助简化这个问题。“在被蒙在鼓里十年后，看到每个人都再次对指标感到兴奋，这太好了，”Rabinovitch 说。

通过使用 Datadog 独特的图像标记方法，开发人员可以获得他们整个架构的细粒度图片，而不会产生不必要的噪音。由于能够标记从发生在组织中每个数据中心的事件到仅发生在单个数据中心位置的事件，Datadog 在监控其堆栈时为开发人员提供了灵活性和动力。

“想象一下，如果我们 10 年前有这样的对话。宠物和牛的类比是存在的，因为人们以希腊神的名字命名他们的服务器，因为他们认为它们会存在很多年。我们将继续看到这种下降，因为人们正在采用像 Kubernetes、[、OpenShift 和 DC/操作系统这样的](https://thenewstack.io/solving-spoiled-broth-open-source/)[编排工具。](/tag/ebook3-automation-and-orchestration/)

Datadog 每天处理近 3 万亿个数据点，不仅利用自己的专有解决方案，还利用各种开源工具，如 Cassandra、PostgreSQL 和 Apache Kafka。

“大多数基础设施管道的默认答案是开源，这太神奇了。十年前，情况并非如此。如今，它是默认的。如果你看看容器领域，那里没有很多专有的容器解决方案。它们都是开源的，”拉宾诺维奇解释道。

Red Hat OpenShift 是新堆栈的赞助商。
