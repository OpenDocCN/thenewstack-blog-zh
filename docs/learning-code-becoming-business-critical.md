# 学习编码是否变得对业务至关重要？

> 原文：<https://thenewstack.io/learning-code-becoming-business-critical/>

通过学习编码，首席执行官们可以更好地洞察影响传统行业的“颠覆性转变”。Scott Anthony 是一家成长咨询公司的管理合伙人，著有《[创新小黑皮书](http://hbr.org/product/the-little-black-book-of-innovation-how-it-works-h/an/10430-HBK-ENG)》一书，他分享了《[哈佛商业评论](https://hbr.org/2015/09/leading-a-digital-transformation-learn-to-code)》一书，这是亚洲领先银行星展银行(DBS Bank)集团高管兼集团技术和运营主管 Dave Gledhill 的例子。

银行是各种传统行业面临新一轮颠覆的一个很好的例子，除非它们能够利用数字化转型，建立一个新的微服务架构，使它们能够创造出客户需要的下一代产品。银行面临的混乱无处不在，以至于来自适应性实验室的詹姆斯·海科克写了“[拜拜银行？](https://www.adaptivelab.com/byebyebanks/)"形容影响深远。他将在伦敦 API Days 上发表演讲，讲述银行如何被金融科技初创公司“取代、削弱和去中介化”，这些公司能够提供一套更灵活的移动、支付、转账和贷款计算器产品。(披露:我也为 API Days London 写了几篇关于他们即将到来的会议的文章。)

安东尼指出，传统企业的高管“在遇到市场颠覆性转变时，并没有第一手经验。”例如，Gledhill 领导的团队正在快速迭代移动应用产品开发，但 Gledhill 正在“努力”理解构成应用体验的完整云和 web 架构流程。

通过学习编码，访问苹果的开发工具包，了解如何将 iBeacons 用于内部 GPS，并直接看到用户界面中的内容，Gledhill 表示，这种体验“使我能够在非常技术的层面上更好地为数字银行提供指导，因为我真的明白发生了什么。”

这种经历与近一年前新堆栈的创始人亚历克斯·威廉姆斯(Alex Williams)接触亚马逊网络服务的经历没有什么不同。当时，CohesiveFT CTO Chris Swan [坚持让我们的编辑动手](https://thenewstack.io/who-knows-how-to-use-aws/)。在随后的一篇关于他的经历的文章中，威廉姆斯还提到，他听说业务部门的经理们正在学习用 New Relic 的 Insights 平台制作应用程序:这是一项通常分配给开发人员的任务。

就像斯万坚称他的工作室不是为观众准备的一样，越来越多的数字公司要求管理岗位上的新员工至少具备一些编程技能，并了解支持其产品的技术体系。例如，Yelp 国际业务开发主管 Sheera Gendzel 分享了她的申请过程中如何使用 Yelp 的 API，以及如何构建某种新的应用程序。它不需要是 flash，但它需要展示对 API 调用以及 Yelp 通过其 API 提供的数据集和功能的理解。增长咨询公司 FABERNOVEL 的合伙人兼战略与发展副总裁 Cyril Vart 认为，管理层，尤其是高管层，需要提高他们的技术水平，甚至需要一些粗略的编码或架构设计技能，以便能够引领下一波公司增长。

毫无疑问，技术对于企业快速进入市场、应对新机遇、在全球范围内提供服务和产品以及保持全天候交付的能力至关重要。现在变得越来越重要的是，对于管理层来说，要真正理解“数字化转型”对他们的业务和行业意味着什么——要能够理解并抢先抓住数据科学、实时反馈、持续交付和以客户为中心的产品设计中的机会——在某种程度上需要编码知识和对分布式架构设计的全面了解。像 Gledhill 这样的高管已经在试图解决这个问题。
