# 应用程序可能很快就需要软件材料清单

> 原文：<https://thenewstack.io/a-software-bill-of-materials-could-be-a-requirement-for-applications-soon/>

Synopsys 去年调查的中，几乎一半的人认为媒体报道已经导致他们的组织对开源软件的使用进行了更严格的控制。那是在网络安全管理软件产品袭击之前，所以我们已经预计会看到更多的采用。现在，软件组合分析(SCA)的趋势可能会加速。

美国政府[正在考虑](https://www.reuters.com/article/us-usa-biden-cyber-exclusive/exclusive-software-vendors-would-have-to-disclose-breaches-to-u-s-government-users-under-new-order-draft-idUSKBN2BH37I)发布一项行政命令，要求供应商提供软件材料清单(SBOMs)。这将有效地推动现有的特定行业和[跨行业的努力](https://www.ntia.gov/SoftwareTransparency)来标准化我们跟踪软件依赖和其他元数据的方式。 [SPDX](https://github.com/spdx) 、 [SWID](https://csrc.nist.gov/projects/software-identification-swid/) 和 [CycloneDX](https://cyclonedx.org/) 是目前为止最常用的标准。

SCA 工具通常在扫描应用程序中的组件后创建自己的 SBOM。这很重要，但更重要的是如何利用这些库存。目前，大多数组织不得不手动更新旧版本，或者根据具体情况缓解安全问题。市场的未来不是能够创建列表，而是能够基于这些列表自动操作。
