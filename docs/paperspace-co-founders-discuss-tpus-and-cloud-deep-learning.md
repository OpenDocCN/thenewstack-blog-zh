# Paperspace 联合创始人讨论 TPUs 和云深度学习

> 原文：<https://thenewstack.io/paperspace-co-founders-discuss-tpus-and-cloud-deep-learning/>

[Paperspace 联合创始人讨论 TPUs 和云深度学习](https://thenewstack.simplecast.com/episodes/paperspace-co-founders-discuss-tpus-and-cloud-deep-learning)

如果你是一家机器学习公司，这是一个拥挤的市场。世界上的每个供应商都集成了一些新的人工智能驱动的服务，使得真正的投资回报率很难在流行词汇和功能蔓延的丛林中发现。 [Paperspace](https://www.paperspace.com) 希望通过提供 [Gradient](https://www.paperspace.com/gradient) ，一个易于管理的深度学习基础设施平台，让企业的旅程变得更加轻松。

在引擎盖下，Paperspace 不仅仅是一些人工智能初创公司:它为开发人员提供了对谷歌 Tensorflow 处理器的访问。Paperspace 目前在其深度学习平台中提供 TPU 访问和 GPU 访问。

仅此一点就值得与首席执行官 Dillon Erb 和首席技术官 Tom Sanfilippo 坐下来聊聊。这些联合创始人花时间讨论了使用 TPU 构建深度学习应用程序的感受。

“人们在谈论在一个特定的工作负载上同时运行 400 或 500 个 TPU。为了同时满足所有这些 TPU 的需求，您需要一个架构来运行一个虚拟机场来满足它们的需求。您需要定位所有数据，以便 TPU 可以按需轮询数据，这样它就永远不会有空队列要做。而设置这一切是相当复杂的，”Sanfilippo 说。

Erb 表示，深度学习已经成为企业的一个诱人的产品，但指出实际实施机器学习管道并产生投资回报对企业来说可能很困难。

“现实情况是，要将这种方法付诸实施，并将研究和学术领域的新发展转化为能够为企业带来实际投资回报的东西，仍然非常困难。这种困难存在于许多方面。我们花费最多时间的领域是基础设施管理和自动化，”Erb 说。

这两个人希望他们的深度学习平台也能让高性能计算(HPC)开发人员更容易处理他们的日常工作。Erb 和 Sanfilipo 指出，HPC 的学术世界可能比开发人员希望的要慢一些，他们的 Jupyter 笔记本电脑支持的管理和自动化服务可以加快传统研究社区对 GPU 和 TPU 计算的采用。
