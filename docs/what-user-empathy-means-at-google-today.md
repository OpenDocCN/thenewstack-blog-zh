# 用户同理心在今天的谷歌意味着什么

> 原文：<https://thenewstack.io/what-user-empathy-means-at-google-today/>

据说，当谈到同理心时，我们都可以做出改进。在软件工程中，需要同理心来创造最终用户能够容易理解的东西；构建一些你认为很棒的东西，但是仅仅因为你认为客户应该自己解决，就期望他们自己解决，这是不可接受的。搜索引擎巨头、云服务领导者和 Kubernetes 创造者谷歌意识到了这一点。

在最新一集的 New Stack Makers 播客中，New Stack 创始人兼出版商 Alex Williams 和 TNS 新闻编辑[Darryl Taft](https://twitter.com/darrylktaft)与谷歌移情工程项目经理 [Kim Bannerman](https://www.linkedin.com/in/kimbannerman) 和首席开发者倡导者 [Kelsey Hightower](https://twitter.com/kelseyhightower) 、[谷歌云平台(GCP)](https://cloud.google.com/) 一起讨论谷歌的客户移情项目和最终用户满意度。

[用户同理心在今天的谷歌意味着什么](https://thenewstack.simplecast.com/episodes/what-user-empathy-means-at-google-today)

谷歌的客户同理心项目将成为明年所有“新手”(刚加入谷歌的人)工程入职流程的一部分。“如果有人开始在谷歌工作，他们将通过一个移情工程会议，以及学习它是什么和它不是什么，模式和非模式——这对我们来说是巨大的，”Bannerman 说。“这是我们真正想全面做的事情，因为你不可能真正为你的客户提供伟大的工程和伟大的产品，除非你能让构建这些产品的人理解他们，真的，这一切都是为了站在他们的立场上。”

班纳曼说，同理心不仅仅关乎情商。“很多人谈论情商，但他们并不真正知道它是什么，甚至不知道发挥作用的移情类型。你可以有认知同理心，这是在智力水平上考虑问题，也可以有同情同理心，这是对某人的感受，并在需要时采取支持行动，”Bannerman 说。“但是，除非你亲身感受到别人的感受，否则你不会有完整的移情周期，除非你经历了所有三个移情环节。”

Hightower 指出，DevOps 本身很可能是同理心的产物。就在 Docker 之前，有一股推动基础设施自动化的力量，“在许多方面有错误的抽象，或者没有抽象，”Hightower 说。“你让人们尝试自动化一切，而容器社区说你在自动化错误的东西。Hightower 说，Docker 没有把 Ruby、Python、Java 和所有编程语言作为独立的问题来解决，而是采取了一种整体的方法，并说，“让我们把应用程序的概念抽象成一个包，一个通用的包。”。“我认为 ops 推动了配置管理世界的发展，但我认为开发人员推动了容器化和分布式系统运动。当这两个世界融合在一起时，人们并不清楚如何将它们结合在一起。”

Hightower 继续说，Kubernetes 可以被视为弥合这一差距的一种方式，因为它“层”在这两个概念之上。“Kubernetes 将这种配置语言带到了这个容器领域。因此，尽管大多数人将 Kubernetes 称为容器编排工具，但它将基础设施的声明性方法引入了容器世界，”Hightower 说。“我认为这是我们开始看到这两个社区开始融合的时候，但这为移情留下了很大的空间，以确保两个社区都了解各自的来源，以便我们获得所有的价值。”
