# 预测 API 正在推动机器学习

> 原文：<https://thenewstack.io/predictive-apis-driving-machine-learning/>

机器学习越来越多地使公司能够通过将大数据转化为有洞察力和可操作的信息来获得战略优势。但是他们如何进入这个人工智能的新世界呢？

预测性 APIs 应用编程接口]提供了对机器学习的轻松访问，正在成为机器学习(ML)和人工智能(AI)整个世界的关键驱动因素。预测 API 提供了决定在云中、内部或两者兼而有之地托管机器学习的灵活性，让开发人员可以自由地使用他们想要的语言和工具。

“预测 API 公开了机器学习能力，因此它既可以是一个提供预测模型访问权限的 API，也可以公开学习新模型和从数据中创建新模型的能力，”路易斯·多拉德说，他拥有机器学习博士学位，是 [*引导机器学习*](http://www.louisdorard.com/machine-learning-book/) 的作者，也是一系列预测 API 会议的创始人，包括本周在西班牙巴伦西亚举行的 PAPIs.io Connect。

多拉德说:“有机器学习 API，你可以发送数据集——例如，一列是推文，另一列是正面或负面或中性的——然后他们(通过)API 发回数据，然后你可以重复使用数据模型。”

预测 API 与像 Amazon 一样将数据放入云中联系最紧密，但是预测 API 也可以在内部轻松使用。

例如， [Upwork](https://www.upwork.com/) 自由职业者平台已经创建了自己的预测 API 供内部使用，可以预测自由职业者的可用性。

“所以当你寻找某人做某项工作时，他们在返回搜索结果时会使用预测。他们使用数据科学团队提供的 API。对最终用户的好处是，如果你在寻找自由职业者来完成你的工作，如果他们不回复或者很忙，你会很沮丧。它只是让平台工作得更好，因为人们浪费的时间更少了，”多拉德说。

多拉德说，如果公司没有建立内部 ML 系统的管理知识，公司可以找到许多服务工具，如亚马逊机器学习、微软 Azure 机器学习和 BigML。

## API 和 AI

预测 API 的一个主要原因是，数据科学家使用的语言不是最常用的，坦率地说，他们需要高薪的开发人员。预测 API 允许初创公司和较小的公司利用机器学习，而不必雇用内部数据科学家。

但是还有其他的优势。

“预测 API 的好处是，如果你想使用机器学习，没有什么需要安装的，你只需要调用一个 API，”多拉德说。

然后还有一个问题，就是制作一个预测模型，它可以在某人的笔记本电脑上工作，也可以在一个生产就绪的系统上工作。

生产环境通常使用不同的语言或技术，尤其是对于 Web 应用程序，而数据科学家则使用不同的语言或技术。API 可以充当从 Scala 到 Ruby 的翻译器，消除了一个非常耗时的任务。除此之外，实现生产系统的人通常不是机器学习专家，但是能够自如地使用 API 集成。

## 融入工作场所

下一步是找到合适的工具来实现它。在这里，您需要决定是完全在内部开发还是在云中开发。这完全取决于你需要多少预测模型创建过程的所有权，以及你的商业机器学习有多核心。

多拉德建议你问问自己:你需要拥有模型创建过程吗？有些人想拥有整个基础设施。在这种情况下，他们可以使用许多开源机器学习框架中的任何一个。然而，对于那些喜欢将人工智能作为服务的人来说，仍然很少有开源的预测 API。其中包括去年被 Salesforce 收购的 PredictionIO 和 Seldon。

最后，多拉德说，“如果你想把事情做好，你确实需要让首席执行官或高管层的人参与进来，就用例达成一致。”
