# git 15 岁:现在怎么办？

> 原文：<https://thenewstack.io/git-is-15-years-old-what-now/>

15 年前，也就是 2005 年 4 月 7 日，Linus Torvalds 首次发布了他的 Git 版本控制软件，试图在 Linux 内核开发中培养一种更具创造性的精神。从那时起，Git 在软件开发中的作用已经远远超出了它作为版本控制系统和软件仓库的根源。它已经成为当今世界分布式团队和开源开发者开发软件的基石。

在这个[新栈制造者](/podcasts/makers)播客中，我们与三位 Git 思想领袖谈论了 Git 的根源、它的当前背景和未来。我们了解到，尽管 Git 目前取得了成功，但它的未来并不确定。

本期节目的嘉宾是:

对于许多人来说，Git 提供的可能性是令人兴奋的，无论是在个人层面还是在宏观层面，当多方必须在一个项目上合作时，尤其是对于 CI/CD。您可以使用 Git 上传个人喜爱的项目，比如您是否想要共享一个简单的代码样本，或者只是与软件不一定相关的文档。对于一个大型企业来说，开发团队可以同时协作开发应用程序，无论他们是分散在世界各地，还是仅仅被小隔间隔开。

[Git 15 岁了:现在怎么样？](https://thenewstack.simplecast.com/episodes/git-is-15-years-old-what-now)

然而，很容易忘记为什么 Git 和版本控制会如此普遍地被采用。追溯到 15 年前，Davis 描述了 Git 超越源代码控制系统的锁定和信任概念的发展。

“安全很重要，但我们认为实现安全的方法是通过严格的控制。现在，我们已经打开了我们的思维，说，'好吧，实际上，这是两个东西吗？是吗？它们是互斥的吗？“我真的可以放松控制而仍然安全吗？””戴维斯说我想我们已经证明了答案是肯定的。但是我们不能用旧的源代码控制系统做到这一点，因为没有原语，比如…将完整的状态存储在一个特定的实例中，如果你愿意，可以说是快照。这些类型的事情让我们有一个安全，而没有这种非常严厉的控制。"

分支也是帮助解释 Git 被大量采用的一个关键功能。沃纳解释了分支的重要性，他将分支与谷歌文档现在的工作方式联系起来。您可以看到哪些用户以前在不同版本中对文档进行了更改。在不知道以前版本的历史的情况下对文档进行修改的前景是可怕的。“分支和 Git 对于 15 年前的我们来说非常非常重要，廉价、简单的分支彻底改变了人们在软件上合作的方式，”华纳说。“我们今天认为这是理所当然的，但 15 年前，没有这样的事情。”

然而，Git 提供的所有优点及其在开发社区中的流行并不一定意味着它一定会满足未来所有的计算和 it 需求。无状态数据管理就是一个很好的例子。

“版本控制数据仍然是非常非常困难的事情之一。我认为现在随着 ML 和 AI 的出现，它变得越来越重要，”Sijbrandij 说。“我们用来训练它的数据是什么？什么是输入和输出数据，在其上运行的模型的输出数据和版本是什么？”他说。“我看到了很多工具，但我认为在文件级别也有一些基本的东西需要改变…我们需要更好的数据版本控制，因为数据和代码一样重要。”

事实上，Warner 说 Git 环境中的文件系统必须改变，特别是它的抽象，“以及我们对它的看法。这不仅仅是数据本身的版本问题，而是它的应用方式，尤其是大规模应用将成为一个真正的问题，”华纳说。“所以，这些概念，如果可以应用，我认为会很有趣。我希望有人能推进它，因为这仍然是这个行业的一个痛点。”
