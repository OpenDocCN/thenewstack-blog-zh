# 开源的社会模式

> 原文：<https://thenewstack.io/the-social-model-of-open-source/>

在这一集的开源峰会上，思科[开源项目办公室](https://www.cisco.com)的开源技术负责人 [Julia Ferraioli](https://www.linkedin.com/in/juliaferraioli/) 向新堆栈讲述了一些定义什么是“开源”什么不是“开源”的替代方法

当有人说，好吧，那是“技术上的”开源，这通常是对一个满足开源的法律标准，但不遵循开源精神的项目的刻薄。Ferraioli 并不认为像 Kubernetes 或 Linux 这样的“经典”开源项目是开源的唯一有效模式。她给出了一个研究项目的例子——代码可能是开源的，以便其他人可以看到代码并自己复制结果。然而，为了保持研究的有效性，它不能接受任何贡献。

[开源的社交模式](https://thenewstack.simplecast.com/episodes/the-social-model-of-open-source)

“它的开源性不亚于其他产品，”费拉约利谈到这个假想的研究项目时说。“如果你有目的地分解事物，这并不总是你试图建立一个健壮的社区。”Ferraioli 说，开源的社会模型是关于理解开源的不同用例，以及提供一个框架，根据项目的动机来决定什么是合适的成功标准。如果你只是为了好玩和朋友一起做一个项目，那么量化乐趣并不容易。
