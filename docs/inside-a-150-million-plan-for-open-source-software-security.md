> 原文：<https://thenewstack.io/inside-a-150-million-plan-for-open-source-software-security/>

德克萨斯州奥斯汀。—每个人都使用开源软件——越来越明显的是[软件的安全性没有得到足够的重视。在德克萨斯州奥斯丁的基金会开源峰会上，Linux 基金会和 Snyk 发布了一份由 T2 发布的调查报告。这个月，41%的组织说他们对他们使用的开源软件的安全性](https://thenewstack.io/log4shell-we-are-in-so-much-trouble/)[没有信心。](/category/security/)

在奥斯汀的活动中，新堆栈的制造者播客与开源安全基金会(OpenSSF)总经理[布莱恩·贝伦多夫](https://www.linkedin.com/in/brianbehlendorf/)坐在一起，谈论从多个角度解决这个问题的新计划。他接受了《新书库》特稿编辑希瑟·乔斯林的采访。

Behlendorf 自 10 月以来一直领导 OpenSSF，并在电子前沿基金会和 Mozilla 基金会的董事会任职，他将 2021 年底 Log4j 漏洞的发现和其他最近的安全“地震”列为关键的转折点。“我认为，今年软件行业不仅真正意识到了地震正在发生的事实，”他说，“而且意识到从地震中恢复的成本越来越高。”

开源安全动员计划源于五月份的一次开源安全峰会。根据 OpenSSF 和 Linux 基金会发布的报告，它确定了 10 个需要关注的领域:

Behlendorf 告诉《我们的制造者》的观众，在最初的两年里，这些倡议的价格标签预计将达到 1.5 亿美元。

该计划是由白宫对正在进行的改善开源软件安全性的各种倡议的询问引发的——它们的成本，以及解决方案建立者心目中的时间框架。“我们无法真正回答这个问题，除非我们能够说，如果我们要投资，需要什么？”贝伦多夫说。“因为大多数时候，我们坐在那里，等待人们出现，并期待最好的结果。”

他说，最终价格远低于他的预期。他说，OpenSSF 内部的各种成员组织已经承诺提供资金。“150 英镑只是一个估计。这些计划仍在完善中，”贝伦多夫说。但通过说明具体步骤及其成本，他感到有信心的是，到了兑现这些承诺的时候，感兴趣的各方也会感到有信心。
