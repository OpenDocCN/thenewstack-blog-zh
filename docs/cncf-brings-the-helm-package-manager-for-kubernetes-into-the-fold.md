# CNCF 将 Kubernetes 的头盔包装经理纳入麾下

> 原文：<https://thenewstack.io/cncf-brings-the-helm-package-manager-for-kubernetes-into-the-fold/>

[云原生计算基金会](https://www.cncf.io/)继续扩展其开源工具栈，用于运行云原生计算操作。周四，CNCF [技术监督委员会](https://www.cncf.io/people/technical-oversight-committee/) (TOC)投票接受[掌舵](https://helm.sh/)包管理器作为孵化级托管项目。

对于许多人来说，Helm 实际上是运行在 CNCF Kubernetes 开源容器编排引擎上的应用程序的包管理者。根据最近的 CNCF 调查，大约 64%的 Kubernetes 用户使用 Helm。

“Helm 通过使配置可重用来满足用户在 Kubernetes 上部署应用的共同需求，”TOC 代表和项目发起人、谷歌首席工程师、Kubernetes SIG 架构联席主席和指导委员会成员[布莱安·葛兰特](https://www.linkedin.com/in/brian-grant-3317a6/)在 CNCF 博客文章[中宣布接受](https://www.cncf.io/blog/2018/06/01/cncf-to-host-helm/)。

软件[由](https://thenewstack.io/deis-aims-extend-kubernetes-platform/) [Matt Butcher](https://github.com/technosophos) 和 [Jason Hansen](https://github.com/slack) 于 2015 年创建，一家名为 [Deis](https://deis.com/about/) 的公司在发布后不久进行了重组，以支持该软件并构建额外的工具。[微软](https://azure.microsoft.com/en-us/?v=17.14)在 2017 年收购了 Deis 。

与操作系统包管理器一样，Helm 旨在简化在平台上设置和运行应用程序的过程，在本例中，平台是 Kubernetes。“在构建 Helm 的过程中，我们开始构建一个工具，作为对 Kubernetes 的一个冲击——一个经验丰富的开发人员不仅会使用，还会做出贡献的工具，”Butcher 在 CNCF 的博客帖子中说。

该软件允许用户以 Kubernetes 图表的形式共享应用程序。在 Helm 的管理下，应用程序本身可以跨不同的 Kubernetes 部署进行一致的设置。该软件还提供了管理单个 Kubernetes 清单或配置文件的方法。

Helm 加入了越来越多的 CNCF 项目，这些项目都旨在以厂商中立的方式简化在云服务上运行工作负载的过程。其他项目包括 Prometheus、OpenTracing、Fluentd、Linkerd、gRPC、CoreDNS、containerd、rkt、CNI、Envoy、Jaeger、公证人、TUF、Vitess 和 NATS。

云本地计算基金会和 T2 微软是新堆栈的赞助商。
