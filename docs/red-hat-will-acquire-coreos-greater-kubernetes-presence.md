# 红帽将收购 CoreOS 以提升 Kubernetes 的地位

> 原文：<https://thenewstack.io/red-hat-will-acquire-coreos-greater-kubernetes-presence/>

为了进一步巩固企业 Linux 市场，Red Hat 已经签署了一项协议，收购 CoreOS Inc。它是以前同名的面向容器的小型 Linux 发行版的制造商。CoreOS Inc .还提供 [Quay](https://coreos.com/quay-enterprise/) container registry，一个 [Kubernetes](https://kubernetes.io/) 发行版，被称为“构造”,并维护着 [etcd](https://coreos.com/etcd/) key-value store，它是 Kubernetes 开源容器编排软件的核心。

红帽将为 CoreOS 支付约 2.5 亿美元，预计将在本月底完成交易。

此举将把所有 130 名 CoreOS 员工纳入红帽帐下。这一举动可能会让 Linux 观察人士感到惊讶，因为两家公司都在努力提供精简的、以容器为中心的 Linux 发行版。然而，真实的故事可能与 Kubernetes 有更多的关系:Red Hat 将[constructive](https://coreos.com/tectonic/)和 Kubernetes 放在了所有关于拟议收购的原因和问题列表的最前面。在宣布此次收购的新闻稿中，Red Hat 特别指出 CoreOS 是 Kubernetes 和容器原生解决方案的创新者。

CoreOS 公司的成立是为了追求一个在虚拟机和容器中使用的更小的 Linux 发行版的愿景。自 2013 年成立以来，该公司一直在推动容器管理，并实现向微服务的过渡。与此同时，该公司还创建了自己的容器引擎，名为 [rkt](https://coreos.com/rkt/) ，捐赠给了[云本地计算基金会](https://www.cncf.io/kubecon-cloudnativecon-events/) (CNCF)，并且是开放容器倡议(OCI)标准的基本组成部分。

今天，对微型 Linux 的最初看法已经大部分被 Alpine Linux 所包含，而 CoreOS 已经转向将重点放在整个大地构造和 Kubernetes 上。

在 2015 年的亚马逊 re:Invent 大会上，CoreOS 首席执行官 [Alex Polvi](https://www.linkedin.com/in/polvi/) 表示，“我们希望确保企业能够通过在 [AWS](https://aws.amazon.com/) 上使用 Kubernetes 来扩展他们在容器中的应用程序。Kubernetes、CoreOS 和 AWS 的结合是帮助公司达到超大规模的又一步。”

CoreOS 最终将其操作系统重命名为[容器 Linux](https://coreos.com/os/docs/latest/) 。“ Container Linux，”Polvi 在 2016 年说，“具有自动管理、更新和补丁的能力。我们称之为自动驾驶基础设施；这个概念就是容器 Linux 总是与最新的上游特性保持同步。”
