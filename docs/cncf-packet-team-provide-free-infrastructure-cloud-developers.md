# CNCF，Packet 为云开发者提供免费基础设施

> 原文：<https://thenewstack.io/cncf-packet-team-provide-free-infrastructure-cloud-developers/>

为开发者提供托管裸机服务器的 Packet T1 和 T2 云计算基金会 T3 联手为推进云计算项目提供免费的基础设施。

这个名为 [CNCF 社区基础设施实验室](http://www.prnewswire.com/news-releases/cloud-native-computing-foundation-expands-developer-access-to-on-demand-hardware-with-new-infrastructure-lab-300501118.html)的实验室，实际上是 Packet 为其现有基础设施每月捐赠 25000 美元的资源。

该基础架构包括分布在超过 15 个全球位置的高性能计算和存储节点，这些位置包括纽约市、硅谷、阿姆斯特丹和东京。

开发人员可以从 x86 和 ARMv8 服务器的五种配置中进行选择。每个裸机配置都是 API 驱动的、云原生友好的，部署时间不到 10 分钟。

默认情况下，Packet 不会强制实施多租户、虚拟化或覆盖网络，允许用户选择自己的工具。它还包括电信级功能，如宣布您自己的 IP 空间的能力、BGP/Anycast 和本机 IPv6 支持，这些功能支持跨多个位置和延迟进行网络测试。

“这是为了各种各样的开发原因，为了测试——规模测试，性能基准测试，”Packet 的高级参与副总裁[雅各布·史密斯](https://www.linkedin.com/in/jacobsmith79/)说。“Packet 可以为没有虚拟化层的开发人员提供自动化基础设施，当您开发越来越深入堆栈的技术时，这真的很方便。”

CNCF 执行董事丹·科恩说，总部位于纽约的 Packet 将管理基础设施，CNCF 将分配资源，Packet 将以信用的形式捐赠。

CNCF 将有一个开放、透明的过程，任何成员项目、成员公司或开源用户都可以申请积分。Kohn 说，该组织将对他们进行排名，并对自己的项目和成员进行优先排序，但“我们确实希望将资源留给其他愿意做有趣事情的人”。

“我们希望在这个基础设施上运行的任何东西都是开源的，然后我们希望他们能就此写一篇博文。但对于每一项云原生技术来说，并不一定是“哦，这是一次美妙的体验”。他们还可以写一篇博文，描述为什么会感到沮丧或他们遇到的问题，”他说。

史密斯说，Packet 正在与生态系统合作伙伴合作，为云带来新的基础设施原语。

“因此，我们能够让 CNCF 用户尽早接触到新兴硬件，”他说。“我们正在寻求一定的规模，这对于开发人员来说可能很难实现—如果您希望部署 100 或 200 个裸机服务器节点，这将非常昂贵和困难。”

Kohn 说，一些成员、项目和团体，包括 CoreDNS(服务发现)、[Prometheus(监控)和持续集成工作组，已经在使用分组基础设施。一项名为](https://thenewstack.io/history-prometheus-julius-volz/)[交叉云](https://github.com/cncf/cross-cloud/)的跨项目集成服务也已经开始使用。它还处于早期阶段，但将于下周向 CNCF 技术监督委员会报告。

科恩谈到他希望利用这些资源的项目时说:“CNCF 正在构建这一系列技术——如何让它们协同工作，如何在多个不同的云上成功部署——这些类型的项目是这一基础设施的非常实际的应用，但我们完全欢迎其他想法。”。

“开发人员只是想过来问，‘当你以这种高带宽在 15 个不同的地方联网时，[CNI](https://thenewstack.io/standardizing-container-networking-cni/)[联网]工作得如何？’或者选择我们的任何东西——gRPC【远程过程调用】在 500 个节点上都互相通信。我们对支持不同类型的测试持开放态度。"

史密斯提到了 Packet 去年与 Docker 在 LinuxKit 上的工作，作为推进技术需要更多资源的例子。

“他们真的需要接触裸机，他们需要接触多种架构来开发产品，因为他们真的开始尝试为容器构建操作系统。他们不想在他们之间有任何层；他们想成为这一层。因此，这是一个很好的例子，与获得他们拥有的其他资源相比，获得金属真的很有价值，”他说。
