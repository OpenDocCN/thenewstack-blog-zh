# 微软 Visual Studio 的新 GitHub 扩展已经开源

> 原文：<https://thenewstack.io/new-github-extension-microsoft-visual-studio-open-sourced/>

我们开始尝试写简报——强调与开发人员和管理大规模应用程序开发的人员相关的一些新闻或主题的短文。这将是大约 400 字或更少，将触及突发新闻或只是有趣的东西，我们想强调。新的堆栈往往遵循新闻周期。我们并不关注突发新闻，但保持同步和提供背景对我们来说很重要。我们提供如何操作的帖子和复杂主题的解释。这让我们能够更专注于我们所涵盖的内容。简报使我们能够就我们认为相关但无需深入解释就能涵盖的主题发表见解、观点和分析。新的 Stack 将很快开始出版一系列关于 Docker 和容器生态系统的电子书。这将加深我们对这一新运动带来的快速发展的报道。但我们确实需要取得平衡，简报是我们使我们讨论和涵盖的内容多样化的一种方式。

[GitHub 和微软今天宣布](https://github.com/blog/2038-github-extension-for-visual-studio-is-open-source)他们将开源为 Visual Studio 2015 构建的 GitHub 扩展。

自今年 4 月发布以来，微软的 Visual Studio 2015 已经包含了 GitHub 和 GitHub Enterprise 的预装[扩展。该扩展允许 Visual Studio 开发人员在 Visual Studio IDE 中搜索和即时克隆 GitHub 存储库，并相应地将 Visual Studio 项目直接发布到 GitHub 上。](https://visualstudio.github.com/)

今天，GitHub 宣布该扩展已经在 MIT 许可条款下开源。

此举允许 Visual Studio 开发人员社区更积极地参与 GitHub 扩展工具的开发，或者直接提出问题。

为了鼓励开源参与，GitHub 的 [Brandon Keepers](https://github.com/bkeepers) 刚刚发布了[贡献说明](https://github.com/github/VisualStudio/blob/master/CONTRIBUTING.md)，肯定了开放行为准则，并提出了一种开发者贡献的方式。特定的限制主要适用于 Visual Studio 的团队资源管理器接口，该接口必须保持很少的依赖性，以便扩展的这一部分保持其轻量级和速度。

这一最新版本再次确认了微软对基于其平台开发开源工具的重要性的认可。去年 10 月，[Saugatuck Technologies 为微软进行的一项调查](https://docs.microsoft.com/en-us/archive/blogs/technet/devops/)发现，在接受采访的 300 多名开发和 IT 运营专业人员中，在 Windows 环境中工作的 52%的人预计到 2016 年将使用一个单一的 DevOps 平台，该平台“支持使用最佳、松散耦合的 DevOps 工具”。

微软正在做的工作是让 GitHub Enterprise 用户能够从 Windows dev 生态系统中访问回购，这是这一工作的核心，并展示了从锁定供应商平台到让开发人员能够将开源工具和项目集成到他们的工作流中的转变。
