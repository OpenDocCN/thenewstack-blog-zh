# CoreOS CEO:在 Kubernetes 上重新搭建平台是值得努力的

> 原文：<https://thenewstack.io/coreos-alex-polvi-orchestration-needs-winner/>

“重新平台化”是上周四在旧金山举行的英特尔云日会议上的一个词(如果确实是一个词的话)，在那里，它被用来指组织应该如何通过使用容器化将工作负载迁移到云。这是一个大问题。

CoreOS 首席执行官 Alex Polvi 在上周旧金山英特尔云日活动期间接受新堆栈制造商播客采访时承认:“过渡是企业讨厌的事情。

“这就像是一件附加的事情，就像制作一棵盆景树——这件事你要花很长时间在 T2 完成，”波尔维继续说道。尽管如此，“我们相信向云的过渡是一个值得公司关注的平台迁移，以及我们如何做和如何做的过程。”

[CoreOS 首席执行官 Alex Polvi 谈将企业迁移到云](https://thenewstack.simplecast.com/episodes/coreos-ceo-alex-polvi-on-moving-the-enterprise-to-the-cloud)

在某种程度上，该活动是关于英特尔如何试图让企业更容易地进行平台迁移。

[英特尔牵头宣布了英特尔、CoreOS 和 Mirantis 之间的联合努力，以生产所谓的通用资源调度程序。虽然该定义的某些方面还很模糊，但很明显，该合作伙伴关系已经接受了虚拟机管理程序驱动的工作负载(如 VMware 虚拟机)不会消失的概念，当然不会很快消失，也可能永远不会消失。容器和虚拟机将共存，英特尔认为自动化整个数据中心的管理将需要开放(如果不是完全开源)的流程编排方法。](https://thenewstack.io/intel-coalesce-coreos-mirantis-universal-resource-scheduler/)

但是通过在 CoreOS 上下大赌注，英特尔正在投资对 Kubernetes 的信心，CoreOS 将 Kubernetes 重新包装成商业上的构造。Polvi 说，他认为信任投票对于在编排领域围绕单一力量建立一种行业驱动的轴心是必要的。

Polvi 说:“你会看到现在 Kubernetes 的焦点发生了巨大的变化，即使是大公司——英特尔，进来说，这是我们选择继续前进的方向。市场真正需要的是在这个领域成为赢家。你需要 Linux 你需要一个大家都认同的，我们可以合作的人。”

CoreOS、Docker 和 Intel 是新堆栈的赞助商。
