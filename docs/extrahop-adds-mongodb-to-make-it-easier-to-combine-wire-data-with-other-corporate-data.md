# ExtraHop 现在将有线数据传输到 MongoDB，但这只是故事的一部分

> 原文：<https://thenewstack.io/extrahop-adds-mongodb-to-make-it-easier-to-combine-wire-data-with-other-corporate-data/>

ExtraHop 今天推出了一项新功能，允许用户将他们 IT 环境中几乎任何活动的数据传输到 MongoDB。其结果是，用户可以将 IT 数据与其他类型的业务数据结合起来，ExtraHop 表示，这种方式可以帮助他们更快地解决问题或获得新的业务洞察力。

[ExtraHop](http://www.extrahop.com/) 是一个采用独特方法的 IT 运营分析平台。它的设备不在用户的交换机上，而是对通过它的所有数据进行复制。这种设置意味着 ExtraHop 正在收集大多数组件之间的通信信息，如防火墙、负载平衡器、Web 服务器和 DNS 服务器。ExtraHop 营销和业务发展高级副总裁 Erik Giesa 说，这样做的一个好处是，客户不必每次在网络中添加新组件时都改变设置。

这与 Splunk 的方法不同，用户可以指向应用服务器、网络服务器、数据库、虚拟机和电信设备等组件，从每个组件收集数据流。

使用 ExtraHop 调用 Open Data Stream 的新功能，用户可以从 ExtraHop 数据库收集关于其 IT 环境的信息，并将特定数据传输到 MongoDB 数据库。从那里，用户可以将它与其他公司数据结合起来，如财务信息和机器数据。他们还可以使用 Tableau 或 Chartio 等工具来实现可视化。

理论上，他们还可以使用 BI 工具——传统工具或基于云的工具——来尝试从广泛的数据集中获取情报，现在包括来自 ExtraHop 的有线数据。

Giesa 说，IT 运营数据通常存储在专有数据库中，很难与其他公司数据结合。

ExtraHop 的设想有一些缺点。例如，用户必须能够提前预测他们想要在 MongoDB 数据库中保存什么样的数据。企业管理协会的分析师吉姆·弗雷说:“实时网络数据可以相当于消防水管，以每秒几千兆比特来衡量。”他说，ExtraHop 建议用户首先关注他们知道自己需要的数据。“这样，它的客户可以将新的开放数据流功能集中在最重要/最有价值的指标上，”他说。“我相信，随着时间的推移，他们将不得不适应那些一直想要所有数据的人，因为许多人不知道他们想要寻找或查看什么，无论是在未来还是现在。”

ExtraHop 并不根据用户通过其设备传输的数据量来收费。因此，Giesa 认为保留大量数据没有太大的负面影响，特别是因为用户将在一个开源的 MongoDB 数据库中这样做。然而，存储可以通过 ExtraHop 收集的大量数据涉及到硬件和管理成本，因此企业不想保留所有数据。
