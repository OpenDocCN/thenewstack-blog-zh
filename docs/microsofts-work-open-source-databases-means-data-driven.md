# 微软与开源数据库的合作以及如何实现数据驱动

> 原文：<https://thenewstack.io/microsofts-work-open-source-databases-means-data-driven/>

[你真的是数据驱动吗？](https://thenewstack.simplecast.com/episodes/are-you-really-data-driven)

Rohan Kumar 是微软数据库系统事业部的总经理。他涵盖了 SQL Server、RBDMS 和所有用于关系数据库的 Azure PaaS 属性。这包括 Azure SQL 数据库和 Azure SQL 数据仓库。它还扩展到基于开源的数据库，包括 MySQL 的 Azure 数据库、Postgres 的 Azure 数据库和 MariaDB 的 Azure 数据库。他还赞助了 Databricks 和微软之间的合作。

在 Microsoft Connect 录制的采访中，Kumar 和 TNS 记者 Swapnil Bhartiya 讨论了开源的重要性，以及微软如何在不同层次上与不同社区合作，特别是在数据库领域。该公司加入了 MariaDB 基金会，并与 Postgres 和 MySQL 等开源数据库社区密切合作。“这不仅仅是把代码拿走，做我们自己的事情，不做回报。我们绝对希望保持接触，”库马尔说。他们还谈到了微软开源他们自己的数据库产品的可能性

在稍后的播客中，他们谈到了数据驱动世界的含义。环顾四周——从脸书到特斯拉汽车，一切都是由数据驱动的。越来越多的公司都在宣称自己是数据驱动的，但是做一个数据驱动的公司意味着什么呢？

“很多时候，公司认为他们正在成为数据驱动的，但他们只是在看数据来支持他们的直觉。库马尔说:“它们并不是真正由数据驱动的，首先要认识到你正在落入那个陷阱，然后承认并改变它。数据驱动并不意味着要有一堆工具来做这件事。而是真正拥抱它，即使它与你的直觉相反。”

最后，他们谈到了微软在塞特亚·纳德拉的领导下所经历的转变。在 YouTube 上观看完整的采访。
