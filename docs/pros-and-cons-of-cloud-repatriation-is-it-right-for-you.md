# 云遣返的利与弊:它适合你吗？

> 原文：<https://thenewstack.io/pros-and-cons-of-cloud-repatriation-is-it-right-for-you/>

云遣返是现在的热门话题。将工作负载从云中转移出来的想法可能会让人觉得有违初衷，但特别是对于那些在业务早期就迁移到云的人来说，这种想法越来越受欢迎。如果您跑向云，以获得超快的上市时间和大型基础设施公司的支持，您可能会问，留在云中的长期好处是什么？所以，对于那些考虑这种变化的人，让我们来探讨遣返的五个理由，还有，不要这么快，不遣返的五个理由。

## 何时遣返是正确的选择(或不是)

### 1.性能已经成为一个问题

网络的复杂性不容忽视，如果您希望改善特定位置的最终用户的体验或本地应用程序的性能，本地解决方案可能会让您如愿以偿。例如，工厂或医院中的员工应用程序，或者体育场或竞技场等单一场所的访问者所使用的产品。

**然而……**

大多数时候，云可以提高您的性能，根据需求提供对无限资源的访问，非常适合意外的流量或服务高峰。您还可以使用地理区域的智能最佳实践，这可能会对延迟或带宽产生很大影响。

### 2.金融已经失去控制

如果您迁移到云上进行现收现付定价，您可能会对每月发票的螺旋上升的成本感到有点不知所措。任何有云经验的人都知道，按需资源会使成本飙升，而预订实例和其他基于承诺的购买是控制成本的唯一方法。还记得人们对避免资本支出而增加运营支出有多兴奋吗？对许多人来说，这是一个“回到资本支出，一切都被原谅了”的例子

**然而……**

自动化和人工智能正在幕后掀起管理云成本的浪潮，消除了对资源支出进行手动治理的需要。此外，我们一开始就不喜欢资本支出也是有原因的，它不会突然变得便宜。

### 3.您会注意到基于 IT 的孤岛和风险

尤其是在多云时代，很容易发现某些团队、部门和项目陷入了自己的技术，存在于 AWS 或 Azure 形状的泡沫中。遣返可能会迫使员工在跨越单一业务领域的简化 IT 环境中从事通用技术工作。这也有助于治理和合规性，减少对影子 IT 或新技术在难以管理的办公室角落涌现的担忧。

**然而……**

不要忘记，云提供了协作和交流工具，这可能是你工作文化的一个重要部分。尽量不要把婴儿和洗澡水一起倒掉！围绕如何使用公共云创建治理和云政策可能会带来同样的好处，而无需额外的遣返工作。

### 4.供应商锁定感觉像一个套索

云的复杂性让许多企业感觉他们被捆绑在一个特定的云供应商身上，并且随着成千上万的工作负载在成千上万的虚拟机上运行，这变得越来越困难。如果所有的安全性和合规性配置、访问管理、通信路径和网络使您无法转移到其他地方，您可能只是想要回一些控制权。

**然而……**

没有什么比云更快节奏的了。终极敏捷性意味着能够在几秒钟内部署新的工作负载，毫无疑问，内部构建是一种长期关系，比您在大学时的关系更为重要。考虑考虑 Kubernetes 或另一个可以使您的云更加敏捷的可移植环境，而不是自己构建一个环境。

### 5.安全性是一项重要的优势

还记得在手机有自己的连接之前，你妈妈关掉了无线网络吗？#艰难时期。这就是您可以在内部实现的目标:一个完全断开连接的环境，这样您的客户数据或敏感文件就可以完全不受网络影响。非常适合备份您最重要的合规性相关数据，也非常适合帮助您晚上高枕无忧。

**然而……**

你是詹姆斯·邦德吗？除非你把已知世界的安全扛在肩上，否则云是相当安全的！正如 [Gartner 喜欢提醒我们的那样](https://www.gartner.com/smarterwithgartner/is-the-cloud-secure)，云上 99%的安全故障都是客户的错，而不是云的错。人们也可能在内部搞砸。

## 嘿！拥有一切，为什么不呢？

我们说为什么不鱼和熊掌兼得呢？

有些工作负载您可能希望迁移到私有云或本地云，这可能是因为它们的特殊安全性或合规性要求，您正在运行的应用程序需要性能提升，或者是因为成本不值得挤压。

然而，在其他情况下，尤其是如果你没有一个超人的愿望清单，这可能只是一个用一些最佳实践优化你所拥有的情况。
