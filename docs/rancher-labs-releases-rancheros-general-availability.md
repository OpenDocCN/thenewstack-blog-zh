# Rancher Labs 向大众发布 RancherOS

> 原文：<https://thenewstack.io/rancher-labs-releases-rancheros-general-availability/>

Rancher Labs 将为那些寻求建造集装箱的人提供一种新武器。总部位于库比蒂诺的公司正式推出了 RancherOS，这是一个精简的操作系统，避免了许多图书馆和服务的需求。

Rancher 本周宣布了该软件的全面上市，声称它通过减少可能被网络罪犯利用的组件数量，缩短了启动时间，同时增强了安全性。

鉴于已经有大量的 Linux 发行版，Rancher 提供了更流行的操作系统所不能提供的东西吗？“[Rancher Labs](https://twitter.com/shengliang)首席执行官盛亮，[说，“我们有一个小的、相当技术性的用户群，RancherOS 满足了那些希望运行容器的用户的需求——它与 Docker 有很多重叠的功能。](http://rancher.com)

这种重叠意味着牧场主使用 Docker 进行管理。因为功能存储在云中，并使用 cloud-init 进行配置，所以大大减少了操作系统的占用空间。“它没有传统的 Linux 配置服务，而是从云中获取，”梁说。

RancherOS 支持多种平台，包括亚马逊 EC2、裸机、KVM 和 OpenStack。

该公司发现，对该产品最感兴趣的是需要快速扩大规模的管理员。“人们希望大规模运行，不想担心操作系统，”梁说，“这是因为当你担心的不是一台服务器，而是整个服务器群时。”

他说发布会已经计划了一段时间。“我们实际上在过去的一年里一直在提供它，”梁说。“但我们已经为非 GA 产品的付费用户签约了”，他说公司不能在没有发布的情况下拿走他们的钱。

[大卫·卡特赖特](https://www.linkedin.com/in/davidscartwright/?ppe=1)，Linux 专家，一家国际金融机构的首席信息安全官，看到了该产品的吸引力。“如果你把你的 Linux 系统作为一个通用的处理平台——通用的、非关键任务的东西，经常变化/发展，你经常添加新的应用程序——那么它不适合你，”他说。

但是，他说，牧场主确实解决了许多管理员面临的一个头痛问题。“Linux 的一个问题是它太多了:有数不清的库和内核模块支持各种永远不会被调用的垃圾，但它们仍然占用磁盘空间、系统内存和潜在的 CPU 周期:你在灵活性和大量应用程序支持方面获得的好处会在效率方面损失。”

他说，集装箱化是解决这个问题的一种方法。“通过容器化内容并将内容限制在需要的范围内，您将获得性能并看到资源使用量下降。”

梁强调了牧场主安全性的增强，正是操作系统的这一方面吸引了卡特赖特。“有趣的一点是容器化和服务器加固之间的潜在联系。如果你“加固”一台服务器，你基本上是卸载了所有未使用的应用程序和系统扩展，这样它们就不会无意中成为黑客进入系统的途径。”

Rancher Labs 已经有多家公司使用该软件，梁说，吸引他的是资源的有效利用和更容易的维护，例如更容易的补丁。

然而，牧场仍将是一个特殊的环境，“牧场并不适合所有人，但在它适合的特定领域，它有很大的机会变得非常有吸引力，”卡特赖特说。
