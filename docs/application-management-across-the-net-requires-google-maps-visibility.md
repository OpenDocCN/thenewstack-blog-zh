# 当管理应用程序体验需要互联网可视性时

> 原文：<https://thenewstack.io/application-management-across-the-net-requires-google-maps-visibility/>

网络监控系统提供商[千里眼](https://www.thousandeyes.com/?utm_content=inline-mention)的产品负责人[乔·瓦卡罗](https://www.linkedin.com/in/joevaccaro)说:“云已经成为如何开展业务的新数据中心，互联网是如何连接服务的新企业网络主干。”。如果一个组织不“拥有”互联网上的所有网络连接，“你就不能采取与过去相同的方法，通过它们来理解数字体验，”瓦卡罗解释道。

在这个由 New Stack 的创始人和发行人 Alex Williams 主持的 [The New Stack Makers](/podcasts/makers) 播客中，Vaccaro 讨论了现代应用体验和管理后端相互依赖的当今数字供应链，以及千眼万眼带来的价值主张[。](https://www.thousandeyes.com/?utm_content=inline-mention)

[当跨网络的应用程序管理需要“谷歌地图”可见性时](https://thenewstack.simplecast.com/episodes/when-application-management-across-the-net-requires-google-maps-visibility)

瓦卡罗说，帮助组织“真正理解互联网”的雄心，类似于谷歌地图如何让用户找到并定位世界上任何地方的位置，是创建 ThousandEyes 的灵感来源。

“本质上，ThousandEyes 所做的是帮助人们看到每一个网络，就像你拥有它一样，并能够理解从用户到应用程序的完整数字体验之旅。”

由于应用程序已经从整体部署架构转移到微服务，因此核心功能也“严重依赖”第三方 API。瓦卡罗说:“因此，如果你考虑所有这些改变不同道路复杂性的因素，它们只会增加最终可能出错的地方。”。“出了什么问题最终会影响访问该应用程序的数字体验。”

一个案例可能是一家 SaaS 公司交付了一个难以访问或访问速度很慢的应用程序。虽然一些用户可能会认为是应用程序导致了问题，但也可能是缓慢的网络连接干扰了体验。相反，努力访问应用程序的员工可能会归咎于网络，而错误可能是由于“用户和他们试图访问的应用程序之间的部分路径，”Vaccaro 说。

“由于网络——互联网本身——是由网络的网络组成的，在网络上，用户随时可能选择多种不同的路径。你不仅需要能够测量正确的指标，还必须能够通过汇集在一起并相互关联的正确路径进行测量:我们称之为路径可视化，”瓦卡罗说。

这种“可视化之路”也有望继续吸引更多的兴趣，因为劳动力和应用程序开发人员很可能会继续远程工作，并在未来保持与高度分布式环境的连接。

“这是一个迷人的时代，身处这个行业，思考数字供应链如何在所有不同的地方继续发展，然后能够从端到端的基础上衡量这种体验，”瓦卡罗说。
