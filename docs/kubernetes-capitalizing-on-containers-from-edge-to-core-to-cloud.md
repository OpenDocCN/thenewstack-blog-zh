# Kubernetes:利用从边缘到核心再到云的容器

> 原文：<https://thenewstack.io/kubernetes-capitalizing-on-containers-from-edge-to-core-to-cloud/>

KubeCon + CloudNativeCon 赞助了这篇文章，期待虚拟 [KubeCon + CloudNativeCon EU](https://events.linuxfoundation.org/kubecon-cloudnativecon-europe/) ，8 月 17-20 日。

[Kubernetes](https://kubernetes.io/) 于六年前推出，在相对较短的时间内，它巩固了其作为容器编排和管理事实上的标准的地位——并且全球粉丝群每年都在快速增长。Kubernetes 和云原生技术具有广泛的应用，因为它们是无数开源创新的坚实而无处不在的结缔组织——从简化开发人员和业务应用程序的生活，到支持不同类型的基础设施和添加 AI/ML 功能。

对于许多首席信息官来说，Kubernetes 是顶级的容器管理系统。GitHub 上与 Kubernetes 相关的条目现在的数量为 [60，682](https://github.com/search?q=Kubernetes) ，随着我们体验到 Kubernetes 在企业和边缘计算等新用例中的快速扩展，这一数字预计还会增加。

Kubernetes 提供了许多优势，尤其是在支持 DevOps 时。它可确保快速、轻松地管理和逻辑组织容器化的应用和服务，并自动执行许多运营任务，如扩展和可用性管理。Kubernetes 服务产品在云提供商中的普遍存在和容器本身的可移植性确保了企业甚至可以在不同的提供商之间转移工作负载。有了这些好处，它的受欢迎程度飙升也就不足为奇了。

SUSE 最近委托独立市场研究机构 Insight Avenue 进行了一项全球研究，探索 IT 领导者如何应对当今的挑战。研究发现，集装箱化的平均生产工作量预计将从目前的 27%增长到一年后的 34%，两年后的 47%。随着对容器化需求的增加，对容器编排的需求也在增加，这反映了随着组织继续其完全云原生解决方案之旅，容器日趋成熟。

简而言之，Kubernetes 对于现在和未来寻求推动创新的组织来说是必不可少的。它在帮助企业通过容器化和云原生工作负载加快应用交付方面发挥着至关重要的作用。随着边缘计算和混合云的兴起，新的需求浮出水面。根据 Insight Avenue Research 的研究，IT 领导者希望通过采用物联网(82%)或边缘计算(80%)以及开源(70%)为他们的业务成功带来切实的变化。

让我们来看看这些数字背后的一些用例。

## **边缘到核心到云**

今天，边缘计算已经在许多不同的行业中实施；它有很多应用。与云计算或核心计算一样，边缘计算需要基础架构、数据和应用管理，以及与内部、私有数据中心和公共云的无缝集成。虽然云的“即服务”方法和基于设备的 edge 方法最初看起来非常不同，但它们紧密相连，相互依赖。

智能自动驾驶汽车是边缘到核心到云用例的一个非常好的例子。这些汽车完全配备了传感器和计算单元，是大规模分布式综合系统边缘架构的端点。这些智能汽车产生了大量的数据。一些数据在本地处理，因为延迟不是驾驶反应所有方面的选项，而一些数据返回到汽车制造商的数据中心(现场或集中)进行存储、过滤、处理和分析。这个系统是双向的，因为汽车系统也需要更新和指示。最后，托管集中式数据和终端用户应用的公共云正在连接到汽车，并与汽车一起使用。

在这个场景中有几个明显的组成部分-汽车、数据中心、云-但我们不能错过的是广泛的基础:使这一切工作的人、解决方案和流程。活跃用户以及 IT 和开发团队利用工具和流程来管理应用程序、数据、基础架构、安全性和可观察性。这一层使整个系统成为现实，并为所有组件提供必要的支持。

当然，边缘计算网络变得非常大，非常快；它们包括各种类型的云和基础设施。但也有开源项目，如 [K3s](https://k3s.io/) 和 [KubeEdge](https://kubeedge.io/en/) ，让我们能够在边缘环境中使用 Kubernetes，以及其他云原生技术，使我们能够管理整个生态系统，例如多云或多集群。

当然，这种情况很容易被汽车行业之外的行业采用，比如零售业或医疗保健业。跨行业所有用例的先决条件是，边缘计算需要一个从边缘到核心再到云的集成基础设施。云原生技术是关键，现在已被改编并用于边缘计算。

最重要的是，所有这些进步将使 Kubernetes 和其他云原生项目成为统一平台，使企业能够访问边缘计算。在当今革命性的技术格局中，Kubernetes 和云原生解决方案对于寻求推动当前和未来创新的组织来说至关重要。
