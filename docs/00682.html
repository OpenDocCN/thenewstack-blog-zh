GitHub Actions为自动化开发工作流创造了一个热点

原文：https://thenewstack.io/github-actions-creates-a-buzz-for-automated-dev-workflows/#0001-01-01

本周在三藩市举行的年度用户大会上，GitHub推出了一款新产品，【GitHub Actions】，它允许开发者创建小代码，这些代码可用于触发代码库服务中的事件或一组事件。

GitHub平台负责人萨姆·兰伯特，在一次采访中表示，GitHub希望创造一个环境，让开发者能够专注于他们最感兴趣的事情，即编写代码。 “我们的问题是，‘我们如何让开发人员保持在最佳状态？’答案是除了编写代码本身之外，自动化代码生产所需的一切。

“我们采用CI/CD并在从创意到生产的整个流程中应用原则，为工作流程带来灵活性，”Lambert说。 “你可以用动作做CI/CD，但这样就多了。”

例如，您可以将所有手动步骤包装到can action中，以将您的代码 推送到云服务。动作也适用于第三方软件:使用launch darky，您可以使用一个动作来翻转功能标志，只需关闭一个问题。 稍后您可以使用单个操作在Terraform中批准和应用操作。

开发人员可以使用预先做好的动作，或者用任何语言编写自己的动作。 动作成为GitHub代码库的一部分，因此一旦编写完成，所有开发者都可以使用。

GitHub产品设计高级总监Max Schoening解释说，开发人员可以创建自己的工作流程。 “你可以想链多少就链多少，可以从GitHub平台的任何事件触发一个工作流。”他警告说，对于beta来说，50岁是有上限的，他指出，这意味着一次要把很多动作串起来。

关键是行动是简单的、可操作的和可重复的。唯一的要求就是要有一个顶级docker文件。 “动作让你指向别人的repo，一个已有的docker文件，或者一个路径，动作就会相应的表现出来。这对于开源的可能性和生态系统来说是一个全新的难题，”微软、 的高级开发人员Sara Drasner在的一篇文章中写道，这篇文章举例说明了如何编写自己的操作。
