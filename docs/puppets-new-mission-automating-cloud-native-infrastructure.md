# Puppet 的新任务:自动化云原生基础设施

> 原文：<https://thenewstack.io/puppets-new-mission-automating-cloud-native-infrastructure/>

如果没有自动化，一个雄心勃勃或希望在云原生环境中扩展应用程序部署的组织不会走得太远。

从 CI/CD 支持、提高应用程序部署速度(通常跨不同环境)以及维护法规遵从性和安全性来看，运营团队在某个时间点之后手动管理这些流程是不可能的。

在最新一集的 New Stack Makers 播客中， [Abby Kearns](https://www.linkedin.com/in/abbykearns) 、 [Puppet](https://puppet.com/?utm_content=inline-mention) 的首席技术官兼研发主管和 [Chip Childers](https://www.linkedin.com/in/chipchilders) 、Puppet 首席架构师讨论了云原生部署的基础设施管理自动化对 Puppet 和 IT 行业意味着什么。TNS 的创始人兼发行人亚历克斯·威廉姆斯主持了这次采访。

[傀儡的新任务:自动化云原生基础设施](https://thenewstack.simplecast.com/episodes/puppets-new-mission-automating-cloud-native-infrastructure)

行业媒体对原生云、容器和 Kubernetes 给予了很大关注。然而，事实仍然是，在云原生架构中实际运行的工作负载数量仍处于早期阶段，Kearns 说。

Kearns 说:“就部署而言，它还处于萌芽阶段，尽管有如此大的关注，但大多数企业仍有如此大的内部期望，这包括他们的主机托管设施或私有云。

“出于数据主权原因或要求，他们不想移出数据，甚至当他们试图找出如何利用这些数据湖中的数据做更多事情时，最终对这些本地功能的需求仍在增长。”

然而，Kearns 指出，组织正在将更多的工作负载转移到一个或多个公共云。由于 Puppet 多年来一直致力于自动化内部基础架构管理，因此将这一技术应用到云环境中为该公司创造了机会。

“我们希望在本地成功的基础上再接再厉，并开始将其扩展到云中，并找出如何将云中确实存在的自动化需求带到那些云原生工作负载中，”Kearns 说。“这就是我们在过去一年半的时间里走过的旅程。我们一直在重建，或者实际上将许多核心自动化功能转移到一个新的基于 Kubernetes 的平台上。”

推动将计算资源转移到云原生环境也意味着更多的复杂性。帮助管理这种复杂性的能力在很大程度上吸引了 Childers 加入 Puppet，他最近从 [Cloud Foundry Foundation](https://www.cloudfoundry.org/?utm_content=inline-mention) 加入了 Puppet，并在那里担任了长期的执行董事(Kearns 也在 Cloud Foundry 担任执行董事，直到 2020 年)。

“信息技术的复杂性呈指数级增长，因为企业对更多技术的需求更快、更强，因为人们开始认为技术和软件是战略上良好的战略目标，而不仅仅是成本中心，”Childers 说。

“随着交付模式、基于云的模式、云原生体系结构、容器化以及所有这些东西开始真正起飞，它也在快速增长。”

Childers 说，随着复杂性的增加，Puppet 已经上升到帮助管理基础设施目标的场合。“Puppet 拥有惊人的经验，因为它拥有惊人的知识产权、背景和经验。坦率地说，寻找一种方法来帮助运营商将所有这些遗产带入目前难以驾驭的领域。”
