# 成为“绿色开发商”意味着什么？

> 原文：<https://thenewstack.io/what-does-it-mean-to-be-a-green-developer/>

当丑闻袭击技术部门时，我们会将责任归咎于高层，但我们通常不会深究个人责任。但是我们从大众汽车排放丑闻中发现[即使是编写代码的人也可能是有罪的](/are-programmers-ethically-and-legally-responsible-for-their-code/)。

如果我们每个人都在发布的代码中承担一些个人责任，那么“绿色开发者”的概念是否有力量？在这一集的[新堆栈制造商](/podcasts/makers)中，我们与微软绿色云倡导负责人 [Asim Hussain](https://twitter.com/jawache) 坐下来讨论什么是绿色开发者。我们试图揭示 web 开发人员、机器学习工程师、DevOps 人员或拥有大量物联网设备的部门的实际情况。

[微软的阿西姆·侯赛因——一个绿色开发者的诞生](https://thenewstack.simplecast.com/episodes/microsofts-asim-hussain-the-making-of-a-green-developer)

Hussain 说，这样一来，当他们应该从头到尾审视事物时，他们最终只会关注自己的角色。我用 去 想全栈本意 就像 一个 网站 一个 一个 数据库。现在 我 了解全栈意思是 喜欢， 从 用户 行为 到 如何 用电 是 买来的 和

事实上，侯赛因预测一个全新的角色将会出现:可持续软件工程师。或者更好的是，一个多部门的团队，负责拼凑完整的软件开发生命周期，从采购硬件材料，到为数据中心供电，再到工具和设备的废弃。

这可以从热情的、跨职能的环保志愿者开始，他们让自己在组织中为人所知，并试图拼凑这个生命周期。微软就是这样开始的，成长了一个 2000 人的绿色团队。

你从哪里开始？侯赛因说，首先要检查你的应用的碳效率和碳强度。Hussain 指出了一些具有重大影响的小举措，如选择何时运行工作负载，这“取决于可再生能源组合和能源网格，您可以通过改变运行工作负载的时间，将每个应用的碳排放量减少 48%。”

不要以为这只是针对最现代的微服务，他说，当你在传统应用上运行某些任务时，这甚至会更有影响。

侯赛因还谈到了绿色公共协议的创建，以及微软的可持续发展计算器，它可以让你开始衡量你的系统对环境的影响。
