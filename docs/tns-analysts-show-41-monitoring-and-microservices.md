# 监控和微服务

> 原文：<https://thenewstack.io/tns-analysts-show-41-monitoring-and-microservices/>

在本期新的 Stack 分析播客中，亚历克斯·威廉姆斯(Alex Williams)和回归的联合主持人唐尼·伯克霍尔茨(Donnie Berkholz，现就职于 451 Research)与菲利普·刘(Phillip Liu)一起讨论如何监控现代分布式应用。

在新堆栈的播客部分查看更多剧集[。](https://thenewstack.io/podcasts/)

[#41:监控与微服务](https://thenewstack.simplecast.com/episodes/41-monitoring-and-microservices)

“创建虚拟机非常容易，”Phillip 说，“而且虚拟机基本上是根据您的应用程序来调整大小的，因此您不再处于拥有大型金属服务器并且在同一台物理服务器上运行许多应用程序的世界。”

然而，如果在所有这些抽象中出现问题，就不容易确定原因了。就在几年前，IT 团队还会在内部对这些内部问题进行分类。

Alex 想知道，“监控如何转移到用户正在构建的新一代微服务架构的中心？”

[![phillip-liu](img/a5cd7690525e99625acb6c34e4ea1e8d.png)](https://thenewstack.io/wp-content/uploads/2015/05/phillip-liu.jpg)

菲利普·刘

随着微服务和实例的激增，Phillip 说“人们不知道从哪里开始。”他指出，有很多工具可以报告特定实例的失败，但不能提供“关于整体服务运行情况的洞察”

菲利普说，SignalFx 在博客上推广这种类型的对话，并将其带入客户关系，从初创公司到财富 500 强企业。“人们构建微服务的原因是，他们可以很容易地在已经运行的服务上添加组件，而无需关闭整个服务，并且能够向服务添加零碎的功能，因此这是他们有机增长的方式。”

Phillip 说:“这很好，但问题是，随着时间的推移，复杂性也会随之增加，这就是你需要开始从数据的角度看待问题的地方。这是您需要开始收集跨所有服务实例的信息的地方，以便使用分析技术来了解您的运行时应用程序正在发生什么。”

这就是 SignalFx 的用武之地，它提供指标和流分析，以帮助促进开发团队和部署这些现代分布式应用的组织之间的交互通信，Phillip 说，并指出“用于构建这些微服务的相同原理将需要相同的工具集来操作它们。”
