# 云计算之后的下一个时代——天空计算

> 原文：<https://thenewstack.io/sky-computing-the-next-era-after-cloud-computing/>

根据加州大学伯克利分校两位杰出的计算机科学教授 T2·扬·斯托伊察和斯科特·申克的说法，我们即将从云计算时代过渡到天空计算时代。

顾名思义，云计算是云平台之上的一层——它的目标是实现云之间的互操作性。如果你认为这听起来像当前的行业术语，多云，你就对了。为了了解更多关于天空计算的信息，我采访了斯托伊察教授(顺便说一下，他也是 [Anyscale](https://www.anyscale.com/) 和 [Databricks](https://databricks.com/) 的联合创始人)。

我很高兴能与斯托伊察交谈，因为他有着正确预测云计算未来的记录。早在 2009 年 2 月，他和一群伯克利的学者发表了一篇关于当时新兴的云产业的有影响力的论文。当时，亚马逊网络服务刚刚成立几年，谷歌唯一的云产品是 App Engine(仍处于预览版)，微软的 Azure 也尚未正式发布。2009 年的论文总结道，“将计算作为一种工具的长期梦想终于实现了。”

虽然云计算确实从根本上改变了 IT 行业以及应用程序在 2010 年的构建和部署方式，但在斯托伊察和申克看来，有一个突出的问题——云计算没有像互联网或网络一样成为公共事业。在 2021 年，没有一个单一的底层云平台拥有一套任何人都可以使用的开放标准。相反，云计算已经演变成一系列彼此基本不兼容的专有平台:[亚马逊网络服务](https://aws.amazon.com/?utm_content=inline-mention)(AWS)[微软 Azure](https://azure.microsoft.com/en-us/)[谷歌云](https://cloud.google.com/)等等。斯托伊察和申克的新论文展示了“云计算的更商品化版本，我们称之为天空计算”的愿景

## 实施多云平台

本质上，这是关于支持多云应用开发。“为了实现效用计算的愿景，应用程序应该能够在任何云提供商上运行(即，编写一次，在任何地方运行)，”这篇新论文断言。

> 本质上，sky computing 是关于支持多云应用程序开发的。

但是为什么开发人员想要为多云环境构建一个应用呢？传统观点认为，选择一家云提供商并使用该公司提供的服务套件更容易(AWS 实际上有数百家！).然而，该论文拒绝了这种观点，而是建议具有“计算密集型工作负载”的应用程序更适合多云环境。我问斯托伊察这是为什么？

扬·斯托伊察教授

“计算机只是更容易些，”他回答道。“你不需要支付出口费用，将数据放入云中不需要任何成本，但将数据从云中取出却需要大量成本。特别是在机器学习中，如果你正在做训练或高参数调整。这些都是计算密集型的工作，所以把这些工作转移到你能更快更便宜地完成的地方是很有意义的。当然，如果移动计算，您还需要移动训练数据，但是，一般来说，移动这些数据的成本与训练或调整模型的成本相比就相形见绌了。”

该论文提出，天空计算由三层组成:“一个兼容层，用于掩盖底层技术差异；一个云间层，用于将作业路由到正确的云；以及一个对等层，允许云之间就如何交换服务达成协议。”这三层反映了互联网本身是如何设计的，例如，互联网协议(IP)提供了网络间的兼容性。

> 天空计算由三层组成:兼容性、云间性和对等性。

兼容性层将使应用程序开发人员能够轻松地将他们的应用程序从(例如)AWS 移动到 Google Cloud。多云的用武之地是云间层，因为它将允许应用程序跨多个云提供商运行——取决于用户需求。斯托伊察是这样解释的:

“云间层(从兼容性层)上升了一个级别。理想情况下，借助 intercloud 层，您可以指定作业的首选项，比如我希望最大限度地降低成本或缩短时间，或者我需要在本地处理这些数据，intercloud 层将决定在哪里运行作业以满足这些首选项。”

关于数据位置的例子，斯托伊察解释说，应用程序必须使用特定的地理位置，这可能有地缘政治或其他方面的原因。考虑一个应用程序，它希望处理一些不能离开国家边界的数据，并且该国家只有一个 AWS 云数据中心。在这种情况下，云间层会自动将该应用程序路由到 AWS 的数据中心。但所有其他应用程序可能会使用不同的云平台，这取决于应用程序开发人员定义的云间规则。(顺便说一下，用户不知道他们在哪个云平台上；这都是在应用程序部署级别。)

谁来提供这个云际层？斯托伊察认为它可以由现有的云平台提供，或者也许一种新型的“虚拟云”公司将出现，专门从事这种路由功能。他建议使用术语“无基础设施云”，因为云间层不需要基础设施(服务器、数据库等)。

## 云企业将如何应对？

随着多云成为天空计算的优先事项，一个关键的挑战将是买入当今市场领先的云平台——特别是 AWS、微软和谷歌。我问斯托伊察，他认为哪个主要平台将首先走向天空计算，他们的动机是什么？

“根据经济学理论，市场上排名第二或第三的云——如谷歌——最有可能做到这一点，因为这是他们获得更多市场份额的一种方式。如果他们提供更快或更便宜的基础设施，天空将使他们更容易从其他云获得更多工作负载。”

然而，他也指出，应用程序开发人员不一定需要大云平台的许可才能获得“天空计算”功能。

“你今天就可以做到。我可以有一个应用程序——比如说一个机器学习管道——并进行一些数据处理、一些培训和一些为模型服务的工作。我可以在谷歌上进行培训，在亚马逊上提供服务。”

用斯托伊察的话来说，现在做多云的问题是“笨重”，“不是自动的，是手动的”当然还有外出费！

> Sky computing 有可能扩展云提供商的软件即服务业务。

大公司面临的另一个挑战是，他们会正确地将此视为将其核心云平台商品化。但斯托伊察指出，这些组织的其他部分将受益——例如，对于微软的 Office 团队来说，“这将允许他们在亚马逊的云或谷歌云上运行 Office。”

因此，天空计算有可能扩展大型云提供商的软件即服务业务。微软是否想这样做是另一回事，但如果他们所有的云竞争对手都转向天空计算模式，那么他们将别无选择，只能跟随市场。这将如何在未来几年上演，这很有意思。

## 下一个 Kubernetes 会是什么？

当斯托伊察和他的同事在 2009 年发表他们的论文时，Docker 和 Kubernetes 几年后才作为大规模管理云计算的方式出现。所以我问斯托伊察，他是否预测在 DevOps 工具中会有类似的创新，也许在接下来的几年里，会推动天空计算的采用？

“展望未来，我认为会有很多创新，因为考虑到云提供的无数服务，将云抽象化并不容易。即使他们提供同样的服务——比如 Kubernetes——由谷歌托管，与亚马逊或微软托管相比，也是不一样的。它们并不相同。因此，基本上，发布和公开服务 API 的能力以及差异，我认为我们将会看到很多创新。”

他还认为在数据层(“因为你必须透明高效地跨云移动数据”)和安全性(“因为你需要授权和认证，每个云略有不同”)方面也会有创新。

> “展望未来，我认为会有很多创新，因为考虑到云提供的无数服务，将云抽象化并不容易。”

-伊恩·斯托伊察

因此，或许类似于 2010 年代的云计算革命如何为这些云平台之上的服务开辟了一个巨大的市场——我们现在称为“云原生”产业——初创公司将有大量机会提供解决方案来促进或构建天空计算层。

同样，前端也需要解决方案。开发人员将如何指定上面提到的应用程序首选项(使用 intercloud 层)？相关的，斯托伊察说，另一个挑战将是“你如何指定什么是可以分发的应用程序的大组件，以及在哪里。”例如，也许你想在谷歌上做机器学习方面的工作，但在 AWS 或 Azure 上做另一个关键任务。因此，随着天空计算时代的到来，这似乎是未来十年创业公司探索的一片绿地。

## 结论

扬·斯托伊察和他的伯克利同事在 2009 年对云计算的未来有先见之明，我认为斯托伊察和申克在新的论文中为实用云计算提供了一个令人信服的案例。但要实现这一愿景，至少有一家大型云提供商需要朝着构建兼容性和云间层迈出第一步。和斯托伊察一样，我怀疑这将是谷歌(毕竟，它是开发 Kubernetes 的公司)。但微软也证明了它愿意支持开源，并转向新兴的云趋势。市场领导者 AWS 迈出第一步的可能性更小——但话说回来，亚马逊是实际上发明云计算的公司。

无论哪个大玩家迈出第一步，我都很兴奋，随着未来十年天空计算平台的建立，数百甚至数千家新的创业公司将有机会大放异彩。
