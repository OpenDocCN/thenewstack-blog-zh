# NS1 建立在 DNS 之上，以加速流量管理

> 原文：<https://thenewstack.io/ns1-builds-on-dns-to-speed-traffic-management/>

当用户体验越来越成为速度和可靠性的同义词时，新的交通管理子团队正在[精英数字企业](https://thenewstack.io/googles-formula-for-elite-devops-performance/)出现。

在这一集的[新堆栈制造商](https://thenewstack.io/podcasts/makers)播客中，我们与[克里斯·比弗斯](https://twitter.com/beevek)谈论流量经理角色的重要性，以及我们对今年企业技术的展望。七年前，比弗斯共同创立了网络自动化公司 NS1，或者用他的话说，是“当今互联网上许多关键领域和应用程序的记录系统”

[混沌工程 w/ Kris Beevers](https://thenewstack.simplecast.com/episodes/chaos-engineering-w-kris-beevers)

他说，我们每个人每天都要与 NS1 互动几十次，就像我们在 LinkedIn 上联系或在 DropBox 上共享文件一样。

NS1 位于这个新的流量管理堆栈的基础，在我们日益复杂和分布式的系统中引导流量。该堆栈还包括内容网络交付网络(CDN)、负载平衡工具、边缘网络足迹、服务网格以及用于服务发现和出口优化的软件。这个新角色不仅仅是测量流量是否正常工作，而是真正了解您的用户和系统，以了解流量如何到达应用程序，您是否使用了正确的网络提供商和供应商，以及您是否有效地选择了数据中心、云和 cdn。

这个新的交通管理角色位于更大的站点可靠性工程趋势的核心，如[混沌工程](https://ns1.com/blog/chaos-engineering-and-the-modern-traffic-stack)和[渐进交付](https://thenewstack.io/the-rise-of-progressive-delivery-for-systems-resilience/)。它是关于使用您的工具栈来理解您的整个工具如何反应以准备最坏的情况。

Beevers 解释说，NS1 依靠 DNS(互联网域名系统)来“释放巨大的杠杆作用，以解决应用性能、可靠性、安全性、运营效率、复杂性和应用交付堆栈现代化方面的大问题。”

“DNS 的机会和我们为什么启动 NS1 的原因是，当你打开一个应用程序或访问一个网站时，它是你手机或笔记本电脑上的第一个接触点，”他说。“在一个为应用程序提供服务的基础设施非常动态、非常分散的世界中，有很多杠杆作用，”这意味着 DNS 所指向的东西是多种多样的。

Beevers 说 NS1 试图回答一个非常基本的问题:现在哪个基础设施将为你提供与这个应用程序交互的最佳服务？

他说他们的答案是“我们将最终用户与应用基础设施连接起来，这将最大限度地提高他们的体验质量，确保这一点，并提高应用程序开发人员的运营效率。”

他说，开发工具或基础设施即代码的趋势是自信地自动化这些决策。随着该基础架构变得越来越复杂和异构，NS1 利用 DNS 来管理这一选择。

NS1 是一种技术，它适合在堆栈的 DNS 级别将用户连接到正确的基础架构，但也会检查用户，包括他们的数据、他们的连接路径、它在哪里运行，以及它是正常还是不正常。

Beevers 将此视为 NS1 的工作，为我们的云原生现实实现这一层堆栈的现代化。

NS1 最近控制了超级碗直播的所有视频流量——这是一次大规模的努力。比弗斯说，用户体验是最重要的驱动因素，但效率紧随其后。这种效率通常会将流量导向成本最低的基础设施，从而立即提供所需的服务质量。在未来，随着可再生能源变得越来越便宜，它也将包括环境成本。

NS1 是新堆栈的赞助商。
