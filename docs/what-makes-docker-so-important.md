# 是什么让 Docker 如此重要

> 原文：<https://thenewstack.io/what-makes-docker-so-important/>

[是什么让 Docker 如此重要](https://thenewstack.simplecast.com/episodes/what-makes-docker-so-important)

在本期 [The New Stack Makers](https://thenewstack.io/podcasts/makers) 播客中，我们将与 Docker 的两位员工——工程师 [Christopher Crone](https://www.linkedin.com/in/christopher-crone-015ab521) 和营销专家[Jim Armstrong](https://www.linkedin.com/in/jdarmstro/)—就使用 Docker 容器时的某些最佳实践(针对开发人员和企业用户)进行交谈。任何被信任部署容器的开发人员和企业用户都会对这种敏捷技术有所了解。

从一开始，采访就以容器的定义开始，以及是什么使得这种后起之秀技术对开发人员如此重要，对企业如此重要。但这不是维基百科对容器的定义，这些定义来自那些帮助创建负责容器部署的技术的人。根据 Crone 的说法，“容器是一种用于隔离应用程序的技术…您可以从在物理机上运行它开始，也可以虚拟化它…最后还有容器，您可以在其中使用名称空间等内核概念来隔离内核中的进程。”阿姆斯特朗补充道，“Docker……创造了一些工具(如 [Docker Desktop](https://www.docker.com/products/docker-desktop) )来处理底层操作系统部件……并使这些工具易于使用。”

从那些开发和营销 Docker 的专业人士的口中，您将获得关于部署、安全性、容器的未来、Kubernetes 或 Docker Swarm 是否是您任务的正确工具、如何避免使用恶意图像以及 Docker 开发的最佳工具的提示。你会发现从小处着手，从头开始学习 Docker 是多么重要。

但是是什么让 Docker 容器对企业如此重要呢？根据 Crone 的说法，从纯开发人员的角度来看，“对开发人员来说，最重要的事情是它使建立您的开发人员环境变得更加容易……您可以使用 docker 文件、docker 平台和 docker 本身来确定一切。”阿姆斯特朗补充道，“如果我在本地创造了一些东西，并且我想与我团队中的人分享……我可以很容易地拿起那个容器与他们分享。”

另一个贯穿本集的问题是社区问题。对此，Crone 说，“如果你在 Stack Overflow 上搜索，如果你阅读论坛，如果你加入 Docker Slack 社区…人们已经完全做了你以前试图做的事情。事实上，Docker 周围的社区是很棒的事情之一。”

从开发人员的角度，Armstrong 提供了在企业中部署容器的三个最佳技巧。他说:从小处着手，向他人学习，并与社区分享。

这个播客主要面向开发者和从业者，但是也有适合每个人的东西……不管你对 Docker 和容器有多熟练。

TNS Linux 记者杰克·沃伦担任本期播客的主持人。
