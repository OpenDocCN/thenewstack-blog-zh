# 主厨亚当·雅各布谈配置管理与人的因素

> 原文：<https://thenewstack.io/chefs-adam-jacob-configuration-management-versus-human-element/>

[大厨亚当·雅各布论配置管理与人的因素](https://thenewstack.simplecast.com/episodes/chefs-adam-jacob-on-configuration-management-versus-the-human-element)

技术流程的设计没有考虑到人的因素。设备可能会变得更容易，但只能通过更复杂、通常看不见的技术基础设施网络来实现。这一令人惊讶的声明来自首席技术官 Adam Jacob，Habitat 系统的联合创始人，该系统旨在为配置管理增加更多的人为因素。雅各布说，安全系统不是为接受变化而设计的。因此，他们倾向于直接拒绝新的流程，即使这些流程可能是创新的、有用的，而且总体上是对我们现有系统的改进。

Jacob 在最近的[page duty Summit 2017](https://www.pagerduty.com/summit/)会议上的讲话中提到，DevOps 转型依赖于人，而不是技术。讲述技术背后的人的故事，而不是技术本身，是持续发展的关键。

“在很大程度上，我们一直在开发的技术并没有很好地为我们服务。我们首先把它作为技术来建造，而不是为所有连接到这项服务的人，”后来补充说，“我们建造的技术将走向世界，对人类产生影响。”
