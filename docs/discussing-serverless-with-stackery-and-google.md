# 与 Stackery 和 Google 讨论无服务器

> 原文：<https://thenewstack.io/discussing-serverless-with-stackery-and-google/>

[与斯塔克里和谷歌](https://thenewstack.simplecast.com/episodes/discussing-serverless-with-stackery-and-google) 讨论无服务器

在最近的一次波特兰 PDX 无服务器用户组会议上，我们采访了[谷歌](https://plus.google.com/106678312289948845134)的开发人员倡导者凯尔西·海托华和[斯塔克里](https://www.linkedin.com/in/nathantaggart/)[的首席执行官内特·塔加特](https://www.stackery.io)。两人就无服务器的状态进行了一次炉边聊天，不仅对这项新技术如何适应当前的应用，而且对它如何适应计算的整个历史有着敏锐的洞察力。

“我认为无服务器现在正处于炒作周期中。我们已经达到了对潜力的极度狂热。它永远是灵丹妙药。这项技术感觉很新，但实际上它可能只是我们做的同样的旧东西的新版本。塔加特说:“我们经历了一个从抽象中走出来，然后又收回控制权的循环。

“说我们要运行一个 Kubernetes 集群，以便扩展我们的计算，这很容易。这不是一个可怕的问题。我们如何把计算机和数据库连接起来？消息流和队列。我们如何将其标准化和规范化，以便我们公司的任何工程师都可以将其从货架上取下，并将其应用于他们的问题？自我托管、自我管理的基础设施，我不知道它是否能达到那个成熟度。在一些专家企业中确实如此，但是云基础架构的本质是它已经产品化了。它非常标准化，非常一致和可预测。塔加特说:“我们努力关注的是如何将这些碎片整合在一起，形成一个有凝聚力的整体。”。

无服务器提供了一条更紧密地使用所有这些底层组件的途径，但不一定仅仅通过现有组件就能提供更好的凝聚力。Hightower 说，与功能即服务的概念相比，即使是无服务器计算的概念也会变得模糊。“我认为 CDN 是一个很好的使用案例，我们需要世界各地的每个人都能快速访问数据。缓存是思考这个问题的一种方式，但是一旦缓存模式变得健壮并被很好地理解，那么我认为你会看到一个完整的市场围绕着它成长。Hightower 将无服务器架构与基于内容交付网络的架构相比较，认为这是一种成熟的模式。

“计算的时机已经成熟，”他继续说道，“现在它的定义非常松散——大部分都是虚拟机甚至容器。我认为无服务器的一部分是尝试做类似的事情。如果我们说你可以给我一些代码，我可以在世界上的任何地方运行它，并处理任何挑战。就像在 CDN 中一样，没有人会考虑 CDN 的规模。如果有人第一次在澳大利亚提取内容，它将能够在那里进行连续提取，在任何地方都是这样处理的。云提供商碰巧在任何地方都有基础设施，而你却没有。Hightower 说:“你几乎可以按照我们复制代码的方式为计算开发 CDN。

Stackery 和 Google 是新堆栈的赞助商。
