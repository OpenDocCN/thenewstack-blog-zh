# Node Summit 重点介绍无服务器并行 JavaScript

> 原文：<https://thenewstack.io/node-summit-highlights-serverless-parallel-javascript/>

本周在旧金山举行的 [Node 峰会](http://www.nodesummit.com/)上，JavaScript 和无服务器技术的企业未来成为人们关注的焦点。展会以宣布将[建筑师](https://thenewstack.io/javascript-gets-onramp-lambda-serverless-open-source-architect-tool/)项目捐赠给 [JavaScript 基金会](https://js.foundation/)拉开序幕，这清楚地表明，无服务器云功能是开发者未来的一部分。演讲者还谈到了一些话题，如最近对版本 8 的更新，以及微软的 TypeScript。

[twist lock](https://www.linkedin.com/in/joshuathorngren/)的营销副总裁 Josh Thorngren 表示，他的团队在活动中设立了一个展位，专门参与围绕无服务器解决方案的讨论。Twistlock 为整个开发生命周期提供安全和漏洞分析，并特别关注解决云环境中安全策略的治理和实施所固有的问题。

Thorngren 说，他看到很多讨论都集中在无服务器上，把它作为一种工具来补充基于容器的架构，但不一定要取代它们。“我们正积极地从容器向无服务器空间转移。就在上周，[我们向我们现有的客户发布了一个测试版](https://www.twistlock.com/2017/07/18/vulnerability-explorer-deep-dive/)，让他们能够为他们的无服务器功能做同样的漏洞管理，就像他们可以为他们的容器应用程序做一样。Thorngren 说:“这与公司通过 Amazon Lambdas 提供功能的方式有很多重叠。

在展会的其他地方，[微软](https://www.linkedin.com/in/bowdenkelly/)[的项目经理鲍登·凯利](http://www.microsoft.com)向挤满了时髦的年轻开发人员和企业程序员的观众讲述了 TypeScript。他的意思是，TypeScript 是一种类型安全的 JavaScript 版本，可以编译成 JavaScript，随着时间的推移，只需添加一个库，就可以轻松采用。一旦添加，类型可以由开发人员逐渐引入，而不是移植到现有代码中。

Kelly 说，这种语言的真正优势在于大型开发团队可以理解堆积如山的代码。这是因为语言的类型安全和其他增强功能确保了印刷错误、拼写错误和其他潜在的编码错误更容易被识别，因为它们在编译时就被发现了。

[Todd Moore](https://www.linkedin.com/in/todd-moore-7985402b/) ，IBM[的开放技术副总裁](http://www.ibm.com)和 [Node.js 基金会](https://nodejs.org/en/foundation/)董事会成员，说 Node 的增长继续在企业内部进行。

“大型企业已经开始使用 Node，我们在自己的云中看到的情况就是证明。Node 用于从连接 Watson 中的服务到使用[open whish](http://openwhisk.incubator.apache.org/)创建各种无服务器项目的所有事情。它已经成为一种标准语言，并且是目前工具箱中最常用的工具之一。摩尔说:“它已经渗透到古老的大型 Java 商店。

Moore 说 IBM 完全支持 OpenWhisk 作为它的无服务器解决方案。他说，他预计围绕无服务器计算的标准将会出现，跨云的可移植性在云功能的未来将非常重要。

摩尔说，随着[无服务器](/category/serverless/)，该行业需要“实现标准化赛事”。事件模型需要标准化。我们有一系列这样的事情涌现出来，让我们所有人一起努力解决这个问题，并在那里创造一些一致性，允许编程模型起飞。我喜欢我们在 OpenWhisk 构造中引入规则的事实，这样你就可以使用规则来做转换和结果发生的事情。我认为这是非常强大的，”摩尔说。

[Twistlock](https://www.paloaltonetworks.com/prisma/cloud) 是新堆栈的赞助商。
