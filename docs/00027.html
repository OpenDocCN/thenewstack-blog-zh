IBM和微软对彼此云的好奇背书

原文：https://thenewstack.io/ibm-and-microsofts-curious-endorsements-of-each-others-clouds/#0001-01-01

IBM和微软今天早上宣布他们将在各自的云服务上支持对方的企业软件。

这是一个奇怪的举动。结果是，这些公司让客户在其他公司的云服务上使用他们的产品变得更加容易。

例如，像WebSphere Liberty、MQ和DB2这样的IBM中间件产品将在Azure上工作。同样，SoftLayer将支持Windows Server和SQL Server。这些公司还表示，他们还将制造。Bluemix支持. Net。

此外，他们含糊地表示，IBM将“扩大对其运行在”微软Windows Server Hyper-V上的软件的支持，并且，IBM的纯应用服务将在Azure上可用。

合作中的Bluemix部分尤其有趣。它将允许。Net开发者在IBM的Cloud Foundry平台上构建应用程序作为服务，这与Azure直接竞争。此举对开发者有利，但从微软的角度来看有点违反直觉，因为该公司肯定更喜欢漏斗。Net用户到Azure。

让IBM的中间件产品在Azure中可用更有意义，因为IBM继续受益于其中间件的使用，而微软获得了Azure的新用途。

这些公司还表示，他们正在努力帮助客户获得许可。例如，微软将为Azure上的WebSphere Liberty、MQ和DB2等产品提供IBM许可证，采用按使用付费的价格。许可对于企业来说是一个很大的难题，因为在某些情况下，许可模式还没有赶上云模式。

IBM并不是微软宣布支持Azure的第一个企业软件。Oracle是Azure支持的其他公司之一。

一旦SoftLayer完全支持OpenStack，SoftLayer对微软产品的支持可能会有一些更广泛的影响，如果这意味着微软产品将在整个OpenStack云中得到支持。然而，T2已经在这方面做了一些工作。

从这一声明中可以看出，两家公司似乎都承认，不可避免地会有多家成功的云提供商。让他们自己的产品在竞争对手的云上工作，比阻止竞争对手的云的用户使用你的产品要好。

但同时，这笔交易似乎也是一种认可，即两家公司都认为对方的云服务将长期存在。否则，他们不会把工作放到这种集成中。

此外，我们不可能忽视这种安排与亚马逊网络服务的关系，亚马逊网络服务仍然是目前市场上最大的云提供商。虽然AWS在过去几年里一直在努力宣传其企业印章，但它仍未被视为一家企业公司。不管正确与否，企业并不认为AWS是一家可以迎合企业需求的公司。IBM/微软的合作提升了两家公司的企业形象。IBM和微软建立在企业业务的基础上，通过在彼此的云服务上支持客户，他们至少让这些客户远离AWS。

这一安排也加强了微软和IBM对抗谷歌的企业游戏。像AWS一样，谷歌试图迎合企业，但未能动摇其作为消费者公司的声誉。虽然谷歌的云业务仍处于起步阶段，但大多数人都承认，由于其资源和技术优势，该公司是一个威胁。
