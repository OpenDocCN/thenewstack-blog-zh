# 如何保障软件供应链

> 原文：<https://thenewstack.io/how-to-safeguard-the-software-supply-chain/>

敏捷开发团队也许能够以越来越快的速度满足软件发布和更新的节奏——但是最终，他们的部署只和底层代码一样好。缺乏健壮性或具有直到为时已晚才被发现的漏洞的应用程序可能会挫败敏捷开发运维的整个目的。严酷的事实是，策略和实践必须从代码开发的一开始就涉及测试和监控，同时贯穿整个 CI/CD 生命周期。

保护软件供应链在很大程度上也取决于“坚固软件”的理念事实上，这个术语已经使用了一段时间，它有助于描述在 CI/CD 环境中应该有什么样的供应链软件过程。王说，通过这种方式，它在很大程度上适用于“具有特定开发过程的软件，这样它就可以真正地去任何地方，做任何事情”。“今天，这意味着采取许多新的现代开发实践，如 CI/CD，所有这些都可以纳入‘坚固的’保护伞。”

Rushgrove 说,“加固软件”的概念是“就像你可以把[软件]带到任何地方，你可以打败它，它就会继续运行。”。拉什格罗夫说:“所以不管是对运行软件的现实有弹性的东西。"所以这个术语在运营界出现了很多次. "

[云原生安全直播，2020 虚拟峰会——保障软件供应链](https://thenewstack.simplecast.com/episodes/cloud-native-security-live-2020-virtual-summit-safeguarding-the-software-supply-chain)

马特恩说，关键是要创造出在“任何类型的环境”中都必须是可移植的软件。“当我想到这一点时，我会想到 Kubernetes。“Kubernetes 和容器是人们实现这一目标的一种机制，”Mattern 说。发布基于容器的软件的 CI/CD 过程也在发展。我们 Google Cloud 正在尝试的一种方法是向左转，将安全性引入 CI/CD 平台，并在基于 Kubernetes 的软件的发布和部署周期中构建安全性。因此，基本上就像确保您在发布软件时通过的每个检查点都包括安全检查一样…您可以放心，您的软件是安全的，并且它已经通过了贵公司同意的安全标准或您开发的软件标准。”
