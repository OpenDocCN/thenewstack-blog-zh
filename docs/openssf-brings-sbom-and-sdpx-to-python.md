# OpenSSF 为 Python 带来了 SBOM 和 SDPX

> 原文：<https://thenewstack.io/openssf-brings-sbom-and-sdpx-to-python/>

爱尔兰都柏林——在[欧洲开源峰会](https://thenewstack.io/open-source-summit-introducing-linux-foundation-europe/)上，[开源安全基金会(OpenSSF)](https://thenewstack.io/the-open-source-security-foundation-looks-to-unite-and-conquer/) 宣布，他们已经获得了实施 [SBOM 无处不在](https://thenewstack.io/sbom-everywhere-the-openssf-plan-for-sboms/)所需的资金，他们计划将[软件材料清单(SBOM)](https://thenewstack.io/securing-the-software-supply-chain-with-a-software-bill-of-materials/) 引入到 Python 中的所有编程语言和框架中。

目标是提高所有开源软件的弹性和安全性。随着对[软件包数据交换(SPDX)](https://spdx.dev/) Python 库的资助，它向 SBOM 成功迈出了第一步。该项目的工作于 9 月 1 日开始。

## SBOMs、SPDX 和 Python

[SPDX 是描述 sbom](https://thenewstack.io/spdx-software-supply-chain-spec-becomes-an-iso-standard/)的 ISO 标准。虽然 Python 已经有了 SPDX 库，但由于缺乏支持，它已经过时了。我想我们都知道，大多数开发人员不喜欢在安全方面下功夫。

[Josh Bressers](https://www.linkedin.com/in/joshbressers/) 、 [Anchore](https://anchore.com/) 安全副总裁和 [Kate Stewart](https://www.linkedin.com/in/katestewartaustin/) SPDX 技术负责人解释说:“SPDX python 库需要更新，以便与更现代版本的 SPDX 保持一致，并使代码更易于维护，从而降低社区贡献的难度。SPDX python 库缺少的是拥有合适技能或资金来完成工作的志愿者。然而，OpenSSF 确实有资金可以完成这项工作。”

一旦这样做了，为任何 Python 程序创建 SBOMs 就容易多了。反过来，这将使从代码开始到生产的整个开发过程变得更加容易。我喜欢这个计划。

## OpenSSF 控件

OpenSSF 为代码改进买单并不意味着它会控制 SPDX Python 库。这不是问题的关键。Python 软件基金会[和创始人/总裁](https://www.python.org/psf/)[吉多·范·罗苏姆](https://en.wikipedia.org/wiki/Guido_van_Rossum)仍将发号施令。OpenSSF 领导层这样做是因为保护所有开源软件的工作“不仅仅是 OpenSSF 的工作，而且将惠及整个开源社区。”

OpenSSF 总经理布莱恩·贝伦多夫在峰会上表示，更多的语言将很快获得所需的支持，将 SBOMs 作为管理开发管道不可或缺的一部分。它不能来得太快。
