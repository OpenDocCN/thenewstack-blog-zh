# 请告诉我们您的服务网格使用情况

> 原文：<https://thenewstack.io/service-mesh-3-questions/>

调查现已结束。

自 2017 年以来，新栈已经发布了超过 65 篇[服务网格](https://thenewstack.io/category/service-mesh/)文章和播客。为了准备接下来的 65 年，我们创建了一个有三个问题的[投票](https://www.surveymonkey.com/r/servicemesh1)，需要您的参与。请点击以下按钮或本页底部的按钮，参加这个由三个问题组成的调查。

第一个问题询问组织在 Kubernetes 环境中使用服务网格控制微服务的程度。然后，我们来看看服务网格对于改善分布式系统的应用流量控制、可观察性和安全性的更广泛的努力有多重要。调查结果将区分对这项被大肆宣传的技术的普遍兴趣和实际计划。

第三个问题是关于我们可以调查的其他研究问题的输入请求，无论是通过调查还是老式的皮鞋报告。根据社区的兴趣，我们可以关注诸如安全策略如何在混合环境中工作、集成控制和数据平面组件的重要性以及特定技术和供应商解决方案的采用率等主题。

新的堆栈计划进行更多的小型调查，以提供数据知情的背景下，未来的编辑报道。如果您或您的公司有兴趣对云原生主题进行更深入的调查研究，请联系 [Lawrence Hecht](mailto:research@thenewstack.io) 。
