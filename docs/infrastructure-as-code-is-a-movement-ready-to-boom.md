# 作为代码的基础设施是一场准备蓬勃发展的运动

> 原文：<https://thenewstack.io/infrastructure-as-code-is-a-movement-ready-to-boom/>

该播客由来自帕洛阿尔托网络公司的 [Prisma Cloud](https://www.paloaltonetworks.com/prisma/cloud) 赞助，先于[构建可扩展的云安全战略:1 月 26 日的虚拟活动](https://register.paloaltonetworks.com/building-scalable-strategy-for-cloud?utm_source=thene%5B%E2%80%A6%5Dte&utm_campaign=PANWDigitalSummit-Jan2021-Registration)。

作为代码的基础设施是一场准备蓬勃发展的运动。帕洛阿尔托网络公司 Prisma Cloud 高级副总裁 [Varun Badhwar](https://www.linkedin.com/in/vbadhwar) 在本期由 TNS 创始人兼发行人 [Alex Williams](/author/alex/) 主持的 [The New Stack Makers](/podcasts/makers) 节目中表示，它也正在成为云安全的三大支柱之一，在不断发展的 DevSecOps 市场中将开发运维与安全结合在一起。

[基础设施为代码是一个运动准备好的吊杆](https://thenewstack.simplecast.com/episodes/infrastructure-as-code-is-a-movement-ready-to-boom)

作为代码的基础设施也是 DevOps 左移趋势的主要组成部分。Badhwar 说:“左移安全性现在意味着应用程序安全性，意味着软件组合分析，意味着代码扫描等基础设施——所有这些现在都可供 DevOps 团队在管道中进行。”

Badhwar 解释说，我们的目标是为 DevOps 团队使用的工具制定一套政策，他将在讨论 Prisma Cloud 下一波平台创新的虚拟活动[Building A Scalable Strategy for Cloud Security:A Virtual Event](https://register.paloaltonetworks.com/building-scalable-strategy-for-cloud?utm_source=thene%5B%E2%80%A6%5Dte&utm_campaign=PANWDigitalSummit-Jan2021-Registration)中担任主讲人。然而，尽管作为代码的基础设施应该越来越多地在 DevSecOps 世界中占据一席之地，但是为了使它以最好地满足组织的安全需求的方式发展，挑战仍然存在。

Badhwar 说:“在我与数百名客户的交谈中，我认为安全团队仍在寻找如何与 DevOps 团队合作，真正说同一种语言，并共享一个通用的工具集。“问题是 infosec 团队使用的大多数工具都是以运行时为中心的。它们不是真正为代码带来的基础设施这一范式转变而构建的。”

Badhwar 说，以前，DevOps 中的安全性主要是关于部署，当时“我们过去生活在一个许多软件产品和应用程序每年或每半年发布一次的世界中”。

“我认为很明显，企业需要更加敏捷，更快地行动，跟上业务发展的步伐，否则就有可能变得无关紧要，被那些知道如何更快行动的公司挤垮。[这是]因为很多都是关于如何进行开发和部署实践，并使它们更加精简和迭代——我认为这就是这个世界开始的地方，”巴德瓦尔说。“如果你看看今天的 it，它已经从非常以部署和开发为中心转变为我所说的全周期和全堆栈，其中安全性显然是很重要的一部分-可观察性、性能、可扩展性、质量，都开始以某种形式或方式融合在一起。”

开源对 DevSecOps 的影响，以及对作为安全代码的基础设施的影响，也不容忽视。Badhwar 说，虽然该软件以前是基于专有的商业模式，但 DevOps 团队此后“不仅带来了智能和自动化，而且还为 DevOps 带来了社区、共享和众包”。“他们发现的一件大事是为什么每次都要重新发明轮子——现在你看到大多数代码都是开源的，很少是专有的，”他解释道。“所以就有了分享的概念……你猜怎么着？我去码头中心或其他地方，我通常会发现不同类型的预包装集装箱已准备好供我挑选和使用。”
