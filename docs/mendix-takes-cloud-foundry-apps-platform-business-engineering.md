# Mendix 如何将 Cloud Foundry 从应用平台转变为“业务工程”

> 原文：<https://thenewstack.io/mendix-takes-cloud-foundry-apps-platform-business-engineering/>

随着大规模部署和管理应用程序的想法逐渐成熟，组织应该重新考虑他们的业务理念了。在某种程度上，将他们的商业运作放在平台上意味着将他们的商业模式也转移到这些平台上——这可能更像心脏*和*肺移植，而不仅仅是后者。

在圣塔克拉拉举行的上一届 Cloud Foundry 峰会期间， [Mendix](https://www.mendix.com/) 首席技术官 [Johan den Haan](https://twitter.com/JohanDenHaan) 讨论了他的公司的移动应用开发平台——围绕 Cloud Foundry nucleus 构建——是如何根据这一新兴事实进行设计的。他还谈到了在开发过程中，用业务经理取代专业程序员是多么重要。

“某些类型的应用，企业应用，可以使用我们平台的视觉模型来构建，”den Haan 在接受新堆栈制造商播客采访时说。“但在某些情况下，你需要扩展它。因此，我们也支持在前端使用 Java 或 JavaScript 扩展您的模型。这意味着，如果你有某种你需要自己构建的集成，或者特定的算法，你仍然可以使用传统的编程语言来扩展它。

“所以我把它看做一个光谱，在那里你有可以由公民开发者构建的应用程序，”den Haan 继续说道，“可以由更懂技术的业务工程师、领域专家构建[*和*。”这一评论促使联合创始人兼 Cloud Foundry Foundation 技术副总裁 [Chip Childers](https://twitter.com/chipchilders) 强调*业务工程*与 Mendix 平台的相关性。
