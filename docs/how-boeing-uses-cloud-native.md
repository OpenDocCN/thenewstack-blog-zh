# 波音如何使用原生云

> 原文：<https://thenewstack.io/how-boeing-uses-cloud-native/>

在最新的[播客](https://thenewstack.io/podcasts/)中，我们采访了[瑞卡多·托瑞斯](https://www.linkedin.com/in/ricardo-torres-96075471/)，他是航空业巨头[波音](https://www.iafastro.org/membership/all-members/the-boeing-company.html)的开源和云原生首席工程师。Torres 也在 5 月加入了[云计算原生计算基金会](https://cncf.io/?utm_content=inline-mention)，担任董事会成员。在上个月录制于 [KubeCon+CloudNativeCon](https://www.cncf.io/kubecon-cloudnativecon-events/?utm_content=inline-mention) 的采访中，Torres 谈到了波音公司对[开源软件](https://thenewstack.io/the-challenges-of-securing-the-open-source-supply-chain/)的使用，以及对[云原生技术](https://thenewstack.io/what-is-cloud-native-and-why-does-it-matter/)的采用。

虽然我们可能会认为波音是一家飞机制造商，但更准确的说法是，该公司是一家大规模系统集成商，使用大量软件。因此，像其他大公司一样，波音公司看到了与开源社区保持良好关系的明显优势。

[波音如何使用云原生](https://thenewstack.simplecast.com/episodes/how-boeing-uses-cloud-native)

“能够利用世界其他地方最优秀的技术专家对我们来说具有重要的战略价值，”托雷斯说。这一战略使波音公司能够“区分我们作为核心业务所做的事情，而不是一直在所有技术上重新发明轮子。”

像许多其他大公司一样，波音公司创建了一个开源办公室来更好地与开源社区合作。虽然波音公司主要是开源软件的消费者，但它仍然希望与社区合作。“我们希望确保我们有一个关于如何回报开源社区的战略，然后利用这些知识进行内部采购，”他说。

波音还管理它如何在内部使用开源软件，对它使用的开源软件的供应链保持严格的控制。“作为软件工程组织的一部分，我们与内部 IT 组织合作，检查我们的互联网流量，确保没有人会直接从不受信任的存储库或注册表中下载。然后我们主持，我们有内部批准的来源。”

毫不奇怪，与许多政府机构打交道的波音公司接受了[使用软件材料清单](https://thenewstack.io/how-to-prepare-your-apps-for-regulated-markets/)(sbom)的做法，这提供了软件系统中使用的组件的完整列表。事实上，根据 Torres 的说法，该公司一直在努力扩展 SBOMs 的全面性。

“我认为现在有趣的事情之一是自动化，”他谈到 SBOMs。“因此，我们一直在寻求加强启发式，因为许多工具相对来说是幼稚的，它们相信指定的依赖关系实际上代表了交付的一切。对于波音这样的公司来说，这还不够好。我们必须绝对肯定，那里的东西正是我们所期望的。”

## 云计算原生计算

虽然波音公司建立了许多驻留在私有数据中心的系统，但该公司也越来越依赖云。今年早些时候，波音公司与三大云服务提供商(CSP)签署了协议:[、亚马逊网络服务](https://aws.amazon.com/?utm_content=inline-mention)、微软 Azure 和谷歌云平台。

“我们的很多云都与我们的开发环境有关。所以，你知道，我们有基于云的软件工厂，它们使用许多 CNCF 和 CNCF 邻近的技术，使我们的开发人员能够快速前进，”Torres 说。
