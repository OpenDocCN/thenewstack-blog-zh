# 背景:对观察先锋慈善专业学生的采访

> 原文：<https://thenewstack.io/context-an-interview-with-observability-pioneer-charity-majors/>

[采访观察先锋慈善专业学生](https://thenewstack.simplecast.com/episodes/an-interview-with-observability-pioneer-charity-majors)

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论来自云原生社区的最新新闻和观点。本周，我们与 [Charity Majors](https://twitter.com/mipsytipsy) 、Honeycomb.io 的首席技术官和可观察性的先驱谈论这场运动的三周年纪念日，这场运动极大地改变了在云原生时代监控和维护应用程序的方式。

Majors [发表了一篇文章](/observability-a-3-year-retrospective/),这篇文章出现在本周的新堆栈上，在这篇文章中，她仔细研究了为什么可观察性运动会形成以及为什么其他方法和手段会失败。她解释了为什么工程团队必须继续关注这种方法，以便更好地理解、调试和维护大规模分布式系统。

“当我们把这个庞然大物分解成许多服务时，我们失去了用调试器单步调试代码的能力:它现在在网络上跳跃。梅杰斯写道:“我们的工具仍在努力应对这种巨大的转变。

然后在节目的稍后部分，我们将讨论本周网站上的热门播客和故事，包括 [Mesosphere 的更名](/d2iq-formerly-mesosphere-brings-enterprise-management-to-kubernetes/)，以及[对 Michael Crosby](/qa-dockers-michael-crosby-on-how-libcontainer-enabled-kubernetes/) 的采访，Michael Crosby 是 Docker 核心团队的工程师，负责 libcontainer 内核接口和 runc 容器运行时，并监督这些项目周围的开源社区。

TNS 的编辑和营销总监 Libby Clark 在 New Stack 的创始人兼发行人 Alex Williams 和 New Stack 的执行主编 T21 的帮助下主持了这个播客。
