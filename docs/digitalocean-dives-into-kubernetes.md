# 数字海洋潜入库伯内特斯

> 原文：<https://thenewstack.io/digitalocean-dives-into-kubernetes/>

以开发者为中心的云提供商 DigitalOcean 希望通过一个新的由 Kubernetes 托管的产品来吸引更多的用户。[digital ocean](https://www.digitalocean.com/)[Kubernetes](https://kubernetes.io/)将在其现有云计算和存储选项的基础上提供免费的容器管理和编排平台。用户不必为使用这个 Kubernetes 管理层支付额外费用，只需支付标准计算费用。

“最大的驱动因素是，我们有许多企业客户试图扩展和容器化他们的应用程序。DigitalOcean 产品副总裁 Shiven Ramji 表示:“他们找到我们说，‘我喜欢你的平台，但你没有对 Kubernetes 的本地支持，所以我必须自己管理这一切。

他接着说“第二个驱动因素是有很多开发者不知道或者不理解 Kubernetes 的复杂性。有很多开发人员不熟悉并有点害怕转向容器世界，这是 DigitalOcean 提供价值的另一个领域。我们相信我们可以让产品更容易使用，我们可以推动学习和发展。我们可以让那些还没有容器化的开发者变得更容易，这样他们就有了一条通往容器化的道路。

随着今天的宣布，DigitalOcean 已经开放注册，可以提前使用其 Kubernetes 产品。今年晚些时候，该公司希望向公众开放该平台。该平台还支持将 Kubernetes 绑定到外部系统，允许用户使用相同的管理系统构建内部和外部协调的混合系统。

这第一个版本已经包括支持集成到 DigitalOcean 的块和对象存储服务，确保开发人员不必手动建立到持久数据存储库的管道。“我们的初始版本将与我们的原语集成在一起。在这里，我们将把它与块存储紧密结合，并与我们的对象存储产品轻松集成。随着时间的推移，在这两个地方集成数据即服务将变得非常容易。我们也会让用户更容易做到这一点，”拉姆奇说。

DigitalOcean 还通过将其会员级别从银牌升级到金牌来加强对[云本地计算基金会](https://www.cncf.io/) (CNCF)的支持。该公司还向 Kubernetes 项目和其他内部使用的 CNCF 项目贡献代码，如 [OpenTracing](http://opentracing.io/) 和 [GRPC](https://grpc.io/) 。

该公司在本周于哥本哈根举行的 CNCF kube con+CloudNativeCon Europe 上宣布了这项服务。

DigitalOcean 和[Cloud Native Computing Foundation](https://www.cncf.io/)是新堆栈的赞助商。
