# 新的堆栈环境:云中的无服务器应用流

> 原文：<https://thenewstack.io/the-new-stack-context-serverless-application-flows-in-the-cloud/>

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论云计算领域的最新新闻和观点。在本周的节目中，我们采访了 [Sebastien Goasguen](https://twitter.com/sebgoa) 、 [TriggerMesh](https://triggermesh.com/) 联合创始人兼首席产品官，讨论如何使用该公司的软件，从跨多个云的无服务器功能构建应用程序。

TNS 编辑和营销总监[利比·克拉克](https://thenewstack.io/author/libby/)主持了这一集，旁边还有 TNS 创始人兼发行人[亚历克斯·威廉姆斯](https://thenewstack.io/author/alex/)，TNS 高级编辑[理查德·麦克马努斯](https://thenewstack.io/author/richard/)，以及 TNS 执行主编[约阿布·杰克逊](https://thenewstack.io/author/joab/)。

[第 115 集:云中无服务器应用流](https://thenewstack.simplecast.com/episodes/episode-115-serverless-application-flows-in-the-cloud)

我们与 Goasguen 讨论了 TriggerMesh 对于 GitLab 和企业客户的作用。上个月，TriggerMesh 发布了[云原生集成平台](https://triggermesh.com/cloud_native_integration_platform/)以及用于 OpenShift 的 [AWS 事件源，发布时间与虚拟红帽峰会一致。通过后一种产品，TriggerMesh 为 OpenShift 生态系统带来了类似亚马逊 EventBridge 的功能，允许开发人员跨云和传统数据中心触发功能。TriggerMesh 用户现在可以将任何地方的事件链接到 Red Hat OpenShift 工作负载。](https://triggermesh.com/2020/05/sources-from-amazon-web-services-saws/)

“无服务器不仅仅是功能即服务。不仅仅是功能。这其实是一个整合的问题。我们将 TriggerMesh 称为云原生集成平台:我们将云服务组合在一起，借助事件驱动的架构将它们粘合在一起，”Goasguen 说。

然后，在稍后的播客中，我们讨论了该网站的热门播客和新闻故事，包括[对敏捷专家 Emily Webber 关于远程工作的采访](https://thenewstack.io/agile-coach-emily-webber-on-fostering-inclusion-for-remote-work/)、无服务器[如何帮助将安全性](https://thenewstack.io/agile-coach-emily-webber-on-fostering-inclusion-for-remote-work/)嵌入开发过程、[提供数据库作为无服务器服务的想法](https://thenewstack.io/why-your-next-database-will-be-a-serverless-api/)，以及标准在无服务器应用中的重要性[。](https://thenewstack.io/serverless-needs-standards-to-be-the-future-of-application-infrastructure/)

Red Hat 和 TriggerMesh 是新堆栈的赞助商。
