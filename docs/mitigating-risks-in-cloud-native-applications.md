# 降低云原生应用中的风险

> 原文：<https://thenewstack.io/mitigating-risks-in-cloud-native-applications/>

二十年前，安全是事后的想法；它通常被“附加”到现有的应用程序上，使企业对威胁可见性和实施采取被动的方法。但是，随着[云原生应用](https://thenewstack.io/10-key-attributes-of-cloud-native-applications/)和采用随处工作模式的企业的激增，传统的安全方法正在被重新设想，以在从开发到运营的整个过程中发挥不可或缺的作用。从识别、评估、区分优先级和适应应用程序中的风险，组织通过在整个生命周期中采用安全性，逐渐全面了解其风险状况。

在本期 New Stack Makers 播客中，[总裁兼首席执行官 Ratan Tipirneni](https://www.linkedin.com/in/ratantipirneni/)， [Tigera](https://www.tigera.io/) 讨论了组织如何通过引入零信任原则来减少应用的攻击面，利用机器学习来应对运行时安全风险，并实现持续合规，同时通过安全策略的变化来降低漏洞和攻击的风险，从而采取积极的安全措施。

新书库的创始人兼发行人亚历克斯·威廉姆斯主持了这次播客。

[降低云原生应用的风险](https://thenewstack.simplecast.com/episodes/mitigating-risks-in-cloud-native-applications)

根据一份 [451 研究报告](https://cpl.thalesgroup.com/cloud-security-research)，全球 40%的组织在过去 12 个月中遭遇过基于云的数据泄露。“零日攻击中的软件组件、漏洞和恶意软件的数量正在飙升，”Tipirneni 说。“已知和未知的威胁层出不穷。但是，即使你做得一切都是正确的，你仍然会遇到一些违规行为，需要自动降低环境中的风险，以保持应用程序的运行，”Tipirneni 说。

Tipirneni 说，随着云原生环境的发展，应对这些新挑战需要新的“云原生应用程序安全性的架构构建模块”。Tipirneni 补充说，一种持续监控合规性、搜索漏洞、从开发到生产减少攻击面的方法需要“零信任基础上的安全策略，以便您可以最大限度地减少您的攻击面”。

Tipirneni 说，随着向随处工作模式的转变成为主流，云应用程序继续激增，it 正在重新定义新的发展，如“安全性和可观察性正在融合”。虽然开发运维与 IT 安全传统上被视为独立的学科，但它们的角色和职责正日益向开发运维趋势发展。Tipirneni 说:“解决安全问题和可观察性问题是你能够在一个非常精细的层次上检测系统中发生的一切——从收集数据到真正理解数据。“开发人员试图绕过复杂的安全控制，但将这两者结合在一起会将权力交到开发人员手中，”他补充道。

信息安全和开发团队传统上管理 Tigera 的解决方案，如 Calico 和 Envoy，但对于没有传统应用程序的云优先公司来说，“开发运维，云运营工程师端到端非常负责，”Tipirneni 说。从部署应用程序到故障排除以及管理合规性和安全性，“他们面临的挑战是他们有太多的事情要做，”Tipirneni 补充道。

Tigera 已经采取了这种方法，“给不同的团队不同级别的权限，以正确的权限级别参与安全性和可观察性的设计和管理，以便开发人员可以设置安全权限和规则，”他补充道。

一个关键的挑战仍然存在:“数据量太大了，”蒂皮涅尼说。Tigera 致力于获得“关于工作负载在进程级别的行为、文件系统访问、系统调用和网络行为的实时数据。”通过在这些级别中的每一个级别上寻找偏差，“就可以确定折衷的指标。这也是该公司下一步投资和创新的目标，”Tipirneni 补充道。
