*   Jayendra 的博客:这是一张可用资源的伟大地图。这些数据大部分可以在 AWS 文档网站上找到，但是这是一个很好的提炼；

## 我的建议

### 这些问题很奇怪

在任何选择题测试中，当你没有立即看到正确答案时，你应该排除其他可能性。AWS 考试更是如此。有些问题的措辞非常奇怪。我举一个概括的例子，假设考试是关于图书馆学的:

问:一个用户想要查找大量的单词以找到它们的定义，哪个选项适合这个目的？

1.  同义词库；
2.  卡片目录；
3.  犹太法典的伪经；
4.  书籍。

以下哪一个是正确的？答案是 4:书。选择 1 和 2 都是可行的，但问题是“这是为了什么:无论是词典还是卡片目录都不是为了这个。选项 3 指的是不存在的东西。因此，选项 4，尽管是一个非常无用和笼统的答案，却是唯一正确的答案。你会用书。你不会用杂志，你会用书。

我记得测试中的 AWS 特定示例提到了几个计算选项的名称，但正确的答案只是“AWS Compute ”,因为特定的答案是错误的。

### 随机考试意味着问题分布不均匀

我是一个很好的应试者，这真的让我很困惑。当参加一个所有问题都由人来选择的测试时，我脑海中有一个想法说:“我们还没有关于 Elastic Beanstalk 的任何问题，这是一个顶级产品，所以一些问题正在出现。”但是，虽然 AWS 云从业者考试的问题是由人类编写的，但选择是随机的——你可能在 AWS 雪球上答对了三个问题，但在 S3 上没有。

这也意味着你可以得到两个本质上相同的问题。这又一次让我感到困惑。如果有人选择了这些问题，我会认为问我:

*   “以下哪些是缩略图的大容量存储选项；”

其次是:

*   “以下哪一项是存储图像缩略图的最佳方式。”

这意味着问题之间有细微的差别；一个我本该确认的会改变我答案的问题。没有。抽签的运气。从分布中提取没有意义。

## 该测试有一些关键主题

该测试被描述为“对 AWS 服务和产品的熟悉程度”，但基本上有四组关键知识:

*   使用哪个计算选项？我再次计划写一整篇关于这个的文章:
*   哪些数据库选项？哪些类型的数据库格式可以在哪些 AWS 产品上托管；
*   哪些支持级别与哪些服务一起提供？这个问题可以通过本页来回答[，所以也许可以把它放在一些闪存卡上？](https://www.amazonaws.cn/en/support/compare-plans/)
*   谁负责加密？当您在 AWS 上托管时，谁管理什么的变化。参见下面的**了解亚马逊何时会说“不是我的问题”**部分。

## 名字很重要

花点时间了解一些顶级服务的确切名称，因为你会被问到是弹性街区商店还是弹性街区服务(这是一家商店)。

也就是说，它不会变得太具体。我最近了解到，有些产品名称前面应该有“AWS ”,而有些没有，这不在测试中。

## 了解亚马逊何时会说“这不是我的问题”

你会得到很多关于“亚马逊有责任管理这些东西中的哪一个？”如果你运用一点常识，这些都是很简单的问题。对于 EC2 中托管的虚拟机(VM ), Amazon 不负责虚拟机中运行的代码。这包括操作系统。当您启动时，AWS 可能会让您选择一个特定的 Windows 服务器映像安装在您的虚拟机上，但保持它的更新不是 AWS 的责任。这就是[亚马逊的“共享责任模式”](https://aws.amazon.com/compliance/shared-responsibility-model/)，它与承包或自有的数据中心有很大不同。

另一方面，Amazon *将*保证运行 VM 的服务器的物理安全。没有一个亚马逊服务会有保安突击费。

一般来说，记住 AWS 希望参加这个考试的人是一个好主意:在一家大型企业工作的人，他提倡使用 AWS。AWS 方面的担忧是，当 AWS 的云服务不要求你维护物理基础设施时，他们的服务会以某种方式与数据中心服务混淆，如果你对自己的需求过于具体，亚马逊甚至不会让你深入检查。

这里有一个很好的比喻，AWS 服务就像麦当劳的菜单项:它们很便宜，任何规模都可以买到。你不需要担心每一种成分来自哪里，如果你问；"我能参观这个汉堡的马场吗？"人们会奇怪地看着你。

## 了解实用工具玩家

虽然亚马逊的计算、数据库和存储服务有不同的技术规格，但有几项服务与业务相关，对所有客户都有用。以下是我想到的一些:

*   [AWS 工件](https://aws.amazon.com/artifact/):记录您的标准(例如 HIPAA)符合性；
*   [CloudWatch](https://aws.amazon.com/cloudwatch/) :查看你的服务日志，设置价格警报；
*   [CloudTrail:](https://aws.amazon.com/cloudtrail/) “昨晚我们的哪个用户提供了 800 个新的数据库实例？”以及这个问题的变体。基本上记录任何 AWS 配置更改；
*   [CloudFormation:](https://aws.amazon.com/cloudformation/) 将您对 Lambdas 和其他服务的配置存储在一个模板文件中，以实现复杂堆栈的一致部署。
