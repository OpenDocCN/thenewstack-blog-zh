# 在疫情的这些日子里学到的云原生经验

> 原文：<https://thenewstack.io/cloud-native-lessons-learned-during-these-pandemic-days/>

在过去的一年里，世界确实发生了变化。始于疫情的我们生活和工作方式的范式转变，为众多组织带来了跨云本地和分布式环境扩展运营的更大推动力。这在很大程度上是为了应对远程工作人员和开发人员的激增，他们的分布式数据和应用程序需求也相应增加。

在这个由新堆栈的创始人和发行人 [Alex Williams](/author/alex/) 主持的 [The New Stack Makers](/podcasts/makers) 播客中，Veeam 旗下 [Kasten 的首席技术官](https://www.kasten.io?utm_content=inline-mention) [Vaibhav Kamra](https://www.linkedin.com/in/vaibhavkamra) 讨论了他观察到的变化，并最终讨论了过去一年中吸取的教训。在此期间，Kasten 为应用程序和数据管理提供了必要的平台，组织依靠这些平台来扩展 Kubernetes 应用程序。

不变的是组织如何依赖 Kasten by Veeam 的平台在 Kubernetes 上部署其应用程序，同时安全地管理其有状态数据，同时使用该平台进行大规模部署，通常在多云和混合环境中进行。

卡姆拉说:“你可以大规模地运行这些应用程序，而不用担心数据丢失、勒索软件或任何其他你在 Kubernetes 上运行应用程序的事情。”

但是，客户的扩展需求经常发生巨大变化。卡姆拉说，他记得“三月初或二月底坐在办公室里，为我们将如何对待客户制定所有这些计划。”这些计划包括当“一两周内，对话完全改变”时，如何在 Kubernetes 上更好地安全扩展。

“谈话从‘我们正在做 Kubernetes，我们正在扩张，我们真的对向云转移很感兴趣’突然变成‘等等，我们没时间谈了。我们需要想办法确保一切正常运转，”卡姆拉说。“最终，在我们度过第一季度后，我认为对话发生了很大变化，变成了‘我们真的希望加快向云的迁移。’"

随着客户开始更快地转向云原生环境，“我们在 Kubernetes 跟踪的所有这些项目，就采用而言，刚刚真正[开始] …满怀热情地向前推进，”Kamra 说。

Kamra 解释说，进入疫情几个月后，由于迫切需要“确保一切都在运行，这样他们就可以在家操作一切，并继续做他们正在做的事情”，所以迅速转向云原生。这一时期之后，企业开始寻求扩展和降低成本，同时提高灵活性，以便客户能够根据需要扩展和缩减基础架构。

“人们开始更多地关注他们的云原生投资真正会给他们带来什么，”Kamra 说。

同时，如何在存储和数据库的云原生环境中管理有状态数据是 Kasten 继续支持其客户的方式之一，正如它在疫情之前和期间所做的那样。这种支持包括强调最佳实践和帮助工具选择过程，包括如何最好地利用 Kasten 的平台。

卡姆拉说，归根结底，一旦一个组织有了可靠的数据管理流程，“它就会给你想要的安全和安心”。
