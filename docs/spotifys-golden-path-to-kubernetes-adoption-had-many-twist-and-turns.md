# Spotify 采用 Kubernetes 的黄金之路有很多曲折

> 原文：<https://thenewstack.io/spotifys-golden-path-to-kubernetes-adoption-had-many-twist-and-turns/>

Spotify 因其音乐服务而闻名全球。不太为人所知的是，它通往 Kubernetes 部署的道路是一条曲折的道路。

可能让许多人感到惊讶的是，Spotify 是 Kubernetes 的资深用户，它的产品交付能力在很大程度上归功于其敏捷的 DevOps。事实上，Spotify 继续越来越依赖于容器和微服务基础设施以及云原生部署来提供许多优势。这使得其 DevOps 团队能够持续改善数百万用户的整体流媒体体验。

在本期的 [The New Stack Analysts](/podcasts/analysts) 播客中，作为 New Stack 最近对终端用户 Kubernetes 报道的一部分，[基础设施和运营负责人 Jim Haughwout](https://www.linkedin.com/in/haughwout/?originalSubdomain=se) 分享了 Spotify 的云原生采用战争故事，并讨论了其过去和现在的 Kubernetes 挑战。 [Alex Williams](/author/alex/) ，新书库创始人兼发行人；[云原生计算基金会(CNCF)](https://uk.linkedin.com/in/cheryljhung) 生态系统副总裁 Cheryl Hung；以及[云原生工程副总裁 Ken Owens](https://www.linkedin.com/in/kenowens12/) 、[万事达](https://www.mastercard.us/en-us.html)都主持了播客。

[为什么 Spotify 采用 Kubernetes 的黄金之路有很多曲折](https://thenewstack.simplecast.com/episodes/why-spotifys-golden-path-to-kubernetes-adoption-has-many-twist-and-turns)

自几年前采用 Kubernetes 以来，Spotify 每月都在继续扩大对它的使用。此前，Spotify 在开始考虑 Kubernetes 可能带来的潜在好处之前，就已经开始将其运营转向集装箱化的基础设施。

Haughwout 说:“当我们最初关注 Kubernetes 时，我们处于一个有趣的境地，因为我们已经有了自己构建的内部流程编排解决方案，而且，有趣的是，在 Kubernetes 推出的同一周，我们也推出了开源[替代方案]。”“因此，我们做了大量工作，从根本上使开发人员过渡到 Kubernetes 变得非常容易，并使我们能够让数百个团队安全地在共享集群中一起工作。”

尽管 Spotify 很早就被采用，但大约一年半前，它开始“认真地”转向 Kubernetes。此后，Kubernetes 在 Spotify 的 DevOps 中以两种关键方式发挥了关键作用。豪格沃特说，这包括该平台如何帮助“减少辛劳”。

他解释说:“我们希望让必须管理基础设施的工程师不再需要担心规模的扩大和缩小，这样他们就可以简单地构建和部署功能了。”

Kubernetes 提供的第二个主要优势是云原生基础设施的采用如何使音乐流媒体巨头增加了许多新的工具和平台，以改善其生产渠道和运营。Haughwout 说:“我们一直在努力与云计算原生计算基金会合作的一个重要原因是，让采用大量基础设施变得容易。”“Kubernetes 已经成为云原生技术的一种通用语，为我们进入许多其他技术打开了大门。”

在过去的一年里，Spotify 一直在扩大其在 Kubernetes 上运行的服务数量，同时利用其高度分布式的结构。例如，Haughwout 描述了 Spotify 如何将数据管道和机器学习转移到 Kubernetes，并“依靠它来建立短暂的环境”。

当然，Kubernetes 的采用之路充满了挑战。在 Spotify 的案例中，Haughwout 列举了与授权数百个自主工程团队“尽快行动”相关的挑战。

“他们正致力于构建和迭代人们每天在 Spotify 上使用的功能和实验，我们希望他们继续努力，”Haughwout 说。“因此，我们面临的挑战之一是，当你每月有 2.99 亿用户时，我们如何迁移他们，而不中断像 Kubernetes 这样的音乐流，也不减慢他们的速度。”
