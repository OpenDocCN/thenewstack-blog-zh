# 开源安全的石器时代

> 原文：<https://thenewstack.io/the-stone-ages-of-open-source-security/>

问一个开发人员他们是如何进入编程的，你会对他们了解很多。

开源安全的石器时代

在本周的科技创始人奥德赛播客系列[中，Chainguard](https://www.chainguard.dev/) 创始人[丹·洛伦克](https://www.linkedin.com/in/danlorenc/)表示，他在大学中途开始编程，同时学习机械工程。

“我从事编程是因为我们必须在 MATLAB 中做模拟和其他事情，”洛伦茨说。“然后我转向 Python，因为它很相似。我们不需要那些许可证或任何我们需要的东西。然后我就想，‘哦，这比你知道的要快得多，订购零件，去机械商店，节省时间，’所以我就这样进入了状态。”

四年前，Lorenc 涉足开源安全领域。

“开源安全和供应链安全在当时并不流行，”洛伦茨说。“没有人在谈论它。我对此有些偏执。”

洛伦茨曾在谷歌从事 Minikube 开源项目，在那里他第一次看到从事开源项目是多么的不安全。在采访中，他谈到了他在那部作品中看到的威胁。

对洛伦茨来说这太奇怪了。开源安全的最新技术根本不是最新技术。那是石器时代。

Lorenc 说，他觉得在 MiniKube 中构建第一个没有提出安全问题的版本很奇怪。

“但我的意思是，这就像一个 200 兆字节的 Go 二进制文件，人们只是在 Kubernetes 社区的笔记本电脑上以 root 用户身份运行，”Lorenc 说。“没有人知道我在里面放了什么，如果它与 GitHub 上的来源或任何东西相匹配。所以那是非常可怕的。这让我对空间产生了偏执，有点像掉进了这个长长的兔子洞，最终导致了 Chainguard 的出现。”

今天，世界正在燃烧，这对像 Chainguard 这样的安全初创公司来说是一件好事。

“是的，我们有一个混乱的行业需要解决，”洛伦茨说。“如果你一直在关注新闻，可能会发现由于所有这些安全问题，软件行业正在着火或倒下。这对很多人来说是个坏消息，但如果你在安全领域，这是个好消息。”

好消息，是的，但它如何融入一个更大的故事？

“现在，我们的一大重点是弄清楚我们如何解释我们在更大的景观中的位置，”洛伦茨。说道。“因为安全市场规模庞大，令人困惑，到处都是厂商在他们的网站上放一些流行语，比如零信任之类的。在那种混乱中很容易迷路。因此，弄清楚我们如何定位自己，我们如何处理品牌、营销，并让潜在客户和社区成员清楚我们在做什么，我们的产品减轻了哪些威胁，以确保我们在这方面是准确的。并传达给我们的客户。这是我目前的主要关注点。”
