# 发布经理——持续交付的独角兽

> 原文：<https://thenewstack.io/release-managers-the-unicorns-of-continuous-delivery/>

好的发布经理是难以捉摸的，特殊的，有神奇的管理力量。然而，像独角兽一样，好的发布经理通常很难找到。那么，公司如何才能雇佣一个神奇的发布经理来确保他们找到合适的人，并给他们最好的工作环境和工具呢？让我们从基础开始。

## 什么是发布经理？

 [Jeff Keyes，Plutora 产品营销总监

杰夫·凯斯是普卢托拉公司的产品营销总监。Keyes 的职业生涯包括编写代码、设计软件功能和 UI、管理开发和测试团队、咨询和宣传产品信息。在微软工作的六年之外，他主要关注成长型初创公司。](http://www.plutora.com/) 

发布经理通常负责将不同的开发人员、操作和项目组合到一个紧密的时间表中，该时间表考虑了资源、最终目标和公司范围的时间表。具体来说，他们拥有发布管理生命周期，包括从项目开始一直到生产的应用程序交付的计划、协调和总体管理。他们必须在整个企业产品组合中这样做。在企业中，每个版本都由数百个并发运行的项目组成，这些项目混合了供应商团队、地理上分散的开发和测试、开发方法、混合云以及虚拟化和物理基础架构。他们必须与所有参与者协作，协调项目和时间表之间的依赖关系，以及开发和测试工作的相关预生产环境。

似乎这还不够，在生命周期本身之外，发布经理还负责向公司高层汇报并更新信息，例如 IT 高层领导、首席信息官和首席技术官，以及 IT 组织之外的业务领导。现在想象一下，当不可避免的项目延迟、范围变更、环境中断或严重错误出现时，他们的世界会发生什么。

这归结为一个有效的发布经理必须有深入的技术知识，并且是组织的主人——看，一个完全的独角兽。这些技能的结合将使他们能够与各种团队和人员一起工作，同时还监督发布生命周期的技术方面。此外，他们需要能够将所有这些信息提升到面向 IT 和业务高级团队成员的演示或报告中。

一旦你找到了你的独角兽，你如何确保这个过程顺利进行并提供真正的商业利益？

## 发布管理最佳实践

发布管理过程是开发和测试/生产之间的关键桥梁，它的存在是为了确保实现涉众所期望的服务功能。itSMF ITIL V3 框架提供了发布管理最佳实践的一般概述，然后可以对其进行定制和调整，以满足特定组织的需求，包括:

*   发布计划:发布经理设计并正式批准发布政策指南，该指南不仅包含治理方面，还包含在组织间实现发布的详细过程设计。
*   构建和配置发布:在这个阶段，发布经理监督故事的编码方面的主要方面——从代码的构建，到确保构建完整性的初始测试，最后在目标环境中交付。发布经理在那里确保所有的团队都在同一页上，并保持在轨道上，以满足商定的目标。当应用程序在与目标生产不匹配的测试环境中进行测试时，项目挑战经常出现——不要跳过这一步。
*   质量评审:发布经理需要确保 QA 团队评审发布的质量，确保它满足最低可接受的标准和业务需求。他们通常评估与功能的关键程度相互参照的项目的历史跟踪记录，以决定将他们的时间集中在哪里。
*   推广计划:一旦 QA 团队接受了一个版本，它就可以部署到生产环境中，最终用户或客户将能够访问发布的新功能。发布经理在这里确保从内部到面向客户的顺利推出。移交和上线计划应包括批准。
*   实现发布:在这个阶段，在 SDLC 测试阶段测试过的发布单元被部署到生产环境中，以供实时使用。同样，发布经理在这里是为了确保平稳过渡到现场使用，并帮助监控发布，并确保所有继续平稳运行和无错误。

通过实现上述以及其他的最佳实践和指导方针，组织可以最大化发布过程的效率，同时节省资金并提高服务的商业价值。

如果你没有真正了解独角兽是什么，你可能会把它误认为是马，反之亦然。同样的想法也适用于发布经理。许多组织没有雇佣合适的人，因为他们不明白角色职责到底是什么。然而，如果使用得当，发布经理是对一个组织的成功至关重要的神奇生物。
