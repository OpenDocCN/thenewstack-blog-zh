# 以开发人员喜欢的方式保护微服务

> 原文：<https://thenewstack.io/secure-microservices-in-ways-that-developers-like/>

云提供商在过去几年中开始提供的服务数量呈爆炸式增长，这可能会使数量呈指数级增长的微服务暴露于漏洞，而这些微服务在多个云和内部环境中支持这些服务。

在这个新的 Stack Makers 播客中，安全提供商 [Check Point](https://www.checkpoint.com/products/cloud-native-security/?utm_content=inline-mention) 的云安全负责人 [TJ (Tsion)戈宁](https://www.linkedin.com/in/tgonen/)，将微服务安全置于背景中，并描述了安全工具发挥的关键作用以及人工智能(AI)和机器学习(ML)提供的支持。《新书库》的记者杰克·沃伦主持了这一集。

[如何以开发者喜欢的方式保护微服务](https://thenewstack.simplecast.com/episodes/how-to-secure-microservices-in-ways-developers-like)

戈宁说，考虑到过去几十年网络连通性的爆炸，网络继续发挥作用这一事实本身就是一个惊人的壮举。他将今天的基础设施比作道路，现在的交通流量比它们刚建成时要多得多。

戈宁说:“如果现在的汽车数量比 30 年前多 10 亿辆，就会有大量的交通事故和人员伤亡。”。在网络安全方面，“我们仍在运行，我们仍在运作，这要归功于许多安全公司和网络安全技术以及一些意识。”

随着潜在安全威胁载体的数量继续以指数速度增长，“根据定义，更多的事情将会发生，因为有更多的机会发生事情，”戈宁说。“我们还活着，我们还在行走，但会有更多的痛苦——会有超过 100%的更多的痛苦，”戈宁说。“我认为在变好之前会有更多的痛苦，但这只是一个自然的演变——坏事发生的机会更多了。”

戈宁说，在许多方面，没有首先进行适当的安全检查就使用云服务就像“从地上捡起一个 u 盘，然后把它插到你的服务器上”。戈宁说:“我们鼓励你快速行动，鼓励你做这些事情，鼓励你使用所有这些服务。”。

例如，在云上使用微服务时，必须设置容器权限。“现在，作为一名开发人员，你有两个选择:一个是说，‘好吧，这个容器只需要访问这个、那个和那个，而这个服务只需要这四个 API 调用——这是你的途径之一。另一个是“允许一切”，戈宁说。后一个选项要简单得多，那么您认为哪个选项是忙碌的开发人员使用最多的呢？

戈宁说:“你可以找到像我们提供的这样的技术，允许你从开发到运行时自动化安全，同时与开发者建立信任，给他们他们喜欢使用的工具。”。“你知道对安全公司来说，早上醒来说‘让我们做一些开发者喜欢的东西’有多难。”"
