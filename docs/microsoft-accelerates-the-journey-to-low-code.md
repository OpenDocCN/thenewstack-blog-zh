# 微软加速低代码之旅

> 原文：<https://thenewstack.io/microsoft-accelerates-the-journey-to-low-code/>

低代码和无代码在软件开发中越来越受欢迎，特别是在那些希望扩大能够为数字化转型工作创建应用程序的人数的企业中。虽然在 2020 年，不到 25%的新应用是使用无代码/低代码开发的，但 Gartner 预测，到 2025 年，70%的应用将使用这种方式。微软是为这一转变铺平道路的供应商之一，它通过减轻业务线和开发人员的负担来换取速度。但是低代码/无代码软件开发的潜力和最佳实践是什么？

在这一集的 New Stack Makers 播客中，[微软公司负责商业应用和平台的公司副总裁](https://www.microsoft.com/en-us/)[查尔斯·拉曼纳](https://www.linkedin.com/in/charleslamanna/)讨论了该公司通过其 Power 平台产品在低代码/无代码领域所做的工作，包括将无代码/低代码专业人士聚集在一起交付应用。

《新书库》的主编约布·杰克逊和《新书库》的新闻编辑达里尔·塔夫特主持了这个播客。

[微软加速低代码之旅](https://thenewstack.simplecast.com/episodes/microsoft-accelerates-the-journey-to-low-code)

低代码开发平台市场[可以追溯到 2011 年](https://en.wikipedia.org/wiki/Low-code_development_platform)，但微软的旅程可以追溯到更早，“低代码真的有好几代。拉曼纳说:“如果你回头看看 Excel、Visual Basic 和 Access，微软在这个领域已经有 25-30 年的历史了。“在过去的 4-5 年里，我们真的处于第三代低代码时代，这是微软通过 Power 平台等东西在企业中真正做大的时候，这些东西包括 Power 应用程序、Power BI power、Power Automate、Power 虚拟代理，”拉曼纳补充道。

这些组件共同提供了一个解决方案，用户可以利用它来输入信息、分析、处理和自动化数据和流程。“我们从微软 Office 中汲取了许多设计元素:Word、Excel、PowerPoint 以及 Visio，人们已经习惯于使用可视界面来设计和定义流程和体验，”拉曼纳补充道。

从人力资源到客户支持，低代码在广泛的用例中表现出色。“为了高效运营您的业务并真正实现数字化转型，您可能需要 10 到 20，000 个不同的应用程序和工作流。对于所有这些，不会有现成的东西，”拉曼纳说。“你永远不能用代码优先的方法去追求它们，因为那会花费太多的时间和金钱，”拉曼纳补充道。

微软使构建应用程序变得容易，并使培育基本安全实践的过程到位，但仍然建议“积极主动地使用低代码策略”。您可以定义政策、规则和治理，并对您的数据有一个强有力的姿态，”拉曼纳说。“因为管理对数据的数据访问，无论是在数据库还是在 API 层，都是你可以管理的最敏感的信息，”拉曼纳补充道。

“我认为，在时机成熟的时候，将会有 10 亿低代码开发人员。任何拥有智能手机或个人电脑的人都可以进行低代码开发，”拉曼纳说。虽然这种新技术在应用程序的构建方式上有很大的前景——使用它们的团队负责自己组装和组成应用程序——但微软的方法将获得多少采用仍有待观察。
