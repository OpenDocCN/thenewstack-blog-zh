# 开放:为绿地环境开放遥测技术

> 原文：<https://thenewstack.io/opening-up-opentelemetry-for-greenfield-environments/>

作为技术专家，我们经常被最新的技术淹没，所有这些技术都保证让你的环境更难、更好、更快、更强。虽然这些承诺在纸面上听起来很棒，但通常只有最健壮的操作才能负担得起时间，更不用说成本了，来实现最新和最棒的工具。

可观察性是一个好处显而易见的领域，但成本可能并不明显。对于许多操作来说，对一个精简的技术堆栈的需求使得实现任何新技术，比如可观察性，成为一件“好事”真正的挑战是平衡这一点和对您的组织的潜在好处。幸运的是，一个开源的(有积极支持的)工具出现了，它在灵活性、特性集，当然还有价格方面为你的可观察性提供了最好的基础。它的名字叫 OpenTelemetry。

# 什么是 OpenTelemetry？

OpenTelemetry 是一个非常活跃的云本地计算基金会(CNCF)项目，实际上是第二活跃的项目。它为开发人员提供了他们需要的库和文档，让他们的应用程序发出了解其性能所需的数据——特别是指标、跟踪和日志，还有更多。这些数据是可观察性的基础，即在复杂的分布式应用程序中确定发生了什么以及发生在哪里的能力。

这里有两个重点:CNCF 的支持使 OpenTelemetry 在积极开发方面与 Kubernetes 等“小项目”处于同一组，同时在可观测性方面得到了最受尊敬的供应商的广泛支持，包括 Splunk、LightStep、Dynatrace 和 New Relic。这种支持通过确保 OpenTelemetry 具有支持您的架构的灵活性(无论架构有多复杂),以及广泛的供应商支持允许进行长期规划，以决定最适合您团队未来的可观测性工具需求，从而让您在实施 open telemetry 时更加安心。

## 为什么 OpenTelemetry 适合您

尽管 OpenTelemetry 听起来很棒，但作为开发人员，我们仍然需要审查我们环境中的直接好处，增加 OpenTelemetry 仪器可能带来的间接好处，如更可靠的应用程序和更好的跨团队合作，我们需要确保实施 OpenTelemetry 将有助于我们避免未来的技术债务。

决定这些基本元素对您的应用程序如何成熟有很大的影响。随着您的成长，您需要做出几个决定，例如选择公共云提供商以及如何构建您的开发管道。将 OpenTelemetry 作为应用程序开发的支柱，可以确保您从一开始就使用行业标准技术，并拥有处理不断扩展的应用程序的规模和复杂性的可靠记录。

早期使用 OpenTelemetry 可以让您继续运行您的应用程序，无论您对云提供商、开发管道等做出什么决定。特别是随着您的业务增长，OpenTelemetry 将通过让您以新的和强大的方式分析和使用相同的数据，更好地调整您的团队，从而在开发中实现更强大的协作，并更快、更准确地进行故障排除。

## 如何打开遥测

那么，如何以合理的人力成本来实现和利用它呢？OpenTelemetry 最好的一点是，它不需要重构整个环境。您可以从检测单个应用程序开始。这可以通过 OpenTelemetry 的标准发行版或您认为合适的特定于供应商的发行版来实现。

从那里，对于许多应用程序，比如大多数 Java 应用程序，可以自动地进行插装，或者您可以在[文档](https://opentelemetry.io/docs/)中找到关于如何开始的更多详细信息。

OpenTelemetry 为您提供了今天更好地构建应用程序所需的工具，以及随着时间的推移取得成功所需的框架。今天就开始免费看看为什么它已经成为这样一个行业的力量，并准备你的应用许多快乐可观的岁月。
