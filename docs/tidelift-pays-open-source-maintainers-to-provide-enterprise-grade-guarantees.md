# Tidelift 向开源维护者付费，以提供企业级保证

> 原文：<https://thenewstack.io/tidelift-pays-open-source-maintainers-to-provide-enterprise-grade-guarantees/>

虽然许多开源项目都有提供企业级支持和特性的母公司，但其他项目则没有。这些项目的贡献者一般都是利用自己的时间无偿完成的。总部位于波士顿的初创公司 Tidelift 致力于“让开源软件更好地为每个人服务”，旨在通过创造一种 [gig 经济类型的模式](https://blog.tidelift.com/open-source-the-rise-of-the-rest)来改变这种情况，在这种模式下，用户订阅费将支付给软件的维护者。

联合创始人兼首席执行官[唐纳德·费希尔](https://www.linkedin.com/in/donaldfischer/)说:“(我们想添加)一些人们用开源软件构建时需要的功能:保证软件安全，持续维护，验证软件来自哪里，谁写的，许可。”。他说，这种模式类似于优步或 AirBnB。

该公司最近达到了 100 万美元的里程碑，为那些维护开源软件项目的人提供资金。

“我们看到了一个机会，一个市场缺口。他说:“对于一些开源项目来说，甚至连基本的卫生问题都没有人去解决。”。

Fischer 和联合创始人[hahov Pennington](https://github.com/havocp)、 Jeremy Katz 和 [Luis Villa](https://www.linkedin.com/in/luisv/) 在开源领域有着悠久的历史，包括参与开发用于 Red Hat Enterprise Linux (RHEL)的开源模型。5 月，Tidelift [宣布](https://www.prnewswire.com/news-releases/tidelift-raises-15m-series-a-to-make-open-source-work-betterfor-everyone-300651931.html)获得由 General Catalyst、Foundry Group 和前红帽董事长兼首席执行官 Matthew Szulik 领投的 1500 万美元 A 轮投资。

“我们直接与软件的创造者合作，”费舍尔说。“我们说，‘如果你愿意按照这个标准维护软件——我们写下这个标准的含义——我们会付钱给你让你这么做。’

我们去找那些最了解软件的人，也就是软件的创造者——他们通常已经在做这些活动了，但不是以统一和可靠的方式。与我们的用户一起，并代表他们的利益，我们为那些维护者创造了激励，让他们花费额外的精力来照看开源组件。"

他说，这让用户更有信心，他们可能已经在使用的开源软件会得到更好的保护。

“在用户方面，我们为他们提供了一个软件工具，可以与他们的软件开发过程集成，有点像是附加到他们的源代码库中，密切关注他们正在构建的应用程序中的软件包，并就此向他们提供持续的反馈，”他说。

“我们和其他开源扫描仪的一个区别是，我们不只是告诉订户有问题；我们去找维护者说，‘嘿，和我们一起解决它吧。’他说:“不是红帽支持 RHEL，而是软件开发人员积极使用的各种开源软件。

维护者不是因为创造了开源软件而获得报酬，而是因为完成了 Tidelift 指定的与安全、维护、许可和营销相关的任务。订阅费不是进入一个普通基金，而是付给维护者客户使用的特定软件包。

参与的维护者完全控制他们的软件包和技术路线图，并为他们的软件提供维护，而不是帮助台或咨询服务。报酬取决于订户的数量和所需的工作量。维护者也可以从每个注册用户那里获得推荐奖金。

它最初专注于支持 JavaScript、Java、Python、PHP 和 Ruby 编程语言生态系统中的包。今天精选的顶级包包括 Vue，Material-UI，Babel，Gulp，Fabric 和芹菜。

该公司承认，在这个早期阶段，支付的金额不是很大。例如，项目维护人员的收入估计是每月 1000 美元。该公司还预先批准了一组精选的开源包，保证每个包的最低支付额为 1 万美元。网站上有一个[搜索工具](https://tidelift.com/about/lifter?hsCtaTracking=804c5708-f751-4414-a2ff-09b42fc29039%7C4f35d3fe-d080-4d14-a7b2-e8c00ab97bf0)，可以找到特定套餐的付款估算。

Fischer 说，围绕开源软件建立一个企业是很困难的，因为很多开源软件并不特别与技术相关。

“我们一次建立业务，并作为共享服务提供，所以他们可以专注于他们的项目，如果你愿意，我们将成为业务前端，”他说。
