 [马克·赫林，CMO，InfluxData

马克·赫林是一位全面发展的硅谷高管，在采用复杂技术并使其为更广泛的受众所理解方面拥有丰富的经验。目前，Herring 是 InfluxData 的首席营销官。在加入 InfluxData 之前，Herring 在 Hortonworks 担任公司营销和开发人员营销副总裁，在 Software AG 担任产品副总裁 s VP，在 Sun Microsystems 担任中间件、Java 和 MySQL 营销副总裁，在 Forte Software 担任营销副总裁。](https://www.influxdata.com/) [](https://www.influxdata.com/)

在互联设备(即物联网)的大规模增长和下一代软件快速增长的仪器需求的推动下，时间序列技术变得更加。自 2013 年推出开源时间序列平台 InfluxDB 以来，我们已经看到了数百万次下载，建立了一个不断扩大的企业客户列表，并培养了一个不断增长的社区，该社区一直在寻找新的方法来部署和构建我们的平台。大多数数据最好在时间维度上理解，所以当涉及到这个空间时，我们才真正开始。

亚马逊网络服务最近宣布，它将在明年的某个时候带着一个托管时间序列数据库 Timestream 进入市场。我们一直在宣传高压缩、超高速引擎和专门为时间序列数据构建的堆栈的重要性，亚马逊似乎非常紧密地遵循了这一模式——这进一步验证了对像 [InfluxData](https://www.influxdata.com/products/) 这样健壮、成熟、专门构建的解决方案的需求。关于 Timestream 的技术能力的细节有限，但根据 AWS 模型，我们的产品之间可能存在几个重大差异，包括:

*   **开源** — InfluxData 首先是一家开源公司。我们致力于公开分享想法和信息，合作开发解决方案，并提供充分的透明度来推动创新。我们的产品由一群精力充沛的开发人员不断改进，使其更加可靠、安全和出色。开源社区推动创新的力量是任何专有软件解决方案都无法超越的。
*   **混合云和本地支持** —公司有不同且特定的功能需求，跨多个云托管环境分布资产通常是最佳选择。在这些情况下，避免供应商锁定非常重要，因为它会限制定制系统和协商更好价格的能力，最终使改变云提供商以满足不断发展的业务和技术需求变得非常困难和昂贵。混合云系统提供了选择最适合公司需求的服务的灵活性，无论是支持 GDPR 法规要求还是支持分布在多个提供商的团队。从运营的角度来看，多云系统提高了效率，并提供了另一层安全性，以确保不会停机。
*   **经过时间考验的**——交付一个健壮的时间序列平台并不容易。我们在四年多前推出了 InfluxDB，并在 2016 年推出了我们的业务产品 InfluxEnterprise。多年来，我们一直在调整、重新设计和重新配置我们的产品，以不断提高整体功能。缺陷和错误是软件自然进化中不可避免的障碍，我们已经在经过时间和社区测试的产品中解决了很多问题。

随着对时间序列技术认识的加深，我们将会看到更多的竞争者进入这个领域，这是一件好事，因为竞争推动了创新，促进了整个类别的发展。我们保持警惕，专注于提供一个优秀的平台，帮助公司从实时分析中收获商业价值，并支持由富有创造力和才华的开发人员组成的社区，他们正在探索用我们的平台解决现实世界问题的新方法。我们将继续推动这项技术的发展——利用专门构建的系统的优势，开发新的语言，与新的合作伙伴保持一致，支持新的平台和新的工作负载，并在令人兴奋的机会到来时抓住它们。

InfluxDB 现在*可用*，我们鼓励开发者 [下载](https://portal.influxdata.com/downloads) 并在几分钟内开始运行。
