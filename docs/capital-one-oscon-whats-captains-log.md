# 为开源生态系统做贡献

> 原文：<https://thenewstack.io/capital-one-oscon-whats-captains-log/>

随着开源软件在软件世界中不断发展壮大，许多企业已经参与进来，要么开源他们自己的内部项目，要么支持围绕内部使用的第三方软件的开源社区。银行业巨头 Capital One 是一家热切拥抱开源方式的公司。

在这一集的 [The New Stack Makers](/tag/the-new-stack-makers/) 中，我们了解了更多关于 Capital One 在开源方面的进展，其即将推出的开源项目水文图，以及其他公司如何在自己的堆栈中采用开源。在 OSCON 2016 上，新栈创始人 [Alex Williams](https://twitter.com/alexwilliams) 与第一资本高级软件工程师 [William Ma](https://www.linkedin.com/in/williamma87) 对话；[贾里德·史密斯](https://twitter.com/jsmith)，他负责管理 Capital One 的开源社区外联活动，以及[迈克尔·科特](https://cote.io/)，他是 [Pivotal](http://pivotal.io/) 的技术营销总监。

首都一号 OSCON:你的船长日志里有什么？

马首先介绍了 Capital One 下一个开源项目的背景，这是一个用于 Hadoop 进程的 ETL 提取工具，被称为 Hydrograph。尽管还没有开源，Capital One 计划在今年晚些时候开源这个项目。“我们开源 Hydrograph 的原因是，我们认为这是许多不同组织的需求，如 Capital One。通过开源，我们可以从其他有类似兴趣的组织那里得到贡献。

开源 Capital One 的软件是 Smith 的工作重点，许多项目从一开始就被定为开放的。他指出，特别是新项目，开始时会询问它们是否应该开源。然而，这样做需要承诺。

“最重要的是公开透明的沟通。这是每个开源社区的关键。第二，试图——无论是通过文档还是其他方法——帮助保存机构知识并将其传递给下一代用户。不管是好的维基、网站上的搜索引擎还是文档，”史密斯说。

Coté指出，有了 GitHub 这样的代码库，推广自己的开源项目变得更加简单。“我认为现在你基本上是在 GitHub 做事情。那是你船长的日志。这类产品的作用是自动进行自我推广。您有一个集中的地方，所有代码、网站和问题跟踪都在这里进行。”

在讨论开源时，科蒂强调，在过去的 20 年里，焦点已经从 20 世纪 90 年代末的“讨伐金钱大亨”转移到了分析、讨论和解释开源背后的思想和概念。话虽如此，他承认，就让企业变得更加开放而言，让企业参与开源一直是“有点让人失望”。

史密斯反驳道，通过提供企业可以做些什么来成为更好的开源公民。“我希望看到更多的企业站出来，为基层做出更多的贡献。一个很好的例子是我们最近在 [OpenSSL](https://thenewstack.io/decades-old-sslv2-drown-bug-hides-third-https-servers/) 中看到的一些缺陷。很多人都在使用这个软件，但没有多少人对此做出贡献。如果你依赖于某个特定的库或堆栈中的某个特定部分，那就贡献出来，贡献时间、开发人员资源，甚至是一个可以见面讨论的空间。”

“我认为过程线的美妙之处在于，我们在内部使用它，同时也在公开采购它。我们这样做不仅仅是为了创造一些开源的东西。我们正在自己使用它，”史密斯说。
