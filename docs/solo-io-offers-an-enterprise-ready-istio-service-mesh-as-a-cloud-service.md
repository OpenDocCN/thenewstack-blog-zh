# Solo.io 提供了一个企业就绪的 Istio 服务网格，即云服务

> 原文：<https://thenewstack.io/solo-io-offers-an-enterprise-ready-istio-service-mesh-as-a-cloud-service/>

Solo.io 推出了 Gloo Cloud，它提供 Gloo Mesh Istio 服务网格和其他工具，但不包括安装、维护和其他相关的操作任务。

事实上，尽管被视为微服务管理的必要组件，Istio 服务网格本身仍然会给开发运维团队带来运营挑战。

“我们看到我们的客户仍然负责安装和管理我们的产品和 Istio，确保它能正常工作等等，” [Solo.io](https://solo.io?utm_content=inline-mention) 创始人兼首席执行官[艾迪·莱文](https://www.linkedin.com/in/iditlevine)告诉新堆栈。“我们发现，通过管理这些业务以及为它们提供 Istio 服务，我们可以提供更好的帮助。基本上，客户需要做的唯一事情就是管理他们自己的简单集群，并将它们注册到系统中。”

Gloo Mesh 和其他 Solo.io 软件提供了一个集成不同 Kubernetes 管理开源组件的单一网关，包括[特使](https://envoyproxy.io/)分布式代理和 [Istio](https://istio.io/) 开源服务 Mesh。例如，Solo.io 的软件范围包括 API gateway Gloo Edge 和 Gloo Mesh，这是一个基于 Istio 构建的服务网格，具有多种集群管理功能，以及用于 API 定义和策略的 [Gloo Portal](https://docs.solo.io/gloo-portal/latest/) ，以及 web 组装功能。

Gloo Cloud 将首先被未来的客户采用，然后他们将其他 Solo.io 工具和平台集成到他们的 DevOps 环境中，[solo . io 的首席营销官 Erik Frieberg](https://www.linkedin.com/in/efrieberg) 向新堆栈预测。

“如果你在寻找服务网格功能，你会去 Gloo cloud，然后你会说‘现在我已经管理了内部的流量，我会开始担心 Gloo Edge 或者以后的开发者门户，’”frie Berg 说。

企业管理协会(EMA)的分析师 Torsten Volk 说:“Solo.io 似乎旨在将网格计算的优势带到仍然非常混乱的 Kubernetes 世界，这对任何目睹开发人员花费大量时间调试环境的人来说都是一个令人兴奋的消息。”。

*要了解更多关于 Solo.io 服务网格软件系列的信息，请查看该公司将于 3 月 23 日至 24 日周二和周三举行的 [SoloCon 2021 用户大会](https://solocon.io/)。*
