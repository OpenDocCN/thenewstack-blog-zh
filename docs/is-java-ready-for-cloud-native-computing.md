# Java 为云原生计算做好准备了吗？

> 原文：<https://thenewstack.io/is-java-ready-for-cloud-native-computing/>

Java 的编程语言于 1995 年首次发布，已经成为领先的开发人员平台，成为数百个企业应用程序的主力。随着每一次新技术的发展，Java 都成功地适应了变化。但是，尽管最近的一项 Java 生态系统研究发现，生产环境中超过 70%的 Java 应用程序都是在容器中运行的，但这种语言要适应云的原生世界，仍然需要克服一些障碍。

在本期 New Stack Makers 播客中，Azul Systems 副首席技术官 [Simon Ritter](https://www.linkedin.com/in/siritter/?originalSubdomain=uk) 和 JetBrains 的 Java 开发倡导者 [Dalia Abo Sheasha](https://daliashea.com/about/) 讨论了该语言正在努力克服的一些挑战，并分享了一些对开发人员要求的新功能的见解。《新书库》的新闻编辑达里尔·塔夫脱主持了这次播客。

[Java 为云原生计算做好准备了吗？](https://thenewstack.simplecast.com/episodes/is-java-ready-for-cloud-native-computing)

现代应用程序的复杂性要求开发人员掌握越来越多的技能、技术和概念，以便在云中进行开发。Abo Sheasha 说:“我所看到的是技能上的差距，以及如何将现有的 Java 应用程序移植到云中。

“开发者真正想要的是专注于开发 Java 代码的想法，”Ritter 说。他补充说:“能够插入不同的云提供商，还能够与 CI/CD 工具等东西集成，这样您就可以获得持续集成和内置的持续部署。”

Ritter 说，让 Java 为云做好准备是“从云提供商到工具提供商的所有人的分布式责任”。“每个人都认识到，我们有越多的人在上面，我们有越多的想法，我们对开发者的语言就有越好的结果，”Abo Sheasha 说。

随着 JEP 或 JDK 增强提案的引入，使开发人员更加高效和多产正在成为主流——一种在 Java 平台本身的开发中添加新特性的轻量级方法。“但也有一些更大的项目，如 Amber 项目，它是对 Java 语言语法的微小改变，目的是通过去掉一些样板代码来提高生产率，”Ritter 说。

通往 Java 下一章的旅程是多维的。虽然“大多数开发人员专注于完成工作，但掌握新事物的技能是一项挑战，因为这需要时间。Ritter 说:“许多公司仍然存在使用他们公司使用的 Java 版本的问题。“不是因为开发商不想做；而是他们需要让管理层相信这是值得投资的，”Abo Sheasha 补充道。
