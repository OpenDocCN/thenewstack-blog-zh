# 与法斯特丽的泰勒·麦克马伦一起生活在边缘

> 原文：<https://thenewstack.io/living-edge-fastlys-tyler-mcmullen/>

[与法斯特丽的泰勒·麦克马伦一起生活在边缘](https://thenewstack.simplecast.com/episodes/living-on-the-edge-with-fastlys-tyler-mcmullen)

在几秒钟内将内容送到全球用户手中的需求促使内容交付网络(cdn)的发展，如 Akamai、 [CloudFlare](https://www.cloudflare.com/) 和 [Fastly](https://www.fastly.com/) 。cdn 正在成为许多大规模系统的核心组件，因此很自然地，它们的用途将扩展到不仅仅是缓存高需求的文件。

Fastly 首席技术官兼联合创始人泰勒·麦克马伦(Tyler McMullen)正在努力思考这个边缘计算场景可能会是什么样子。当我们在最新一期的 [The New Stack Makers 播客](https://thenewstack.io/podcasts/makers)中与他聊天时，他从整体上思考了边缘计算的可能性甚至定义。

对一些人来说，这意味着在除核心系统层之外的任何地方进行计算:在管道中，在 CDN 或在最终用户的设备中。最后一点甚至暗示了物联网设备中边缘计算的使用。这个想法很有吸引力，因为它将节省往返和服务器负载，但在实践中，它确实给已经很棘手的分布式内容交付增加了另一层复杂性。

麦克马伦说，最大的问题是*到底是什么* 边缘计算？“我经常被问到这个问题，这个问题取决于你问谁……如果你问某种类型的移动开发者，他们的优势是设备本身……我听说过一些(电信)公司正在考虑在高速公路旁放置微型服务器的故事。所以，对他们来说，边缘本质上是他们网络的边缘。”

他继续说道，“你认为什么是优势似乎完全取决于你在网络中的视角。在我看来，当我们谈论边缘时，我们谈论的是用户和应用程序起源之间的任何东西。”

McMullen 将边缘计算视为提供更好服务的推动者，因为当单个节点可以自己做出决定时，可以缩短响应时间。“我们为人们提供了一种编写代码的方式，这些代码可以同时传播到数千台服务器上，尽可能地传播到用户身边，而不需要在设备上。”

McMullen 对边缘计算和现代分布式计算生态系统有很多其他想法。哦，他还喜欢 Lisp！一定要看看上面的整个播客。
