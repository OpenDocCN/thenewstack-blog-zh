# 提高 Web 应用程序和 API 安全性的 5 种方法

> 原文：<https://thenewstack.io/5-ways-to-improve-your-web-application-and-api-security/>

学习使用 web 应用程序和 API 来利用数据的黑客数量迅速增加。然而，Gartner 预测，到 2025 年，[只有不到 50%的企业 API 将被管理](https://www.gartner.com/en/documents/4009103)，这显示了 web 应用和 [API 安全性](https://thenewstack.io/10-ways-to-manage-api-security-risks/)日益增长的重要性。

## **为什么攻击者的目标是 Web 应用程序和 API？**

随着 web 应用程序和 API 的数量持续飙升，了解 web 应用程序面临的威胁非常重要。web 应用程序威胁(WAT)通过组织的网站或应用程序来攻击组织。组织应该在开发的每个阶段解决这些安全问题。wat 分为几种不同的类型。一些最常见的包括:

*   外部 WATs
*   内部 WATs
*   社交媒体 WATs
*   恶意代码 wat
*   网络钓鱼/黑客 wat

未能解决 web 应用程序的安全性会导致严重的威胁和长期的损害。API 也是如此。免费开放给公众使用的 API 的出现使得几乎整个计算世界都可以使用它们来改善应用程序和数据之间的连接。虽然这可能提供一些很大的优势，但开放性也使它们成为攻击者的目标。

随着时间的推移，攻击者已经掌握了专门针对滥用 API 的代码编写方法。就像开发人员可以编写代码从组织的系统中获取数据一样，攻击者也可以对一个恶意软件做同样的事情。他们可以使用恶意应用程序和 API 对毫无防备的用户造成严重破坏。目标是用恶意软件感染无辜的用户，以便以后可以对组织甚至个人发起攻击。

当 web 应用程序或 API 遭到破坏时，攻击者可以轻松访问数据。此外，攻击者能够访问私人数据，并在多个设备上传播恶意软件。为了保护自己免受此类攻击，组织必须采取严密的安全措施。

WATs 和 API 威胁在未来只会变得更加复杂和危险。

因此，找到正确的保护措施是必须的。

## **保护组织的 5 种策略**

找到保护您的组织免受 WATs 和 API 威胁的最佳策略将取决于您最想避免的威胁类型，以及您拥有的资源种类和您愿意在这些保护措施上投入的时间和金钱。

让我们来讨论其中的一些策略，以及为什么它们对您的组织很重要。

### **1。使用网络应用防火墙(WAF)**

攻击者经常利用分布式拒绝服务(DDoS)攻击来攻击不安全的 web 应用。通过这种攻击，多个 web 应用程序被劫持，并被用来用流量轰炸单个目标。这使得攻击者更容易获得对受限信息的访问。为了减少此类攻击，组织需要部署适当的防火墙。一个 [web 应用防火墙](https://thenewstack.io/4-essential-tools-for-protecting-apis-and-web-applications/)可以是基于网络、基于云或者基于主机的。

### **2。API 发现和状态管理**

随着 API 越来越受欢迎，大多数攻击者现在都瞄准了它们。因此，组织需要能够监控 API 及其相关的安全风险。有[工具](https://orca.security/resources/blog/multi-cloud-api-security-visibility-and-risk-management/)来检查风险、漏洞、错误配置、恶意软件、敏感数据的位置和横向移动风险。这些工具帮助我们有效地对给组织带来最大危险的 API 风险进行优先级排序。

### **3。使用 OAuth**

组织也可以在系统内部实施保护措施。这可以通过在关键 web 应用程序上要求双因素身份验证来实现，以阻止未经授权的用户进入。此外，基于时间的一次性密码(TOTPs)的使用最近有所增加，尤其是在云应用程序提供商中。此方法使用一天中的当前时间作为身份验证因素之一。

在设置新的 web 应用程序时，将员工列入白名单还可以确保他们在远程工作时只访问可信的系统和服务器。这将使您能够在第三方试图在您不知情的情况下通过外部服务器渗透您的内部网络时进行监控并接收通知。

### **4。数据加密**

由 API 管理的每一个敏感数据都必须被很好地加密。拥有良好的加密方法可以确保攻击者无法访问任何敏感数据。它还确保授权用户拥有可用于解密或修改数据的唯一签名。

### **5。手动穿透**

虽然听起来很可怕，但这是迄今为止防范 WATs 和 API 威胁的最安全的方法之一。如果您手头没有安全专家，强烈建议您为此咨询一位专家。安全专家的主要任务包括从扫描漏洞到执行安全审计和监控恶意活动。此外，在安全专家手动检查的同时实施自动化有助于确保不会遗漏威胁。这使组织能够从专家的角度获得真实的反馈，以及关于攻击者最有可能以何处为目标的详细信息。

## **推荐的入门安全策略**

基于上面描述的策略，您可以看到一个好的 web 应用程序和 API 安全策略是从理解风险开始的。这些知识将帮助你记住预防措施。虽然我们已经讨论了您可以使用的多种策略，但我们建议所有组织从以下几点开始:

*   **利用 API 发现和状态管理** —您可以通过物理硬件或利用软件解决方案(如 [Orca 的无代理 API](https://orca.security/platform/api-security/) )来实现这一点，这有助于组织识别、优先处理和解决跨多云环境的 API 错误配置和安全风险。在单个解决方案中，它为用户提供了完整且持续更新的托管和非托管 API 清单、关于 API 错误配置和漏洞的可操作数据，以及关于潜在风险 API 漂移和可能发生的变化的警报。

*   **扫描漏洞** —使用合适的工具，这是确保 web 应用程序安全的最有效方法之一。执行连续扫描使组织更容易识别他们面临的漏洞。Orca 的[漏洞管理工具](https://orca.security/platform/vulnerability-management/)，覆盖您云的每一层，包括云工作负载和配置。它将所有这些信息整合到一个统一的数据模型中，以确定风险的优先级，并识别何时看似不相关的问题可能会结合起来形成危险的攻击路径。
