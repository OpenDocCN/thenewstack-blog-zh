# 收起大帐篷 OpenStack 的下一步是什么

> 原文：<https://thenewstack.io/folding-big-tent-whats-next-openstack/>

[折叠大帐篷:OpenStack](https://thenewstack.simplecast.com/episodes/folding-the-big-tent-whats-next-for-openstack) 下一步怎么办

OpenStack 正在收起[大帐篷](https://thenewstack.io/tns-makers-openstack-tokyo-canonical-ubuntu-project-midokura-ibm/)的管理风格，采用更加模块化的方法。这将如何影响 OpenStack 已经成为的这个大型开源项目的开发？

在 9 月份的开源峰会上，TNS 创始人 [Alex Williams](https://twitter.com/alexwilliams) 与 [Linux Academy](https://linuxacademy.com/) 讲师 [Amy Marrich](https://github.com/spotzz) 交谈，以了解更多关于 OpenStack 的下一步，以及 Marrich 的 Linux Academy 课程如何帮助开发人员利用 OpenStack 做更多事情。Marrich 的课程中有一个这样的重点是基于场景的故事，开发人员专注于在 Keystone 中为一个虚构的政府客户创建云评估组。

Marrich 解释说，随着大帐篷的折叠，有关于模块化和独立服务的讨论。“关于什么是大帐篷，人们有很多困惑。这个想法就是你要找到适合你的情况和场景的项目。模块化过程仍处于早期阶段，Swift 是唯一一个从模块化开始的。”
