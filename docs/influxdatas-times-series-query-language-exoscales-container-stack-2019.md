# InfluxData 的时间序列查询语言，Exoscale 的容器堆栈 2019

> 原文：<https://thenewstack.io/influxdatas-times-series-query-language-exoscales-container-stack-2019/>

[InfluxData 的时间序列查询语言，Exoscale 的容器栈 2019](https://thenewstack.simplecast.com/episodes/influxdatas-times-series-query-language-exoscales-container-stack-2019)

本周，在我们的每周新闻总结播客 New Stack Context 中，我们采访了 [InfluxData](https://www.influxdata.com/) 的创始人兼首席技术官[保罗·迪克斯](https://www.linkedin.com/in/pauldix/)，谈论为什么现在需要时间序列数据库。

新的 Stack 赞助商 InfluxData 在 2013 年推出了 InfluxDB，[一个开源时间序列平台](https://www.influxdata.com/modern-time-series-platform/)。从那时起，该软件已经有了数百万次的下载，建立了一个不断扩大的企业客户名单，并培养了一个不断增长的社区。influx data CMO·马克·赫林[本周在新的堆栈上写道](https://thenewstack.io/the-need-for-time-series-data-is-now/)大多数数据最好在时间维度上理解，所以时间序列空间才刚刚开始。Paul 与我们讨论了为什么会出现这种情况，还分享了一些关于公司新的时间序列数据查询语言的信息，这种语言叫做 Flux。

在节目的后半部分，我们将回顾本周的头条新闻，并讨论 Exoscale 的[容器堆栈](https://www.containerstack.net/)，这是一个关于与 Kubernetes 进行 DevOps 的会议，将于 1 月 30 日至 31 日在苏黎世举行。在此次活动中，来自制药、银行、媒体和研究行业的最终用户以及云原生专家将分享他们的经验和最佳实践。Aqua 的 Liz Rice、Weaveworks 的 Alexis Richardson 和 CERN 的 Ricardo Rocha 将在会上发言。将涵盖堆栈的所有层，包括网络层、存储层、打包层和专业服务。

在最近一次关于新堆栈的直播中，Exoscale 首席技术官[Pierre-Yves Ritschard](https://twitter.com/pyr)分享了他对欧洲市场集装箱化状况的看法。他说，虽然在容器化工作负载方面，欧洲组织可能有点落后于美国同行，但他们正在迅速赶上。

TNS 编辑总监 [Libby Clark](https://twitter.com/libbymclark?lang=en) 主持了这次活动，并得到了 TNS 创始人 [Alex Williams](https://twitter.com/alexwilliams?lang=en) 和 TNS 执行主编 [Joab Jackson](https://twitter.com/Joab_Jackson) 的协助。
