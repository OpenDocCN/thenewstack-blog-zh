# 下一代云计算不需要将服务器放入容器中

> 原文：<https://thenewstack.io/next-generation-cloud-computing-does-not-involve-putting-a-server-in-a-container/>

当传统的服务器计算模式让位于容器化和微服务环境时，所谓的下一代云计算可以说是从向云原生的转变开始的。

[容器解决方案](https://www.container-solutions.com/)的云原生工程师 [Charlotte Mach](https://nl.linkedin.com/in/charlotte-d-mach) 说:“你实际上不必依赖那里的服务器——你可以考虑使用新的编排系统，许多人必须改变他们的思维方式，以实际达到他们可以更有效地使用新版本和新系统的地步。“Kubernetes 确实有所帮助，但它有时也会让我们想到‘让我们将服务器放入容器中吧’——这不是 Docker 容器中的一整块东西后最美好的事情。因此，我们所做的一件大事就是试图帮助人们重新思考下一代。”

在本期[的新堆栈制造商](/podcasts/makers)播客中，Mach 和[集装箱解决方案首席技术官 Ian Crosby](https://www.linkedin.com/in/iandcrosby/) 讨论了许多关于下一代计算的话题。它们包括边缘计算、混合和多云的采用、环境可持续的云计算和文化。《新书库》的创始人兼发行人亚历克斯·威廉姆斯主持了这一集。

[下一代世界并不意味着将服务器放入容器](https://thenewstack.simplecast.com/episodes/a-next-gen-world-does-not-mean-putting-a-server-in-a-container)

Mach 和 Crosby 最近还借出了他们的 Kubernetes 和云原生计算专业知识，以支持他们评论的新堆栈的[“Kubernetes 生态系统的状态”](/ebooks/kubernetes/state-of-kubernetes-ecosystem-second-edition-2020/)电子书。

下一代云计算的许多方面(如果不是大部分的话)仍在进行中。马赫说，云环境中的边缘计算仍然是一个“特例”。她解释说，这是因为“你真的不能只拿一个 Kubernetes 集群，把它放在一个离你的移动塔更近的数据中心，然后说你有一个边缘集群，可以做你想做的一切，因为仍然有一些特殊的网络考虑需要做”。

然而，用于边缘计算的工具类似于用于云原生环境的工具。Mach 的团队还使用 HashiCorp 的 [Terraform](https://www.terraform.io/) 来编写通用基础设施代码，并使用 GitLab 来部署资源。马赫解释说:“这更多的是关于你最终创建的平台(针对边缘应用)有什么具体要求，而这些要求还没有完全实现。”。“因此，Kubernetes 为您做了很多事情，但它并不真正适合大多数边缘用例。”

下一代云计算将不可避免地变得更加环境可持续。克罗斯比说，为此，可再生能源受到了很多关注。有证据表明，一些云提供商的环境正在向“碳中和”模式转变。“这是我们看到人们开始询问的更多需求——在我选择哪个云提供商方面，甚至在我运行的哪个数据中心方面，都有一些选择，”他说。“如果这些信息是公开的，我可以决定‘这个区域或数据中心正在运行 100%的可再生能源，这就是我想要运行我的资源的地方。’"

下一代云计算也带来了文化转变。毕竟，Kubernetes 是开源的，这种思想应该继续贯彻到新兴的云应用程序中。克罗斯比说:“另一件我很想看到的事情是——随着这种云原生空间开始扩展到不同的行业或不同的部门——看到[开源文化]如何影响他们。”。

克罗斯比说，传统的数据中心管理和软件开发及运营团队面临着更大的压力，他们需要在协作时采用更多的开源文化，这“过去是许多封闭的大门和专有的东西”。“所以我认为，随着 cloud native 扩展到这些新领域，这是我真正感到兴奋的事情。”
