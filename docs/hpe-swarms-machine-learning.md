# HPE 蜂拥机器学习

> 原文：<https://thenewstack.io/hpe-swarms-machine-learning/>

分布式机器学习的想法并不新鲜。谷歌是最先大规模实现这一功能的公司之一，它训练 Android 手机执行键盘自动完成功能。随着物联网的发展，有一种将 ML 模型的处理[推向边缘](https://onlinelibrary.wiley.com/doi/abs/10.1002/9781119551713.ch12)的趋势，这一概念通常与新兴的[雾计算](https://www.onlogic.com/company/io-hub/edge-computing-vs-fog-computing)概念相关联。这个想法并不新鲜，但实施和商业化仍处于起步阶段。现在 [HPE](https://hpe.com/) 提出了“群体学习”，一种使用区块链技术的联合学习的新方法。

联合学习是一种在模型所在的数据源上远程训练模型的方法，然后仅将训练好的模型传送回中枢，在中枢中通过一致的过程来确定完成的模型。一个明显的用例是物联网，其中有大量由远程设备生成的数据，在这种规模下，将所有这些数据传送回云进行模型训练或推理是没有意义的。在商业上，像 [Integrate.ai](https://integrate.ai/integratefl/) 和 [Devron](https://www.devron.ai/) 这样的提供商已经开始提供管理联合学习的解决方案，但是正如刚刚提到的，这个解决方案空间[仍然处于起步阶段](https://integrate-ai.gitbook.io/product-docs/McUohjARcUMVQJalc0fB/)。

## 进入区块链

这些方法的一个潜在缺点是它们依赖于一个中枢，这可能成为一个瓶颈或单一故障源。这就是 HPE 的群体学习方法的用武之地。它不仅聚合和下推训练和推理工作负载，而且还使用区块链方法。

乍一看，还有什么比“机器学习”和“区块链”更符合流行词汇的呢但是有一个真正的疯狂的方法。正如预想的那样，“群体机器学习”的目标是涉及隐私或监管的场景，这些场景阻止或阻碍数据被移动。这就是区块链的基本原理。例如，您可能有一组医院正在合作进行一项研究，将机器学习应用于疾病预防、检测或结果，但患者数据是不可移动的障碍。

HPE 实现了一个基于以太坊技术的区块链，允许数据留在原地，模型在本地训练和运行，在这样的环境下，交换的模型结果变得防篡改。HPE 建立了一个群体网络，个体节点在其中注册，然后这些节点执行建模。

它包含几个组件。它始于作为容器交付的群体学习库，这些容器可以在基于 Kubernetes 的任何目标基础设施上运行。模型本身保持不变；HPE 声称，只需增加四行代码，这些模型就可以部署在 swarm 上。然后是群网络，它是区块链、控制平面和许可服务器。

## 潜在的使用案例

分布式学习有许多潜在的用例。在医疗保健领域，世界各地的医院都可以应用 ML 来识别 MRI 图像上的癌症，从而提供一个不必移动到中心位置的大型训练数据基础。金融机构联盟可以协作构建跨全球数据集的欺诈检测或个性化模型。跨特许经销商零售网络的全球营销活动是另一个例子，其中建模财富可以以尊重本地数据所有权的方式共享。当然，也有全球模型跨越数据的情况，根据规定，这些数据是不允许跨境的。

目前，Swarm ML 在 HPE 仍处于早期采用阶段，计划最终将其作为 [HPE Ezmeral MLOps](https://www.hpe.com/us/en/solutions/ezmeral-machine-learning-operations.html) 的一部分进行产品化。虽然不一定需要专业服务参与，但我们预计早期采用者可能需要专家帮助才能快速启动。
