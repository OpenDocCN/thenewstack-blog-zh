# MongoDB 抓住了 WiredTiger 及其人才，包括 Sleepycat Software 的联合创始人和首席架构师

> 原文：<https://thenewstack.io/mongodb-grabs-wiredtiger-and-its-talent/>

NoSQL 数据库 [MongoDB 收购了数据库存储引擎技术 WiredTiger](https://www.mongodb.com/blog/post/welcome-wiredtiger-mongodb) ，并在此过程中获得了数据库专家 Keith Bostic 和 Michael Cahill 博士的人才。

在创办 [WiredTiger](http://www.wiredtiger.com/) 之前，Cahill 是 Sleepycat 软件公司 Berkeley DB 的架构师，而 Oracle Corp. Bostic 是 Sleepycat 软件公司的联合创始人，该公司于 2006 年被 Oracle 收购。

MongoDB 之前宣布了 WiredTiger，它将其描述为高性能、高吞吐量的存储引擎，作为即将于 1 月份发布的 MongoDB 2.8 的一个选项。相反，它选择了收购。

“Keith 和 Michael 都是数据存储和事务管理领域的杰出人物，他们和整个 WiredTiger 团队与我们完全一致。他们是自由软件运动的真正典范，在他们的团队中，15 年的领域经验使你相对来说是个新人，”MongoDB 首席技术官艾略特·霍洛维茨[在博客](https://www.mongodb.com/blog/post/welcome-wiredtiger-mongodb)中写道。

虽然其他公司已经提供了多种存储引擎，但这是 MongoDB 的首次尝试。WiredTiger 有望大大提高高写入量工作负载的性能。

该公司表示，虽然其现有的存储引擎 MMAPv1 可以很好地处理读取密集型应用程序，但 WiredTiger 可能是写入密集型和读写密集型混合应用程序的更好选择。

与此同时，DynamoDB 云服务和 Cassandra 将写性能置于一致性和读性能之上。

MongoDB 2.8 中新的可插拔存储 API 允许第三方为特定的工作负载、硬件优化或部署架构构建定制的存储引擎。

收购后，WiredTiger 仍将是自己的项目，正在积极开发中，MongoDB 的目标是使其成为 MongoDB 3.0 的默认存储引擎。Horowitz 说，它还将积极支持并继续 MMAPv1 的开发。

它已经延长了寻找漏洞的时间，在整个假期为那些发现关键问题的人提供奖励。

[DB-Engines](http://db-engines.com/en/ranking) 将 MongoDB 列为最受欢迎的 NoSQL 数据库，也是第五大最受欢迎的数据库，仅次于 Oracle、MySQL、Microsoft SQL Server 和 PostgreSQL。
