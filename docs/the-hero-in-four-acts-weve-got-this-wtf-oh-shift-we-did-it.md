# 四幕中的男主角:我们成功了，WTF，哦，Shift，我们成功了

> 原文：<https://thenewstack.io/the-hero-in-four-acts-weve-got-this-wtf-oh-shift-we-did-it/>

欢迎来到[新的堆栈制造商](/podcasts/makers):攀登新的高度，由 [Scalyr](https://www.scalyr.com/) 首席执行官 [Christine Heckart](https://www.linkedin.com/in/christineheckart/) 主持的一系列采访，采访工程师们关于他们面临的问题和他们寻求的解决方案。

这些是关于工程管理以及如何为扩展架构以支持业务需求而做出技术决策的故事。

Pooja Brown 是在线个人风格购物网站的工程副总裁，也是 ENG 的创始成员之一，ENG 是一个由 SaaS 领先公司的副总裁和首席技术官组成的同行网络。Brown 花了大部分时间讨论她在 Stitch Fix 工作之前开发现代基础设施的工作。

[四幕主人公:我们做到了，WTF，哦 Shift，我们做到了](https://thenewstack.simplecast.com/episodes/the-hero-in-four-acts-weve-got-this-wtf-oh-shift-we-did-it)

布朗用四幕讲述了她的故事，讨论了她在巨石柱上的工作以及随着招聘而来的向现代建筑的转移。开发人员是因为他们的 Node.js 技能而被雇佣的。而且它们与。他们曾经非常熟悉的网络开发者。

“前提是:‘拿起这个庞然大物，至少将其构建成水平层，在前端有清晰的 API 分离，用更现代的技术栈来发展前端应用程序，这样你就可以快速创新，从 CI/CD 的角度迭代，所有我们希望在软件中看到的东西，’”Brown 说。“我组建了一个团队，我们在开发软件，我们有一个计划，我们在重新想象这种体验，这让我开始了第二步。”

她说:“这是什么他妈的(WTF)时刻:工具与旧工具不同，技能组合也不同。他们在寻找节点开发者。Net 开发人员。流程在不断变化，人们在多个办公室工作。

Pooja 说:“当你开始考虑这种差异以及它至少会对项目的成功和采用产生的影响时，你就会意识到，就像‘哦，该死’。

然后思维的转变就来了。可以说是支点。

在 StitchFix，应用不一定是传统的，但规模是最大的挑战。

“你如何实现规模化的个性化零售？”Pooja 说。那不是一件容易的事，对吗？它必须对[客户]个性化，但必须大规模运作。"

最后，Pooja 的故事是关于文化和人民的。我们经常听说 DevOps 是一种文化。Pooja 的叙述告诉我们，当应用程序架构发生变化，新方式的现实在日常工作中变得明显时，会发生什么。
