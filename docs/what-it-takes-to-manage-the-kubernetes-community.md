# 管理 Kubernetes 社区需要什么

> 原文：<https://thenewstack.io/what-it-takes-to-manage-the-kubernetes-community/>

[管理 Kubernetes 社区需要什么](https://thenewstack.simplecast.com/episodes/what-it-takes-to-manage-the-kubernetes-community)

有时候，管理软件社区的工作量和编写实际的软件一样多。 [Paris Pittman](https://www.linkedin.com/in/parispittman/) ，谷歌[的开发者关系项目经理，知道这一点，因为她已经帮助管理 Kubernetes 社区一段时间了。她说她喜欢这份工作，因为这个社区的态度很好，对新来的人很有吸引力。](https://cloud.google.com/kubernetes-engine)

“我认为这是我们作为一个项目发展如此之快的原因之一，因为我们有一个热情的社区。令人惊讶的是，我们遇到并处理了这么多问题——bug，而不是个人问题，然后仍然设法对来找我们的人如此热情友好，”皮特曼在今年早些时候 Kubernetes 五周年派对现场录制的采访中说。

然而，情况并非总是如此。因为 Kubernetes 如此迅速地成为如此受欢迎的项目，这有点像在航行中造船。传统企业系统的许多基础并不是在项目开始时构建的。皮特曼说，当 Kubernetes 开始流行时，根本没有时间为这样的工作建立适当的基础。但是，她补充说， [CNCF](https://www.cncf.io/) 已经成为一个地方，如果需要，围绕治理、安全和控制的更加枯燥和传统的工作可以从一开始就建立起来。

“我认为我们开始摆脱很多事情。我想一开始是一片蛮荒的西部。即使在贡献者的经验中，我们仍然在解决很多你认为我们已经解决的事情，但是项目一直在以指数速度前进，所以那些事情中的一些在那个速度中丢失了。像 [GitHub](https://github.com/kubernetes/kubernetes) 管理这样的事情，现在我们正在记录那些策略，你也将看到它与治理一起。皮特曼说:“我们开始研究一些工程师可能认为很无聊的东西，这很好，因为这让我们做好准备，为未来做好准备，这真的很重要。”。

“指导委员会现在正在做很多非常重要的工作，就像搭积木一样。我认为你实际上看到了很多项目，比如说，像 Istio T1，他们首先采用这些积木的方法，而 Kubernetes 起飞得如此之快，以至于很难真正放下那些积木。…我们开始放慢脚步，集中注意力。你也会听到这种说法，特别是从维护者和退伍军人那里，特别是当他们说他们想把东西从核心中取出来，让生态系统围绕它建立并蓬勃发展，”皮特曼说。
