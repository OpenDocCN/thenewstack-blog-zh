# 作为攻击者青睐的目标，API 需要的不仅仅是安全基础

> 原文：<https://thenewstack.io/a-favored-target-for-attackers-apis-need-more-than-the-security-basics/>

Palo Alto Networks 产品副总裁 John Morello 长期以来一直在谈论云原生安全性的基础知识。在本期由新堆栈的创始人兼发行人 [Alex Williams](/author/alex/) 主持的 [The New Stack Makers](/podcasts/makers) 播客中，Morello 讨论了 API 与其说是最薄弱的环节，不如说是它们自身成功的受害者。API 已经获得了更多的关注，这反过来也使它们成为犯罪分子利用的目标。开发 API 的人越来越多，使用 API 的人越来越多，利用 API 的攻击者也越来越多——这使得基础知识比以往任何时候都更加重要，无论是现在还是随着更多的应用程序上线。

安全专业人员需要更好的智能，尤其是在机器人和其他形式的攻击者越来越擅长模仿人类行为的情况下。

“人们所处的不同网络和环境的不同类型攻击模式的应用程序激增，使得人类不可能真正保持领先，因为变化的速度和环境的广度都很大，”Morello 说。“我认为，我们行业中的所有人都有责任帮助客户解决这个问题，在我们如何保护环境以及我们实际上用什么来打扰用户(其他人类操作员)方面提高智能水平，这样我们就不只是向人们提供大量数据，而是向他们提供他们实际上需要采取一些行动的数据，这些行动必须由人类来采取。”

[John Morello，Palo Alto Networks–API 安全基础知识是一回事，但更大的需求是什么？](https://thenewstack.simplecast.com/episodes/john-morello-palo-alto-networks-api-security-basics-are-one-thing-but-what-is-the-greater-need)

因此，适当的安全工具不会用低优先级警报轰炸安全团队，或者在最坏的情况下，不会有太多的误报。

Morello 说:“我经常告诉人们，建立一个安全工具，对一切可能有害的东西发出警报是非常容易的，但最终你会得到太多的噪音，以至于你什么也做不了。“我认为，目前真正无与伦比的挑战是，如何在用于保护这些环境的软件中创建一个学习级别，以便尽可能少地打扰人类操作员，并且只在真正需要人类干预的情况下进行。”

考虑到 API 是入侵者的常见入口点，可行的安全平台还需要非常精通 API。然而，尽管 API 不一定代表最薄弱的安全环节，但 API“往往是攻击者试图更简单地瞄准的东西，因为它被明确设计为可通过编程访问，”Morello 说。

Morello 说，API“主要是为一些人提供一种更有趣的攻击面，让他们去探测并弄清楚除了它打算做的事情之外，它还可以做什么”。

因为微服务和分解的应用程序“确实是你看到的许多组织运行的方向，其中那些单独的组件被明确设计为通过网络一起工作，因此通过网络暴露了更多的内部机制，”Morello 解释道。“它允许攻击者或怀有恶意的人拥有更广泛的终端，他们可以探测更多的东西。”
