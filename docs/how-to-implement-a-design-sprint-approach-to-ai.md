# 如何实现人工智能的设计冲刺方法

> 原文：<https://thenewstack.io/how-to-implement-a-design-sprint-approach-to-ai/>

胭脂红里米

Carmine Rimi 是 Canonical 的人工智能产品经理，负责包括 Kubernetes 在内的全球人工智能/人工智能战略。在 2018 年 5 月加入 Canonical 之前，Carmine 曾在亚马逊网络服务、VMWare、WorkDay、Mirantis 和华为等公司担任云产品管理和工程方面的高级职位。他在整个云堆栈(基础架构、平台和软件)中提供了 web 级的技术和解决方案。除了目前在 Canonical 的职务之外，Carmine 还是 Telstra 的技术顾问，之前是 StackStorm 的董事会成员。Carmine 拥有斯坦福大学的计算机科学学士学位。

人工智能(AI)的商业潜力与日俱增，特别是当该技术超越预测和分析走向真正的自主决策并释放出前所未有的洞察力、知识和创新时。

然而，尽管“大多数高管都知道人工智能有能力改变他们做生意的方式，并可能在 2030 年前为全球经济贡献高达 15.7 万亿美元的价值，”普华永道会计师事务所在最近的一份报告中断言，“但许多商业领袖不知道如何部署人工智能，不只是在这里或那里的试点，而是在整个组织中，在那里它可以创造最大价值。”

对于刚刚开始使用人工智能的组织来说，理解完整的画面是必不可少的——如何从想法到生产部署。这就是为什么追求为期一周的人工智能设计冲刺可以提供一个关于什么是可能的清晰视图，这将有助于设定对时间表、功能和质量的期望。

为期一周的设计冲刺将帮助你的整个团队了解现代人工智能技术和方法。一定要解决真正的业务挑战，并取得切实的成果。这将作为你所有人工智能创新的概念验证。

下面让我们仔细看看初始设计冲刺应该如何进行。每个阶段的具体步骤应该根据您的特定需求进行调整。

**第一天:发现号**。在启动前，召集人工智能团队进行这种结构化的对话，这将创建一周的路径。从长期目标开始，规划出挑战，请你的领域专家分享他们的知识，并选择一个冲刺目标——一个雄心勃勃但可管理的问题，可以在本周解决。规划的挑战应该包括业务需求、预期结果、数据源、部署场景、潜在机会和风险。

第二天:评估。领域专家应该根据发现阶段提供初始反馈。他们将报告数据并提出解决方案建议。他们将根据您的具体业务案例讨论构建项目的战略路线图。在此步骤中，应该突出显示对基础架构的任何更改。

**第三天:设计**。为 sprint 目标设计几个解决方案的原型，并确定一个看起来最合适的候选解决方案。数据工程师应该准备数据。数据科学家应该讨论和选择适当的功能和机器学习算法。机器学习工程师应该在你的原型神经网络上设计、构建和执行初步测试。如果时间允许，开始对数据和神经网络模型进行设计和发现的迭代过程。

**第四天:实施**。完成设计过程，并开始训练和测试您的 AI 模型，直到它达到所需的精度阈值。建立一个管道，将您的模型放入一个合适的环境中，用于测试和来自其他利益相关者的反馈。领域专家应该在评估机器学习预测和将发现的见解付诸行动方面提供指导。

**第五天:运营与反馈**。这可能包括 UI 开发、技术文档以及开发和运营团队之间的实践知识传授，以确保他们能够操作该解决方案。讨论生产部署选项非常重要，例如，黑暗启动或与原型业务应用程序的集成。在这个步骤中，从关键涉众那里获得反馈，包括基线架构和模型。

这个为期五天的项目依靠几个额外的步骤来确保成功的体验:

**第 0 天**是前面提到的预启动。这包括为第一天准备好足够的初步信息、数据、利益相关者和系统。它包括由一小组领域专家进行的初步发现和评估，这些专家有能力交付这个阶段的关键工件。这些工件作为团队参加第一天的输入。

**第六天**是验尸。这是第五天更长的后续行动。这最后一步的关键输出是在冲刺周学到的一系列经验教训。例子:在设计冲刺阶段学到了什么？为更广泛的计划创建的初步计划的含义是什么？向项目发起人推荐和提出什么行动？什么效果好？什么没有？这些经验作为人工智能程序的输入，有助于创造一个良性循环。

对于任何新的主要新技术，有时刚开始是最难的部分。通过遵循设计春天的哲学，组织可以开始他们的人工智能之旅。
