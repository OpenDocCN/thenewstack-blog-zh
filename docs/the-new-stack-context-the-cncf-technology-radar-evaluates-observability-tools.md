# 新的堆栈环境:CNCF 技术雷达评估可观测性工具

> 原文：<https://thenewstack.io/the-new-stack-context-the-cncf-technology-radar-evaluates-observability-tools/>

应用程序和系统的可观察性是上周发布的最新[云本地计算基金会](https://www.cncf.io/)的[技术雷达](https://www.cncf.io/blog/2020/06/12/introducing-the-cncf-technology-radar/)终端用户调查的[焦点](https://www.cncf.io/blog/2020/09/11/cncf-end-user-technology-radar-observability-september-2020/)。因此，在本周的[新堆栈背景](/podcasts/context)中，我们邀请了[CNCF 生态系统副总裁谢丽尔·洪](https://www.oicheryl.com/)来讨论这些发现。为了获得关于可观测性的额外行业观点，我们还邀请了 [Buddy Brewer](https://www.linkedin.com/in/bbrewer/) ，他是 [New Relic](https://newrelic.com/) 的全栈可观测性副总裁。

在云本地空间中，可观察性已经成为一个至关重要的主题，因为考虑到此类应用程序的分布式性质，深入了解特定应用程序或系统组件性能不佳的原因的能力可能会很复杂。CNCF 理工大学的雷达充满了关于管理员如何解决这个问题的见解。

例如，这份报告发现，云原生世界严重依赖开源工具，尽管该领域很少整合——各种各样的工具正在全面使用。另一个发现对该领域的许多人来说并不意外:普罗米修斯度量收集器通常与可视化仪表板一起使用。

“一旦你用仪器测量了某样东西，你就必须能够正确地观察它。否则，就没什么好处，”洪指出。这两个项目都出现在云原生生态系统的早期，并且都非常稳定。阅读一个项目的教程的人会发现经常会提到另一个项目。“所以这两者真的相互反馈，并导致这两者同时获得成功的反馈回路，”她说。

Brewer 肯定了 Hung 的观察，并补充说，不存在单一类型的 Prometheus/Grafana 用户，初创公司和大公司都使用监控堆栈。“我回想起过去几个月与客户的对话，不可避免地会提到 Prometheus。这是他们的生态系统，他们正在利用它。当你与部署普罗米修斯的人交谈时，他们也有 Grafana，”他说。
