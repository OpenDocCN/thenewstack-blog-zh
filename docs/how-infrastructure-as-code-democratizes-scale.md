# 基础设施即代码如何使规模民主化

> 原文：<https://thenewstack.io/how-infrastructure-as-code-democratizes-scale/>

[戴尔科技](https://www.delltechnologies.com/en-us/index.htm)赞助了这次播客。

DevOps 团队依赖于基础设施即代码的工具和技术肯定已经发生了变化，尤其是当它已经开始针对存储进行扩展时。与此同时，始于超大规模云原生应用及其 DevOps 环境的工具、技术和平台实际上已经实现了“规模化”。配置管理和基于状态的声明性范例的概念实际上强化了一个事实，即“云不是一个地方”，而是您如何管理运营以实现自助服务、弹性扩展和敏捷应用程序开发等目标。

在这个[新堆栈制造商](/podcasts/makers)播客中，作为[戴尔技术虚拟播客日](https://www.eventbrite.com/e/virtual-day-of-podcasts-with-dell-technologies-registration-104428791326#)系列的一部分，我们讨论了许多与代码形式的基础架构相关的主题，以及它如何应用于存储，以及这在当今日益以云为中心的世界中意味着什么。

我们的嘉宾有[凯瑟琳·帕格尼尼](https://www.linkedin.com/in/catherinepaganini/en-us)，ku BLR[的市场营销负责人、](https://kublr.com/)帕拉萨尔·科达蒂，产品营销高级顾问、[戴尔科技](https://www.delltechnologies.com/en-us/index.htm)和[英特尔](https://de.linkedin.com/in/patrickohly)[的高级软件工程师 Patrick Ohly](https://www.intel.com/content/www/us/en/homepage.html) 。

[基础设施如何作为规范民主化的尺度](https://thenewstack.simplecast.com/episodes/how-infrastructure-as-code-democratizes-scale)

作为代码的基础设施的一个关键部分，因为它与 DevOps 团队的存储相关，由容器存储接口(CSI)组成。Ohly 说，API 允许为容器编排器创建插件或存储驱动程序。Ohly 说，CSI 已经发展到“存储供应商可以部署一个可以单独提供它的驱动程序，完全独立于 Kubernetes”。"而且它在 Kubernetes 上运行，作为一个真正的添加在它自己的容器上."

“因此，部署现在也容易得多，因为你可以像管理任何其他应用程序一样管理 CSI 驱动程序，但你可以在 Kubernetes 上运行它——它现在真的非常灵活，”Ohly 说。

自助服务模式也越来越重要。Kodati 说，在更高的层面上，业务利益相关者使用基础设施和其他 IT 服务，就像他们从服务目录中进行选择一样。“由于现代平台，依赖服务目录类型的方法变得越来越容易，”他说。

与此同时，在较低的层面上，“自助服务正在获得新的意义，”科达蒂说。“这是一种自我消费，就像开发人员、开发人员、开发人员或部署环境一样，能够自助服务基础架构，而不必经历任何人的手动步骤，如调配存储、按照给定的时间表进行备份、拍摄快照或将数据复制到不同的站点，”Kodati 说。"所有这些任务也在代码中被消耗."

虽然存在采用基础架构作为代码的需求，例如，为了使云原生采用更简单、更灵活，以满足不同组织的需求，但所有利益相关方(包括业务团队)必须首先对其组织采用的内容以及它将如何影响其业务成果有相当深入的了解。

帕格尼尼说:“单靠云原生并不能让你达到目标，这一点必须明确，它的实施方式将最终影响你的灵活性。”。

帕格尼尼说，许多基于云的原生技术也可以“锁定你”。“锁定并不总是可以避免的，但无论何时你进入那个方向，它都应该是一个有意识的决定，而不是因为你不理解其中的含义而不得不做的事情。帕格尼尼说，锁定技术替代方案真正显示了“这些影响有多大，因为它将影响你未来对市场需求的反应能力”。

“如果你正在参与这些讨论，你必须了解全局，所以简而言之，技术太重要太复杂，不能依靠它来用几张幻灯片来概括。帕格尼尼说:“在你做出重大决定之前，你真的需要仔细阅读并在高层次上理解它——你不必成为专家，但你必须在高层次上理解它。”。
