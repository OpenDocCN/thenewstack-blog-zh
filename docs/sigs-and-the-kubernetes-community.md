# Kubernetes SIGs 更新

> 原文：<https://thenewstack.io/sigs-and-the-kubernetes-community/>

[西格和库伯内特社区](https://thenewstack.simplecast.com/episodes/sigs-and-the-kubernetes-community)

在本期来自 [KubeCon 和 CloudNativeCon](https://events.linuxfoundation.org/events/kubecon-cloudnativecon-north-america-2018/) 的新堆栈制造商节目中，TNS 创始人 [Alex Williams](https://thenewstack.io/author/alex/) 与 Kubernetes 特别兴趣小组(SIG)指导委员会成员 [Joe Beda](https://twitter.com/jbeda) 、Kubernetes 社区负责人 [Sarah Novotny](https://twitter.com/sarahnovotny) 和 [Michael Rubin](https://www.linkedin.com/in/michael-rubin-9b32961) (谷歌 GKE/Kubernetes 的高级员工工程师兼经理)讨论了他们的特定 SIG 中发生的事情以及 SIG 在总体上如何反映

特殊兴趣小组是为开发人员和其他社区成员设计的，他们可以定期会面并在 Kubernetes 社区内就他们的兴趣达成一致。Novotny 说，他们为代码库、子项目或相邻项目的一个领域做出贡献，并对愿景和技术决策拥有一些所有权。

“SIGs 被用作一种治理和扩展机制，”Novotny 说。“随着 Kubernetes 变得越来越大，越来越多的人希望做出贡献，为了防止我们陷入单一的基础架构，我们希望分散决策。SIGs 是我们实现这一目标的途径。”

“效果相当不错，”鲁宾说。

[一款 Kubernetes App SIG 深潜](https://thenewstack.simplecast.com/episodes/a-kubernetes-app-sig-deep-dive)

另一个来自 KubeCon 和 CloudNativeCon 的 TNS livestream 讨论了 Kubernetes 社区中的 SIG，采用了不同的方法，深入探讨了 Kubernetes 1.9 中应用 SIG 的进展。在 App SIG 的带领下，微软高级软件工程师 [Michelle Noorali](https://twitter.com/michellenoorali) 和三星 SDS 高级员工工程师 [Matt Farina](https://twitter.com/mattfarina) ，我们还讨论了 SIG 正在解决的最紧迫的问题，以帮助缓解开发人员在 Kubernetes 上开发应用时的痛点。

“我们真的想代表一半的开发人员和一半的运营商。这么多的精力都集中在构建 Kubernetes 上，但一旦你有了 Kubernetes，你就需要写作，这是如何工作的？”法里纳说。“我们希望确保这一点得到体现，并确保开发人员的问题在经常被忽视的另一层得到解决。”

“Kubernetes 中的控制器是工作负载 API 的一部分，我们都希望有一个地方来讨论这些概念和使用这些工具的最佳实践，也希望有一个地方来提供对这些事情的反馈，”Noorali 说。“工作负载团队的开发人员现在将来到 SIG apps 进行交流，并形成反馈回路。”
