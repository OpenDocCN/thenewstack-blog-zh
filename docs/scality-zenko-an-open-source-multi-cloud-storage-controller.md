# Scality Zenko:一个开源的多云存储控制器

> 原文：<https://thenewstack.io/scality-zenko-an-open-source-multi-cloud-storage-controller/>

在一个流行词汇的世界里，有两个短语目前特别流行:随着公司努力应对虚拟化、云部署和容器化的新世界，数字化转型和敏捷性几乎已经成为 it 基础设施的必备要素。

Scality 是利用这一趋势的最新公司之一，它推出了 [Zenko](http://www.zenko.io/) ，这是一种开源的多云存储控制器，使公司能够在不同的云部署之间移动工作负载。

Scality 首席执行官[杰罗姆·勒卡特](https://twitter.com/jlecat)称，无论使用何种云部署，Zenko 都提供相同的用户界面，这为客户提供了更多的灵活性。Zenko 还可以处理结构化和非结构化数据，并为客户提供搜索企业存储系统的方法。

Zenko 处理多个云平台的能力意味着客户不会局限于一个提供商，也不会局限于他们自己的私有系统。“敏捷系统通常被认为意味着迁移到公共云，但这与此无关，我们谈论的是迁移到分布式、高度弹性的系统，”勒卡特说。

Lecat 说，两类客户将从 Zenko 中受益。

一类是具有严格数据存储需求的大型企业。

“我们有一款名为 [RING](http://www.scality.com/products/ring/) 的产品，[管理横向扩展存储的软件]可以处理所有类型的文档:视频、日志、大数据分析。它唯一没有做的事情就是抓图/视频编辑。这是我们卖给 15 大银行中的 6 家的东西。有充分理由部署自己的基础设施的公司。使用 RING 的公司与使用传统存储的公司有很大不同，”他说。

另一类用户是希望在其私有云与公共云服务之间工作的客户。“你不会想成为某个供应商的一员，你会被那个供应商挟持。我看到很多人选择亚马逊作为唯一的供应商，但当亚马逊提高价格时会发生什么？”

然而，虽然 Lecat 质疑基于亚马逊的方法是否明智，但他很乐意使用该公司的软件。Zenko 基于亚马逊[简单存储服务(S3) API](https://aws.amazon.com/s3/) 的实现。

“这并不矛盾，”他说，“关键是亚马逊如何对待它的 API，到目前为止，它还是一个封闭的产品。”

尽管 Zenko 是一款开源产品，但 Scality 并没有不愿意走专有路线；例如，RING 基于专有软件。但是，他补充说，Zenko 是完全开源的，“这是软件的未来。”

Lecat 表示，Zenko 是一个“伞状产品”，它由几个元素组成:S3 API、RING、Backbeat 数据工作流(尚未发布，但将于秋季发布)和克鲁索乐队元数据搜索(也将于今年晚些时候发布)。

虽然它似乎与多云控制器如[戴尔 Boomi](https://boomi.com/) 有一些相似之处，但 Lecat 表示，这些控制器并不专注于存储。他说最接近的竞争对手是 EMC，但他说 Zenko 在其更智能的软件上得分。“EMC 总是说它是软件定义的，但最终它总是与硬件有关，”他说。

对于 Lecat 来说，Zenko 的推出是一场存储革命，堪比特斯拉对汽车的变革——从传统的内燃机到有 17 个移动部件的内燃机，升级由软件驱动。

“Zenko 允许客户部署已经为云编写的应用程序，而不必决定编写哪个云。这让他们可以自由改变主意。这都是政策驱动的:写一个政策比改变供应商容易得多，”他说。
