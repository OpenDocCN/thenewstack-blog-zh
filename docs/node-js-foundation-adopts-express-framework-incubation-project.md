# IBM 将 Express 框架留给了 Node.js 基金会

> 原文：<https://thenewstack.io/node-js-foundation-adopts-express-framework-incubation-project/>

Node.js 基金会已经将 [Express](http://expressjs.com/) Node.js 框架收归旗下。

Express 将是基金会新的孵化项目。去年九月[收购了](http://www-03.ibm.com/press/us/en/pressrelease/47577.wss) Express 维护者 StrongLoop 的 IBM 正在贡献代码。

“表达将作为它自己的独立实体发挥作用；类似于 Node.js 基金会如何通过开放治理支持 Node.js，技术指导委员会、导师和贡献者将有效地支持该框架，”Node.js 基金会的社区经理 Mikeal Rogers 在一封电子邮件中写道。

Express 是一个用于构建 web 应用程序和框架的工具包。它通常与 Node 一起使用。可以通过 **NPM** (节点包管理器)安装。它通常与 MongoDB 数据库和 AngularJS 前端框架一起使用。Express 是最流行的节点框架。在过去的两年里，它已经被下载了 5300 万次，仅上个月就被下载了 5 次。

据罗杰斯说，Express 的发布节奏和数量将由其贡献者决定。“传统上，这些用户和贡献者更喜欢稳定性，而不是新功能，”他写道。

允许基金会监督 Express 的部分原因是为了建立一个多样化的贡献者基础，考虑到该框架的受欢迎程度，这一点很重要。

该软件用于 PayPal 的开源软件 kraken.js 中，这是 Node 的企业扩展。它还为 [Sails.js](http://sailsjs.org/) MVC(模型-视图-控制器)Web 框架和[Loopback](http://loopback.io/)(node . js API 框架)提供了基础功能。

基金会的孵化过程将为 Express 提供一个开放的治理和贡献协议，这应该会吸引依赖 Express 的组织，这些组织可能会担心一个供应商主导 Express 的未来方向。

“作为孵化器的一部分，导师将与 Express 的贡献者合作，组成一个‘Express 技术委员会’，其成员完全由 Express 的贡献者和相关依赖者组成，”罗杰斯说。“技术委员会将决定谁是主席，如果他们觉得有必要话。当 Express 从孵化中毕业时，他们也可以选举一名代表加入 Node.js 基金会技术指导委员会。”

Express 最初由 TJ Holowaychuk 开发，他最近为 Lambda 无服务器计算服务创建了 T2 Apex 框架。像 Node 本身一样， [Express framework](https://github.com/strongloop/express/?_ga=1.156918590.486351045.1455053248) 是最小的，它提供了一个干净的接口来包含额外的特性作为插件。

[Node.js Foundation](https://nodejs.org/en/foundation/) 是[在一年前](https://thenewstack.io/foundation-node-js-will-bring-back-fork/)成立的，目的是作为一个独立的治理机构来管理软件，为 JavaScript 提供一个[服务器端运行时](https://thenewstack.io/node-js-long-term-support-ready-enterprise/)。Node.js 基金会孵化器计划下的项目从基金会的技术指导委员会和相关工作组获得援助和治理指导。

IBM 是新堆栈的赞助商。
