# “蜜月”结束后与库伯内特一起生活

> 原文：<https://thenewstack.io/kubernetes-after-the-honeymoon-ends/>

西班牙巴伦西亚——在谷歌将开源[容器](https://thenewstack.io/category/containers/)orchestrator[Kubernetes](https://thenewstack.io/category/kubernetes/)发布到一个不为人知的世界近七年后，[全球有 560 万开发者使用它](https://www.cncf.io/wp-content/uploads/2022/02/CNCF-AR_FINAL-edits-15.2.21.pdf)。

但是这个数字，来自最新的[云本地计算基金会(CNCF)](https://cncf.io/?utm_content=inline-mention) 年度调查，掩盖了很多挫折。Kubernetes (K8s)可以让采用它的组织的生活变得更容易——在它使它变得更加困难之后。随着规模的扩大，它可以创造一个无休止的胜利和挑战的节奏。

换句话说:很复杂。

上周在西班牙巴伦西亚举行的欧盟 [KubeCon + CloudNativeCon](https://www.cncf.io/kubecon-cloudnativecon-events/?utm_content=inline-mention) 大会上，三位专家——[Saad Malik](https://www.linkedin.com/in/saad-a-malik/)，Spectro Cloud 的联合创始人兼首席技术官； [Bailey Hayes](https://www.linkedin.com/in/baileyhayes/) ，SingleStore 首席软件工程师；VMware 的员工工程师 Fabrizio Pandini 和我一起参加了一个直播活动。

其结果是一个关系状态报告/治疗会议，内容是 Kubernetes 可能比云原生世界第一次爱上它时看起来更难相处。

## 每季度更新的挑战

对话涉及了许多领域:Kubernetes 季度发布与许多企业运行的 K8s 的过时版本之间的矛盾；多集群与巨型集群的增长趋势，以及 WebAssembly 的前景。

Kubernetes 的一个核心挑战是:组织经常运行过时的 it 版本。马利克说，他见过监管严格的行业(医疗保健、石油和天然气等)中的组织。)运行在两年或者更久没有更新的 K8s 上。

Pandini 说，让客户更容易更新 K8s 为软件提供商提供了“一个很好的机会”:“我们必须为用户提供简单的按钮，我们必须满足他们的需求，但要把他们带到我们的世界，这更具可持续性。”

他在随后的活动中说，Kubernetes 社区正在整合，超越早期的创新阶段，进入更加关注稳定性和安全性的阶段。与早期工具相比，现在创新是以“一种更可控的方式”进行的。

“社区对此做出了反应，”潘迪尼说。“有很多工作没有以前那么有魅力了，以前每个月都有令人兴奋的消息。”

一名观众评论说，Kubernetes 可能发布更新过于频繁。“问题不在于升级 Kubernetes，”他说，而是必须每三个月更新一次所有核心组件，如 Prometheus 和 ingress 控制器。潘迪尼和马利克说，增加自动化的使用可以帮助解决这个问题。

[WebAssembly，或 WASM](https://thenewstack.io/how-webassembly-could-streamline-cloud-native-computing/)——“构建一次，在任何地方运行”的技术——提供了一种“赶上 Kubernetes”的有前途的方法，Hayes 说。但 WASM 标准仍在创建中，这将允许它在浏览器之外的服务器环境(WASI)中运行，她说。

作为进步点，她指出了更新的平台，如 WASM 云和 Spin，允许用户使用 WASM 运行生产工作负载，以及 [Krustlet](https://thenewstack.io/krustlet-brings-webassembly-to-kubernetes-with-a-rust-based-kubelet/) 技术，允许用户使用 OCI 容器并行运行 WASM。

“对于规模，我认为这将是一个很好的答案，但我不认为我们必须有，”海斯说。“还是很新的。”
