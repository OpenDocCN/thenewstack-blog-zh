# SecOps 每天都在监控

> 原文：<https://thenewstack.io/secops-spends-day-monitoring/>

开发人员、安全和运营:DevSecOps。该术语的运营部分通常指 IT 运营。然而，今天的范围缩小到了在安全运营中心(SOC)和网络事件响应小组(CIRTs)工作的 secop。 [Cyentia Institute 的](https://www.cyentia.com/) [对 160 名安全分析师的调查](https://go.respond-software.com/voice-of-the-analyst/)显示，他们面临着开发人员和 IT 运营团队面临的一些同样的挑战。他们花在监控上的时间比任何其他活动都多，但他们更愿意解决问题和“搜寻”新的威胁。SecOps 不喜欢报告或称之为转移操作的东西——变更控制的实际细节，并确保团队不会筋疲力尽。鉴于信息安全专业人员的短缺，令人担忧的是，只有 45%的受访者表示他们的工作经历符合他们的期望。

Cyentia 建议，自动化可以减少花在监控上的时间，让分析师专注于入侵防御和威胁情报。这很可能是真的，尤其是如果监控是在托管安全服务提供商(MSSP)处大规模进行的话。然而，像 DevOps 一样，SecOps 也在寻求让他们的仪表板更具可操作性，使用人工智能来筛选不重要的噪音。
