# 软件创业瞄准集群计算壁垒

> 原文：<https://thenewstack.io/software-startup-aims-at-cluster-computing-barriers/>

写得不好的代码通常会导致硬件资源利用率低下。硬件制造商正在积极尝试解决这个问题，但软件制造商也在提供工具，以利用地理上分布的云环境中可用的最佳计算资源。

植根于高性能和量子计算的 Agnostiq 升级了一个名为[共价](https://github.com/AgnostiqHQ/covalent)的工具，作为一种简单的方式，将代码执行部分卸载到云中最好的硬件资源。一个程序员只需要写几行代码，就可以释放[亚马逊网络服务](https://aws.amazon.com/?utm_content=inline-mention)中的计算资源，用于科学计算和模拟。

该工具可帮助 IT 组织、研究人员和科学家扩大在内部和公共云中部署的混合云的范围。该工具旨在将内部部署和 AWS 基础设施视为一个系统，共价扩展了硬件可用性，具体取决于所解决问题的规模。

## 超越 CPU

Agnostiq 的工具符合计算从 CPU 向图形和人工智能处理器等专业加速器的更广泛转变，这些加速器正在承担机器学习和科学任务的重任。在许多情况下，编码人员仍然需要输入特定的硬件来执行代码，但是英伟达和英特尔公司的工具包现在已经可以自动完成这个过程。

Agnostiq 量子软件负责人威尔·坎宁安(Will Cunningham )说，共价位于并行编程工具包(如英伟达的 [CUDA](https://thenewstack.io/cuda-12-harnesses-a-nvidias-speedier-gpu-architecture/) )之上。

Cunningham 说:“用户能够在他们的环境中为特定的任务管理所有不同的软件依赖性。

例如，一个软件栈可能部署在多个云中，而共价将跨多个集群编排作业、管理软件修订、提交脚本和管理数据。

“如果没有这样一个能够跨多个本地集群的工具，就很难理解——最新版本的代码在哪里？Cunningham 说:“如果数据变得支离破碎，它在哪里，我的计算在哪里最容易获得。

## 该工具

共价是一个基于 Python 的工具，几行代码将 AWS 的 Lambda、ECS 和 EC2 计算资源添加到工具包中。当前的实现侧重于输入特定的代码集以便在 Amazon 集群上执行，并等待输出到达，然后可以将输出放入堆栈。

该工具包括一个用于构建工作流的软件开发工具包组件和一个涉及后端调度服务和用户界面的服务器组件。

“在某些节点可用于工作流工具的情况下，该服务器本身可以在 HPC 集群上运行，这在当今更为常见。或者它可以运行在云中，如果用户使用混合云配置，这可能更合适，”Cunningham 说。

内部服务器可能没有像 Nvidia 的 GPU 那样的大量硬件资源，而这些资源是机器学习等应用程序所需的。机器学习通常需要大量内存和存储，靠近服务器内部的 GPU 和 CPU，这在 AWS 上很容易获得。

一个典型的到云的卸载将涉及发送输入到远程云服务器，并等待接收输出，然后输出可以被合并到软件栈中。代码执行的每一秒都很重要，在具有内部和公共云基础架构的混合云中进行实时模拟被视为云计算中最大的挑战之一。障碍是互连，这对于实时数据交换来说不是最佳的。

该工具包不包括流技术，该技术在硬件上执行时增加了实时处理和分析数据的元素。
