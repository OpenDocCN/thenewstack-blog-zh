# 上下文:“如果数据为王，那么分销就是女王”

> 原文：<https://thenewstack.io/context-if-data-is-king-then-distribution-is-the-queen/>

[‘如果数据是王道，那么分配就是女王’](https://thenewstack.simplecast.com/episodes/if-data-is-king-then-distribution-is-the-queen)

欢迎来到[新的堆栈环境](https://thenewstack.io/podcasts/context)，在这个播客中，我们将讨论来自云原生社区的最新新闻和观点。本周，我们采访了体育视频流媒体网站 [DAZN](https://watch.dazn.com/en-US/sports/) (Da-Zone)的首席架构师 [Luca Mezzalira](https://lucamezzalira.com/) ，讨论了微前端的概念和发展，或者相当于扩展前端开发的微服务方法。

Mezzalira 曾[撰写并发表演讲](https://medium.com/@lucamezzalira/micro-frontends-resources-53b1ec7d512a)讲述他设计公司横向扩展前端流媒体应用的旅程，他称之为他所创建的最具挑战性的架构之一。DAZN 在试图快速、大规模扩展时，遇到了许多拥有庞大、单一后端的公司所面临的相同问题。我们将听到他的团队如何开发 DAZN 微前端。如果内容是王道，那么发行就是女王，他在这个生动的采访中提醒我们。

随后在节目中，我们将讨论 Capital One 最近因“防火墙错误配置”导致的[数据泄露事件，该事件允许黑客访问该公司 AWS S3 存储桶中超过 1 亿名客户的数据。正如普通读者所知，在过去的几年里，新的堆栈已经发布了几十篇关于云安全的危险的帖子。它们都有相同的基本信息:即使你使用第三方云服务，安全仍然是你的责任。](https://thenewstack.io/capital-ones-cloud-misconfiguration-woes-have-been-an-industry-wide-fear/)

没有人会因为没有实现多因素身份验证、密码管理器，甚至没有进行基本的修补而被解雇。但是一旦出现漏洞，人们就会被解雇。“行业需要有一个基本的网络能力构建模块列表。如果你没有做到这一点，你就应该被解雇，不管是否存在违规行为。”

我们还讨论了网站上的一些其他故事，包括[可能回归单核](/why-the-unikernel-might-outpace-generic-linux-for-cloud-native-ops/)、[可能在编程中过度使用循环](https://thenewstack.io/4-reasons-not-to-use-programming-loops-and-a-few-ways-to-avoid-them/)，以及[对 Cloud Foundry 执行董事 Abby Kearns](https://thenewstack.io/cloud-foundry-sees-exciting-challenges-in-china/) 的采访，内容是关于开源成熟和中国令人兴奋的可能性。

TNS 的编辑和营销总监 Libby Clark 在 New Stack 的创始人兼发行人 Alex Williams 和 New Stack 的执行主编 Joab Jackson 的帮助下主持了这个播客。

Cloud Foundry Foundation、Cloud Native Computing Foundation 和 KubeCon+CloudNativeCon 是新堆栈的赞助商。
