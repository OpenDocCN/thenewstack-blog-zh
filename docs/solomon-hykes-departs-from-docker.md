# 所罗门·海克斯离开码头

> 原文：<https://thenewstack.io/solomon-hykes-departs-from-docker/>

所罗门·海克斯走向成功的道路是曲折的。他打造 Docker 的旅程始于 10 年前巴黎 dotCloud 的创立。这条路今天随着他从公司辞职而终结。在此期间，Hykes 的领导将 dotCloud 从一家 PaaS 公司转变为 Linux 容器革命的中心，于 2013 年发布了 Docker 开源项目。

一路走来，dotCloud 变成了 Docker，containers 扩展到了 Linux 之外，Hykes 和公司设法筹集了近 2.5 亿美元的风险投资。有了这么多的资金，难怪 Hykes 可能已经厌倦了 Docker 的日常运作。在他的辞职博客中，Hykes 表示他将专注于成为 Docker 的董事会成员和股东，同时也帮助他多年来投资的许多企业家。

Hykes 跟随其他知名创始人的道路，他们不得不离开他们的中型风险投资公司，因为他们努力转向更大规模的运营。Luke Kanies 在 2016 年将 Puppet 的控制权移交给了一个更具商业风格的团队，同时也将注意力转移到了投资上。

虽然 Hykes 使 Docker 因其转向企业容器化软件而声名大噪，但该公司的增长和企业产品的扩展也带来了一些成长的烦恼。特别是，Docker 一直在努力控制容器编排平台的叙事，Hykes 认为该公司早期主导这个领域的时机已经成熟。Docker 坚持认为，Hykes 的离开与这些斗争无关，事实上，将 Swarm 和 Kubernetes 的合并归功于 Hykes。

“不，这和他的决定绝对没有关系。Docker 发言人在回应新堆栈时写道:“Solomon 是[云本地计算基金会](https://www.cncf.io/)技术监督委员会的成员，在 Docker 容器平台中 Kubernetes 与 Swarm 的集成中发挥了重要作用。Hykes 本人表示，他将承担为公司寻找新 CTO 的工作，这是他在 2017 年底辞去该职务后承担的工作。

早在 2016 年，在该公司的 Docker Con 上，Hykes 向人群解释了基于容器的基础设施的大规模编排平台的重要性。“今天，流程编排与码头之前的集装箱化处于相同的阶段。你要么需要一群专家来构建它，要么把自己锁在一个单一的平台上，这将大大减少你对供应商的选择。三年前，我们让集装箱化成为主流，让非专业人士也能使用，而不会受到限制。我们认为是时候对编排做同样的事情了。这是行业向前发展的必要一步，作为集装箱化市场的领导者，我们有责任引领这一变革。”

不幸的是，在接下来的一年中，Kubernetes 项目成为了编排层之争中事实上的赢家。2017 年年中，Docker 开始改组，其首席执行官 Ben Golub 被前 SAP 高管兼 Concur 前首席执行官 Steve Singh 取代。与此同时，Docker 产品执行副总裁 Marc Verstaen 也离职。Docker 今年早些时候接受了 Kubernetes，现在提供自己的商业 Kubernetes 发行版。

2017 年 11 月[，Hykes 从 CTO 的角色转为首席架构师。Hykes 的](https://thenewstack.io/solomon-hykes-changes-roles-docker/)[推文](https://thenewstack.io/container-format-dispute-twitter-shows-disparities-docker-community/)和[新闻发布会](https://thenewstack.io/solomon-hykes-leader-open-source-world-needs/)引发了一些小争议，但他一直是容器和开源软件的忠实拥护者。

Hykes 在他的辞职博客中写道，“今天，在我 34 岁的时候，在我们的首席执行官传奇人物 Steve Singh 的领导下，Docker 已经悄悄地转变为一个收入爆炸性增长的企业业务和数百万的开发人员社区。我们的策略很简单:世界上的每一家大型企业都在准备将其应用程序和基础设施一起迁移到云中。他们需要一种解决方案来可靠、安全地做到这一点，不需要昂贵的代码或流程更改，也不需要将自己局限于单一操作系统或云。目前，满足这些要求的唯一解决方案是 Docker 企业版。这使得 Docker 处于巨大增长机会的中心。为了利用这个机会，我们需要史蒂夫身边的首席技术官有几十年为世界上最大的公司运送和支持软件的经验。因此，我现在有了一个新的角色:帮助寻找理想的 CTO，偶尔提供一些建议，并在团队继续建立庞大的业务时，离开他们。作为股东，我非常乐意接受这个角色。”

CNCF 是新堆栈的赞助商。
