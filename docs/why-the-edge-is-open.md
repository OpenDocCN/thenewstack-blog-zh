多亏了开源社区，将工作负载转移到边缘正迅速成为现实。新的延迟敏感型工作负载，如智能汽车、增强现实和其他实时驱动的应用，正在推动这种边缘采用。

边缘包括部署到各个位置的计算和存储资源，以最大限度地减少现场连接设备(如物联网和移动设备)的延迟。对于开发人员来说，这些边缘位置提供了支持这些现场部署设备的资源。

我们看到开源技术在整个边缘堆栈中引领潮流:从物理基础设施(Open19)、计算机处理器(RISC-V)，当然还有整个软件堆栈(Akraino、Kubernetes、OPNFV，等等)。

过去，公众无法在边缘位置托管工作负载。初创公司和不那么根深蒂固的公司一直无法将应用部署到由电信运营商拥有和运营的边缘位置。分布式边缘基础设施提供商，如 Akamai，是为单一目的设计的，不对其他一般用例开放。

然而，最近手机信号发射塔所有权的变化、公民宽带无线电频谱(CBRS)的可用性以及开放技术的广泛采用改变了格局，使边缘真正为所有人所用。

## 在塔楼见！

多年来，电信运营商通过出售和回租的方式剥离了手机信号塔和区域数据中心等实体资产。这有助于创建像 Crown Castle、SBA Communication 和 American Tower 这样的大型运营商，所有这些运营商也都投资了光纤和“小蜂窝”资产。这些发射塔运营商以及 Brookfield 等房地产投资信托基金(REITs)正在寻求带来额外的租赁收入，开始将这些手机发射塔位置提供给边缘运营商。

Packet 正在数十个区域和边缘位置部署小型数据中心，为具有早期“边缘”用例(如 Sprint 和 Hatch)的客户提供服务。与此同时，他们正在以完全自动化的公共云方式提供这些位置的基础架构和网络服务。需要在网络边缘进行一些计算？只需进行一次 API 调用，您就可以在几分钟内获得在数十个位置运行边缘应用所需的裸机计算。

## 开放不再仅仅是软件

Open19 和开放计算项目(OCP)等项目在数据中心基础设施的物理方面的标准化方面一直处于领先地位，使 Packet 和像 VaporIO 这样的公司能够以合理的运营成本开放这些边缘位置。这些项目将效率技术从超大规模云环境(想想电源母线)带到了可以少量部署的标准 19 英寸机架。Open19 在优化这种“子规模”部署方面尤其积极，从服务器上移除电缆以大幅降低安装成本。

芯片也在为边缘生活进行创新:高核心数 Arm 和 AMD EPYC 处理器是这些边缘位置的理想选择。开源 RISC-V 体系结构正在迅速发展，允许公司为他们特定的硬件或应用需求定制芯片。这些技术有可能降低成本并增加允许这些边缘位置对公众可用的选择。

## 你现在能听到我吗？

如果没有配套的无线或最后一英里连接，真正的边缘访问就没有用。除了传统运营商，我们现在还有公民宽带无线服务(CBRS)。这种新改变用途的公共频谱以前由美国海军使用，并由 Federated Wireless 等公司商业化，允许开发人员在局部区域部署可靠的无线服务。早期的用例包括体育场、机场和工业区。

CBRS 不仅有潜力降低延迟，还能为开发人员提供广泛的无线技术，使下一代边缘应用蓬勃发展。只需几个额外的 API 调用，边缘的计算基础设施就可以连接到 CBRS 天线，以及边缘应用所需的任何额外存储或定制处理器(如 GPU)。

## 现在都在一起

在软件层，云计算和集群项目中的边缘感知开源技术正在蓬勃发展。除了在云本地计算基金会(CNCF)的工作，Linux 基金会已经创建了 LF Edge 来容纳专注于 Edge 的软件项目，OpenStack 基金会已经发布了 Airship 和 StarlingX(两者都旨在广泛的分布式计算)。这些边缘感知软件堆栈支持边缘感知应用的开发。

这种对堆栈所有层的开放方法既迷人又重要。随着边缘技术的发展，我们需要继续投资开放方法，从物理基础设施、无线连接和芯片到软件堆栈。这将确保边缘创新加速，新的想法和商业模式能够生根发芽。

云计算原生计算基金会和 Linux 基金会是新堆栈的赞助商。
